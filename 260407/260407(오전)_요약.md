## 2. Tensor와 NumPy 기초

### 2.1 Tensor의 정의
**Tensor (텐서)** : 다차원 배열. 차원 수에 따른 명칭:
- 0D : 스칼라
- 1D : 벡터
- 2D : 행렬
- 3D+ : 고차 텐서 (예: 이미지 = H×W×C, 배치 이미지 = N×H×W×C)

### 2.2 배열 속성
| 속성 | 의미 | 예 |
|------|------|-----|
| `shape` | 각 차원 크기 | `(2, 3)` = 2×3 행렬 |
| `dtype` | 데이터 타입 | `int32`, `float32`, `bool` |

### 2.3 NumPy 기본 연산
```python
# 배열 생성
np.array([1,2,3])          # shape (3,)
np.zeros((2,3))            # 0으로 채운 2×3
np.ones((2,3))             # 1로 채운 2×3

# Shape 변환
a.reshape(3, 2)

# 축 기반 연산
a.sum(axis=0)              # 행 방향 합
a.mean(axis=1)             # 열 방향 평균

# 슬라이싱
a[0], a[0:2], a[-1]
```

### 2.4 Broadcasting (브로드캐스팅)
서로 다른 shape의 배열끼리 연산할 때 **자동으로 작은 쪽을 확장**.
```
A : (2, 3)
B : (1, 3)
A + B → (2, 3)   (B가 2행으로 자동 복제)
```
이미지(64×64×3) + RGB 평균 벡터(3,) 같은 연산에서 필수.

## 3. PyTorch 기초

### 3.1 PyTorch 학습의 3단계 구조
1. **데이터셋 준비** (Dataset / DataLoader)
2. **모델 정의** (`nn.Module` 상속 + `forward`)
3. **학습 루프** (Forward → Loss → Backward → Optimizer.step)

### 3.2 표준 학습 루프
```python
class Model(nn.Module):
    def __init__(self, in_size, hidden, out_size):
        super().__init__()
        self.fc1 = nn.Linear(in_size, hidden)
        self.fc2 = nn.Linear(hidden, out_size)
    def forward(self, x):
        x = torch.relu(self.fc1(x))
        return self.fc2(x)

for epoch in range(num_epochs):
    for data, label in dataloader:
        output = model(data)
        loss = loss_fn(output, label)

        optimizer.zero_grad()   # ① 그래디언트 초기화
        loss.backward()         # ② 역전파
        optimizer.step()        # ③ 가중치 업데이트
```

## 4. Perceptron (단일 퍼셉트론)

### 4.1 구조
```
입력 x → Σ(wᵢ·xᵢ) + b → 활성화함수 → 출력 ŷ
```
- 이진 분류기
- **선형 결정 경계** 만 가능

### 4.2 Delta Rule (학습식)
```
Δwᵢ = -η · (y - ŷ) · xᵢ = -η · error · input
```
- η : Learning Rate (보통 0.01~0.1)
- y : 정답 / ŷ : 모델 예측

### 4.3 한계 : XOR 문제
단일 직선으로 분리 불가능한 XOR은 단층 퍼셉트론으로 해결 불가 (Minsky-Papert, 1969).

## 5. MLP (Multi-Layer Perceptron)

### 5.1 구조
```
Input → Hidden (1개 이상, 비선형 활성화) → Output
```
- 은닉층이 비선형 특징 추출
- 출력층이 최종 분류/회귀

### 5.2 비선형 활성화 함수
- **ReLU** : `max(0, x)` — 기울기 소실 완화, 현대 표준(은닉층)
- **Sigmoid** : `1/(1+e⁻ˣ)` — 이진 분류 출력층
- **Tanh** : `-1 ~ 1` 범위 — 과거 은닉층
- **Softmax** : 다중 분류 출력층

### 5.3 XOR 해결
은닉층(2개 이상 뉴런) + 비선형 활성화 ⇒ 곡선 경계 형성 ⇒ XOR 해결.

## 6. 주요 하이퍼파라미터

| 파라미터 | 역할 | 일반 값 |
|----------|------|---------|
| Learning Rate (η) | 스텝 크기 | 0.001 ~ 0.1 |
| Batch Size | 한 번에 처리하는 샘플 수 | 16, 32, 64, 128 |
| Epoch | 전체 데이터 반복 횟수 | 10 ~ 100+ |
| 활성화 함수 | 비선형성 부여 | ReLU / Sigmoid / Tanh |

### 6.1 Learning Rate의 영향
- η 너무 크면 : 발산, 최적점 건너뜀
- η 너무 작으면 : 학습 느림, 지역 최솟값에 갇힐 수 있음
- 실험 : loss가 크다가 갑자기 작아지면 η가 커서 불안정하다는 신호

## 7. 핵심 정리

1. Tensor = 다차원 배열, `shape` 이해가 디버깅의 핵심.
2. NumPy의 Broadcasting 으로 shape가 다른 배열도 자동 연산.
3. PyTorch = Dataset + Model(`nn.Module` + `forward`) + Train Loop.
4. 표준 삼단 : `zero_grad` → `backward` → `step`.
5. Perceptron은 선형 분리 가능 문제만 해결 가능.
6. XOR 은 단층 퍼셉트론 실패, **은닉층 + 비선형 활성화** 로 해결.
7. Learning Rate가 수렴 속도·안정성의 핵심 튜닝 포인트.
8. ReLU(은닉) + Sigmoid/Softmax(출력)가 현대 표준.
9. `Delta Rule : Δw = -η·(y-ŷ)·x`.
10. End-to-End 학습 : 중간 특징을 네트워크가 자동 학습.
11. Batch size는 메모리와 안정성의 trade-off.
12. Epoch당 전체 데이터셋 1회 처리, 일반적으로 수십~수백 반복.
13. Dataset → DataLoader로 배치·셔플링·병렬 로딩 제공.
14. `torch.relu` 는 inplace 형태 `F.relu_` 도 있음.
15. `nn.Module`의 파라미터는 `parameters()` 로 접근, optimizer에 전달.
