# 260409 오전 강의 요약 — Transformer·Attention 실습

## 1. Attention의 등장 배경

### 1.1 Seq2Seq의 Information Bottleneck
RNN 기반 Encoder-Decoder 에서, 긴 문장을 **단일 Context 벡터** 하나로 압축하면 초기 단어의 정보가 뒤로 갈수록 희미해진다. → 장거리 의존성 소실.

### 1.2 Attention의 해결 아이디어
각 출력 토큰을 생성할 때 **입력의 모든 단어를 다시 참고** 하되, **어느 단어에 집중할지** 를 스코어로 결정한다.

## 2. Scaled Dot-Product Attention

### 2.1 공식
```
Attention(Q, K, V) = softmax( Q · Kᵀ / √d_k ) · V
```

### 2.2 QKV 의미
- **Query (Q)** : 현재 위치가 "찾는 것"
- **Key (K)** : 각 위치의 "표지 / 인덱스"
- **Value (V)** : 각 위치가 담은 "실제 정보"

### 2.3 3단계 연산
```
① score = Q · Kᵀ            (유사도)
② α     = softmax(score/√d_k) (확률 분포)
③ out   = Σ αᵢ · Vᵢ           (가중합)
```

### 2.4 √d_k 로 스케일링하는 이유
차원 d_k가 클수록 `Q·Kᵀ` 의 분산이 커져 softmax 입력값이 거대해지고 분포가 saturate → 그래디언트 소실. √d_k 로 나누어 **softmax 분포를 안정화**.

## 3. Self-Attention 예시

입력 `[t, o, (space), b, e]` 에 대해 첫 토큰 `t` 의 Self-Attention:
```
Q = W_Q · x_t
K = [W_K·x_t,  W_K·x_o,  W_K·x_b,  W_K·x_e]
V = [W_V·x_t,  W_V·x_o,  W_V·x_b,  W_V·x_e]

scores        = [2.4, 4.0, 1.2, 3.0]
attention α   = softmax(scores/√d) ≈ [0.13, 0.43, 0.10, 0.34]
output        = 0.13·V_t + 0.43·V_o + 0.10·V_b + 0.34·V_e
```
→ 첫 토큰은 주로 두 번째(o), 네 번째(e) 토큰을 참고.

## 4. Transformer 아키텍처

### 4.1 Encoder
- Multi-Head **Self-Attention**
- Feed-Forward Network (위치별 독립)
- Layer Normalization + Residual Connection

### 4.2 Decoder
- **Masked Self-Attention** : 미래 토큰 차단 (auto-regressive)
- **Cross-Attention** : 디코더가 인코더 출력을 참고
- Feed-Forward Network

### 4.3 3가지 Attention 유형
| 위치 | Q 출처 | K, V 출처 | 역할 |
|------|--------|-----------|------|
| Encoder Self-Attn | 입력 | 입력 | 입력 문장 내부 관계 |
| Decoder Masked Self-Attn | 출력(과거) | 출력(과거) | 이전 토큰만 참고 |
| Cross-Attn | 디코더 | 인코더 | 번역·요약 등에서 입력 참조 |

## 5. Multi-Head Attention

### 5.1 공식
```
head_i   = Attention(Q·W_iQ, K·W_iK, V·W_iV)
MultiHead = Concat(head₁, …, head_h) · W^O
```

### 5.2 의미
- 여러 Attention 을 **병렬로 수행**
- 각 head가 다른 관계(의미, 문법, 구조)를 학습
- 결과를 연결(concat)한 뒤 선형 투영

## 6. Positional Encoding

### 6.1 필요성
Self-Attention은 **permutation invariant** → 순서 정보 없음. "duck bite man" = "man bite duck" 문제 발생.

### 6.2 원 논문의 sin/cos 인코딩
```
PE(pos, 2i)   = sin( pos / 10000^(2i / d_model) )
PE(pos, 2i+1) = cos( pos / 10000^(2i / d_model) )
```

### 6.3 sin/cos 사용의 이점
- **일정 주기성** → 학습 시 보지 못한 긴 시퀀스도 외삽 가능
- **상대 거리 표현** : PE(pos+k) = PE(pos) 의 선형 결합으로 표현 가능
- **파라미터 추가 없음**
- 차원마다 다른 주파수 → 다양한 스케일의 위치 패턴 포착

### 6.4 최종 입력
```
최종 입력 = Token Embedding + Positional Encoding   (element-wise add)
```

## 7. RNN vs CNN vs Self-Attention 복잡도

| 항목 | Self-Attention | RNN | CNN |
|------|----------------|-----|-----|
| per-layer 복잡도 | O(N²·D) | O(N·D) | O(K·N·D) |
| 순차 연산 | O(1) | O(N) | O(log N) |
| 최대 경로 길이 | O(1) | O(N) | O(log N) |

- Self-Attn : N²로 계산량 많지만 **완전 병렬화** 가능, 장거리 의존 유리
- RNN : 순차 처리 필수
- CNN : 장거리는 층 쌓기로 해결

## 8. 핵심 정리

1. Attention = Q·K 유사도로 V를 가중합하는 연산.
2. √d_k 로 스케일링해 softmax saturate 방지.
3. Self-Attention은 Q/K/V가 같은 입력에서 생성.
4. Cross-Attention은 Q=디코더, K/V=인코더.
5. Masked Self-Attention은 디코더의 미래 토큰을 mask(-∞).
6. Multi-Head = 여러 관점에서 관계 학습.
7. Positional Encoding 없으면 순서 정보 0.
8. sin/cos PE의 장점 : 외삽, 상대 거리, 파라미터 0, 다주파.
9. Transformer는 완전 병렬화 가능 (RNN 대비).
10. Max Path O(1) → 장거리 의존 포착.
11. 트랜스포머 3요소 : Attention + FFN + LayerNorm(+Residual).
12. *"Attention is All You Need"* (2017) 가 패러다임 전환.
13. 이후 BERT/GPT/ViT/Diffusion 등 모든 현대 아키텍처의 기초.
14. 실제 구현은 학습 가능한 positional embedding도 많이 사용.
15. 행렬 곱으로 한 번에 전체 시퀀스의 attention을 계산 → GPU 최적화에 유리.

### 강의 중 주요 직관 비유
- **"기계 번역 시 현재 단어에 필요한 입력 부분에 집중"** : Attention 메커니즘의 직관.
- **"Attention is All You Need"** : 시퀀스 처리에는 RNN·CNN 없이 Attention만으로 충분하다는 혁신적 주장.
