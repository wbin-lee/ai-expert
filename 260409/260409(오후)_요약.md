# 260409 오후 강의 요약 — LLM·BERT vs GPT·디코딩 전략·Scaling Law

## 1. BERT vs GPT 구조 비교

### 1.1 개념
- **BERT** (Bidirectional Encoder Representations from Transformers)
  - Transformer **Encoder만** 사용
  - **양방향(bidirectional)** 문맥
  - Masked Language Modeling(MLM)으로 사전학습
- **GPT** (Generative Pre-trained Transformer)
  - Transformer **Decoder만** 사용
  - **일방향(unidirectional, causal)** 문맥
  - Next Token Prediction 으로 사전학습

### 1.2 한눈 비교표
| 항목 | BERT | GPT |
|------|------|-----|
| 구조 | Encoder-only | Decoder-only |
| 문맥 | Bidirectional | Unidirectional |
| 사전학습 | MLM (+ NSP) | Causal LM |
| 입력 예시 | `[CLS] 나는 [MASK] 간다 [SEP]` | `나는` → 예측 `학교에` |
| 활용 | 분류·NLU·임베딩 | 텍스트 생성·대화 |

### 1.3 학습 목표 수식
**BERT (MLM)**
```
P(xᵢ | context \ xᵢ) 예측
```

**GPT (Causal LM)**
```
P(xₜ | x₁, x₂, …, x_{t-1}) 예측
Loss = -Σₜ log P(xₜ | x₁:t-1)
```

## 2. Scaling Law

### 2.1 3요소
- **N** : 모델 파라미터 수
- **D** : 데이터(토큰) 크기
- **C** : 연산량(FLOPs)

### 2.2 경험적 관계 (Power-Law)
```
Loss ≈ c_N · N^(-αN)
Loss ≈ c_D · D^(-αD)   (αD ≈ 0.09)
Loss ≈ c_C · C^(-αC)   (αC ≈ 0.08)
```
→ 파라미터·데이터·연산이 늘어날수록 테스트 손실이 예측 가능하게 감소.

### 2.3 고정 예산 하 최적 할당
- 초기 OpenAI 관점 : **모델 크기 우선**.
- 이후 **Chinchilla** 관점 : N과 D를 **균형 있게** 증가시켜야 최적.

## 3. In-Context Learning (Few-shot Learning)

### 3.1 정의
사전학습된 거대 모델이 **별도 Fine-tuning 없이**, 프롬프트에 주어진 몇 개 예시만으로 새 태스크를 수행하는 능력. GPT-3 이후 본격 창발(emergent) 능력으로 관찰됨.

### 3.2 유형
- **Zero-shot** : 예시 없음, 지시만 주기
- **One-shot** : 예시 1개
- **Few-shot** : 예시 3~5개

### 3.3 예시 (Few-shot 번역)
```
Q: Capital of Germany?   A: Berlin
Q: Capital of Japan?     A: Tokyo
Q: Capital of France?    A: ?   → Paris
```

### 3.4 Chain-of-Thought Prompting
예시에 **단계별 풀이 과정** 까지 포함시켜 복잡한 추론을 유도. 산수·논리 문제에서 큰 성능 향상.

## 4. 디코딩 전략 (Decoding Strategies)

### 4.1 Greedy Decoding
```
x_t = argmax P(x_t | x₁:t-1)
```
- 빠르고 결정적, 단조·국소 최적에 취약

### 4.2 Beam Search
- 각 스텝에서 상위 **k개 경로** 유지
- 최종엔 누적 확률이 가장 높은 경로 선택
- 비용 : k배

### 4.3 Top-K Sampling
- 상위 K개만 후보로, 나머지 확률 0
- K 값은 고정
- 다양성 확보하되 터무니없는 토큰 배제

### 4.4 Top-P (Nucleus) Sampling
- 확률 누적이 P(예: 0.9)를 넘는 지점까지만 후보
- 분포가 뾰족하면 소수 토큰, 평평하면 많은 토큰 → **adaptive**

### 4.5 Temperature Scaling
```
P_i ∝ exp(z_i / T)
```
- T = 1 : 원본
- T > 1 : 분포 평탄 → 다양성 ↑
- T < 1 : 분포 예각 → 결정적
- T → 0 : Greedy와 같음

### 4.6 상황별 선택
- **수학 정답** : T ≈ 0 또는 Greedy
- **창의적 글** : Top-P + T 0.8~1.2

## 5. Language Modeling 기본 원리

### 5.1 확률 분해
```
P(sentence) = P(w₁) · P(w₂|w₁) · P(w₃|w₁,w₂) · …
log P(sentence) = Σ log P(wᵢ | w₁:i-1)
```

### 5.2 Perplexity
```
Perplexity = exp( -Σ log P(wᵢ) / N )
```
- 낮을수록 모델 확신 / 성능 우수
- 높을수록 불확실

## 6. 핵심 정리

1. BERT = Encoder-only, 양방향, MLM 사전학습.
2. GPT = Decoder-only, 일방향, Causal LM 사전학습.
3. BERT ↔ 분류·NLU / GPT ↔ 텍스트 생성.
4. Scaling Law : N·D·C 증가 → 성능 power-law 향상.
5. OpenAI 관점(N 우선) vs Chinchilla(N·D 균형).
6. In-Context Learning = 파라미터 고정 상태로 프롬프트만으로 태스크 적응.
7. Chain-of-Thought 프롬프팅으로 복잡 추론 가능.
8. Greedy = 빠르지만 국소 최적.
9. Beam Search = 상위 k 유지, 비용 k배.
10. Top-K는 고정 K, Top-P는 누적확률 P (adaptive).
11. Temperature로 다양성/결정성 조절.
12. T → 0 = Greedy와 동일.
13. 디코딩 전략은 **속도-품질-다양성** trade-off.
14. Perplexity로 언어모델 평가.
15. LLM = Transformer + 거대 데이터 + Scaling Law 의 결과.

### 강의 중 주요 직관 비유
- **대학 교양 비유** : Pre-training = 초·중·고 교양, Fine-tuning = 대학 전공.
- **"ChatGPT는 인터넷 전체를 공부한 상식 부자"** : 대규모 pretraining의 본질.
- **수학 문제 = 확률 1 필요** : Greedy/T→0. **시 = 창의성** : Top-P + 높은 T.
