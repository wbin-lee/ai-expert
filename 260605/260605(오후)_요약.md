# 260605 오후 실습 요약 — Semantic Segmentation 모델 직접 구현 · FCN-32 · FCN-8 (Skip Connection) · Pre-trained VGG Backbone

> 오후 흐름: **20-class PASCAL Segmentation 데이터셋 준비 → FCN-32 모델 구조 직접 작성(Conv6/Conv7 + Score 분류기 + Deconv 32× upsampling) → VGG16 사전학습 모델에서 FC weight 로딩(view 함수로 shape 변환) → Forward 함수와 Crop 연산으로 입력 이미지 크기 정합 → FCN-8 모델 구현(3개 branch: pool3/pool4/pool5에서 각각 score 추출 후 Skip Connection으로 합산, 3단계 progressive upsampling) → Multiplier scalar(0.01, 1e-4)로 보조 brancha 가중치 조정 → 학습 loop(get_prediction 함수, argmax로 클래스 prediction, cross-entropy loss) → 체크포인트 로딩으로 미리 학습된 모델 평가(Pixel Accuracy, mIoU)**
> 강의 주제: **금요일 오전 강의에서 배운 FCN의 두 가지 변형(FCN-32: 최종 단일 scale, FCN-8: 멀티 scale skip connection)을 PyTorch로 직접 구현하면서 (1) FC→Conv 변환, (2) Transposed Convolution upsampling, (3) Skip Connection으로 디테일 복원, (4) Pre-trained backbone 활용을 코드로 체득하는 것**.

# 2. FCN-32 모델 구조 ⭐

## 2.1 핵심 구성요소

```
입력 이미지 → VGG Backbone (학습 X, weight 로딩만) → pool5 feature (512 ch)
                                                  → Conv6 (512→4096, kernel=7)
                                                  → Conv7 (4096→4096, kernel=1)
                                                  → Score Classifier (4096→num_class, kernel=1)
                                                  → Deconv (32× upsampling)
                                                  → Crop으로 입력 크기에 맞춤
```

## 2.2 핵심 레이어 정의 ⭐

| 레이어 | 코드 |
|---|---|
| **Conv6** | `nn.Conv2d(512, 4096, kernel_size=7)` |
| **Conv7** | `nn.Conv2d(4096, 4096, kernel_size=1)` |
| **Score Classifier** | `nn.Conv2d(4096, num_class, kernel_size=1)` — 1×1 conv로 pixel-wise classification |
| **Deconv (32×)** | `nn.ConvTranspose2d(num_class, num_class, kernel_size=64, stride=32, bias=False)` |

### Conv7의 의미 ⭐

원래 FC layer는 입력 크기가 고정이어야 하지만, **1×1 Conv로 해석**하면 입력 크기 무관하게 적용 가능. 픽셀마다 4096차원 벡터를 매핑하는 fully-connected layer가 모든 위치에 공유되는 효과.

### Deconv layer

- Transposed convolution. 사이즈가 32분의 1로 줄어든 feature map을 다시 32배 키움.
- `bias=False`: upsampling만 하는 역할이라 bias 불필요.
- 학습 가능한 보간 — 단순 nearest-neighbor 업샘플링보다 정교.

## 2.3 Pre-trained Weight 로딩 ⭐

VGG16의 FC6/FC7 weight를 conv layer에 옮긴다 — **`view()` 함수로 shape 변환**:

```python
def load_pretrained(self, vgg):
    self.conv6.weight.data = vgg.classifier[0].weight.data.view(self.conv6.weight.size())
    self.conv6.bias.data = vgg.classifier[0].bias.data
    self.conv7.weight.data = vgg.classifier[3].weight.data.view(self.conv7.weight.size())
    self.conv7.bias.data = vgg.classifier[3].bias.data
```

→ FC layer의 \(W \in \mathbb{R}^{4096 \times (7 \cdot 7 \cdot 512)}\) 매트릭스를 **\(4096 \times 512 \times 7 \times 7\) conv 커널**로 reshape.

## 2.4 Forward 함수와 Crop ⭐

```python
def forward(self, x):
    pool5 = self.vgg_features(x)         # backbone
    out = self.conv6(pool5)              # 1×1 → 7×7 conv
    out = self.relu(out); out = self.dropout(out)
    out = self.conv7(out)                # 1×1 conv
    out = self.relu(out); out = self.dropout(out)
    out = self.score(out)                # num_class 채널
    out = self.deconv(out)               # 32× upsample
    
    # Crop으로 입력 사이즈에 맞춤
    out = out[:, :, 9:9+x.shape[2], 9:9+x.shape[3]]
    return out
```

**Crop의 이유**: Deconv가 64 kernel size로 upsampling하므로 정확히 입력과 같은 크기가 아닌 약간 더 큰 출력이 나옴 → 픽셀 9개씩 잘라 정합.

# 3. FCN-8 모델 구조 ⭐⭐

## 3.1 핵심 차이 — Skip Connection ⭐

FCN-32는 32× downsampled feature 하나만 사용하지만, FCN-8은 **3개의 다른 scale branch**를 합쳐 사용:

```
이미지 → VGG → pool3 (8× downsample)
            → pool4 (16× downsample)
            → pool5 (32× downsample) → Conv6/7 → Score(32×) → Deconv(2×) ───┐
                                                                            ▼
                                       pool4 → Score(16×) ──→ Crop ───→ ⊕──┘
                                                                       Deconv(2×)
                                                                            ▼
                          pool3 → Score(8×) ──→ Crop ──────────────→ ⊕─────┘
                                                                     Deconv(8×)
                                                                            ▼
                                                                        최종 출력
```

## 3.2 3개 Score Layer

각 풀링 레이어의 채널 수에 맞춰 입력 채널만 다름:

| Score Layer | 입력 채널 | 출력 채널 |
|---|---|---|
| `score_x32` (pool5/conv7) | 4096 | num_class |
| `score_x16` (pool4) | 512 | num_class |
| `score_x8` (pool3) | 256 | num_class |

→ 모두 1×1 conv (커널 사이즈 1).

## 3.3 3단계 점진적 Upsampling

| Deconv | 역할 |
|---|---|
| `deconv1` (kernel 4, stride 2) | 32× → 16× |
| `deconv2` (kernel 4, stride 2) | 16× → 8× |
| `deconv3` (kernel 16, stride 8) | 8× → 원본 |

## 3.4 Forward와 가중치 조정 ⭐

```python
def forward(self, x):
    pool3, pool4, pool5 = self.vgg_features(x)
    
    # 메인 branch (32× → 16× → 8× → 원본)
    out = self.conv6(pool5); out = self.conv7(out)
    main = self.score_x32(out)              # 32× downsample
    main = self.deconv1(main)               # 16× downsample
    
    # 보조 branch 1 (pool4, 16×)
    aux1 = self.score_x16(0.01 * pool4)     # multiplier 0.01
    aux1 = aux1[:, :, 5:5+main.shape[2], 5:5+main.shape[3]]
    main = main + aux1                       # Skip Connection (element-wise sum)
    main = self.deconv2(main)               # 8× downsample
    
    # 보조 branch 2 (pool3, 8×)
    aux2 = self.score_x8(1e-4 * pool3)      # multiplier 1e-4
    aux2 = aux2[:, :, 9:9+main.shape[2], 9:9+main.shape[3]]
    main = main + aux2                       # Skip Connection
    main = self.deconv3(main)               # 원본 사이즈로
    
    # 최종 crop
    return main[:, :, 31:31+x.shape[2], 31:31+x.shape[3]]
```

### Multiplier(0.01, 1e-4)의 의미 ⭐

보조 branch는 보조적인 역할이므로 **메인 branch보다 약하게** 가중. 0.01, 1e-4 같은 scalar로 곱해 너무 강한 영향을 막음 — 일종의 정규화.

### 멀티 스케일 결합의 효과 ⭐

| Branch | 정보 |
|---|---|
| pool5 (32×) | 시멘틱 정확도 高, 공간 정보 低 |
| pool4 (16×) | 중간 |
| pool3 (8×) | 시멘틱 정확도 低, 공간 정보 高 |

→ 세 가지를 결합해 **시멘틱 정확도와 공간 디테일을 모두 잡음**. 경계 부분 디테일이 FCN-32보다 훨씬 좋아짐.

# 4. 학습 함수 ⭐

## 4.1 `get_prediction` 함수

```python
def get_prediction(net, image, label, criterion):
    output = net(image)                      # (B, num_class, H, W)
    loss = criterion(output, label).mean()   # cross-entropy per pixel
    prediction = output.argmax(dim=1)        # 가장 높은 확률 클래스
    return output, loss, prediction
```

### 핵심 디테일

- 출력 shape: `(B, num_class, H, W)` → argmax는 클래스 차원(dim=1)에서.
- Loss는 배치 평균 (스칼라화).
- **argmax 대신 random sampling**도 가능 (모델 출력이 확률 벡터). 애매한 경우(자동차 vs 사람 확률이 비슷)에 항상 1등만 고르는 argmax보다 유연성을 줄 수 있음. 실습에서는 단순 argmax 사용.

## 4.2 Training Loop

```python
for epoch in range(num_epochs):
    for image, label in train_loader:
        output, loss, pred = get_prediction(net, image, label, criterion)
        
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
        
        total_loss += loss.item()
    
    # Validation
    for image, label in val_loader:
        with torch.no_grad():
            output, val_loss, pred = get_prediction(net, image, label, criterion)
```

## 4.3 평가 메트릭 ⭐

| 메트릭 | 정의 |
|---|---|
| **Pixel Accuracy** | (정답 픽셀 수) / (전체 픽셀 수) — 가장 나이브 |
| **mIoU (mean IoU)** | 클래스별 (intersection 영역) / (union 영역)의 평균 |

mIoU가 더 의미 있는 메트릭 — 모델이 예측 영역과 GT 영역이 얼마나 겹치는지를 측정.

## 4.4 학습 시간 vs 체크포인트

- 한 epoch에 약 1시간 → 실습 시간 안에 의미 있는 학습 불가.
- 강사가 미리 학습한 체크포인트(`Trial0/best.pt`)를 다운받아 평가만 진행.

# 6. 실습 핵심 코드 패턴 ⭐

## 6.1 FC → Conv 변환 코드 패턴

```python
# VGG의 classifier (FC layer)의 weight를 conv shape으로 reshape
conv_layer.weight.data = fc_layer.weight.data.view(conv_layer.weight.size())
conv_layer.bias.data = fc_layer.bias.data
```

## 6.2 Skip Connection 패턴

```python
# 메인 branch를 deconv로 키우고
main = self.deconv(main)

# 보조 branch를 메인의 spatial size에 맞춰 crop
aux = score_layer(multiplier * skip_feature)
aux = aux[:, :, offset:offset+main.shape[2], offset:offset+main.shape[3]]

# element-wise sum으로 결합
combined = main + aux
```

## 6.3 Final Crop 패턴

```python
# Deconv 후 출력이 입력보다 약간 큰 경우 crop
output = output[:, :, offset:offset+input_H, offset:offset+input_W]
```

→ Crop offset은 deconv 커널 사이즈, stride에 따라 결정되며, 실험적으로 맞춰야 함.

# 7. 오후 핵심 정리 ⭐

- **FC → Conv 변환**은 `nn.Linear`를 `nn.Conv2d`로 바꾸고 weight를 `view()`로 reshape하는 단순 작업으로, FCN의 핵심 트릭이다.
- **FCN-32**는 pool5 단일 scale만 사용해 빠르고 단순하지만, 32× upsampling으로 인한 마스크 뭉뚝함이 한계다.
- **FCN-8**은 pool3/pool4/pool5의 세 가지 scale을 **Skip Connection으로 결합**해 시멘틱과 공간 디테일을 모두 살린다.
- **Multiplier(0.01, 1e-4)**로 보조 branch의 영향을 줄이는 것은 메인 branch를 너무 흔들지 않으면서 디테일만 추가하는 정규화 기법이다.
- **Transposed Convolution**의 kernel size와 stride에 따라 출력이 입력보다 약간 커지므로 **Crop 연산**으로 정합해야 한다.
- **Pre-trained VGG backbone**을 freeze하고 새 layer만 학습하는 transfer learning 패턴은 작은 데이터셋에서 segmentation을 빠르게 학습할 수 있게 한다.
- 평가 메트릭은 단순 Pixel Accuracy보다 **mIoU**(클래스별 IoU 평균)가 더 의미 있으며, FCN-8이 FCN-32보다 mIoU에서 명확한 향상을 보인다.
