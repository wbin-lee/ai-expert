# 260605 오전 강의 요약 — Semantic Segmentation · CNN 복습(Receptive Field) · FCN · Encoder-Decoder(U-Net/DeconvNet) · Atrous Convolution · Spatial Pyramid Pooling

> 오전 흐름: **CNN 기초 복습(Convolution 입출력 공식, Padding/Stride, Pooling, Receptive Field) → Semantic Segmentation 문제 정의(픽셀 단위 클래스 레이블, MSRC만 쓰던 시대 → 2014년 폭발) → 초창기 방법(Selective Search proposal + CNN feature + SVM) → Fully Convolutional Network(FC를 Conv로 해석, 큰 이미지에 sliding window 효율적 적용) → FCN의 두 문제(낮은 score map 해상도, 작은 고정 receptive field) → 인코더-디코더 모델(U-Net의 Skip Connection, DeconvNet의 Unpooling + Transposed Convolution) → Atrous Convolution(rate로 receptive field만 키우기) → Spatial Pyramid Pooling(멀티 윈도우 풀링 동시 적용) → Pose Estimation도 같은 문제(이미지→이미지 변환), 모든 비전 90%가 같은 구조로 풀린다**
> 강의 주제: **Semantic Segmentation을 풀기 위한 신경망 설계의 핵심은 "Receptive Field 키우기 vs Output Resolution 유지"의 트레이드오프를 어떻게 풀 것인가이며, FCN의 두 한계(작은 RF, 작은 score map)를 인코더-디코더/Atrous Conv/SPP로 보완하는 흐름이 모든 이미지→이미지 변환 문제에 공통적으로 적용된다는 것을 이해하는 것**.

# 1. CNN 기초 복습 ⭐

## 1.1 Convolution의 입출력 공식

입력 \(N \times N \times C\), 커널 \(K \times K\), Stride \(S\), Padding \(P\), 필터 개수 \(D\):

\[
N' = \left\lfloor \frac{N + 2P - K}{S} \right\rfloor + 1
\]

- 출력 채널 = 필터 개수 \(D\).
- 흔한 세팅: 해상도 유지하려면 \(K=3, S=1, P=1\).

## 1.2 Receptive Field ⭐

**정의**: 중간 레이어의 한 액티베이션이 **입력 데이터의 어느 영역**을 관측하고 있는가.

- 항상 입력 데이터 기준으로 정의된다.
- Receptive field 바깥의 픽셀을 바꿔도 그 액티베이션은 변하지 않는다.
- 누적 공식: \(K-1\)만큼씩 커지고, 중간에 stride/pooling이 있으면 그 배율만큼 커진다.

## 1.3 Pooling을 쓰는 이유

Receptive field를 키우려면:

| 방법 | 트레이드오프 |
|---|---|
| Conv kernel 크기 증가 | 파라미터 증가 |
| Conv stride 증가 | 파라미터는 그대로지만 feature map 작아짐 |
| **Pooling** | **파라미터 없이** receptive field 키움, 단 feature map 작아짐 |

→ Pooling은 "파라미터를 늘리지 않고 RF를 키우는" 핵심 도구.

# 2. Semantic Segmentation 문제 정의 ⭐

## 2.1 테스크

이미지 분류와 달리 **각 픽셀마다 클래스 레이블**을 추정. 학습 데이터는 (이미지, 픽셀 단위 마스크) 쌍.

- 마스크 라벨링 비용이 매우 큼(반도체 결함 탐지의 경우 전문가만 가능 → 비쌈).

## 2.2 클래시피케이션과 비교했을 때 어려운 점 ⭐

1. **특징적인 영역만으로는 부족**: 클래시피케이션은 얼굴만 봐도 사람이라고 알 수 있으나, segmentation은 단색 옷처럼 특징 없는 모든 픽셀에 레이블을 줘야 함.
2. **해 공간이 매우 큼**: 클래스 20개 기준 이미지 분류는 찍어서 5% 정확도지만, 32×32 segmentation은 \(20^{1024}\)가지 가능성 → 초기 그래디언트 신호 거의 없음.

→ 2012년 알렉스넷, 2014년 본격적 brick-through로 논문 폭증.

---

# 3. 초창기 Segmentation: Proposal + CNN feature + SVM ⭐

1. Selective Search로 **2000개 segmentation proposal**(픽셀 색이 비슷한 그룹들) 생성.
2. ImageNet pre-trained CNN으로 이미지 전체에서 feature map 추출.
3. 각 proposal 영역의 feature를 평균 풀링.
4. SVM으로 (개, 사람, 배경) 분류.

**문제**:

- Proposal 알고리즘에 성능 cap이 묶임 (제대로 된 마스크가 없으면 망함).
- 여러 모듈로 나뉘어 있어 **end-to-end 학습 불가** — gradient가 안 흐름.
- Feature extractor가 classification용이지 segmentation용이 아님.

→ "전부 한 번에 학습되는 end-to-end pipeline"이 필요해진다.

# 4. Fully Convolutional Network (FCN, 2014) ⭐

## 4.1 핵심 아이디어 — CNN을 거대한 필터로 ⭐

이미지 분류 CNN(224×224 입력 → 1000-d 벡터)을 그대로 **거대한 필터**로 보고, 더 큰 입력 이미지에 **slide**하면 출력이 score map이 된다.

- 나이브 구현: 패치를 떼서 CNN 2000번 돌림 → 너무 비효율(겹치는 영역 재계산).
- **Convolution은 계산 재사용에 최적화** → 전체를 conv로 만들면 한 번의 forward로 끝.

## 4.2 Fully Connected → Convolution 변환 ⭐⭐

마지막 7×7×512 feature를 4096-d vector로 매핑하는 FC layer:

**FC 관점**: \(W\)의 사이즈는 \(4096 \times (7 \cdot 7 \cdot 512)\). 입력 크기가 고정되어야 함.

**Conv 해석**: 같은 weight를 **7×7×512×4096 conv 필터**로 보면, 입력이 커지면 **sliding window**가 가능 → 입력 크기 무관!

- 22×22×512 입력에 7×7 conv (stride 1, no padding) → 16×16×4096 출력.
- 다음 FC도 1×1 conv로 해석 → 최종 16×16×1000 score map.

## 4.3 전체 파이프라인

```
이미지(500×500) → CNN(VGG/AlexNet) → 작은 score map(10×10×21)
                                  → Upsampling(이미지 크기로 키움)
                                  → 픽셀별 cross-entropy loss
```

End-to-end로 backprop 가능. 모든 연산이 미분 가능.

## 4.4 FCN의 두 가지 한계 ⭐⭐

1. **낮은 score map 해상도**: 500×500 입력이 10×10으로 줄어듦. Upsampling으로 키워도 **소실된 디테일은 복원 안 됨** → 마스크가 뭉뚝.
2. **고정된 작은 Receptive Field**: CNN 자체가 224×224에 맞춰 설계 → 큰 이미지를 넣어도 RF는 작음 → **국부만 보고 분류** → 말의 몸통(특징 없음)을 자전거나 개로 오분류.

→ "RF는 키우고 싶고, 해상도는 줄이고 싶지 않다"는 **트레이드오프**가 핵심 문제.

# 5. Encoder-Decoder 모델 ⭐⭐

## 5.1 Encoder vs Decoder

| 단계 | 역할 |
|---|---|
| **Encoder** | 풀링/stride로 정보 추상화 (해상도↓, 채널↑) |
| **Decoder** | 추상화된 정보를 원래 해상도로 복원 (해상도↑, 채널↓) |

- 이상적으로 decoder는 encoder의 역함수 — 단 정보가 소실되었으므로 정확한 역함수는 불가능, 근사.
- 핵심 연산: **Transposed Convolution**

\[
\text{Convolution} \to \text{stride } 2 \to \text{feature map } \div 2
\]
\[
\text{Transposed Convolution} \to \text{stride } 2 \to \text{feature map } \times 2
\]

## 5.2 Transposed Convolution

- 일반 conv: \(y = Wx\), \(W\)는 \(D' \times D\) 매트릭스 (\(D\) 차원 → \(D'\) 차원).
- 역연산은 이상적으로 \(W^{-1}\)이지만, **\(W^T\)도 역할 수행** (\(D' \times D \to D \times D'\)).
- 시각적 의미: conv는 입력 patch를 stride만큼 건너뛰며 dot product → 하나의 값 출력. Transposed conv는 **하나의 입력 값을 stride만큼 건너뛰며 kernel을 곱해 놓는다** → 출력 키움.

## 5.3 U-Net ⭐⭐

```
Input → Encoder (Conv + Pooling, 해상도↓) → Bottleneck
                                          → Decoder (Transposed Conv, 해상도↑) → Output

   ↑                                          ↑
   └────── Skip Connection ──────────────────┘ (각 해상도별로)
```

**Skip Connection의 역할** ⭐:

1. Encoder에서 **소실된 공간 정보**를 decoder에 직접 전달.
2. **Gradient flow 보장** — backprop 시 깊은 layer까지 gradient가 잘 흐르도록 고속도로 역할 (ResNet의 residual connection과 동일 원리).

→ Decoder는 단순히 upsampling만 하는 게 아니라, **non-linearity는 계속 증가시키면서 해상도를 복원**한다.

## 5.4 DeconvNet — 또 다른 Encoder-Decoder ⭐

극단적 설계: 입력을 24×24 → **1×1까지 축소**(전체 이미지가 한 픽셀로 요약) → 다시 24×24로 복원.

**Unpooling**:

- Max pooling 시 **선택된 픽셀의 원래 위치**를 메타데이터로 저장.
- Decoder에서 unpooling 시 그 위치에 값을 갖다 놓고, 나머지는 비워둔 채로.
- 비어있는 픽셀은 **transposed conv로 보간**.

**장점**:

- 1×1까지 압축 → **receptive field = 이미지 전체** → 전역 맥락 활용 가능.
- 위치 정보 저장으로 mask shape이 세밀해짐.

→ FCN보다 훨씬 정확한 마스크.

# 6. Atrous Convolution (Dilated Convolution) ⭐

## 6.1 동기

Receptive field를 키우려면:

- Kernel 키우기 → 파라미터 폭증.
- Pooling/stride → 해상도 손실.

**해결**: 커널 안에 **공간을 비워주기**.

```
일반 3×3 conv:   [ * * * ]
                 [ * * * ]
                 [ * * * ]   파라미터 9개, RF 3×3

Atrous (rate=2): [ *  *  * ]
                 [          ]
                 [ *  *  * ]   파라미터 9개, RF 5×5
                 [          ]
                 [ *  *  * ]
```

## 6.2 특성

- 파라미터 수는 그대로, **effective receptive field만 확장**.
- Feature map 사이즈도 유지(stride 1).
- 너무 듬성듬성하면 로컬 패턴을 못 봐서 적당히 사용해야 함.

→ DeepLab(Semantic Segmentation)에서 도입, 이후 광범위하게 사용.

---

# 7. Spatial Pyramid Pooling (SPP) ⭐

여러 윈도우 사이즈의 풀링을 **동시에** 수행:

| 풀링 단계 | 출력 |
|---|---|
| Global Average Pooling | 1 벡터 (전체 이미지) |
| 2×2 Pooling | 4 벡터 (4분할) |
| 4×4 Pooling | 16 벡터 (16분할) |
| ... | ... |

여러 스케일의 컨텍스트를 동시에 보존 → 다양한 사이즈의 receptive field를 한 layer에서 통합.

→ Dilated convolution으로도 구현 가능 (둘 다 conv의 일반화).

---

# 8. 일반화 — 모든 비전 문제로 ⭐⭐

## 8.1 이미지 → 이미지 변환 문제

오늘 배운 모든 모델은 사실 **이미지를 이미지로 변환**하는 문제다.

| 테스크 | 출력 |
|---|---|
| **Segmentation** | 픽셀별 클래스 레이블 |
| **Depth Estimation** | 픽셀별 depth 값 (3D 거리) |
| **Object Detection** | 픽셀별 bounding box + class (YOLO도 사실 FCN과 거의 같다) |
| **Pose Estimation** | 픽셀별 joint 위치 (관절을 클래스로 봄) |
| **Instance Segmentation** | 픽셀별 인스턴스 레이블 |

→ "비전에서 푸는 90% 이상의 문제는 이미지→이미지 변환"이며, 같은 인코더-디코더 백본을 공유한다.

## 8.2 Pose Estimation 예시

- 관절(목, 무릎, 팔꿈치)을 클래스로 정의.
- 각 클래스에 대해 "현재 픽셀이 그 관절인가?"를 prediction.
- 좌우 대칭이라 왼쪽 vs 오른쪽 팔꿈치를 구분하려면 **전역 맥락 필요** → FCN의 RF 문제와 동일.
- → 같은 U-Net 구조로 풀린다.

## 8.3 Multi-View와 한계

- Depth/Pose 추정은 사실 **single view로는 ill-defined**. 본질적으로 2개 이상의 view가 필요.
- 실제 production motion capture는 **여러 카메라**가 둘러싼 방에서 수행.
- 그래도 single view에서 푸는 건 그 근사일 뿐.

# 9. 큰 그림 — 도메인 통합 ⭐

기계학습 발전의 방향:

- 원래는 테스크마다 특화 모델 (CNN for 이미지, RNN for 시퀀스, GNN for 그래프).
- **신경망이 들어오면서 점점 하나로 통합**.
- Vision 도메인 내에서 segmentation/detection/depth/pose가 같은 U-Net 백본으로 통일됨.
- Vision과 Language도 통합 진행 중 (예: 나노바나나 같은 이미지 생성과 GPT 텍스트 생성은 결국 통합될 것).

→ "사람 뇌가 하나인 것처럼 모델도 하나로" 가는 흐름.

# 10. 오전 핵심 정리 ⭐

- **Semantic Segmentation**은 픽셀 단위 클래스 레이블을 추정하는 문제로, classification보다 훨씬 어렵다(특징 없는 영역도 분류해야, 해 공간 폭증).
- **FCN의 핵심 아이디어**는 FC layer를 conv로 해석해 임의 크기 이미지에 sliding window를 효율적으로 적용하는 것이며, 한계는 **낮은 score map 해상도**와 **작은 고정 receptive field**다.
- **U-Net의 Skip Connection**은 (1) encoder에서 소실된 공간 정보를 decoder에 직접 전달하고, (2) gradient flow를 보장해 깊은 레이어 학습을 가능하게 한다.
- **DeconvNet**은 1×1까지 압축해 receptive field를 이미지 전체로 만들고, max pooling의 **선택 위치를 저장**해 unpooling 시 정확히 복원하는 영리한 방식이다.
- **Atrous Convolution**(rate로 커널 공간을 비움)은 파라미터를 늘리지 않고 receptive field만 키우며, feature map 크기도 유지한다.
- **Spatial Pyramid Pooling**은 여러 윈도우 크기의 풀링을 동시 적용해 멀티 스케일 컨텍스트를 통합한다.
- **이미지→이미지 변환**이라는 큰 틀에서 segmentation/detection/depth/pose 모두 같은 U-Net 백본으로 풀리며, 한 가지 문제의 해결책이 다른 모든 문제에 그대로 적용된다.
