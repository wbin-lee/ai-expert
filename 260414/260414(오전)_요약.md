# 260414 오전 강의 요약 — CNN 기초 및 아키텍처

## 1. CNN의 3가지 귀납적 편향(Inductive Bias)

1. **Local Connectivity (지역 연결)** — 전체 이미지 대신 작은 영역만 봄
2. **Parameter Sharing (가중치 공유)** — 동일 필터를 이미지 전역에 재활용
3. **Pooling (풀링)** — 공간 해상도를 축소하며 정보 압축

## 2. CNN의 동기(Motivation)

### 2.1 MLP의 비효율
200×200 흑백 이미지 = 40,000 픽셀. 첫 은닉층이 100뉴런이면 **4,000,000개 파라미터**.

### 2.2 CNN 적용 시
3×3 필터는 채널 수에만 의존 → RGB 이미지도 수천 개 파라미터로 충분. 파라미터 수 **1,000 배 이상 감소**.

### 2.3 컴퓨터 비전의 요구 특성
- 고차원 데이터 : FHD(1920×1080) RGB = 6,220,800 차원
- **Translational Invariance** (위치 이동에 강건)
- **Rotational Invariance** (회전에 강건)
- **Illumination Invariance** (조명에 강건)
- CCTV 영상의 색감 차이 → 추적 어려움의 실전적 예

## 3. CNN의 기본 구성

### 3.1 4대 레이어
1. **Convolution Layer** — 핵심 연산, 가장 많은 계산량
2. **Activation (ReLU)** — 비선형성
3. **Pooling Layer** — 차원 축소
4. **Fully Connected Layer** — 최종 분류

### 3.2 출력 크기 공식
```
H_out = ⌊(H_in − K + 2P) / S⌋ + 1
```
| 설정 | 효과 |
|------|------|
| K=3, P=1, S=1 | 입력 크기 유지 |
| K=5, P=2, S=1 | 입력 크기 유지 |
| K=3, P=0, S=2 | 크기 약 절반 |

### 3.3 Pooling 종류
- **Max Pooling** : 영역 내 최댓값 → 강한 신호 유지
- **Average Pooling** : 평균 → 노이즈 강건
- **Min Pooling** : 최솟값

2×2, stride 2 → 공간 크기 ½, **정보 75% 제거하되 강한 특징 유지**.

## 4. 유명 CNN 아키텍처 계보

| 연도 | 아키텍처 | 특징 |
|------|----------|------|
| 2012 | **AlexNet** | ReLU, Dropout, GPU 학습, ImageNet 우승 |
| 2014 | **VGG** | 매우 깊은 단순 구조 (3×3 반복) |
| 2014 | **GoogLeNet (Inception)** | Inception 모듈, 다양한 커널 병렬 |
| 2015 | **ResNet** | **Skip Connection (잔차)** — 수백 층 가능 |
| 2016~ | **DenseNet** | 모든 이전 층 연결 |

### 4.1 ResNet의 혁신
```
y = F(x, W) + x       # Skip Connection
```
- **잔차(residual)만 학습** → 기울기 흐름 개선
- 깊은 네트워크에서도 학습 가능
- 현재까지도 백본(backbone)으로 많이 쓰임

## 5. 필터 예시


## 6. 파라미터 비교 (구체)

200×200 RGB → 64채널 출력층:
| 구조 | 파라미터 수 |
|------|-------------|
| FC (완전연결) | 약 **7,680,000** |
| Conv 3×3 | 약 **4,800** |
| 감소율 | 약 **1,600 배** |

## 7. 핵심 정리

1. CNN 3축 : Local Connectivity · Parameter Sharing · Pooling.
2. 출력 공식 `(H − K + 2P)/S + 1` 와 P=1/K=3/S=1 유지는 필수 암기.
3. Conv는 FC 대비 1000배↑ 효율.
4. AlexNet(2012)이 딥러닝 모멘트의 시작.
5. VGG → Inception → **ResNet** → DenseNet 계보.
6. ResNet의 Skip Connection이 수백 층 학습 가능케 함.
7. 이미지 전처리 = ImageNet 평균/표준편차 정규화.
8. BatchNorm + 잔차 = 현대 CNN 필수 조합.
9. 필터는 edge/blur/texture 등 특징 자동 탐지.
10. 파라미터 저장보다 중간 feature map 메모리가 더 큼.
11. CNN은 경량/엣지 환경에서 여전히 강력.
12. 아키텍처 설계는 공학 노하우(경험적 시행착오).
13. 데이터 증강(flip, rotate, color jitter)로 일반화 향상.
14. ReLU · Dropout · BN · 스킵의 조합이 현대 CNN의 표준.
15. 컴퓨터 비전은 신호처리 전통과 딥러닝의 결합으로 진화.

### 강의 중 주요 직관·일화
- **Carlo Tomasi의 "3×3 박사학위"** : 스탠포드 시절 "필터 9개 숫자를 완벽히 채우면 박사학위" — 전통 비전이 얼마나 수공업적 난제였는지의 단면.
- **학파 갈등** : 전기전자(신호처리)가 "6천만 파라미터는 비효율" 이라며 CNN을 거부 → 2년 만에 AlexNet으로 뒤집힘.
- **변기 공유 비유** : Parameter Sharing의 경제성.
- **레이더 필터** : 필터가 입력을 특정 특징 강조로 왜곡하는 비유.
