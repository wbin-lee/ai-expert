# 260414 오후 강의 요약 — CNN 실습 (NumPy·PyTorch·CIFAR-10)

## 1. Notation 통일

### 1.1 텐서 shape
- **입력** : `N × C × H × W` (Batch, Channel, Height, Width)
- **필터** : `F × C × KH × KW` (필터 수, 입력 채널, 커널 높이·너비)

### 1.2 중요 규칙
- 배치 크기 **N은 절대 변하지 않음** (입력 N → 출력 N)
- 채널 수는 out_channels 에 의해 변경됨
- 혼동 방지 : 채널과 필터 수를 구분

## 2. 출력 크기 계산

### 2.1 공식
```
H_out = ⌊(H_in − K + 2P) / S⌋ + 1
```

### 2.2 예시
| 입력 | K | P | S | 출력 |
|------|---|---|---|------|
| 7×7 | 3 | 0 | 1 | 5×5 |
| 7×7 | 3 | 1 | 1 | 7×7 (유지) |
| 28×28 | 5 | 2 | 1 | 28×28 (유지) |
| 28×28 | 3 | 0 | 1 | 26×26 |

## 3. Conv Forward 구현 (NumPy)

### 3.1 핵심 알고리즘
```
for n in batch:
    for f in filters:
        for i, j in output_positions:
            window = input[n, :, i:i+KH, j:j+KW]
            output[n, f, i, j] = sum(window * filter[f]) + bias[f]
```
- 4중 루프 : 배치 → 필터 → 출력 위치(H, W)
- 각 위치에서 **element-wise 곱 후 합산 + bias**

### 3.2 PyTorch API
```python
nn.Conv2d(in_channels, out_channels, kernel_size,
          stride=1, padding=0)
```

## 5. Max Pooling 구현

### 5.1 동작
```
window = input[i:i+KH, j:j+KW]
output[i, j] = max(window)
```
- 2×2 윈도우, stride 2 → 겹침 없음
- 100×100 → 50×50 (정보 75% 제거)

### 5.2 Stride별 손실
| Stride | 정보 손실 |
|--------|-----------|
| 2 | 75 % |
| 3 | 89 % |

## 6. CIFAR-10 데이터셋

### 6.1 스펙
- 이미지 크기 : 32×32 RGB (3채널)
- 클래스 : 10개 (airplane, bird, cat, dog 등)
- Train : 50,000 / Test : 10,000

### 6.2 전처리
- 정규화 : (값 − 채널별 평균) / 채널별 표준편차
- ImageNet 통계 자주 사용 :
  - mean = (0.485, 0.456, 0.406)
  - std  = (0.229, 0.224, 0.225)

### 6.3 데이터 증강
- 좌우 반전 (RandomHorizontalFlip)
- 회전 (RandomRotation)
- 밝기/채도 조정 (ColorJitter)
- Crop & Resize

### 6.4 PyTorch Transforms 연쇄
```python
transforms.Compose([
    transforms.ToTensor(),
    transforms.Normalize(mean, std),
    transforms.RandomHorizontalFlip(),
])
```

## 7. CNN vs MLP 파라미터 효율성 (복습)

200×200 RGB → 64채널 기준
- FC : 약 **7,680,000** 파라미터
- Conv 3×3 : 약 **4,800** 파라미터
- 감소율 : **≈1,600배**

## 8. Cross-correlation vs Convolution

- 수학적 convolution : 필터를 뒤집어서 계산
- 딥러닝의 "convolution" = 실제로는 **Cross-correlation**
- 그러나 필터 자체가 학습되므로 결과는 동일 효과

## 9. 핵심 정리

1. 입력 shape = N×C×H×W, 필터 shape = F×C×KH×KW.
2. 배치 N은 항상 보존.
3. 출력 공식 `(H − K + 2P)/S + 1` 암기.
4. K=3, P=1, S=1 이면 크기 유지.
5. K=5, P=2, S=1 이면 크기 유지.
6. Conv = 4중 루프 + element-wise 곱 + bias.
7. PyTorch Conv2d 파라미터 순서 주의 (in, out, kernel, stride, padding).
8. Max Pool 2×2, stride 2 → ½ 크기.
9. Stride 3 pooling = 89 % 손실.
10. 필터는 edge/blur 등 특징 탐지기로 학습됨.
11. 정규화 : (값-평균)/표준편차, ImageNet 통계 사용 일반적.
12. 데이터 증강으로 일반화 향상.
13. `transforms.Compose` 로 전처리 파이프라인 구성.
14. CNN 의 파라미터 효율이 MLP 대비 1000배↑.
15. NumPy 구현 → PyTorch 효율화 의 2단계 학습이 이해에 유효.

### 강의 중 주요 직관·팁
- **Cross-correlation vs Convolution** : 필터가 학습되므로 뒤집느냐의 차이는 무시 가능.
- **LAION-5B 라이선스 분쟁** : 대규모 이미지 데이터셋의 저작권 문제 언급.
- **실습 단골 실수** : 채널 수와 필터 수 혼동 (조교 멘트).
- **element-wise(*) vs matmul(@)** 구분의 중요성 (초심자 실수 포인트).
