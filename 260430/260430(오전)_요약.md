# 260430 오전 강의 요약 — Encoder-Decoder(T5) · Decoder(GPT 시리즈) · Instruction Tuning · RLHF

> 오전 흐름: **T5 (Encoder-Decoder + Span Corruption) → UL2 / Mid-training / Multi-task → GPT-1·2·3 (Decoder-only) → In-Context Learning → Scaling Law / Chinchilla → CoT (Few-shot, Zero-shot) → Prompt Optimization → Instruction Tuning (Flan / Alpaca / LIMA) → Alignment vs Training → RLHF (Reward Model · Bradley-Terry · PPO) → DPO 예고**

## 1. ⭐ Encoder-Decoder 모델 — T5

### 1.1 왜 Encoder-Decoder를 썼나
- 번역(영→불) 같은 task는 입력/출력의 vocabulary가 달라 통합 시 vocab이 너무 커짐.
- 입력 표현(이해)과 출력 표현(생성)이 **분리되어야 한다**는 믿음이 있었음.
- 지금은 한쪽(Decoder)에 몰아서 깊게 쌓는 게 더 효과적임을 발견 → 인코더 디코더 구조의 사용은 줄었지만, **인사이트는 현재 Pre-training에 큰 영향**.

### 1.2 ⭐ T5 = Text-to-Text Transfer Transformer (시험 출제 가능)
- T가 5번 등장 → T5.
- **Transfer Learning** (전이): task A 학습이 다른 task B, C, D로부터 정보를 전달받음. 의학의 "전이"(암 전이 = metastasis)와 같은 개념.
  - **Multi-task Learning**: 여러 task 동시에 학습 (예: BERT의 MLM + NSP).
  - **Sequential Learning**: task A 학습 후 그 파라미터로 task B 이어서 학습 (Fine-tuning).
- **Text-to-Text**: 4가지 task(번역 / CoLA 문법오류 / STS-B 유사도 / 요약)를 **모두 텍스트 입출력 형태로 통합**.
  - 분류(CoLA)도 "not acceptable" 같은 **string으로 출력**.
  - 회귀(STS-B 3.8 같은 numeric) → "3.8"이라는 **string** 출력으로 변환.
  - → **모델 모형을 바꾸지 않고 통합 학습** 가능.

### 1.3 ⭐ Prefix vs Instruction (시험 출제 가능)
- T5에서 task 구분을 위해 입력 앞에 `Translate English to German:`, `Cola sentence:`, `summarize:` 등을 붙임.
- 이때는 **Prefix**라고 부름 (Instruction이 아님).
  - **Prefix** = 일종의 태그, 미니멀한 표시. (`Translate` 한 단어로 충분).
  - **Instruction**(나중 등장) = 자연어 명령, 사람도 이해 가능한 풀어쓴 형태.
- BERT는 task별 classifier가 따로 있었지만, T5는 **동일 디코더**로 모든 task → 신호 부여 위해 prefix 필요.

### 1.4 ⭐ T5의 학습 — Span Corruption + Denoising (시험 출제 핵심)
- Pre-training: **일반 LM** vs **Span Corruption Denoising** 비교.
- Span Corruption:
  - 원문: `Thank you for inviting me to your party last week`
  - Corrupted: `Thank you <X> me to your party <Y> week` (span을 특수기호로 치환)
  - Decoder가 corrupt된 span을 복원하는 task = **Denoising**.
- MLM(BERT)이 단어 단위 마스킹이라면, Span Corruption은 **연속된 N개 토큰(span)** 마스킹.
- 결과: **Denoising loss가 일반 LM보다 GLUE/SuperGLUE/SQuAD/번역에서 더 좋음**.
- 이유: 일반 LM은 뒷부분만 예측 → 인코더가 앞부분 모델링을 게을리할 수 있음. Span Corruption은 앞/뒤 어디서든 망가뜨릴 수 있어 **인코더가 전체 맥락을 압축해 표현하는 훈련**.

### 1.5 ⭐ T5의 발견 — 모델은 "Database"다 (시험 출제 가능)
- T5의 빈칸 맞추기 능력이 **지식 응답에 활용** 가능.
- 예: `Franklin Roosevelt 대통령은 언제 태어났나` 같은 질문에 모델 자체가 답변.
- **DPR(Retrieval) 기반 시스템과 비등하거나 더 좋은 성능** → 검색 없이도 됨.
- **QA의 진화**:
  - **Extractive QA** (SQuAD): 문서 + 질문 → 문서에서 answer span 추출.
  - **Open-domain QA**: 질문만 → 검색 후 답변.
  - **Closed-book QA** (T5가 불씨): **검색 없이 모델 파라미터에서 답** — "모델이 거대 데이터베이스".
- → 모델을 **계산기**가 아닌 **Neural / Vector Database**로 보는 철학적 전환.

### 1.6 UL2 — Mixture-of-Denoisers
- 다양한 LM 방식을 **동시에** 학습:
  - **Regular** (일반 span corruption)
  - **Sequential** (일반 LM, 뒷부분 생성)
  - **Extreme** (긴 span 또는 많이 가린 corruption)
- Pre-training 단계에서도 **R / S / X prefix**로 어떤 LM 방식인지 태깅.
- 인코더-디코더의 가능한 디자인을 다 시도한 셈.

### 1.7 BERT의 친구 ERNIE (참고)
- BERT가 단어 마스킹이라면, ERNIE는 NER 먼저 수행 → **유명 엔티티(예: Donald Trump, 시진핑)를 통째로 마스킹**.
- 마지막 분류기는 vocab 분포가 아닌 **엔티티 DB(예: 5천 개) 위 분포**로 학습.
- → 엔티티 단위 인식 능력 ↑.

### 1.8 ⭐ Mid-Training / Multi-task의 함의
- T5의 학습 = Pre-training + **Mid-training**(또는 intermediate pre-training) + Fine-tuning.
- Mid-training은 학계에 새로운 패러다임을 선물 (좋은 점) / 모델만 보면 단계가 늘어 부담 (나쁜 점).
- BERT는 단일 fine-tuning으로 충분했지만, T5는 **생성형 task의 어려움** 때문에 더 많은 단계 필요.

### 1.9 ⭐ Multi-task Learning의 함정 — Negative Transfer (시험 출제 가능)
- 멀티태스크 러닝은 **공짜 기술이 아니다** — 연구 분야 그 자체.
- 충돌하는 두 지식을 동시 학습하면 **negative transfer** 발생.
  - 비유: "양의학 + 한의학 동시 학습 → 명의 or 끔찍한 혼종"
  - 모델 capacity가 작으면 충돌 시 학습 실패 / 크면 시너지 가능.
- 학습 단계가 길어지면 앞 단계가 **forgetting** 되므로, 가능하면 **multi-task로 통합**하는 게 유리.

## 2. ⭐ 기술을 보는 시각의 전환 — "금광 비유"

> "여기는 금광이고, 모두가 대박나는 LM을 찾아 금을 캐고 있다.
> 옆 10m 갔더니 금이 나오는데, 10m 차이 안 난다고 할 수 없다."

- 메서드(아키텍처)의 큰 변화로만 가치를 평가하면 놓침.
- 별것 아닌 발견(예: DeepSeek의 정답 1 / 오답 0 단순 reward)이 **산업 임팩트**를 만든다.
- 트랜스포머 이전: 새 아키텍처가 핵심.
- 트랜스포머 이후: **호주 사막에서 좌표 찍기** — 누군가 알려주면 당연해 보이지만 발견 자체가 어려운 시대.

### DeepSeek 사례
- "정답 맞으면 1, 못 맞히면 0"이라는 **검증 가능한 reward(RLVR)**.
- 원래 reward 설계는 매우 어려운 문제 (BLEU/ROUGE 같은 평가 메트릭은 불완전).
- 수학처럼 **정답이 명확한 문제**만 학습 → 적은 리소스로 빅테크 능가.
- "Robertra가 BERT보다 데이터 더 많이 / 학습 더 오래" 같은 단순 발견과 동급.

## 3. ⭐ Decoder Model — GPT 시리즈

### 3.1 GPT-1 (2018)
- Decoder-only Transformer + Auto-Regressive.
- 12 layers, 768 dim, FFN 3072 → **약 1억 파라미터** (BERT base와 유사 — BERT가 GPT 스펙을 따라했음).
- **NLU 문제 위주**: NLI, 감성분석 등.
- 입력 형태: `<start> ... <delimiter> ... <extract>` — **extract 토큰**의 표현으로 분류 (BERT의 CLS와 유사한 역할).
- 결과: 일부 task만 SOTA, GLUE에서는 **BERT가 곧 압도** → 대박은 못 침.
- 의미: **하나의 모델이 여러 벤치마크에서 동일하게 잘함**을 처음 보여줌 (이전엔 task별로 LSTM 깊이 / 임베딩 종류를 따로 설계).

### 3.2 GPT-2 (2019)
- 사이즈 ~10억 (10배 ↑).
- 생성형으로 발견된 능력: "안데스 산맥에서 영어하는 유니콘" 같은 입력에 **그럴듯한 이야기 생성** → 처음으로 "말 잘하네"라는 충격.

### 3.3 ⭐ GPT-3 (2020) (시험 출제 핵심)
- **1750억 파라미터** (GPT-1의 1000배, GPT-2의 100배).
- ⭐ **In-Context Learning (ICL)** 능력 발현.
  - **Learning ≠ Training** (강의자 강조):
    - **Learning** = 패턴 인식만 해도 learning.
    - **Training** = 파라미터 업데이트.
  - ICL = training 없이 learning.
- **Zero-shot / One-shot / Few-shot** 정의:
  - 예시 0개 / 1개 / 다수.
  - few-shot은 5개일 수도 50개일 수도 — context size에 의해 제약.

### 3.4 GPT-3의 발견
- 100B 이상 모델부터 **명령(prompt)과 무명령의 성능 차이**가 발생 → **명령을 이해할 수 있는 능력 발현**.
- 예시가 많아지면 명령의 효과는 사라짐 → 예시가 충분히 task를 표현.
- LLM이라는 용어는 **100B 이상 모델**부터 부르기 시작.

### 3.5 ⭐ "발명이 아닌 발견" (시험 출제 가능)
- LLM은 **발견**이라 부름. OpenAI도 만들었지만 **동작 원리를 다 이해하지 못함**.
- "스케일업 했더니 이런 결과가 나오더라" — report에 가까움.
- GPT-3 이후 학계는 **외계인 기술을 분석**하는 단계 — "왜 잘하는가, 왜 병목인가, 어떻게 대화할 것인가".

### 3.6 ICL의 패턴 인식 가설
- 사람의 언어에는 **병렬 구조**(repetitive parallel pattern)가 있음.
  - 예: "Among Palestine" → "in Gaza"
  - 예: "We will estimate" → "it can be"
- ⭐ **Burst-style 데이터 실험**: 학습 시 병렬 패턴을 의도적으로 망가뜨리면 ICL 성능이 **반 이하로 감소** → 인간 언어의 병렬성에서 ICL이 발생한다는 가설.

### 3.7 LLM과의 "프로토콜" 발견
- **Stylistic Output** 발견: System Prompt + Style 예시 제공 → 성능 ↑.
- 비유: 신임 조교가 학생들에게 "보고서 써오세요"라고만 하면 못 함 → "필요성 / 내용 / 추진 체계 + 표 2개 + 그림" 가이드 주면 잘함.
- **Prompt Engineering** = LLM에게 지식을 잘 뽑아내는 형태를 찾는 작업.

### 3.8 ⭐ Scaling Law (2020) (시험 출제 가능)
- **Compute / Data / Parameters**가 exponential하게 ↑ → **Loss는 linear하게 ↓**.
- 로그-로그 스케일에서 직선.
- 이것을 처음 공식화한 게 2020년 (Kaplan et al.).
- 이 직선을 **언제 깨질지**가 핵심 — 깨지면 새 아키텍처/데이터가 필요하다는 뜻.

## 4. ⭐ Chinchilla — Compute-Optimal Scaling

### 4.1 발견
- 큰 모델은 **수렴이 안 됨** = "이만큼 컴퓨트 썼는데도 데이터를 다 수용 못함".
- ⭐ **Scale = Parameters × Tokens** — 두 요소의 균형이 핵심 (시험 출제 가능).
- GPT-3는 **사이즈에 비해 데이터가 적었음**.

### 4.2 Chinchilla 처방
- GPT-3 175B → **70B로 사이즈 줄임**.
- 대신 데이터를 늘려서 **trained tokens 증가**.
- 결과: 성능 ↑ → **모델 사이즈 / 데이터 사이즈 밸런스의 중요성**.

### 4.3 데이터의 불법 수집
- 거대 LM은 데이터가 **굉장히 많이 필요** → **불법적으로 모은 데이터** 다수.
- "허락보다 용서가 쉽다" — 일단 쓰고 들키면 합의금 (투자금의 상당 부분).
- Anthropic도 마찬가지 / GPT-4o의 Scarlett Johansson 목소리 사례 등.
- 모델은 더 큰 수혜를 입었음.

### 4.4 데이터 침해 탐지 — Membership Inference
- "내 데이터 썼니?" 탐지 모델 학습 가능.
- 학습 데이터 → label 1, 미학습 → label 0 → attack model 훈련.
- **Watermarking**: 알파벳처럼 보이는 다른 코드를 섞어 AI 사용 여부 판별.
- 그러나 탐지 모델이 있으면 **adversarial training**으로 안 들키는 방어 모델 → **창과 방패의 경쟁**.
- 개인정보 노출 사례: **이루다 사건** (이름/주소 노출).

## 5. ⭐ Chain-of-Thought (CoT)

### 5.1 ⭐ Few-shot CoT (Jason Wei, 시험 출제 가능)
- 제이슨 웨이 (학부 졸업 후 박사 안 한 유명 연구자).
- Few-shot 예시 안에 **Rationale (중간 추론)** 포함 → 모델이 그 패턴을 따라 답 전 reasoning 생성.
- ⭐ **Emergent Ability**: 작은 모델은 효과 없음, **일정 사이즈 이상**에서 성능 갭이 갑자기 벌어짐.
- 모델: LaMDA(구글, 작은 대화형) / GPT-3 / PaLM(구글) — 사이즈별 갭 차이 발생.

### 5.2 ⭐ Zero-shot CoT — "Let's think step by step" (시험 출제 핵심)
- 코지마 et al. (도쿄대) 논문.
- 예시 없이 끝에 **"Let's think step by step"** 한 줄 → 성능 4배 ↑.
- 예시 제공이 어려운 시나리오(법률, 의료, 로봇)에서 매우 유용.
- ⭐ **Zero-shot vs Few-shot은 시나리오가 다르다**:
  - Few-shot은 upper bound 역할.
  - Zero-shot이 예시 없이도 비슷한 성능에 가까이 가면 좋은 결과.
- 변형 시도: `Let's think step by step`, `First`, `Let's think about this logically` 등 — 평균적으로 **"Let's think step by step"**이 잘 됨.

### 5.3 o1 / o3 — CoT의 내재화
- 추론 모델(o1, o3)은 **Reasoning Step을 내재화** → "Let's think step by step"을 굳이 쓰지 않음.
- 두 번 쓰면 오히려 예상치 못한 오류 발생 가능.

### 5.4 Prompt Optimization
- 사람이 prompt 엔지니어링 = `프롬프트 v1 → 테스트 → 분석 → v2 → 테스트 → 분석`.
- 이걸 **기계가 자동화** = Prompt Optimization.
- 결과: 사람이 만든 것보다 성능 더 좋음. + 24시간 / 병렬 가능.

### 5.5 한계
- 결국 모델 함수 `Y = f(X; θ)`에서 **X(prompt)만 바꾸는 것**.
- θ(파라미터)는 못 바꿈 → 본질적으로 모델이 못 풀던 문제는 prompt로 한계.
- → **θ도 바꾸자** = Instruction Tuning, Reinforcement Learning.

## 6. ⭐ Instruction Tuning

### 6.1 왜 필요한가
- LLM의 prior 확률은 **명령 수행에 align되지 않음**.
- 명령에 대해 답이 아니라 **유사한 문장**을 생성하는 경향.
- 사람을 돕는 목표와 LM의 목표가 **mis-aligned** → 정렬 필요.

### 6.2 ⭐ Fine-Tuning vs Instruction-Tuning (시험 출제 가능)
| | Fine-Tuning | Instruction-Tuning |
|---|---|---|
| 학습 대상 | 하나의 task | **수많은 task의 instruction following** |
| 데이터 | 적음 | 매우 많음 (50~수천개 task) |
| 능력 | task 자체의 능력 학습 | **명령 자체를 따르는 능력** 학습 |
| 가정 | task 능력은 키워야 함 | **이미 능력은 있음, 끄집어냄** |

### 6.3 ⭐ Plan / Flan-T5 (시험 출제 가능)
- Plan(=FLAN) Dataset = 기존 text-to-text 데이터 + **Instruction**.
- T5의 Prefix와 다른 점: Prefix는 태그, Plan은 **자연어 instruction**.
  - 예: `Please answer the following questions.` / `Answer the following questions by reasoning step by step.`
- 50개 task 정도 → 처음 보는 task에서도 명령 수행 가능.

### 6.4 데이터셋 변천
- **초기 (Plan)**: 약 50개 task. 기존 NLP 데이터(MNLI / MRPC / QQP / SST-2 등)에 instruction 부착.
- **Super Natural Instructions**: 1600개 task — **기계 자동 생성** task 다수.
- ⭐ **양질의 데이터의 중요성** (Alpaca, LIMA — 시험 가능):
  - **Alpaca**: Instruct-GPT 3.5(빅모델)로 175개 시드 task에서 5000개 데이터 생성 → LLaMA 7B에 학습 → 좋은 성능.
  - **LIMA** ("Less is More for Alignment"): **양질의 1000개**만으로도 잘 작동.

### 6.5 ⭐ Alignment vs Training (시험 출제 가능)
- **Training** = 능력이 없으니 길러줌.
- **Alignment** (정렬) = 능력은 있는데 결만 안 맞음 → 조율.
- 비유: 뛰어난 학생을 와서 **사람 자체를 바꾸려 하지 말고 결만 맞추라**.

### 6.6 ⭐ MMLU 벤치마크
- **Massive Multi-Task Language Understanding** — 대학생 수준 객관식 문제.
- 한국어판: KMLU.
- Random = 25%.
- **GPT-2 → GPT-3 → Gopher → Chinchilla → Flan-PaLM → GPT-4 → Gemini** 순으로 100에 근접.
- 학부생의 모든 학과 지능을 객관식 수준에서 따라잡았음.

### 6.7 ⭐ Instruction Tuning의 한계 (시험 출제 가능)
1. **사람이 쓴 정답 흉내내는 학습** (maximum likelihood) — 정답이 없는 응답에서 부적합.
2. **토큰 레벨 cross-entropy의 부적절성**:
   - 예: `Avatar is a fantasy TV show`에서 정답은 `fantasy`, `show`.
   - `adventure`는 fantasy의 거의 동의어 → OK.
   - `musical`을 show 자리에 두면 잘못 → 그러나 cross-entropy는 동등하게 페널티.
3. **사람을 흉내내봤자 사람 수준** — Super-Human / ASI 목표에 미달.
- → **RLHF 등장**.

### 6.8 ⭐ AGI / ASI (참고)
- **AGI** (Artificial General Intelligence) — 다양한 분야에서 사람 수준.
- **ASI** (Artificial Super Intelligence) — 사람을 **능가**.
- 학계 목표가 "사람 흉내" → "사람 능가"로 이동.

## 7. ⭐⭐ RLHF — Reinforcement Learning from Human Feedback

### 7.1 시작 — InstructGPT (= GPT-3.5)
- 논문: **"Training Language Models to Follow Instructions with Human Feedback"**.
- 코드 네임 `text-davinci-003`.
- ChatGPT = InstructGPT의 **대화형 버전**.
- 명령은 잘 듣지만 **Human Preference**(공공 사회의 가치)와 미정렬:
  - 예: "청와대에 폭탄 설치 방법 알려줘" → 명령은 수행하지만 **원하는 응답이 아님**.
  - Safety / Hallucination / 정치적 답변 등 → 상용화의 큰 벽.

### 7.2 ⭐ RLHF Pipeline 3단계 (시험 출제 핵심)

#### Step 1 — SFT (Supervised Fine-Tuning) = Instruction Fine-Tuning
- "6살 아이에게 달 착륙 설명해줘" → **사람이 직접 답을 작성** → 학습.
- 양질 데이터를 **저비용으로 만들려고 영어권 비선진국 노동력** 활용 (나이지리아 / 필리핀 / 케냐). **Scale AI**가 그 시기 부상.

#### Step 2 — Reward Model 학습
- 모델이 응답 여러 개 생성 → **사람이 상대적 선호 평가** ("D가 제일 좋고 A·B가 제일 나쁘다").
- 사람이 매번 평가하기엔 비싸므로, 사람을 **대체할 수 있는 Reward Model**을 LM에서 학습.
- 입력: prompt + answer → 출력: scalar reward.

#### Step 3 — Reward Model로 RL
- LM이 응답 샘플링 → Reward Model이 점수 → **점수를 최대화**하는 방향으로 학습.
- = RLHF.

### 7.3 ⭐ Bradley-Terry Objective (시험 출제 가능)
- **Calibration 문제**: 딥러닝 모델은 0.8을 진짜 80%처럼 못 씀. 사람도 98 vs 97 정확히 못 씀.
- 절대 점수 ❌ → **두 응답의 페어** (Preferred vs Dispreferred = Winning vs Losing)에서 **선호되는 것의 점수가 더 높도록**.
- 점수 자체가 아닌 **점수의 차이**를 최대화.
- 비유: 신입사원 평가에서 "98점인지 97점인지"는 못 정해도 "B보다 A가 좋다"는 정확.

### 7.4 ⭐ PPO — Proximal Policy Optimization (시험 출제 가능)
- 의미: **Proximal** = 가까운 (스페인 지하철 "다음 역" 안내처럼 "가장 가까운"이라는 뜻).
- OpenAI가 RLHF를 위해 만든 알고리즘.
- 핵심 텀: `reward 최대화 - β·KL(현재 모델 ∥ 초기 모델)`
  - 마이너스가 붙은 텀 = **초기(pre-training된) 모델에서 멀리 떨어지지 마라**.
  - 이미 양질의 파라미터 → 모델 상태 자체를 바꾸진 말고 **결만 조율**.
- = Regularization의 한 형태 (시험 답변 가능).
- 일반적인 RL은 "엉망 상태에서 좋은 상태로" → RLHF는 "이미 훌륭한 상태를 너무 망가뜨리지 마".

### 7.5 ⭐ 강화학습의 3시대
| 시대 | 특징 | 대표 |
|------|------|------|
| **1세대 (전통 RL)** | 딥러닝 X. Value function 등을 수동 설계. | 고전 RL |
| **2세대 (Deep RL)** | Value / Policy / Reward 모델을 **딥러닝**으로. | **AlphaGo** |
| **3세대 (LLM RL)** | LLM에 RL 적용. PPO 등 LLM 전용. | **ChatGPT** |

### 7.6 ⭐ RLHF의 또 다른 효과 — 아부 / 사고
- 사람 피드백에 동조하다 보니 **답변에서 아부성 멘트** 경향 발생 ("핵심을 찔렀어!").
- **Human-in-the-loop**: 학습 과정에서 사람이 판단/검수 단계로 개입.

### 7.7 ⭐ DPO — Direct Preference Optimization (예고)
- Reward Model **없이** 선호 / 비선호 데이터로 직접 학습 → **RLHF 효과를 fine-tuning처럼 쉽게**.
- 비용 ↓, 메모리 ↓, 접근성 ↑.
- 데이터 구성:
  - Preferred: 사람 라벨링 or 좋은 AI로 생성.
  - Dispreferred: AI로 자동 생성 or 안 좋은 AI로 생성.
- 합성 데이터 만들기 어렵지 않음 → 신입사원 면접에서 "데이터 합성 능력" 보는 트렌드.

## 8. 미래 / 메타 — 강사 인사이트

### 8.1 스케일링은 끝났는가?
- 강사 의견: 스케일링은 거의 **끝났음** (80% 확신).
  - 데이터 더 쓸 게 없음 (개인 플랫폼 로그 수집 미약하게 진행 중).
  - 파라미터 더 키우기도 적자 부담.
  - Compute 한계 + 환경 문제 (탄소, 전기 — "GPT에 '감사하다' 한번 쓸 때마다 소가 방귀 N번").

### 8.2 차기 패러다임
- **Reinforcement Learning as the next paradigm**:
  - 이미 학습된 모델이 시뮬레이션하며 경험을 통해 학습 → "시간과 정신의 방"에서 100억년 사는 셈.
  - Anthropic CEO 등의 팟캐스트 토픽: "RL만으로 AGI 도달 가능한가?"
- **Self-Evolving**: T → T+1 시점에 모델이 **스스로 발전**할 수 있는가?
  - 한 번만 가능하면 inductively 반복 → 변곡점.
  - 방법 1: 모델이 데이터를 만들어내고 스스로 학습.
  - 방법 2 (실용적): **Agent의 사용 로그**를 데이터베이스로 활용 → 좋은 것만 남기고 나쁜 건 제거하며 성장.

### 8.3 ⭐ 새로운 회사 구조 비전 (강사 비전)
```
개인 전문가 → 자기 에이전트 사용 → 개인 DB
       ↓ 검증된 것만
       팀 DB
       ↓ 검증된 것만
       전사 DB (기업 자산화 / 지능화)
```
- 일을 할수록 **AI가 똑똑해지는 조직** — "데이터를 잘 모아 쓴다"가 아닌 **새로운 과학**.
- 거버넌스 / 권한 / 자동화가 핵심.
- ⭐ 중국에서 가장 빠르게 진행 — **퇴사한 직원의 데이터로 그 직원을 흉내내는 에이전트 제작**.

### 8.4 ⭐ Mamba / Nemotron — Transformer의 한계?
- 현재까지 가장 큰 변화 가능성.
- 트랜스포머는 셀프어텐션을 **메타 아키텍처**로 활용 (어텐션이 아키텍처를 학습).
- Stanford의 이론적 그룹 → NVIDIA가 채택해 **Nemotron** 같은 형태로 등장.
- 피지컬 AI / AX 시장 확장에는 트랜스포머 본질적 한계 극복 필요.

### 8.5 ⭐ 데이터 거버넌스 / 에러 확산
- 에이전트 데이터가 **noise + 유용한 것**이 섞여있음.
- 에러를 가지고 에이전트가 참조하면 **바이러스처럼 확산** → "예방접종"이 필요.
- 베이스라인: "많이 쓰이는 = 유용한" → 단순하지만 오류도 많이 쓰일 수 있음.
- 거버넌스 자동화 → **AI가 거버넌스도 만든다** (NRF 제안서 토픽).
- 사람이 할 수 있는 건 기계도 할 수 있음 → 다 자동화 방향.

### 8.6 ⭐ 빌리프(Belief) 모델링
- LLM이 "물리학자가 답하듯" 답하면 → 모델 내부에 **belief**가 생긴 것.
- **Fact ≠ Belief**:
  - **Fact**: 실제 일어난 일.
  - **Belief**: 내 마음속의 믿음.
- 챗봇은 원래 belief를 따로 모델링해야 했는데, LLM은 **암묵적으로 belief를 가지게 됨**.
- → 에이전트가 되려면 환경과 상호작용하면서 belief를 업데이트할 수 있어야 함.

## 9. ⭐ Context Size / Attention의 한계 (Q&A 인사이트)

- Self-attention은 **N×N** 스코어 매트릭스 → N=10이면 100, N=100이면 10,000 → quadratic 비용.
- 하드웨어(서빙) 한계가 먼저, 모델 자체 한계가 후순위.
- 단어가 너무 많으면 attention이 **uniform해져 효과 ↓**.
- 해결:
  - **Local Attention**: 범위 제한.
  - 어떤 attention을 안 할지 선택하는 메커니즘.
  - **RoPE (Rotary Position Embedding)**: position embedding을 더하지 않고 함수 자체를 회전 → 위치 정보를 의미 임베딩에 내재.
- BERT → RoBERTa 시기엔 position embedding을 안 써도 잘 됐음.

## 10. 한 페이지 요약

### 10.1 ⭐ 시험 핵심 포인트 (오전)

| 주제 | 핵심 |
|------|------|
| **T5** | Text-to-Text Transfer Transformer, 모든 task를 텍스트 입출력으로 통합 |
| **Prefix vs Instruction** | T5의 prefix는 태그(미니멀), Instruction은 자연어 명령 |

| **UL2** | R/S/X prefix로 다양한 LM 방식 동시 학습 |
| **ERNIE** | 엔티티 단위 마스킹 + 엔티티 DB 분포 |
| **Multi-task / Negative Transfer** | 멀티태스크는 공짜 아님, 충돌 시 학습 실패 |
| **GPT-1** | Decoder-only, ~1억 파라미터, NLU 위주, BERT가 곧 압도 |
| **GPT-2** | 10억 파라미터, 생성 능력 발현 |
| **GPT-3** | 175B, In-Context Learning 발현, LLM 용어 시작 |
| **Learning ≠ Training** | Learning은 패턴 인식, Training은 파라미터 업데이트 |
| **Scaling Law** | Compute / Data / Params exponential ↑ → Loss linear ↓ |
| **Chinchilla** | Scale = Params × Tokens, GPT-3는 데이터 부족 |
| **Few-shot CoT** | Rationale 포함된 예시 + emergent ability(사이즈 임계값) |
| **Zero-shot CoT** | "Let's think step by step" 한 줄 |
| **Prompt Optimization** | 사람의 프롬프트 엔지니어링을 기계가 자동화 |
| **Fine-Tuning vs Instruction-Tuning** | 전자는 task 능력, 후자는 instruction following 능력 |
| **Plan / Flan-T5** | 자연어 instruction을 부착한 데이터로 학습 |
| **Alpaca / LIMA** | 데이터 양 못지않게 **질**이 중요 |
| **Alignment vs Training** | Training = 능력 키움, Alignment = 결만 조율 |
| **Instruction-Tuning 한계** | maximum likelihood, 토큰 cross-entropy 부적절, 사람 수준 한계 |
| **RLHF 3단계** | SFT → Reward Model → RL |
| **Bradley-Terry** | 점수 absolute 아닌 페어의 점수 차이 최대화 |
| **PPO** | reward 최대화 + KL(초기 모델과 가깝게) regularization |
| **3세대 RL** | 1세대 전통 RL → 2세대 Deep RL(AlphaGo) → 3세대 LLM RL(ChatGPT) |
| **DPO** | Reward Model 없이 선호/비선호 데이터로 직접 학습 |
| **AGI / ASI** | AGI = 사람 수준 범용, ASI = 사람 능가 |

```
~2018: Word2Vec / FastText 임베딩 + LSTM 설계

2019-2020: T5 (Text-to-Text) / Span Corruption

2023+: DPO, Thinking 모델, Test-Time Scaling, Agent
```

### 10.4 Pipeline 도식
```
[1] Pre-training            (Causal LM / Masked LM / Span Corruption)
       ↓
[2] (Mid-training)          (T5/UL2의 Multi-task, intermediate pretraining)
       ↓
[3] Instruction Tuning      (SFT, Plan/Flan/Alpaca)
       ↓
[4] RLHF                    (Reward Model → PPO with KL constraint)
       or DPO               (no reward model, direct preference loss)
       ↓
       사용자 align된 LLM
```
