# 1부. 260626 오전 — Explainable AI의 세계

## 1-1. 왜 XAI를 배우는가

곽노준 교수님은 서울대 융합과학기술대학원 지능정보융합학과에서 **Explainable and Responsible AI**를 3일에 걸쳐 강의합니다.

- **1일차**: XAI (설명 가능한 AI)
- **2일차**: Uncertainty, Robustness, Adversarial Attack
- **3일차**: Quantization (모델 압축)

몇 년 전에는 “Responsible AI”라는 말도 쓰였지만, 요즘은 다시 **XAI**라는 용어가 널리 쓰입니다. Physical AI, Agentic AI 같은 새 이슈가 많아지면서, 예전부터 있던 XAI가 다시 주목받는 흐름입니다.

## 1-2. 딥러닝 vs 전통 ML — 설명 가능성

왼쪽은 **딥러닝**(레이어가 수십·수백 개), 오른쪽은 **Decision Tree·Random Forest** 같은 전통 ML입니다.

은행 대출 심사를 생각해 보세요. “왜 대출이 안 되나요?”라고 물으면, Decision Tree는 **“소득이 기준 미만이라서”**처럼 명확한 규칙으로 답할 수 있습니다. 반면 딥러닝은 레이어가 깊어질수록 **비선형성**이 쌓여 decision boundary가 복잡해지고, **왜 원숭이로 분류했는지** 설명하기 어렵습니다.

> ⭐ **시험 포인트**  
> XAI가 필요한 이유: 딥러닝은 **설명이 어렵다**. 전통 ML(Decision Tree, SVM 등)은 decision boundary가 비교적 명확하다.

## 1-3. 해석 가능성의 네 가지 패러다임

교수님은 해석 방법을 **네 가지**로 나눕니다.

| 패러다임 | 보는 것 | 핵심 질문 |
|----------|---------|-----------|
| **Behavioral** | 블랙박스 입·출력 | 어떤 입력에 어떤 출력이 나오나? |
| **Local (Attribution)** | **한 샘플** | 이 예측에 **어디(Where)** 가 기여했나? |
| **Global (Concept-based)** | **레이어·피처** | 모델이 **무엇(What)** 을 배웠나? |
| **Mechanistic** | **레이어 간 연결** | 이전 레이어 피처가 다음 레이어에 **어떻게** 합쳐지나? |

### Local vs Global — Where vs What

- **Local**: 이미지 **한 장**에 대해 “모델이 **어디**를 보고 이 예측을 했는가?”
- **Global**: 모델 전체가 “**무슨 컨셉**을 담고 있는가?” (예: TCAV, 클러스터링)

### Mechanistic — 레이어 사이를 본다

예: 사람을 인식한다면, 이전 레이어에서 **길쭉한 막대(팔)** 에 반응하는 피처와 **동그란 것(얼굴)** 에 반응하는 피처가 합쳐져, 다음 레이어에서 **사람 전체**에 반응하는 피처가 만들어진다.  
이 **레이어 간 인터커넥션**을 분석하는 것이 **Mechanistic Interpretability**입니다. 아직 연구 초기 단계이며, 계산량이 많아 **빠르게 설명하는 방법**이 연구 중입니다.

> ⭐ **시험 포인트**  
> 교수님이 **시험 문제를 낼 것**이라고 직접 말한 컨셉: **Local / Global / Mechanistic**이 **서로 어떻게 다른지** 설명할 수 있어야 합니다.

## 1-4. Local Attribution — CAM과 Grad-CAM

### CAM (Class Activation Mapping, 2016)

1. CNN 마지막 conv 레이어 직전에서 **끊는다**
2. **Global Average Pooling(GAP)** 으로 채널별 평균을 구한다
3. 클래스별 **가중치 W**를 곱해 **weighted sum** → 히트맵

“고양이” 클래스라면, 어떤 채널(노란색 등)의 가중치가 크면 그 채널의 activation map이 강조됩니다.

**CAM의 가장 큰 단점**: 이미 학습된 CNN 위에서 **마지막 레이어를 따로 다시 학습**해야 합니다.

### Grad-CAM (2017) — 재학습 없이

Grad-CAM은 **gradient**로 채널 가중치를 구합니다.

\[
w_k^c = \frac{1}{Z}\sum_{i,j}\frac{\partial y^c}{\partial A_{ij}^k}
\]

- \(A^k\): k번째 채널 feature map
- \(y^c\): 클래스 c에 대한 logit
- **미리 학습된 모델**에 backprop만 하면 됨 → CAM의 재학습 문제 해결

> ⭐ **시험 포인트**  
> **CAM**: GAP + class weight, **마지막 레이어 재학습 필요**.  
> **Grad-CAM**: gradient로 채널 weight, **재학습 불필요**.  
> Grad-CAM 핵심: **feature value × gradient** — gradient만으로는 부족하고, **입력 값 자체**도 중요하다.

### 기타 Local 방법 (개념만)

| 방법 | 한 줄 요약 |
|------|-----------|
| **Integrated Gradients (IG)** | baseline(검은 이미지 등)에서 입력까지 경로 적분; gradient saturation 완화 |
| **SHAP** | Shapley value로 각 feature의 marginal contribution; 표 형 데이터에 강함 |
| **LRP** | gradient가 아니라 **weight**로 relevance를 역전파; 의료 영상 등 |
| **Guided BackProp, Grad-CAM++** | CAM/Grad-CAM 변형 |

## 1-5. Global — TCAV와 로컬의 한계

로컬 방법(Grad-CAM, IG 등)은 **“어디를 봤는지”** 는 알려주지만, **“왜 그게 중요한지”** 는 모릅니다.

- 얼룩말 이미지에서 줄무늬만 강조 → “줄무늬 = 얼룩말”이라고 **착각**할 수 있음
- 같은 이미지를 **플루트(악기)** 로 해석하면, 비슷한 위치가 강조됨 → **클래스만 바꿔도** 설명이 그럴듯해 보임
- **Explanation Can Be Manipulated** — 설명 자체가 조작될 수 있음

**TCAV (Testing with Concept Activation Vectors)** 는 **컨셉** 단위로 접근합니다.

1. 컨셉 이미지(예: 줄무늬) vs 랜덤 이미지로 feature 추출
2. 레이어 L에서 **SVM** 학습 → **Concept Vector C**
3. 클래스 k 이미지에 대해 \(\nabla h_L(x) \cdot C\) (방향 미분) 계산
4. **TCAV score** = 클래스 k 중 양의 score 비율 (예: 970/1000 = 0.97)

> ⭐ **시험 포인트**  
> **로컬의 한계**: Where는 알지만 What/Why는 모름. **글로벌(TCAV)** 은 **컨셉** 단위로 “이 클래스가 이 컨셉에 얼마나 민감한가”를 본다.

## 1-6. Mechanistic — Sparse Autoencoder (SAE)

2024년 LLM 분석에서 센세이션을 일으킨 방법입니다.

**일반 Autoencoder**: 차원을 **줄여** reconstruction  
**Sparse Autoencoder**: 차원을 **4~8배 뻥튀기**한 뒤, **극히 일부(약 0.1%)만** activation되도록 **sparsity** 제약

- 목적: **컨셉을 분리** — SAE feature 하나 ≈ **하나의 해석 가능한 컨셉**
- LLM의 MLP block activation을 reconstruction + sparsity loss로 학습
- 예: 특정 feature가 “Golden Gate Bridge”, “현수교” 등에 반응

**Patch SAE**: Vision Transformer(ViT) 패치 단위로 동일 아이디어 적용.

교수님 연구실은 **SRD**(벡터 단위 relevance + Effective Receptive Field), **CAFE**(클러스터링 기반 글로벌), **Inter-layer Concept Graph**로 local → global → mechanistic을 연결합니다.

# 2부. 260713 오전 — Uncertainty와 Adversarial Attack

## 2-1. 강의의 출발 — Bias와 Uncertainty

XAI와 연결되며, 요즘 **post-training**에서 bias가 많이 생긴다는 논의도 있습니다. Persona, Digital Twin, LLM 여론조사 등에서 **편향(bias)** 을 잘 다뤄야 합니다.

**Bias가 생기는 곳**

1. **데이터** — 특정 집단 데이터가 많으면 그쪽으로 편향 (얼굴 인식에서 darker female 성능 저하 등)
2. **모델** — loss가 ERM(평균 리스크 최소) 위주라 bias·uncertainty를 직접 줄이는 항이 적음
3. **Deployment** — train/test distribution shift
4. **Evaluation·Interpretation** — 사람 주관이 개입

## 2-2. Confidence vs Calibration

모델이 softmax로 **0.8 confidence**로 “고양이”라고 하면, 우리는 80% 확률로 해석합니다.  
실제로 그런 confidence의 예측들 중 **정답 비율**이 80%에 가까우면 **잘 calibrated**된 모델입니다.

**Calibration error** = |accuracy − confidence|  
이를 줄이는 것이 바람직한 모델의 한 목표입니다.

## 2-3. Aleatoric vs Epistemic Uncertainty

불확실성의 **원인**은 크게 두 가지입니다.

### Aleatoric Uncertainty (데이터 불확실성)

- **데이터 자체**에 내재된 noise·겹침
- 같은 x에서 클래스가 겹쳐 있으면 **예측 불가**에 가깝다
- **Irreducible** — 데이터가 주어지면 줄이기 어렵다 (더 많은 feature/차원으로 분리 가능)
- 예: 키가 같을 때 몸무게 분산이 크면, 키만으로 몸무게 예측이 불확실

### Epistemic Uncertainty (모델·시스템 불확실성)

- **데이터가 비어 있는 영역** — 학습 데이터가 없어 모델이 확신할 수 없음
- **데이터를 추가**하면 줄일 수 있다
- SVM 그림에서 decision boundary 사이 **데이터 없는 구간** — 여러 boundary가 모두 100% train accuracy → **어떤 모델이 맞는지 모름**

| | Aleatoric | Epistemic |
|--|-----------|-----------|
| **원인** | 데이터 noise·겹침 | 데이터 부족·모델 지식 부족 |
| **줄일 수 있나** | 차원·feature 추가로 일부 완화 | **더 많은 데이터**로 완화 |
| **비유** | 데이터에 노이즈가 너무 많다 | 데이터가 너무 없다 |

> ⭐ **시험 포인트**  
> **Aleatoric vs Epistemic** 차이를 **설명**할 수 있어야 합니다.  
> - 겹친 클래스 영역 → **Aleatoric**  
> - 데이터 없는 빈 영역 → **Epistemic**

## 2-4. Adversarial Attack — 왜 모델을 속이나

**Evasion attack**(추론 시점): 입력에 **눈에 안 보이는 perturbation**을 더해 분류를 틀리게 함.  
사람 눈에는 원본과 비슷하지만, 모델은 **금붕어 → 이집션 마우**처럼 완전히 다른 클래스로 예측합니다.

### 공격 분류 (시험 핵심)

| 축 | 옵션 |
|----|------|
| **시점** | Evasion(추론) vs Poisoning(학습 시 backdoor) |
| **접근** | **White-box**(모델+gradient) vs **Black-box**(입출력만) |
| **목표** | **Untargeted**(아무 클래스나 틀리게) vs **Targeted**(특정 클래스로) |
| **제약** | Bounded(ε 이내) vs Unbounded |

### White-box vs Black-box

- **White-box**: 모델 구조·가중치·gradient 접근 → FGSM, PGD
- **Black-box**: 모델 내부 모름, 입출력만 관찰 → finite difference 등으로 gradient **추정** (시간 많이 듦)

### Untargeted vs Targeted

- **Untargeted**: loss를 **maximize** — 정답 클래스에서 **멀어지기만** 하면 됨 (어떤 클래스로 바뀔지는 상관없음)
- **Targeted**: loss를 **minimize** — **특정 타겟 클래스** \(y_{target}\)에 가깝게 (예: STOP 표지판을 **50km**로 인식)

> ⭐ **시험 포인트**  
> **Targeted vs Untargeted**, **Black-box vs White-box** — 교수님이 시험에 개념을 잘 갖고 있으면 된다고 강조.

### FGSM (Fast Gradient Sign Method, 2014)

한 스텝 공격:

\[
x_{adv} = x + \epsilon \cdot \text{sign}(\nabla_x L(x, y))
\]

- **빠르다** — gradient 한 번, closed-form
- 단점: **모든 픽셀**이 바뀌어 salt-and-pepper 노이즈

### PGD (Projected Gradient Descent, 2017)

- FGSM을 **iterative**하게 반복 (BIM과 유사)
- 특정 픽셀에 perturbation이 **너무 누적**되면 **clipping** — 원본 x의 **ε-ball** 밖으로 나가지 않게
- 벤치마크에서 가장 많이 쓰는 **기본 white-box untargeted** 공격
- PPO의 clipping과 **수학적으로 유사** (Trust Region / Constraint Optimization)

### Physical-world attack

STOP 표지판에 adversarial 패치를 **프린트해 붙이면**, 사람 눈에는 STOP인데 자율주행차는 **50km**로 인식 — **Targeted** 공격의 실제 사례입니다.

# 3부. 260714 오전 — Quantization

## 3-1. 왜 Quantization을 배우는가

엣지 디바이스에서는 **메모리·연산**이 부족합니다. Knowledge Distillation, NAS, LoRA, Domain Adaptation 등은 이미 다른 강의에서 다뤘고, 오늘은 **Quantization**에 집중합니다.

딥러닝에서는 수많은 연산이 **더해지기** 때문에, 개별 weight·activation의 해상도가 조금 떨어져도 성능이 크게 안 무너질 수 있습니다.

## 3-2. 숫자 표현과 기본 Quantization

| 형식 | 특징 |
|------|------|
| **FP32** | 1 sign + 8 exponent + 23 mantissa (4 byte) |
| **FP16 / BF16** | exponent·mantissa 비율 조정; BF16은 dynamic range 확대 |
| **INT8 / INT4** | integer + **scale factor** (zero-point 등) |

**Symmetric quantization** (가장 구현 쉬움):

\[
Q = \text{clip}(\text{round}(X / \text{scale})),\quad \hat{X} = Q \times \text{scale}
\]

**RTN (Round-To-Nearest)**: 가장 단순한 baseline.

| | **PTQ** | **QAT** |
|--|---------|---------|
| **시점** | 학습 **후** | 학습 **중** |
| **장점** | 빠름, 재학습 불필요 | 저비트에서 정확도 최고 |
| **단점** | 극저비트에서 성능 하락 | LLM 규모에서는 비현실적으로 느림 |

**Calibration**: PTQ 시 activation 통계 수집 — 대략 **100 sequences, ~200K tokens**.

## 3-3. LLM의 Activation Outlier

Transformer block마다 여러 matmul이 있고, **특정 채널**에 activation이 **비정상적으로 튀는(outlier)** 현상이 있습니다.  
이 outlier 때문에 단순 INT8 quantization이 어렵습니다.

**LLM.int8()**: outlier 채널만 **FP16**, 나머지 **INT8** — mixed precision.

## 3-4. Weight-only PTQ — GPTQ vs AWQ

### GPTQ (2022)

- **Hessian** \(X^TX\) 기반으로 weight를 **column-wise greedy** quantization
- 민감한 weight부터 quantize, 오차를 남은 weight에 전파
- **Cholesky factorization**으로 효율적 업데이트
- 목표: **W4A16** (weight 4-bit, activation FP16)
- OBQ는 element-wise → 메모리·연산 부담 큼 / GPTQ는 **block-wise**

### AWQ (2024, Activation-Aware Weight Quantization)

- Activation은 **FP16 유지**, **weight만** quantize
- Outlier activation 채널에 **diagonal scaling S** 적용 → 해당 weight 채널을 **촘촘한 grid**로 quantize
- S를 **RMSNorm γ**에 fuse 가능
- 목표: **W4**, perplexity degradation ~1% 미만

> ⭐ **시험 포인트**  
> **GPTQ**: Hessian 기반 **greedy column-wise** weight PTQ.  
> **AWQ**: **Activation-aware** diagonal scaling, weight만 4-bit.

## 3-5. Activation + Weight — SmoothQuant vs QuaRot vs SpinQuant

### SmoothQuant (2023) — W8A8

- **Weight와 Activation 둘 다** quantize → **메모리 + 연산 속도** (INT8 GEMM)
- Per-channel diagonal **S**: \(X' = X \cdot S^{-1}\), \(W' = S \cdot W\)
- Outlier를 X와 W 사이에 **α로 분배** (α=0.5 geometric mean 등)
- Activation outlier를 **suppress**, 대신 해당 channel weight variance 증가

### AWQ vs SmoothQuant 철학 차이

| | **AWQ** | **SmoothQuant** |
|--|---------|-----------------|
| **Activation** | FP16 유지 | INT8 quantize |
| **전략** | 어려움을 activation에 몰아 weight 정밀 quantize | Outlier를 X↔W之间 분배 후 **둘 다** quantize |
| **목표** | **메모리** (weight 4-bit) | **메모리 + 연산 속도** (W8A8) |

### QuaRot (2024) — W4A4

- SmoothQuant의 **연장**이지만 목표가 더 aggressive: **W4A4 + KV cache 4-bit**
- Diagonal scaling만으로는 4-bit 극한이 어려움 → **Random Hadamard rotation**
- Orthogonal matrix: \(R^T R = I\) — outlier energy를 채널에 **분산**
- Quantize **직전**에 rotation → activation을 **평평하게**

### SpinQuant (2024)

- QuaRot의 **random** rotation 대신 **learned rotation**
- **Cayley transform** + STE로 학습
- 4-bit에서 QuaRot보다 perplexity degradation **훨씬 적음** (거의 FP16 유지)

### ReSpinQuant (교수님 lab, ICML 2025)

- SpinQuant는 **R1을 모든 layer에서 공유** (residual connection 제약)
- ReSpinQuant: **layer-wise** rotation → 성능 향상, 대신 online transform 비용

> ⭐ **시험 포인트**  
> **SmoothQuant vs QuaRot**: SmoothQuant = **diagonal scaling S**, W8A8. QuaRot = **random Hadamard rotation**, W4A4+KV4.  
> (오후 실습 전 시험 안내 녹취에서 순서가 뒤바뀐 부분 있음 — **오전 강의 기준**이 맞습니다.)  
> **SpinQuant** = **learned** rotation (Cayley).

# 시험 출제 포인트

교수님 260714 오후 안내를 바탕으로 정리했습니다.  
**모든 문제가 설명형**입니다.

## 1일차 — XAI (260626)

1. **Local / Global / Mechanistic** 해석이 **서로 어떻게 다른지**
   - Local: 한 샘플, **Where** (CAM, Grad-CAM, IG, SHAP, LRP…)
   - Global: 레이어·컨셉, **What** (TCAV, 클러스터링)
   - Mechanistic: **레이어 간** 연결, SAE 등
2. **CAM vs Grad-CAM**: 재학습 필요 여부, gradient 기반 weight
3. **로컬의 한계**: Where는 알지만 Why/What은 모름; explanation manipulation
4. **TCAV**: concept vector, directional derivative score

## 2일차 — Uncertainty & Adversarial (260713)

1. **Aleatoric vs Epistemic uncertainty**
   - Aleatoric: 데이터 noise·겹침, irreducible
   - Epistemic: 데이터 부족·모델 uncertainty, 데이터 추가로 완화
2. **Targeted vs Untargeted attack**
   - Untargeted: loss maximize, 아무 클래스나 틀리게
   - Targeted: 특정 클래스로 유도, loss minimize toward \(y_{target}\)
3. **Black-box vs White-box**
   - White-box: 모델+gradient (FGSM, PGD)
   - Black-box: 입출력만, gradient 추정
4. **FGSM vs PGD** (개념): one-step vs iterative + ε-ball clipping

## 3일차 — Quantization (260714)

1. **SmoothQuant vs QuaRot**
   - SmoothQuant: diagonal **scaling**, W8A8, outlier를 X·W에 분배
   - QuaRot: **random Hadamard rotation**, W4A4, KV 4-bit
2. **GPTQ vs AWQ**
   - GPTQ: Hessian-aware greedy weight PTQ, W4A16
   - AWQ: activation-aware scaling, weight만 4-bit, activation FP16
3. **SpinQuant** (보너스): learned rotation, QuaRot 대비 성능
4. **AWQ vs SmoothQuant 철학**: activation quantize 여부, 목표(메모리 only vs 메모리+속도)
