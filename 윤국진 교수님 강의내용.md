# 1부. 260709 오전 — Domain Adaptation의 출발

## 1-1. 왜 Domain Adaptation을 배우는가

윤국진 교수님은 카이스트 시각지능 연구실에서 **Domain Adaptation**과 **Domain Generalization**을 연구합니다.  
오늘·내일 강의는 이 두 주제를 다루며, 둘 다 결국 **Knowledge Transfer** — 기존에 배운 지식을 새로운 상황에 옮기는 문제입니다.

Weekly SL, Continual Learning과 마찬가지로, “처음부터 다시 학습”이 아니라 **이미 있는 지식을 활용**하는 것이 핵심입니다.

> ⭐ **시험 포인트**  
> DA와 DG는 **테크닉이 유사**합니다. DA를 잘 이해하면 DG가 수월합니다. 용어와 loss function의 **목적**을 이해하는 것이 중요합니다.

## 1-2. 슈퍼바이즈드 러닝의 전제와 깨짐

일반적인 딥러닝 학습은 이렇게 진행됩니다.

1. **트레이닝셋**에 입력(X)과 정답(Y, GT)이 있다.
2. 모델 f에 X를 넣고, 예측값과 GT의 차이(**loss**)를 구한다.
3. loss가 줄어드는 방향으로 모델을 업데이트한다.
4. 이 과정을 반복하면 **트레이닝 에러 E_train**이 작아진다.

여기서 암묵적으로 깔린 **가정**이 있습니다.

> 트레이닝 데이터의 **분포 P(X)** 와 테스트 데이터의 **분포 P(X)** 가 **비슷하다**면,  
> 트레이닝에서 잘 맞춘 모델은 테스트에서도 잘 맞출 것이다.

이걸 **In-distribution**이라고 합니다. 같은 데이터셋을 train/test로 나눈 경우가 대표적입니다.

### 수학 공부 비유

미적분 문제만 열심히 풀고 대수·기하는 안 풀었다면, 시험에 미적분이 많이 나올 때는 잘하고, 대수·기하가 많이 나오면 망합니다.  
모델도 마찬가지입니다. **트레이닝 데이터가 많이 몰린 영역**은 잘 맞추고, **적은 영역**은 못 맞춥니다.

## 1-3. Domain Gap — Out-of-Distribution

현실에서는 트레이닝과 테스트의 **분포가 다릅니다**.

| 상황 | 예시 |
|------|------|
| **합성 → 실제** | 시뮬레이터로 학습 → 실제 도로에 배포 |
| **지역 차이** | 미국 도시 데이터로 학습 → 유럽 도시에서 테스트 |
| **기기·환자 차이** | A 병원 MRI로 학습 → B 병원·다른 기기에서 테스트 |
| **제조사 변경** | A 회사 장비 데이터로 학습 → B 회사 장비로 교체 |

이때 테스트 데이터는 **Out-of-distribution(OOD)** 이고, 트레이닝과 테스트 사이의 분포 차이를 **Distribution Discrepancy** 또는 **Domain Gap**이라고 합니다.

트레이닝은 열심히 했는데 테스트에 넣으면 성능이 **엉망**이 됩니다.  
세그멘테이션 맵이 엉뚱한 곳에 찍히거나, 분류가 틀리는 현상이 나타납니다.

> ⭐ **시험 포인트**  
> **Domain Gap** = 트레이닝·테스트 **분포가 다를 때** 성능이 떨어지는 현상.  
> In-distribution(같은 분포) vs Out-of-distribution(다른 분포) 구분을 알아두세요.

## 1-4. 가장 쉬운 해법과 한계

가장 직관적인 해결책은 **테스트 환경의 데이터를 다시 모아서 재학습**하는 것입니다.

하지만 현실에서는:

- 데이터 수집에 **시간·비용**이 많이 든다
- 테스트 환경마다 **매번 재학습**하는 것은 비효율적이다
- 라벨링도 **비용**이 든다

그래서 **Domain Adaptation**이 필요합니다.

> **재학습 없이**, 또는 **최소한의 추가 정보**만으로,  
> 트레이닝에서 배운 모델이 **새로운 분포(도메인)** 에서도 잘 동작하게 만드는 것.

## 1-5. Domain Adaptation 문제 정의

DA 문제는 다음과 같이 정의합니다.

| 구성 요소 | 설명 |
|-----------|------|
| **Source Domain** | 트레이닝에 쓴 데이터. **라벨(Y)이 있다** |
| **Target Domain** | 테스트·적용할 환경의 데이터. **라벨이 없거나 극히 적다** |
| **목표** | Source에서 학습한 모델이 Target에서도 잘 동작하게 |

가장 흔한 가정은 **Covariate Shift**입니다.

- **P(X)** 는 source와 target에서 **다르다** (입력 분포가 다름)
- **P(Y|X)** 는 **동일하다** (같은 X면 같은 Y — 분류·회귀 규칙은 같음)

> ⭐ **시험 포인트**  
> **Covariate Shift**: P(X)는 다르지만 P(Y|X)는 같다. DA의 기본 가정.

### DA의 핵심 목표

Source와 Target의 **feature space를 align(정렬)** 하는 것입니다.  
두 도메인의 특징 분포를 겹치게 만들면, source에서 학습한 분류기가 target에서도 동작할 수 있습니다.

# 2부. 260709 오후 — DA의 전통적·딥러닝 접근

## 2-1. DA의 세 가지 전통적 접근

교수님은 DA를 크게 **세 가지 방향**으로 나눕니다. 각각 **무엇을 맞추려 하는지**가 다릅니다.

### (1) Metric Learning — 거리를 줄인다

**아이디어**: 같은 클래스는 가깝게, 다른 클래스는 멀게.

- Source·Target 데이터를 공통 feature space에 매핑
- **Intra-class compactness**: 같은 클래스끼리 가깝게
- **Inter-class separability**: 다른 클래스끼리 멀게

분포(distribution) 자체를 직접 맞추기보다, **거리(metric) 구조**를 맞추는 접근입니다.

### (2) Subspace Alignment — 서브스페이스를 겹친다

**아이디어**: PCA 등으로 각 도메인의 **서브스페이스**를 구한 뒤, 변환 행렬 M으로 직접 정렬.

- Source 서브스페이스와 Target 서브스페이스를 **겹치게** 만듦
- **Geodesic Flow Kernel** 등은 중간 서브스페이스까지 고려하는 확장

**한계**: 도메인 갭이 크면 한 서브스페이스만으로 정렬하기 어렵습니다.

### (3) Distribution Matching — 분포를 맞춘다

**아이디어**: Source와 Target의 **통계적 분포**를 직접 맞춤.

대표 기법: **MMD(Maximum Mean Discrepancy)**

- 데이터를 **Hilbert space**(고차원 특징 공간)로 올림
- 두 도메인의 **mean(평균)** 이 Hilbert space에서 가까워지도록 학습
- MMD를 최소화하면 분포 차이가 줄어듦

추가 조건으로 **intra-class compactness**, **inter-class separability**를 함께 둘 수 있습니다.

> ⭐ **시험 포인트**  
> DA 3대 접근의 **차이**를 알아두세요.  
> - **Metric Learning**: 거리(같은 클래스 가깝게)  
> - **Subspace Alignment**: 서브스페이스 겹침  
> - **Distribution Matching**: MMD 등으로 분포 자체를 맞춤

## 2-2. TCA vs DIP — Hilbert와 Latent의 순서

둘 다 **변환 행렬 W**를 구해 source·target을 정렬하지만, **어디서 MMD를 최소화하느냐**가 다릅니다.

| 방법 | 순서 | 설명 |
|------|------|------|
| **TCA** (Transfer Component Analysis) | Input → **Hilbert** → **Latent** | 먼저 Hilbert space로 올린 뒤, Latent에서 MMD 최소화하며 W 구함 |
| **DIP** (Domain-Invariant Projection) | Input → **Latent** → **Hilbert** | 먼저 Latent로 투영한 뒤, Hilbert space에서 MMD 최소화하며 W 구함 |

**핵심**: 둘 다 W를 구하지만, **Hilbert projection과 Latent projection의 순서**가 반대입니다.  
시험에서 “TCA와 DIP의 차이”를 물으면 이 **순서 차이**에 집중하세요.

> ⭐ **시험 포인트**  
> **TCA vs DIP**: Input→Hilbert→Latent vs Input→Latent→Hilbert. **순서가 다르다**. W를 구하는 목적은 같음(MMD 최소화).

## 2-3. 딥러닝 시대의 Domain Adaptation

전통적 방법은 **shallow feature**에 적용되었고, 딥러닝에서는 **end-to-end**로 feature extractor + classifier를 함께 학습합니다.

### 기본 구조

```
입력 → Feature Extractor (G) → Feature → Classifier (C) → 예측
```

DA의 목표: Feature Extractor가 뽑은 **feature distribution**을 source·target에서 비슷하게.

### 딥러닝 DA의 대표 기법

| 기법 | 하는 일 |
|------|---------|
| **Deep Coral** | Source·Target feature의 **공분산(covariance, 2차 통계)** 을 맞춤 |
| **DANN** (Domain-Adversarial NN) | Feature extractor vs **Domain Discriminator** — 도메인 구분 못 하게 adversarial 학습 |
| **DAN** | MMD를 딥 feature에 적용 |
| **BatchNorm mixing** | Source·Target 통계를 섞어 domain-invariant feature 유도 |

### DANN / GAN 스타일

- **Feature Extractor**: 도메인을 **구분하지 못하는** feature를 뽑도록 학습
- **Domain Discriminator**: feature가 source인지 target인지 맞추려 함
- **Gradient Reversal**: extractor는 discriminator를 속이는 방향으로 업데이트

> ⭐ **시험 포인트**  
> **Deep Coral** = feature **공분산** 맞춤 (2차 통계).  
> **DANN** = adversarial로 domain-invariant feature 학습.

## 2-4. 최근 DA 트렌드

- **Foundation Model·CLIP** 활용: 강한 pretrained representation으로 domain gap 완화
- **Source-free DA**: Source 데이터 없이, **pretrained 모델만**으로 target에 적응
- Foundation model이 강해지면서 **DA 자체의 중요성**은 다소 줄어드는 논의도 있음

# 3부. 260710 오전 — Domain Generalization의 세계

## 3-1. DA vs DG — 무엇이 다른가

| | **Domain Adaptation (DA)** | **Domain Generalization (DG)** |
|--|---------------------------|----------------------------------|
| **Target** | **특정** target domain이 정해져 있음 | Target domain이 **미지정**, target **데이터 없음** |
| **Target 데이터** | UDA 등에서 target **비라벨 데이터 접근 가능** | Target 데이터를 **사실상 쓰지 않음** |
| **목표** | Source–Target **discrepancy를 줄임** | **여러 source**로 domain-invariant feature 학습 |
| **테스트** | 정해진 target에서 평가 | **보지 못한** 새 domain에서 평가 |

한 줄로: DA는 “**이 target**에 맞추자”, DG는 “**어떤 target이 와도** 견디자”입니다.

> ⭐ **시험 포인트**  
> **DA vs DG**: DA는 target이 정해져 있고 target 데이터(비라벨) 활용 가능. DG는 target 미지정·데이터 없이 여러 source로 일반화.

## 3-2. DG의 목표 — 두 가지를 동시에

DG에서 feature는 **두 가지**를 만족해야 합니다.

1. **Domain invariant**: 도메인이 바뀌어도 feature가 비슷
2. **Class discriminative**: 클래스를 구분할 수 있어야 함

**둘 다 필요**합니다. “모든 시험에서 0점이어도 consistent하다”는 의미 없는 일반화입니다.  
Task(분류·회귀) 성능을 유지하면서 도메인 변화에 강해야 합니다.

## 3-3. DG의 세 가지 접근

### (1) Data Manipulation

- **Augmentation**: 기존 데이터를 변형해 다양한 “가짜 도메인” 생성
- **Generation**: sim-to-real, **diffusion** 기반 synthetic data, CLIP 스타일 생성
- 목적: 학습 시 **더 다양한 분포**를 보게 해 일반화력 향상

### (2) Representation Learning

- **Domain-invariant + task-discriminative** feature 학습
- Kernel method, adversarial training
- **Feature disentanglement**: shared(공통) vs private(도메인별) feature 분리

### (3) Learning Strategy

- **Ensemble**: 여러 모델·도메인 조합
- **Meta-learning (MLDG)**: source domain을 meta-train/meta-test로 나눠 **self-simulation**
- **Group DRO**: 평균이 아니라 **최악 그룹** 손실 최소화
- **Flat minima**: loss landscape에서 평탄한 최솟값 추구

> ⭐ **시험 포인트**  
> DG 3대 접근: **Data manipulation**, **Representation learning**, **Learning strategy**.  
> 각각 대표 기법(augmentation, adversarial, MLDG, Group DRO 등)을 연결해 이해하세요.

## 3-4. Feature Disentanglement

Representation learning의 한 축으로, feature를 **나눕니다**.

- **Shared feature**: 모든 도메인에 공통 — **domain invariant**
- **Private feature**: 도메인마다 다름 — 도메인 특화 정보

Task는 shared feature로, 도메인 구분은 private로 분리하는 그림입니다.

# 4부. 260710 오후 — DG 기법·응용·마무리

## 4-1. MLDG — Meta-Learning for Domain Generalization

**MLDG**는 meta-learning을 DG에 적용한 대표 방법입니다.

**아이디어**:

1. 여러 source domain이 있다 (예: 도메인 A, B, C)
2. 일부를 **meta-train**, 나머지를 **meta-test**로 나눔 (self-simulation)
3. meta-test domain에서도 잘 되도록 **meta-parameter** 학습
4. “가상의 새 도메인”에서 테스트하는 경험을 학습 과정에 넣음

> ⭐ **시험 포인트**  
> **MLDG**: source domain을 meta-train/meta-test로 나눠 **self-simulation**. Meta-learning으로 DG.

## 4-2. MixUp

**MixUp**은 data augmentation이자 regularization입니다.

- 두 샘플 \(x_i, x_j\)와 라벨 \(y_i, y_j\)를 **선형 보간**
- \(\tilde{x} = \lambda x_i + (1-\lambda)x_j\), \(\tilde{y} = \lambda y_i + (1-\lambda)y_j\)
- Multi-class일 때는 **Cross-Entropy**로 interpolated label에 맞춤

목적: decision boundary를 부드럽게, **다양한 중간 샘플**을 보며 일반화.

## 4-3. Group DRO — Worst-Case Generalization

**ERM(Empirical Risk Minimization)** 은 **평균** 손실을 최소화합니다.

**Group DRO(Distributionally Robust Optimization)** 는 **그룹(도메인)별** 손실 중 **최악(worst)** 인 그룹의 손실을 최소화합니다.

- 평균만 좋으면 한 도메인에서만 잘 되고 다른 도메인에서 망할 수 있음
- **Worst-case**를 올리면 모든 도메인에서 최소한의 성능 보장

> ⭐ **시험 포인트**  
> **Group DRO**: ERM은 **평균** 손실 최소 vs Group DRO는 **최악 그룹** 손실 최소.

## 4-4. DIFEX — 주파수(Phase) 도메인

**DIFEX**는 **주파수 영역**에서 domain-invariant feature를 찾습니다.

**핵심 통찰**:

- **Amplitude(진폭)**: 도메인마다 **다를 수 있음** (스타일·조명 등)
- **Phase(위상)**: **구조·형태**를 담음 — 도메인에 덜 민감

→ Phase 쪽 정보를 활용해 **domain-invariant** feature 추출.  
Teacher–student **distillation** 구조로 학습.

> ⭐ **시험 포인트**  
> **DIFEX**: **Amplitude**는 도메인별로 다르고 **Phase**는 구조 보존 → phase 기반 domain-invariant feature. Diffusion 기반 augmentation이 DG에 도움이 되는 이유와도 연결됩니다.

## 4-5. 응용 분야

| 분야 | Domain Gap 예시 | DA/DG 활용 |
|------|-----------------|------------|
| **Medical** | 병원·MRI 기기·환자 cohort 차이 | DG로 기기·환자 간 일반화 |

# 시험 출제 포인트

교수님 강의와 260710 오후 조교 서머리를 바탕으로 정리했습니다.  
**총 4문제 (DA 2 + DG 2)** 예상.

## DA 핵심

1. **문제 정의**: Source(라벨 O) + Target(라벨 X 또는 극소), **Covariate Shift** P(X) 다름·P(Y|X) 동일
2. **3대 접근 차이**
   - Metric Learning: **거리** (같은 클래스 가깝게)
   - Subspace Alignment: **서브스페이스** 겹침
   - Distribution Matching: **MMD**, Hilbert space에서 mean discrepancy 최소화
3. **TCA vs DIP**: **Hilbert ↔ Latent 순서** — TCA는 Hilbert 먼저, DIP는 Latent 먼저. W 구하는 목적은 MMD 최소화
4. **딥러닝 DA**: Deep Coral(공분산), DANN(adversarial·gradient reversal), feature alignment
5. **최근**: Source-free DA, foundation model·CLIP

## DG 핵심

1. **DA vs DG**: DG는 target **미지정·데이터 없음**, 여러 source로 **domain-invariant + class discriminative** feature
2. **3대 접근**: Data manipulation / Representation learning / Learning strategy
3. **MLDG**: meta-train·meta-test **self-simulation**
4. **MixUp**: 샘플·라벨 선형 보간, multi-class CE
5. **Group DRO**: **최악 그룹** 손실 최소 (ERM은 평균)
6. **DIFEX**: **Phase**는 구조·domain-invariant, Amplitude는 도메인별
7. **Diffusion augmentation**: synthetic domain 확장이 DG에 도움이 되는 **이유** 설명 가능해야 함
8. **응용**: Medical(기기·환자), autonomous driving(sim-to-real)

## 체크리스트

- [ ] Covariate shift 정의를 말할 수 있는가?
- [ ] DA 3대 접근(Metric / Subspace / Distribution) 차이를 한 줄로 설명할 수 있는가?
- [ ] TCA와 DIP의 Hilbert·Latent **순서** 차이를 그릴 수 있는가?
- [ ] Deep Coral vs DANN이 각각 무엇을 맞추는지 아는가?
- [ ] DA vs DG 표의 차이(target 지정, target 데이터)를 아는가?
- [ ] DG에서 domain invariant + class discriminative가 **둘 다** 필요한 이유를 아는가?
- [ ] MLDG의 meta-train/meta-test self-simulation을 설명할 수 있는가?
- [ ] Group DRO가 ERM과 어떻게 다른가(평균 vs worst-case)?
- [ ] DIFEX에서 phase vs amplitude 역할을 아는가?
- [ ] Loss function·용어의 **목적**을 이해했는가? (교수·조교 반복 강조)
