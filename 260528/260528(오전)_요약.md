# 260528 오전 강의 요약 — 트랜스포머 계산 복잡도 · Knowledge Distillation · Pruning · Quantization(SmoothQuant) · Attention 근사(Reformer/Linformer/Performer) · CNN-Transformer Hybrid

> 오전 흐름: **트랜스포머 구조 복습과 계산 복잡도(Attention vs Feed-Forward) → Transpose/Softmax/GELU/LayerNorm 오버헤드 → BERT(인코더)/GPT(디코더)/ViT → 트랜스포머=뇌(상관관계 기반 인식) 가설 → embedded(FFN 지배) vs server(attention 증가) → Knowledge Distillation과 DEiT → Pruning 대상의 다양화(weight/head/layer/FFN/token/patch, 2:4 semi-structural, sparse 저장 포맷 COO/CSR/CBR) → Quantization granularity와 I-BERT/outlier/SmoothQuant → Attention 근사(Reformer/Linformer/Performer/Cosformer) → CNN+Transformer hybrid(MobileViT/EfficientFormer/MetaFormer) → FlashAttention/ELSA → 실습(Ollama LLM 추론)**
> 강의 주제: **CNN에서 배운 소프트웨어 최적화 기법(Approximation/Parallelization)이 트랜스포머에 어떻게 적용되는지, 적용 대상이 어떻게 다양해지는지 이해하고, 트랜스포머의 계산 복잡도와 임베디드/서버 환경별 bottleneck의 차이를 파악하는 것**.

# 1. 트랜스포머 계산 복잡도 ⭐

## 1.1 구조 복습

입력 토큰 → Embedding → Positional Encoding → Attention(토큰 간 연관도) → Feed-Forward Network → Softmax 출력.

## 1.2 복잡도 분석 ⭐

토큰 개수 \(n\), hidden dimension \(d_k\), head 개수 \(h\), head별 차원 \(d_k = d/h\)로 둘 때:

**Attention**: \(QK^T\)는 \(n \times n\) 행렬, 각 원소 계산에 \(d_k\)만큼 연산 → matmul은 \(n^2 d_k\). Softmax/scaling도 \(n^2\). 이를 head 수만큼 반복하고 출력에 weight를 곱하면:

\[
\text{Attention complexity} = O(n \cdot d_k^2) \quad (\text{전체 } d_k\text{ 기준})
\]

즉 attention의 matmul은 \(n^2 d_k\) 오더(head로 복원하면 \(n \cdot D_k^2\)).

**Feed-Forward Network**: 처음 4배로 확장 → GELU → 다시 축소. 복잡도:

\[
\text{FFN} = 8 \cdot n \cdot d_k^2
\]

→ 핵심 비교: **attention(\(n^2 d_k\)) vs FFN(\(n d_k^2\))**, 즉 토큰 개수 \(n\)이 큰가 hidden dimension \(d_k\)가 큰가의 차이.

## 1.3 행렬곱 외의 오버헤드 ⭐

GPT-2를 CPU/GPU에서 측정하면 matmul(QK 곱 등)이 약 50%이고, 나머지가 다른 연산이다.

- **Transpose/Reshaping**: 비중이 큼. 연산기를 늘려도 빨라지지 않음(메모리에서 읽어 돌려야 함). transpose를 자동으로 해주는 하드웨어(메모리 컨트롤러)를 넣기도 하지만, 기존 GPU(나노/제온/타이탄)는 하드웨어 지원이 없어 시간이 걸림.
- **Softmax/GELU/LayerNorm**: non-linear 연산이라 시간이 걸림.

비선형 연산 특징:

- **GELU** (Gaussian Error Linear Unit): CNN의 ReLU/LeakyReLU/SiLU(Swish)와 다름. SiLU와 모양이 비슷하나 성능 차이가 있어 트랜스포머에서 GELU를 씀.
- **LayerNorm**: 트랜스포머는 입력 토큰별로 normalize(토큰당 편차 감소). CNN의 BatchNorm처럼 트레이닝 단계에서 필터에 merge할 수 없어(per-token, row 방향) 시간이 걸림.

# 2. 트랜스포머의 응용과 뇌 가설 ⭐

## 2.1 인코더 vs 디코더

| 모델 | 종류 | 특징 |
|---|---|---|
| **BERT** | Encoder-only | 문장 전체를 이해(빈칸 채우기 등). arithmetic intensity 높아 재사용 가능성 많고 가속 잘 됨 |
| **GPT/LLM** | Decoder-only | 토큰을 하나씩 생성. 병렬성 적고 시간이 오래 걸림 |
| **ViT** | Vision Transformer | 컴퓨터 비전용 |

## 2.2 ViT와 뇌 가설 ⭐

ViT: 2D 이미지를 패치로 쪼개 일렬로 만들고(patch embedding + position embedding), 패치 간 attention으로 연관관계를 구해 분류.

- CNN은 receptive field를 점점 확장하지만, **트랜스포머는 처음부터 전체를 본다(large receptive field, inductive bias 없음)**.
- 패치 개수 = \(H \cdot W / P^2\), hidden size 정의.

**뇌 가설**: 뇌는 절대적 값이 아니라 **취합된 정보들의 상관관계**로 인식한다는 가설. 트랜스포머는 정보가 시각/청각/글자 무엇이든 임베딩된 정보 간 상관관계로 인식하므로 뇌를 모사한다고 볼 수 있다 → 생성형 AI의 기본.

## 2.3 embedded vs server bottleneck ⭐

attention(\(n^2 d_k\))과 FFN(\(8 n d_k^2\)) 비교:

| 환경 | 특징 | 지배적 연산 |
|---|---|---|
| **Server** | \(n\)(토큰)을 계속 키움(LLM), \(n > d_k\) 추세 | **Attention** 비중 커짐 |
| **Embedded** | 메모리 제약으로 \(n\)을 못 키움, \(d_k > n\) | **FFN** 지배(심지어 3:7, 2:8) |
| **ViT** | 패치 고정으로 토큰 수 고정 | attention ≈ FFN (둘 다 중요) |

핵심: **많은 연구는 server 향이라 attention을 줄이려 하지만, 임베디드 향에서는 FFN을 줄여야 한다.** Amdahl's law처럼 한쪽만 줄여서는 한계가 있으므로 **bottleneck을 분석**해야 한다.

- 트랜스포머는 메모리 bottleneck이 많아 모델 사이즈(메모리)를 줄이는 게 기본.
- CNN보다 최적화가 어려움: weight precision 편차가 크고, non-linear 연산(softmax/GELU/LayerNorm)이 많음.
- 좋은 소식: 동일 구조를 반복하므로 redundancy가 많음.

# 3. Knowledge Distillation ⭐

## 3.1 기본 원리

같은 역할을 하는 더 작은(student) 모델을 만들고 싶을 때, **teacher 모델의 힌트**로 학습을 돕는다.

- Accuracy는 모델 토폴로지뿐 아니라 **트레이닝 방법**에 크게 좌우됨(같은 모델도 5% 이상 차이).
- **Distillation loss**: 모든 클래스의 softmax probability에서 teacher와 student가 비슷해지도록 하는 loss + 정답(hard label) loss를 조합.

## 3.2 트랜스포머에서의 KD ⭐

teacher와 student가 **어느 수준에서 정보를 주고받을지**가 다양:

- 마지막 클래스 라벨(output level) — CNN에서 주로 쓰던 방식.
- 중간 layer/attention/embedding layer level (individual level distillation).
- **점진적(stepwise/incremental)**으로 줄여나감(DynaBERT 등 — 한 번에 확 줄이지 않고 여러 단계).

## 3.3 DEiT ⭐

ViT는 처음에 트레이닝이 매우 어려웠다.

- **DEiT**: distillation token을 두고, teacher로 (트랜스포머가 아닌) pre-trained CNN을 사용. hard label distillation loss로 ImageNet 데이터만으로 트레이닝 가능해짐 → **ViT의 트레이닝 코스트를 크게 줄임**(모델 구조는 ViT 그대로).
- KD는 모델 구조 자체가 아니라 **트레이닝을 돕는 기법**.

# 4. Pruning ⭐

## 4.1 Pruning 대상의 다양화 ⭐

CNN에서는 주로 필터 프루닝뿐이었으나, 트랜스포머/LLM에서는 대상이 매우 다양해짐:

| 대상 | 설명 |
|---|---|
| **Weight** | unstructural(메모리만 절감, 계산엔 도움 안 됨) / structural(row/column/block) |
| **Head** | 모든 레이어가 12 head일 필요 없음 — 중간 레이어는 single head도 가능 |
| **Layer** | layer-wise pruning(layer drop) |
| **FFN** | embedded에서 비중 70~80% → 가장 효과 큼 |
| **Token/Patch** | 모든 토큰/패치가 중요하지 않음. ViT는 물체 없는 배경 패치 제거 |

## 4.2 Semi-structural / Sparse 저장 ⭐

- **2:4 semi-structural pruning**: 4개당 2개를 0으로, 균등하게 분산. NVIDIA GPU가 가속을 지원해 메모리·속도 둘 다 이득 → **하드웨어를 알아야 하는** 대표 예(hardware-aware pruning).
- Sparse weight 저장 포맷(0의 위치를 따로 저장):

| 포맷 | 설명 |
|---|---|
| **COO** (Coordinate List) | 0이 아닌 값의 (row, col) 좌표 저장 — 가장 직관적 |
| **CSR** (Compressed Sparse Row) | column index + row 전환 포인터로 정보량 절감 |
| **CBR** (Compressed Block Row) / Block Balance | 블록 단위 |

- 0이 많을 때만 이득(0이 적으면 오히려 오버헤드).
- Pareto Optimal 관점으로 서베이 논문의 표를 읽으면, Pareto에 있는 해(SAViT 등)만 의미 있고 나머지는 무용지물.

# 5. Quantization ⭐

## 5.1 Granularity ⭐

| 단위 | 특징 |
|---|---|
| Per-tensor / Per-layer | 전체를 한 scale로 — 하드웨어 단순, 작은 분포는 손해 |
| Per-channel | 가장 정확하나 하드웨어 복잡(NPU의 병렬성 활용 어려움) |
| Per-head / Per-group | head별 분포가 다르면 비슷한 것끼리 그룹으로 묶어 양자화 |

결정 기준은 weight/activation의 **distribution**(편차)을 보는 것 — CNN과 원리가 같음.

## 5.2 주요 연구 ⭐

- **8bit symmetric linear quantization**: 초기 BERT에서 연구.
- **I-BERT**: GELU/Softmax/LayerNorm(non-linear)을 polynomial expansion으로 근사해 integer 연산기로 처리(보통은 FP로). FP32 대비 accuracy drop 거의 없이 NVIDIA T4에서 2배 이상 성능.
- **Outlier quantization**: 큰 값(0.1%)만 32bit, 나머지는 8bit.
- **PTQ vs QAT** 복습:

| | 정의 |
|---|---|
| **PTQ** (Post-Training Quantization) | 학습 후 양자화. calibration dataset 사용. GOBO/ZeroQuant 등 |
| **QAT** (Quantization-Aware Training) | 양자화 상태로 트레이닝해 정확도 보존. fake/simulated quantization(중간값은 FP32) |

- **LLM 양자화 문제**: 모델 사이즈가 커지면 PTQ accuracy가 갑자기 떨어짐(weight distribution 편차 증가, activation 문제).

## 5.3 SmoothQuant ⭐ (실습 주제)

MIT Song Han(Deep Compression의 저자 — pruning+quantization+run-length coding으로 메모리 10배↓, 속도 3~4배↑) 작품.

- 관찰: **activation은 outlier가 커서 양자화 어렵고, weight는 상대적으로 양자화 쉽다.** 하지만 INT8 연산기는 weight·activation 모두 8bit여야 함.
- 아이디어: **Smoothing** — activation의 크기를 줄이고 weight의 크기를 키워 둘의 절대값 편차를 맞춤.
  - scale ratio \(\alpha\)에 대해 activation은 \(1/\alpha\)로 나누고 weight는 \(\alpha\)로 곱해 곱셈 결과는 동일하게 유지(상쇄).
  - 앞단 weight를 키우면 행렬곱 결과 유지를 위해 activation이 줄어듦.
- **Cross-Layer Equalization**(CNN PTQ 기법)과 유사한 발상 — 두 레이어 weight 크기를 balance.
- LLM.int8과 비교: large 모델(175B/176B)에서 SmoothQuant는 FP16 대비 accuracy drop 거의 없음. (LLM.int8은 activation을 FP16으로, SmoothQuant는 INT8 행렬곱 가능, non-linear는 FP.)
- 그 외: PSAQ-ViT(ViT 양자화, Pareto 1등), binary/ternary quantization(BiBERT), pruning+quantization 결합, SpAtten(cascade token/head pruning + progressive quantization).

# 6. Algorithmic Optimization: Attention 근사 ⭐

Attention의 \(O(n^2)\) complexity를 줄이거나 CNN과 결합.

| 기법 | 출처 | 아이디어 |
|---|---|---|
| **Reformer** | Google | **LSH(Locality Sensitive Hashing)**로 연관도 높은 토큰을 같은 버킷에 모아 인접한 것끼리만 attention → \(n^2 \to n\log n\) |
| **Linformer** | Facebook | hidden/중간 dimension을 **low-rank factorization**으로 \(n \to k\) 차원으로 projection |
| **Performer** | Google | softmax를 random orthogonal **feature map**으로 근사 → **행렬곱 순서 변경**(\(K^TV\) 먼저)으로 softmax 제거, \(nd^2 \to nd\) |
| **Cosformer** | SenseTime | softmax를 두 linear 함수의 곱으로 근사 |

**CNN + Transformer Hybrid**:

| 기법 | 아이디어 |
|---|---|
| **MobileViT** | depthwise convolution 기반 MobileNet 중간에 ViT block 삽입 |
| **MobileFormer** | CNN과 Transformer가 병렬로 정보 교환 |
| **EfficientFormer** | iPhone에서 bottleneck이 matmul이 아닌 **reshaping**임을 발견 → reshaping 없애는 아키텍처 + CNN 결합 |
| **MetaFormer / PoolFormer** | attention을 **token mixer** 개념으로 일반화. attention 대신 단순 pooling을 넣어도 ViT에서 동작 → **attention-FFN topology 자체가 중요**할 수 있음 |

기타:

- **FlashAttention**: 전체를 한 번에 하지 않고 **블록으로 나눠** on-chip(캐시) 메모리에 맞춰 attention 수행 → 메모리 오버헤드 감소.
- **ELSA/A3** (서울대 이재욱 교수 연구실): attention의 의미 있는 토큰만 미리 필터링해 softmax/곱셈 연산을 줄임.

# 7. 실습: Ollama LLM 추론

- **Ollama**: llama.cpp를 사용자 친화적으로 감싼 **런타임 계층**. 다양한 하드웨어 지원, GGUF 기반 LLM을 컴파일 없이 실행.
- TensorRT-LLM(NVIDIA 환경 특화)과 달리 범용적. 모델: Llama 3.2 1B.
- 관찰 대상: **Quantization, GGUF, KV cache, context size**가 추론 성능에 미치는 영향.

# 8. 오전 핵심 정리 ⭐

- 트랜스포머 계산 복잡도는 **Attention(\(n^2 d_k\))**과 **FFN(\(8 n d_k^2\))**으로 나뉘며, 토큰 수 \(n\)과 hidden dimension \(d_k\) 중 무엇이 큰가가 핵심이다. matmul 외에 Transpose/Reshaping/Softmax/GELU/LayerNorm 오버헤드가 약 절반을 차지한다.
- **Server는 attention(\(n\) 증가), embedded는 FFN(\(d_k > n\))이 bottleneck** — 임베디드 최적화는 FFN을 줄여야 한다.
- ViT는 패치 임베딩 + 전체 attention(large receptive field)으로 동작하며, 트랜스포머는 "정보 간 상관관계로 인식"하는 뇌 가설과 부합해 생성형 AI의 기반이 된다.
- **Knowledge Distillation**은 teacher 힌트로 작은 student를 학습시키는 기법으로, 트랜스포머에서는 output/layer/attention 등 다양한 수준에서 점진적으로 적용하며, **DEiT**는 CNN teacher로 ViT 트레이닝 코스트를 줄였다.
- **Pruning** 대상이 weight/head/layer/FFN/token/patch로 다양해졌고, **2:4 semi-structural pruning**(NVIDIA 가속)과 sparse 저장 포맷(COO/CSR/CBR)이 중요하다.
- **Quantization**은 granularity(tensor/channel/head/group)를 분포에 따라 선택하며, I-BERT(non-linear를 integer로), outlier quantization, PTQ/QAT, 그리고 **SmoothQuant**(activation outlier를 weight로 옮기는 smoothing)가 핵심이다.
- **Attention 근사**는 Reformer(LSH), Linformer(low-rank), Performer(행렬곱 순서 변경으로 softmax 제거), Cosformer가 있고, MobileViT/EfficientFormer/MetaFormer 같은 **CNN-Transformer hybrid**와 FlashAttention(블록 단위)이 있다.
