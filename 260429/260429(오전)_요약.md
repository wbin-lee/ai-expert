# 260429 오전 강의 요약 — Transformer 마무리 · 사전학습 언어모델 · BERT(Encoder-only)

> 오전 흐름: **Multi-Head Attention 마무리 → Residual / LayerNorm → Encoder-Decoder Attention → Transformer 파라미터 계산 실습 → Coarse-grained 관점(Attention = Neural Architecture Search) → Decoding 복습 → 사전학습 언어모델 등장 → BERT (MLM + NSP)**

## 1. Transformer 잔여 트릭 정리

### 1.1 Attention is All You Need (2017) 트리비아
- 2017년 Google 논문. 저자 중 한 명이 비틀즈 팬 → "All You Need is Love" → "Attention is All You Need"로 작명.
- 트랜스포머에 추가로 들어가 있던 트릭들:
  - **Multi-Head Attention**
  - **Scaled Dot-Product**
  - **Residual Connection**
  - **Add & Layer Normalization**
  - **Encoder-Decoder Attention**

### 1.2 Single-Head Attention 복습 (한 번 더)
- 입력: N개 단어 × D 차원 임베딩 (트랜스포머 원본은 D=512, 강의에선 N=1000으로 가정해 N과 D 구분).
- W_Q, W_K, W_V는 D×D 행렬 (헤드 1개일 때 차원 보존).
- Q, K, V 모두 N × D.
- Q · K^T → N × N (모든 단어 쌍 사이 유사도 = complete graph score).
- Softmax → 분포(N × N).
- 분포 · V → N × D = **Attention output**.

### 1.3 ⭐ Multi-Head Attention (시험 출제 가능)
- 헤드의 인덱스 ℓ = 1…H. 트랜스포머 원본 H = 8.
- 각 헤드의 Q, K, V는 D × (D/H) 차원으로 변환.
  - 512 / 8 = **64** 차원짜리 Query, Key, Value 벡터.
- 계산 자체는 single-head와 동일, 차원만 64로 줄어듦.
- 헤드별 attention output (N × 64)을 H개 만큼 **Concat** → N × D (다시 512).
- Concat은 물리적 결합 → 정보를 섞기 위해 또 다른 D × D 변환 행렬 (W_O / Y / P 등 표기 혼용)을 통과 → 최종 N × D.

> **노테이션 통일은 안 되어 있음**: 같은 행렬을 W_O, Y, P, WO 등으로 표기. 강사 의도적으로 통일 안 함 — robustness 훈련.

### 1.4 🔥 Scaled Dot-Product (시험 출제 가능)
- $\text{Attention}(Q,K,V) = \text{softmax}\!\left(\dfrac{QK^T}{\sqrt{d_k}}\right) V$
- $\sqrt{d_k}$로 나누는 이유:
  - Dot-product는 두 벡터의 element-wise 곱을 다 합치는 것이라 **벡터 사이즈가 클수록 값이 커진다**.
  - Softmax는 지수함수($e^x$)이므로 입력이 크면 값이 폭발 → 분포가 극단적으로 뾰족(0 or 1)해지거나 발산.
  - 그래서 $\sqrt{d_k}$ (=$\sqrt{64}=8$)로 나눠 **분포가 안정적으로 형성**되도록 함.

### 1.5 ⭐ Residual Connection (시험 출제 가능)
- **레이어를 통과하는 라인 + 통과하지 않는 라인** 두 갈래.
- **Backpropagation 관점**: 한 레이어 미분을 건너뛰므로 **Vanishing Gradient 완화** & 손실 지형(loss landscape)을 단순화 → 최적화 용이.
- 초기값이 중요한 이유: non-convex 지형에서 시작점이 어디냐에 따라 학습 난이도가 달라짐.
- **Forward 관점에서 Residual의 역할 (디코더에서 특히 중요)**:
  1. **Encoder-Decoder Attention** 수행 후 디코더 표현이 사라지지 않도록(프랑스어 표현 살아남게) 도움.
  2. **Position Embedding** 정보가 레이어 깊어질수록 희석되는 것을 막아줌.
  3. 최적화(Vanishing Gradient 완화).

### 1.6 Layer Normalization
- 비유: 시험을 쳤는데 누구는 10점 만점, 누구는 100점 만점, 누구는 50점 만점이면 **혼란스럽다** → 같은 스페이스에선 분포가 일관되어야 한다.
- 학습이 진행되면서 분포가 휙휙 바뀌면 그래디언트도 불안정.
- 평균/분산을 이용해 값 안정화 → 그래디언트가 잘 먹히도록.
- (배치 노멀, 레이어 노멀 등 종류 다양 — 이 강의에선 LayerNorm이라는 사실만)

### 1.7 ⭐ Encoder-Decoder Attention (시험 출제 가능)
- Sequence-to-Sequence와 동일한 구조이지만 차이점:
  - **Q는 Decoder에서**, **K, V는 Encoder의 마지막 블록에서** 가져옴.
- 영어→프랑스어 번역 예시:
  - Decoder의 Q는 프랑스어 표현(샤를 만들기 위한 시그널).
  - Encoder의 K, V는 영어 표현.
  - Attention output은 V의 가중합 → **영어 표현**.
  - 프랑스어 정보는 Q로 쓰이고 사라져버림.
- 그래서 **Residual Connection이 프랑스어 표현을 살려내는 역할** (Add: 컨캣이 아닌 "더하기"여도 벡터의 방향 정보는 유지된다).

> **벡터 더하기의 직관**: 스칼라 2+3=5는 흔적이 안 남지만, 두 벡터 더하기는 두 방향이 합쳐진 새 방향 → 흔적이 보존됨.

### 1.8 Sequence-to-Sequence vs Transformer Encoder-Decoder Attention
- Seq2Seq에서는 디코더 표현 sₜ + 인코더 attention output aₜ를 **Concat** 후 linear → 두 표현 모두 유지됨.
- Transformer Encoder-Decoder에서는 **Q는 그냥 증발**, V만 가중합 → Residual로 보강.

## 2. Transformer 파라미터 계산 실습 ⭐ (시험 출제 가능)

### 2.1 문제 세팅
- Embedding size = 10
- Head 수 = 2
- Q/K/V size = 5 (= 10/2)
- Encoder block 수 = 3
- Feed-Forward 중간 차원 = 20
- Bias 없음
- 임베딩 레이어 제외하고 **인코더 블록 총 파라미터 수**?

### 2.2 풀이
| 단계 | 계산 | 파라미터 수 |
|------|------|-------------|
| Q, K, V 변환 (한 헤드) | 10 × 5 × 3개(QKV) | 150 |
| 헤드 2개 (멀티헤드) | 150 × 2 | 300 |
| Concat 후 W_O (10 → 10) | 10 × 10 | 100 |
| **Self-Attention 합계** | 300 + 100 | **400** |
| Add & Norm | (파라미터 없음) | 0 |
| Feed-Forward 1층 (10 → 20) | 10 × 20 | 200 |
| Feed-Forward 2층 (20 → 10) | 20 × 10 | 200 |
| **FFN 합계** | | **400** |
| **블록 1개 합** | 400 + 400 | **800** |
| **블록 3개** | 800 × 3 | **2400** |

### 2.3 정답: **2400**
- ALBERT처럼 블록간 파라미터 공유한다면 800에서 끝남.
- 일반적으론 공유 안 함 → 블록 수만큼 곱함.

### 2.4 왜 알아야 하나?
- **KV 캐시**: 트랜스포머 효율성 / 가속기 / 추론 최적화 얘기에서 항상 등장.
- **LoRA 학습**: 어떤 파라미터가 있는지 알아야 어디에 LoRA를 붙일지 결정 가능.

## 3. ⭐ Transformer를 한 차원 위에서 바라보기 (Coarse-grained)

### 3.1 임베딩/Q,K,V 차원 정리
- 임베딩이 D 차원이면 Self-Attention 통과해도, FFN 통과해도 **항상 D 차원**.
- 트랜스포머 원본: 512.
- BERT: 768, 1024 등.
- 사이사이에서 **Q/K/V 차원만** D/H로 줄어든다.

> **왜 Q/K/V를 줄이나?** 차원 큰 채로 멀티헤드를 하면 W_Q, W_K, W_V가 커져 리소스 폭증. 효율성과 정확도 모두에서 줄이는 게 합리적.

### 3.2 🔥 BERT 분석 논문 — Attention의 정체 (시험 출제 가능)
스탠포드 Christopher Manning 연구실의 BERT 분석 논문 결과 (강사가 MSG 첨가하여 단순화 설명):

| 깊이 | Attention 양상 | 비유 |
|------|----------------|------|
| 초반 레이어 | 다음 토큰 / 옆 단어 attention | **RNN처럼 동작** |
| 중간 레이어 | 주변 단어 / 인접 패치 attention | **CNN처럼 동작** |
| CLS 토큰 / 후반 | 모든 토큰 정보 모음 | **MLP/Fully-connected처럼 동작** |

- 헤드마다도 역할이 달라짐 (co-reference 관계, see-her 같은 것 attention 등).
- **SEP 토큰**에 다른 단어들이 attention을 많이 보냄 → SEP가 **아지트** 역할 (문장 정보 모임).
- 마침표/구두점에도 정보가 모임. 의외로 CLS는 그렇게 높지 않은 경우도.

### 3.3 ⭐ Sterling (시험 출제 가능)
- 헤드 중 일 잘하는 헤드만 골라서 가중합 다시 → **성능 일관적으로 향상**.
- 주로 VLM 같은 곳에서 활용 (언어 모델은 이미 잘해서 불만이 적음).

### 3.4 🔥 Self-Attention = Neural Architecture Search (강사 핵심 메시지)
- 모든 네트워크는 그래프(노드 + 엣지)로 표현 가능.
- **RNN**: 과거→미래로만 연결되는 제한된 그래프 (DAG).
- **CNN**: 로컬리티 외엔 연결 안 함.
- **Transformer Self-Attention**: **Complete Graph** — 모든 노드 간 연결 가능, 엣지 가중치(=attention)가 데이터에 따라 결정됨.
- 즉, 트랜스포머는 **사람이 디자인한 그래프가 아닌, 입력 데이터에 따라 스스로 그래프를 만들어내는 모델**.
- 레이어별·문장별로 다른 attention 패턴이 학습됨 → 다양한 NN 구조가 한 모델 안에 가능.

> **궁극의 메시지**: 트랜스포머는 단순한 모델이 아니라 **데이터에 맞춰 신경망 구조 자체를 학습하는 메타 네트워크**.

### 3.5 파라미터 = 성격 + 지능 (강사 비유)
- 파라미터는 곧 W_Q, W_K, W_V → **Attention을 결정하는 것**.
- Attention 패턴 = 모델이 무엇에 집중하느냐 = **성격 + 지능**.
- 사람의 뉴런 신호로 성격을 직접 못 보지만, "어디에 집중하느냐"로 성격이 드러남.
- 모델별로 dark triad (psychopathy, narcissism, Machiavellianism) 점수가 다르게 나옴.
- **작은 모델 → dark 성향 ↑**, frontier 모델 → 낮음.
- 사이즈 작은 LM은 정서적 지지·챗봇 용도로 쓰기엔 위험.
- Multi-Agent 시대엔 보안 문제(시스템 프롬프트 탈취, jailbreak)가 핵심 이슈.

## 4. Decoding 복습 (오전 빠른 정리)

### 4.1 디코딩이란
- 모델이 학습한 확률 분포에서 **어떤 토큰을 채택할지** 결정하는 알고리즘.
- 학습 단계엔 별로 안 중요했지만 **Inference 단계에선 결정적**.

### 4.2 모델 성능 = 3요소의 조합
1. **학습 데이터 + 모델 구조**
2. **최적화** (SFT vs RLHF 등)
3. **디코딩** (휴리스틱)

### 4.3 디코딩 전략
- **Greedy / Beam Search**: 영어가 틀리거나 일반적인 언어 → 적절히 조정.
- **Random Sampling**: 그냥 뽑으면 엉뚱한 결과.
- **Top-K**: 상위 K개에서 샘플링. 분포 broad면 너무 빨리 컷, sharp면 K가 너무 클 수 있음.
- **Top-P (Nucleus)**: 누적 확률 P 넘을 때까지 후보 모음 → **adaptive**.
- **Temperature**: Scaled dot-product의 스케일 팩터를 인위적으로 조정.
  - **T 높음** → 분포 uniform → uncertainty ↑, **창의력 ↑**.
  - **T 낮음** → 분포 sharp → **정확/정밀**.

### 4.4 기타 개념 (간단 언급)
- **Schedule Sampling**: Teacher Forcing 적용 여부 조절.
- **Dataset Aggregation**: Offline data setting.
- **Retrieve-Augmentation (RAG)**: 모델은 처음부터 말하는 것보다 **이어서 말하기**가 잘됨 → 앞에 맥락 붙여주는 게 자연스러움.
  - **Exposure Bias**: 학습 때 매번 어떤 맥락 다음 단어 예측만 하다 보니 처음 말 시작하는 능력이 떨어진다.

> RAG의 두 가지 시각: (1) 파라미터 내 없는 지식 제공, (2) 모델은 맥락 다음 단어 잘 만드니 맥락 붙여주는 게 좋음.

### 4.5 강화학습 (RLHF)
- 포스트트레이닝에서 자세히 다룸. 일단 패스.

## 5. 사전학습 언어모델 (Pre-trained Language Model) 등장

### 5.1 ⭐ NLP의 변곡점 (시험 출제 가능)
- NLP 성능 추이는 완만하게 향상되다 **변곡점**을 맞이해 급격 상승.
- 이 변곡점을 만든 게 **사전학습 언어모델** (= **Foundation Model**).
- 텍스트뿐 아니라 이미지/음성/그래프/테이블에 다 적용 가능 → 그래서 "Foundation Model"이라 부름.

### 5.2 활용 = Adaptation
- 우리가 원하는 task로 바꾸는 모든 것을 adaptation:
  1. **Prompting** (잘 날리기)
  2. **Agent** (만들기)
  3. **Fine-tuning** (고전적)

### 5.3 🔥 Subword Tokenization (시험 출제 가능)
- Vocab을 500K로 잘라도 모든 단어를 못 다룸 → **OOV** (신조어, "tasty"를 "taaaaasty"로 늘여 쓰기, 오타, "transformify" 같은 신조어).
- 단어 단위 토크나이즈로는 신조어 처리 불가능.
- 해결책: **Subword** (접두사·접미사처럼 쪼개기).
  - 예: "professor" = "pro" + "fessor" (앞에서 알려주는 사람).
- **Word2Vec → FastText**: Subword 기반 사전학습 임베딩 (Facebook).
- LLM에서는 사람이 정한 접두/접미사가 아니라 **통계적으로 자주 발생하는 인접 캐릭터**를 서브워드로 사용.
  - BPE류: 캐릭터 단위 시작 → 인접 빈도 보고 머지 → 원하는 vocab 사이즈까지.
  - "headrun" 같은 일반어는 통째 토큰, "tasty(연장형)"·"transformify"는 여러 서브워드로.
- "##" 표기는 단어 중간/끝에 붙는 서브워드.

> **잡지식**: Claude Opus 같은 모델은 동일 단어에 대해서도 토큰을 더 많이 쪼개도록 설계 → 더 많은 토큰 비용.

### 5.4 사전학습 = "조기 파라미터 세팅"
- Word2Vec(임베딩 레이어 사전학습)도 사실은 사전학습의 일종.
- 그러나 **임베딩 레이어 위의 LSTM/RNN 파라미터는 사전학습 안 됨**.
- Transformer 등장으로 **위쪽 파라미터가 무한히 커질 수 있게** → 위쪽도 사전학습 필요.

### 5.5 ⭐ 사전학습으로 얻는 3가지 (시험 출제 가능)
1. **Representation of Language** (인코더 관점에서 언어 표현 잘 학습).
2. **Probability Distribution** (생성 모델로서 언어 학률 잘 학습).
3. **Optimization 시작점** (초기값을 일반화된 좋은 위치로).

## 6. ⭐ 빈칸 맞추기로 배우는 다양한 능력 (시험 출제 가능)

> "Pre-training으로 무엇을 학습할 수 있는가"의 직관적 예시들. 각 빈칸이 어떤 능력을 요구하는지 보면 LLM이 가지는 지식의 종류가 보임.

| 빈칸 예시 | 요구되는 지식 |
|-----------|-----------|
| "경복궁 Palace is located in **___** South Korea" → "Seoul" | **위치/지형 지식** |
| "I put **___** fork down on the table" → "the / a" | **문법** (관사) |
| "The woman crossing the road checked for cars over **her** shoulder" | **공간적 상식** (시각적 상상) |
| "I went to the ocean and saw fish, turtle, seal, **___**" → 고래, 문어 | **사물 카테고리 / world knowledge** |
| "The value I got for 2 hours = popcorn + drink. The movie was **terrible**" | **풍자/유머/감성** |
| "Iroh was in the kitchen. Zuko stood next to Iroh. Then Zuko left **___**" → "the kitchen" | **추론(reasoning)** |
| "1, 1, 2, 3, 5, 8, 13, 21, **___**" → 34 | **수학(피보나치)** |

> 예전엔 "수학은 언어모델이 못 푼다"고 단정했지만 틀린 예측 — LLM은 **이해가 아닌 정보의 연결을 학습**하는 모델이며, 지금은 수학도 잘 푼다.

## 7. ⭐ Encoder vs Encoder-Decoder vs Decoder (시험 출제 가능)

### 7.1 BERT 트리
- 인코더 모델: **BERT** (와 변형들, 예: RoBERTa)
- 인코더-디코더: **T5**
- 디코더: **GPT 시리즈**
- Claude / Gemini도 결국 GPT와 같은 Decoder-only 계열이며, 변형은 한 문단으로 설명 가능한 정도의 차이.

### 7.2 사전학습 → Fine-tuning의 수식
- $\theta \to \hat\theta$ (사전학습으로 일반화된 파라미터) → 그 위에서 fine-tuning.
- $\hat\theta$는 일반화가 잘 되어 있어 fine-tuning 시 **로컬 미니마 부근에 있을 가능성 ↑**.

### 7.3 인코더 모델 — 언제 쓰나
- 분류 / NLU 위주.
- 양방향 문맥(왼쪽+오른쪽) 모두 활용 가능 → 이해 task에 유리.

## 8. ⭐⭐ BERT — Bidirectional Encoder Representation from Transformer

### 8.1 이름 풀이 (시험 출제 가능)
- **B**idirectional: **양방향** = Encoder를 쓴다.
- **E**ncoder **R**epresentations: 인코더로 표현 학습.
- **T**ransformer
- "Deep" = 레이어 더 쌓음 (12층, 24층).
- 즉 **인코더 블록만 가져와서 더 깊이 쌓고, 사전학습한 모델**.

### 8.2 트리비아
- NAACL 2018 베스트 페이퍼.
- 진짜 임팩트는 베스트 페이퍼가 아니라 산업적 임팩트 — 강사가 2018년 9월 입사 후 10월에 회사가 BERT로 난리.
- 당시 사람들은 트랜스포머 자체를 잘 몰랐는데 BERT 때문에 트랜스포머 스터디 붐.

### 8.3 ⭐⭐ Masked Language Model (MLM) — 시험 출제 핵심
- 입력 문장의 토큰을 가린 뒤 그 토큰을 맞추도록 학습.

#### 마스킹 전략 (시험 출제 가능)
- **15%** 의 토큰을 마스킹.
- 그중:
  - **80%**: 진짜 [MASK]로 교체.
  - **10%**: 랜덤 단어로 교체.
  - **10%**: 원래 단어 그대로.
- 이유:
  - **랜덤 단어 교체** → 웹 코퍼스의 노이즈/오타에 robust해짐.
  - **원본 그대로 유지** → 실제 inference 시엔 [MASK]가 없으므로, 정상 토큰을 그대로 인코딩하는 환경에 노출시켜야 함. ("피자가 들어가서 went가 나오기"와 "피자가 들어가서 피자가 나오기"를 동시에 풀게 함 → 모델이 똑똑해짐).

#### 마스킹 비율의 균형
- 다 가리면 빈칸 맞출 수 없음 (신데렐라가 ___을 흘렸다 → 뭐든 가능).
- 너무 적게 가리면 학습 효율 저하 (한 문장에서 한 번만 loss).
- → Sweet spot = 15%.

### 8.4 ⭐⭐ Next Sentence Prediction (NSP) — 시험 출제 핵심
- 두 문장을 [SEP]로 구분해 입력.
- 두 문장이 실제로 이어지는지 / 아닌지 binary classification.
- [CLS] 토큰의 출력으로 분류.
- 자연스럽게 만들기: **document에서 연속 문장** vs **랜덤 매칭**.
- 학습 데이터 자동 생성 가능 (레이블링 불필요).
- 효과: [CLS]에 문장 전체 정보가 모이도록 훈련됨 → downstream 분류에 유용.

### 8.5 ⭐ BERT의 입력 임베딩 (3가지 합) — 시험 출제 가능
1. **Token Embedding** (xᵢ)
2. **Position Embedding** (pᵢ): **학습되는 임베딩** (Transformer 원본의 sin/cos가 아님).
3. **Segment Embedding**: 문장 A/B 구분용 bias.

### 8.6 BERT 스펙
| 모델 | Layers | Hidden | Heads | Params |
|------|--------|--------|-------|--------|
| **BERT-base** | 12 | 768 | 12 | 110M (1.1억) |
| **BERT-large** | 24 | 1024 | 16 | 340M (3.4억) |

> **숫자 단위 잡지식**: K(천) → M(백만) → B(십억). 100M ≈ 1억. GPT-3가 175B면 1750억 ≈ BERT-base 대비 1100배.

### 8.7 학습 데이터
- **BookCorpus + English Wikipedia** (영어권 정보).
- 위키피디아 = 나무위키 같은 것. BookCorpus = 책 디지털라이저.

### 8.8 학습 자원
- 64 TPU × 4일.
- 한국 기업 당시 GPU 환경에선 1~2개월 걸림.

### 8.9 사전학습 vs Fine-tuning 단계
- **사전학습 1회**, **fine-tuning은 task별로 여러 번**.
- 30개 부서가 다른 task를 원하면 30개 fine-tuning model.

### 8.10 ⭐ Fine-tuning 시 학습되는 부분 (시험 출제 가능)
- BERT 인코더 자체는 사전학습된 것을 가져와서 시작.
- **Classifier (Softmax 레이어)** 는 task별로 클래스 수가 다르므로 **랜덤 초기화**.
- Fine-tuning 시 **인코더 + classifier 둘 다 업데이트**.
- 인코더는 많이 바뀔 필요 없어 학습이 빠름 → fine-tuning이라 부르는 이유.

## 9. GLUE 벤치마크 성능

### 9.1 비교 모델
| 모델 | 설명 |
|------|------|
| **Pre-OpenAI SoTA** | GPT-1 이전 task별 최적화된 LSTM/BiLSTM/RNN |
| **BiLSTM + ELMo + Attention** | 범용 아키텍처 (ELMo는 LSTM 기반 사전학습 LM) |

| **BERT-base / large** | 인코더 사전학습 |

### 9.2 결과 요약 (BERT가 전반적으로 SOTA)
- 일부 task(SST-2 감성 분석)는 갭이 작음 — 단어 frequency만으로도 대충 풀려서.
- CoLA(문법 오류)는 단어 개수로는 못 풀던 task인데 BERT가 처음으로 큰 도약.
- QQP/MRPC(문장 paraphrase)는 원래 쉬워서 갭 작음.
- QNLI(자연어 추론)는 어려운 task인데 BERT가 크게 향상.

### 9.3 ⭐ 인코더 vs 디코더 (NLU 측면) — 시험 출제 가능
- **GPT-1보다 BERT가 NLU에서 우수** → 비슷한 파라미터 수일 때 **인코더가 더 효율적**.
- 그러나 OpenAI는 비영리에서 출발 → 엉뚱한(과학적 실험) 시도를 많이 했고, Google은 상용화 잘 만드는 회사라는 캐릭터가 있음.

### 9.4 Hyperparameter scaling
- Layers ↑, Heads ↑, Hidden ↑ → 성능 ↑.
- 그러나 **2배 키운다고 2배 좋아지진 않음** — Scaling Law 발견은 2020년 GPT-3 시기.

## 10. ⭐ BERT의 다양한 Downstream Task (시험 출제 가능)

> BERT는 토큰별 hidden vector를 잘 뽑는 모델 → classifier 설계의 창의력에 따라 다양한 task에 적용 가능.

### 10.1 Sentence-level Task
- Single sentence: SST(감성), CoLA(문법).
- Sentence pair: MNLI, QQP, MRPC, QNLI, RTE.
- [CLS] 토큰 출력 → classifier.

### 10.2 Token-level Task: NER (Named Entity Recognition)
- **BIO 태그**:
  - B: 엔티티 시작.
  - I: 엔티티 지속.
  - O: 엔티티 아님.
- 7개 클래스: B-Person, I-Person, B-Location, I-Location, B-Org, I-Org, O.
- 예: "President Donald Trump" → O B-Per I-Per.
- 각 토큰에서 7-way softmax.
- **Entity Linking** = NER 결과를 위키피디아 등 KG와 연결 → 자동 하이퍼링크 생성.

### 10.3 ⭐⭐ Extractive Question Answering (시험 출제 핵심)
- SQuAD 벤치마크가 대표.
- 질문 + 문서를 입력, 정답이 **문서 내 span**으로 존재.

#### 설계 방법 (창의력 끝판왕)
- BERT 위에 **두 개의 선형 변환** (768 → 1):
  - **시작 위치 score**: span의 시작점.
  - **끝 위치 score**: span의 끝점.
- 모든 토큰마다 두 종류 스칼라 score 출력.
- Inference: 시작 score 가장 높은 위치 ~ 끝 score 가장 높은 위치를 정답 span으로.
  - 끝이 시작보다 앞이면 휴리스틱으로 보정.

#### 학습
- Ground truth는 **두 개의 one-hot 벡터** (시작·끝 위치).
- 토큰 차원에 대해 **softmax** → 시작 ŷ, 끝 ŷ.
- 두 개의 cross-entropy loss를 더해 최적화.

> **새로운 점**: 기존 BERT는 한 토큰 내에서 softmax(긍정/부정, BIO)였지만, **여기서는 토큰을 가로지르는 softmax** (token-wise softmax over sequence). → BERT 분류 패턴 중 가장 신박.

> 나중에 검색(Retrieval)에서도 비슷한 창의적 softmax 활용이 등장.

### 10.4 ⭐ GPT의 신기함 (보너스)
- GPT는 파라미터를 안 바꾸고도(zero-shot) 모든 task를 풀어냄.
- → **모든 명령에 대응하는 universal "Golden Attention"이 존재**한다는 것을 시사.
- BERT 시대(2018)엔 99.9%가 이런 가능성을 부정. 0.1%(OpenAI 일부)가 그 길을 갔음.

## 13. Mamba & 새 아키텍처 (Q&A 보너스)

- **Transformer가 만능은 아님** — 주식, 의료 등 시계열·연속값엔 적합하지 않음.
- **Mamba**: Recurrent 모델의 후손, **연속(continuous) 값에 대한 RNN-like** 모델. 인간다운 기억을 만들기 위해.
- **NVIDIA Nemotron** = Transformer + Mamba 혼합.
- Mamba는 어렵고 niche, Transformer가 제일 쉽다.

## 14. 한 페이지 요약

### 14.1 ⭐ 시험 핵심 포인트 (오전)

| 주제 | 핵심 |
|------|------|
| **Multi-Head Attention** | D 차원 임베딩 → H개의 헤드, 각 헤드 D/H 차원 Q/K/V → Concat → W_O로 D 복귀 |
| **Scaled Dot-Product** | $QK^T / \sqrt{d_k}$ — softmax 폭발 방지 |
| **Residual Connection** | (1) 그래디언트 vanishing 완화, (2) Position embedding 보존, (3) 디코더에서 Q 정보 살림 |
| **Layer Norm** | 분포 안정화 — "100점/10점 만점 섞이면 혼란" |
| **Encoder-Decoder Attention** | Q는 디코더, K·V는 인코더 (cross attention) |
| **파라미터 수 계산** | Q/K/V × 3 × H + W_O + FFN(2단) → 블록당 합 → 블록 수 곱 |
| **Self-Attention = NAS** | Complete graph로 데이터에 따라 그래프(NN 구조) 자체를 학습 |
| **Subword Tokenization** | 통계적 BPE류 — OOV/신조어 대응 |
| **사전학습 효과 3가지** | 표현 학습 + 확률분포 학습 + 좋은 초기값 |
| **BERT** | **B**idirectional **E**ncoder **R**epresentation from **T**ransformer |
| **BERT 입력 임베딩** | Token + Position + Segment (3개의 합) |
| **MLM 마스킹 비율** | 15% (그 중 80% [MASK], 10% 랜덤, 10% 원본) |
| **NSP** | [SEP]로 두 문장 구분, [CLS]로 binary classification |
| **BERT-base** | 12 layers, 768 hidden, 12 heads, 110M params |
| **BERT-large** | 24 layers, 1024 hidden, 16 heads, 340M params |
| **Fine-tuning** | 인코더(사전학습) + Classifier(랜덤) 둘 다 업데이트 |
| **NER BIO 태그** | B(시작) / I(지속) / O(외부) — 3종 엔티티면 7-way softmax |
| **Extractive QA** | 시작·끝 위치 두 score → 토큰 가로지르는 softmax 두 번 |
| **인코더 > 디코더 (NLU)** | 같은 파라미터일 때 BERT > GPT-1 |
| **RoBERTa** | 데이터·학습량만 늘림, peer review reject되었으나 시대 앞섬 |

### 14.2 BERT 한 줄 요약
```
BERT = Encoder-only + Bidirectional + (MLM 15% + NSP) + Token+Position+Segment Embedding
       사전학습 한 번 → Task별 Fine-tuning (Classifier만 랜덤 초기화)
```

### 14.3 비유 모음

| 개념 | 비유 |
|------|------|
| **Scaled Dot-Product의 √d_k** | 큰 벡터끼리 내적하면 값이 폭발 → softmax가 뾰족해짐 → "음량 줄이기" |
| **Layer Norm** | "10점 만점, 100점 만점 섞여서 채점하면 혼란" → 공통 척도로 통일 |
| **Residual Connection** | 영어→프랑스어 번역에서 디코더의 Q가 사라지지 않게 "남겨두기"; 벡터 더하기는 흔적이 남음 (스칼라 2+3=5는 안 남지만 벡터는 방향 보존) |
| **Self-Attention = NAS** | 사람이 그래프를 디자인한 RNN/CNN과 달리, 트랜스포머는 데이터로부터 그래프 자체를 학습 |
| **Attention = 성격** | "전 여자친구가 사소한 말에 attention 폭발" → 같은 정보라도 무엇에 집중하느냐가 사람과 모델의 성격 |
| **Transformer 이름의 유래** | 옵티머스 프라임처럼 입력 시퀀스를 출력 시퀀스로 변환 |
| **트랜스포머 깊이별 attention** | 초반 = RNN, 중간 = CNN, 후반 = MLP — 한 모델 안에 다 있음 |
| **SEP 토큰** | 정보가 모이는 "아지트" |
| **MLM 마스킹 비율** | 다 가리면 풀 수 없고, 너무 적게 가리면 학습 효율 ↓ — sweet spot 15% |
| **Subword "##"** | 단어의 중간/끝임을 알리는 표식 |
| **사전학습 → Fine-tuning** | "일반 교양" 다 배우고 → 부서별 직무 교육 |
| **RoBERTa** | "그냥 BERT인데 더 오래 공부했어" → 단순해 보이지만 사실은 시대를 앞섬 |
| **BERT 분석 = MSG** | 강사가 직관적으로 단순화·과장한 설명 |
| **파라미터 = 뉴런 신호** | 사람의 뉴런 신호 자체로 성격 못 보지만 "어디 집중하는가"로 성격 드러남 |
