## 1. ⭐ 시험 출제 포인트 (10문제 복습)

### 1.1 RNN — 파라미터 크기와 학습 가능 여부 (시험 1번)
- ⭐ X와 hidden vector, 모델 파라미터(W) 등이 **각각 몇 차원**인지 정확히 파악.
- ⭐ **학습 가능한 파라미터 vs 입력값 / 중간 출력값** 구분이 핵심:
  - X = 입력값 (학습 X).
  - 중간 layer 출력값 (학습 X).
  - 모델 파라미터 W = **최적화 대상** (학습 O).

### 1.2 RNN으로 풀 수 있는 문제 (시험 2번)
- 가변 길이 시퀀스 처리:
  - **Many-to-One**: 시퀀스 입력 → 분류 (예: 감성분석).
  - **One-to-Many**: 입력 → 시퀀스 출력 (예: 캡셔닝).
  - **Many-to-Many**: 시퀀스 → 시퀀스 (예: 번역, 요약).
- ⭐ 시험은 카테고리에 예시를 매핑하는 형태지만, **현업은 반대로** "내 문제가 어떤 카테고리인지" 역추적해야 함.
- 본질을 파악하면 적합한 모델링이 결정 → 인코더 / 디코더의 역할도 결정.

### 1.3 Attention (시험 3번)
- Seq2Seq 모델의 **문제점**(고정 길이 context vector의 정보 압축 손실)을 명확히 설명할 수 있어야 함.
- ⭐ Attention의 출발점 = "Seq2Seq의 정보 압축 한계 극복".
- Decoding step에서의 attention 연산 과정을 그림 + 수식 + 차원까지 설명 가능해야 함.
- ⭐ Attention은 단순 기법이 아니라 **이후 모든 LM의 근간**:
  - 사고 / 수학 / 코딩 / 안전성 / 명령 따르기 모두 **모델이 어떻게 attention하는가**로 해석 가능.

### 1.4 Transformer 파라미터 크기 (시험 4번)
- Encoder-Decoder 구조 전반.
- Q/K/V 변환 파라미터 + Multi-head 구성 + Multi-head attention 합치는 부분의 비율.
- Feedforward layer는 **입력 차원 = 출력 차원** 제약 → 중간 layer만 알아도 파라미터 크기 계산 가능.
- 추천 자료: **Illustrated Transformer** (영문/한국어 블로그) — 전체 그림 이해에 최적.

### 1.5 언어 모델 (시험 5번) — 인코더 / 디코더 / 인코더-디코더
- ⭐ 모델 분석은 **3가지 축**으로:
  1. **Architecture**: 어떤 구조인가.
  2. **Data**: 어떤 데이터를 썼는가.
  3. **Learning**: 어떤 loss / 어떤 학습을 했는가.
- 이 3축으로 BERT / GPT / T5 / GPT-3 모두 표현 가능.
  - GPT-3 = 거대 디코더 + GPT-3 데이터 + LM 학습 (+ 추후 SFT/IT/RLHF).

### 1.6 In-Context Learning (시험 6번)
- ⭐ **Learning ≠ Training**:
  - Training = 파라미터 변경.
  - Learning = 패턴 인식만으로도 learning.
- ICL = 파라미터 변경 없이 예시만으로 task 인식 → learning.
- 발전 흐름: **Parallel Structure → Few-shot → CoT → Stylistic Output** 등 프롬프팅 기법.

### 1.7 IT / RLHF / SFT (시험 7~8번)
- Instruction Tuning 데이터 수집 및 합성 (Super Natural Instructions).
- ⭐ Reward 점수 calibration 어려움 → **상대적 페어 학습**.
- Preference Learning이라 부르기도 함 (용어 그라운딩 X, 다양한 명칭).

### 1.8 RLHF의 부담 — 모델 3개 동시 운영
- PPO 기반 RLHF는:
  1. 학습 모델.
  2. Reference Model (KL term 유지).
  3. Reward Model.
- ⭐ 3개나 메모리에 올려야 → 리소스 부담.
- Reward Model 학습 단계 자체가 **현업 실용주의자에게 부담**.

### 1.9 DPO와 데이터 합성
- DPO: Reward Model 없이 chosen/rejected 페어로 직접 학습.
- 이론적으로 Reward Model을 두는 것과 다르지 않음 (수학적으로 입증).
- ⭐ **데이터 합성 능력이 필수** — preference pair는 기본 데이터에 없는 경우가 많음.
- 신입사원 면접에서 "데이터 합성 능력"을 보는 트렌드.

### 1.10 RLHF의 KL Penalty + Chain-of-Thought (시험 9~10번)
- KL Divergence: 현재 policy가 reference에서 얼마나 벗어났는지.
- Chain-of-Thought 두 번 다루는 문항.

## 2. ⭐ 자기회귀 vs Diffusion Language Model

### 2.1 자기회귀 LM
- 맥락 → 다음 토큰 예측 → 그 토큰을 다시 입력 → 반복.
- 순차적 생성 (auto-regressive).

### 2.2 ⭐ Diffusion Language Model (시험 출제 가능 — 미래 트렌드)
- 자기회귀 X.
- 토큰을 **순서대로** / **하나씩** 생성하지 않음.
- **드문드문 / 병렬적으로** 생성 → 속도 ↑.
- ⭐ **Humanoid Robot의 사고 엔진**에 활용:
  - 깊게 생각 (slow) → 자기회귀 LM.
  - 빨리 생각 (fast) → **Diffusion LM** (로봇의 즉각적 움직임 생성).
- 교과서적으로 많이 다루지 않지만 **다가올 패러다임**이므로 기억할 것.

## 3. ⭐ GPT 시리즈 발전사

### 3.1 GPT-1 → GPT-2
- GPT-1: NLU (텍스트 분류) 위주.
- GPT-2: 데이터 + 파라미터 10배 ↑ → **생성형 AI의 두각**, 단 기계적 느낌.

### 3.2 ⭐ GPT-3와 발견의 시대 (2020.6)
- **In-Context Learning** 능력 발현.
- ⭐ GPT-3가 필요한 이유 = Pre-training + Fine-tuning 패러다임의 **3가지 한계**:
  1. **데이터 수집 부담** — task가 즉흥적/세밀할수록 fine-tuning 데이터 모으기 비용 ↑.
  2. **모델 운영 부담** — 2000개 task별로 거의 비슷한 파라미터 모델 1만 개 운영 비현실적.
  3. **오버피팅 / Spurious Correlation**:
     - 입력-출력 사이의 **바보 같은 correlation**을 학습.
     - 예: 감성분석에서 "봉준호" 또는 "스필버그" → 무조건 긍정.
     - 학습 데이터 bias에 의존 → 실제 환경에서 신뢰 불가.
  4. **(추가) 인간스럽지 않음** — 인간은 적은 데이터로도 task를 풀고, 번역/요약을 따로 학습하지 않음.

### 3.3 ⭐ Scaling Law (2020) → Anthropic 창업 멤버
- Kaplan et al. 2020 — 스케일업하면 능력이 발현된다는 발견.
- ⭐ **Scaling Law 저자들 = 현재 Anthropic 공동창업자들**.
- Dario Amodei (CEO).

### 3.4 2021 — 상용화의 물밑 작업
- 새 모델 별로 안 나오고 **상용화 준비**.
- ⭐ OpenAI 분열:
  - Scaling Law 저자들 vs Sam Altman의 비즈니스 방향 갈등.
  - 안전 장치 부족하다고 판단 → **Anthropic** (인류라는 뜻) 창립.
  - Elon Musk 이탈 → 별도 행보.
  - Ilya Sutskever 후일 이탈해 안전한 AI 스타트업 창립.
- Anthropic 기여 — **Constitutional AI**: 모델이 지켜야 할 헌법(가이드라인) 설정.
- ⭐ "안전한 AI"가 결과적으로 **성능 향상**에도 기여:
  - Claude Code 등에서 inference time에 **Self-Validation / Self-Verification / Self-Reflection** → 성능 ↑.
  - 메타 인지 = 자기 응답의 quality 인지 능력.

### 3.5 ChatGPT (2022.11)
- 5일 만에 100만 사용자 돌파 — 마케팅 없이 입소문만으로.
- ⭐ "단순한 챗봇이 아니라 **인터넷/스마트폰의 발명에 비견되는 사건**".
- 코드 형태로도 출력 가능 → **소프트웨어 대체 / 기계 조정 / 사람의 업무 대체**의 시작점.
- GPT-3.5 = **InstructGPT + ChatGPT**.

### 3.6 GPT-4
- 추산 1조+ 파라미터.
- 멀티모달 (이미지 이해/생성).
- 추론 / 창의력 ↑.
- STT + TTS 결합 → **Agent의 시작점**.

## 4. ⭐ LLM 사용법의 진화 — 프롬프트 엔지니어링

### 4.1 초기 프롬프트 엔지니어링
- 가스라이팅에 가까운 기법:
  - **역할 부여** (You are a helpful assistant).
  - 가이드라인 제공.
  - **맥락 풍부하게 제공** — "맥락의 조건부로 의미가 결정"되는 LM의 본질 활용.
  - "너가 가지고 있는 지식을 많이 생성해라" — 일종의 parametric RAG.
  - **Step-by-step 사고**: 정답까지의 거리를 점진적으로 메꿈.
  - **감정적 호소**: "정말 중요한 일이야".
  - "넌 정말 똑똑한 AI야" 류의 칭찬.
- **Self-Reflection / Self-Verification**: 답변 후 스스로 검증/피드백 반복.

### 4.2 ⭐ 프롬프트 엔지니어링이 사라지는 이유
- 일반적 프롬프팅 기법은 **모델 내부에 내재화**됨.
- 모델이 명령을 받으면 **자동으로 메타 프롬프트 생성**해서 운영.
- → 엔지니어가 직접 적용해야 하는 영역 X.
- 단, **작은 모델**에는 여전히 중요.

## 5. ⭐ Chain-of-Thought 계열 기법 발전

### 5.1 Few-shot CoT (Jason Wei)
- Rationale 포함된 예시 → reasoning 따라하기.
- ⭐ **Emergent Ability**: 100B+ 모델에서만 발현.

### 5.2 Zero-shot CoT — "Let's think step by step"
- 도쿄대 Kojima et al.
- 예시 없이도 한 줄 추가만으로 성능 ↑.
- ⭐ **Therefore the answer is** 같은 prefix로 답을 추출.
- 의료 / 법률 / 로봇 같이 **예시 제공이 어려운 시나리오**에 유용.

### 5.3 ⭐ CoT의 산업 임팩트 — Spurious Correlation 극복
- PaLM + CoT (58%) ≈ Fine-tuned GPT-3 (55%) 수학 벤치마크.
- ⭐ 단 Fine-tuned 결과는 **신뢰 불가**:
  - 학습 데이터의 bias를 잘 활용했을 뿐 → 실 환경에서는 더 낮음.
  - CoT는 **순수한 58%** — 학습 데이터를 안 봤기 때문에.
- → 의심스러운 55%보다 신뢰할 수 있는 58%가 압도적.

### 5.4 ⭐ Self-Consistency (Jason Wei)
- "여러 번 생성 → 다수결" — 매우 단순.
- 성능 +20% 정도 ↑ → 압도적.
- ⭐ **Test-Time Scaling**의 시초.
- 후속 연구: 다수결이 잘 안 되면 **Hallucination 강력한 근거** → Hallucination Detection 기술로 발전.

### 5.5 ⭐ Plan-and-Solve
- CoT를 더 시스템적/체계적으로:
  - **Step 1**: High-level Plan을 CoT로 생성.
  - **Step 2**: 각 Plan을 execute하는 CoT 수행.
- 예: "댄스 학원 20명, 컨템포러리 20%, 나머지 25% 재즈, 나머지 힙합 → 힙합 % 계산" 같은 다단계 문제.
- ⭐ **Claude Code 같은 에이전트의 근간**.

### 5.6 Auto-CoT
- 예시를 자동 생성:
  - 이미 경험한 문제들의 CoT 데이터베이스 구축.
  - 새 문제 들어오면 관련 CoT 검색 → 자동으로 예시 구성.
- ⭐ RAG와 연결 가능, 최근에는 **Memory에 가까움**.
- Agentic Memory: 과거 워크플로우 기록 → 학습 데이터화.

### 5.7 ⭐ Program-of-Thought (PoT)
- CoT의 rationale을 **자연어 → 프로그래밍 언어**로:
  - 자연어로 수학식을 표현하면 토크나이저/엄밀성 문제로 자주 틀림.
  - **Python 코드로 rationale 생성 → Python compiler로 실행** → 정확.
- → 자연어와 프로그램의 장점 혼합.

### 5.8 ⭐ "이젠 더 이상 공부할 필요 없다"
- 강의자 의견: Gemini / ChatGPT 책임자라면 **이미 나온 수백 개 기법을 다 내장**시킴.
- 모델이 명령에 따라 적절한 프롬프팅을 내부에서 자동 선택.
- 엔지니어가 일일이 공부해서 적용할 영역 X.
- **이해도를 위해서만** 다루는 것.

## 6. ⭐⭐ 시스템 1 vs 시스템 2 (Daniel Kahneman)

### 6.1 책 — Thinking, Fast and Slow
- 미국 심리학자 Daniel Kahneman.
- ⭐ LLM 연구자들에게 **영감의 원천**.
- 현재 NVIDIA Robot Foundation Model 등에도 적용됨.

### 6.2 인지 시스템 3가지
| 시스템 | 특징 | 비율 |
|--------|------|------|
| **Perception** | 시각/청각 등 감각 | — |
| **System 1** | 빠르고 충동적/직관적 | 95% |
| **System 2** | 느리지만 심사숙고/노력 多 | 5% (중요한 의사결정) |

### 6.3 ⭐ 딥러닝 모델 = System 1
- 20층짜리 CNN MRI 모델도 인간 뇌 크기에 비하면 작음.
- 마치 의사가 환자 진단 시 **1초 만에 "감기입니다"** 라고 답하는 것과 같음 → 신뢰성 ↓.
- ⭐ 모든 이전 딥러닝의 노력은 **System 1에 가까움**.

### 6.4 ⭐ CoT = System 2의 발현
- LM이 자기 사고를 언어로 펼쳐 더 좋은 답을 도출 → System 2.
- → System 2를 더 발전시킬 여지가 있음.

### 6.5 ⭐ Self-Discover (DeepMind)
- 문제를 보고 어떤 reasoning 전략을 쓸지 **AI가 스스로 결정**:
  1. **Select** — Step-by-step / Plan-and-Solve / Propose-Verify 등 **전략 선택**.
  2. **Adapt** — 그 전략의 구체적 step 설계.
  3. **Implement** — 각 step 실행.
- ⭐ Agent적 — 의사결정을 스스로.

### 6.6 ⭐ Pseudocode CoT (강의자 연구)
- 자연어 CoT는 **구조 부족** → 프로그램 언어보다 표현력 ↓.
- 프로그램 언어 CoT는 **컴파일 오류 발생 가능**.
- 중간형 = **Pseudocode**:
  - 알고리즘 설계 시 사람이 쓰는 중간 메모.
  - 실제 동작 X, 그러나 LM이 **컴파일러처럼 이해**.
- 논문 제목: **"Language Models as Compilers"** for Pseudocode CoT.
- ⭐ Pseudocode 안에서 if/for문 등 구조 + 자연어를 동시에 → 자연어/프로그램 능력 모두 활용.
- ⭐ Self-Discover의 카테고리 결정조차 **pseudocode로 통합** 가능 — 잠재적 무한 전략을 코드로 표현.
- "DeepMind의 Self-Discover를 이긴 논문" — 사실은 인턴이 만든 논문에 비교한 것.

## 7. ⭐ Symbolic Knowledge Distillation (학습 + 추론 강화의 결합)

### 7.1 Knowledge Distillation의 기본
- ⭐ Teacher Model → Student Model.
- 데이터가 모델 학습 ❌ → **큰 모델이 작은 모델 학습 ✓**.
- Hinton 교수 논문 (텍스트 이해 문제).

### 7.2 ⭐ Logit-Based KD (원래 방식)
- Cross-entropy loss에서 정답 y 대신 **Teacher의 확률분포** 사용.
- 예: 사람=0.8 / 고양이(머리띠 때문)=0.2 / 개=0 → bin 정답(1,0,0)이 아닌 실수 분포.
- ⭐ **Dark Knowledge** = 정답이 아닌 영역의 미세 확률에도 정보가 있다는 직관.
  - 우주 = 보이지 않는 암흑 물질이 더 많다는 비유.

### 7.3 ⭐ Symbolic KD
- Teacher가 **확률값 대신 Y 자체를 샘플링**해서 제공 → 학습 데이터 생성.
- Student LM은 그 데이터로 학습.
- ⭐ **데이터를 만들고 그 데이터를 LM이 학습** = 머신-투-머신 학습.

### 7.4 ⭐ Critic Model (데이터 필터링)
- 기계 생성 데이터는 노이즈 있음 → 필터 필요.
- **Critic Model**:
  - Correct vs Incorrect 분류 (NLI 데이터 등으로 binary classification).
  - 사람이 1만 개 정도 라벨링 → 작은 모델 학습.
- ⭐ 사람이 직접 데이터 만들면 **수개월** (cloud sourcing + 검수) vs 기계 4일.
- Black/Green = 노이즈 비율 → Critic Model로 노이즈 ↓.

### 7.5 ⭐ 결과 — Student가 Teacher를 능가 가능
- Student 모델이 GPT-3 같은 Teacher보다 **더 좋은 성능**도.
- ⭐ Critic으로 잘 가공된 데이터가 사람이 만든 데이터보다 더 효과적인 경우도.
- 단, 시뮬레이션 환경 vs 실제 사람 환경 격차는 별개 이슈.

### 7.6 ⭐ 의료 영역 적용 — System 2 Reasoning
- 의료 영상 분석은 전통적으로 특수 랩에서 수십 년간 연구.
- ChatGPT 등장 후:
  - EMR 테이블 + 영상 데이터 → 텍스트화 → LM 입력 → 병 예측 → **더 잘 예측**.
- ⭐ **Rationale 포함**:
  - "70세 노인, MMSE 28점, 뇌의 어떤 부위 수축 관측 → 경도인지장애".
- 양질 데이터를 좋은 모델로 미리 만들어 → 7B 모델에 학습 → **작은 추론 모델**.
- 성능: 추론 모델 > 비추론 모델 일관적으로.
- ⭐ x축 = 학습 데이터 사이즈. **데이터 적을수록 차이 ↑** → System 2 reasoning이 일반화에 유리.

### 7.7 적용 분야
- Healthcare / Software Engineering / Claude Code / 일반 챗봇.
- 7B 모델로도 큰 모델 성능 발현 가능 → **Agent 속도 ↑ / 폐쇄망 운영 가능**.

## 8. ⭐ Data Synthesis 시대 — 논문 한 편급 파이프라인

### 8.1 합성 데이터의 중요성
- 논문 한 편이 보통 **데이터 합성 파이프라인 하나**.
- 예: DPO 학습용 코드 생성 모델 데이터 파이프라인 (강의자 연구실):
  - 데이터 수집 (난이도/다양성 기반).
  - 자동 피드백 생성 + 테스트 케이스 생성.
  - 양질 데이터 → DPO 학습.

### 8.2 ⭐ 문제 풀이의 두 접근
1. **데이터셋 만들기** → 학습.
2. **프롬프트 / Context / Harness 엔지니어링** → 추론.
- 후자가 빠름 → 가능하면 후자.
- 단, 후자가 항상 가능한 건 아님.

### 8.3 ⭐ 강의자 사례 — 박사 vs 영스터
- 미국 구글의 박사팀: "데이터셋 만들어 풀자".
- 새로 합류한 영스터들: "프롬프트로 풀 수 있다".
- ⭐ **데이터셋 준비도 안 됐을 때 영스터들이 프롬프트로 구현 완료** → 박사들 퇴사 → 마이크로소프트 같은 추격조로 이직.
- 특히 **JS/Cursor 능숙한 젊은 친구들이 위험 요소**.
- 일부 박사들은 "3년 남았다"며 두려워함.

## 9. ⭐⭐ LM-Based Agent — 강의자 연구 분야

### 9.1 Agent의 구조
- ⭐ **LM Agent** = LM 중심 + 주변 도구들 (인터넷 검색기 / 계산기 / Excel / PowerPoint / API).
- 모듈 분리 관점:
  - **Environment**: 환경.
  - **Perception**: 환경 관측 (이미지/HTML/접근성 트리 등).
  - **Reasoning + Memory**: 의사결정.
  - **Action**: 출력 (텍스트/API 호출/로봇 조정).

### 9.2 ⭐ Generalist Agent
- ⭐ AGI는 마케팅 용어, 학계는 **Generalist Agent**.
- 강의자 랩 방향:
  - **Test-bed = Web Browser** (단순 검색 X, 실제 브라우저 조작):
    - DC인사이드, 삼성 사이트, 비행기 표 구매, 쇼핑몰 등 다양한 웹사이트 자동 조작.
  - **Single Software → Multi Software → 운영체제** 수준으로 발전.
  - 디지털 월드 → **Embodied Agent**(로봇)로 확장.

### 9.3 ⭐ Web Agent — 비행기 표 구매 예시
- "서울→얼바인 비행기 표 사줘" → 무의 상태에서:
  1. 웹사이트 방문.
  2. 출발지/도착지 입력.
  3. 출발/도착 날짜 입력.
  4. 표 후보 중 선택.
  5. 카드 입력 / 구매.
- ⭐ **키보드/마우스를 사용**, API 호출 X.
- 성능:
  - 사람: 70% (다양한 웹사이트, 모르는 사이트도 있어 100% X).
  - GPT-4 (Agent화 X): **20% 미만**.

### 9.4 ⭐ Long Horizon Task의 어려움
- 액션 수십~수백 step 필요.
- ⭐ 학습 잘 안 됨.
- 해결 접근 3가지:
  1. **Test-Time Scaling**: 트리 구조로 시행착오 + Value Function 평가 → 비용/시간 ↑.
  2. **강화학습**: 작은 모델 사이즈 + 효율성.
  3. **Context / Harness Engineering**: LLM에 도구 모듈 붙이기 → 스타트업이 주로.

### 9.5 ⭐ Test-Time Scaling — Value Function
- VLM(Vision-Language Model)으로 명령-스텝 부합 평가.
- 평가 낮으면 뒤로가기 → 시행착오.
- 비용 ↑ — VLM 자체가 비싸고 트리 탐색은 시행착오 비용 ↑.
- "1분 걸릴 일을 AI가 10분 → 사용자가 못 기다림".

### 9.6 ⭐ 강화학습 — Reward Model 설계의 어려움
- 작은 모델로 평가 모델(Reward Model) 만들면 학습 시간에 적용 가능.
- ⭐ 기본은 **Outcome-based reward** (성공=1, 실패=0):
  - 너무 sparse → 학습 어려움.
  - 처음부터 실패 vs 거의 성공한 실패 동일 페널티.
- ⭐ **Process-based reward**가 필요.

### 9.7 ⭐ Checklist 기반 Reward Model (강의자 랩 핵심 연구)
- 사용자 명령에 대한 **중간 마일스톤(체크리스트)** 추출.
- 각 마일스톤 만족 여부로 부분 점수 산정.
- ⭐ 절차:
  1. 70+ 다양한 웹사이트에서 **바람직한 trajectory** 사람이 수집 (manual).
  2. 그 trajectory를 보고 LLM이 체크리스트 자동 해석.
- 두 가지 모델 산출:
  - **체크리스트 생성기**.
  - **체크리스트 기반 Reward Model**.
- 결과: **3B 사이즈** 작은 reward model로도 강력한 평가 능력 → 학생 발표 후 학교/산업계 큰 관심.

### 9.8 ⭐ Context Engineering — World Model
- LLM은 **미래 예측 능력 부족**.
- 비행기 표 구매 시 비즈니스석 구매 같은 부적절 의사결정 발생.
- ⭐ **World Model**:
  - 상태 + 액션 → 미래 상태.
  - LM과 비슷한 학습 (입력-출력 분리).
- 실험: ChatGPT에게 액션 후 무엇이 발생할지 2지선다 → **50%** (랜덤 수준) → 환경 미이해 입증.
- ⭐ 적용: 에이전트가 액션 후보 생성 → World Model이 미래 시뮬레이션 → 좋은 value 측정 → 안정적 결정.
- CoT가 World Model에 자연스럽게 포함됨.
- 성능: 어퍼바운드(주황) > 현재(노랑).

### 9.9 ⭐ Multi-Software Agent
- Web → 멀티 소프트웨어로 가면 **HTML 없음** → GUI 화면만.
- PowerPoint 같은 익숙한 SW도 버튼 너무 많고 작고 설명 없음.
- 사람의 상식 부족 → 기계는 더 부족.
- **Frontier Lab들도 1개도 제대로 못 만든 수준**.
- ⭐ B2B 시장 (CAD / 과학자용) 잠재력이 큼.
- 한국에서는 강의자 랩 + 삼성(SDS 또는 전자) 정도가 연구 중.

### 9.10 ⭐ Embodied Agent (로봇)
- 디지털 월드 능력 + 물리적 몸 = Embodied.
- 단순 공장 블루칼라 대체 로봇은 LLM 약한 로보틱스 연구자들이 담당.
- ⭐ 더 큰 흐름: **소프트웨어 다룰 수 있는 뇌를 탑재한 로봇** → 화이트칼라 / 골드칼라까지 대체.
- 엔터테이너 로봇 — BTS의 머신 버전 같은 개념.
- 가정 환경 시뮬레이션을 LLM이 **3D 시뮬레이터로 직접 코드 생성**.
- 메모리 / Safety / Guardrail 연구 부족.

## 10. ⭐ Multi-Agent 시스템

### 10.1 협력 형태
- Hierarchical / Cooperative / Competitive 다양.
- ⭐ 핵심은 **정보 교환** — 컴퓨터 사이언스의 모든 문제는 정보 다루기.
- "삼성과 협력하려면 정보를 내놔야 하지만, 모두 내놓으면 안 됨" — **선을 지키는 것이 art**.

### 10.2 ⭐ 정보 보안 vs 업무 능력 — 양립 불가능한 트레이드오프
- 정보 유출 확률 ↑ → 업무 능력 ↑.
- ⭐ AI는 이 art를 **잘 못 함**.
- 사람도 사회적 활동에서 작은 비밀 공유로 라포 형성하는 기술을 누구나 사용.

### 10.3 ⭐ 벤치마크 (강의자 랩)
- 회사 A의 에이전트 / 회사 B의 에이전트 미팅 스케줄 협상.
- 룰: 다른 미팅 일정 (회사 C와)을 직접 언급 X, "다른 일정 있다" 정도로 표현.
- 모델별 성능 (x축: 정보 보안 / y축: 협력 목표 달성):
  - GPT 모델 = 둘 다 높음 (내재 기술 多).
  - QN / LLaMA 같은 작은 오픈소스 = **둘 다 낮음**.
- ⭐ 빅테크 모델조차 100점 X → 다국적 멀티에이전트 시스템에 큰 위험.
- 삼성 내부 멀티에이전트 시스템도 한계 명확.

## 11. ⭐ Human-AI Collaboration

### 11.1 ⭐ 의료 도메인 충격적 결과
- 임상의가 풀기 어려운 문제로 실험.
- 성능: **사람 < AI 단독 < AI + 사람**? **아님** → **AI 단독 > AI + 사람**.
- ⭐ 사람이 들어가면 **AI 단독보다 성능 ↓** — 의사결정 방해.
- AI는 혼자 푸는 훈련만 했지 **사람과 상호작용 훈련은 없음**.
- 비유: "고시원에서 10년 공부한 친구가 회사 입사하면 어떨지 상상해보라" — 지식 ↑ 사회성 ↓.

### 11.2 ⭐ AI의 사회성 한계
- 동조하는 능력 vs 반박하는 능력 별개.
- 빅테크 모델: 균형 OK.
- 작은 모델: 균형 X.
- ⭐ 의료 데이터 학습한 의료 특화 모델 (BioMistral / MedLLaMA 등):
  - **지식 ↑ 그러나 사회성 ↓** → 협력 성능 더 떨어짐.
  - 지식 ≠ 협력 능력.

### 11.3 X-Y 분석
- x축: 모델 단독 성능.
- y축: 사람과 협력 시 정확도 (수용/반박/방어).
- 같은 지식 수준에서도 협력 성향 매우 다양.

- Innovator = AlphaFold 같은 **과학적 발명** 가능.
- 시퀀스-시퀀스가 아닌 **에이전틱한 풀이**로 영역 확장.

- 영스터 승 → 박사들 퇴사 / 추격조 회사로 이직.
- 강의자: "AI에 연결되면 여러분의 영역은 없고, 매년 적응해야 한다."

### 13.5 ⭐ Reasoning ≠ Agent (관계)
- Reasoning = 단일 턴 문제 풀이 (수학/코딩).
- Agent = 다단계 액션, 한 액션마다 reasoning.
- ⭐ Agent는 **Reasoning 위에 만들어짐** — 짜장면(reasoning) 위 해물(agent) 비유.
- 별개 기술 X, 누적 발전.

### 13.6 ⭐ "흐름을 따라 적응"
- 강의자 진로: 텍스트 이해 → 챗봇 → 프롬프팅 → 명령/코드 능력 → 멀티모달 → 추론 → 도구 사용 Agent → Software Agent.
- 매년 모든 게 바뀜.
- 일주일 전에 나온 논문 보고 1년 뒤를 예측.
- 학생들은 **5~6년 뒤** 잘 될 기술을 지금부터 시작해야.

|------|------|
| **RNN 파라미터** | X / W / 학습 가능 vs 입력값 분간 |
| **RNN 카테고리** | Many-to-One / One-to-Many / Many-to-Many + 예시 |
| **Attention의 동기** | Seq2Seq 정보 압축 한계 → 그림+수식+차원 |
| **Transformer 파라미터** | Q/K/V + Multi-head + FFN 입출력 동일 |
| **언어 모델 3축** | Architecture / Data / Learning |
| **ICL** | Learning ≠ Training, 패턴 인식만으로도 learning |
| **Pre-training 한계** | 데이터 부담 + 모델 운영 부담 + Spurious Correlation + 인간스럽지 않음 |
| **PPO 부담** | 모델 3개 동시 메모리 (Reference + Policy + Reward) |
| **DPO** | Reward Model 없이 페어로 직접 학습, 데이터 합성 능력 필수 |
| **CoT (Few-shot)** | Rationale 포함 예시, emergent ability 100B+ |
| **CoT (Zero-shot)** | "Let's think step by step" |
| **CoT의 산업 임팩트** | Spurious Correlation 극복, 신뢰할 수 있는 성능 |
| **Self-Consistency** | 다수결, Test-Time Scaling 시초 |
| **Plan-and-Solve** | 계층적 CoT, Claude Code 근간 |
| **Auto-CoT** | 검색-기반 자동 예시, Memory 개념 |
| **PoT (Program-of-Thought)** | 자연어 → 프로그램 언어 rationale |
| **시스템 1 vs 2** | 빠른 직관 vs 느린 심사숙고, CoT = System 2 |
| **Self-Discover** | Select-Adapt-Implement 전략 자기결정 |
| **Pseudocode CoT** | 자연어/프로그램 장점 결합 (LM-as-Compiler) |
| **Symbolic KD** | 데이터 합성 + Critic Model |
| **Generalist Agent** | AGI = 마케팅, 학계는 Generalist |
| **Long Horizon Task** | Outcome reward sparse → Process reward (Checklist) |
| **World Model** | 액션 후 미래 예측, LM의 약점 |

### 14.2 전체 구조 도식
```
[시험 출제 9~10문제]
RNN → Seq2Seq → Attention → Transformer → 언어모델
  → ICL → Instruction Tuning → RLHF / DPO

[자기회귀 vs Diffusion LM]
빠른 사고 (Diffusion) + 느린 사고 (Autoregressive) = 휴머노이드 로봇 사고 엔진

[GPT 발전사]
GPT-1 (NLU) → GPT-2 (생성) → GPT-3 (ICL) → InstructGPT/ChatGPT (정렬) → GPT-4 (멀티모달)

[프롬프팅 발전]
초기 PE → CoT → Self-Consistency → Plan-and-Solve → Auto-CoT → PoT
                                          ↓
                                  Self-Discover (전략 자기결정)
                                          ↓
                                  Pseudocode CoT (구조 + 자연어)
                                          ↓
                          Symbolic KD (학습 + 추론 결합)
                                          ↓
                            의료 / SE / Claude Code 응용

[LM-Based Agent]
Web Agent → Software Agent → Embodied Agent (Robot)
   ↓
Long Horizon Task 해결:
  Test-Time Scaling / 강화학습 / Context Engineering
   ↓
Checklist 기반 Reward Model (강의자 랩)
World Model (미래 예측)

[Multi-Agent / Human-AI]
정보 보안 vs 업무 능력 트레이드오프
의료 도메인: AI 단독 > AI + 사람 (사회성 부족)

[Self-Evolving]
환경에서 스스로 발전, 사용자 로그가 기업 자산
→ 특이점 (Singularity) 도전
```

### 14.3 비유 / 직관 모음

| 개념 | 비유 / 직관 |
|------|-----------|
| **Diffusion LM의 의미** | "휴머노이드 로봇은 빠른 사고와 느린 사고가 둘 다 필요하다" |
| **Spurious Correlation** | "감성분석에서 봉준호/스필버그 보이면 무조건 긍정" |
| **System 1** | "20층 CNN MRI 모델도 1초 만에 '감기입니다'와 같다" |
| **System 2** | "CoT를 통해 모델이 자기 사고를 펼치는 능력" |
| **Self-Discover** | "AI가 어떤 reasoning 전략을 쓸지 스스로 결정" |
| **Pseudocode CoT** | "자연어와 프로그래밍의 중간 — 사람이 쓰는 알고리즘 메모" |
| **Knowledge Distillation** | "교사 모델 → 학생 모델, 데이터가 아닌 모델이 모델을 가르친다" |
| **Dark Knowledge** | "정답이 아닌 영역에도 미세 확률 = 우주의 암흑 물질" |
| **Symbolic KD** | "데이터 합성 + Critic Model로 사람보다 빠르고 깨끗한 데이터" |
| **Web Agent** | "비행기 표 구매를 키보드/마우스로 무의 상태에서 끝까지" |
| **Long Horizon = Sparse Reward** | "성공만 1점, 실패 0점이면 학습이 막힌다" |
| **Checklist Reward** | "마라톤 중간 표지판처럼 마일스톤마다 점수" |
| **World Model** | "맥락에서 다음 단어 = 상태/액션에서 다음 상태" |
| **Multi-Agent 정보 보안** | "삼성과 협력하려면 정보 내놔야 하지만 다 내놓으면 안 됨" |
| **AI 사회성 부족** | "고시원에서 10년 공부한 친구가 회사 입사하면 어떻겠나" |
| **Innovator** | "AlphaFold처럼 과학적 발명 — Reasoning > Agent > Innovator" |
| **Microhard** | "앱이 없는 핸드폰 — SW를 그때그때 동적 생성" |

### 14.5 LM-Based Agent 연구 핵심 발견 (강의자 랩)

```
[연구실 진화]
텍스트 이해 → 챗봇 → 프롬프팅 → 명령 수행 / 코드 → 멀티모달
   → 추론 → 도구 사용 Agent → Software Agent → Generalist Agent
                                                       ↓
                            Web → Multi-SW → 운영체제 → Embodied
                                                       ↓
                            Long Horizon Task (성능 20%)
                                                       ↓
              [3가지 해결]
              (1) Test-Time Scaling (트리 + Value Function, 비싸다)
              (2) 강화학습 (작은 reward model)
                    ↓
                  Checklist 기반 Reward Model (3B로 강력한 평가)
                    ↓
                  사람 + LLM 반자동 데이터 합성 파이프라인
              (3) Context Engineering
                    ↓
                  World Model (액션 후 미래 예측)
                    ↓
                  CoT 결합 액션 평가
                                                       ↓
              [Multi-Agent]
              - 협력의 본질 = 정보 교환의 art
              - 정보 보안 vs 업무 능력 트레이드오프
              - 사이즈 작은 오픈소스 모델: 둘 다 낮음
                                                       ↓
              [Human-AI Collaboration]
              - AI 단독 > AI + 사람 (사회성 부재)
              - 의료 특화 모델: 지식 ↑ 사회성 ↓
                                                       ↓
              [Self-Evolving Agent]
              - 환경에서 스스로 발전
              - Parametric 지능 + Non-Parametric DB 지능
              - 사용자 에이전트 사용 기록 = 기업 자산
              - 특이점 (Singularity) 도전
```

### 14.6 핵심 발견 사이의 연결
```
시험 복습 (RNN → Attention → Transformer → LM → IT → RLHF/DPO)
   ↓ 정렬 단계 → 추론 시대로 이동
ICL (예시로만) → CoT (자기 사고) → Self-Consistency (다수결) → Plan-and-Solve → PoT
   ↓ "외부에서 도와주기"
시스템 2 발현
   ↓ "AI가 스스로 결정"
Self-Discover / Pseudocode CoT
   ↓ "학습으로도 추론 능력 강화"
Symbolic Knowledge Distillation (큰 모델로 데이터 생성 → 작은 모델 학습)
   ↓ 의료 도메인 추론 모델
LM-Based Agent (행동까지 확장)
   ↓ Long Horizon
Process-based Reward (Checklist) + World Model
   ↓ Multi-Agent
정보 보안 vs 업무 능력의 art
   ↓ Human-AI
사회성과 협력 능력의 별개 학습
   ↓
Self-Evolving Agent → 특이점
```
