## 2. 소리(Sound)의 물리적 본질

### 2.1 소리가 만들어지는 원리
- 횡경막이 호흡을 내뱉어 **성대를 진동**시키고, 그 진동이 공기 분자를 **압축(compression) ↔ 팽창(rarefaction)** 시키며 고막에 전달된다.
- 신호처리에서는 압축 시점을 **양수**, 팽창 시점을 **음수**로 표현 → **시간에 따른 진폭 그래프(주기함수 형태)** 가 된다.

### 2.2 사인파(Sine Wave)의 3대 파라미터
| 용어 | 의미 | 청각적 영향 |
|------|------|-------------|
| **Amplitude (진폭)** | 신호가 위·아래로 얼마나 흔들리는가 | **소리의 크기 (Volume)** |
| **Period (주기)** | 한 번 진동하는 데 걸리는 시간 | (Frequency의 역수) |
| **Frequency (주파수)** | 1초당 진동 횟수, 단위 Hz | **음의 높낮이 (Pitch)** |
| **Phase (위상)** | 사인파의 시작 시점 | **청각적으로 거의 영향 없음** → 당분간 무시 |

> 직관 : 남성 목소리(저음) → 성대가 느리게 진동, 아이 목소리(고음) → 빠르게 진동.
> 440Hz 사인파 = 1초에 440번 진동 = 음계 "라" 정도.

### 2.3 가청주파수 (Audible Frequency Range)
- 남성 목소리 : 100~150Hz, 여성 : 200~300Hz, 아이 : 300~400Hz.
- 사람의 귀는 **저주파에 민감하고 고주파엔 둔감**. 나이가 들수록 들을 수 있는 최대 주파수가 감소(노화).
- 🔥 **(중요)** "사람은 낮은 주파수 위주로 말하고 듣는다"는 개념은 **뒤에서 멜 스펙트로그램의 동기**가 된다. 꼭 기억할 것.

## 3. 디지털화 (Analog → Digital Conversion)

컴퓨터는 연속(continuous) 신호를 직접 저장 못 함 → 파이썬 리스트/넘파이 배열 형태로 바꿔야 함. 미술관의 **점묘화** 처럼 점을 무수히 찍어 표현.

### 3.1 Sampling — 시간 축 이산화
- **1초 동안 점을 몇 개 찍을지** = **Sample Rate (샘플링 레이트)**, 단위 **Hz**.
- 통상값 : 통화(저품질) 8kHz, 일반 16kHz, 음악 44.1kHz, 고품질 96kHz.
- 강의 기본값 : **16kHz** (1초에 16,000개 점).

> ⭐ **시험 출제 포인트 — Sampling Rate vs Frequency 혼동 금지**
> 둘 다 단위가 Hz지만 의미는 전혀 다름.
> - **Frequency** : 단위 시간당 주기함수 반복 횟수
> - **Sampling Rate** : 단위 시간당 신호에 찍는 점의 개수
> 항상 use case에 맞게 구분해 사용해야 함.

### 3.2 Aliasing (에일리어싱)
- 점을 너무 적게 찍으면 → 원 신호 복원 불가 → 이상한 신호로 보임.
- 예시 : 빠르게 도는 프로펠러를 느린 카메라로 찍으면 천천히 도는 듯 보이는 현상.
- 🔥 **나이키스트-섀넌 이론**의 핵심 메시지 : "**점을 충분히 많이 찍어야 신호를 복원할 수 있다**". 정확한 공식은 외울 필요 없고 직관만 알면 된다.

### 3.3 Quantization — 진폭 축 이산화
- 진폭값은 항상 **[-1, 1] 범위로 정규화**된 후 저장.
- [-1, 1] 구간을 N등분(보통 **16-bit = 2^16 등분**)해서 그 중 하나로 매핑.
- bit가 적으면(예: 4-bit) 지지직 잡음 발생 (정보 손실).
- ⚠️ **헷갈리지 말 것** : 샘플링은 시간축 이산화, 양자화는 진폭축 이산화. 서로 다른 개념.

### 3.4 컴퓨터에 저장된 오디오 파일의 메타데이터
- 샘플 속도(Sample Rate), 샘플 크기(Bit depth), 채널 수(mono/stereo) 등이 함께 저장된다.
- 파이썬에서 오디오를 로드하면 항상 **(waveform 1D 배열, sample_rate)** 두 값이 함께 리턴.

## 4. Resampling — 모델 갈아끼울 때 가장 자주 하는 실수

### 4.1 왜 필요한가
- 학습 모델마다 요구하는 sample rate가 다르다.
- 예 : Whisper는 16kHz로 학습 → 44.1kHz 오디오 그대로 넣으면 성능 박살.
- 🔥 **(시험 출제, 실무 핵심)** 모델 사용 전 **반드시 입력 오디오를 모델 학습 sample rate에 맞춰야** 한다. 이미지의 resize와 같은 개념.

### 4.2 Down-sampling vs Up-sampling
| 연산 | 의미 | 직관적 방법 |
|------|------|-------------|
| **Down-sampling** | 점 개수 줄이기 (예: 16k → 8k) | 점을 듬성듬성 다시 찍음 |
| **Up-sampling** | 점 개수 늘리기 (예: 16k → 44.1k) | 점 사이에 점 추가 (interpolation) |

### 4.3 정보 손실의 비가역성
- **샘플링 과정은 비가역적**. 한 번 down-sampling한 뒤 다시 up-sampling해도 원본 음질 복원 불가.
- 이 영역을 AI로 해결하려는 분야가 **Audio Super-Resolution** (삼성에서도 연구 중).

## 5. 실제 음성 신호의 복잡성 — 사인파 ≠ 실제 음성

### 5.1 Voiced Sound vs Unvoiced Sound
| 유형 | 정의 | 예시 |
|------|------|------|
| **Voiced (유성음)** | 성대가 진동하며 발음 | "아", "이", "우" 모음, "바" |
| **Unvoiced (무성음)** | 성대 진동 없이 공기 흐름만 | "스", "크" 등 파열음·치찰음 |

### 5.2 Harmonics (배음/하모닉스)
- 같은 진폭·같은 음정이라도 **음색(Timbre)이 다른 이유** = 사인파 구성이 다름.
- 실제 사람·악기 소리 = **가장 낮은 주파수 F0**와 그것의 **자연수배 주파수(2F0, 3F0, 4F0, …)** 인 사인파들의 **중첩**.
- 각 자연수배 사인파의 **상대적 진폭**이 음색을 결정.

```
복잡한 신호 = A1·sin(2π·F0·t) + A2·sin(2π·2F0·t) + A3·sin(2π·3F0·t) + ...
```

### 5.3 Formant (포먼트)
- 하모닉스 진폭들을 막대 그래프로 그리면 **볼록 솟아오르는 지점** 들이 생긴다.
- 이 볼록 위치 = **포먼트**. 사람 발음(/a/, /i/, /u/ 등)을 결정.
- 강사가 강조 : "**포먼트와 하모닉스 자체를 외울 필요는 없다**. 핵심은 복잡한 신호를 사인파들로 분해하면 음색·발음을 유추할 수 있다는 것."

> 🔥 **(시험 출제 포인트)** 음성 AI에서 웨이브폼을 직접 쓰지 않고 **주파수 도메인으로 변환**하는 가장 큰 이유 = 사인파의 진폭 관계만 알면 음색·발음을 유추할 수 있기 때문.

## 6. Fourier Transform (푸리에 변환)

### 6.1 직관 — 프리즘 비유
- 백색광을 프리즘에 통과시키면 빨주노초파람보로 분해.
- 마찬가지로 복잡한 음성 신호를 **다양한 주파수의 사인파들로 분해**하는 연산이 푸리에 변환.

### 6.2 계산 직관 — "곱하고 적분"
1. 미스터리한 신호 x(t)에 다양한 주기의 사인파를 하나씩 곱한다.
2. 곱한 결과를 적분(면적 합산)한다.
3. 결과 해석:
   - **큰 값** → 그 주파수 성분이 신호 안에 있다.
   - **0에 가까움** → 양음이 상쇄 → 그 주파수 성분이 없다.

### 6.3 분석 범위와 간격
- **최대 분석 주파수** : `f_s / 2` (Nyquist limit).
  예) 16kHz 신호 → 최대 8kHz까지만 분석.
- **주파수 간격** : `f_s / N` (N = 샘플 개수).
  예) 16kHz, 1600 샘플(0.1초) → 10Hz 간격으로 0Hz부터 8kHz까지 분석.

### 6.4 출력의 복소수 처리
- 푸리에 변환 결과는 **복소수(complex number)**.
- 복소수는 직접 분석 어려움 → **Magnitude(크기)** 만 사용.
  ```
  |F(ω)| = √(Real² + Imaginary²)
  ```
- **Phase(위상)** 는 청각적으로 중요하지 않으므로 보통 버림.

### 6.5 프리에 변환의 한계 — 시간 정보 손실
- 1D waveform → 1D 주파수 스펙트럼으로 변환.
- 즉, **전체 구간의 주파수 평균** 만 알 수 있고, **특정 시점에 어떤 발음을 했는지는 알 수 없다**.
- 그러나 사람의 발음·음색은 **시간에 따라 계속 변하기 때문에** 시간 정보가 반드시 필요 → **STFT** 등장.

## 7. Short-Time Fourier Transform (STFT)

### 7.1 핵심 아이디어
> 전체에 한 번 푸리에 변환하는 대신, **짧은 구간(window)으로 쪼개서 각각 변환** 한다.

```
긴 신호 → [세그먼트1][세그먼트2][세그먼트3]... → 각 세그먼트마다 FT → 시간×주파수 2D
```

### 7.2 Spectral Leakage 문제
- 신호를 두부 썰듯이 뚝 자르면 → 잘린 끝에서 **갑작스러운 0 → 진폭 급변** 발생.
- 급변은 곧 **고주파 성분** → 원하지 않는 주파수가 분석 결과에 나타남(스펙트럴 리키지).

### 7.3 Hann/Hanning Window — 부드러운 절단
- 양 끝을 0으로 부드럽게 눌러주는 **종 모양 윈도우** 사용.
- 가운데는 그대로, 양쪽 끝으로 갈수록 진폭이 0에 수렴.
- 결과 : 고주파 누설 사라짐. 다만 양끝 정보 손실은 감수.
- 청각 효과 : 오디오에 윈도우를 적용하면 **볼륨이 커졌다 작아졌다 반복** 하는 바이브레이션 같은 효과.

### 7.4 STFT의 3대 하이퍼파라미터
| 파라미터 | 의미 | 통상값 (TTS 기준) |
|----------|------|-------------------|
| **n_fft** | 한 윈도우의 샘플 길이 (= 한 번에 보는 구간 크기) | **1024** |
| **win_length** | 실제 윈도우 길이. 보통 n_fft와 같게 설정 | 1024 |
| **hop_length** | 다음 윈도우로 **이동하는 거리** (Overlap이 아님!) | **256** |

> ⚠️ 🔥 **(시험 출제 주의)** **hop_length는 "겹치는 길이"가 아니라 "이동하는 거리"** 다.
> n_fft=1024, hop_length=256 → 75%(=768/1024) 중첩이 발생.

### 7.5 왜 Overlap을 두는가
- 윈도우를 나란히 (겹침 없이) 자르면 **경계 부분의 이벤트(발음 변화·화자 전환)** 를 놓칠 수 있다.
- Overlap을 두어 누락 없이 분석하기 위함.

## 8. Linear Spectrogram

### 8.1 출력 형태
- STFT 결과는 **2D 데이터** (시간축 × 주파수축).
- 가로 축 : 시간 프레임 (윈도우 인덱스).
- 세로 축 : 주파수 빈 (0Hz ~ f_s/2).
- 각 픽셀 값 : 해당 시간·주파수에서의 사인파 진폭(Magnitude).

### 8.2 세로 축 길이 유도
세로 길이 = `n_fft / 2 + 1`
- n_fft = 1024 → 세로 길이 = **513**.

> ⭐ **시험 출제 포인트** : Linear Spectrogram의 세로 길이 = `N/2 + 1`. **513은 너무 커서 비효율적** → 멜 스펙트로그램의 동기.

### 8.3 Linear Spectrogram의 시각적 해석
- 모음 발음(예: "바") → 가로로 평행한 선들이 일정 간격으로 (= **하모닉스**).
- 무성음(예: "파"의 ㅍ 부분) → 전 주파수에 뿌연 노이즈처럼 퍼짐.
- 박수 → 음정 없으므로 전 영역에 뿌연 형태.
- 휘파람 → 음정 있으므로 선 형태.

### 8.4 Phase Spectrogram
- Phase 정보로 그림을 그릴 수도 있지만, **사람도 모델도 해석 어려움**(가우시안 노이즈처럼 보임) → 거의 사용하지 않는다.

### 8.5 왜 그대로 쓰지 않는가
1. **세로 차원이 너무 큼 (513)** — 인공지능 처리에 비효율.
2. 위쪽(고주파) 대부분이 **검고 정보가 없음** — 사람 목소리는 저주파에 모여있기 때문.

## 9. Mel Spectrogram — 최종 목표

### 9.1 동기 (Motivation)
- 사람의 달팽이관은 바깥쪽(고주파)·안쪽(저주파) 영역으로 나뉘는데, **저주파 담당 영역이 훨씬 넓다**.
- 사람은 **저주파로 말하고 저주파로 듣는다**.
- 그렇다면 Linear Spectrogram의 8000Hz 전 영역을 균등 분석하지 말고, **저주파를 강조하고 고주파를 압축** 하자.

### 9.2 Mel Filterbank — 핵심 아이디어
- **저주파 영역** : 좁은 구간씩 평균(weighted average) → 정보 보존.
- **고주파 영역** : 넓은 구간씩 평균 → 정보 압축.
- 이 비균등 평균 필터들의 모음을 **Mel Filterbank** 라 함.

```
Linear Spec (513) ─[Mel Filterbank]→ Mel Spec (80 or 128)
   세로 차원 압축, 저주파 디테일 유지
```

### 9.3 Mel Spectrogram 차원
| Task | n_mels (세로 차원) |
|------|--------------------|
| 음성합성 (TTS) | **80** |
| 음성인식 (ASR, Whisper 등) | **128** |
- 차원 자체는 정하기 나름이지만, 위 값이 표준에 가깝다.

### 9.4 시각적 비교
- Linear Spec : 위쪽이 검은 영역(정보 없음).
- Mel Spec : 저주파 정보가 살아 있고 전체적으로 정보가 꽉 차 있음. **인공지능에 훨씬 유용**.

> 🔥 **(시험 출제·핵심)** Day 2(음성인식), Day 3(음성합성)에서 다룰 **모든 AI 모델은 Mel Spectrogram을 입력 또는 출력으로 사용** 한다. 따라서 오전 강의의 결론은 "**waveform → mel spectrogram 변환이 음성 AI의 표준 전처리**"이다.

### 9.5 음성 vs 일반 사운드
- 음성(스피치) → Mel Spec (저주파 강조).
- 음악·새소리·기차경적·휘파람 등 **고주파 정보가 중요한 사운드** → 그대로 Linear Spec 사용.
- 즉 **Mel은 사람 음성에 특화된 피처**.

### 9.6 멜 스펙트로그램으로 알 수 있는 것들
- **하모닉스 간격** : 좁으면 저음 화자(남성), 넓으면 고음 화자(아이).
- **포먼트 위치** : 발음(모음 등) 추측 가능.
- **피치 곡선** : "비가 옵니다" (평서문, 끝 내려감) vs "비가 옵니까?" (의문문, 끝 올라감).
- **Voiced/Unvoiced** : 진한 줄무늬(Voiced) vs 뿌연 분포(Unvoiced).

## 10. 추가 피처와 현대 동향

### 10.1 MFCC (Mel-Frequency Cepstral Coefficient)
- Mel Spec(80차원)을 추가 변환으로 13~20차원까지 압축한 피처.
- 1990년대~2000년대 초 AI 성능이 낮을 때 유리(저장 공간 절약).
- 현재는 거의 안 쓰임. AI가 발전하면서 더 풍부한 정보(Mel Spec)를 처리할 수 있게 됨.

### 10.2 Waveform 직접 처리
- 최근에는 GPU·데이터·모델 크기 증가로 **waveform을 그대로 입력으로 받는 모델**도 연구됨.
- 그러나 **현재 주류는 여전히 Mel Spectrogram** (8~90% 이상).

## 11. 한 페이지 요약 (핵심 정리)

### 신호 처리 흐름
```
연속 아날로그 음성
   ↓ Sampling (시간축 이산화, 보통 16kHz)
   ↓ Quantization ([-1,1] 16-bit 등분)
   ↓ Resampling (모델 sample rate에 맞추기)
1D Waveform (해석 어려움)
   ↓ STFT (Window=1024, Hop=256, Hann window)
Linear Spectrogram (시간 × 513 주파수)
   ↓ Mel Filterbank (저주파 강조)
Mel Spectrogram (시간 × 80 or 128)
   ↓
음성 AI 모델 (Whisper, TTS 등)
```

### 시험·핵심 포인트 정리
1. ⭐ **Frequency vs Sampling Rate** — 단위는 같지만 의미는 다름.
2. ⭐ **Phase는 청각적으로 무시 가능** — Magnitude만 사용.
3. ⭐ **Sampling Rate Mismatch** — 모델 사용 시 입력 오디오를 반드시 학습 sample rate로 resample.
4. ⭐ **STFT 파라미터** : `n_fft`(=1024), `hop_length`(=256, 이동거리 / 겹치는 길이 아님), `win_length`(=n_fft).
5. ⭐ **Linear Spec 세로 = N/2 + 1 = 513** → 너무 커서 멜로 압축.
6. ⭐ **Mel Spec 차원** : TTS 80, ASR 128.
7. ⭐ **Spectral Leakage 방지를 위해 Hann/Hanning Window 사용**.
8. ⭐ **Mel은 사람 음성 특화 피처** — 음악·일반 사운드는 Linear 사용.

### 강의 중 주요 직관·비유
- **점묘화** : 연속 신호를 점으로 찍어 디지털화 (Sampling).
- **프로펠러** : Aliasing(낮은 sample rate일 때 빠른 회전이 느리게 보임).
- **프리즘** : Fourier Transform — 백색광을 색깔별로 분해.
- **두부 자르기 vs 부드럽게 누르기** : Hann window의 동기.
- **달팽이관** : Mel Spectrogram의 동기 — 저주파에 집중하는 사람의 청각 구조.
