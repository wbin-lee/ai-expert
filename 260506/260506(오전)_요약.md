# 260506 오전 강의 요약 — Distributed Semantics · Dense vs Sparse · TF-IDF · Cosine Similarity · BM25

> 오전 흐름: **Distributed Semantics 학파 vs 사전 학파 → WordNet (Synonym/Similar/Related/Antonym/Isa) → Word2Vec (Skip-gram / CBOW) → 시간에 따른 의미 변화 / Bias 분석 → Contextual Embedding (ELMo / BERT) → Transformer = Soft Memory (Q-K-V Database 비유) → BERT 기반 검색 엔진 만들기 → ⭐ 왜 모든 검색 엔진이 BERT가 아닌가? → Sparse 세계 (Term-Document Matrix / 원핫) → Boolean Retrieval의 한계 → Ranked Retrieval → TF / DF / IDF / TF-IDF → Cosine Similarity (왜 코사인인가) → BM25 (TF saturation + Length normalization) → 실습 (토크나이즈 / Vocabulary / TF / IDF / TF-IDF / Cosine)**
> 강의 주제: NLP의 두 패러다임(Dense / Sparse)을 통합 이해 + Sparse Retrieval의 핵심 — 황승원 교수님.

## 1. ⭐ Distributed Semantics — 두 가지 학파

### 1.1 두 가지 의미론
- **사전 학파 (Lexical / Symbolic)**: 단어의 의미를 사전·언어학자의 합의로 정의. WordNet 등.
- ⭐ **Distributed Semantics 학파**: **"문맥은 곧 의미다"** — 짧은 글짓기로 충분히 의미를 알 수 있다.
  - "후추를 고기에 뿌리면 맛있다 / 후추는 어디에서 자란다" 같은 문장만으로 의미 파악 가능.
  - 같은 컨텍스트에 들어가는 두 단어는 사실상 **동의어**.
  - ⭐ **언어모델은 전적으로 이 학파를 따름**.

### 1.2 ⭐ 두 학파의 비유 (강사 관점)
- 사전 학파: 황승원 교수에게 7일 강의 듣고 커피 마시며 알아가는 방식.
- Distributed: 황승원의 SNS / 친구 리스트만 보고 누군지 파악.
- ⭐ "충분한 데이터(웹 코퍼스)가 있으면 본인보다 더 잘 알 수도 있다."
- 두 학파는 **상보적** — 상황에 따라 선택 / 결합 / Transfer / Hybrid.

### 1.3 ⭐ 결론
- Knowledge Graph도 사전 학파 vs Distributed (Embedding) 학파 양쪽 존재.
- 오늘의 핵심: **Sparse(원핫·TF-IDF)도 Dense(BERT)도 양쪽 다 살아있고, 어떤 경우에 무엇을 쓸지가 중요한 결정**.

## 2. ⭐ WordNet — 사전 학파의 단어 관계

### 2.1 주요 Relation
| 관계 | 예시 | 특징 |
|------|------|------|
| **Synonym (동의어)** | car ↔ automobile | 대체 가능, 의미 거의 동일 |
| **Similarity** | car ↔ bicycle, cow ↔ horse | 대체 불가, 비슷한 역할/카테고리 |
| **Related (Relatedness)** | car ↔ gasoline / tire / highway | 불가분 관계, 유사성은 X |
| **Antonym** | up ↔ down (상쇄), long ↔ short (스펙트럼) | 두 가지 톤 |
| **Hypernym / Hyponym (Isa)** | mango ⊂ fruit, chair ⊂ furniture | 상하위 포함관계, OOP 영향 |

### 2.2 ⭐ Basic Level
- 컨셉 계층에서 사람들이 **가장 일반적으로 호명**하는 레벨.
- 교탁 → 가구 → 무생물 중에서 모두가 "교탁"이라고 부름.
- Predict 연구 분야가 활발.

### 2.3 ⭐ 강사 어록
- "Princeton에서 모인 분들이 한 땀 한 땀 만든 게 WordNet."
- "Thesaurus (동의어 사전) — 글 다양화에 활용."
- "Similarity Score 0~1 사람 라벨링 데이터셋도 유명."

## 3. ⭐ Word2Vec — Skip-gram / CBOW

### 3.1 학습 원리
- ⭐ **타겟 단어 ↔ 주변 단어 (context)** 예측을 통해 자연스럽게 비슷한 벡터 학습.
- **Skip-gram**: 타겟 → context 예측.
- **CBOW**: context → 타겟 예측.
- "Apricot ↔ tablespoon" 자주 등장 / "Apricot ↔ Aardvark" 절대 안 등장 → 분포 차이 학습.

### 3.2 ⭐ Window Size — Hyperparameter
- **작은 윈도우** → **Similarity** 캡처 (같은 카테고리 — 시애틀 마리너스, 자이언츠 등).
- **큰 윈도우** → **Relatedness** 캡처 (해리포터, 호그와트, 마법 — 관련된 모든 것).
- 애플리케이션에 맞게 조정.

### 3.3 ⭐ 시간에 따른 의미 변화 (Diachronic Embedding)
- 시대별 코퍼스로 따로 학습 → 같은 단어의 벡터 위치 변화 추적.
- **Gay**: 1900년 활발한 → 현재 동성애 관련.
- **Broadcast**: 옛날 농업(씨 뿌리다) → 현재 방송.
- **Awful**: 1850년대 좋은 뜻 → 현재 나쁜 뜻.
- ⭐ 사전은 10년에 한 번 개정 / 임베딩은 수학적으로 자연스럽게 변화 표현.

### 3.4 ⭐ 사회적 Bias 측정 / 보정
- 유명 예시: **"파리 - 프랑스 + 제팬 ≈ 도쿄"**.
- 2016 NeurIPS 논문: "아빠가 의사면 엄마는?" → **Nurse**, "남자 컴퓨터 프로그래머면 여자는?" → **주부**.
- ⭐ **Embedding Fitting / Debiasing** — 벡터 위치를 이상적인 곳으로 이동시켜 bias 완화 가능.

## 4. ⭐⭐ Contextual Embedding — ELMo / BERT

### 4.1 차이점
- **Word2Vec (비-contextual)**: "play"라는 단어 주면 → `[1,3,5]` 벡터 그냥 줌. `word2vec.txt`로 배포 가능.
- **Contextual (ELMo / BERT)**: ⭐ "play"만 주면 **거부** — "주변(context)도 같이 줘".
  - "Yankees won the play" → `[1,3,5]` (야구).
  - "Broadway play" → `[7,7,7]` (연극).
- ⭐ **API로만 배포 가능** — 매번 context 따라 다른 vector.

### 4.2 ⭐ ELMo (2018)
- = **"E**mbeddings from **L**anguage **Mo**dels".
- RNN 양방향 → 좌→우 + 우→좌 hidden vector를 concat.
- 이후 모든 Contextual Embedding의 이름이 **세사미 스트리트 인형**으로 (BERT, ERNIE 등).

### 4.3 ⭐ BERT (Google) — 충격의 그래프
- ELMo의 RNN을 **Transformer 양방향**으로 교체.
- 학습 task: **빈칸 맞추기 (MLM)** + ⭐ **NSP (Next Sentence Prediction)**.
- Self-supervised — 교과서만 있으면 학습 가능 (사람 라벨 X).
- ⭐ 그 해 NeurIPS / NLP **모든 최고 논문상**을 ELMo + BERT가 휩쓸었음.
- 충격의 발견: **허름한 baseline 모델 + Word2Vec 벡터를 ELMo 벡터로 교체** → 모든 task에서 SOTA 능가.

### 4.4 ⭐ NSP의 함의 — 문장 임베딩
- 두 문장 입력 → 앞/뒤 순서 분류 → ⭐ **문장 단위 Dense Embedding 획득**.
- Document를 벡터화 가능 → BERT 기반 **Dense 검색 엔진** 가능.
- 파인 튜닝: query-document 클릭 데이터로 학습 → BERT 기반 IR.
- BERT 기반으로: 검색 / QA / 스팸 필터까지 모두 가능 → "모든 NLP 문제 해결!" 분위기.

## 5. ⭐ Transformer = Soft Memory (Database 비유)

### 5.1 일반 데이터베이스
- **Query → Key 정확 매치 → Value 가져옴** (성공 0/1).
- 예: "황승원의 연봉?" → 키 매치 → 1 million 가져옴 OR 실패.

### 5.2 ⭐ Transformer의 Soft Memory
- ⭐ Query와 Key의 **Similarity (0~1)** 를 계산 → 그 비율로 **Value들의 가중 평균**.
- 예: 0.7×1M + 0.2×2M + 0.1×3M.
- ⭐ **하드 매치가 아니라 소프트 매치** — Transformer의 천재성.
- 데이터베이스 디자인을 사람이 아니라 **러닝에 맡긴 것**.

### 5.3 의미
- **Distributed Semantics**의 컴퓨테이션화 — 나의 representation = 주변 친구들의 영향력 합.
- 다음 레이어로 갈수록 representation 풍부 (사회적 관계 누적).

## 6. ⭐ Sparse 세계로 — 왜 BERT만으로는 안 되나?

### 6.1 Sparse Representation
- 단어 2만 개면 **2만 차원 원핫 인코딩** — 대부분 0.
- 셰익스피어 코퍼스 예시: 각 희곡(다큐멘트)별 단어 빈도 → 다큐멘트 벡터 / 단어 벡터.
- ⭐ Term-Document Matrix만으로도 Word2Vec 효과 (로맨스 작품끼리 모임).

### 6.2 ⭐ Sparse가 살아남는 이유
- ⭐ **인터넷 코퍼스 전체를 BERT로 임베딩하기엔 GPU/메모리 비용이 천문학적**.
- 키워드 매치는 **Inverted Index 룩업**으로 매우 빠름.
- 의료/법률처럼 **전문 용어가 잘 정의된 도메인**에서는 키워드 매치만으로 충분.

### 6.3 Boolean Retrieval의 한계
- "이 키워드 정확히 포함" 0/1 → 결과가 **0개거나 너무 많거나**.
- 일반 사용자 query는 매치되는 게 너무 많거나 적음.
- 해결 → **Ranked Retrieval**: similarity score 순으로 정렬해서 Top 10.

## 7. ⭐ TF (Term Frequency)

### 7.1 직관
- "삼성"이라는 query에 대해 **삼성을 200번 언급한 D1 vs 50번 언급한 D2** → D1이 더 관련 가능성 ↑.

### 7.2 ⭐ Log Normalization
- ⭐ 200번이 50번보다 **4배 더 관련 있는 건 아님**.
- 그래서 `log(1 + tf)`로 압축 (선형 → 로그).
- 딸기 위키피디아 예: 딸기 10번 vs 20번 → 2배 관련 X.

---

## 8. ⭐ DF / IDF (Document Frequency / Inverse DF)

### 8.1 Document Frequency
- ⭐ "the"처럼 거의 모든 문서에 나오는 단어 → **변별력 X**.
- "Arachnocentric" 같은 희귀 단어가 한 문서에만 → **변별력 ↑**.
- ⭐ **소중한 단어 = DF가 낮은 단어**.

### 8.2 IDF
- ⭐ 그래서 **역수**를 취함: `IDF = log(N / DF)`.
- Stopword (the, 은/는/이/가) → IDF ≈ 0 → 거의 무시.
- ⭐ **로그 씌우는 이유**: 선형이면 "apple" 같이 자주 나오는 단어가 완전히 무시됨 → 로그로 완만하게.

### 8.3 ⭐ Collection Frequency vs Document Frequency
- **Collection Frequency** = 코퍼스 전체 등장 횟수.
- **Document Frequency** = 등장한 다큐멘트 수.
- ⭐ 같은 1만 회여도, **3,000개 문서에 몰린 "Insurance"** vs **8,000개 문서에 분산된 "Try"**.
- "Try"는 **고르게 분포** → 사실상 stopword와 비슷.
- "Insurance"는 **금융 도메인에 SKU(skewed)** → 금융 문서 매칭에 강력.
- ⭐ **검색은 다큐멘트 변별이 목적이므로 DF가 더 중요**.

### 8.4 강사 어록 — 멜론 가요 분석
- 60년대 가요 분석 → "마도로스" (원양어선) — DF 낮은 시대 특징어 발견.
- "은/는/이/가" 같은 stopword는 정보량 0.
- 정보량 = 1/확률 (Information Theory).

## 9. ⭐ TF-IDF

### 9.1 결합
- **TF × IDF** → 다큐멘트가 **자주 나오면서도(TF↑) 흔치 않은 단어(IDF↑)** 매치를 강조.
- 단어 한 개 query면 IDF 무관 (상수) / 다중 단어 query면 매치된 희귀 단어에 가중치.
- ⭐ 라이브러리에 코퍼스 주면 자동으로 TF-IDF 매트릭스 변환.

### 9.2 식의 다양한 변형
- 로그 베이스 / 로그 씌우기 / 1 더하기 / smoothing 등 다양한 변형 모두 **TF-IDF 스코어링**으로 통칭.
- "외울 의미 없음 — 라이브러리에서 옵션으로 다 제공".

## 10. ⭐ Cosine Similarity — 왜 코사인인가?

### 10.1 문제
- Euclidean Distance / 내적은 **벡터 길이에 영향받음**.
- 예: "I love you"를 1번 말한 D1 vs copy-paste로 2번 말한 D2 → **의미는 같은데 거리는 달라짐**.
- ⭐ 다큐멘트 길이가 제각각 → 길이로 차별하면 검색 결과 왜곡.

### 10.2 ⭐ Cosine Similarity의 원리
- ⭐ **각도만 보고 크기는 무시** — "계급장 떼고 방향만".
- 식: `cos(θ) = (A·B) / (|A| × |B|)` = 내적을 각자 norm으로 나눔.
- ⭐ **L2 Normalize한 두 벡터의 내적 = 코사인 유사도**.

### 10.3 ⭐ 예시 — 제인 오스틴 vs 폭풍의 언덕
- **SAS (Sense and Sensibility)** + **PAP (Pride and Prejudice)** = 같은 작가, 코사인 유사도 ↑.
- 같은 결의 로맨스 소설인 **Wuthering Heights**도 비슷할 것 같지만:
  - "Wuthering"이 **고유 명사**(언덕 이름이 아니라 저택 이름)로 자주 나옴 → IDF가 매우 높음.
  - 그래서 SAS-PAP 쌍이 SAS-WH보다 코사인 유사도 ↑.
- ⭐ **TF-IDF + Cosine Similarity의 완벽한 시너지**.

## 11. ⭐ BM25 — TF-IDF의 진화

### 11.1 ⭐ TF Saturation
- TF가 무한히 늘어나면 score도 무한히 늘어남 (log여도) → **상한이 없음**.
- ⭐ BM25: TF가 충분히 크면 **2 정도로 saturate** — `(K1+1) × tf / (K1 + tf)` 형태.
- K1 파라미터로 곡선의 완만함 조절.

### 11.2 ⭐ Length Normalization
- 다큐멘트 길이가 길면 자연스럽게 TF가 높아짐 → query 관련도와 무관.
- 위키 전체를 한 string으로 넣으면 모든 단어 TF 높지만 query 적합 X.
- ⭐ BM25: `(1-B + B × |d|/avg|d|)` — **평균 길이 대비 보정**.
- B 파라미터로 정규화 강도 조절.

### 11.3 ⭐ BM25 식 종합
```
BM25(q, d) = Σ IDF(t) × [(K1+1) × tf(t,d)] / [K1 × (1-B + B × |d|/avg|d|) + tf(t,d)]
```
- ⭐ K1, B는 **사용자가 경험적으로 튜닝**하는 하이퍼파라미터.
- ⭐ **BM25는 검색 논문의 빠지지 않는 베이스라인** — 단순한 카운트 기반인데도 검색 성능 강력.

### 11.4 IDF의 변형
- BM25에서는 IDF에 **Laplace smoothing 형태** — 분모에 0.5 추가 등.
- 작은 코퍼스에서 IDF=0 되는 문제 회피.

## 12. ⭐ Sparse vs Dense 검색 비교 (오전 결론)

| 측면 | Sparse (TF-IDF, BM25) | Dense (BERT, Word2Vec) |
|------|----------------------|------------------------|
| **속도** | ⭐ 매우 빠름 (Inverted Index 룩업) | 느림 (벡터화 + 시밀러리티 서치) |
| **GPU/메모리** | 거의 불필요 | 인터넷 코퍼스 임베딩 시 천문학적 |
| **정확한 매치** | 강력 (의료/법률) | 약함 |
| **의미적 매치** | 약함 (Spinach vs Ong Choi 매치 X) | ⭐ 강력 (한↔영 cross-lingual도 가능) |
| **단점** | "옹초이" query → 시금치 결과 X | "구글 비번 변경" → 페이스북 결과 가능 |

### 12.1 ⭐ 두 방법은 **상보적**
- 앙상블이 매우 흥미로운 새로운 애플리케이션을 만들어냄.
- ⭐ **Dense IR의 가장 큰 함정**: "구글 비밀번호 변경"으로 query → "페이스북 비밀번호 변경" 결과를 줌 (둘 다 빅테크 CEO 컨텍스트로 비슷하게 임베딩되기 때문).

## 13. 실습 — Sparse Retrieval 직접 구현

### 13.1 빌딩 블록
1. **Tokenize** (whitespace 기준).
2. **Vocabulary 구축** (set of unique terms).
3. **Term-Document Matrix** (rows = terms, columns = documents).
4. **Log TF normalization**: `np.where(tf > 0, 1 + np.log(tf), 0)`.
5. **Document Frequency** 계산.
6. **IDF** = `log(N/DF)` (smoothing 포함).
7. **TF-IDF** = TF × IDF.
8. **L2 Normalize** + **Cosine Similarity** = 내적.
9. **Search**: query → 벡터화 → 모든 doc과 cosine → 정렬 → top-K.

### 13.2 BM25 (오후로 이어짐)
- TF saturation + Length normalization 추가.
- K1, B 파라미터 튜닝.

### 13.3 코퍼스 — 셰익스피어 희곡
- 시저 검색 → 줄리어스 시저 다큐멘트 1위 → 안토니우스와 클레오파트라 (로마 관련) 2위.
- ⭐ "작은 예제에서도 의미 있는 검색이 가능하다는 것을 확인".

## 14. ⭐ 한 페이지 요약

### 14.1 오전 핵심 포인트

| 주제 | 핵심 |
|------|------|
| **Distributed Semantics** | 문맥 = 의미 / "친구 리스트로 그 사람 알 수 있다" |
| **사전 학파 vs 분포 학파** | 상보적, Hybrid 가능 |
| **WordNet 관계** | Synonym / Similar / Related / Antonym / Isa |
| **Word2Vec** | Skip-gram (target→context) / CBOW (context→target) |
| **Window Size** | 작으면 Similarity, 크면 Relatedness |
| **Diachronic / Bias** | 시대별 임베딩 / Debiasing 가능 |
| **Contextual Embedding** | ELMo (RNN bidirectional) → BERT (Transformer bidirectional) |
| **BERT의 NSP** | 문장 임베딩 가능 → 검색 엔진 / 스팸 / QA 다 됨 |
| **Transformer = Soft Memory** | Hard match → Query-Key Similarity 가중 평균 Value |
| **Sparse vs Dense** | Sparse는 빠름·정확 매치 / Dense는 의미적·고비용 |
| **TF Log Normalization** | 200번이 50번의 4배 관련 있는 건 아님 |
| **DF vs IDF** | 흔한 단어는 변별력 X → 역수가 의미 있음 |
| **Collection vs Document Frequency** | "Insurance"가 SKU된 단어 → 변별력 ↑ |
| **TF-IDF** | TF × IDF — 자주 + 희귀한 단어 매치 강조 |
| **Cosine Similarity** | 길이 무시, 각도만 — 다큐멘트 길이 차이 보정 |
| **BM25** | TF saturation (K1) + Length normalization (B) |

### 14.2 전체 구조 도식
```
[두 학파]
사전 학파 (WordNet)  ↔  Distributed Semantics 학파
                                    ↓
                          Word2Vec (Skip-gram / CBOW)
                                    ↓
                    Contextual Embedding (ELMo → BERT)
                                    ↓
                  Transformer = Soft Memory (Q-K-V)
                                    ↓
              [BERT 기반 Dense IR — 만능?]
                                    ↓
              ⭐ "왜 모든 검색엔진이 BERT가 아닌가?"
                                    ↓
        [Sparse 세계로 — 효율성 / 정확 매치 / 도메인 강점]
                                    ↓
                  Boolean Retrieval (0/1 너무 많거나 적거나)
                                    ↓
                  Ranked Retrieval (Top-K)
                                    ↓
        TF (log) → DF / IDF → TF-IDF → Cosine Similarity
                                    ↓
                  BM25 (TF saturation + Length norm)
                                    ↓
              [Sparse vs Dense는 상보적 — Hybrid가 핵심]
```

### 14.3 비유 / 직관 모음

| 개념 | 비유 / 직관 |
|------|-----------|
| **Distributed Semantics** | "친구 리스트만 봐도 누군지 알 수 있다 — 본인보다 더 잘" |
| **WordNet 사전 학파** | "Princeton에 모인 분들이 한 땀 한 땀 만든 사전" |
| **Skip-gram의 Window** | "해리포터를 작은 윈도우 → 같은 기숙사 단어 / 큰 윈도우 → 마법 일반" |
| **사회적 Bias** | "남자 컴퓨터 프로그래머의 여자 버전 = 주부" |
| **Contextual Embedding** | "play만 주면 거부 — 주변을 줘야 vector 줄게" |
| **BERT의 충격** | "허름한 베이스라인 + ELMo 벡터 → 모든 SOTA 능가" |
| **Transformer Soft Memory** | "DB는 0/1 / Transformer는 0.7×1M + 0.2×2M 가중평균" |
| **Sparse 효율성** | "인터넷 전체 BERT 임베딩 = 천문학적 GPU / Inverted Index = 즉시 룩업" |
| **Dense의 함정** | "구글 비밀번호 변경 → 페이스북 결과" |
| **TF Log Norm** | "딸기 10번 vs 20번이 정말 2배 관련 있는가?" |
| **IDF의 정의** | "흔한 단어는 정보량 0 — 시도 때도 없이 나오는 'the'" |
| **Collection vs DF** | "Insurance(금융 SKU) vs Try(도처 분산) — DF가 더 중요" |
| **Cosine Similarity** | "I love you를 1번 말하든 10번 말하든 의미는 같다" |
| **Wuthering Heights 예시** | "고유명사 'Wuthering'의 IDF가 너무 높아 작가 매칭 강력" |
| **BM25 Saturation** | "TF가 아무리 커져도 약 2까지만 — bound가 있다" |

### 14.4 강사 어록
- "문맥은 곧 의미다 — 짧은 글짓기로 충분히 의미를 알 수 있다."
- "구글은 나를 만나본 적도 없지만 내 유튜브 시청 기록만으로 나를 더 잘 안다."
- "ELMo는 Embeddings from Language Models의 약자 — 이후 모든 contextual 모델 이름이 세사미 스트리트 인형이 됐다."
- "허름한 모델에 ELMo 벡터만 갈아 끼웠는데 모든 SOTA를 이겼다."
- "Transformer의 Soft Memory는 천재적이다 — DB 디자인을 러닝에 맡긴 것."
- "왜 모든 검색엔진이 BERT가 아닌가 — 그게 오늘의 답입니다."
- "아라크론크노센트릭 같은 단어가 매치되면 모두가 너무 해피한 거죠."
- "Try는 사실상 더와 비슷한 단어다 — Collection Frequency만 보면 안 된다."
- "Wuthering Heights는 언덕 이름이 아니고 저택 이름이었다 — 영화 보면서 알게 됐다."
- "BM25를 외우는 건 정말 의미가 없다 — 라이브러리에서 K, B 옵션으로 다 제공한다."
- "구글 비밀번호 변경 검색에 페이스북 결과가 나오면 안 된다 — Dense의 가장 큰 함정."
