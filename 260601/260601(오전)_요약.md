# 260601 오전 강의 요약 — 이미지 클래시피케이션 · Convolution의 Inductive Bias · CNN 아키텍처(LeNet/AlexNet/VGG/GoogLeNet/ResNet) · Vision Transformer · PVT/Swin Transformer · NAS

> 오전 흐름: **Fully Connected 한계(파라미터 폭증·공간 변동에 취약) → Convolution이 도입하는 inductive bias(translation equivariance/invariance) → 채널·필터·receptive field·stride·padding → Pooling(Max/Avg)과 다운샘플링 → CNN과 Fully Connected의 양방향 변환(FC↔ConvLayer) → CNN 아키텍처 계보(LeNet → AlexNet → VGG → GoogLeNet/Inception → ResNet 152) → Residual Connection이 풀어낸 vanishing gradient → Attention 복습(Multi-Head Self-Attention, Causal Masking) → Vision Transformer(ViT)의 패치 임베딩 + CLS token → ViT vs CNN의 데이터 스케일 의존성 → Hierarchical Vision Transformer(PVT/Swin) → NAS 잠깐 소개**
> 강의 주제: **Fully Connected가 풀지 못하던 공간적 일반화 문제를 Convolution이 어떤 inductive bias로 해결했는지 이해하고, CNN 아키텍처가 깊이를 늘리며 발전해 온 계보(특히 ResNet의 residual connection)를 짚은 뒤, Vision Transformer가 그 inductive bias를 제거한 대안으로 어떻게 작동하며 어떤 조건에서 더 우월한지 파악하는 것**.

# 1. Fully Connected의 한계 ⭐

## 1.1 파라미터 폭증

1000×1000 RGB 이미지는 \(3 \times 10^6\) 차원 벡터. hidden representation 100개만 만들어도 첫 레이어 파라미터는:

\[
3{,}000{,}000 \times 100 = 3 \times 10^8
\]

→ overfitting에 매우 취약.

## 1.2 Fully Connected는 패턴 매칭이다 ⭐

\(W^T x\)의 한 칼럼은 dot product이므로 본질적으로 **similarity 측정**. 두 unit vector의 dot product는:

- 같은 벡터: 1
- 직교: 0
- 반대: \(-1\)

→ FC는 학습한 weight 패턴과 입력이 얼마나 비슷한지를 측정. **학습된 패턴 = 데이터셋에서 유용했던 벡터 패턴**.

## 1.3 공간 변동에 취약

"7" 이미지를 살짝 옆으로 옮기면 동일한 weight과의 dot product가 0이 되어버릴 수 있음. 트레이닝셋에서 본 공간 패턴에만 fitting되어 트랜슬레이션이 일어나면 실패.

→ 인간이 가진 인사이트("7은 위치가 옮겨져도 7이다")를 네트워크에 구조적으로 넣는 게 **inductive bias**.

# 2. Convolution과 Inductive Bias ⭐

## 2.1 핵심 아이디어

전체 이미지에 대한 패턴 대신 **로컬 패치 패턴**을 찾고, 그 패턴을 공간 전체에 슬라이딩하며 적용 → 위치가 어디든 같은 패턴은 같은 활성화를 만든다.

## 2.2 Translation Equivariance vs Invariance ⭐

| 성질 | 정의 |
|---|---|
| **Translation Equivariance** | 입력 시프트 시 출력도 동일하게 시프트 (Conv 자체의 성질) |
| **Translation Invariance** | 입력 시프트와 무관하게 출력 동일 (Conv + Pooling 등을 통해 달성) |

Convolution은 **equivariant**, 클래시피케이션 모델은 **invariant**가 목표.

## 2.3 Convolution 정의

이산 시그널 1D:

\[
(f * g)[n] = \sum_m f[m] \cdot g[n-m]
\]

이미지에 쓰는 건 2D Convolution(슬라이딩 디멘전이 2D). 채널 디멘전은 슬라이딩하지 않고 동시에 사용.

## 2.4 핵심 파라미터 ⭐

입력 \(H \times W \times C_1\), 커널 \(F_H \times F_W \times C_1\), 필터 개수 \(K = C_2\) → 출력 \(H_2 \times W_2 \times C_2\).

| 파라미터 | 역할 |
|---|---|
| 커널 사이즈 \(F\) | 로컬 패치 크기 (하이퍼파라미터) |
| 필터 개수 \(C_2\) | 출력 채널 수 (얼마나 많은 패턴을 동시에 볼지) |
| Stride \(s\) | 슬라이딩 보폭 |
| Padding \(p\) | 경계 패딩(보통 zero padding) |

출력 공간 사이즈:

\[
H_2 = \left\lfloor \frac{H + 2p - F}{s} \right\rfloor + 1
\]

전체 weight 파라미터:

\[
F_H \cdot F_W \cdot C_1 \cdot C_2 \quad (\text{+ } C_2 \text{ biases})
\]

⭐ **커널의 채널 디멘전은 항상 입력 채널과 같다** — 채널은 같이 보고, 공간만 슬라이딩.

## 2.5 Receptive Field

마지막 hidden representation 하나가 원본 이미지의 어떤 영역을 보는가. 5×5 conv를 두 번 쌓으면 effective receptive field는 9×9.

\[
\text{RF}_{\text{new}} = \text{RF}_{\text{prev}} + (F-1) \cdot s_{\text{accumulated}}
\]

## 2.6 작은 필터를 여러 번 쌓는 이유 ⭐

7×7 conv 하나(\(49 C^2\) 파라미터) vs 3×3 conv 3번(\(27 C^2\) 파라미터).

- 후자가 파라미터 ↓, **사이에 비선형성 끼워넣을 수 있어** 더 복잡한 패턴 학습 가능.
- 7×7로 하면 그 안의 spatial location 변동이 다 한 패턴에 뭉개져 overfitting.

# 3. Pooling ⭐

| 종류 | 역할 |
|---|---|
| **Max Pooling** | 윈도우에서 최대값 — 가장 강하게 활성화된 시멘틱만 살림 |
| **Average Pooling** | 윈도우 평균 |

목적: **공간 해상도 축소**(연산 부담 ↓) + **로컬 픽셀 미세 변동에 둔감**(시멘틱 강조).

- 단순 stride conv도 가능하나, max pooling은 "패턴이 가장 강한 위치의 시그널"을 보존하는 의미적 해석이 있다.
- Backprop 시 max pool은 **선택된 위치만 gradient 전달**(switching).

# 4. CNN ↔ Fully Connected 양방향 변환 ⭐

## 4.1 Convolution은 FC의 special case

특정 위치에만 weight가 있고 나머지는 0인 sparse FC matrix + weight sharing → Convolution. 그래서 **선형 두 conv 사이에 nonlinearity 없으면 결국 더 큰 FC 하나**.

## 4.2 FC를 Convolution으로 해석

마지막 FC layer(7×7×512 → 4096)는 **7×7 커널을 가지는 conv** 한 번과 같다.

- 장점: 학습된 모델을 **임의 사이즈 이미지에 적용**할 수 있게 됨.
- FC로 짜면 입력 디멘전이 고정되어야 하지만, conv로 해석하면 큰 이미지 입력 시 conv output map이 커질 뿐 적용 가능 → 큰 이미지를 슬라이딩 윈도우로 평가한 효과(세그멘테이션의 토대).

# 5. CNN 아키텍처 계보 ⭐

## 5.1 데이터 + GPU가 가능하게 한 시대

- **ImageNet**(100만 장, 1000 클래스) 등장으로 large-scale data가 마련됨.
- GPU 병렬화로 행렬곱 빠르게 처리 가능 → 딥러닝 폭발.

## 5.2 주요 아키텍처

| 아키텍처 | 특징 | 레이어 |
|---|---|---|
| **LeNet** (1998) | 첫 CNN. Conv-Pool-FC | 약 7 |
| **AlexNet** (2012) | 11×11/5×5 큰 필터, ReLU/Normalization, 2 GPU 분산 | 8 |
| **VGG-16/19** (2014) | **모든 conv가 3×3**, 깊이 증가, FC가 파라미터 대부분 | 16/19 |
| **GoogLeNet/Inception** (2014) | 병렬 멀티 스케일 브랜치, **1×1 bottleneck**으로 채널 압축, Auxiliary Loss로 학습 도움 | 22 |
| **ResNet** (2015) | **Residual Connection**, 152 레이어까지 | 152+ |

## 5.3 1×1 Bottleneck Convolution

1×1 conv는 **각 픽셀을 독립적으로 다른 표현 공간으로 매핑**하는 FC. 채널 디멘전을 압축해 3×3, 5×5의 파라미터 폭증을 막음 → Inception 핵심.

## 5.4 ResNet과 Vanishing Gradient ⭐⭐

**문제**: 레이어를 늘리니 테스트 에러뿐 아니라 **트레이닝 에러**가 올라간다. capacity 부족이 아니라 **옵티마이제이션이 어렵다**는 신호 — vanishing gradient.

**해결**: residual connection.

\[
y = F(x) + x
\]

- Identity 함수를 학습하는 데 conv layer만으로는 어려운데, residual을 더하면 \(F(x) \approx 0\)으로 만들면 됨(weight 0에 가까이) — 학습 용이.
- Backprop 시 \(\frac{\partial y}{\partial x} = \frac{\partial F}{\partial x} + I\)로 **항상 1이 남아** gradient가 깊은 레이어까지 흐름.

⭐ ResNet 논문은 **전 분야 통틀어 인용 1위** 수준의 영향력. Transformer 등 모든 현대 아키텍처가 residual 사용.

# 6. Attention과 Self-Attention ⭐

## 6.1 Attention 일반 형태

\[
y = \sum_i \text{softmax}\left( \frac{q^T k_i}{\sqrt{d}} \right) v_i
\]

- Query \(q\), Key \(k_i\), Value \(v_i\).
- Dot product = similarity, scaling은 dimension에 따른 폭증 방지.

## 6.2 Self-Attention

\(Q, K, V\)가 모두 같은 입력 set에서 나옴. **셋 내부의 상호 contextualization**.

### Permutation Equivariance ⭐

입력 순서를 바꾸면 출력도 같은 순서로 바뀜 → 순서 정보 없음. **Positional Encoding** 추가로 시퀀스 정보 주입.

## 6.3 Multi-Head Self-Attention ⭐

Feature dimension을 \(h\) 등분해 각 head가 **다른 측면의 similarity**를 봄.

- "비슷하다"가 단일 dot product로 표현되기 어려움(고양이와 개가 다리 측면에서 비슷, 다른 측면에서는 다름).
- 여러 aspect로 쪼개 동시에 보면 각 head가 다른 관점의 컨텍스트를 모음.

## 6.4 Masked / Causal Attention

Auto-regressive 생성 시 미래 토큰을 못 보게 lower triangular matrix로 마스킹. \(-\infty\)를 softmax 통과시켜 0으로.

# 7. Vision Transformer (ViT) ⭐

## 7.1 핵심 아이디어

논문 제목: "An image is worth 16×16 words". 이미지를 **16×16 패치 N개**로 쪼개 토큰 시퀀스로 만들고 BERT를 그대로 적용.

- Patch embedding은 stride=16의 single conv로 구현(자르기 + projection 동시).
- **Classification(CLS) Token** 추가 → 196 + 1 = 197 토큰. 마지막에 CLS token만 뽑아 분류.
- Positional Embedding으로 패치 위치 주입.

## 7.2 ViT vs CNN ⭐

CNN의 inductive bias: **로컬리티**(가까운 픽셀 우선) + **translation invariance**. ViT는 이걸 모두 버리고 모든 패치를 동시에 봄.

데이터 스케일별 성능:

| 데이터셋 | 성능 |
|---|---|
| 작은 데이터(ImageNet) | **ResNet > ViT** — inductive bias 덕분 |
| 매우 큰 데이터(JFT-300M, 약 3억 장) | **ViT > ResNet** — bias가 오히려 제약이 됨 |

→ "**데이터가 많으면 쉬운 문제, 적으면 어려운 문제**". 데이터가 충분하면 모델이 스스로 inductive bias를 학습.

## 7.3 왜 MLP가 아니라 ViT인가

MLP도 inductive bias 없음. 차이는:

- **MLP**: 학습된 weight 패턴과의 매칭(고정 패턴).
- **Self-Attention**: 입력 토큰 간 similarity 측정(동적 컨텍스트).

→ ViT가 일반화에 훨씬 유리.

# 8. Hierarchical Vision Transformer ⭐

## 8.1 동기

ViT는 처음부터 끝까지 같은 해상도. 이미지의 왼쪽 끝과 오른쪽 끝을 raw color space에서부터 같이 볼 필요가 있나? → 점진적으로 해상도를 줄이며 hierarchy를 만들자.

## 8.2 PVT (Pyramid Vision Transformer)

각 블록에서 다시 patch embedding을 적용해 feature map을 단계적으로 축소. CNN처럼 멀티 스케일 backbone 역할.

## 8.3 Swin Transformer ⭐

- 첫 레이어들은 어텐션 패턴이 **로컬에 집중** → 굳이 글로벌 어텐션이 필요 없음.
- **Windowed Attention**: \(M \times M\) 윈도우 안에서만 attention 수행.
- 윈도우 경계에 걸친 패치들이 서로를 못 보는 문제 → **Shifted Window**: 짝수/홀수 레이어에서 윈도우를 반만큼 시프트해 상호 컨텍스트 확보.

→ 효율성-정확도 trade-off의 새 frontier.

---

# 9. NAS (Neural Architecture Search) ⭐

피처 엔지니어링을 데이터로 대체했듯, **아키텍처 설계 자체도 학습**.

- 서브 컴포넌트 후보군 정의 → policy network가 추가/제거 추천 → validation set으로 평가 → 좋은 것만 채택.
- 메타 논문 중에는 **랜덤하게 와이어드된 네트워크**가 사람이 설계한 것에 견줄 만한 성능을 낸 사례도 있음.
- 현재는 트랜스포머/파운데이션 모델이 강력해 활발히 연구되진 않음.

# 10. 오전 핵심 정리 ⭐

- **Fully Connected**는 본질적으로 패턴 매칭이며, 공간 변동에 취약하고 파라미터가 폭증한다.
- **Convolution**은 로컬 패치 패턴을 슬라이딩하는 inductive bias를 주입해 translation equivariance를 달성하며, 채널은 함께 보고 공간만 슬라이딩한다.
- 작은 필터(3×3)를 깊이 쌓으면 같은 receptive field를 더 적은 파라미터와 더 많은 non-linearity로 표현할 수 있다 — **VGG의 핵심**.
- **ResNet의 residual connection** \(y = F(x) + x\)는 identity 학습을 쉽게 만들고 backprop의 vanishing gradient를 막아 100+ 레이어 학습을 가능하게 했다.
- **Self-Attention의 permutation equivariance** 때문에 positional encoding이 필수이며, **multi-head**는 similarity를 여러 aspect로 분해해 본다.
- **ViT**는 이미지를 16×16 패치 토큰으로 보고 BERT를 적용하며, CNN의 inductive bias를 버린 대가로 **대규모 데이터가 있을 때만 CNN을 능가**한다.
- **PVT/Swin**은 ViT에 계층적 해상도와 윈도우 어텐션을 도입해 효율성을 개선했고, Swin의 shifted window는 윈도우 경계의 정보 단절을 해결한다.
