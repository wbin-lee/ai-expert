# 260601 오후 실습 요약 — Oxford-Flowers 102 클래시피케이션 · 커스텀 CNN/VGG/ResNet/ViT 스크래치 학습 · Transfer Learning(Linear Probing)

> 오후 흐름: **Oxford-102 Flowers 데이터셋 + augmentation 준비 → 커스텀 단순 CNN으로 스크래치 학습(약 22%) → 3×3 conv 다층 + 더 적은 파라미터의 커스텀 VGG로 향상(약 35%) → 같은 파라미터에 residual만 더한 커스텀 ResNet으로 추가 향상(약 47%) → 비전 트랜스포머 직접 구현(Patch Embedding/CLS Token/Positional Embedding/Multi-Head Self-Attention) → 스크래치 ViT(약 35%) → ImageNet 프리트레인 ResNet-18/ViT-Small 백본을 freeze하고 마지막 linear head만 학습(Transfer Learning) → 코사인 유사도 히트맵으로 ViT가 ResNet보다 클래스 분리가 잘 됨을 확인 → ViT transfer learning이 약 99% 테스트 정확도 달성**
> 강의 주제: **CNN 아키텍처 진화(평범한 CNN → VGG 스타일 3×3 → ResNet residual)를 직접 코딩하며 같은 파라미터 예산에서 어떻게 성능이 향상되는지 체감하고, 스크래치 학습의 한계를 transfer learning(linear probing)으로 극복하는 표준 워크플로우를 익히는 것**.

# 3. 스크래치 학습 ① — Custom CNN ⭐

## 3.1 아키텍처

```
Conv 7×7 (s=1, p=3) → ReLU → MaxPool 2×2
Conv 3×3 (p=1) → ReLU → MaxPool 2×2  → 채널 64→128
Conv 3×3 (p=1) → ReLU → MaxPool 2×2  → 채널 128→256
Conv 3×3 (p=1) → ReLU → MaxPool 2×2  → 채널 256→384
AdaptiveAvgPool2d(1×1)
Linear → 102
```

## 3.2 핵심 공식 — 해상도 유지 ⭐

\[
H_{\text{out}} = \left\lfloor \frac{H + 2p - F}{s} \right\rfloor + 1
\]

\(s=1\)일 때 \(H_{\text{out}} = H\)가 되려면 \(p = (F-1)/2\). 따라서:

| 커널 사이즈 \(F\) | 패딩 \(p\) |
|---|---|
| 3 | 1 |
| 5 | 2 |
| 7 | 3 |

**해상도는 conv가 유지하고, pooling이 절반으로 줄인다** — 채널은 깊어지고 공간은 줄어드는 표준 패턴.

## 3.3 결과

- 파라미터: 약 190만
- 25 epoch 후 **validation accuracy 약 21%**

# 4. 스크래치 학습 ② — Custom VGG ⭐

## 4.1 아키텍처 변화

- 모든 conv를 **3×3 (p=1)**로 통일.
- 각 stage에서 conv-ReLU를 **여러 번 연속 적용**(non-linearity 자주 삽입) 후 max pool.
- 스테이지 4개, 총 ReLU 12개 / Conv 약 12개.

## 4.2 의미 ⭐

같은 receptive field를 더 적은 파라미터 + 더 많은 non-linearity로 달성:

| 비교 | 파라미터 | 성능 |
|---|---|---|
| Custom CNN | 약 190만 | 약 21% |
| **Custom VGG** | **약 110만** (오히려 적음) | **약 35%** (val) |

→ **큰 필터 1개보다 작은 필터를 여러 번 + non-linearity 끼우기가 효율적**임을 실험적으로 확인.

# 5. 스크래치 학습 ③ — Custom ResNet ⭐

## 5.1 변경점

VGG 구조 그대로, **forward에서 residual만 추가**:

\[
\text{out} = \text{ReLU}(F(x) + x)
\]

- Conv가 입력 채널을 바꾸므로 residual 더하기 전에 \(x\)도 동일 채널로 매칭(1×1 conv 등).
- 파라미터 개수는 VGG와 거의 동일(약 112만).

## 5.2 Residual의 두 가지 효과 ⭐

| 효과 | 설명 |
|---|---|
| **Identity 학습 용이** | 깊은 레이어에서 각 레이어가 조금만 바뀌면 되는데, conv만으로 identity 학습은 어려움. Residual이 있으면 \(F(x) \approx 0\)만 만들면 identity → weight를 0 근처로만 하면 OK |
| **Gradient 흐름 보장** | \(\nabla(F(x)+x) = \nabla F + I\) → identity term \(I\)가 항상 살아 있어 vanishing gradient 방지 |

## 5.3 결과

| 모델 | 파라미터 | Val Acc (25 epoch) |
|---|---|---|
| Custom CNN | 190만 | 21% |
| Custom VGG | 110만 | 35% |
| **Custom ResNet** | **112만** | **47%** |

→ **단순히 residual 한 줄 추가로 10%p 향상**.

# 6. 스크래치 학습 ④ — Mini ViT 직접 구현 ⭐

## 6.1 컴포넌트

| 컴포넌트 | 구현 |
|---|---|
| **Image → Patch Token** | 224×224 → 16×16 패치로 자르고 각 패치를 임베딩 벡터(128차원)로 변환 — Conv2d(in=3, out=128, kernel=16, **stride=16**)로 한 번에 처리 |
| **CLS Token** | `nn.Parameter`로 학습 가능한 1×128 벡터 추가 → 196 + 1 = **197 토큰** |
| **Positional Embedding** | `nn.Parameter`로 학습 가능한 (197, 128) 벡터 — 토큰에 더해줌 |
| **Transformer Encoder** | `nn.TransformerEncoderLayer` 사용(직접 구현 X). depth 인자로 블록 수 결정 |
| **Classification Head** | 마지막 출력의 **첫 토큰(CLS)**만 슬라이싱 → Linear → 102 |

## 6.2 Forward

```
x → patch tokens (B, 196, 128)
concat CLS token → (B, 197, 128)
+ positional embedding
→ TransformerEncoder
→ output[:, 0]  (CLS 슬라이스)
→ Linear(128, 102)
```

## 6.3 결과

- 파라미터: 약 100만 (VGG/ResNet과 유사)
- 15 epoch val acc **약 35%** — ResNet(40% @ 15 epoch)보다 낮음

→ **데이터셋이 작아 inductive bias 없는 ViT가 불리**. 데이터 스케일(3억 장 이상)일 때만 ResNet 능가(이론 일치).

# 7. Transfer Learning (Linear Probing) ⭐

## 7.1 동기

데이터(2040장)가 너무 적어 스크래치는 한계. **ImageNet에서 프리트레인된 백본**을 가져와 freeze하고 **마지막 linear head만 102 클래스에 맞춰 새로 학습**.

## 7.2 사용 모델

| 모델 | 백본 파라미터 | 학습할 head 파라미터 |
|---|---|---|
| **ResNet-18** | 약 1100만 | 약 50,000 (512×102) |
| **ViT-Small** | 약 2000만 | 약 39,000 (384×102) |

## 7.3 Feature Quality 시각화 ⭐

학습 전에 백본만으로 같은/다른 클래스 5장씩 임베딩 후 **코사인 시밀러리티 히트맵**:

| 모델 | 같은 클래스끼리(대각 박스) | 다른 클래스끼리 |
|---|---|---|
| **ResNet-18** | 약간 진함 | 약간 연함 (분리 약함) |
| **ViT-Small** | **매우 진함** | **매우 연함** (분리 명확) |

→ ViT가 이미 클래스를 잘 분리하는 임베딩을 줌. 학습 안 해도 nearest-neighbor 시각화에서 ViT는 같은 꽃이 매칭됨.

## 7.4 결과 ⭐

| 모델 | Val Acc |
|---|---|
| ResNet-18 transfer | **약 87%** |
| **ViT-Small transfer** | **약 99%** (테스트셋도 99%) |

→ 스크래치 35~47% vs transfer 99% — **수십~수백 배 적은 학습 파라미터로 압도적 성능**.

## 7.5 핵심 인사이트 ⭐

**좋은 백본이 클래스 분리가 잘 된 피처를 주면 linear classifier가 거의 자동으로 잘 작동**. 

- 99% 테스트 acc인데 1% 정도는 틀림 — Confusion matrix로 헷갈리는 클래스 쌍 확인 가능.
- 클래스별 confidence(0~1)로 모델이 얼마나 확신하는지도 확인.

# 8. 오후 핵심 정리 ⭐

- **CNN 진화의 직접 검증**: 단순 CNN(21%) → 3×3 + 깊은 VGG(35%, 더 적은 파라미터) → Residual만 추가한 ResNet(47%, 같은 파라미터). **작은 필터 + non-linearity + residual**의 효과가 모두 실험으로 입증된다.
- **Conv 공간 유지 공식** \(p = (F-1)/2\) (stride 1)을 활용해 conv는 해상도 유지, pooling이 줄이는 표준 패턴을 따랐다.
- **ResNet residual**은 forward에 `F(x) + x` 한 줄 추가하는 단순 구현이지만, identity 학습 용이성과 gradient 흐름 보장이라는 두 효과로 성능을 크게 끌어올린다.
- **Mini ViT 구현**의 핵심은 (1) Conv2d(stride=patch) 한 번으로 patch embedding, (2) 학습 가능한 CLS token과 positional embedding을 `nn.Parameter`로 추가, (3) 마지막 출력의 CLS 토큰만 슬라이싱해 linear head로 분류하는 것이다.
- **데이터셋이 작을 때 ViT는 ResNet보다 약함** — inductive bias가 없어서 데이터로부터 모든 걸 학습해야 하기 때문(스크래치 ViT 35% vs ResNet 40%).
- **Transfer Learning(Linear Probing)**: ImageNet 백본을 freeze하고 linear head만 102 클래스로 새로 학습하면 ResNet 87%, **ViT-Small 99%** 달성. 좋은 백본의 피처가 이미 클래스를 잘 분리하면 linear classifier만으로 충분하다.
- **백본 품질 시각화**는 코사인 시밀러리티 히트맵(같은 클래스끼리 진해야)과 nearest-neighbor 시각화로 학습 전에 검증 가능하다.
