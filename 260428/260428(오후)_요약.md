## 1. Transformer 빠른 복습

### 1.1 발전 타임라인
- **2017**: Attention is All You Need (Transformer 등장).
- **2018**: GPT, BERT 등장.
- **2022**: ChatGPT → 대중화.
- 그 이전: RNN 기반 신경망 LM 발전에 30년 이상 소요.
- 2025년 시점에는 GPT-5, Claude 등 다양한 모델 폭발적 발전.

### 1.2 Transformer 정의
- **Self-Attention 중심으로 입력 토큰 간의 관계를 학습하는 Encoder-Decoder 모델.**
- Attention: 시퀀스에서 유사성 높은 단어를 참조하는 과정.
- Self-Attention: 한 시퀀스 내에서 각 단어가 다른 단어들과 얼마나 관련 있는지를 attention score로 계산하고 가중합.

### 1.3 ⭐ Self-Attention의 세 가지 문제와 해결 (시험 출제)
| 문제 | 해결책 |
|------|--------|
| 순서 정보 X (병렬 처리) | **Positional Encoding을 입력 임베딩에 더함** |
| 표현력 제한 (선형 변환 + 가중합) | **Feed-Forward Network로 비선형성 추가** |
| 미래 정보 누설 | **Masked Self-Attention** |

### 1.4 Multi-Head Attention
- 첫 head는 문맥적 관계, 두 번째는 시제, 세 번째는 명사… 다양한 관점.
- 문장 내 다양한 측면 정보 동시 학습.

### 1.5 Cross-Attention
- 디코더가 인코더에서 학습한 표현을 전달받음 → 디코더 성능 향상.

### 1.6 Transformer 정리
- Long-range Dependency 해결 + 병렬 연산 + Multi-Head + Next Token Prediction → **범용적 표현학습 모델**.

## 2. Hugging Face Transformers 실습

### 2.1 Hugging Face란
- 대규모 사전학습 모델을 누구나 쉽게 사용할 수 있도록 API/라이브러리/데이터셋/모델허브/Spaces/커뮤니티 제공 플랫폼.

### 2.2 Pipeline API
- 트랜스포머스에서 **가장 쉬운 사용 방식**.
- 모델, 토크나이저, 전처리, 추론, 후처리를 직접 작성하지 않고 **task 이름만 지정하면 바로 사용**.

### 2.3 Pipeline으로 가능한 Task
| Task | 설명 | 예시 |
|------|------|------|
| **Feature Extraction** | 텍스트 벡터 표현 추출 | 문법적 특징 반영 벡터 |
| **Sentiment Analysis** | 감정 분석 | "I have been waiting for this course my whole life" → POSITIVE 99% |
| **Text Generation** | 다음 문장 예측 | "in this course we will teach you how to..." → 완성 |
| **Mask Filling** | [MASK] 토큰 채우기 | mathematical / computational 등 그럴듯한 단어 |
| **NER** | 개체명 인식 | Hyejoon=person, Hugging Face=organization, Brooklyn=location |
| **Question Answering** | 질문 답하기 | context 기반 정답 위치 추출 |
| **Zero-shot Classification** | 학습 안 한 클래스 분류 | "transformers library" → education 84% |

### 2.4 NMT 실습 (Helsinki-NLP MarianMT)
**순서**: tokenizer 전처리 → 모델 통과 → 디코드 후처리.
```python
input_ids = tokenizer(input_text)
output = model.generate(input_ids)
result = tokenizer.decode(output, skip_special_tokens=True)
```
- `skip_special_tokens=True`: EOS/SOS 등 special token 제외.

### 2.5 KoBART 한국어 요약 실습
- **KoBART**: 인코더-디코더 생성 모델, 한국어 요약 데이터로 fine-tuning됨.
- `AutoTokenizer`, `AutoModelForSeq2SeqLM` (T5/BART/MarianMT 등 seq2seq 계열 자동 로더).
- Hyperparameter: `min_length`, `max_length` 지정.

### 2.6 🔥 Logit → Softmax 정규화 (직관)
- 모델 raw 출력은 정규화되지 않은 logit (예: -4.36, 4.69).
- "5면 좋은 거야 나쁜 거야?" 직관적이지 않음.
- **Softmax**: $\text{softmax}(z_i) = e^{z_i} / \sum_j e^{z_j}$
  - $e^{-4.36} \approx 0.0$, $e^{4.69} \approx 108$
  - → 정규화 결과: 0.11% (negative), 99.89% (positive).
- Label 0 = negative, Label 1 = positive (관행).

## 3. BERT — Natural Language Understanding (NLU)

### 3.1 ⭐ BERT 정의 (시험 출제: "Encoder-only" 매우 중요)
- 2018년 Google이 공개한 pre-trained 언어 모델.
- **Transformer의 Encoder만 사용**한 형태 — **Encoder-only Model**.
- NLU task에 적합 (감정 분석, 유사도 예측, QA, 분류).

### 3.2 🔥 Bidirectional Representation
- 특정 단어를 이해할 때 **왼쪽(앞 문맥) + 오른쪽(뒤 문맥)을 모두 반영**.
- 빈칸 단어 예측 시 앞뒤 문맥을 모두 참고 → 문맥 이해의 강점.

### 3.3 BERT의 입력 표현 (3가지 임베딩 합)
1. **Token Embedding** — 각 단어의 토큰.
2. **Segment Embedding** — 문장 구분 (A=첫 문장, B=두 번째 문장).
3. **Position Embedding** — 위치 정보.

### 3.4 사전학습 데이터
- BookCorpus + English Wikipedia에서 약 **33억 개 단어**.

### 3.5 ⭐ BERT의 두 가지 사전학습 방식 (시험 출제)

#### 3.5.1 Masked Language Modeling (MLM)
- 입력의 일부 단어를 **랜덤하게 마스킹**, 해당 단어 예측.
- 양쪽 문맥을 받아들여 정확한 단어를 예측 → 문맥 파악 능력 학습.
- 또는 다른 단어로 대체 (예: "I went to the store" → "I pizza to the store" → "went" 복원).

#### 3.5.2 Next Sentence Prediction (NSP)
- 두 문장 입력 → 두 번째가 실제로 첫 문장 다음에 이어지는지 판단.
- "is_next" / "is_not_next" 분류.
- 문장 간 관계 학습.

### 3.6 BERT 토크나이징
- **사전학습 시 사용한 토크나이저와 vocabulary를 모델 사용 시에도 동일하게 사용해야 함.**
  - 예: "사과"가 사전학습 시 10번이면, 사용 시도 10번이어야 함. 25번이면 다른 단어 취급.
- **Subword Tokenizing**: "unaffordable" → "un" + "##afford" + "##able".
  - "##" 표시는 중간이나 끝의 subword를 알리는 표기.
- 예시: "Embeddings"는 vocab에 직접 없음 → "Em" + "##bed" + "##ding" + "##s"로 분할.

### 3.7 BERT 특수 토큰
| 토큰 | ID | 역할 |
|------|-----|------|
| `[CLS]` | 101 | 시작 토큰, 분류 시 사용 |
| `[SEP]` | 102 | 문장 구분 토큰 |
| `[MASK]` | 103 | 마스킹 토큰 |
| `[PAD]` | 0 | 패딩 토큰 |

### 3.8 BERT 입력 텐서
- **input_ids**: 정수 인코딩된 토큰 ID 시퀀스.
- **token_type_ids (Segment ID)**: 한 문장이면 모두 0, 두 문장이면 두 번째 문장 토큰만 1.
- **attention_mask**: 패드 토큰이 아니면 1, 패드 토큰이면 0 → 학습에서 패드 제외.

### 3.9 BERT 활용 실습 결과

#### Mask Filling
- "Samsung is [MASK] company"
- → telecommunication 22%, semiconductor 6%, shopping, multinational, media …

#### Next Sentence Prediction
- "I work at Samsung" + "Samsung is a good company in Korea" → is_next 100% (잘 잇어짐).
- "Samsung is a good company in Korea" + "I am a student" → is_next 9.9%, is_not_next 90.1%.

### 3.10 영화 리뷰 감성 분석 (한국어, 이중 분류)
- **DistilBERT**: BERT 경량화 모델 (구조/성능 보존, 파라미터 수 감소).
- 학습 전 정확도 ≈ 48% (반타작).
- 학습 설정: epoch=3, learning rate, **CrossEntropyLoss**.
- 학습 후 정확도가 약간 향상 (한 문제 정도 더 맞히는 수준 — 데이터·epoch 적음).

### 3.11 다중 분류 (감정 7가지: 공포/놀람/분노/슬픔/중립/행복/혐오)
- **`num_labels=7`** 지정 (이전엔 2였음).
- 정확도 ≈ 53.8%, 반타작보다 약간 위.

## 4. GPT — Natural Language Generation (NLG)

### 4.1 ⭐ GPT 정의 (시험 출제: "Decoder-only" 매우 중요)
- 2018년 OpenAI가 공개한 pre-trained 모델.
- **Transformer의 Decoder만 사용** — **Decoder-only Model**.
- Generation task에 적합.

### 4.2 핵심 메커니즘
- Decoder-only이므로 **미래 시점 정보를 반영할 수 없음**.
- 따라서 이전 단어들을 바탕으로 다음 단어를 예측하는 **Next Token Prediction**으로 사전학습.

### 4.3 GPT 시리즈 발전
| 모델 | 특징 |
|------|------|
| **GPT-1** | Decoder 기반 사전학습 후 fine-tuning. 현재 frontier 모델들의 출발점. 모델 크기 작아 한계 존재 |
| **GPT-2** | 파라미터 수 대폭 증가. 더 긴 문맥, 일관된 문장 생성. **Fine-tuning 없이 Zero-shot Multi-task** 가능 |
| **GPT-4, o3, o4** | 그 이후 발전된 시리즈 |

### 4.4 Zero-shot이란
- **예시 없이** 프롬프팅만으로 task 수행.
- 다음 시간(prompting 수업)에 자세히. (Few-shot은 예시 몇 개 제공.)

### 4.5 ⭐ BERT vs GPT 비교 (시험 출제 명확히 구분)

| 항목 | **BERT** | **GPT** |
|------|----------|---------|
| 구조 | **Encoder-only** | **Decoder-only** |
| 방향성 | **Bidirectional** | **Unidirectional (왼쪽→오른쪽)** |
| 사전학습 | **Masked Token 예측** | **Next Token 생성** |
| 활용 방식 | **Fine-tuning 필요** (task별로) | **Fine-tuning 없이 Multi-task 가능** |
| 적합 task | 분류, 이해, QA (NLU) | 생성, 대화 (NLG) |

> GPT는 매우 general한 모델 — 여러 문장이 연결된 긴 시퀀스에 next token prediction을 학습하기 때문. 즉 **input과 task가 주어졌을 때 output 확률을 예측**하는 방식.
> 예: 번역 task → input + 프롬프트 → 그대로 번역문 생성.

### 4.6 GPT 실습
#### Text Generation
```python
generator("Once upon a time in a faraway land")
```
- max_token 미지정 시 매우 긴 문장 생성.
- 보통 max_token을 지정해서 비용 절약 (OpenRouter 같은 중계 플랫폼 활용 시 더 중요).

#### Chatbot 구현
- `pipeline`으로 `conversation=True` 옵션 → 입력 칸 등장.
- GPT-2는 옛날 모델이라 헛소리가 많음.
- **이전 대화 히스토리 주입** → 맥락 기반 응답 가능. 그러나 한정된 맥락.

## 5. ⭐ Decoding Strategies (시험 출제 핵심)

> 언어 모델 출력은 다음 토큰의 **조건부 확률 분포**. 어떤 토큰을 채택할지 결정하는 전략이 필요.

### 5.1 결정적 (Deterministic) Search 기반 전략

#### 5.1.1 🔥 Greedy Search
- 매 시점 **확률이 가장 높은 단어**를 선택.
- 전체 일관성·자연스러움 고려 X.
- **단점**: 반복 단어 빠르게 발생, 백트래킹 불가.

**Greedy 알고리즘의 함정**:
```
0 → 3 → ?         (3 다음에 100이라는 정답)
0 → 10 → 8        (10이 더 커서 그리디는 10 선택, 결국 8로 끝남)
```
지역 최적이 전역 최적이 아님.

#### 5.1.2 🔥 Beam Search
- 매 시점 **K개의 후보(beam)** 를 유지하며 각 후보의 확률 추적.
- 전체 문장 일관성을 높이려 함. 하지만 K 클수록 계산 비용 증가.

**예시 (beam = 2)**:
```
"The" 다음:  nice (0.5), dog (0.4)        ← top 2 유지
"The nice" 다음: woman (0.4), house (0.3), guy (0.3)
"The dog"  다음: has (0.9), runs (0.05), and (0.05)

문장 확률:
"The dog has" = 0.4 × 0.9 = 0.36
"The nice woman" = 0.5 × 0.4 = 0.20
```
Greedy면 "The nice woman"(0.20)을 선택했겠지만, **beam search는 "The dog has"(0.36)** 를 발견.

### 5.2 확률적 샘플링 기반 전략 (OpenAI Playground의 temperature/top_p)

#### 5.2.1 🔥 Top-k Sampling
- 확률이 높은 **상위 k개 토큰**만 후보로 두고 그 중에서 샘플링.
- 무작위성을 통제 (top 4900까지 가는 극단적 샘플링 방지).

#### 5.2.2 🔥 Top-p (Nucleus) Sampling
- 누적 확률이 **p 이상**이 될 때까지의 토큰만 유지.
- 분포가 sharp하면 적은 후보, flat하면 많은 후보 → 적응적.
- 예: 누적 80% / 5% / 1%... → 90%까지 누적 → 6번째에서 끊음.

#### 5.2.3 🔥 Temperature
- 확률 분포의 **모양** 자체를 조절.
- $p_i = e^{z_i / T} / \sum_j e^{z_j / T}$
- **T < 1**: 분포가 sharp → 높은 것 더 높게 (보수적, deterministic).
- **T = 1**: 기본 (일반적).
- **T > 1**: 분포가 flat → 다양성 증가.

> 시각화: T=0.5일 때 첫 토큰이 우뚝 솟지만, T=2.0이면 토큰 1은 작아지고 2, 3 토큰이 커짐.

> Temperature, Top-k, Top-p는 **동시에 조합 가능**.

### 5.3 반복 제어 전략

#### 5.3.1 🔥 N-gram Penalty
- 같은 n-gram이 여러 번 나타나지 않도록 패널티.
- 반복 패턴 감소, 자연스러운 문장 생성에 유리.

#### 5.3.2 🔥 Token-based Penalty (Repetition Penalty)
- 이미 등장한 토큰이 다시 나타날 때 패널티 적용 → 생성 확률 감소.
- 예: "love"의 logit=3.0, repetition_penalty=1.2 → 3.0/1.2 = 2.5로 낮춤.

### 5.4 Decoding 전략 실습 결과

#### 영어 ("Once upon a time" 시작)
- **Greedy**: "I'm not sure if I will. I'm not sure if I will..." 매우 반복.
- **Beam Search**: 반복 일부 잔존, 문장 길이는 짧아짐.
- **Beam + N-gram penalty**: "I've been thinking" / "I think"로 다양화.
- **Repetition Penalty**: 더 다양한 표현 등장.

#### 한국어 ("안녕하세요" 시작)
- **Greedy**: "그럼 그건 뭐예요"가 무한 반복.
- **Beam**: 여전히 반복.
- **N-gram penalty**: 다른 문장이 나오기 시작.
- **Repetition Penalty 2.0**: 좀 이상하지만 반복 감소.

#### 확률적 샘플링
- 기본 샘플링: "Great"가 두 번 등장, 문장 반복.
- **Temperature 1.5**: 더 다양한 단어.
- **Top-k**: Beam/Greedy보다 자연스러운 결과.

> 직접 실습 권장 — 창의적 답이 필요한 경우 / 안정적 답이 필요한 경우 / 반복 제어가 필요한 경우 다른 전략.

## 6. BERT vs GPT 성능 비교 실습

### 6.1 감성 분석 (Sentiment Analysis)
- **BERT (DistilBERT)**: 정확도 **91%**, "The movie was surprisingly good" → POSITIVE 99% 이상.
- **GPT-2**: 정확도 비슷하지만 동일 문장에서 **낮은 확률로 positive 예측**.
- GPT-2를 분류로 쓸 때는 padding token이 기본 없어 **직접 설정** 필요.
- GPT 계열은 분류 시 **풀 컨텍스트를 담은 마지막 hidden state**에 classification head를 달아 분류.

### 6.2 Question Answering
**Context**: "윤여정은 1966년 연극 배우로 연기 경력을 시작했고, 2021년 영화 미나리의 순자역으로 아카데미 여우조연상을 수상했다."

| 질문 | BERT 답변 | GPT-2 답변 |
|------|-----------|------------|
| 윤여정의 직업은? | "연극 배우" (정확) | 동문서답 |
| 21년에 무엇을 했나? | "아카데미 여우조연상을 수상" (정확) | 엉뚱한 말 |
| 미나리에서의 역할은? | "순자" (정확) | 잘못된 답 |

### 6.3 ⭐ Extractive QA에는 BERT가 우수한 이유 (시험 출제)
- **BERT**: 문맥 전체를 양방향으로 인식 + 질문과 지문을 함께 입력 → 지문에서 정답 span 추출하도록 fine-tuning 가능.
- **GPT-2**: Decoder-only autoregressive → 다음 토큰 생성에 최적화 → 같은 조건의 Extractive QA에서는 불리.

## 7. Retrieval & RAG (Retrieval-Augmented Generation)

### 7.1 BERT의 현대적 역할
- 최신 BERT 계열 모델은 **RAG 시스템의 Retrieval Backbone**으로 활용.
- **Retrieval**: 대규모 문서 집합에서 질문과 관련된 문서/문장을 찾는 과정.
- BERT는 문서를 의미적으로 인코딩 → 벡터 공간 유사도 기반 검색.
- 검색된 문서 → GPT 등 생성 모델의 **컨텍스트로 제공** → 정확하고 풍부한 응답.

### 7.2 Retrieval 실습 흐름
```
1. 문서 리스트 두기
2. BERT로 각 문서 임베딩
3. Question 입력
4. Question을 BERT로 임베딩
5. 각 문서 임베딩과 question 임베딩의 코사인 유사도 계산
6. 가장 유사한 문서 반환
```
- 예: "삼성 라이온즈와 관련된 뉴스가 있니?" → 정확하게 삼성 라이온즈 관련 뉴스 검색.

### 7.3 GPT 주류이지만 BERT 여전히 중요
- 생성형 모델이 주류이지만, **NLU 모델(BERT)은 정보 검색·의미 파악 측면에서 여전히 핵심**.

## 8. 최신 Language Model 흐름

### 8.1 발전 방향
- 트랜스포머 이후 LM은 점점 더 **커지고, 다양해지고, 특정 목적에 맞는 모델**도 계속 등장.
- 단순 문장 생성 수준을 넘어 **추론, 코딩, 도구 사용, 의료, 과학** 등 전문 영역으로 확장.

### 8.2 주요 모델
- 처음엔 Decoder 계열 generator만 사용 (GPT, Gemini, DeepSeek, Claude).
- 현재는 Encoder 계열도 RAG에 활용.
- 최근: GPT-5, GPT-5.5, DeepSeek v4, Gemini 3.1 Pro 등.

> 중요: 개별 모델 이름을 외우는 것이 아니라, **트랜스포머 이후 흐름**을 이해하는 것.

## 9. 한 페이지 요약

### 9.1 ⭐ 시험 핵심 포인트 정리

| 주제 | 핵심 |
|------|------|
| **Self-Attention 3대 문제** | 순서 X / 비선형 X / 미래 누설 → PE / FFN / Mask |
| **BERT** | **Encoder-only, Bidirectional, MLM+NSP, Fine-tuning 필요** |
| **GPT** | **Decoder-only, Unidirectional, Next Token Prediction, Zero-shot Multi-task** |
| **Logit→확률** | Softmax로 정규화 |
| **BERT 입력 임베딩** | Token + Segment + Position |
| **BERT 사전학습 task** | MLM (Masked LM), NSP (Next Sentence Prediction) |
| **Subword Tokenization** | "##" 표기는 중간/끝 subword |
| **CLS=101, SEP=102, MASK=103, PAD=0** | 특수 토큰 ID |
| **Greedy** | 매 step 최고 확률, 반복·백트래킹 불가 |
| **Beam Search** | K개 hypothesis, length normalization |
| **Top-k** | 상위 k 토큰만 |
| **Top-p (nucleus)** | 누적 확률 p 이상 |
| **Temperature** | T<1 sharp, T>1 flat |
| **N-gram penalty** | 같은 n-gram 반복 방지 |
| **Repetition penalty** | 토큰 logit / penalty |
| **Extractive QA** | BERT >> GPT-2 (양방향 + 질문/지문 함께) |
| **RAG** | BERT(retrieval) + GPT(generation) 결합 |

### 9.2 BERT vs GPT 한눈에
```
BERT  =  Encoder-only  +  Bidirectional  +  MLM/NSP  +  NLU
GPT   =  Decoder-only  +  Unidirectional +  NTP      +  NLG
```

### 9.3 Decoding 전략 한눈에
```
[Deterministic]
  Greedy     : 매 step 최고 확률
  Beam Search: K개 후보 유지

[Stochastic Sampling]
  Top-k      : 상위 k 토큰
  Top-p      : 누적 확률 p
  Temperature: 분포 sharp/flat 조절

[Repetition Control]
  N-gram penalty       : 같은 n-gram 패널티
  Repetition penalty   : 등장 토큰 logit 나누기
```

### 9.4 강의 중 주요 직관/비유 모음
- **반타작 (50%)** — 학습 전 모델 정확도, 랜덤과 비슷.
- **Greedy의 100 vs 8** — 지역 최적의 함정.
- **온도 = 분포 모양** — sharp(보수적) vs flat(다양성).
- **도시락처럼 fixed size** — Attention output은 가변 입력에서도 고정 차원 summary.
- **사과 10번 vs 25번** — 토크나이저 일관성 중요.
- **"##" 표기** — subword가 단어 중간/끝임을 알림.
- **사전학습 → Fine-tuning** — BERT는 task별로 fine-tuning, GPT는 zero-shot 가능.
