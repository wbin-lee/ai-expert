# 260428 오전 강의 요약 — Language Model · Seq2Seq · Attention · Transformer 입문

> 오전 흐름: **RNN Language Model → Seq2Seq → Beam Search/BLEU → Attention(일반화) → Self-Attention → Transformer**
> 핵심 메타 메시지: "텍스트가 시퀀스라는 건 **고정관념**일 수 있다." 트랜스포머의 본질은 텍스트를 **병렬 처리 가능한 집합 + 위치정보**로 보는 발상의 전환.

## 1. Language Model이란 무엇인가

### 1.1 정의 (원형)
**맥락 다음에 올 단어의 확률을 근사하는 모델.**
"The student opened ____" 다음에 올 만한 단어(book / laptop / …) 50만 개에 대해 확률 분포를 부여한다.

### 1.2 단어 확률 → 문장 확률 (Chain Rule)
```
P(the cat sat on the mat)
 = P(the) · P(cat|the) · P(sat|the cat) · P(on|the cat sat) · …
```
즉 "맥락이 주어졌을 때 다음 단어의 확률"만 잘 예측하면 **문장 확률을 근사**할 수 있다. 그것을 무한히 이어 붙이면 corpus 자체를 "외운 듯" 작동한다. → **거시적 언어 이해 ≡ next-token prediction**.

### 1.3 활용
- 핸드폰 자동 완성 (작은 통계 모델 → 성능 낮음)
- 검색/추천 엔진 (개인화된 conditional LM)
- 번역, 챗봇, 요약, 코드 생성 — 모두 결국 LM

### 1.4 N-gram Language Model (전통적 통계 LM)
- Unigram, Bigram, Trigram, … : 연속된 n개 단어 묶음을 통계 단위로.
- 문제 : n이 커지면 frequency가 0에 수렴 → 4-gram, 5-gram이 한계.
- Smoothing(분모/분자 0 방지를 위해 hyperparameter 더하기) 등의 trick.
- 직관도 부족 — "시계가 시작했으니 시험을 본다"가 정답인데 더 흔한 "book"이 뽑히는 식.
- 그래도 **꽤 그럴싸한 문장은 만들어진다**는 점에서 의외로 강력.

### 1.5 RNN Language Model (딥러닝 LM)
구조: `x_t (원핫) → W_embed → x̃_t → RNN cell(h_{t-1}, x̃_t) → h_t → W_y → y_t (vocab 확률)`

```
SOS → "the" → "dog" → "is" → "chasing" → "a" → "cat" → EOS
        ↑       ↑       ↑          ↑          ↑       ↑
       각 step에서 negative log-likelihood 계산 후 합산하여 backprop
```

- **Cross-entropy loss = Negative log-likelihood**: 정답 단어의 확률이 얼마인지를 묻는 것. 정답 외 단어의 확률은 loss에서 다 0으로 곱해져 사라짐.
- 학습 과정: 빅 코퍼스 모음 → ŷ 예측 → CE loss 계산 → backprop → 평균 loss를 모든 step에서 합산.
- **장점**: 사람의 레이블링이 필요 없음. 코퍼스만 있으면 학습 가능.
- Pre-trained Word Embedding (Chapter 1)도 가능, 안 써도 랜덤 초기화 행렬로 학습 가능.

## 2. Seq2Seq (Sequence-to-Sequence) 모델

### 2.1 등장 배경
- 2014년 **Ilya Sutskever** (Hinton 교수의 제자) 등이 발표.
- 그 전엔 **SMT(Statistical Machine Translation)**: Bayesian rule + likelihood + prior, 단어 alignment 등 복잡한 통계 시스템. 한 학기짜리 대학원 수업, 수백 명이 운영.
- Seq2Seq 등장으로 **NMT(Neural Machine Translation)** 시대 → 한 사람이 한 시간 만에 수백 명의 노력보다 뛰어난 모델 생성. 기존 MT 커뮤니티 증발.
- 2024년 NeurIPS에서 **10주년 기념 톡** 진행될 정도로 상징적.

### 2.2 구조
```
[Encoder RNN]              [Decoder RNN]
 x1 → x2 → … → xn  ──▶  s0=h_n → s1 → s2 → … (한 단어씩 생성)
 (소스 언어, h들)        (타겟 언어, conditional LM)
```
- Encoder: 입력(소스 언어)을 **고정 크기 벡터**로 압축. 마지막 hidden state h_n을 쓰는 방법, 모든 hidden state pooling, bidirectional RNN 등 여러 옵션.
- Decoder: 그 벡터를 **조건**으로 받아 한 단어씩 출력(타겟 언어 생성).

### 2.3 🔥 핵심 직관: Decoder First (Conditional LM)
- 일반적으로는 Encoder→Decoder 순서로 배우지만, **동기적으로는 Decoder가 먼저**.
- Decoder = Language Model = 언어를 자유롭게 생성하는 모델.
- 그러나 LM 단독은 "내 명령을 따르지 않고 자기 멋대로 발화하는 동료"와 같아 유용하지 않음.
- 우리는 **입력→출력 매핑**을 원함 → 그 입력 조건을 제공하는 게 Encoder.
- 즉 **Seq2Seq = Conditional Language Model** = "Encoder 조건 하에 Decoder가 LM처럼 생성".
- 적용 가능 task : 번역, 요약, 챗봇, 코드 생성, paraphrasing, 등 거의 모든 시퀀스→시퀀스.

### 2.4 왜 기계 번역이 벤치마크였나
1. **데이터가 많음** — 인류 최초의 빅데이터 = **성경(모든 언어로 번역됨)**.
2. AI 서비스가 거의 없던 시절, 회사에 도움 되는 거의 유일한 응용.
3. 가장 어려운 종합 올림픽 — multilingual + 단어 정밀성 모두 요구.

> 다만 번역은 **closed-ended** task. 보고서/소설 같은 **open-ended** task가 LLM 시대의 주된 평가 대상으로 옮겨감.

### 2.5 Teacher Forcing (학습 트릭)
- Decoder 학습 시 **이전 step의 정답을 그대로 입력**. 모델 자기가 만든 출력을 넣지 않음.
- 이유: 첫 번째 단어를 잘못 생성하면 두/세/네 번째까지 에러가 풀어팩(연쇄 오염) → 학습 불안정.
- **단점**: Train ↔ Inference 환경이 달라짐 (오버피팅 우려).
- 해결: **Scheduled Sampling** — 학습 초반엔 teacher forcing 많이, 후반으로 갈수록 줄임. (네발 자전거 → 두발 → 외발 자전거 비유)

## 3. Inference Strategy: Greedy vs Beam Search

### 3.1 Greedy Inference
- 매 step에서 **확률 최대 단어**를 그대로 채택. 한 번 잘못 뽑으면 백트래킹 불가.
- 예시: 프랑스어 "Le chat noir"(검은 고양이)을 번역하는데 첫 단어 "Le" 다음에 "saint"(계란)이 잘못 뽑히면 그 문장은 망함.

> 그리디 알고리즘은 알고리즘 이론에서 **lower bound 보장**이 잘 증명된 베이스라인. 인생도 그리디로 살면 최적은 못 찾아도 망하진 않는다는 비유.

### 3.2 🔥 Beam Search
- "Beam = 레이저 빔. 입력에서 출력까지 일정 크기 K를 유지하며 탐색."
- 매 step마다 K개의 hypothesis를 유지하고, 그 K개에서 다시 vocabulary 전부에 대해 확장 → 50만 × K 연산 → 다시 K개로 줄이고 반복.
- EOS 토큰이 나온 hypothesis들을 모아 **length normalization**(log 확률을 길이로 나눔)을 거쳐 최종 선택.
  - 이유: log 확률은 길이에 비례해 커지거나 작아지므로 그대로 합치면 길이 편향 발생.

### 3.3 Beam의 한계
- K가 늘어나면 연산량 증가, 노이즈 hypothesis가 끼어 우연히 채택될 수 있음.
- Neural net은 수학적으로 멋진 부분과 휴리스틱·기계적 부분이 혼재.

## 4. 평가 지표: BLEU, ROUGE

### 4.1 평가 자체의 어려움
- **Human evaluation**: 비싸고 결과가 매번 다름(논문은 원하는 결과 나올 때까지 돌리는 불편한 진실).
- **Crowdsourcing** (Amazon Mechanical Turk): 저렴하지만 노동력 윤리 이슈 + 품질 변동.
- **LLM-as-a-Judge**: 평가 대상 능력이 부족하다는 자기모순.
- **다중 모델 평가 + correlation 기반 outlier 제거** 같은 메타 평가법도 존재.

### 4.2 BLEU Score (Precision 기반)
- 모델이 생성한 응답을 **n-gram으로 쪼갠 뒤**, 정답에 존재하는 n-gram의 비율(precision)을 잼.
- 예: 정답 "How are you?", 모델 "How is you?" → unigram precision = 2/4.
- 짧은 문장은 BLEU가 높게 나오는 부작용 → **brevity penalty** 추가.
- 번역에 주로 쓰임.

### 4.3 ROUGE Score (Recall 기반)
- 정답 n-gram 중 모델이 얼마나 가져왔는지(recall).
- 길수록 유리 → 길이 penalty.
- **요약**에 주로 쓰임.

> 나중엔 BLEU/ROUGE/PPL 모두 한꺼번에 다각도로 평가하는 방식이 주류.

## 5. Attention — Bottleneck 문제 해결

### 5.1 ⭐ 시험 출제 포인트: Bottleneck 문제
**Encoder가 마지막 hidden state h_n 하나로 모든 정보를 압축** → 디코더는 그 고정 벡터만 보고 생성. 길이가 짧을 땐 괜찮지만:
- 해리포터 700개 단어 → h_700에 모든 정보가 담겨있어야 함. 사람도 책 읽고 덮으면 다 기억 못 함.
- LSTM도 한계 → bidirectional RNN으로 일부 보완 가능하지만 여전히 **고정 차원 bottleneck**.

### 5.2 Attention 아이디어 — "책을 다시 펼친다"
**번역은 일종의 search.** Decoder의 매 step마다 Encoder의 어떤 부분에 주목할지 동적으로 검색.

### 5.3 메커니즘 (시퀀스 시퀀스 + Attention)
```
디코더 step t:  s_t (query)
인코더 hidden states: h_1, h_2, …, h_n (candidates)

1) Attention score:    e_t,i = score(s_t, h_i)
2) Attention dist:     α_t,i = softmax(e_t,i)        ← attention distribution
3) Attention output:   a_t   = Σ_i α_t,i · h_i        ← weighted sum
4) Decoding:           [s_t; a_t] concat → U → y_t
```
- Encoder hidden 100차원 + Decoder hidden 100차원 → concat 200차원 → vocab 사이즈로 projection.
- **별도 파라미터가 거의 없음**(병렬 연결로 구조 추가만 함). dot product, softmax, 가중합으로 끝.

### 5.4 효과
- **매 디코딩 step마다 새 정보를 요청** — bottleneck 해소.
- **Vanishing gradient 완화** — encoder→decoder가 한 번에 직접 연결됨(skip connection 효과).
- 성능: length가 길어져도 attention 쓰면 BLEU 유지. attention 없는 큰 모델보다 attention 있는 작은 모델이 더 좋음 (모델 크기보다 구조가 중요).

### 5.5 Attention의 또 다른 효과: 설명 가능성
- 딥러닝은 black box지만 attention distribution은 시각화 가능.
- 영어→프랑스어 번역에서 어순이 비슷한 언어라면 attention map이 **대각선**으로 흐름 → 자연스러운 번역의 증거.
- 어순이 같은 언어인데 흩어진 attention → 모델 디버깅 단서.
- 보수적 분야(의료/금융)에서 신뢰성·해석성은 채택의 핵심.

## 6. Attention 일반화 — Query / Key / Value 관점

### 6.1 일반화된 정의
- **Values**: 후보들의 집합(Encoder의 hidden states 등).
- **Query**: 정보를 요구하는 측(Decoder의 현재 state 등).
- **Attention = query에 기반한 values의 weighted sum**.
- Encoder/Decoder 구조에만 갇히지 않음 → 어디든 적용 가능.

### 6.2 ⭐ 도시락 비유 (시험 직관)
정보가 가변적이어도 **출력 차원은 고정**되어야 함. 그래서 values를 다 가중합 → 일종의 **selected summary**. "어떤 건 쓰고 어떤 건 안 쓰고"는 안 됨; 모든 values에 가중치를 주고 합친다.

### 6.3 Attention Score 계산 방식
| 방법 | 식 | 적용 조건 |
|------|-----|----------|
| **Dot product** | s · h | 두 벡터 차원 같을 때 |
| **Multiplicative (Bilinear)** | s · W · h, W: d1×d2 | 차원이 다를 때, 학습 가능한 W로 score 계산기 역할 |
| **Additive** | vᵀ tanh(W₁ h + W₂ s) | 작은 NN을 둬서 regression 문제처럼. 더 유연 |
| **Scaled dot product** | (s · h) / √d_k | 트랜스포머에서 사용 (다음 단계) |

> Q: "Additive는 유사도 기반이 아닌데 왜 동작?" → 사실 W 자체가 end-to-end로 학습되므로, 유사도 기반이라는 건 사람 직관일 뿐. **"필요한 정보를 찾아오기 위한 방법"**이 더 정확한 표현.

### 6.4 Self-Attention 맛보기
- Query와 Values가 **같은 시퀀스 내**에 있을 때 = **Self-Attention**.
- 예: text classification에서 query 역할의 학습 가능한 vector c를 두고 단어들과 attention → "like"라는 감정 단어를 강조.
- 트랜스포머가 self-attention 아키텍처.

### 6.5 Attention Supervision (강사 본인의 연구)
- Attention 분포가 데이터 bias에 오염될 수 있음 (예: "Steven Spielberg" 단어만 보고 무조건 긍정 분류).
- 사람이 중요한 단어를 어노테이션 → attention distribution과 비교하는 cross-entropy loss를 추가 → 성능 + 설명 가능성 동시 개선.

## 7. Transformer — 텍스트는 시퀀스가 아니다?

### 7.1 🔥 핵심 발상의 전환
- "텍스트는 시퀀스다" = **고정관념**.
- **속독 학원 비유**: 글의 중간에 점을 찍고 1, 2, 3, 4, 5로 눈동자만 이동하면서 한 번에 한 줄을 병렬 인식. 순차로 읽는 게 궁극은 아님.
- 트랜스포머는 시퀀스를 시간순 hidden state로 보지 않고, **각 위치를 병렬로 + 위치 정보 보강**으로 처리.

### 7.2 RNN의 두 가지 근본 문제
1. **Long-Term Dependency (정확도 문제)** — 앞부분 정보가 뒤로 갈수록 희석.
   - 소설에서 "왕자가 죽었다"가 너무 앞에 있으면 "부활했다"의 주체를 모델이 잊음.
2. **Sequential 처리 (효율성 문제)** — t번째 hidden은 t-1을 기다려야 함 → 데이터가 클수록 치명적.
   - SK텔레콤 BERT 학습이 한 달 걸리는데, 병렬 안 되면 T배 더 걸림 → 누구도 시도 안 함.

### 7.3 Self-Attention의 해결
- 각 레이어에서 모든 토큰이 동시에 다른 모든 토큰과 attention → 병렬 + 직접 연결.
- Long distance도 한 번에 → vanishing gradient 완화.

### 7.4 Self-Attention 형식 정의
```
입력 단어 i:  w_i (one-hot) → x_i = E·w_i (embedding, d차원)
각 단어:      q_i = W_Q·x_i, k_i = W_K·x_i, v_i = W_V·x_i

Attention:
  score_ij = q_i · k_j
  α_ij     = softmax_j(score_ij)
  o_i      = Σ_j α_ij · v_j        ← 단어 i의 새 표현 (병렬 계산)
```
- 모든 단어에 대해 q, k, v를 행렬곱으로 한 번에 구함 → 병렬.
- 모든 attention output o_i를 한 번에 구함 → 병렬.

### 7.5 ⭐ Self-Attention의 세 가지 문제와 해결책 (시험 출제)

| # | 문제 | 해결책 |
|---|------|--------|
| 1 | **순서 정보 없음** (set처럼 처리) | **Positional Embedding** p_i를 x_i에 더함 |
| 2 | **비선형성 없음** (가중합은 선형 결합 → 1억 번 쌓아도 한 번 곱한 거랑 같음) | **Feed-Forward Network** (각 토큰 독립 적용, 같은 파라미터, ReLU) |
| 3 | **미래 정보 누설** (디코더가 미래 토큰 attention) | **Masked Self-Attention** (미래 위치 score = −∞ → softmax 후 0) |

#### 7.5.1 Positional Embedding 상세
**정보 표현의 두 가지 요건**:
1. **고유성** — 모든 위치가 서로 다른 표현.
2. **유사성** — 가까운 위치는 비슷한 표현, 먼 위치는 다른 표현 (그라데이션).

> "1, 2, 3, 4" 같은 정수 인덱스는 고유성은 OK지만 dot product 했을 때 거리감을 못 줌.

**해결**: sin/cos 주기함수.
- D=256 차원 → sin/cos pair 128쌍.
- 각 차원은 서로 다른 frequency(고주파→저주파)로 변동 → 위치마다 고유한 d차원 벡터.
- 가까운 위치는 dot product 값이 크고, 먼 위치는 작음 → 거리 정보 보존.
- **균질성** — (1번,2번)과 (5번,6번)은 비슷한 관계.

> Q: x_i (단어 의미)와 p_i (위치)를 더하는 게 이상하지 않나?
> A: 신경망의 본질은 항상 `Wh + Wx + bias` 형태. **p_i는 일종의 bias**. "apple is apple"에서 첫 번째 apple과 세 번째 apple에 위치 bias만 살짝 다르게 더한 것.

> 요즘은 sin/cos 대신 **학습 가능한 position embedding**(m by d) 또는 더 고등적인 방법(RoPE 등)을 사용.

#### 7.5.2 Feed-Forward (Non-linearity)
- 각 토큰별로 독립 적용, **모든 토큰이 같은 파라미터 공유**.
- 2-layer: 512 → 1000(or 2048) → 512, 첫 layer 후 ReLU.
- Self-attention(input 512) + FFN(input/output 512) + 한 블록의 입출력 차원 일치 → 블록을 m번 쌓을 수 있음.

#### 7.5.3 Masked Attention
```
Query "the"의 attention score row에서 미래 위치는 −∞로 강제
softmax(−∞) = e^{−∞}/Σ = 0
```
- 학습 시엔 정답을 다 알지만 inference 환경(미래 모름)을 모방.
- "Start" 시점엔 자기 자신만, "Start the" 시점엔 둘만 attention.

## 8. Transformer 전체 구조 (Multi-Head, Residual, LayerNorm)

### 8.1 인코더-디코더 구조
- Encoder N개 + Decoder N개 (각각 같은 구조, 파라미터는 공유 X).
- 모든 sub-layer 입출력 차원 = 512.
- 모든 Decoder는 **마지막 Encoder 출력**과 cross-attention으로 연결 (Encoder-Decoder Attention).

### 8.2 Multi-Head Attention
- 원래 Q/K/V를 512×512로 했으면, 이를 **64차원 8개로 쪼갬**(8 heads).
- 각 head 별로 attention 수행 → 64차원 output 8개 → concat → 512.
- Concat된 512는 단순 물리적 결합 → **W_O (512×512)** 로 한 번 더 변환해서 **화학적 결합**.

#### 8.2.1 트랜스포머의 세 가지 병렬 처리
1. **모든 단어**에 대해 Q/K/V를 동시에 (행렬곱 한 번).
2. **모든 단어**의 attention output을 동시에.
3. **여러 head**가 동시에.

#### 8.2.2 왜 Multi-Head?
- 한 head로 모든 관계 학습 어려움 — coreference("it" → "the animal"), 시제, 명사 관계 등 다양한 관점.
- 여러 head가 서로 다른 W_Q, W_K, W_V로 다채로운 attention 학습.
- Task에 따라 head 수가 너무 많으면 줄이기도.

### 8.3 Residual Connection + Layer Normalization
- Self-attention/FFN 통과한 값과 통과하지 않은 입력을 **더함** (skip connection).
- 그 후 LayerNorm 적용 → vanishing gradient + 학습 안정화.
- **이 두 부분엔 학습 파라미터가 사실상 없음** (LayerNorm의 γ, β 정도).

### 8.4 ⭐ 시험 출제 포인트: 트랜스포머 파라미터 수 계산
시험에 **한 layer라도 틀리면 다 틀리는** 형태로 출제 (10문제, OX/객관식 X).
- 입력 d_model = 512, FFN 중간 차원 = 1000(예시), Q/K/V head 차원 = 64, head 수 = 8.
- **FFN**: W₁ (512×1000) + W₂ (1000×512).
- **Attention**: W_Q, W_K, W_V 각각 (512×64) × 8 heads + W_O (512×512).
- 시험에선 d=5 같은 작은 값으로 출제, 원리만 파악할 것.

## 9. 발상 전환의 교훈 — 학습 팁

### 9.1 트랜스포머의 본질적 가치
- 구조 자체는 복잡할 뿐 새로운 게 없어 보임.
- **메타적 발상**: "텍스트가 시퀀스라는 것은 환상이다." — 이걸 그 시대(2015~2017)에 떠올렸다는 점이 천재적.
- 지금 트랜스포머를 너무 당연하게 보면 다음 혁신을 못 떠올림.

### 9.2 학습법: 정의가 아닌 "문제와 해결" 관점
- RNN을 "이렇게 구성됐다"로 끝내지 말고, **이런 문제가 있다 → 어떻게 해결?** 으로 보기.
- T → T+1 → T+2의 트랜지션을 학습해야 미래 예측 가능 (Agent AI 다음을 떠올리기).
- 박사 학위 훈련된 사람이 이걸 잘 하는 편.

## 10. 한 페이지 요약

### 10.1 모델 발전 흐름
```
N-gram LM (통계)
   ↓
RNN LM (Cross-entropy = NLL, 사람 레이블 불필요)
   ↓
Seq2Seq (Encoder-Decoder, Conditional LM, Teacher Forcing)
   ↓
+Attention (Bottleneck/Vanishing 해소, 설명 가능성)
   ↓
Self-Attention (Q/K/V, 병렬, Long-distance)
   ↓
Transformer (PE + FFN + Mask + Multi-Head + Residual + LayerNorm)
```

### 10.2 ⭐ 시험 핵심 포인트
| 항목 | 요점 |
|------|------|
| **LM 정의** | 맥락 다음 단어 확률 → chain rule로 문장 확률 |
| **Cross-Entropy = NLL** | 정답 단어의 확률만 보면 됨 |
| **Teacher Forcing** | 학습 안정성, train-test gap 발생 |
| **Greedy vs Beam Search** | beam은 K개 hypothesis 유지, length normalization |
| **BLEU(precision, 번역) vs ROUGE(recall, 요약)** | brevity / length penalty |
| **Attention 정의** | values의 weighted sum, query 기반 |
| **Q/K/V 일반화** | "도시락 비유" — 출력은 고정 차원 |
| **Score 종류** | dot, multiplicative, additive, scaled dot |
| **Self-Attention 3대 문제** | 순서 X, 비선형 X, 미래 누설 |
| **해결** | Positional Encoding(sin/cos), FFN, Masked Attention |
| **Multi-Head 병렬** | Q/K/V 병렬 + attention output 병렬 + heads 병렬 |
| **트랜스포머 파라미터 수** | FFN 두 layer + W_Q,K,V + W_O. **시험 출제 가능성 매우 높음** |

### 10.3 강의 중 주요 비유 모음
- **속독 학원의 점 찍기** — 텍스트를 시퀀스 아닌 병렬로 처리하는 트랜스포머의 직관.
- **해리포터 700단어를 다 읽고 책을 덮은 뒤 번역** — Bottleneck 문제. Attention은 "책을 다시 펼치는 것".
- **르 샤 누아 / 르 생** — Greedy의 함정 (saint=계란 잘못 뽑으면 끝).
- **레이저 빔** — Beam Search의 후보 K개 일정 유지.
- **도시락** — Attention output은 고정 차원으로 selected summary.
- **포르쉐 / 마크의 말** — 정보 표현은 고유성 + 유사성을 가져야 함.
- **네발 자전거 → 외발** — Scheduled Sampling으로 점진적 강건성.
- **신임 사원 GPA/토익** — BLEU/ROUGE는 보조 metric일 뿐, 완벽하지 않음.
- **Apple is Apple** — Position Embedding은 단어 의미에 위치 bias만 더한 것.
- **검은 고양이 / 르 샤 누아** — Greedy 디코딩 실패 사례 (saint=계란 잘못).
- **스티븐 스필버그** — Attention bias 사례 (bias 단어 하나로 분류 결정).
