# 1부. 260601 오전 — 이미지 분류, CNN, Vision Transformer

## 1-1. 왜 CNN이 필요한가

딥러닝 입문 때 배운 **Fully Connected(FC) 레이어**부터 다시 생각해 보면, 이미지 분류의 출발점이 보입니다.

1000×1000 RGB 이미지는 약 300만 차원의 벡터입니다. 첫 FC 레이어만 해도 파라미터가 수억 개가 될 수 있어 **오버피팅**에 취약합니다. FC의 연산은 본질적으로 **dot product = 패턴 매칭**입니다. 학습된 weight 벡터와 입력이 비슷할수록 출력이 커집니다.

그런데 숫자 7이 화면 한가운데 있을 때와 살짝 옮겨졌을 때, FC는 **완전히 다른 벡터**로 보기 때문에 같은 7을 인식하기 어렵습니다. 공간 이동(translation)에 지나치게 민감한 것이 FC의 한계입니다.

> ⭐ **시험 포인트**  
> FC는 **전역 패턴**을 한 번에 학습해야 해서 공간 이동에 취약하고, dot product는 **유사도(패턴 매칭)** 의미를 가진다.

## 1-2. Convolution과 inductive bias

CNN은 “**로컬 패치**에 대한 패턴을 여러 위치에 **공유(weight sharing)** 하며 반복 적용한다”는 **inductive bias**를 구조에 넣은 것입니다.

- **Translation equivariance**: 입력이 이동하면 feature map도 같은 방향으로 이동한다.
- **Translation invariance**: pooling 등과 결합하면 최종 분류 결과는 위치에 무관해질 수 있다.

2D convolution은 슬라이딩하는 차원이 가로·세로 2개이고, RGB 채널은 한꺼번에 dot product에 들어갑니다. 출력은 **activation map** 형태로 공간 구조가 유지됩니다.

**Stride**, **padding**, **receptive field**도 이 맥락에서 이해합니다.

- stride가 크면 map이 줄어들고, receptive field는 레이어가 쌓일수록 커집니다.
- 3×3 conv를 3번 쌓으면 effective receptive field는 7×7과 같지만, **중간에 non-linearity**가 들어가 더 복잡한 표현이 가능하고 **파라미터는 더 적다**.

> ⭐ **시험 포인트 (확정 Q4 — VGG)**  
> VGG는 큰 커널 대신 **3×3 conv를 반복**한다.  
> - 같은 effective receptive field를 더 **적은 파라미터**로 확보  
> - 층마다 ReLU가 들어가 **더 복잡한 feature** 표현 가능

## 1-3. Pooling

Pooling은 (1) spatial 크기를 줄여 연산량을 낮추고, (2) 픽셀 단위 미세 차이에 덜 민감하게 만들어 **semantic한 패턴**을 강조합니다. Max pooling은 해당 영역의 최댓값만 남기므로 “어느 위치에서 왔는지”는 잃지만, “그 패턴이 있었는지”는 남깁니다.

## 1-4. CNN 아키텍처의 흐름: LeNet → AlexNet → VGG → GoogLeNet → ResNet

| 모델 | 핵심 아이디어 |
|------|----------------|
| **LeNet** | conv + pooling + FC로 손글씨 분류, 딥러닝 부활의 시작 |
| **AlexNet** | ImageNet + GPU, ReLU·정규화, 큰 커널(11×11, 5×5) |
| **VGG** | **3×3 conv만 깊게 쌓음**, FC 파라미터가 대부분 차지 |
| **GoogLeNet** | Inception 블록, **1×1 conv**로 채널 압축(bottleneck) |
| **ResNet** | **residual connection**으로 매우 깊은 net 학습 |

### ResNet과 degradation problem

깊게 쌓기만 하면 **training error까지** 나빠질 수 있습니다. 이건 overfitting이 아니라 **optimization(degradation) 문제**입니다. identity mapping $H(x)=x$는 항상 가능한데, plain net은 그보다 못할 수 있다는 뜻입니다.

ResNet은 $H(x) = F(x) + x$ 형태로 **residual $F(x)$** 를 학습합니다. $F(x)\approx 0$이면 identity에 가까워져, 깊은 층도 최소한 얕은 net 이상은 되도록 유도합니다. vanishing gradient 완화에도 기여합니다(LSTM의 additive connection과 유사한 직관).

> ⭐ **시험 포인트 (확정 Q4 — ResNet)**  
> - degradation = capacity 문제가 아니라 **최적화 어려움**  
> - residual: $H(x)=F(x)+x$, **identity mapping을 쉽게 유지**  
> - vanishing gradient 완화, 100층 이상 학습 가능

## 1-5. FC ↔ Conv 해석

- Conv는 **weight sharing constraint**가 있는 FC의 special case.
- 마지막 FC를 feature map 크기와 같은 **conv kernel**로 바꾸면, **입력 해상도가 달라도** conv head는 동작 가능(분류 map → 평균 등).

## 1-6. Attention → Self-Attention → ViT

Attention은 query가 “지금 필요한 정보”를 key와 비교해 value를 가중 합치는 과정입니다. Self-attention은 query·key·value가 모두 같은 set에서 나오며, **permutation equivariant**합니다. 순서 정보는 **positional encoding**으로 보완합니다.

**ViT(Vision Transformer)** 는 BERT를 거의 그대로 가져온 것입니다. 이미지를 16×16 패치로 자르고 linear embedding 후 transformer encoder를 통과시킵니다.

> ⭐ **시험 포인트 (확정 Q3 — CNN vs ViT)**  
> **CNN**  
> - local convolution, **weight sharing**, **translation equivariance**  
> - 이미지 local structure를 구조적으로 활용 → **작은 데이터**에서 안정적  
>  
> **ViT**  
> - patch token + self-attention, **positional encoding 필수**  
> - inductive bias가 약해 **데이터·pretrain 규모가 커질수록** 유리  
> - ImageNet만으로는 ResNet보다 약했으나, **JFT-300M 등 대규모**에서 역전

### ViT 이후 변형(개념만)

- **PVT**: pyramid처럼 resolution을 줄이며 transformer 적용
- **Swin**: **window attention** + **shifted window**로 local·global 정보 교환, 효율성
- **NAS**: 아키텍처 자체도 탐색(현재는 foundation model 시대에 상대적 관심 감소)

온디바이스·저지연에는 CNN이 여전히 유효하고, 고해상도 특수 목적이 아니면 **vanilla ViT**가 dominant하다는 맥락도 언급되었습니다.

# 3부. 260618 오전 — 비디오 이해와 Action Recognition

## 3-1. 비디오 표현과 기본 개념

- 이미지: $H \times W \times 3$  
- 비디오: $H \times W \times T \times 3$ (4D tensor)
- **FPS**: 1초에 몇 프레임을 쌓는지. 높을수록 부드럽지만 용량·연산 증가
- **Trimmed video**: 관심 액션 구간만 잘린 클립 (UCF-101 등)
- **Untrimmed video**: 긴 영상 안 어디선가 액션 발생 (ActivityNet, THUMOS 등)

비디오는 **spatial + temporal** 패턴을 함께 봐야 합니다. 카메라 모션과 객체 모션이 합쳐진 **temporal dynamics**가 핵심 단서입니다.

## 3-2. Optical Flow

**Optical flow** = 연속 두 프레임 사이 픽셀별 변위 $(u, v)$. dense per-pixel motion입니다.

- **Brightness constancy**: 짧은 시간 안 색·밝기 유사
- **Aperture problem**: 한 픽셀·작은 패치만 보면 motion 방향이 **여러 개**로 해석될 수 있음 → 주변 픽셀을 묶어 over-determined system으로 풂

> ⭐ **시험 포인트 (확정 Q2)**  
> - flow 한 장 = **(u, v) 2채널**  
> - $L$장 스택 → **$2L$ 채널** 입력  
> - 정지 **appearance**가 아닌 **단기 움직임(방향·크기)** 제공  
> - 배경(정지)은 억제되고 **움직이는 객체**가 강조됨

## 3-3. 데이터셋 한눈에

| 데이터셋 | 특징 |
|----------|------|
| UCF-101 | 101클래스, 짧은 trimmed, 초기 벤치마크 |
| HMDB-51 | 51 human motion 클래스 |
| Kinetics-400/600 | ~10초 클립, action recognition **대표** 벤치마크 |
| ActivityNet | untrimmed + temporal localization |
| Something-Something | **모션·방향**이 중요, single frame으로는 불가능한 클래스 의도적 포함 |

> ⭐ **시험 포인트**  
> Kinetics는 **무엇(appearance)** 이 중요한 클래스가 많고,  
> **Something-Something**은 **어떻게 움직이는지**를 반드시 봐야 맞출 수 있어  
> “비디오가 정말 필요한가?”를 검증하는 벤치마크로 강조됨.

## 3-4. Action Recognition의 핵심 질문

이미지 분류와 달리 **멀티 프레임에서 feature를 어떻게 뽑을 것인가** + **연산 비용**이 핵심입니다.

초기 실험(Single frame vs Early/Late/Slow fusion)에서 **single frame만으로도 꽤 높은 성능**이 나왔습니다. 이는 기존 벤치마크가 appearance에 치우쳤다는 신호이고, Something-Something이 나온 배경입니다.

> ⭐ **시험 포인트 (확정 Q1)**  
> **단일 RGB CNN**  
> - 한 프레임 = **appearance만**, 시간·움직임 정보 없음  
> - 정지 화면이 비슷한 행동 구분 어려움  
>  
> **Two-Stream**  
> - **Spatial stream**: RGB → 외형  
> - **Temporal stream**: optical flow → 움직임  
> - “무엇이 보이는가” + “어떻게 움직이는가” **상보적(complementary)** 단서를 분리 학습 후 fusion

### Two-Stream과 Fusion

- 기본: 두 stream **독립** 학습 후 **class score fusion**(앙상블)
- **Spatial-Temporal Fusion** (같은 net 안): sum, max, concat, bilinear, **3D conv(comp fusion)** 등
- Bilinear fusion은 표현력은 크지만 $D^2$로 차원 폭발 → compact bilinear 등으로 축소

Temporal stream 입력: 연속 프레임 쌍의 flow를 **$L$장 스택** → **$2L$ 채널**.

## 3-5. 3D CNN, R(2+1)D, 한계

**3D convolution**: spatial + temporal 차원에서 동시에 sliding. C3D 등.

- 채널 수만큼만 “패턴 가짓수”가 정해져 **spatio-temporal 패턴 공간**을 다 커버하기 어렵다는 한계
- 데이터·연산 모두 heavy

**R(2+1)D**: 3D conv를 **2D spatial conv + 1D temporal conv**로 factorization → 더 data-efficient.

**Inflation**: ImageNet pretrained 2D weight를 temporal 차원으로 복제·나눠 3D conv 초기화.

## 3-6. Non-local, SlowFast, TSM

| 방법 | 요지 |
|------|------|
| **Non-local** | 모든 위치 쌍 관계 모델링 ≈ **self-attention** formulation |
| **SlowFast** | Slow(저FPS·고채널)=appearance, Fast(고FPS·저채널)=motion, 중간 fusion |
| **TSM** | 2D CNN 유지, **채널별 temporal shift**로 이웃 프레임 정보 교환, 거의 추가 연산 없음 |

## 3-7. Video Transformer: TimeSformer, ViViT

공통 이슈: full space-time attention은 **$O((HW)^2 T^2)$** 에 가까워 너무 비쌈.

- **Factorized attention**: spatial attention ↔ temporal attention 분리
- **Tubelet embedding**: 프레임별 grid가 아니라 **시공간 tube** 단위 토큰화가 유리할 수 있음
- Kinetics처럼 scene 전체로 판단 가능한 데이터는 factorized encoder도 충분한 경우 많음

# 5부. 260619 오전 — 비디오 ViT 확장, 다양한 테스크, Representation Learning

## 5-1. ViViT 복습과 MultiViT

**ViViT** 이후 연구는 (1) **complexity** 줄이기, (2) **multi-scale** 토큰 처리입니다.

**MultiViT**: self-attention의 Q·K에 **pooling**을 적용해 해상도를 단계적으로 줄임. CNN의 pooling·receptive field 누적과 유사한 직관.  
※ 논문 그림은 downsample → conv → upsample 순이 맞고, 슬라이드 일부는 오류(교수님 확인).

**Swin(비디오)**: 3D window + **alternating shift**로 window 경계 정보 교환. 효율적인 local attention.

## 5-2. UniFormer — CNN + Attention 하이브리드

ViT attention map을 보면 **lower layer에서 local·반복 패턴**이 많습니다. 굳이 매번 global self-attention으로 학습할 필요가 있나?

- **Lower layer**: **Local MHRA** = pointwise conv + **depthwise conv**(공통 attention pattern 학습) + pointwise conv  
  → depthwise conv의 weight = **고정된 local aggregation pattern**(redundant attention pattern 재활용)
- **Higher layer**: global **MHRA**(일반 transformer block)

비디오 버전은 이미지 UniFormer를 temporal inflation으로 확장.

## 5-3. StructViT — structure from attention map

Self-attention의 **affinity map**은 appearance가 아닌 **구조·관계·모션 패턴**을 담을 수 있습니다.

- 기존: affinity로 value를 가중 합
- StructViT: affinity map 자체에 **conv $U_k$** 를 적용해 **패턴**을 찾고, 그 패턴으로 aggregation  
- Something-Something에서 특히 이득 — **모션 구조**를 직접 활용

## 5-4. Encoder–Decoder 패턴과 비디오 테스크

많은 vision/speech 문제의 공통 구조:

```
입력 → Encoder backbone → Task-specific Decoder → 구조화된 출력
```

| 테스크 | 출력 구조 | 2D 대응 |
|--------|-----------|---------|
| Action Recognition | 클래스 1개 | Image classification |
| Temporal Action Detection (TAD) | 시간 구간 + 클래스 | Object detection (1D) |
| Spatio-Temporal Action Detection | tube + 클래스 | Object detection (2D+T) |
| Video Object Segmentation (VOS) | mask 시퀀스 | Segmentation + tracking |
| Action Anticipation | 미래 액션 클래스 | 조건부 generation |

> ⭐ **시험 포인트**  
> **TAD** = 2D object detection을 **시간 1축**으로 옮긴 문제.  
> DETR류 **learnable query** + encoder feature cross-attention이 최근 주류.

### TubeR / DETR for video

- **Action query**(learnable)가 encoder 비디오 feature와 cross-attention
- 출력: **시작 시각 $t$**, **duration $d$**, **class**
- **Actionness regression**: 배경·비액션 구간 필터 (DETR의 objectness와 유사)
- Diffusion 기반 TAD: 랜덤 temporal proposal에서 점진적 refine (generative formulation)

### XMem — Video Object Segmentation

초기 프레임 **mask**가 주어지면 이후 프레임에서 같은 객체 segmentation 유지.

- **Working memory**: 최근 프레임 key–value (value = **mask representation**)
- **Sensory memory**: RNN/LSTM으로 직전 hidden state 전달
- **Long-term memory**: working memory overflow 시 prototype 압축 저장

옛날 **online lightweight adapter** 방식은 feature encoder가 약해서 test-time adaptation이 필요했으나, 요즘은 강한 encoder + **memory로 이전 mask 정보 전달**이 핵심.

## 5-5. Self-Supervised Learning & Video MAE

레이블 없는 비디오가 훨씬 많습니다. **Self-supervised**는 인위적 **pretext task**로 목표를 만듭니다.

- **Masked Autoencoder (MAE)**: 비디오 패치 일부 마스크 → 나머지로 복원  
  → 주변 맥락·의미 이해를 장려
- **픽셀 복원보다 feature(HOG, motion) 예측**이 downstream에 더 유리하다는 finding
- **Distillation**: Image teacher(appearance·global semantic) + Video teacher(temporal distinctiveness)를 student가 동시에 mimic

## 5-6. ST-Adapter & LoRA — 효율적 transfer

**ST-Adapter**: Frozen ViT block 사이에 **lightweight adapter** 삽입. Spatial adapter 먼저, 이후 temporal adapter·temporal self-attention 추가. **Full fine-tune보다 적은 파라미터**로 비디오 적응.

**LoRA**: weight update를 low-rank $BA$로 제한. $W' = W + BA$. 대형 matrix 전체 fine-tune 대신 **residual만** 학습 → 비용·과적합 완화.

# 6부. 260619 오후 — 교수님 강의 (UniFormer2, 시험 관련 안내)

※ 오후 세션 앞부분만 교수님 강의입니다. 이후 AIM 실습(TA)은 제외.

## 6-1. UniFormer2

기존 UniFormer video는 **이미지 UniFormer 전체**를 temporal inflation해 pretrain해야 해서, 이미 학습된 **ViT 생태계(BERT-style transfer)** 를 쓰기 어렵습니다.

**UniFormer2** 아이디어:

1. **Pretrained ViT** encoder는 그대로 freeze 또는 활용
2. **Temporal Adapter** 자리에 UniFormer의 **Local Multi-Head Relation Attention**을 inflation
3. **Global Uni-Block** 추가: frame별 feature를 query로, **전체 spatio-temporal feature**와 **global cross-attention** → spatial·temporal이 분리된 adapter만으로는 부족한 **통합 contextualization**

> ⭐ **시험 포인트**  
> UniFormer2 = **ViT pretrained 가중치 재활용** + **local MHRA(temporal)** + **global uni-block(cross-attn)**

## 6-2. 교수님 시험·학습 관련 코멘트

- 행정실 요청으로 **“100문항” 등 구체 수치** 언급은 자제
- 대신 **테스크별 특성**을 이해하는 것이 중요:
  - **Tracking/VOS**: 초기 **target(mask/box)이 주어지는** 조건부 테스크
  - **Localization/Detection**: 비디오 전체에서 **이벤트 구간·위치**를 찾는 테스크
  - **TAD**는 본질적으로 **object detection과 같은 detection formulation**
