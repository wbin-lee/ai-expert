# 1부. 260629 오전 — 멀티모달 러닝과 Vision-Language의 큰 그림

## 1-1. 왜 멀티모달인가

서홍석 교수님은 멀티모달 러닝을 **두 가지 이상의 데이터 타입**을 함께 쓰는 학습으로 정의합니다. 이미지+텍스트, 오디오+텍스트는 물론, 같은 문자열이라도 JSON과 자유 텍스트는 **서로 다른 모달리티**로 볼 수 있습니다.

사람을 모델로 생각하면, 인텔리전스는 **여러 감각을 동시에** 씁니다.

- 눈이 안 보이면 소리·촉각으로 공간을 이해합니다.
- 아이는 보기만 하지 않고 만지고, 먹어보며 시각·촉각·미각을 함께 학습합니다.
- **McGurk effect**: 같은 소리 “바바바”도 입 모양(비주얼)에 따라 “바” 또는 “파”로 다르게 들립니다. 청각만으로 이해하는 것이 아닙니다.

AGI를 지향한다면, 세상 데이터 자체도 이미 멀티모달합니다. 웹 페이지, 비디오(프레임+오디오+자막+메타정보)를 한 모달리티만으로 설명하는 것은 부자연스럽습니다.

> ⭐ **시험 포인트**  
> 이 수업의 초점 = **Vision + Language**.  
> Vision = perception(환경 인식), Language = communication(사람과의 소통). AI도 결국 사람과 대화하려면 언어가 핵심 미디엄입니다.

## 1-2. Vision-Language 태스크 지도

| 태스크 | 한 줄 설명 |
|--------|-----------|
| **Visual QA (VQA)** | 이미지 + 질문 → 답 (단일 턴) |
| **Visual Dialog** | 멀티턴 QA — 앞 대화·대명사 맥락 이해 필요 |
| **Captioning** | 이미지 → 설명 문장 (모달리티 변환) |
| **Text-to-Image / Retrieval** | 텍스트 ↔ 이미지 생성·검색 |
| **Grounding / Referring Expression** | 텍스트로 지목한 객체의 위치·마스크 예측 |

이번 강의의 중심은 **VQA**와 **Image Captioning**입니다.

## 1-3. 전처리 리캡 — 이미지·텍스트를 모델에 넣기까지

### 이미지

- 3D 배열 → 고정 크기로 resize, 값 정규화
- **Augmentation**: color jittering, random crop(학습) / center crop(테스트), rotation 등
- 학습 시 random crop, 테스트 시 center crop으로 **분포를 맞추는 것**이 일반적

### 텍스트

- 원본: 문자 시퀀스 → **토크나이제이션** → 토큰 시퀀스
- 토크나이제이션 방식: character / word / **subword(BPE)** — OOV·복수형 문제 완화
- 인코더: LSTM·GRU 또는 **Transformer**
- 생성: **autoregressive** — 이전 토큰 조건으로 다음 토큰 예측, Transformer는 **causal masking**

> ⭐ **시험 포인트**  
> 텍스트는 반드시 **토크나이제이션**을 거칩니다. RNN의 vanishing gradient → **LSTM**(cell state + hidden state).  
> Answer generation = 오토리그레시브 = **스텝별 분류**의 연속.

## 1-4. VQA — 왜 유연하고, 왜 어려운가

VQA는 **하나의 포뮬레이션**으로 여러 비전 태스크를 풉니다.

- “What is this?” → 객체 분류
- “What is she doing?” → 행동 인식
- “Is this elephant?” → 검증(yes/no)
- “What color is the teapot?” → **참조(reference) 해석** 후 속성 예측

### 핵심 챌린지

1. **Reference problem**: 여러 객체 중 “무엇”을 묻는지 찾기
2. **Spatial / semantic relation**: “그 여자가 들고 있는 것” — 공간·의미 관계
3. **Zero-shot / few-shot**: 학습에 없던 단어·표현
4. **Counting 등**: 정답 숫자보다 **세는 과정**이 중요한 weakly supervised 문제

### 일반 구조

```
이미지 인코더 + 텍스트 인코더 → 멀티모달 퓨전 → 답 생성
```

- 초기: **고정 답 클래스 집합**으로 flat classification (평가·노이즈 때문에)
- 현재: **LLM/BLM**으로 autoregressive answer generation이 자연스러운 포뮬레이션

## 1-5. VQA 데이터셋

### VQA v1

- MS COCO 이미지 + crowdsourcing QA
- 질문당 **10개 답** 수집 → 시맨틱 모호성 완화
- 대부분 **단어 한 개** 답 → classification으로 풀기 reasonable
- 평가: 10명 중 몇 명과 일치하느냐 → 3명 이상이면 1점, 그 미만은 부분 점수

### Language Bias → VQA v2

- Yes/No 질문에 **Yes 편향** 심함 → 이미지 없이 Yes만 찍어도 60~70%
- **Balanced eval set**: 답 분포 균형
- **Complementary images**: 같은 질문에 답이 바뀌는 이미지 쌍 → 진짜 비전 이해 검증

> ⭐ **시험 포인트**  
> VQA v2의 동기 = **language bias** 제거·이미지 의존성 검증.  
> “바나나 색?” → 텍스트만으로 Yellow, 실제 초록 바나나는 Green이어야 함.

### CLEVR

- **시뮬레이터 렌더링** → GT·위치·규칙 기반 분석 가능
- 목적: **spatial reasoning**, **compositional generalization**
- 학습: 빨간 고양이·파란 개만 봤을 때, 테스트 **빨간 개** 인식 가능한가?
- 색·속성 조합을 train/test에서 **의도적으로 분리**해 일반화 측정

> ⭐ **시험 포인트**  
> CLEVR = 분석용 synthetic 데이터. **Compositional generalization**이 제일 중요한 이슈.

## 1-6. VQA 방법의 진화

### 초기: Concat + FC

- CNN 이미지 + question one-hot → concat → FC → classification  
- 텍스트를 one-hot만 쓰면 당연히 부족

### RNN 퓨전

- 이미지를 LSTM **첫 hidden**으로만 넣으면 → gradient가 멀리서만 옴 (**vanishing**)
- 개선: **매 토큰마다** 이미지 concat

### Visual Attention (핵심)

- 질문 = Query, 이미지 피처 = Key/Value
- “What color are the cows?” → **cow 영역 먼저**, 그 안에서 color
- **Stacked Attention**: 참조를 한 번에 못 찾으면 여러 번 refine
- **Dual Attention**: 텍스트→이미지뿐 아니라 **이미지→텍스트**도

> ⭐ **시험 포인트**  
> VQA에서 attention = **reference resolution**(타겟 찾기). Stacked·Dual attention의 직관을 설명할 수 있어야 합니다.

### 구조·파라미터를 동적으로

| 방법 | 아이디어 |
|------|----------|
| **Neural Module Networks** | 질문 syntax parse → **모듈 그래프를 동적 구성**(attention, combine, classify 등) |
| **Dynamic Parameter Prediction (FiLM 계열)** | 네트워크 구조 고정, **질문에 따라 FC 가중치 생성** — 해싱으로 파라미터 수 절감 |
| **Relation Networks** | 모든 spatial location **pair** + 질문으로 관계 모델링 |
| **FiLM** | 질문 벡터로 이미지 feature map을 **채널별 scale·shift**(modulation) — DiT 등에도 사용 |
| **MCB** | Bilinear pooling을 **FFT로 compact**하게 — 풍부한 cross-modal interaction |

### Bottom-up / Top-down Attention

- **Bottom-up**: object detector로 **의미 단위 region** 추출 (objectness, class-agnostic proposal)
- **Top-down**: 그 region들 중 질문 관련 영역 **soft attention**
- VQA·캡셔닝 모두 성능 향상

### DeVQA — Language Bias Debiasing

- **두 갈래**: (1) image+question, (2) **question-only**
- question-only로도 맞는 샘플 → 텍스트 bias → **가중치 축소**
- question-only는 틀리는 샘플 → 이미지를 봐야 함 → **가중치 강화**

> ⭐ **시험 포인트**  
> Bottom-up = detector region / Top-down = 그중 선택 attention.  
> DeVQA = dual path + **sample reweighting**으로 language bias 완화.

# 2부. 260629 오전 (후반) — Image Captioning

## 2-1. 문제 정의

이미지 $I$가 주어졌을 때, 문장 $S$의 확률 $P(S \mid I)$를 최대화.  
**Chain rule**로 분해 → **autoregressive** 토큰 생성.

### MS COCO

- 이미지당 **5개 캡션** (다양한 표현·평가용)
- Stanford Fei-Fei 팀 대규모 데이터 + neural generation → 캡셔닝 연구 활성화

## 2-2. 평가 지표 — 왜 까다로운가

GT가 하나가 아니고, “틀린 표현”이 아닐 수 있음 → **여러 reference** 필요.

| 지표 | 핵심 |
|------|------|
| **BLEU** | n-gram precision (MT에서 유래) |
| **METEOR** | alignment 기반 |
| **CIDEr** | n-gram + **TF-IDF** 가중 — 도메인에서 중요한 n-gram에 높은 점수 |
| **SPICE** | 텍스트·이미지 **scene graph** 유사도 — GT 캡션만이 아니라 **이미지 내용** 반영 |

최근에는 **CLIP score** 등 foundation model 기반 평가도 시도.

> ⭐ **시험 포인트**  
> CIDEr = TF-IDF로 n-gram **중요도** 반영 (조사 vs content word).  
> SPICE = scene graph로 **이미지-텍스트** 정합성.

## 2-3. Show and Tell → Show, Attend and Tell

### Show and Tell

- CNN → single vector → RNN decoder (머신번역과 동일, 입력만 이미지)
- 문제: 이미지 정보가 **처음 한 번만** 들어감 → 긴 문장 후반은 언어 모델만 의존, **hallucination**

### Show, Attend and Tell

- 매 스텝 **soft attention**으로 관련 이미지 영역 aggregate 후 디코딩

### Hard Attention

- attention map에서 **하나의 location 샘플링** → gradient 끊김
- **Policy Gradient(REINFORCE)** 로 학습 — Neural Module Networks layout learning과 동일한 spirit
- 당시 hard가 soft보다 나은 경우도 있었으나, **optimization·exploration**이 어려워 soft가 주류

### Retrieval-augmented attention

- 다음 단어를 모르는데 “어디를 볼지” 모호 → **비슷한 이미지의 기존 캡션**을 retrieval해 힌트로 사용

### Bottom-up Attention in Captioning

- Object proposal feature + top-down attention → **semantic level** 캡셔닝에 유리 (VQA와 동일 논문)

## 2-4. 캡셔닝의 근본적 딜레마 (260630으로 이어짐)

1. GT가 **unique하지 않음**, 완벽하지도 않음
2. 학습 loss = **cross-entropy (MLE)** ↔ 평가 = **BLEU/CIDEr** 등 → **misalignment**
3. 평가 지표는 **non-differentiable** → 직접 최적화 불가

→ 해결 시도: **Reinforcement Learning**

# 3부. 260630 오전 (전반) — Captioning + Reinforcement Learning

## 3-1. 왜 RL인가

**Differentiable vs non-differentiable**가 핵심.

- MLE: log probability에 gradient → backprop 가능
- BLEU/CIDEr: 이산 토큰 시퀀스 → **gradient 끊김**
- RL의 기본 정신: **exploration** — 해보고 reward 높으면 encourage, 낮으면 discourage

> ⭐ **시험 포인트**  
> RL 쓰는 조건: (1) 목표가 non-differentiable, (2) 정답 annotation 어렵지만 **좋은지 판단**은 가능.  
> LLM 시대: **생성은 어렵지만 평가는 상대적으로 쉬운** 경우 RLHF 등에 활용.

### RL 기본 요소 (리캡)

- Agent(policy) — Environment — State — Action — **Reward** — **Return**
- **Policy Gradient / REINFORCE**: $\nabla \log \pi(a \mid s) \times R$

### Policy Gradient 실무 이슈

1. **좋은 초기점** 필요 → MLE로 **pretrain** 후 RL
2. **Baseline**: 항상 positive reward면 variance 큼 → 기준값 빼서 positive/negative 구분
3. **Exploration**: argmax가 아니라 distribution에서 **sampling**

## 3-2. SCST — Self-Critical Sequence Training

- Reward = **CIDEr** (또는 BLEU, METEOR, ROUGE — objective에 따라 해당 metric 상승)
- **Baseline** = inference 시 **greedy/beam**으로 만든 자기 캡션
- 샘플링 캡션의 CIDEr > baseline → encourage, 아니면 discourage
- 논문명 그대로: **“자기 자신을 기준으로”** reward balancing

> ⭐ **시험 포인트**  
> SCST = CIDEr 등 **evaluation metric을 직접 reward**로 + **self-critical baseline**으로 variance 감소.  
> CIDEr로 학습하면 CIDEr 최대화; BLEU reward면 BLEU 상승 (다른 metric은 떨어질 수 있음).

## 3-3. Mixer & Critic

- **Mixer**: cross-entropy 기반 RL + evaluation metric
- 전체 시퀀스 reward만 쓰면 **한 토큰 실수로 전체 discourage** → **Critic(Q)** 으로 토큰별 quality estimation
- Critic = “이 상태·행동 이후 기대 return” 회귀

## 3-4. Human Rating & Off-Policy PG

- 궁극적 품질 = **human evaluation**
- **Caption Ratings** 데이터셋: machine-generated caption + human score
- 학습 중 매번 사람에게 물을 수 없음 → rating **estimator(critic)** 학습 후 proxy reward
- **Off-policy PG**: 현재 policy가 아닌 **데이터셋 distribution**에서 샘플 + importance weighting
- 문제: estimator mismatch → **reward hacking** (점 하나 같은 trivial caption)
- **PPO** 등: pretrained 모델을 크게 망가뜨리지 않는 범위에서 reward 최대화

# 4부. 260630 오전 (후반) — General Multimodal & Foundation Models

## 4-1. 멀티모달 학습의 두 갈래

| 접근 | 활용하는 정보 |
|------|--------------|
| **Cross-modal supervision** | 한 모달리티로 다른 모달리티 representation **예측** (redundant) |
| **Fusion** | 두 모달리티 **결합** (complementary) |

멀티모달 확장의 어려움: annotation 비용, 데이터 수집, computation.

## 4-2. Foundation Model이란

- 대규모 데이터로 사전학습된 large model
- **핵심 특성 = Adaptiveness** — 다양한 downstream·프롬프팅으로 적응
- “Large scale”, “self-supervised”는 수식어; **adaptiveness**가 본질
- 적응 방식 변화: task-specific **fine-tuning** → **prompting**(튜닝 없이)

> ⭐ **시험 포인트**  
> Foundation model의 키 = **Adaptiveness**.  
> LLM 프롬프팅뿐 아니라 vision의 **visual prompting**(SAM point/box 등)도 동일 계열.

## 4-3. 웹 스케일 데이터셋

| 데이터셋 | 특징 |
|----------|------|
| **Conceptual Captions** | 웹 이미지 + **alt-text** 크롤링, 필터링·후처리 → 3.3M |
| **CLIP training set** | alt-text 기반 **400M** image-text pairs |
| **ALIGN** | Google, **1.8B** pairs, minimal filtering |
| **HowTo100M** | YouTube **instructional** video — visual·speech 정렬 |
| **WebVid** | 스톡 비디오 사이트 — 태깅된 텍스트 큐 |
| **VideoCC** | CC 이미지-캡션으로 유사 프레임 찾아 **비디오에 캡션 transfer** |

> ⭐ **시험 포인트**  
> 웹 멀티모달 데이터의 핵심 = **alt-text**(사람이 작성한 이미지 설명).  
> Instructional video는 visual·narration **alignment**가 좋아 멀티모달 학습에 유리.

## 4-4. 대표 모델

### ViLBERT

- BERT spirit → **dual stream** + **cross-attention** (양방향)
- 학습: masked LM + **masked frame prediction**
- 하나의 모델로 VQA, VCR, referring, retrieval 등

### CLIP (Contrastive Language-Image Pre-training)

- 배치 내 matched pair = positive, 나머지 = negative
- **Symmetric cross-entropy** on dot product → matched embedding **같은 방향**
- 활용:
  - Image-text **retrieval**
  - **Zero-shot classification**: `"a photo of a {class}"` 텍스트 embedding과 이미지 similarity
  - **Domain robustness**: ImageNet 학습 모델보다 sketch 등 domain shift에 강함
- **Linear probing**: 마지막 classifier layer만 학습해도 strong baseline

> ⭐ **시험 포인트**  
> CLIP = **contrastive learning** + 웹 스케일 데이터.  
> Zero-shot = 프롬프트 템플릿(`a photo of a ...`) + **가장 높은 similarity** 클래스 선택.  
> Few-shot보다 zero-shot이 나은 경우도 많음 (domain robustness).

### CoCa

- Contrastive + **generative** decoder — retrieval과 caption generation 동시

### OFA

- 비전 태스크를 **모두 텍스트 토큰**으로 통일 → 지금 BLM들의 선행 접근

### Flamingo

- **Pretrained LLM** + **Perceiver Resampler**(이미지 토큰 압축) + **gated cross-attention**
- **Zero-init gating**: 처음엔 LLM 출력 그대로 유지 → 점진적 vision 적응
- Interleaved image-text 데이터, zero-shot·few-shot VQA 등

### CLIP 응용 — Segmentation

- **CLIPSeg**: seen class로 mask proposal 학습 + **CLIP text embedding**으로 unseen class
- **ZegFormer / IFSeg**: OFA 등으로 dummy image embedding → zero-shot seg
- **Referring segmentation**: global+local feature — 관계 표현("lying on the seed of the scooter")에 local context 필요

## 4-5. 비디오·오디오 확장

| 모델 | 요약 |
|------|------|
| **VideoBERT** | ViLBERT를 비디오로 — frame tokenization + masked modeling |
| **MMT** | 비디오 retrieval — visual+audio+speech **multi-modality transformer**, pretrained BERT |
| **MERLOT / Coref MBT** | 대화·비디오 — visual 정보가 future utterance prediction에 유의미, **pretrained weight** 중요 |
| **Bottleneck tokens** | 모든 modality pair attention 대신 **bottleneck**으로 fusion → complexity↓ |
| **Frozen / LLaVA** | frozen LLM + **lightweight adapter**(projection)만 학습 → instruction VQA |
| **UniVLP** | 여러 loss 합산 — 현재는 **task를 텍스트 토큰으로 통일**이 더 scalable |

### Multimodal Video Captioning (MVGPT)

- 비디오+스피치 → 캡션 — visual·speech **complementary** 정보 모두 반영 필요

교수님이 강의에서 반복 강조한 개념 위주로 정리했습니다. (260629·260630 강의록에 “시험” 직접 언급은 없었습니다.)

## A. 멀티모달·Vision-Language 기본

1. **멀티모달 동기**: 사람·AGI·실세계 데이터는 멀티모달 — McGurk effect, 아이의 촉각 학습
2. **Vision+Language**가 핵심인 이유: perception + communication
3. VL 태스크 구분: VQA / Dialog / Captioning / T2I / Grounding
4. **Cross-modal supervision vs Fusion** 차이

## B. VQA

5. VQA가 여러 비전 태스크를 **하나의 QA**로 통합
6. **Reference problem**, spatial·semantic relation, counting
7. 초기 **flat classification** vs 현재 **autoregressive generation**
8. **VQA v1** 평가(10 answers, 3/10 rule) / **VQA v2** (balanced, complementary images, **language bias**)
9. **CLEVR**: synthetic, spatial reasoning, **compositional generalization** (빨간 개 테스트)
10. 방법 진화: concat → RNN fusion → **Visual Attention** → Stacked/Dual → Module Networks / Dynamic params / **Relation Networks** / **FiLM** / **MCB**
11. **Bottom-up**(detector region) + **Top-down**(attention) — VQA·캡셔닝 공통
12. **DeVQA**: image+question vs question-only dual path, **reweighting**

## C. Image Captioning

13. MLE = chain rule, conditional token prediction
14. **MS COCO** — 이미지당 5 captions
15. 평가: **BLEU, METEOR, CIDEr(TF-IDF), SPICE(scene graph)**
16. **Show and Tell** vs **Show, Attend and Tell** — vanishing gradient·attention
17. **Hard attention** + **Policy Gradient** — gradient 끊김
18. 학습(MLE) vs 평가(metric) **misalignment** → **RL** 동기

## D. Captioning + RL

19. Non-differentiable → **REINFORCE / Policy Gradient**
20. **Baseline**·pretrain·exploration(sampling)의 필요성
21. **SCST**: CIDEr reward + **greedy/beam self-critical baseline**
22. **Human rating**, off-policy PG, reward hacking, **PPO** 언급

## E. Foundation Models

23. Foundation model 키 = **Adaptiveness** (fine-tune → **prompting**)
24. 웹 데이터: **Conceptual Captions, CLIP 400M, ALIGN 1.8B**, HowTo100M, WebVid, VideoCC
25. **ViLBERT**: dual stream, cross-attention, masked LM + masked frame
26. **CLIP**: contrastive, zero-shot (`a photo of a {class}`), domain robustness, linear probing
27. **CoCa**: contrastive + generative
28. **OFA / BLM**: 모든 태스크를 **텍스트 토큰**으로
29. **Flamingo**: frozen LLM + Perceiver Resampler + **zero-init gated cross-attention**
30. BLM 성공 배경: 비전 커뮤니티의 **대규모 데이터 수집·변환** 노력

## F. 응용·확장

31. CLIP 기반 **zero-shot segmentation** (seen/unseen class 구분)
32. 비디오: **MMT** — visual+audio+speech fusion이 중요
33. **Pretrained weight** 없으면 멀티모달 gain 제한적 (Coref MBT)
