# 1부. 260701 오전 — 3D 세상을 어떻게 볼 것인가

## 1-1. 이 수업이 다루는 것

박재식 교수님은 서울대 컴퓨터공학부에서 **3D 비전(3D Scene Understanding)** 을 가르칩니다. 생성 모델 수업과 달리, 여기서는 **3D 데이터를 어떻게 획득·인지·응용하는가**에 집중합니다.

3일간의 여정은 대략 이렇습니다.

```
3D 데이터 획득·인식 (Point Cloud, Sparse Conv)
    → 3D 응용 (Detection, Segmentation)
        → Neural Implicit (SDF, NeRF)
            → Gaussian Splatting (실시간 복원)
                → 응용 (Feature GS, Dynamic GS)
```

## 1-2. Depth Map: 픽셀마다 거리를 재다

우리가 흔히 보는 RGB 이미지는 “무슨 색인가”를 담습니다. **Depth Map**은 같은 크기의 이미지인데, 픽셀마다 **카메라로부터의 거리**를 담습니다. 가까울수록 파란색, 멀수록 빨간색처럼 시각화하기도 합니다.

- 한 장의 사진에서 **모든 픽셀의 거리**를 한 번에 측정
- 값은 보통 **밀리미터 단위**, 10~12비트로 저장
- Depth Map 여러 장 → **3D 복원** 가능

그런데 검게 나온 픽셀이 있습니다. **측정 불가 영역**입니다.

- 코너·무특징 벽면
- **유리** — 빛이 반사되어 거리 측정이 어려움

> ⭐ **시험 포인트**  
> Depth Map = 픽셀당 거리. RGB와 해상도는 같지만 **저장 의미가 다름**.

## 1-3. 네 가지 3D 센서 — 사람 눈에서 시작

여러분 모두 **스테레오 센서**를 가지고 있습니다. 바로 **양쪽 눈**입니다.

손가락을 눈 앞 10~20cm에 대고 한쪽 눈만 감았다 뜨면, 손가락이 옆으로 움직이는 것처럼 보입니다. 이게 **시차(parallax)** 입니다. 가까울수록 시차가 크고, 멀수록 작습니다. 천문학에서 별까지의 거리도 이 원리로 잰다고 교수님이 강조하셨습니다.

| 센서 | 원리 | 특징 |
|------|------|------|
| **Passive Stereo** | 양안 시차 (사람 눈 모방) | 저렴, **무특징 벽**은 매칭 불가 |
| **Active Stereo** | + 중앙 **프로젝터**로 패턴 투사 | 무특징 표면도 가능, HW 복잡 |
| **Structured Light** | IR 패턴 투사 + 카메라 (Kinect) | 실시간 30fps, 가정용 |
| **Time of Flight (LiDAR)** | 레이저 왕복 **위상**으로 거리 | 50m+, mm 정밀, 자율주행 |

실무에서 가장 많이 쓰는 두 가지는 **Structured Light**와 **ToF/LiDAR**입니다.

### Kinect의 역사 (재미있는 사실)

2010년 Kinect 포즈 추정 논문은 AlexNet(2012)보다 이릅니다. 당시엔 **딥러닝 없이 Random Forest**로 Depth 패치 → 3D 포즈를 추정했습니다.

## 1-4. 자율주행의 두 길: Waymo vs Tesla

| | Waymo (LiDAR) | Tesla (영상만) |
|---|---------------|----------------|
| 센서 | LiDAR로 **물리적 거리 측정** | 카메라만으로 추론 |
| 안정성 | 높음, 사고 적음 | 탈옥 쉬움 (현수막 등) |
| LiDAR 역할 | **실제 추론에 사용** | **Ground Truth 학습용만** |
| 목표 | Driveable Area (주행 가능 영역) | Occupancy Map (점유 격자) |

테슬라가 Velodyne LiDAR를 대량 구매한 이유도 **학습용 GT** 때문이었습니다. 추론 때는 쓰지 않습니다.

> ⭐ **시험 포인트**  
> Tesla Occupancy Map = 영상만으로 3D 점유 예측. LiDAR는 **학습 라벨**로만 활용.

## 1-5. Depth → Point Cloud

Depth Map의 각 픽셀 값은 카메라 중심으로부터의 **Z(거리)** 입니다. 초점거리(Focal length)와 픽셀 위치를 알면 **(X, Y, Z)** 3D 좌표로 역투영할 수 있습니다. 이렇게 모인 점들의 집합이 **Point Cloud(포인트 클라우드)** 입니다.

## 1-6. 3D의 본질적 난제: Sparse(희소) 데이터

2D 이미지는 **Dense** — 모든 픽셀에 값이 있습니다.  
3D는 **Sparse** — 3D 공간의 약 **3%만** 물체 표면이 차지합니다.

> ⭐ **시험 포인트**  
> 3D의 핵심 관건 = **Sparse 데이터를 어떻게 처리하느냐**.

### 표현 방법 비교

| 방법 | 아이디어 | 장단점 |
|------|----------|--------|
| **Voxel** | 3D 공간을 큐브로 분할 | 3D CNN 가능, 해상도↑=무거움 |
| **PointNet** | 이웃점 수집 → MLP → **Max Pooling** | 원시적, KNN **O(N log N)** |
| **Voxelization** | 포인트→복셀, 빈 복셀 제거 | voxel size 선택이 중요 |
| **Sparse Convolution** | **채워진 복셀만** convolution | **가장 빠름**, Hash로 **O(N)** |

### Sparse Convolution의 핵심

**Indicator matrix** — 값이 있는(occupied) 복셀만 골라서 convolution합니다.  
PointNet의 KNN은 “우주에서 별 찾기”(**O(N log N)**), Hash grid는 “호텔 방번호 조회”(**O(1)**)에 비유하셨습니다.

**MinkowskiNet**은 Sparse Conv의 표준 구현으로, 속도 베이스라인으로 계속 비교됩니다.

## 1-7. PointNet → Point Transformer

- **PointNet**: 이웃점 상대좌표 + MLP + **동일 가중치 pooling**
- **Point Transformer**: pooling 대신 **Attention weighted sum** (Q, K, V)

차이는 **어그리게이션 하나**지만, 자유도와 성능이 크게 달라집니다.

### Point Transformer v3 (최근)

KNN·상대위치 인코딩 병목을 **Space-filling curve**(Z-order, Hilbert)로 직렬화 → 언어 모델처럼 처리. 교수님은 성능은 올라가지만 **재현성·실속도**에 의문을 표하셨습니다.

## 1-8. 3D Foundation Model의 필요성

Velodyne 64채널로 학습한 모델에 32채널 데이터를 넣으면 **완전히 망가집니다**. 센서·씬마다 모델을 따로 쓰는 게 현실인데, **하나의 통합 인코더**가 필요합니다.

- **Sonata**: PTv3 pretrained + Self-supervised + VLM
- **SpinNet** (교수 연구): **PCA**로 물체별 불변 좌표계 → 센서·씬에 관계없이 매칭

# 2부. 260701 오후 — 3D를 어디에 쓰는가

## 2-1. Segmentation과 Detection은 형제

같은 **인코더**를 공유하고 **헤드만 바꾸면** 됩니다. Segmentation이 가장 쉬운 응용입니다.

## 2-2. 3D Object Detection — 7 DoF

3D 물체 하나를 표현하려면 **7자유도(7 DoF)** 가 필요합니다.

1. 중심 **(X, Y, Z)** — 3
2. 크기 **(W, D, H)** — 3
3. **Yaw** (바닥 수직축 회전) — 1

가정: 물체가 **바닥에 붙어 있음** → 회전축은 1개만.

> ⭐ **시험 포인트**  
> **문제 정의가 중요** — 7 DoF, yaw 1개만 쓰는 간략화부터 시작.

## 2-3. VoteNet — 2D 아이디어의 3D 확장

**2D에서**: 부위별 feature(엉덩이·머리)가 **물체 중심으로 vote**(회귀). 잘못된 vote는 산발적 → **클러스터링**으로 진짜 center 수렴.

- Backbone = **PointNet**, XYZ만으로도 geo-only 성능↑
- **한계**: **평평·넓은 물체**(책장, 책상) — 유사 feature → vote 실패
- **Single Shot 아님** — 중간 **클러스터링**이 비미분

## 2-4. GSDN — End-to-End Single Shot

- Sparse **3D U-Net** + 업샘플 시 **복셀 Generate**(살릴지 분류)
- **Pruning**: 물체 center 해당 복셀만 남김
- 각 복셀 → **7 DoF bbox 회귀**
- **Multi-scale head**: 저해상도=큰 물체, 고해상도=작은 물체
- **End-to-End, Single Shot** — 클러스터링 없음

## 2-5. PointPillars — “상식을 깨는” 아이디어

자율주행에서 가장 많이 쓰이는 방법입니다.

1. 포인트를 **기둥(Pillar)** 로 쌓음 — BEV(위에서 본 뷰)에서 1픽셀 = Z방향 여러 층
2. → **Pseudo-image** 생성
3. → **2D CNN**(SSD/YOLO head)로 detection

**초당 60Hz** — Spinning LiDAR 실시간 detection.

> ⭐ **시험 포인트**  
> PointPillars 핵심 = 3D 고집 말고 **이미 잘 되는 2D detector에 맞게 변환**.  
> Pillar → pseudo-image → YOLO/SSD 재사용.

| | VoteNet | GSDN | PointPillars |
|---|---------|------|--------------|
| 방식 | Vote + 클러스터링 | Sparse U-Net | Pillar → 2D |
| E2E | ✗ (클러스터링) | ✓ | ✓ |
| 속도 | 보통 | 보통 | **60Hz** |

## 2-6. Open Vocabulary · SAM · Grasping (흐름만)

- **Open Vocabulary 3D**: CLIP으로 Text ↔ 2D ↔ 3D point cloud 정렬 → 고정 20클래스 대신 **임의 텍스트**로 detection
- **SAM → 3D**: 2D SAM segmentation → 3D point에 정렬 → 로봇 manipulation
- **GraspGen**: Point cloud → gripper pose 예측 → 로봇 grasping

교수님 말씀: Detection/Segmentation만 연구하는 시대는 지났고, **open vocab + localization + 실제 grasping**까지 가야 합니다.

# 3부. 260702 — Neural Implicit Representation

> 260702 **오전**은 TA 실습(SIREN, SDF, Tiny NeRF 코드)이므로 제외했습니다.  
> **오후** 교수님 이론 강의 + 오전 실습 복습을 정리합니다.

## 3-1. 왜 Neural Implicit인가?

내일 배울 **Gaussian Splatting**으로 가는 **다리**이기도 하고, 그 자체로 재미있는 문제 정의입니다.

### 일반 ML vs Neural Implicit

| | 일반 ML (분류 등) | Neural Implicit |
|---|-------------------|-----------------|
| 목표 | **Generalization** (새 데이터) | **Fitting** (이 데이터 하나) |
| Validation | **필수** (overfitting 경계) | **불필요** — overfitting이 **목적** |
| 학습 | Training curve + validation curve | Loss만 minimize |

> ⭐ **시험 포인트**  
> Neural Implicit = **의도적 overfitting**. 하나의 이미지·장면을 네트워크에 **memorization**.

### 왜 굳이 fitting?

- **장점**: 10MB 이미지 → **~1KB** network weights (압축·코덱 패러다임)
- **단점**: 이미지/장면 **하나당** 네트워크 하나, 복원 시 **모든 픽셀 inference** (느림)

## 3-2. Explicit vs Implicit

| | Explicit | Implicit |
|---|----------|----------|
| 저장 | 배열로 직접 (point cloud N×3, pixel grid) | **함수**로 표현 |
| 예시 | Point Cloud, Voxel, **Gaussian** | MLP로 학습한 SDF, NeRF |
| 장점 | 바로 사용·export | 연속 query, 압축 |
| 단점 | 메모리↑ | export에 **후처리** 필요 |

## 3-3. SIREN — ReLU가 안 되는 이유

MLP + ReLU는 **저주파**만 잘 표현합니다. 자연 신호(이미지, 3D)는 고주파 디테일이 많습니다.

**SIREN**: **\(\sin\)(주기 함수)** 를 activation으로 사용 → 고주파·미분(gradient, Laplacian) 표현 ↑

| | ReLU MLP | SIREN |
|---|----------|-------|
| 용도 | 분류, general feature | **고주파 fitting** |
| 미분 | 불연속 | 닫혀 있음 → PDE·gradient loss 가능 |

NeRF는 SIREN 대신 **Positional Encoding**으로 고주파를 주입합니다.

> ⭐ **시험 포인트**  
> MLP+ReLU = 저주파만 / SIREN(sin) 또는 Positional Encoding = **고주파 표현**

## 3-4. Poisson Reconstruction — Gradient만으로 복원

GT 이미지 대신 **gradient만** 주고 원본을 복원할 수 있습니다 (Poisson image reconstruction).  
절대값은 **ambiguity** — constant offset을 모릅니다.

**Event camera**는 변화량(gradient)만 측정 → 저전력·고속 → Poisson reconstruction과 연결됩니다.

## 3-5. SDF (Signed Distance Function)

Point cloud만으로는 **watertight mesh**를 만들기 어렵습니다. SDF로 표면을 정의합니다.

- **표면** = SDF 값 0
- **내부** = 음수, **외부** = 양수
- 거리에 비례하는 값

**GT SDF는 없습니다** — point + normal만 주고 **constraint(제약)** 만으로 MLP를 최적화합니다. Classical **solver**를 MLP가 대체하는 관점입니다.

### 4가지 Loss (핵심 순)

1. **Surface** (|SDF|≈0) — **가장 중요**
2. Zero-level / off-surface penalty
3. **Eikonal** (|∇SDF|=1) — 등고선 간격 = 거리 비율 보존
4. **Normal alignment** — ∇SDF ∥ surface normal

> ⭐ **시험 포인트**  
> SDF = GT 없이 **제약만**으로 풂. Surface constraint가 핵심.  
> GT mesh는 **평가용**일 뿐, 실전에는 없음.

### Neural Implicit Function의 Input/Output

| 데이터 | Input | Output |
|--------|-------|--------|
| Point cloud | (x,y,z) | **SDF** |
| Image | (u,v) | **Intensity** |
| NeRF | (x,y,z, view_dir) | RGB + density |

**Input/Output 정의**가 가장 중요합니다.

## 3-6. NeRF — Multi-view에서 3D를 복원하다

### 문제 정의

**Novel View Synthesis**: 여러 각도 사진 → **새 시점**에서의 이미지 합성

### 왜 한 장으론 안 되나?

한 장 사진에서 ray 위 **어디에 물체가 있는지 모호**합니다. **Multi-view triangulation**이 필요합니다 (GPS 원리와 동일).

### NeRF 파이프라인

```
3D 좌표 (x,y,z) + 시선 방향 → MLP → RGB + density(σ)
    → Volume Rendering (ray marching + alpha blending)
    → 2D 이미지
    → GT 이미지와 MSE loss → MLP 업데이트
```

- **CG vs CV**: CG는 GT mesh → realistic image, CV는 images → recover 3D
- 2020 NeRF: **3일** 학습 → Instant-NGP 등으로 **초 단위**

> ⭐ **시험 포인트**  
> NeRF = **Implicit function** + **Volume rendering 역산**  
> 학습 = **per-scene optimization** (하나의 씬을 네트워크에 기억)

### Instant-NGP

- 큰 MLP 대신 **Hash Grid**에 feature 저장
- **작은 MLP**는 feature → radiance field **변환기**만
- GPU 병렬 + hash → **학습·렌더링 대폭 가속**
- 여전히 **implicit** → mesh export 어려움

# 4부. 260703 오전 — Gaussian Splatting, 3D 복원의 현재

## 4-1. NeRF에서 Gaussian Splatting으로

교수님이 송신탑 데모를 보여주셨습니다. 웹브라우저에서 **즉시 로드**, 노트북에서 **실시간** 시점 변경, 철탑 프레임 하나하나 **디테일 살아있음**.

어제 NeRF(수 초/프레임)와 달리, 이건 **Gaussian Splatting**입니다. “시간의 문제”라고 하셨을 만큼, 상용 서비스가 이미 나오고 있습니다.

## 4-2. 정적 씬 복원 — 문제와 파이프라인

**정적(Static) 씬**: 물체가 고정 → 여러 각도 사진 → 3D 위치 + 색 복원

```
멀티뷰 이미지 → 카메라 포즈(SfM/COLMAP) → 3D 표현 학습 → 새 뷰 렌더링
```

사진이 부족하면 생성 모델(Trellis, SAM3D 등)로 보완 가능합니다.

### 방법론 비교

| 방법 | 표현 | 렌더링 | 특징 |
|------|------|--------|------|
| **Multi-View Stereo** | 포인트/메쉬 | 느림 | 패치 매칭, triangulation |
| **NeRF** | Neural Implicit (MLP) | **수 초/프레임** | 포토리얼, 무거움 |
| **Instant NGP** | Hash Grid + 작은 MLP | NeRF보다 빠름 | 지글거림, implicit |
| **3D Gaussian Splatting** | **Explicit** 가우시안 | **100+ FPS** | 가볍고, 상용화 |

### 초기화의 중요성

- **랜덤 포인트**로도 수렴 가능하나 **배경/원거리** 약함
- **SfM 포인트** 초기화 → 뒤쪽 텍스처까지 잘 복원
- 비선형 최적화 → **좋은 initial**이 품질을 결정

## 4-3. 3D Gaussian Splatting (3DGS)

### NeRF 한계를 어떻게 극복했나

1. Nearest neighbor 탐색 느림 → **GPU rasterization**으로 대체
2. MLP inference → 가우시안 자체가 **color + opacity** 보유

가우시안은 GPU **geometric primitive**(mesh, triangle과 동일 계열)입니다. GPU가 원래 잘 하는 **alpha blending**을 그대로 활용합니다.

### 가우시안 파라미터

- **Position** (center)
- **Rotation, Scale** (covariance)
- **Spherical Harmonics** (view-dependent color)
- **Opacity**

### 학습 파이프라인

1. SfM 포인트(또는 랜덤)로 초기화
2. **Differentiable rasterizer**로 렌더링
3. GT 이미지와 photometric loss (MSE)
4. **Adam**으로 모든 가우시안 파라미터 업데이트
5. **Adaptive Density Control**
   - Gradient 큼 + scale 작음 → **Clone** (복제)
   - Gradient 큼 + scale 큼 → **Split** (분할)
   - 불필요/투명 → **Prune** (제거)

> ⭐ **시험 포인트**  
> 3DGS = **Differentiable rasterization** + **Adaptive density control**  
> **Explicit** → `.ply`로 바로 export, 뷰어에 drop

### 3DGS 장점 (교수·조교 공통 강조)

| NeRF (Implicit) | 3DGS (Explicit) |
|-----------------|-----------------|
| 네트워크에 정의 → export **후처리** 필요 | 파라미터가 **그대로 저장** (.ply) |
| MLP inference → 무거움 | **가벼움**, 실시간 |
| 네트워크 합성 불가 | 가우시안끼리 **union**으로 합성 |

### Vanilla 3DGS 한계

1. **Projection approximation** — 3D→2D 근사 부정확 → **floater**(떠다니는 가우시안), **needle** 아티팩트
2. **표면(surface) 없음** — 둥둥 떠 있는 구름, 게임 캐릭터가 걸을 평면 없음
3. **Novel view**에서 학습 뷰에 안 보이던 투명 가우시안이 지저분하게 보임

## 4-4. 2DGS vs GSDF — 표면을 어떻게 잡을 것인가

### GSDF (Dual Representation)

3DGS + **SDF branch**를 동시에 복원. Depth/normal + SDF 일치 loss, Eikonal loss.  
**무겁고** 학습 어렵습니다.

### 2DGS (교수: “근간을 흔드는 아이디어”)

표면이 필요하면 SDF를 **붙이는 것**보다, **애초에 2D 가우시안 원판(디스크)**으로 표면을 복원합니다.

| | 3DGS | 2DGS |
|---|------|------|
| 형태 | 3D 구 | **2D 원판** |
| 표면 | 없음 (구름) | **SDF 없이 mesh 추출** |
| 품질 | photometric 약간 우위 | geometry/surface 우위 |
| floater | 많음 | **감소** |

> ⭐ **시험 포인트**  
> 표면 복원 목적이면 **2DGS** 고려. GSDF = dual(GS+SDF), 2DGS = **표현 자체를 2D로 변경**.

## 4-5. Dynamic 3D (시간 축 확장)

정적 씬에서 **시간 축**으로 확장 — 비디오, 움직이는 액터.

- **DyNeRF**: NeRF + time latent
- **4D Gaussian Splatting**: 3D 가우시안 + K-Planes에 motion feature

### Dynamic GS의 한계 (교수 실험 강조)

**Photometric loss만**으로는 부족합니다. 빠른 motion에서 가우시안이 **추적 실패** — opacity 0→1로 **깜빡이며** 나타남/사라짐 (착시).

개선: trajectory prior, 2D tracking + basis curve, foundation model feature 보조.

# 5부. 260703 오후 — Gaussian Splatting 응용 (교수 강의)

> 조교 실습 마무리·서버 이슈 구간은 제외. 교수님 **응용 강의**만 정리합니다.

## 5-1. Feature 3D GS / LangSplat

가우시안에 **CLIP/SAM feature**를 추가로 저장 → **3D segmentation**.

- RGB 렌더링 + **Feature 렌더링** (32채널 등)
- Foundation model에서 뽑은 feature와 비교 loss
- 가우시안 점을 찍으면 **영역 분할** 가능

SAM/CLIP은 frozen, encoder-decoder만 학습. Feature 차원이 커지면 렌더링이 무거워집니다.

## 5-2. Object-level 합성 (Driving Gaussian)

- 배경 + **객체별 따로** GS 복원 → union으로 합성
- LiDAR → 정확한 center 초기화
- Vanilla GS는 **움직이는 객체** 표현 실패 → **분리 복원**이 핵심

## 5-3. Human / Scene Manipulation

멀티뷰 비디오에서 **human GS** 분리 복원. SMPL mesh 위에 가우시안 샘플링.  
사람 삭제·이동·포즈 변경 — 생성 모델이 아니라 **조작(manipulation)**.

## 주제 1. 3D 데이터 획득·표현

**질문 방향**: Depth Map이란? 4종 센서 차이? Dense vs Sparse?

### 핵심 답변 포인트

- **Depth Map**: 픽셀당 거리(mm). RGB와 해상도 같지만 의미 다름. 유리·코너는 측정 불가
- **4종 센서**: Passive/Active Stereo, Structured Light(Kinect), ToF/LiDAR. Passive는 **무특징 벽** 한계
- **시차 ∝ 1/depth**: 가까울수록 시차 큼
- **Depth → (X,Y,Z)**: focal length로 역투영 → Point Cloud
- **Dense(2D) vs Sparse(3D)**: 3D 볼륨 ~3%만 점유 → **처리 방법이 핵심**
- **Sparse Conv**: occupied 복셀만 연산, Hash O(N) vs KNN O(N log N)
- **Tesla**: Occupancy Map, LiDAR는 **GT 학습용만**

---

## 주제 2. 3D 인식·Detection

**질문 방향**: 7 DoF? VoteNet vs GSDN vs PointPillars?

### 핵심 답변 포인트

- **7 DoF**: center(X,Y,Z) + size(W,D,H) + yaw 1개 (바닥 grounding 가정)
- **VoteNet**: part feature → center vote → **클러스터링** (E2E 아님, 평평한 물체 약함)
- **GSDN**: Sparse U-Net, Generate/Prune, **Single Shot E2E**
- **PointPillars**: Pillar → pseudo-image → **2D detector**(60Hz). 3D 고집 대신 2D로 변환

---

## 주제 3. Neural Implicit · SIREN · SDF

**질문 방향**: Explicit vs Implicit? 왜 fitting? SIREN vs ReLU? SDF loss?

### 핵심 답변 포인트

- **Explicit**: 배열 직접 저장 (point, voxel, **gaussian**)
- **Implicit**: 함수(MLP)로 표현, 연속 query, export 후처리
- **Neural Implicit 목적**: generalization이 아니라 **한 데이터 fitting** (overfitting이 목적)
- **MLP+ReLU**: 저주파만 / **SIREN(sin)**: 고주파·미분 표현
- **SDF**: 표면=0, GT 없음, **constraint solver**. Surface loss 가장 중요, Eikonal |∇SDF|=1
- **Input/Output 정의**가 가장 중요 (point→SDF, image→intensity, NeRF→RGB+density)

---

## 주제 4. NeRF · Instant NGP

**질문 방향**: NeRF 파이프라인? 왜 multi-view? Instant NGP는?

### 핵심 답변 포인트

- **NeRF**: Novel View Synthesis. (x,y,z,view_dir) → RGB+density → **volume rendering** → MSE
- **한 장으론 불가**: depth 모호 → **multi-view triangulation** 필요
- **학습 = per-scene optimization** (memorization)
- **한계**: 느림(수 초/프레임), implicit → export 어려움
- **Instant NGP**: Hash Grid + small MLP, 빠르지만 여전히 implicit

---

## 주제 5. 3D Gaussian Splatting · 2DGS

**질문 방향**: 3DGS 원리·장점? Implicit vs Explicit? 2DGS는 왜?

### 핵심 답변 포인트

- **3DGS**: Explicit 가우시안(position, rotation, scale, SH, opacity)
- **학습**: Differentiable rasterization + photometric loss + **Adaptive density control**(clone/split/prune)
- **장점**: Explicit(.ply export), 가벼움, **실시간(100+ FPS)**, 포토리얼, 합성 용이
- **vs NeRF**: Implicit(후처리 필요, 느림) vs Explicit(바로 사용)
- **한계**: floater, needle, 표면 없음
- **2DGS**: 2D 원판으로 표면 복원, **SDF 없이 mesh**, floater↓. 표면 목적이면 3DGS만 볼 것 아님
- **GSDF**: GS+SDF dual, 무겁다
- **SfM 초기화** vs 랜덤: 수렴 품질 차이

---

## 주제 6. Dynamic 3D (개념 수준)

**질문 방향**: 정적에서 동적으로 확장? 한계는?

### 핵심 답변 포인트

- Static → **시간 축** 확장 (DyNeRF, 4D GS)
- **Photometric loss만**으로 부족, 빠른 motion에서 **flickering**(깜빡임)
- Monocular dynamic: unseen view에 loss 정의 불가 → 근본적 한계
