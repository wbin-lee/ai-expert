## 2. DCGAN 실습 (MNIST, 1차시)

### 2.1 데이터셋 & 모델 목표
- **MNIST**: 0~9 손글씨 숫자, 채널 1, 28×28 grayscale.
- **Generator**: 100차원 `z` → 1×28×28 이미지.
- **Discriminator**: 1×28×28 이미지 → 스칼라 (진짜일 확률, sigmoid).

### 2.2 Generator 구조
```
z (100)
  → Linear(100 → 128·7·7) → BatchNorm1d → ReLU
  → reshape (128, 7, 7)
  → ConvTranspose2d(128 → 64, k=4, s=2, p=1) → BatchNorm → ReLU   # 7→14
  → ConvTranspose2d(64 → 1,   k=4, s=2, p=1) → Tanh              # 14→28
```
- 출력 활성화가 **Tanh**인 이유: 픽셀을 **[−1, 1]로 스케일**해서 다룬다는 가정.
- "CNN은 차원을 줄이지만 여기선 반대로 키워야 한다" → **ConvTranspose2d** 사용.

### 2.3 ConvTranspose2d의 동작
- 이미지를 키울 때 "붙일 자리"가 필요. 일반적인 zero-padding처럼 **가장자리**에만 0을 넣는 게 아니라, **픽셀과 픽셀 사이에** 값을 삽입.
- 기본은 **0 삽입**. 이유: 삽입 값은 학습된 정보가 아니라 "자리 채우기"이므로 gradient 관점에서 중립적이어야 함.
- `kernel=4, stride=2, padding=1` 조합이 **공간 2배 확대**의 국룰.

### 2.4 Discriminator 구조
- 대칭으로 가지만 **Generator가 더 탄탄한 것**이 일반적 (Discriminator가 너무 강력하면 학습 실패).
- `Conv2d(1→64) → LeakyReLU → Conv2d(64→128) → LeakyReLU → Flatten → Linear → Sigmoid`.
- DCGAN 원 논문이 **Discriminator에 BatchNorm을 뺐던** 전례를 따름 (학습 성능상의 이유).

### 2.5 관용 규칙
- **공간 ½ 축소 ↔ 채널 2배 증가**. 수학적 법칙이라기보다 GPU 연산 효율 & 정보량 균형.

### 2.6 `view(-1, ...)` / reshape / contiguous
- `view`는 메모리가 **연속(contiguous)**이어야 동작 → 아니면 에러.
- `reshape`는 내부적으로 **필요 시 `contiguous()`를 호출**한 뒤 `view`를 수행하는 감싸진 함수. 실무에서는 reshape가 안전.

## 3. GAN 학습 루프 구현 (1차시)

### 3.1 두 옵티마이저
- VAE와의 차이점: **Optimizer가 2개** — 하나는 Generator용, 하나는 Discriminator용.
- 실제 이미지 레이블 `1`, 가짜 레이블 `0`을 **미리 만들어 둠** (self-supervised 레이블링).

### 3.2 Discriminator 학습 step
1. `z ~ N(0, I)` 뽑기.
2. `fake = G(z)`.
3. Real → D → BCE with 1, Fake → D → BCE with 0 → 평균.
4. `loss.backward()` → `optimizer_D.step()`.

### 3.3 왜 `detach()`가 필요한가
- Fake 이미지를 D에 넣을 때 `G(z).detach()`로 **computation graph를 끊음**.
- 끊지 않으면 D loss의 backward가 **G까지 흘러가** gradient를 남김. 명시적으로 G를 학습하는 step은 아니지만 메모리·계산 낭비, 잘못된 혼선 방지 목적.
- 실제로 검증: `zero_grad()`와 `step()` 루틴이 두 옵티마이저에서 교차 실행되기 때문에 **동작 결과는 같을 수도 있지만**, `detach()`는 **의도 명확화 + 메모리/속도 절약**을 위한 표준 관행.

### 3.4 Generator 학습 step
1. `z ~ N(0, I)` 새로 뽑기.
2. `fake2 = G(z)`, `score = D(fake2)`.
3. BCE with **1** (속이는 방향). ← 이게 **non-saturating loss**.
4. `loss.backward()` → `optimizer_G.step()`.

## 4. Non-Saturating Loss 시각화

- Game-theoretic: `log(1 − D(G(z)))`를 minimize.
- Non-saturating: `log D(G(z))`를 maximize.
- **두 커브의 기울기**:
  - `log(1 − D)` 곡선: D(G(z))가 0 근처일 때 기울기 완만 → **학습 초반 gradient 부족**.
  - `log D` 곡선: D(G(z))가 0 근처일 때 기울기 급함 → **학습 초반에도 강한 gradient**.
- 초기 Discriminator가 압도적이라 D(G(z)) ≈ 0에서 시작하므로, 기울기 뒤집힌 **non-saturating**이 실전에서 훨씬 잘 됨.

## 5. 학습 모니터링 팁

### 5.1 분포 그래프 해석
- Z 히스토그램은 Gaussian. 자명.
- D(x_real) 분포 vs D(x_fake) 분포를 같은 축에 그려보면:
  - 이상적으로 완전히 학습된 상태 = **둘 다 0.5 근처에서 겹침** (D가 구분 불가 → G가 잘 속임).
  - 두 분포가 분리되어 있으면 D가 우세 = G 부족.
- D의 출력이 "어느 한쪽"으로 수렴하지 않았다고 실패가 아니라, **두 분포가 얼마나 겹치는가**가 핵심 판단 기준.

### 5.2 Epoch별 세대 비교 실험
강의에서 보여준 재미있는 관찰:
- Generator를 0, 5, 10 epoch 스냅샷으로 저장. Discriminator도 0, 2, 4, 6, 8, 10 epoch로 저장.
- 각 조합의 D가 G의 출력을 얼마나 "진짜로 분류"하는지 행렬로 찍어보면 **선형적이지 않음**:
  - 중간 epoch의 D가 "속는" 경우가 있음 → D의 성능이 학습 시간에 단조 증가하지 않는다는 증거.
- 원인: G와 D가 **상호 영향**하며 공진화하기 때문. 경로 의존성이 강함.

## 6. Inverse Learning (Generator 학습이 아니라 Z 학습)

GAN의 근본 한계: "숫자 7을 만들어줘" 같은 **조건 생성이 안 됨** (Unconditional).

### 6.1 아이디어
- 이미 학습된 G는 freeze.
- Target 이미지(예: 7)을 주고 Z를 `requires_grad=True`로 둠.
- Loss = `||G(Z) − target||²`, `loss.backward()`를 **Z 방향으로** 흘림.
- **Z가 학습**되어 "이 G에 넣었을 때 7이 나오는 Z"로 수렴.

### 6.2 응용
- 이것이 **Inverse GAN** 분야의 기본 아이디어.
- 핵심 구문: `param.requires_grad_(False)` (G 고정) + Z는 리프 텐서로 생성 후 `requires_grad=True`.

## 7. Conditional GAN (2차시, Fashion-MNIST)

### 7.1 수식
```
D(x, y),  G(z, y)
```
- `y`는 class label. X 쪽에도, Z 쪽에도 **조건이 같이 들어감**.
- Y를 G에만 주고 D에 안 주면 → D가 조건을 무시 → G도 조건을 무시하는 방향으로 학습 → 의미 없음.

### 7.2 `nn.Embedding`을 통한 라벨 주입
- 클래스 개수 = 10이라면, **10개의 학습 가능한 벡터**를 lookup table로 유지.
- **"벤치에 앉아 있는 10명의 선수"** 비유: 요청한 라벨 하나가 그라운드로 나가는 구조.
- Lookup 호출 시마다 랜덤이 아니라 **같은 벡터**가 나옴. 하지만 **gradient가 흘러 학습**됨.

### 7.3 Generator와 Discriminator에 Y 주입 방식
- **Generator**: `z(100) ⊕ label_embedding(10) → 110차원` 입력.
- **Discriminator**: 라벨 임베딩을 **28×28 이미지 한 겹**으로 펼쳐서 입력 이미지에 **channel-wise concat** → 2채널 입력.
- 주입 방법은 여러 가지 가능:
  - MLP projection, concat, inner product, AdaIN 등.
  - StyleGAN은 AdaIN 레이어로 W를 복사해 주입.

### 7.4 학습된 라벨 임베딩 분석
- 학습 후 임베딩 벡터의 **PCA**를 찍으면, 비슷한 카테고리(Ankle boot/Sneaker/Sandal) 끼리 **가까이 모이고** 다른 카테고리는 멀어짐.
- **Cosine Similarity matrix**를 찍어보면:
  - Ankle boot와 가까운 것: Sandal > Pullover > Sneaker
  - Dress와 가까운 것: Coat > Shirt > Trouser
- 모델이 **라벨 자체의 표현**도 학습한다는 증거.

### 7.5 Label Interpolation & Emergent Ability
- Class 3(Dress)와 4(Coat) 사이에는 정수 라벨이 없어서 "3.5"로 학습된 적이 없음.
- 하지만 **임베딩 벡터**는 연속 공간이므로 `0.5 × emb(3) + 0.5 × emb(4)` 주입 가능.
- 결과: **티셔츠↔풀오버, 샌들↔스니커즈** 등 시맨틱하게 자연스러운 중간 형태 생성.
- 학습한 적 없는 능력이 자연히 발현 = **Emergent Ability**.

## 8. Generative Model의 정량 평가

### 8.1 FID (Fréchet Inception Distance)
- Inception V3의 **pool3 레이어** (FC 직전) feature를 사용.
- 이 피처는 classification을 수행하기 위해 **가장 정제된 semantic feature**라는 가정.
- Real 2000장 / Fake 2000장의 feature 분포(μ, Σ)를 구해 두 Gaussian 사이의 거리 계산.
- **주의점**:
  1. Inception이 ImageNet 기반이라 **도메인이 너무 다르면** 절대값은 못 믿음. 같은 도메인 내 상대 변화만 신뢰.
  2. **샘플 크기**가 너무 작으면 통계 추정 불안정. 2000 샘플이 실습 기준.
- 실습 결과: 9.2 → 0.2 → 0.1 → 0.0 → 0.1 (매우 급격히 수렴).

### 8.2 왜 FID를 training loss로 안 쓰는가
- **미분 불가능**. 그리고 충분한 샘플이 필요해 training step 비용이 너무 큼.
- BCE 같은 미분 가능 surrogate loss가 필요.

### 8.3 Precision & Recall for Generative Models
- **Precision = 품질(Quality)**: 생성 샘플이 real 분포 안에 얼마나 들어가 있나.
- **Recall = 다양성(Diversity)**: real 분포를 생성 샘플이 얼마나 넓게 커버하나.

#### 알고리즘 (k-NN 기반 bounded region)
1. `kNN_radii(real, k)`: 각 real 포인트에 대해 **k번째로 가까운 다른 real 포인트까지의 거리** = 해당 포인트의 "자기 영역 반경".
2. `kNN_radii(fake, k)`: fake도 마찬가지.
3. 모든 (fake, real) 쌍의 거리 계산.
4. **Precision** = real 포인트의 반경 안에 들어온 fake의 비율.
5. **Recall** = fake 포인트의 반경 안에 들어온 real의 비율.

#### 직관
- Real의 울타리 안에 fake가 많으면 품질 좋음.
- Fake의 울타리 안에 real이 많으면 다양성 좋음 (fake가 real 전역을 커버).

### 8.4 실습 결과
- Conditional GAN의 Precision ≈ 0.95, Recall도 적당히 높음 → **이상향에 근접**.
- 같은 기준으로 **VAE는 바닥을 기어다님** (에폭 수 적고 CNN 파라미터 적은 점 고려해도).

## 9. FID Feature 공간 시각화
- Inception pool3 feature를 **2D PCA**로 눌러 플롯.
- 학습 전: Real (파랑) 과 Fake (빨강) 이 **전혀 다른 군집**.
- 학습 후 10 epoch: **거의 구별 불가능하게 겹침** → FID 값 추이와 정합.

## 10. VAE vs GAN 직접 비교 (Fashion-MNIST)

| 구분 | VAE | GAN |
|------|-----|-----|
| Reconstruction 이미지 | 블러리 | — |
| Prior 샘플 | 뿌연 얼룩 | **선명한 의류 실루엣** |
| Loss 특성 | L1/L2/BCE 계열 = **Expectation 학습** → 평균 픽셀로 수렴 | Discriminator 기반 → expectation 평균을 강요받지 않음 |
| 샘플 sharpness | 낮음 | 높음 |

### 10.1 VAE가 블러리한 이유
- 픽셀을 "검은색/흰색 중 어느 쪽인가"가 애매할 때 **회색(중간값)으로 수렴**해야 loss가 줄어듦.
- Expectation 기반 학습의 **수학적 필연**.

### 10.2 GAN이 샤프한 이유
- 로스가 **각 샘플의 expectation을 계산하지 않음**.
- Discriminator는 "진짜냐 가짜냐"만 판단 → 평균 이미지를 내놔도 discriminator가 "가짜"라고 판정하면 G는 샤프해지는 방향으로 학습.

### 10.3 Diffusion·AR 시대의 GAN loss 재활용
- Reconstruction loss 기반 모델이 blurry하면, **GAN discriminator만 떼어 붙여** sharpness 보완.
- Latent Diffusion의 Autoencoder, 각종 TTS/Audio 모델에 **GAN loss가 여전히 현역**.

## 11. W-Space & StyleGAN (3차시)

### 11.1 Z에서 W로: Mapping Network의 역할
- **Z**: 그냥 가우시안 노이즈. Generator에겐 "이미지 분포로 변환하라"는 가혹한 요구.
- **W**: Z를 **FC 레이어 8층**에 통과시켜 얻은 리파인된 노이즈. 데이터 분포의 기하에 맞게 재정렬됨.
- Generator에는 Z 대신 W를 넣고, **중간 레이어마다 반복 주입** (AdaIN).

### 11.2 W의 평균
- `w̄ = E[w]` = 모델이 학습한 **"평균 얼굴"**.
- `ψ` (truncation 계수)로 interpolation:
  - `ψ = 0` → 평균 얼굴 (모든 샘플이 똑같이 나옴).
  - `ψ = 1` → 개별 특징 강한 얼굴.
  - `ψ = 0.7` → 평균과 개체 사이의 자연스러운 절충.

### 11.3 W의 분포
- Z는 순수 Gaussian ball. 각 차원 간 correlation 없음.
- **W는 데이터 분포에 맞게 비정규 모양**으로 변형됨 (차원 간 상관 있음).

## 12. ProGAN (Progressive GAN)

### 12.1 아이디어
- 작은 해상도 → 큰 해상도로 **점진적 학습**. 4×4 → 8×8 → 16×16 → ... → 1024×1024.
- 각 해상도에서 학습이 수렴하면 새 레이어를 추가해 키움.

### 12.2 한계
- 처음으로 1K 해상도 사람 얼굴에 도달. 하지만:
  - **머리카락 텍스처**를 잘 못 잡음.
  - 기괴하게 찌그러진 얼굴 등 artifact 빈번.
- 눈/코/입의 **상대적 위치 (spatial layout)** 는 잘 학습하지만, **세부 텍스처**에서 실패.

### 12.3 왜 하필 사람 얼굴?
- 얼굴만큼 **특징적이면서 응용 가치가 높은** 오브젝트가 드묾.
- 딥페이크의 파급력이 이를 증명.

## 13. StyleGAN

### 13.1 구조 요약
- ProGAN 구조 + **W의 계층별 주입** + AdaIN.
- 각 해상도 레이어마다 W를 컨디션으로 삽입.

### 13.2 레이어별 역할 (자동 발견)
| 레이어 해상도 | 담당 스타일 |
|---------------|-------------|
| 4×4 ~ 8×8 (coarse) | **얼굴 쉐이프, 포즈, 식별성** |
| 16×16 ~ 32×32 (middle) | **헤어, 이목구비 형태, 표정** |
| 64×64 ~ 256×256 (fine) | **색감, 피부 질감, 미세 디테일** |

### 13.3 Style Mixing 실험
- Reference 이미지의 W를 **특정 레이어 구간에만** 주입:
  - Coarse만 교체 → **포즈만** 바뀜, 식별성·색감 유지.
  - Middle 교체 → **이목구비**가 바뀜.
  - Fine 교체 → **색감·질감**만 바뀜.
- W-space가 **hierarchical하게 disentangle** 되어 있다는 경험적 증거.

### 13.4 Z-space vs W-space 보간
- **Lerp (Linear interpolation)**: 두 점을 직선으로 쪼갬.
- **Slerp (Spherical linear interpolation)**: 원점 기준 **각도 기반** 보간. 구면 위를 따라 이동.
- StyleGAN 저자들은 **Slerp가 더 선형적이고 disentangle** 하다고 주장. 체감으로는 확인이 애매하지만 W-space에서는 확실히 Z-space보다 부드러움.

## 14. 생성 모델의 주관성과 User Study
- **정량 지표(FID, PSNR)만으로 부족** → 요즘 논문은 대부분 **Amazon Mechanical Turk**류로 **3000명 규모 사람 설문**을 붙임.
- "Real vs Fake 중 뭐가 더 진짜 같아?"를 사람에게 judgment → discriminator 역할 수행.
- 품질 판단은 **개개인의 주관**이므로 공감대 확보가 중요.

## 15. CycleGAN — Unpaired Image-to-Image Translation

### 15.1 구조
- 두 개의 Generator: **G: X→Y**, **F: Y→X**.
- 두 개의 Discriminator: X 도메인용, Y 도메인용.
- **Cycle Consistency**: `F(G(x)) ≈ x`, `G(F(y)) ≈ y`.

### 15.2 Horse ↔ Zebra 데모
- G: 말 이미지의 "말 부분"에 **줄무늬를 칠함**.
- F: 얼룩말에서 줄무늬를 **지움**.
- 두 방향이 동시에 학습되며 reconstruction 제약을 통해 모드 붕괴 억제.

### 15.3 이미지 에디팅의 시초
- 이미지 편집(image-to-image translation) 분야 개척.
- 하지만 **공간 구조는 유지, 텍스처만 교체**하는 제한된 테스크만 성공:
  - 말 ↔ 얼룩말, 여름 ↔ 겨울, 사진 ↔ Monet 화풍 등.
- 강아지 ↔ 병아리 같은 **구조가 크게 다른 객체**는 무참히 실패. 병아리 모양에 강아지가 쑤셔넣어지는 결과.

### 15.4 Error Heatmap
- `x → G(x) → F(G(x))` 의 원본과 재구성 사이 픽셀 에러 히트맵.
- 비슷해 보여도 **국소 영역에서 높은 에러**를 잡아낼 수 있음. 디버깅에 유용.

## 16. GAN의 현재 위치
- GAN은 아직 죽지 않음:
  - **GAN loss**는 diffusion·AR 모델의 blurry 문제 보완에 여전히 쓰임.
  - 아키텍처도 Transformer-GAN 같은 연구가 꾸준히 등장.
  - 수학적 수렴성 담보 논문도 몇 년에 한 번씩 나옴.
- CNN/Transformer = **빌딩 블록**, VAE/GAN/Diffusion = **건물**. GAN이라는 건물은 내부를 무엇으로 채울지 자유.

## 17. Q&A 요약

### 17.1 "스타일 GAN에서 '눈 크기만' 조절하는 latent를 찾을 수 있나?"
전략 세 가지:
1. **데이터셋 큐레이션** (최후의 보루): 눈 크기만 다른 페어 데이터.
2. **눈 영역 detection loss**: 눈 패치에만 걸리는 로스를 W에 흘려, 어느 W 차원이 눈에 작용하는지 탐색.
3. **Conditioning 주입 경로 역추적**: AdaIN/concat 위치에서 눈 토큰/패치에 강하게 반응하는 네트워크 요소를 찾아 "뜯어보는" 수밖에 없음.

### 17.2 "`nn.Embedding` 외에 별도 임베딩 모델이 필요할까?"
- 대부분 경우 **nn.Embedding으로 충분**. lookup + learnable이 본질이고, 레이블 수가 많으면 **embedding dim**을 키우면 됨. 별도 모델 설계의 효용은 작음.

### 17.3 "Emergent ability가 '진짜'인지 아닌지 어떻게 판정?"
- "**귀에 걸면 귀걸이, 코에 걸면 코걸이**" — 발견된 현상이 **실제 응용/컨트롤/성능 향상**에 연결되면 Emergent Ability, 아니면 그냥 artifact.
- Latent interpolation이 매끄럽다고 해서 곧바로 "잘 학습되었다"는 증거는 아님. 실용적 가치와 다수의 공감대가 관건.

## 18. 핵심 정리 (한 페이지 요약)

1. **DCGAN 구현**: 100-dim `z` → `Linear + BN + ReLU` → reshape(128,7,7) → **ConvTranspose2d**(k=4,s=2,p=1) 두 번 → **Tanh** → 28×28 이미지. Discriminator는 대칭 Conv + Sigmoid.
2. **학습 루프**: 두 Optimizer, `detach()`로 graph 분리, Generator에는 **non-saturating BCE** (레이블 1로 속이기).
3. **모니터링**: 두 D 분포가 **0.5 근처에서 겹칠 때** 이상적. Loss 감소보다 **분포 겹침**이 더 정직한 지표.
4. **Inverse Learning**: G freeze + Z `requires_grad=True`로 **Z를 학습** → target 이미지를 만드는 Z 탐색.
5. **Conditional GAN**: `y`를 `nn.Embedding`으로 벡터화, G는 concat으로, D는 **이미지 채널로 펼쳐서 concat**. 라벨 임베딩도 함께 학습됨.
6. **평가 메트릭**:
   - **FID** = Inception pool3 feature의 Gaussian 거리. 낮을수록 좋음, 도메인 일치 필수.
   - **Precision/Recall** = k-NN 기반 bounded region 겹침. Precision=품질, Recall=다양성.
7. **VAE vs GAN**: VAE는 expectation 기반 loss로 blurry, GAN은 Discriminator로 sharp. 현대에는 reconstruction + GAN loss **혼합**이 일반적.
8. **W-space (StyleGAN)**: Z를 FC 8층으로 리파인한 W가 데이터 분포에 정렬됨. `ψ` 계수로 평균↔개체 보간 가능.
9. **Style Mixing**: coarse/middle/fine 레이어에 다른 W 주입 → **포즈/이목구비/질감**을 독립적으로 교체. Disentanglement의 경험적 증거.
10. **CycleGAN**: G·F 두 Generator + Cycle Consistency로 unpaired translation. 구조 보존 + 텍스처 교체 테스크에 특화. 이미지 에디팅 분야의 시초.

### 강의 중 주요 직관 비유 모음
- **먼지 → 이미지** : z → G(z) 의 핵심 비유. 학습이 안 된 상태의 Generator는 "더 큰 먼지"를 낸다.
- **은행나무/자작나무/소나무** : Discriminator는 클래스 하나만 뱉지만, Generator는 분포에서 다양한 인스턴스를 샘플링 → "생성형"의 본질.
- **벤치에 앉은 10명의 선수** : `nn.Embedding`의 lookup 구조 + 학습 가능성.
- **검은색/흰색이 헷갈리면 회색 칠하기** : VAE가 blurry 해지는 expectation-loss의 필연.
- **로스가 예쁘게 떨어졌는데 음성이 이상함** : GAN에서 loss와 품질의 correlation이 낮은 전형 → WGAN 필요성의 동기.
- **강아지↔병아리 = 실패** : CycleGAN은 spatial 구조 차이 큰 변환에는 무력 → "강아지처럼 생긴 노란 병아리, 병아리 형태에 우겨넣어진 강아지".
- **귀걸이/코걸이** : Emergent ability의 판정 기준 = 실용성.
- **3000명 사람 설문** : 최종 심판은 결국 **사람의 인지**. 정량 지표의 한계.
