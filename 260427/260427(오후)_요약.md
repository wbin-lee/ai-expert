## 2. Word Vector & Word Embedding 복습

### 2.1 좋은 워드 벡터의 조건 🔥 **(시험 출제)**
1. **Dense** (sparse 반대) — 차원 무한히 늘리지 않고 모든 숫자가 의미 가짐.
2. **차원이 작다** — 100~300 정도.
3. 의미적으로 유사한 단어는 유사한 벡터로.

### 2.2 One-Hot Encoding 한계 (재정리)
- King = [1,0,0,0], Queen = [0,1,0,0], Food = [0,0,1,0], Milk = [0,0,0,1]
- King-Queen 유사도 = King-Food 유사도 = 0 → **의미 유사성 표현 불가**.
- Sparse, Curse of Dimensionality.

### 2.3 Word2Vec 복습
- 핵심 : 특정 단어가 어떤 단어와 자주 나오는가를 학습.
- **CBOW** : 주변 → 중심 ("I _ U" 빈칸을 Love로). **빠르고 적은 데이터에 안정적**.
- **Skip-gram** : 중심 → 주변 ("Love"가 양옆 I, U를 예측). **느리지만 의미 학습 우수**.

### 2.4 GloVe 🔥 **(시험 가능 - Word2Vec과 차이)**
- 단어 동시 등장 빈도(co-occurrence)를 **미리 계산**하고 **로그 값**을 취함.
- 값이 무한정 커지는 것 방지 → **중복 계산 적음, 학습 빠름, 적은 데이터에서도 동작**.
- 예시 : "the cat is fast", "the cat in the hat" → 매트릭스에 cat-the는 3회, cat-fast는 2회 기록.

| 항목 | Word2Vec | GloVe |
|------|----------|-------|
| 학습 방식 | 페어가 자주 등장하면 여러 번 학습 → 내적값 증가 | co-occurrence 미리 계산 + log 적용 |
| 단어 관계 포착 | 지역적(local context) | **전역적(global)** |
| 학습 속도 | 느림 | 빠름 |
| 적은 데이터 | 약함 | **잘 동작** |

### 2.5 Cosine Similarity 🔥 **(시험 출제 - 수식)**

```
cosine(A, B) = (A · B) / (||A|| · ||B||)
            = Σ(A_i · B_i) / (sqrt(Σ A_i²) · sqrt(Σ B_i²))
```
- 값 범위 : **[-1, 1]**
- 1에 가까울수록 의미 유사 (방향이 비슷)
- 두 벡터 사이 각도 θ가 작을수록 의미 비슷.

### 2.6 ELMo & BERT (간단 소개)
- **ELMo** : Bidirectional LSTM 구조.
- **BERT** : Transformer 인코더 기반 (Day 2에서 배움).

## 3. 실습 1 : One-Hot, Word2Vec, GloVe

### 3.1 One-Hot 결과
- 보케브 = {King, Queen, Food, Milk}
- 히트맵 : 자기 자신만 1, 나머지 모두 0 → 의미 유사성 0.

### 3.2 Cosine Similarity 함수 작성

```python
def cosine_similarity(v1, v2):
    norm1 = np.linalg.norm(v1)
    norm2 = np.linalg.norm(v2)
    if norm1 == 0 or norm2 == 0:
        return 0
    return np.dot(v1, v2) / (norm1 * norm2)
```

### 3.3 Word2Vec 결과 (작은 코퍼스)
- King-Queen 유사도 = **0.11** (one-hot의 0보다 향상).
- King-Milk = -0.22 (관련 없음 → 음수).
- King의 most_similar : Man(0.18), Queen(0.11), Milk(0.08), Bread(0.07).

### 3.4 GloVe 결과 (대형 사전학습)
- King-Queen 유사도 = **0.67** → Word2Vec보다 훨씬 높음.
- 전역 코퍼스 고려 덕분.
- King의 most_similar : Prince(0.68), Queen(0.66), Princes(0.63).

### 3.5 Word Analogy 실습
- King - Man + Woman = **Queen (0.79)**, Princess, Monarch...
- Biggest - Big + Large = **Largest (0.83)**
- Cats - Cat + Dog = **Dogs**
- Asia - Korea + France = **Europe**
- 무관한 단어들(A, B, C)은 "Another" 같은 무의미 결과.

### 3.6 Vector Difference Similarity
- (Best - Good) vs (Worst - Bad) cosine = **0.4490**
- (Student - Professor) vs (Employee - Manager) = 0.2928
- (Paris - France) vs (Seoul - Korea) = 0.6322
- 알파벳 (B-A) vs (D-C) = -0.2505 (관계 없음, 음수)

### 3.7 Doesn't Match (가장 다른 단어 찾기)
- {Breakfast, Cereal, Dinner, Lunch} → **Cereal** (식사가 아닌 음식)
- {Samsung, KAIST, Yonsei} → **Samsung** (대학이 아님)
- {Lee, Kim, Kang, James} → **James** (영어 이름, 한국 성씨가 아님)

### 3.8 PCA 시각화 (수도-국가 관계)

## 4. Contextualized Embedding (BERT) 실습

### 4.1 동기 : 같은 단어, 다른 의미
- "He deposited money in the **bank**" — 금융기관
- "She sat by the **bank** of the river" — 강가
- 고정된 하나의 벡터로는 표현 불가능 → **문맥에 따라 다른 벡터**가 필요.

### 4.2 BERT vs GloVe 비교 (Bank 단어)
- **BERT** : 첫 문장 bank-river 유사도 < 두 번째 문장 bank-river 유사도 → **문맥 반영**.
- **GloVe** : 두 문장 모두 같은 0.307 (고정 벡터).

### 4.3 결론
- BERT 같은 contextualized embedding은 한 단어가 문장에 따라 **다른 벡터**를 가짐.
- 다의어(bank, run 등) 처리에 필수.

## 5. 토크나이제이션(Tokenization) 🔥 **(시험 출제)**

### 5.1 정의
텍스트를 더 작은 단위로 쪼개는 과정. 컴퓨터는 문장을 그대로 이해 못 하므로 단어/서브워드/문자 단위로 분할 필요.

### 5.2 세 가지 방식

| 방식 | 예시 (I love NLP, unhappiness) |
|------|--------------------------------|
| **Word** | I / love / NLP |
| **Subword** | un / happiness (BPE 등) |
| **Character** | I / l / o / v / e / ... |

### 5.3 BPE (Byte Pair Encoding)
- 시작은 문자 단위.
- **자주 등장하는 문자 조합을 반복적으로 합쳐 나감**.
- 너무 작지도 너무 크지도 않은 적절한 단위 도출.
- 현대 LLM에서 가장 많이 쓰이는 방식.

## 6. RNN 복습 + 실습

### 6.1 RNN 핵심
- **Sequential data** : 텍스트, 음성, 시계열. 이전 데이터가 현재에 영향.
- **Hidden state** : 이전까지의 모든 정보를 저장하는 벡터. 매 시점마다 새 입력 + 이전 hidden으로 갱신.
- **같은 RNN cell, 같은 가중치 W** 가 시간 순으로 반복 호출 (recurrent 구조의 핵심).

### 6.2 RNN의 4가지 입출력 패턴
| 구조 | 예시 |
|------|------|
| One-to-One | 이미지 분류 (이미지→cat/dog) |
| One-to-Many | 이미지 캡셔닝 (이미지→"a cat is running") |
| Many-to-One | 감정 분석, 스팸 분류, 문장 주제 분류 |
| Many-to-Many | 기계 번역, POS 태깅, NER |

> **Many-to-Many 주의** : POS 태깅은 입출력 길이 동일, 기계 번역은 길이 불일치 → 같은 카테고리지만 차이 존재.

### 6.3 RNN 한계
1. **장기의존성(long-term dependency) 문제** — 문장이 길수록 앞쪽 정보 망각.
2. **반복 계산으로 느림**.
3. **중요한 앞쪽 정보 소실**.

### 6.4 실습 - One-to-One RNN
- `SimpleRNN(50)` (hidden 50차원)
- Output activation = **Softmax**
- Epoch 200으로 학습
- "Hello World" 시드 → "Hello World Machine Learning Deep Learning Deep" (한 단어씩 5번 생성)

### 6.5 실습 - Many-to-One RNN
- 3개 단어 입력 → 1개 출력
- "Hello World Machine" 시드 → 다음 단어 "Learning" 예측, 5회 반복.
- "Samsung" 시작 → 모델이 학습한 패턴대로 단어 생성.

## 7. LSTM 복습 + 실습

### 7.1 LSTM 핵심 직관
- 정보를 **단순히 곱해서 전달이 아니라**, 잊을 것과 중요한 것을 **더하고 빼면서 기억 조절**.
- 긴 문장에서도 중요한 정보 유지.
- 응용 : 텍스트 생성, 번역, 음성 인식, 시계열 예측.

### 7.2 세 가지 게이트 (오전 강의 복습)
- **Input gate** : 새 정보를 얼마나 반영할지
- **Forget gate** : 이전 cell state에서 얼마나 잊을지
- **Output gate** : 현재 cell state로부터 얼마나 출력할지

### 7.3 실습 - LSTM 감성 분석 (IMDB)
- Input Embedding : 차원 20000 → 128
- LSTM cell : 128차원
- Output : Dense + **Sigmoid** (이진 분류)
- 데이터셋 : IMDB 영화 리뷰 (긍정/부정)
- 배치 사이즈 64, Epoch 2 (시간 단축)
- "This Movie is Awesome" → 0.56 (긍정 쪽). 배치/에폭 튜닝 시 더 정교해짐.

## 8. Sequence-to-Sequence (Seq2Seq) 🔥 **(시험 출제)**

### 8.1 동기
- RNN/LSTM은 **입력 길이 = 출력 길이** 가정 → 번역처럼 길이가 달라지면 문제.
- 새 구조 필요 → **Encoder-Decoder** 분리.

### 8.2 구조
```
[입력 문장] → Encoder → 하나의 벡터(요약) → Decoder → [출력 문장]
```

- **Encoder** : 영어 문장을 읽고 의미를 하나의 벡터로 요약.
- **Decoder** : 그 벡터로부터 프랑스어 문장을 한 단어씩 생성.
- Encoder/Decoder는 RNN, LSTM, GRU 어느 것도 가능 → **앞뒤 모두 LSTM이 성능 가장 좋음**.

### 8.3 GRU (Gated Recurrent Unit)
- LSTM의 3 gate를 **2 gate로 축소**한 버전.
- 기본 의미와 게이트 역할은 LSTM과 비슷. 더 단순/빠름.

### 8.4 평가 - BLEU Score 🔥 **(시험 출제 - 매우 중요)**

정답(reference) 문장과 모델 생성(candidate) 문장의 **n-gram 겹침** 기반 평가.

#### 예시
- Reference : "The cat is on the mat"
- Candidate : "The cat the cat on the mat"

| n-gram | Reference 단위 | Candidate와 겹치는 것 |
|--------|----------------|----------------------|
| 1-gram | The/cat/is/on/the/mat | The, cat, on, the, mat (is 빠짐) |
| 2-gram | "The cat", "cat is", ... | "The cat", "on the", "the mat" (3개) |
| 3-gram | ... | ... |
| 4-gram | ... | ... |

- **최종 = 모든 n-gram 점수의 곱의 평균** (보통 n=1~4)
- 단어 일치 + 순서 일치 + 길이 적절성 모두 고려.

### 8.5 벤치마크
- 모델 성능 표준화 평가 위한 데이터셋 모음.
- **GLUE, SuperGLUE, SQuAD** 등.
- 평가 지표 : BLEU, Accuracy 등.

### 8.6 실습 - 영-불 번역기 (Seq2Seq)
- **Word-level tokenization**.
- LSTM Embedding 256차원, LSTM hidden 256차원.
- `RepeatVector` : Encoder의 hidden vector를 디코더에 반복 전달.
- `return_sequences=True` : 모든 timestep 출력 (False면 마지막만).
- Output activation : Softmax.
- LSTM 모델 vs RNN 모델 비교 → **LSTM이 더 정답에 가까운 번역** (long-term dependency 덕분).

### 8.7 실습 - 영-한 번역기 (Character-level)
- **Character-level tokenizer** (글자 단위).
- 인코더/디코더 모두 LSTM.
- Build Vocab의 특수 토큰 :
  - **PAD** : 길이 맞추기 (패딩)
  - **SOS** : Start of Sentence
  - **EOS** : End of Sentence
  - **UNK** : Unknown (보케브 외 단어)
- 결과 :
  - "Hello" → "안녕"
  - "I am tired" → "피곤해"
  - "I miss you" → "보고 싶어"

## 9. Attention 메커니즘 🔥 **(시험 출제 매우 중요 — Transformer의 기반)**

### 9.1 동기 : Bottleneck Problem
- Seq2Seq는 문장 전체 정보를 **하나의 고정 차원 벡터**로 압축 → 정보 손실.
- 문장이 길수록 압축 손실 ↑ — **Bottleneck 문제**.
- LSTM도 cell state 한계로 완전히 해결 못 함.

### 9.2 Attention 핵심 아이디어
> "문장을 하나로 압축하지 말고, **필요할 때마다 직접 참고**하자."

- Decoder의 각 단계에서 Encoder와 직접 연결.
- 입력 시퀀스의 특정 부분에 **가중치(attention)** 부여.
- 번역 시 현재 만들고 있는 단어와 관련된 입력 단어에만 집중.

### 9.3 Attention 수식 🔥 **(시험 출제)**

```
스코어     :  e_{t,i} = score(s_t, h_i)              (디코더 s_t와 인코더 h_i 유사도)
가중치     :  α_{t,i} = softmax(e_{t,i})             (확률분포로 변환)
컨텍스트   :  c_t = Σ_i α_{t,i} · h_i                (가중합)
출력       :  y_t = f(s_t, c_t)                       (concat 후 출력 계산)
```

| 기호 | 의미 |
|------|------|
| $h_i$ | i번째 인코더 hidden state |
| $s_t$ | t번째 디코더 hidden state |
| $e_{t,i}$ | t시점 디코더와 i번째 인코더의 관련 스코어 |
| $\alpha_{t,i}$ | softmax로 변환된 attention weight |
| $c_t$ | t시점의 context vector |

### 9.4 Attention 동작 예시
- 첫 번째 단어 생성 시 → 인코더의 두 번째 hidden state(예: "the")에 attention 집중.
- 두 번째 단어 "shot"(불어 "chat") 생성 시 → "cat" 위치 hidden state에 집중.
- 세 번째 "noir"(black) 생성 시 → "black" 위치에 집중.
- **확률분포가 적은 위치의 hidden state는 거의 무시됨**.

### 9.5 실습 - Attention 기반 영-한 번역기
- 250개 리뷰로 학습.
- Score = decoder hidden과 encoder output의 **dot product**.
- Attention weight = softmax(score).
- Context = encoder output × attention weight 의 sum.
- 학습 후 paragraph 입력 → 요약 생성 ("태피" 같은 압축된 출력).

#### 실습 코드 핵심 빈칸 정답
- Score : `tf.matmul(encoder_outputs, hidden_with_time_axis, transpose_b=True)`
- Attention weights : `tf.nn.softmax(score, axis=1)`
- Context vector : `attention_weights * encoder_outputs` → `tf.reduce_sum(..., axis=1)`

## 10. Multimodal & Image Captioning

### 10.1 Multimodal AI 시대
- 텍스트, 이미지, 음성 등 다양한 데이터를 통합 이해/처리.
- ChatGPT, Gemini 모두 멀티모달 형태.

### 10.2 Image Captioning 실습 🔥 **(시험 가능 - 구조)**

```
이미지 → VGG16 (Encoder) → 특징 벡터
                                ↓
                         LSTM (Decoder) → 캡션 생성
```

- **Encoder** : VGG16으로 이미지 특징 추출 (Activation = ReLU).
- **Decoder** : LSTM으로 캡션 단어 단위 생성 (Activation = Softmax).
- Embedding 차원 = 128.
- 데이터셋 : 700개 이미지 (시간 단축 위해 축소).
- 결과 :
  - 야구 선수 이미지 → "Man in White"
  - 농구 선수 이미지 → 관련 캡션

## 12. 한 페이지 요약 (핵심 정리)

### 12.1 표현 학습
1. **좋은 워드 벡터 조건** : Dense, 작은 차원, 의미 반영.
2. **Word2Vec** : Local context (페어 빈도). CBOW(빠름) vs Skip-gram(의미 강).
3. **GloVe** : Global co-occurrence + log → 빠르고 적은 데이터에 강함.
4. **Cosine Similarity** : $\cos\theta = (A \cdot B) / (||A|| \cdot ||B||)$, 범위 [-1, 1].
5. **BERT (Contextualized)** : 같은 단어도 문맥에 따라 **다른 벡터** (다의어 처리).

### 12.2 토크나이제이션
6. **3가지 단위** : Word / Subword(BPE) / Character. 현대 LLM은 BPE 위주.
7. **특수 토큰** : PAD, SOS, EOS, UNK.

### 12.3 시퀀스 처리
8. **RNN** : Hidden state 갱신, 같은 cell/W 반복. Long-term dependency 약함.
9. **LSTM** : Cell state(장기) + Hidden state(단기) + 3 gate. 더하기/빼기로 기억 조절.
10. **GRU** : LSTM 게이트 3→2개로 축소.

### 12.4 Seq2Seq + Attention
11. **Seq2Seq** : Encoder-Decoder 분리로 입출력 길이 다른 번역 처리.
12. **Bottleneck 문제** : 한 벡터에 다 담으면 정보 손실 → **Attention** 필요.
13. **Attention 수식** : score → softmax(α) → context = Σα·h → y_t = f(s_t, c_t).
14. **BLEU Score** : N-gram 겹침의 곱 평균. n=1~4 표준.

### 12.5 Multimodal
15. **Image Captioning** : VGG16(이미지 인코더) + LSTM(텍스트 디코더).
16. 멀티모달 AI(ChatGPT, Gemini)가 현재의 주류.

### 12.6 강의 중 주요 직관 비유 모음
- **"단어의 의미는 친구를 봐라"** : Distributional Hypothesis 격언.
- **세서미 스트리트 캐릭터** : ELMo, BERT, BigBird 모두 어린이 TV 캐릭터에서 차용.
- **King-Man+Woman=Queen** : 단어 벡터 산술의 마법. 2013년 처음 본 이들에게 마술상자 같았다.
- **하나의 벡터에 다 담기** : Seq2Seq의 bottleneck — 긴 문장은 하나의 벡터에 담기 어렵다.
- **필요할 때마다 직접 참고** : Attention의 본질 — 압축 대신 참조.
- **데이터 부족 시 단어 반복** : 700개 이미지로 학습한 캡셔닝이 단어 수 늘리면 같은 단어 반복하는 현상.

### 12.7 실습 핵심 코드 패턴
```python
# Cosine Similarity
np.dot(v1, v2) / (np.linalg.norm(v1) * np.linalg.norm(v2))

# RNN One-to-One
SimpleRNN(50, activation='softmax')

# LSTM 감성 분석
Embedding(20000, 128) → LSTM(128) → Dense(1, activation='sigmoid')

# Seq2Seq Encoder-Decoder
Embedding → LSTM → RepeatVector → LSTM(return_sequences=True) → Dense(softmax)

# Attention
score = matmul(encoder_outputs, decoder_hidden, transpose_b=True)
α = softmax(score, axis=1)
context = reduce_sum(α * encoder_outputs, axis=1)

# Image Captioning
VGG16(이미지) → ReLU → Concat → LSTM(텍스트) → Softmax
```
