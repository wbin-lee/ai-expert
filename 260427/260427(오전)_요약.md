## 1. NLP의 시작 : 단어 표현(Word Representation)

### 1.1 언어를 처리하는 두 가지 패러다임

| 시대 | 방식 | 특징 |
|------|------|------|
| 전(前) 딥러닝 | **Symbolic** (Taxonomy / Knowledge Graph) | WordNet, hyper/hyponym, synonym/antonym 관계로 구조화. 트리 거리로 유사도 계산 (distance ↔ similarity는 역수 관계) |
| 딥러닝 | **Vector / Embedding** | 단어 = 숫자 벡터. 머신러닝 모델 입출력에 직접 활용 가능 |

### 1.2 One-Hot Vector
- 50만 차원(보케브) 벡터에서 단어 인덱스 위치만 1, 나머지 0.
- "1"은 숫자가 아닌 **인디케이터(indicator)** — 인덱스를 가리키는 표시.
- 보케브 사이즈 예시 :
  - 음성인식 : 2만 개
  - Penn Treebank : 5만 개
  - 일반 LM : **500K (50만 개, 강의 기준)**
  - 구글 검색 코퍼스 : 13M

#### One-Hot의 한계 (3대 문제)
1. **Orthogonality (직교)** — 모든 벡터가 서로 직교 → 내적(dot product) = 0 → 모든 단어 쌍이 "관련 없음".
   - "Motel ≠ Hotel", "size ≠ capacity" 가 검색 엔진에서 매칭되지 않는 문제.
2. **차원의 저주(Curse of Dimensionality)** — 50만 차원의 비용/저장/패턴 발견 곤란.
3. **Sparse, Discrete** — 0이 너무 많고, 0/1 이산 신호여서 정보를 압축하지 못함.

### 1.3 Distributional Hypothesis (분포 가설)
> "그 사람을 알려면 친구를 봐라" — 단어의 의미는 주변 문맥(context)에 의해 결정된다.

- One-Hot은 distributional 하지 않다(자기 인덱스만 표현).
- Banking을 모를 때 앞뒤 문맥을 보고 의미를 유추하는 영어 학습자처럼, 단어를 **주변과의 관계로 표현**하자는 것이 Word Embedding의 출발점.

### 1.4 Word Embedding이 추구하는 표현의 4가지 속성

| 속성 | 의미 | 반대 개념 |
|------|------|-----------|
| **Dense** | 차원이 작고 (100~300), 모든 차원이 채워져 있음 | Sparse |
| **Continuous** | 실수(real-valued) 값으로 채워짐 | Discrete |
| **Distributional** | 주변 단어와의 관계를 반영 | Independent |
| **Distributed** | 정보가 여러 차원에 분산 | Localist |

## 2. Word2Vec (2013, Mikolov et al., Google) 🔥 **(시험 출제 가능성 매우 높음 - 사이즈 계산 포함)**

### 2.1 배경
- 저자 : Tomas Mikolov, Ilya Sutskever (이후 OpenAI 공동창업자), Jeff Dean (구글 전설적 개발자) 등.
- **Shallow network** — 인코더 1층 + 디코더 1층. 네트워크 자체보다 **무엇을 입력/출력/Loss로 삼는가** 가 핵심.
- 두 변종 : **Skip-gram, CBOW**.

### 2.2 핵심 수식

문장이 주어졌을 때 중심 단어 $w_t$ 주변 $\pm m$ 윈도우의 단어들이 발생할 확률을 최대화 :

```
L(θ) = ∏_t ∏_{-m ≤ j ≤ m, j≠0} P(w_{t+j} | w_t ; θ)
```

수치 안정성/최적화 편의를 위해 **Negative Log-Likelihood**(= Cross Entropy)로 변환 :

```
J(θ) = - (1/T) Σ_t Σ_j log P(w_{t+j} | w_t ; θ)
```

### 2.3 Softmax 확률 정의 (스킵그램)

```
            exp( u_o^T · v_c )
P(o|c) = -----------------------------
         Σ_{w=1..V} exp( u_w^T · v_c )
```

- $v_c$ : 중심 단어(center)의 **인코더 벡터** (query)
- $u_o$ : 출력 단어(output)의 **디코더 벡터** (key)
- 두 벡터의 **내적(dot product)** 으로 관련 스코어 → softmax로 확률화

### 2.4 Skip-gram 네트워크 구조 🔥 **(시험 출제 — 사이즈 계산)**

```
입력(중심 단어) → 원핫 (V×1)
        ↓
   인코더 W (D×V)  →  v_c (D×1)         ← W의 c번째 행 = 임베딩 lookup
        ↓
   디코더 W' (V×D) →  스코어 (V×1)
        ↓
   Softmax        →  확률분포 (V×1)
        ↓
   정답 인덱스의 확률 → NLL Loss
```

- 인코더 입력이 원핫이므로 곱셈 == **lookup table** 참조와 같음 (해시/딕셔너리처럼 동작).
- 학습 후 인코더 W = 워드 임베딩 데이터베이스.
- 디코더 W'는 V×D 매트릭스로, V만큼 모든 단어와 내적 → 확률 분포 생성.

### 2.5 학습 데이터 구성 방식
- 한 문장에서 (중심, 주변) 페어를 다 추출 → **셔플링** 후 SGD.
- 한 문장에서 연속 학습이 아니라, 코퍼스 전체에서 거의 랜덤 위치로 학습됨.
- 한 step의 그림은 **한 step**일 뿐 — 실제 학습은 코퍼스 레벨에서 누적.

### 2.6 Loss 의미 직관
- 정답 단어의 확률이 0.6이면 → 0.4만큼 더 채워야 한다 → 인코더/디코더 파라미터 업데이트.
- 수렴 후 brown을 입력하면 fox, dog, bear 등 관련 단어들이 **밀도 있게 분포**된 확률을 출력 (단일 단어가 0.6이 되진 않음 — 기댓값 1/500K).

### 2.7 분모(softmax denominator) 계산 비용 문제 🔥 **(시험 가능)**

매 step마다 V=500K개 단어 모두에 대해 내적을 계산하는 건 너무 비싸다. 두 가지 해결책 :

| 방법 | 아이디어 |
|------|----------|
| **Negative Sampling** | V 전체가 아닌 **5~20개 랜덤 샘플**만 분모에 사용. 절대 확률값은 부정확하지만 상대 순위는 유지 |
| **Hierarchical Softmax** | Binary search 트리로 분류 모델을 학습. 각 노드에서 left/right 결정 → 검색을 **log scale**로 축소 |

> "확률을 정확히 구하는 게 중요한 게 아니라, 정답 단어가 다른 단어들보다 높다는 개념이 중요하다."

### 2.8 CBOW vs Skip-gram

| 항목 | Skip-gram | CBOW |
|------|-----------|------|
| 입력 | 중심 단어 1개 | 주변 맥락 단어들 (집합) |
| 출력 | 주변 단어 (한 페어씩, 별개 step) | 중심 단어 1개 (동시 입력) |
| 비유 | 자녀가 한 단어로 주변을 떠올리기 | 빈칸 채우기 / 동화책 가리기 |
| Rare word | **잘 동작** (드문 단어도 직접 학습 기회 多) | 약함 (팀플레이라 묻혀버림) |
| 사용 빈도 | **더 많이 사용** | - |
| 문법(syntactic) | 약함 | **더 강함** (주변 합쳐 들어가니 시퀀스 같은 효과) |
| 의미(semantic) | **강함** | - |

#### Rare Word가 중요한 이유 🔥 **(시험 출제)**
- 단어 분포는 **Power Law / Long Tail** (log scale로 보면 선형).
- 개별 rare 단어는 적지만, **rare 단어의 합집합이 코퍼스의 majority**.
- delightful 같은 드문 단어가 잘 학습되어야 진정한 언어 이해.
- Skip-gram이 rare word 학습 기회를 더 많이 줌 → 임베딩 품질 우수.

### 2.9 평가 — Intrinsic Evaluation
비지도학습이라 정답이 없음 → 두 가지 방식.

#### (a) 시각화 (정성적)
TSNE/PCA로 저차원 투영 → bad/poor, best/good 같은 유사어가 가깝게 모여 있는지 확인.

#### (b) Word Analogy (정량적) 🔥 **(시험 가능)**
```
king - man + woman ≈ queen
big - bigger ≈ small - smaller
```
- 두 단어 페어 사이에 의미론적/문법적 관계가 **벡터 공간에서 평행**하게 표현됨.
- 8000개 이상의 질문셋(Semantic + Syntactic)으로 평가.
- 가장 가까운 단어가 정답인지 확인 → 최대 50% 정확도 달성.

### 2.10 Scaling 발견 (역사적 의미)

#### Diminishing Returns (한쪽만 키울 때)
- 모델 사이즈만 키우거나 데이터 사이즈만 키우면 처음엔 6점 오르다가 1점, 0.x점으로 수렴.
- 이 현상을 보고 당시 사람들은 "AI는 한계가 있다, AI는 버블이다" 라고 결론.

#### Scaling Law의 진짜 발견 🔥 **(시험 출제 - 매우 중요)**
> **Scale = 모델 사이즈 × 데이터 사이즈** (둘은 같이 연결된 한 팩터)

- 두 축을 **대각선**으로 같이 키우면 성능이 linear하게 증가.
- Scale을 **exponential**하게 키우면 성능이 **linear**하게 증가.
- 이 인사이트는 2013년에 이미 힌트가 있었지만 그땐 알지 못했고, 2020년대에 와서야 LLM 스케일링 법칙으로 확립됨.
- 옛날 임베딩 차원 50~100 → 300/600도 잘 동작 → 현재는 만~2만으로 거대화.

### 2.11 Skip-gram vs CBOW 성능
- **Semantic accuracy** : Skip-gram > CBOW
- **Syntactic accuracy** : CBOW가 약간 우세 (주변 단어 동시 처리 → 시퀀스적 문법 정보 더 잘 잡음)

## 3. RNN (Recurrent Neural Network)

### 3.1 동기 : 시퀀스 데이터 처리
- 음성, 주식/날씨, 비디오, 추천 로그, 텍스트(GPT) — 모두 시퀀스.
- 기존 MLP는 **고정된 차원**의 입출력만 처리 가능 → 가변 길이 + 순서 정보 처리 불가.

### 3.2 RNN 종류 (입출력 형태별)

| 구조 | 예시 |
|------|------|
| One-to-One | 이미지 분류 |
| One-to-Many | 이미지 캡셔닝 |
| Many-to-One | 감성 분석, 스팸 분류 |
| Many-to-Many | 번역, 챗봇, 요약, 비디오 생성 |

### 3.3 RNN 핵심 수식 🔥 **(시험 출제 - 사이즈 계산)**

```
h_t = tanh( W_hh · h_{t-1} + W_xh · x_t )
y_t = softmax( W_hy · h_t )
```

- **Hidden state $h_t$** : 시간 축 따라 누적되는 **메모리** (사람의 working memory / 컴퓨터의 RAM 비유).
- 동일한 $W_{hh}, W_{xh}$ 가 **모든 시간 t에서 공유** → 가변 길이 처리 가능.
- $h_t$ 는 인코더가 문장 의미를 파악한 결과로 해석.

### 3.4 왜 tanh 활성화? 🔥 **(시험 가능)**
- 출력 범위 [-1, 1] → 양수/음수가 균형.
- 정보가 계속 더해지는 RNN 구조에서 **양수만 더해지면 값이 무한히 커짐** → 분포 안정화 위해 부호가 섞여야 함.
- 미분 안정성 + 표준 분포 유사한 출력으로 학습 안정성 확보.

### 3.5 Loss 구성
- **Sequence-level**: 마지막 $h_T$만 사용 (감성 분류 등).
- **Step-level**: 매 시점마다 출력 (POS tagging, NER, 비디오 프레임 분류 등) → 모든 step의 loss를 합산.

### 3.6 RNN의 한계
- $W_{hh}, W_{xh}$ 가 정보를 **걸러내는 역할이 없음** — 모든 입력을 빼곡히 hidden state에 업데이트.
- HTML 태그 같은 노이즈도 그대로 메모리에 들어감.
- 시간이 길어질수록 **과거 정보가 희석** → 멀리 떨어진 단어들 간 상호작용 학습 어려움.
- **Long-term dependency 문제** + Vanishing Gradient.

## 4. LSTM (Long Short-Term Memory) 🔥 **(시험 출제 — 사이즈 계산 매우 중요)**

### 4.1 핵심 차이 : 두 개의 State + Gate
- **Hidden state $h_t$** : 단기 (현재 출력 용)
- **Cell state $c_t$** : 장기 메모리 (LSTM 고유) — RAM이 아닌 **하드디스크** 비유 (여진영 교수의 비유)
  - 지금 쓸 것 + 나중에 쓸 것 + 영원히 안 쓸 것 = 다 혼재
  - 선택적으로 저장(write), 지움(forget), 가져옴(read)

### 4.2 세 가지 Gate

| Gate | 기호 | 역할 | 활성화 |
|------|------|------|--------|
| **Forget gate** | $f_t$ | 이전 cell state $c_{t-1}$ 에서 무엇을 잊을지 | sigmoid |
| **Input gate** | $i_t$ | 새 임시 정보 $\tilde{c}_t$ 중 무엇을 저장할지 | sigmoid |
| **Output gate** | $o_t$ | cell state에서 무엇을 출력할지 | sigmoid |

### 4.3 수식 흐름

```
f_t = σ(W_f · [h_{t-1}, x_t])
i_t = σ(W_i · [h_{t-1}, x_t])
o_t = σ(W_o · [h_{t-1}, x_t])

c̃_t = tanh(W_c · [h_{t-1}, x_t])      # 임시 new cell content (틸다 c_t)
c_t = f_t ⊙ c_{t-1} + i_t ⊙ c̃_t       # cell state 업데이트
h_t = o_t ⊙ tanh(c_t)                   # hidden state 출력
```

⊙ = **Element-wise multiplication (요소곱, Hadamard product)** — 내적과 다름.

### 4.4 직관 (뇌-뉴런 비유)
- D차원 cell state = D개의 뉴런이 가족 정보, 업무 정보, 수업 정보 등을 저장.
- Forget gate = 0011010 같은 0/1 벡터 → cell state의 어떤 차원을 지우고 살릴지.
- Input gate = 새로 추가될 정보 중 어떤 차원만 받아들일지.
- 조크가 끝난 뒤 조크 부분은 forget, 진지한 수업 내용만 input gate로 통과.

### 4.5 LSTM 차원 ⭐ **(시험 출제 - 반드시 외울 것)**

가정: $V$ = 1000 (보케브), $H$ = 10 (hidden), $C$ = 20 (클래스)

| 변수 | 차원 |
|------|------|
| $x_t$ | 1000×1 |
| $h_{t-1}, h_t$ | 10×1 |
| $c_{t-1}, c_t, \tilde{c}_t$ | **10×1 (hidden과 같은 차원)** |
| $f_t, i_t, o_t$ | **10×1 (스칼라 아님!)** |
| $W_{hh}$ | 10×10 |
| $W_{xh}$ | 10×1000 |
| $W_f$ (h와 상호작용) | 10×10 |
| $U_f$ (x와 상호작용) | 10×1000 |
| $W_i, W_o$ | 10×10 |
| $U_i, U_o$ | 10×1000 |
| $W_{hy}$ | 10×20 |
| $y_t$ | 20×1 |

**핵심**: forget gate, input gate, output gate가 **스칼라가 아니라 cell state와 동일한 차원의 벡터**라는 점을 반드시 기억.

> "사이즈를 모르면 이걸 이해하지 못한 겁니다. 시험 문제도 사이즈에 대해서 나옵니다."
> "Bias는 빼놓은 거고, transpose도 row/column 순서 바꿔도 정답 처리합니다."

### 4.6 Concat 표현
$h_{t-1}$과 $x_t$를 concat한 뒤 한 매트릭스 W로 처리하는 방식과, 분리해서 별도 매트릭스로 처리하는 방식은 **사이즈는 같지만 학습 양상이 다름** — 합치면 균질성 가지고 같이 학습 (도움될 수도, 저해될 수도). 아키텍처 선택 영역.

## 5. Bidirectional RNN/LSTM

### 5.1 동기
- 단방향 RNN은 미래 정보 활용 불가.
- 텍스트 생성에선 OK (미래는 모르는 게 맞음), 텍스트 **이해**에선 결함.

### 5.2 구조
- **Forward RNN** + **Backward RNN** 의 hidden state를 **concat**.
- 일반적으로 두 방향은 **파라미터 공유 안 함**.
- 100차원 → 50/50으로 쪼개면 사이즈 그대로 유지 가능.

### 5.3 Multi-layer (Deep) RNN
- RNN은 이미 시간 축에서 deep (vanishing gradient 발생할 만큼).
- 레이어를 3개 이상 잘 안 쌓음 — 성능 향상이 미미.
- "사이즈만 중요한 게 아니라 **아키텍처가 훌륭해야** 스케일업의 효과가 있다" (개구리 vs 사람의 신경망 비유).

### 5.4 실험 결과 (감성 분류)
- RNN < BiLSTM < Multi-layer BiLSTM
- 차이는 1~2% 수준이지만 **1%는 결코 작지 않다** :
  - 영업이익 1% = 거대 비즈니스 임팩트
  - 광고 정확도 1% = 전체 유저 1%에 영향
  - 논문 게재 가능 최소 성능 향상

## 6. 텍스트 분류 (Text Classification)

### 6.1 정의
- Document → Class 매핑.
- 사전 정의된 클래스 집합. 학습 도중 클래스 변경 불가.

### 6.2 응용 분야 (다양함)
- **감성(Sentiment) 분류** : 극성(polarity, 긍/부정)
- **감정(Emotion) 분류** : 다양성(joy, surprise 등 중첩 가능)
- 토픽 분류, 문서 분류, 스팸 분류
- 나이/성별/언어 추정 (인스타그램 광고 타겟팅 등)

> 극성(gradient)과 감정(diverse boundary)은 다른 문제 — Netflix의 약물/폭력/언어 등급 자동 분류 예시.

### 6.3 Naive Bayes 🔥 **(시험 가능 - 베이즈 정리)**

posterior $P(C|D)$를 직접 못 구하므로 베이즈 룰로 분해:

```
P(C|D) ∝ P(D|C) · P(C)
       = P(C) · ∏_i P(x_i | C)         # bag-of-words + 단어 독립 가정
```

#### 두 가지 강력한 가정
1. **Bag of Words** : 단어 순서 무시.
2. **Conditional Independence** : 단어들이 클래스 조건부로 서로 독립.

#### Prior $P(C)$ 의 중요성
- 온라인 쇼핑몰 방문자가 구매할 prior는 약 **2%** (98%는 윈도우 쇼퍼).
- prior를 무시하면 비현실적 추정 (40%/70%).

#### 한계
- 단어 빈도만 다루므로 의미/문맥 이해 불가.

### 6.4 GLUE Benchmark 🔥 **(시험 출제 - 12개 과제 분류 외울 것)**

2019년에 BERT 같은 LM 평가 위해 통합된 **General Language Understanding Evaluation** 벤치마크.

#### 분류 (3가지)
1. **Single Sentence**
   - **CoLA**: 문법(linguistic acceptability) 분류 (Grammarly 같은 응용)
   - **SST**: 감성 분석
2. **Sentence Pair (Similarity & Paraphrase)**
   - **MRPC**: 두 문장이 같은 의미인지 (paraphrase)
   - **STS-B**: **Regression** 문제 (0-5점 유사도 점수)
   - **QQP**: Quora 질문 페어
3. **NLI (Natural Language Inference)** 🔥 **(시험 출제)**
   - **MNLI, QNLI, RTE, WNLI**
   - 전제(Premise) + 가설(Hypothesis) → **Entailment / Contradiction / Neutral** 3분류
   - 응용 : LLM 응답 필터링 (Galaxy 최신 기종 25 vs 26 사실 검증 모듈)

### 6.5 텍스트 분류의 12가지 챌린지 🔥 **(시험 출제 - 3개 정도 언급할 수 있어야 함)**

> "달달 외우는 건 아니지만 12개 중 3개는 언급할 수 있는 레벨로 공부 필요."

1. **데이터 비용** — 전문 도메인일수록 레이블링 비용 폭증.
2. **데이터 밸런스** — 구매율 2% 같은 극도의 imbalance.
3. **도메인 특화** — 메디컬, 법률 등 일반 언어와 다름.
4. **Sparse Features** — 딥러닝 이전 방법의 한계.
5. **Out-of-Vocabulary (OOV)** — 신조어, 컷오프된 단어 처리 불가.
6. **Long-range Dependency** — LSTM도 cell state 차원 한계로 임시 방편.
7. **Explainability** — 의료/금융/법률 같은 high-stake 분야에서 필수.
8. **Computational Cost** — 큰 모델로 단순 분류 시 비효율 (Claude Code 4만×5만 곱셈에 토큰 폭주, 천원 발생 사례).
9. **Noisy Data** — 트위터 슬랭, 자연 영어의 2%만 해석 가능.
10. **Multilingual / Code-mix** — 한 문장에 여러 언어 혼재.
11. **Concept Drift / Trend** — 시간에 따라 데이터 분포 변화 (캘리포니아 산 추천 시스템을 한국에 적용).
12. **Bias** — "봉준호" 단어만 들어가도 무조건 긍정으로 분류되는 단어 단위 편향.

### 6.6 감성 분석이 어려운 이유
- 단순 단어 카운트로 풀리지 않음.
- "얼굴 별로, 직업 별로, 성격 별로 → **근데 사귀기로 했어**" — 부정어 다수지만 결론은 긍정.
- LLM도 인간의 감성을 완전히 이해 못함, 여전히 발전 중.

## 7. 한 페이지 요약 (핵심 정리)

### 7.1 표현(Word Embedding)
1. **One-Hot의 3대 문제** : 직교(유사성 표현 불가) / 차원의 저주 / Sparse-Discrete.
2. **Word Embedding 4 속성** : Dense, Continuous, Distributional, Distributed.
3. **Distributional Hypothesis** : 단어의 의미는 주변 문맥에서 온다.
4. **Word2Vec** : Shallow encoder-decoder. Skip-gram(중심→주변) / CBOW(주변→중심).
5. **Loss** : NLL(= Cross Entropy). $P(o|c) = \exp(u_o^T v_c) / \sum_w \exp(u_w^T v_c)$.
6. **분모 비용 해결** : Negative Sampling (5~20개 랜덤) / Hierarchical Softmax (log 스케일).
7. **Skip-gram > CBOW** in semantic, **CBOW > Skip-gram** in syntactic. Rare word 학습 → Skip-gram이 강함.
8. **Long Tail / Power Law** : rare 단어가 코퍼스의 majority.
9. **Scaling Law (씨앗)** : Scale = Model × Data, 대각선 키우기가 진정한 스케일업.

### 7.2 시퀀스(Sequential Learning)
10. **RNN 핵심 수식** : $h_t = \tanh(W_{hh} h_{t-1} + W_{xh} x_t)$. 모든 t에서 W 공유.
11. **tanh를 쓰는 이유** : 양/음 균형 → 분포 안정성.
12. **RNN 한계** : 정보 필터링 부재 + Long-term dependency 약함.
13. **LSTM** : Cell state(장기) + Hidden state(단기) + 3 Gate(forget, input, output).
14. **Cell update** : $c_t = f_t \odot c_{t-1} + i_t \odot \tilde{c}_t$, $h_t = o_t \odot \tanh(c_t)$.
15. **Gate는 vector**(스칼라 아님), cell state와 동일 차원. **차원 계산이 시험 출제**.
16. **Bidirectional** : Forward+Backward concat. Multi-layer는 3층 이상 잘 안 쌓음.

### 7.3 분류 & 평가
17. **Naive Bayes** : Bayes rule + BoW + 독립 가정. Prior 무시 시 비현실적.
18. **GLUE** : Single sentence (CoLA, SST) / Pair (MRPC, STS-B regression, QQP) / NLI (entailment/contradiction/neutral).
19. **NLI 응용** : LLM 응답을 회사 DB와 대조해 필터링.
20. **12 Challenges** : 비용/밸런스/도메인/Sparse/OOV/Long-range/설명성/계산비용/노이즈/멀티링걸/Drift/Bias.

### 7.4 강의 중 주요 직관 비유
- **마술상자** : 2013년 Word2Vec 데모로 "한국-서울+일본=도쿄" 보고 충격받은 기억.
- **하드디스크 vs 램** : LSTM Cell state는 RAM이 아니라 하드디스크 — 장단기 정보 혼재 + 선택적 read/write/delete.
- **개구리 vs 사람의 신경망** : 스케일업의 효과는 좋은 아키텍처 위에서만 발생.
- **포켓몬 / 봉준호** : 단어 편향 — "봉준호 한 명 갔어"도 긍정으로 분류.
- **소개팅 마지막 반전** : "별로별로별로... 근데 사귀기로 했어" — 단어 카운트로 못 푸는 감성 분석.
- **자녀에게 한글 가르치기** : CBOW = 동화책 빈칸 채우기.
- **시험에서 자주 나오는 거만 공부** : Skip-gram이 rare 단어를 더 잘 배우는 이유의 비유.
- **그란마리(Grammarly) 스타트업** : CoLA 같은 문법 분류가 산업적으로 가치 있음을 보여줌.
