# 1. Triangulation — 대응점에서 3D 포인트 복원 ⭐⭐

## 1.1 문제 정의

- 카메라 2대(또는 그 이상)가 **칼리브레이션 되어 있음**: 각 카메라의 \(K, R, T\) (즉 \(P_1, P_2\)) 주어짐.
- 누군가 **대응점**을 줌: \(x_1, x_2\)(2D)가 같은 3D 포인트를 보고 있음.
- 목표: 그 **3D 포인트 \(X\)를 찾는 것** — 전형적인 3D reconstruction 문제.

## 1.2 레이는 실제로 만나지 않는다 ⭐

- 이상적으로는 두 레이가 한 점에서 만나야 하지만:
  - 칼리브레이션 파라미터가 약간 부정확 → 레이 방향이 살짝 삐끗.
  - 대응점도 살짝 부정확.
- 라인이 무한히 얇다면 **공중에서 두 라인이 우연히 부딪칠 확률은 0** — 밀리미터/픽셀 오차도 멀리 가면 어긋남.
- → "교점을 찾는다"가 아니라 **두 레이에서 가장 가까운 \(X\)를 찾는다**가 안전한 가정.

## 1.3 진짜 원하는 것 — Reprojection Error 최소화 ⭐

\(X\)를 \(i\)번째 카메라로 프로젝션했을 때 찍힌 점과 대응점 사이의 거리를 **reprojection error**라고 부른다.

\[
X^* = \arg\min_X \sum_i \| \pi(P_i, X) - x_i \|^2
\]

- 이상적으로 모든 카메라에서 이 거리가 0 (= 레이 위에 있음).
- 모든 카메라에 대한 reprojection error의 합(평균)을 최소화하는 \(X\)를 찾는 게 진짜 triangulation 문제.

## 1.4 DLT 풀이 ⭐⭐

대응점, \(P_1, P_2\)는 모두 아는 값, **모르는 건 \(X\) 하나뿐** → 늘 쓰던 연립방정식 패턴.

- \(x \sim PX\) (up to scale) → equality로 만드는 방법: λ 도입, 양변 inhomogeneous화, 또는 **cross product = 0**.
- 슬라이드에서는 cross product 방식: \(x \times PX = 0\).
- \(P\)(3×4)를 3개의 row \(p^1, p^2, p^3\)(각 4차원)로 쪼개면 식 3개가 나오는데, **세 번째 식은 위 두 개의 linear sum** → redundant.
- → **카메라 1대당 독립적인 식 2개**.

\[
\begin{aligned}
&x_1 (p^{3\top} X) - (p^{1\top} X) = 0 \\
&y_1 (p^{3\top} X) - (p^{2\top} X) = 0
\end{aligned}
\]

- 카메라 2대 → 식 4개, 미지수는 \(X\)의 3 DOF → **카메라 2대만 있으면 풀 수 있음**.
- 카메라가 더 많으면? 행을 **계속 concat** → 측정이 많을수록 유리 (개별 측정이 erroneous해도 평균적으로 정확해짐).
- 결국 \(AX = 0\) 형태 → **SVD로 풀이** (Homography 때와 완전히 동일한 리콜 패턴).

## 1.5 Algebraic vs Geometric Error ⭐

| 구분 | DLT (\(AX=0\)) | Reprojection Error 최소화 |
|---|---|---|
| 성격 | Linear, algebraic | **Nonlinear** (L2 distance 포함) |
| 의미 | 대수적으로 0에 가깝게 | **실제 거리** 최소화 |
| 관계 | 해가 정확히 0이면 둘 다 0 | 0이 아닐 때는 미묘하게 다름 |

- 우리가 진짜 원하는 건 reprojection error 쪽 — **더 좋은 솔루션**.
- 하지만 nonlinear → gradient descent 필요 → **이니셜이 필요**.
- → **DLT 해를 이니셜로 주고 gradient descent로 정제**하는 게 일반적 방법론.
- Homography, Fundamental Matrix, Calibration 전부 같은 패턴: DLT init → reprojection error 최소화.

## 1.6 Correspondence가 모든 것 ⭐

- 카네기멜런 Takeo Kanade 교수의 명언: 컴퓨터 비전의 세 가지 중요한 문제는 **"Registration, Registration, Registration"** — 대응점만 주면 비전 문제는 다 풀린다.
- 2000년대 초반 **SIFT** 등장 → in-the-wild 이미지에서도 대응점 제공 → 칼리브레이션 가능 → 3D 포인트 생성 가능 → **Structure from Motion으로 연결**.

# 2. PnP (Perspective-n-Point) — 카메라 포즈 추정 ⭐⭐

## 2.1 문제 정의

- 입력: **\(K\)를 알고 있는 상태**에서, **3D-2D 대응점**.
- 출력: **카메라 포즈 = Extrinsic 파라미터 \(R, T\)** — 카메라가 3D 공간 어디서 보고 있는지.

(2D-2D 대응점을 알고 그 2D에서 3D를 만들었다면 2D-3D 대응점도 자연히 알 수 있음.)

## 2.2 응용 예시

- **사진 위치 맞추기 아티스트**: 유명한 옛 사진/영화 장면이 어디서 촬영됐는지 3D-2D 대응을 보며 수작업으로 카메라 포즈를 맞춤 — 정확히 PnP.
- **건물 리컨에 새 이미지 등록**: 이미 3D 리컨된 건물에 새 이미지의 대응점이 주어지면 그 이미지의 시점을 복원.
- **Augmented Reality** ⭐: 바닥의 알려진 패턴(체커보드)으로 3D-2D 대응을 실시간으로 얻음 → \(R, T\)를 실시간으로 추정 → 3D 캐릭터를 \(KRT\)로 렌더링하면 현실 장면과 정합. **AR 글래스**도 동일 원리 — 내가 어디서 보고 있는지를 알아내고 3D를 프로젝션.

## 2.3 왜 가능한가 — 레이 직관 ⭐

- \(K\)를 안다 = 2D 포인트가 주어지면 **레이를 쏠 수 있다** (\(K\)가 3D 레이와 2D 포인트의 관계를 정의).
- 올바른 포즈 = **각 대응 레이 위에 대응되는 3D 포인트가 올라가 있는 포즈**.
- 레이 위에 3D가 안 올라가는 포즈는 틀린 포즈 → 대응관계와 \(K\)만 있으면 포즈를 정확히 계산 가능.

## 2.4 최소 대응점 — P3P

- **대응점 3개면 카메라 포즈가 정확히 정의됨** → 가장 미니멀한 알고리즘이 **P3P**.
- PnP의 N = 포인트 개수; P3P, P4P, EPnP 등 다양한 알고리즘 존재.

# 3. Structure from Motion (SfM) ⭐⭐

## 3.1 문제 정의

- 입력: **이미지 다수** + 이미지 간 대응관계 (SIFT 등이 찾아준다고 가정).
- 출력: **모든 카메라 포즈(\(R, T\)들) + 3D 포인트 클라우드(\(X\)들)** 전부.
- 데모: Visual SFM 툴 — 이미지 업로드 후 버튼 하나로 53개 이미지 → 약 35K 포인트, 136K 프로젝션으로 리컨.

## 3.2 Incremental SfM 파이프라인 ⭐⭐

```
1. 모든 이미지 쌍에 대해 pairwise correspondence 계산 (SIFT)
2. 대응점 연결 → 테이블(트랙) 구성: "이 점들은 같은 3D를 보고 있으니 나중에 같이 triangulation"
3. F-RANSAC으로 아웃라이어 제거
4. 대응이 충분히 많은 이미지 2장 선택
   - 첫 번째 카메라에 월드 좌표 원점 부여: P1 = K[I | 0]
   - F(E) 매트릭스에서 relative R, T 분리 → 두 번째 카메라 포즈
5. Triangulation → 초기 3D 포인트 클라우드
6. 반복(Incremental loop):
   a. 이미 리컨된 부분과 대응이 충분히 많은 새 이미지 선택
   b. 2D-2D 대응 → 2D-3D 대응 확보 → PnP → 새 카메라 등록(register)
   c. 새 카메라가 추가로 보는 대응점들 Triangulation → 포인트 추가/보정
   d. (수시로) Bundle Adjustment
```

- 핵심 RANSAC 주의점 ⭐: 여기서는 H-RANSAC을 쓸 수 없음(homography는 평면/pure rotation 등 특수한 경우). 대신 **에피폴라 컨스트레인트(에피폴라 라인 위에 있어야 함)를 쓰는 F-RANSAC** 사용 — **카메라 2대 사이에는 F 관계가 항상 성립**하므로 언제든 사용 가능.
- 이미지가 많을수록 Triangulation에도 PnP에도 유리 — 거의 무조건 더 정확.

## 3.3 에러 누적 문제 ⭐

Incremental한 방법은 모두 **에러가 누적**된다:

```
RT가 조금 틀림 → Triangulation 부정확 → 그걸로 한 PnP 부정확
→ 부정확한 포즈로 또 Triangulation → 점점 망가짐
```

대책: 초반부터 철저하게 관리 —
- 매 단계 RANSAC (F-RANSAC, Triangulation RANSAC, PnP RANSAC)으로 아웃라이어 제거.
- **Bundle Adjustment**를 매 스텝마다 수행.

## 3.4 Bundle Adjustment ⭐⭐

전체(bundle)를 다 집어넣고 조금씩 최적화:

\[
\min_{\{P_i\}, \{X_j\}} \sum_{i,j} v_{ij}\,\| \pi(P_i, X_j) - x_{ij} \|^2
\]

- Triangulation 때와 달리 이제 **\(P\)도 모르고 \(X\)도 모름** (둘 다 우리가 추정한 값) — 카메라 포즈와 포인트를 **동시에** 최적화.
- 핵심 논리: 이 에러는 **랜덤하게 0이 될 수 없다** — 모든 레이가 만나는 경우는 \(P\)와 \(X\)가 모두 정확할 때뿐. 따라서 reprojection error를 모든 카메라·모든 포인트에 대해 최소화하면 그게 곧 최적의 칼리브레이션 + 리컨.
- Highly nonlinear인데도 잘 풀리는 두 가지 이유 ⭐:
  1. **이니셜이 매우 좋음** — incremental 과정에서 기하학적으로 잘 큐레이팅된 \(P, X\)를 이미 보유.
  2. **자코비안이 매우 sparse** — 한 포인트를 보는 카메라는 일부뿐(visibility \(v_{ij}\): 안 보는 카메라는 카운트하지 않음) → gradient를 효율적으로 계산 가능.
- 규모 예시: 카메라 3,000개 × 포인트 수백만 개 → 카메라당 ~7 파라미터 + 포인트당 3 파라미터 = **약 3 million 파라미터 최적화**를 GPU 없는 뉴럴넷 이전 시대에 풀어냄.
- 데모: BA 한 번에 reprojection error **1.6px → 1.4px** (35K 포인트 + 53 카메라 미세 조정).
- 구조가 어느 정도 완성되면 BA를 sparse하게(가끔만) 돌려도 됨.

## 3.5 K도 같이 최적화 ⭐

이미지마다 \(K\)를 모르는 게 현실 — 그러나 \(K\)도 옵티마이제이션에 포함시키면 됨:

- skew \(s = 0\)으로 가정.
- \(p_x, p_y\) = 이미지 센터로 가정 (대부분 맞음).
- \(f_x = f_y\)로 가정.
- → **K는 사실상 focal length \(f\) 하나의 파라미터** (+ distortion 1개 정도).
- 이니셜 \(f\)는 이미지 사이즈 기반 임의값으로 줘도 최적화가 잘 동작.

→ **SfM은 패턴 없이도 칼리브레이션을 해주는 툴**. 요즘 3D 딥러닝 논문들이 "칼리브레이션이 주어져 있다"고 가정하는 것은 대부분 **COLMAP을 돌렸다**는 뜻.

## 3.6 고전 SfM의 한계와 러닝 기반 전환 (VGGT) ⭐

3D 비전은 이론적으로 거의 완벽한 분야였음 — 불안정한 건 correspondence 찾기뿐이었고 그것도 SIFT가 해결 → 2018년경까지 3D 하는 사람들은 뉴럴넷을 거의 안 씀 ("수학적으로 완벽한데 구멍이 어딨어").

고전 SfM의 두 가지 약점:

1. **느림** — 큰 스케일 리컨에는 거대한 최적화가 필요.
2. **매칭이 안 되면 답이 없음** — 하얀 벽(텍스처 없음), 뷰가 겹치지 않는 경우(수학적으로 불가능).

러닝 기반(VGGT 등, 작년 등장):

- 이미지에서 **depth를 직접 추정** → 겹침 여부와 무관하게 동작.
- 거대한 스케일로 학습하니 **정확도마저 고전 기법을 능가**.
- 가능한 이유: **사람이 사는 세상은 매우 구조화**되어 있음 — 모니터, 땅, 벽… 다 "봤던 것들"이라 러닝이 먹힐 여지가 큼.
- 추가 장점: 텍스트/오디오 등 **다른 모달리티와의 결합**이 쉬움 (모두 트랜스포머 기반).
- → 발표자는 VGGT 등장을 계기로 3D(기하학) 연구를 접고 러닝 영역으로 넘어갔다고 언급. 그래도 이론이 아름답고 쓸모가 많으니 공부해둘 가치가 있음.

질의응답:
- 카메라 파라미터 7개는 대략적인 수치 (더 많을 수도 있음).
- **렌즈 디스토션은 intrinsic \(K\)에 포함시킬 수 없음** — \(K\)는 linear 변환이라 라인이 라인으로 보존되는데, 디스토션은 라인을 구부리는 **nonlinear** 변환 → 별도 모델링 필요.

# 4. 실습 1 — 카메라 칼리브레이션 (체커보드 + Homography) ⭐

## 4.1 체커보드 포인트 생성

- 입력: board shape, square size(실제 메트릭 스케일) → 출력: \((x, y, 0)\) 형태의 3D 체커보드 코너 좌표.
- 9×6 = 54개 코너 → **54 × 3** 배열 (`meshgrid`/`mgrid`로 구현).
- 체커보드는 **평면**이므로 \(Z = 0\) 고정 — 보드를 고정하고 카메라를 추정하는 구도.
- 체커보드는 **코너 디텍션이 쉬운 형태**라 2D 좌표를 손쉽게 얻을 수 있음 → 3D-2D correspondence 확보.

## 4.2 Homography DLT — A 행렬 구성

- Homography \(H\)는 원소 9개, **up to scale이므로 8 DOF** → 대응점 1개당 식 2개 → **최소 4개의 대응점** 필요.
- 많을수록 오차가 줄어드니 전부 사용: N개의 대응점 → **\(2N \times 9\) 행렬 A** 구성 (`build_homography_A`).
- 대응점마다 두 줄씩 append → numpy array 변환.

## 4.3 SVD로 \(Ah = 0\) 풀기 ⭐

\(A = U \Sigma V^\top\)로 분해하면 **\(V\)의 마지막 컬럼이 솔루션**. 이유:

- trivial solution을 배제하기 위해 \(\|h\| = 1\) 제약하에 \(\|Ah\|\)를 최소화.
- \(U, V\)는 orthogonal → norm에 영향 없음 → \(V^\top h\)를 치환하면 \(\sum_i \sigma_i^2 h_i'^2\) 형태만 남음.
- 이를 최소화하면 **가장 작은 singular value에 해당하는 성분만 살아남음** → \(V\)의 마지막 컬럼.
- 마지막에 \(h_{22}\)(마지막 원소)로 나누어 정규화 — homogeneous 좌표 곱셈 시 마지막 항이 1이 되어 계산이 편함 (정규화 안 해도 해가 아닌 것은 아님).

## 4.4 Homography 적용 + 평가

- \((x, y)\) → homogeneous로 만들고(`np.ones`로 1 append) → \(H\) 곱 → 마지막 원소로 나눠 \((u, v)\) 획득 (디멘션/transpose 주의).
- 첫 번째 이미지에서 **reprojection error ≈ 0.749 px** — 애초에 reprojection error를 줄이는 방향으로 구한 것이므로 작게 나오는 게 정상.

## 4.5 Zhang's Method와 OpenCV 칼리브레이션 ⭐

- 체커보드 이미지 **여러 장**으로 구한 homography들에는 intrinsic과 extrinsic(R, T)이 섞여 있음.
- **Zhang's method**: rotation matrix 컬럼들의 직교성(\(R_1 \perp R_2\)) 등의 성질을 이용해 **최소 3장의 homography로 K/R/T 분리** 가능 (수업에서는 미사용, 개념만 소개).
- 실습에서는 **`cv2.calibrateCamera`** 사용: all object points + all image points 입력 → **RMS, K, distortion coefficients, rvecs, tvecs** 출력.
- 평가: `cv2.projectPoints`로 구한 K/R/T를 이용해 프로젝션 → 대응점과의 차이를 모아 평균 → **mean reprojection error ≈ 0.235 px**.

## 4.6 Radial Distortion ⭐

\[
\text{scale} = 1 + k_1 r^2 + k_2 r^4
\]

- Normalized coordinate의 \((x, y)\)에 위 스케일을 곱하는 형태로 distortion 구현 (`distort_normalized_points`).
- \(r^2\)을 구하고 스케일을 곱하는 단순 구현.
- **\(k_1 < 0\)이면 안쪽으로, \(k_1 > 0\)이면 바깥쪽으로** 점이 끌려가는 것을 시각적으로 확인.
- 마무리: 전통적으론 homography 3장 이상으로 칼리브레이션하지만, 최신 **GeoCalib**(싱글 이미지 기반 칼리브레이션 모델) 데모로 실습 종료.

# 5. 실습 2 — Structure from Motion 직접 구현 ⭐⭐

## 5.1 전체 구성

교수님 강의의 파이프라인을 코드로 재현:

```
Feature Track 구성 → Two-View Geometry로 첫 두 카메라 포즈
→ Triangulation → PnP로 카메라 추가 → 반복 (Incremental SfM)
→ 최신 연구 동향 리뷰
```

- 데이터: 두 사람이 아기를 안고 있는 **조각상(statue)을 여러 방향에서 찍은 40장의 이미지**.

## 5.2 Feature Track

- **여러 이미지에서 같은 3D를 보는 2D 포인트들을 하나로 묶은 것** — 나중에 하나의 3D 포인트가 됨.
- 두 이미지 간 correspondence 탐색 → **F-RANSAC으로 geometric verification** → 트랙 구성 (실습에서는 미리 계산해 제공).
- 공용 **DLT 함수** 구현: \(A = U\Sigma V^\top\) SVD 분해 후 \(V\)의 마지막 컬럼(코드상 \(V^\top\)의 마지막 row) 반환 = \(\|Ax\|\)를 최소화하는 \(x\).

## 5.3 Two-View Geometry — Fundamental / Essential Matrix ⭐

- 에피폴라 지오메트리의 필요성 두 가지:
  1. 로컬하게 비슷하지만 실제로는 다른 매칭(아웃라이어)을 **에피폴라 constraint로 검증/제거**.
  2. SfM 시작 시 **닭이 먼저냐 달걀이 먼저냐 문제** 해결 — PnP는 3D 포인트가 필요하고 Triangulation은 포즈가 필요한데, 처음엔 둘 다 없음 → **two-view geometry로 첫 두 카메라의 포즈를 결정**.
- **Fundamental matrix**: \(x'^\top F x = 0\). 대응점 하나당 linear equation 1개 (\(F\)를 9차원 벡터 \(f_{11} \dots f_{33}\)로 보고 전개) → 쌓아서 \(Ax = 0\) → DLT로 \(F\) 추정. 아웃라이어가 있으므로 **RANSAC과 함께** 사용.
- 시각화: 에피폴라 constraint 없이 매칭하면 아웃라이어(빨간색) 다수 발생 → constraint + RANSAC으로 제거. Fundamental matrix로 그린 **에피폴라 라인들의 교점이 epipole**; 한 이미지의 라인 위 점은 다른 이미지에서 같은 색 라인 위에 있어야 함.
- **Essential matrix**: 칼리브레이션이 끝났다고 가정하고 \(K^{-1}\)을 곱해 normalize → \(x'^\top E x = 0\). F보다 DOF가 작고 **두 카메라의 relative R, T만으로 표현** 가능 → 포즈 복원에 사용.
- **E 분해의 4-해 문제** ⭐: 첫 카메라를 원점(identity rotation)에 두면 수학적으로 \(R, T\) 해가 4개 나옴 → 실제 가능한 포즈는 1개 → **카메라 정면에 조각상(3D 포인트)이 보이는 해**(cheirality)를 선택. 4개 후보 중 1개만 조건 만족.

## 5.4 Triangulation 구현 + Nonlinear Refinement ⭐

- \(x \sim PX\) → 둘이 평행 → cross product = 0 → \(P\)를 row로 쪼개 식 3개, linear combination으로 **독립 식 2개/이미지**.
- 3D 포인트는 3 DOF → **카메라(포즈) 최소 2개** 필요. \(2N \times 4\) 행렬 A → SVD → 4×1 해 → 마지막 원소로 나눠 **inhomogeneous 3D 좌표**로 변환.
- **Nonlinear refinement** ⭐: DLT는 algebraic 풀이일 뿐 geometric 의미가 없음 → \(PX\)를 dehomogenize(나눗셈 때문에 nonlinear)한 projected 2D 점과 실측 2D 점의 **L2 distance를 최소화**.
  - `scipy.optimize.least_squares` 사용 — residual로 거리(차이)를 넘기면 residual 제곱합을 최소화.
  - 1차 미분만 쓰지 않고 2차(second order)까지 활용하는 **LM(Levenberg–Marquardt) optimization** → 매우 빠른 수렴 (Gauss-Newton 방식도 있음).
- 초기 두 뷰로 이니셜 포인트 클라우드 생성: DLT만 한 것 vs refinement까지 한 것이 눈으로는 큰 차이가 없지만, **incremental하게 누적되면 에러가 커지므로 좋은 이니셜 + 중간중간 BA가 필수**. 두 장만으로도 조각상 형태가 꽤 보임.

## 5.5 PnP의 DLT 풀이 ⭐

- \(K^{-1}\)을 곱해 normalize → 남는 것은 extrinsic 행렬 \(M = [R | T]\) (3×4).
- up to scale → cross product = 0 → 대응점 1개당 독립 식 2개.
- \(M\)은 미지수 12개, up to scale이므로 **11 DOF** → \(2N \times 12\) 행렬에서 **최소 N = 6개의 대응점** 필요 → \(AM = 0\)을 DLT로 풀이.
- **Post-processing** (DLT 해는 rotation의 제약을 만족하지 않음):
  - 앞 3×3 부분을 SVD: \(U\Sigma V^\top\)에서 **\(\Sigma\)를 버리고 \(UV^\top\)** → orthogonal 행렬화 (\(R^\top R = RR^\top = I\)); determinant를 항상 +1로 만드는 트릭 적용.
  - \(R\)이 바뀌었으니 \(T\)도 singular value들의 평균으로 나눠 스케일 보정.
  - \(\pm M\) 모두 해가 될 수 있음 → **카메라 앞에 3D 포인트들이 있어야 한다**(cheirality)로 부호 선택.
- **RANSAC의 중요성** ⭐: Triangulation은 앞 단계 F-RANSAC으로 이미 아웃라이어가 걸러졌지만, PnP는 그런 과정이 없어 **그냥 DLT를 돌리면 아웃라이어 때문에 포즈가 완전히 망가짐**. RANSAC을 쓰면 DLT만으로도 꽤 정확한 포즈, refinement까지 하면 더 정확.
- 그 외 P3P, P4P, EPnP 등 알고리즘 존재.

## 5.6 Incremental Loop 완성

```
while 남은 이미지:
    이미 만들어진 3D 포인트와 correspondence가 가장 많은 이미지 선택
    → PnP(RANSAC)로 카메라 포즈 등록
    → 새 카메라로 인해 생긴 새 트랙들 Triangulation
    → 수시로 Bundle Adjustment (전체 reprojection error 최소화로 드리프트 보정)
```

- 데모: 카메라가 하나씩 추가되며 포인트 증가, BA가 돌면 reprojection error가 다시 작아짐.
- 슬라이더 시각화: 정면 카메라만 있을 때는 뒤쪽 관측이 없어 뒤쪽 포인트가 없음 → 카메라가 한 바퀴 이어 붙으면 **조각상 뒷면까지 복원**.

## 5.7 최신 연구 동향 ⭐

| 방향 | 대표 연구 | 내용 |
|---|---|---|
| 고전 Incremental SfM | **COLMAP** | 오늘 구현한 것과 동일: SIFT 매칭 → geometric verification(F-RANSAC) → two-view init → Triangulation/PnP 반복 + BA |
| Global SfM | **GloMap** (2024) | Incremental 대신 전체 페어로 **global rotation → global position** 결정 후 BA — 한 번에 풀기 |
| 러닝 기반 feature/matcher | **SuperPoint, LoFTR, ALIKED / SuperGlue, LightGlue** | Handcrafted SIFT + Lowe's ratio 매칭을 러닝으로 대체 — **low-texture 영역에서도 키포인트** 추출 → 더 dense하고 정확한 리컨 |
| Differentiable BA | **DROID-SLAM, VGG-SfM** | BA 레이어를 미분 가능하게 만들어 전체를 trainable하게 — 앞단(Triangulation/PnP)까지 gradient가 흘러 더 좋은 이니셜 학습 |
| Feed-forward 리컨 | **DUSt3R → MASt3R → VGGT** | 지오메트리(에피폴라 등)를 아예 버리고 **이미지에서 바로 pointmap 예측**. DUSt3R: 2장 → 공통 pointmap; VGGT: N장 → 카메라 + depth + pointmap 동시 예측 (네트워크/데이터 대규모 확장). Sparse 키포인트 매칭과 달리 **전 픽셀 dense 예측** 가능. 이후 Pi3, Depth Anything 3 등 후속 |
| 동적(4D) 리컨 | CVPR 2025 best paper 류 | 고전 SfM은 static 장면 전제(움직이면 geometric verification 실패) — 러닝으로 비디오 입력의 **트래킹 포함 4D reconstruction**까지 발전 |

- 마무리 데모: Depth Anything 3(작은 모델)로 같은 데이터를 feed-forward로 복원해 비교해볼 수 있는 셀 제공 (큰 모델일수록 결과 우수).

# 6. 오후 핵심 정리 ⭐

- **Triangulation**: 칼리브레이션된 카메라들과 2D 대응점이 주어지면 3D 포인트를 복원 — 레이는 실제로 만나지 않으므로 **reprojection error를 최소화하는 \(X\)** 를 찾는다. 카메라당 식 2개, 최소 2대면 풀이 가능, DLT(SVD)로 이니셜을 잡고 nonlinear refinement.
- **Algebraic(DLT) vs Geometric(reprojection) error**: 해가 0이면 일치하지만 0이 아니면 미묘하게 다름 — 진짜 원하는 건 geometric, DLT는 그 좋은 이니셜.
- **PnP**: \(K\)와 3D-2D 대응점에서 카메라 포즈(\(R, T\))를 추정 — 최소 3점(P3P), 리니어 DLT 풀이는 6점 이상 + rotation constraint 후처리. AR의 핵심 알고리즘.
- **Incremental SfM**: two-view geometry(E 분해)로 첫 두 포즈 → Triangulation → PnP로 카메라 추가 → 다시 Triangulation → 반복. **에러가 누적**되므로 매 단계 RANSAC(F-RANSAC, PnP-RANSAC)과 **Bundle Adjustment**가 필수.
- **Bundle Adjustment**: 모든 카메라와 모든 포인트의 reprojection error를 동시에 최소화 — reprojection error는 모두 정확할 때만 0이 될 수 있다는 성질 + **좋은 이니셜 + sparse한 visibility(자코비안)** 덕에 수백만 파라미터도 풀림. \(K\)도 \(f\) 하나로 퉁쳐 같이 최적화 가능 → SfM = 패턴 없는 칼리브레이션 (COLMAP).
- **고전 SfM의 한계**(느림, 텍스처 없음/뷰 안 겹침에 무력) → **VGGT 등 feed-forward 러닝 기반**이 등장해 속도·강건성·정확도 모두 추월 — 세상이 구조화되어 있어 러닝이 먹히고, 트랜스포머 기반이라 타 모달리티 결합도 용이.
- **칼리브레이션 실습**: 체커보드(\(Z=0\) 평면) 코너로 3D-2D 대응 → Homography DLT(\(2N \times 9\), SVD의 \(V\) 마지막 컬럼, 최소 4점) → 여러 장이면 Zhang's method로 K/R/T 분리(원리만 소개), 실습은 `cv2.calibrateCamera` (mean reprojection error 0.235px) → Radial distortion \(1 + k_1 r^2 + k_2 r^4\), \(k_1\) 부호에 따라 안/밖으로 왜곡.
- **SfM 구현 실습**: F/E matrix를 DLT로 추정, E 분해의 4개 해 중 cheirality로 선택 → Triangulation(\(2N \times 4\)) + LM refinement → PnP DLT(11 DOF, 6점 이상) + rotation 직교화 후처리 + RANSAC 필수 → incremental loop로 조각상 40장 복원.
- **연구 동향**: COLMAP(고전) → GloMap(global) → 러닝 feature/matcher(SuperPoint/LightGlue) → differentiable BA(DROID-SLAM) → feed-forward(DUSt3R/MASt3R/VGGT/Depth Anything 3) → 4D dynamic reconstruction.
