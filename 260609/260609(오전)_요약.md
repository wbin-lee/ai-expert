> 강의 주제: **3D 포인트–카메라–2D 이미지의 수학적 관계를 다루기 위한 기초로서, 무한 점까지 포함하는 Projective Geometry와 Homogeneous Coordinate를 익히고, 2D↔2D 평면 매핑인 Homography를 모델링·추정하는 것** (주완별 교수, 멀티뷰 지오메트리 텍스트북의 내용을 이틀 압축 강의 중 첫 오전).

# 1. 컴퓨터 비전 인트로 — Recognition vs Reconstruction ⭐

## 1.1 비전의 두 축

"본다"는 것의 두 가지 정의:

| 분야 | 문제 | 성격 |
|---|---|---|
| **Recognition** | 이미지에서 **뭐가 어디에** 있는지 알아맞히기 (픽셀별 사람/캐비넷 등) | 시맨틱 이해 |
| **Reconstruction** | 2D로 촬영된 이미지를 **다시 3D로 복원** — depth, 거리, 직각 등 지오메트리 추정 | 카메라를 통한 센싱 |

- 딥러닝 혁명의 시작점이 비전(ImageNet 챌린지의 AlexNet, 90년대에 사라졌던 뉴럴넷의 부활).
- 전통적으로 2D 비전(recognition)과 3D 비전(geometry)은 따로 발전했지만 서로 연관:
  - 3D 씬 이해가 좋으면 occlusion 처리 등 recognition에 유리.
  - 물체가 뭔지 알면(예: 사람 조각상) 안 보이는 부분도 복원 가능 → recognition이 reconstruction에 도움.
- 러닝 기반 방법 등장으로 두 분야의 결합이 쉬워짐 (데이터 넣고 트레이닝) → 나아가 비전+언어+오디오 **멀티모달** 통합 학습 시대.

## 1.2 Recognition의 서브필드 정리

| 문제 | 입력 | 출력 | 특징 |
|---|---|---|---|
| **Image Classification** | 이미지 1장 | 클래스 1개 (객관식, 예: ImageNet 1000개) | 어노테이션 쉬움(장당 5~10초), "바보 같은" 문제라 뉴럴넷 적용 쉬움 → 혁명의 출발점 |
| **Detection** | 이미지 1장 | 모든 물체의 **bounding box + 클래스** | 미싱/오탐 모두 패널티, 박스 어노테이션이 비싸서 카테고리 80~100개 수준 (PASCAL, COCO) |
| **Correspondence Search** | 이미지 2장 | 두 이미지 간 **대응점** | 시점/조명/시간 변화에도 같은 점을 찾아야 함 — 3D 비전 쪽의 recognition 문제 |

클래식 시대에는 각 문제의 SOTA 알고리즘이 완전히 달랐음 ⭐:

- Classification → **Bag of Words**
- Detection → **HOG**
- Correspondence → **SIFT feature**

→ 사람의 직관(intuition)에 입각해 알고리즘을 짰기 때문에 문제 성격마다 방법이 따로 발전. 지금은 죄다 end-to-end 러닝으로 통일 — 성능은 좋아졌지만 왜 잘 되는지 모르는 블랙박스.

# 2. 3D 비전의 역사와 현재 ⭐

## 2.1 3D 비전이란

- 그래픽스: 3D 에셋 → 리얼리스틱한 2D 이미지 (렌더링 방향).
- 3D 비전: **2D 이미지 → 3D 씬 복원** (반대 방향) — 촬영하며 사라진 depth를 예측해야 하므로 더 어려운 면이 있음.

## 2.2 3D 비전의 부활

- CVPR 키워드 랭킹: 2021년 "3D"가 3위(detection, segmentation보다 위), 2023년에는 object보다 위 — 매우 핫한 분야가 됨.
- 역사적 흐름:
  1. 2000년대 초중반: SfM 등 멀티뷰 재구성이 사실상 풀림 (2008~2010년경 도시 단위 복원, 구글 스트리트뷰) → "3D는 끝났다"며 detection/recognition으로 이동.
  2. 딥러닝이 detection/segmentation을 너무 잘 풀어버림 → 더 어려운 3D로 회귀.
  3. **NeRF**(Neural Radiance Field): 러닝+리컨 결합으로 포인트 클라우드의 "구멍"이 없는, 어떤 뷰로든 렌더링되는 결과 → 3D 붐의 계기.
  4. Generative 모델: 텍스트 → 3D 에셋 생성, 4D(3D+시간) 생성, 심지어 **이미지 1장 → 원하는 뷰로 돌려보기**(전통적 리컨 없이 view synthesis로 푸는 방식).
  5. 최근에는 **Physical AI / 로보틱스** 키워드와 연결되며 더욱 인기.

## 2.3 본 강의의 목표 ⭐

멀티뷰 이미지(같은 static 씬을 여러 각도에서 촬영)만 주면:

- 씬의 **3D 포인트 클라우드**
- 각 이미지를 찍은 **카메라 포즈** (피라미드로 시각화)

를 동시에 복원. 핵심은 **Correspondence(대응점)만 있으면 지오메트릭하게 계산 가능**하다는 것. 같은 알고리즘을 인터넷 관광 사진에 적용하면 도시 전체도 복원됨.

→ 이를 위해 **3D 포인트 – 카메라 – 2D 이미지의 수학적 관계**를 배운다. 렌즈는 빛을 꺾어 복잡하므로 **핀홀 카메라(바늘구멍 사진기)** 모델로 근사 — 실제 카메라의 매우 좋은 근사이며, 렌즈 디스토션만 별도 처리하면 됨.

# 3. Projective Geometry의 동기 — 소실점 문제 ⭐⭐

## 3.1 소실점(Vanishing Point)의 역설

- 이미지 속 소실점은 **finite한 위치(x, y)** 에 있는 점.
- 그러나 실제로는 **평행선들이 만나는 점 = 무한(infinity)에 있는 점**.
- 카메라 촬영 = 3D → 2D 매핑 **함수**인데, 이 함수가 **infinity의 점을 finite한 점으로 떨어뜨림**.
- 문제: 유클리디안 기하에서는 **infinity를 수학적으로 표기(write down)할 방법이 없음** → 카메라 모델(매핑 함수)을 정의하기 전에 infinity 점부터 처리해야 함.

## 3.2 Projective Geometry = Euclidean + Infinity ⭐

- 유클리디안 기하: 두 점의 최단거리는 직선, 삼각형 내각 합 180° 등 — 이 안에서는 무한 점을 정의할 수 없음.
- **Projective geometry = 유클리디안 기하에 infinity point까지 합친 확장**.
- 장점: 두 직선의 위치 관계에 **if-else가 필요 없어짐**. 만나든(finite) 평행이든(infinite) **항상 교점이 정의**되고 표현법이 동일 → algebraic하게 구분 불필요. 3D 비전 공부가 크게 간편해짐.

# 4. Homogeneous Coordinates ⭐⭐

## 4.1 정의

2D 점 \((x_1, x_2)^T \in \mathbb{R}^2\) (= **inhomogeneous coordinate**, 유클리디안 스페이스의 점)에 대해:

\[
(x_1, x_2)^T \;\longrightarrow\; (x_1, x_2, 1)^T, \quad \text{단 } k \neq 0 \text{에 대해 } k(x_1, x_2, 1)^T \text{는 모두 같은 점}
\]

- 차원을 하나 추가하고 1을 붙임 → 3D 벡터지만 **여전히 2D 점을 표현**.
- 임의의 nonzero \(k\)를 곱해도 같은 점 = **up to scale** — 절댓값이 아니라 **ratio만 중요**.
- (0,0,0)을 제외한 모든 3D 벡터는 어떤 2D 점을 표현.

## 4.2 변환 규칙

- Inhomogeneous → Homogeneous: 마지막에 1 추가.
- Homogeneous → Inhomogeneous: **마지막 원소로 나누고** 1이 된 마지막 원소를 드롭. 예: \((\alpha, \beta, \gamma)^T \to (\alpha/\gamma, \beta/\gamma)^T\).

## 4.3 Degree of Freedom ⭐

- 3차원 벡터지만 up to scale이므로 **DoF = 2** (하나를 고정하면 ratio만 자유) → 일반 규칙: **up to scale이면 차원 수에서 1을 뺀 것이 DoF**.
- 예: \((a, a, a)^T\) 같은 형태는 DoF = 1.

## 4.4 기하학적 해석 — 점을 Ray로 ⭐

2D 평면을 3D 공간의 **depth 1 위치**에 세운다고 생각:

- 2D 점 \((x_1, x_2)\)는 3D에서 \((x_1, x_2, 1)\)이 됨.
- \(k\)를 곱하면 원점에서 그 점을 지나는 **ray 위에서** 멀어지거나 가까워짐 → **하나의 2D 점 = 원점을 지나는 하나의 ray** (ray 위의 모든 점은 같은 점).
- Homogeneous → inhomogeneous 변환 = **ray와 평면(depth=1)의 교점** 찾기.
- **Ideal point**: 이미지 평면과 **평행한 ray** → 평면과의 교점이 존재하지 않음 → 유클리디안으로 돌아올 수 없는 점.

## 4.5 Ideal Point (Point at Infinity) ⭐

\[
(x_1, x_2, 0)^T
\]

- 마지막 원소가 0 → 0으로 나눌 수 없으므로 **inhomogeneous로 되돌릴 수 없음**.
- Projective space에서는 멀쩡히 존재하지만 Euclidean space에는 존재할 수 없는 점 = **infinity의 점**.
- "마지막이 0이면 infinity"는 단순한 약속이 아니라, 실제로 **평행선들의 교점을 계산하면 항상 이 꼴이 나옴** (아래 §5에서 증명).

→ 이제 "무한의 점이 함수를 통과해 finite한 점이 됐다"를 \((x_1, x_2, 0)\to\) (마지막이 0이 아닌 벡터)로 **algebraic하게 쓸 수 있음**. 이것이 projective space 도입의 핵심 이유.

# 5. 점과 선의 관계 — Cross Product ⭐⭐

## 5.1 라인의 Homogeneous 표현

라인 방정식 \(a x_1 + b x_2 + c = 0\)을 내적 형태로:

\[
\mathbf{l}^T \mathbf{x} = 0, \quad \mathbf{l} = (a, b, c)^T
\]

- 라인도 **3D 벡터(계수만 기억)** 로 표현되며 역시 **up to scale** (k를 곱해도 같은 라인).
- **점이 라인 위에 있다 ⟺ \(\mathbf{l}^T \mathbf{x} = 0\)** (inner product = 0).
- 라인의 DoF도 2 (3차원 − up to scale).

## 5.2 두 점을 지나는 라인 ⭐

\[
\mathbf{l} = \mathbf{x} \times \mathbf{x}'
\]

증명: cross product의 정의상 \(\mathbf{l}\)은 \(\mathbf{x}\), \(\mathbf{x}'\) 둘 다와 orthogonal → \(\mathbf{l}^T\mathbf{x} = 0\), \(\mathbf{l}^T\mathbf{x}' = 0\) → 라인이 두 점을 모두 지남. (증명 시에만 3D 벡터의 직교성으로 생각하고, 개념상으로는 2D 점/라인의 incidence로 이해.) **연립방정식을 풀 필요 없이** 한 번에 계산.

## 5.3 두 라인의 교점 ⭐

\[
\mathbf{x} = \mathbf{l} \times \mathbf{l}'
\]

같은 논리로 \(\mathbf{x}\)는 \(\mathbf{l}\) 위에도, \(\mathbf{l}'\) 위에도 있음 → 교점.

## 5.4 예제

- \(x=1\) 라인: \(-x + 1 = 0\) → \(\mathbf{l} = (-1, 0, 1)^T\); \(y=1\) 라인: \(\mathbf{l}' = (0, -1, 1)^T\).
- 교점: \(\mathbf{l} \times \mathbf{l}' = (1, 1, 1)^T\) → inhomogeneous로 \((1,1)\). ✓

**평행선의 경우** ⭐:

- \(x=1\): \((-1, 0, 1)^T\), \(x=2\): \((-1, 0, 2)^T\) → 교점 = \((0, 1, 0)^T\).
- **마지막 원소가 0** → projective space에서만 정의되는 ideal point. 평행선들의 교점은 모두 이런 꼴.

## 5.5 모든 평행선은 같은 점에서 만난다 ⭐

- 임의의 \(c\)에 대해 \(x = c\) 라인 \((-1, 0, c)^T\)는 모두 \((0,1,0)^T\)와 내적이 0 → **\(c\)와 무관하게 같은 ideal point를 지남**.
- 직관: **북극성** — 내가 아무리 이동(offset \(c\))해도 북극성을 향하는 라인은 항상 같은 방향. 너무 멀리 있어서 내 이동이 무의미.
- 방향이 다르면 다른 ideal point에서 만남.

## 5.6 Line at Infinity와 지평선 ⭐

\[
\mathbf{l}_\infty = (0, 0, 1)^T
\]

- 모든 ideal point \((x_1, x_2, 0)\)와 내적이 0 → **모든 ideal point를 지나는 라인 = line at infinity**.
- 카메라로 촬영하면 line at infinity가 **지평선**으로 찍힘 — 어떤 방향의 평행선이든 그 소실점은 모두 지평선 위에 있음.

# 6. Homography — 2D Projective Space 간의 매핑 ⭐⭐

## 6.1 정의

- **Homography = \(\mathbb{P}^2 \to \mathbb{P}^2\)의 invertible한 매핑** (projective space라서 infinity 점도 포함 가능 — 유클리디안에선 정의 불가).
- 핵심 성질: **collinearity 보존** — source에서 한 라인 위에 있던 세 점은 매핑 후에도 한 라인 위에 있음 (순서나 간격은 셔플될 수 있으나 라인은 라인으로; 곡선이 되면 homography가 아님).
- 결론: **라인이 보존되는 매핑 ⟺ invertible한 3×3 행렬로 표현되는 linear 매핑** (양방향 모두 성립):

\[
\mathbf{x}' = H \mathbf{x}, \quad H \in \mathbb{R}^{3\times 3}, \; \text{up to scale}
\]

## 6.2 왜 Homography인가 — Central Projection ⭐

카메라 촬영에서 **3D 공간의 평면(바닥/벽/천장) 위 점들 ↔ 이미지 평면**의 매핑은 핀홀(pinhole center)을 지나는 ray를 통한 **central projection**이고, 이런 류의 평면↔평면 매핑은 모두 3×3 homography로 설명됨.

- 일반적인 3D→2D는 homography가 아니라 **3×4 카메라 모델**(다음 시간)이 필요. 단 "평면 위의 3D만" 생각하면 3×3으로 충분.
- 평면 하나를 두 카메라가 볼 때: (평면→이미지1)도 3×3, (평면→이미지2)도 3×3 → **이미지1↔이미지2도 3×3**.

## 6.3 Pure Rotation의 특수 케이스 ⭐

- 카메라 센터를 움직이지 않고 **순수 회전(pure rotation)** 만 하며 찍은 두 이미지 사이의 매핑은 **depth와 무관하게** homography를 만족.
- 파노라마 촬영 시 UI가 "제자리에서 돌리라"고 하는 이유 — 카메라를 평행이동하며 찍으면 같은 평면(depth)만 맞고 다른 depth는 어긋남(ghosting).
- 일반적인 경우 파노라마 같은 결과를 얻으려면 3D 리컨 후 렌더링해야 하지만, pure rotation이면 2D homography만으로 **파노라마 스티칭** 가능.

## 6.4 응용

- 화면에 그린 그림을 바닥/벽에 정확히 투영하는 **특수효과** (2D↔2D 매핑만 알면 됨).
- 자동차 **서라운드 뷰**: 전/측방 카메라 이미지를 바닥 평면으로 투영해 합성 — 3D 리컨 불필요.
- 비스듬히 찍은 평면(예: 반도체 웨이퍼)을 **정면에서 본 것처럼 펴기** (rectification).

# 7. 2D 변환의 계층 구조 ⭐⭐

## 7.1 Translation

\[
H = \begin{pmatrix} I & \mathbf{t} \\ \mathbf{0}^T & 1 \end{pmatrix}
\]

유클리디안에서의 "더하기" \(\mathbf{x}' = \mathbf{x} + \mathbf{t}\)를 homogeneous에서는 **행렬 곱 하나**로 표현 (3D 비전은 더하기를 싫어함 — 모든 걸 행렬 곱으로).

## 7.2 Euclidean (Rigid) Transform — DoF 3

\[
\mathbf{x}' = R\mathbf{x} + \mathbf{t}, \quad
H = \begin{pmatrix} R & \mathbf{t} \\ \mathbf{0}^T & 1 \end{pmatrix}, \quad
R = \begin{pmatrix} \cos\theta & -\sin\theta \\ \sin\theta & \cos\theta \end{pmatrix}
\]

- \(R\)의 DoF = 1 (\(\theta\) 하나), \(\mathbf{t}\)의 DoF = 2 → 총 **3**.
- 모양과 크기(길이, 각도) 모두 보존. 로보틱스 등에서 매우 중요.

**적용 순서 주의** ⭐: 비전에서는 항상 **Rotation 먼저, Translation 나중** (\(R\mathbf{x} + \mathbf{t}\)). 순서를 바꾸면(T 먼저 후 R) 원점 중심 회전 때문에 완전히 다른 결과. (그래픽스 등에선 반대 관례도 있음.)

## 7.3 Similarity Transform — DoF 4

\[
H = \begin{pmatrix} sR & \mathbf{t} \\ \mathbf{0}^T & 1 \end{pmatrix}
\]

Rotation에 scale \(s\) 추가 (DoF 3+1=4). 모양(각도)은 유지, 크기/길이는 변할 수 있음.

## 7.4 Affine Transform — DoF 6

\[
H = \begin{pmatrix} A & \mathbf{t} \\ \mathbf{0}^T & 1 \end{pmatrix} = \begin{pmatrix} a_{11} & a_{12} & t_1 \\ a_{21} & a_{22} & t_2 \\ 0 & 0 & 1 \end{pmatrix}
\]

- 마지막 row를 \((0,0,1)\)로 고정하고 위 6개는 자유 → 찌그러질 수 있음(각도 변화).
- **핵심 성질: parallelism 보존** ⭐ — 증명: ideal point \((x_1, x_2, 0)^T\)에 적용해도 마지막 원소가 여전히 0 → **infinity의 점은 infinity에 머무름** → 평행선의 교점이 여전히 infinity → 평행선은 평행 유지.

## 7.5 Projective Transform (일반 Homography) — DoF 8

- 마지막 row 제약 없음, 3×3 전체 자유. 9개 원소이나 **up to scale이라 DoF = 8**.
- 마지막 row가 \((0,0,1)\)이 아니면 ideal point가 **finite point로 떨어질 수 있음** → 평행성 깨짐 (소실점 현상이 바로 이것).

## 7.6 불변량(Invariants) 요약표 ⭐

| 변환 | DoF | 보존되는 것 |
|---|---|---|
| Euclidean (Rigid) | 3 | 길이, 각도, 넓이 (모두) |
| Similarity | 4 | 각도 (길이는 ×) |
| Affine | 6 | **Parallelism** (각도는 ×) |
| Projective | 8 | **Collinearity**만 (라인은 라인으로, 곡선 안 됨) |

**실용적 의미** ⭐: 변환의 종류를 미리 알면 **미지수(=필요한 식/constraint 수)를 줄일 수 있음**. Rigid임을 알면 식 3개면 충분, 모르면 full 8개 필요. 클래식이든 모던이든 결국 "함수를 어떻게 정의하고 미지수가 몇 개냐"의 문제.

# 8. Homography 추정 — DLT (Direct Linear Transform) ⭐⭐

## 8.1 문제 설정

- 알고 있는 것: 매핑이 homography라는 **모델** (\(\mathbf{x}'_i = H\mathbf{x}_i\)) + 누군가 준 **대응점들** \((\mathbf{x}_i, \mathbf{x}'_i)\).
- 모르는 것: 3×3 행렬 \(H\).

## 8.2 Up to Scale 처리 ⭐

\(\mathbf{x}' = H\mathbf{x}\)는 **등호가 아니라 up to scale** (양변 모두 homogeneous). ratio 핸들링을 피하려면 양변을 **inhomogeneous로 변환**: \(H\mathbf{x}\)를 계산해 마지막 원소로 나눈 뒤 드롭:

\[
x' = \frac{h_{11}x + h_{12}y + h_{13}}{h_{31}x + h_{32}y + h_{33}}, \qquad
y' = \frac{h_{21}x + h_{22}y + h_{23}}{h_{31}x + h_{32}y + h_{33}}
\]

→ 이제 진짜 등호 성립. 분모를 양변에 곱해 정리하면 **대응점 1개당 \(\mathbf{h}\)(9개 원소를 벡터화)에 대한 선형 식(row) 2개**가 나옴 (아는 값은 좌표들, 모르는 건 \(h\)들만).

## 8.3 필요한 대응점 수 ⭐

- 미지수: 9개이지만 up to scale → **DoF 8**.
- 대응점 1개 = 식 2개 → **최소 4개의 대응점**이면 식 8개로 \(H\)를 결정.

## 8.4 Ah = 0 형태와 SVD 풀이 ⭐

모든 대응점의 row들을 **concatenation**해 하나의 큰 시스템으로:

\[
A\,\mathbf{h} = \mathbf{0} \quad (\text{대응점 } n \text{개} \to A \in \mathbb{R}^{2n \times 9})
\]

- **Homogeneous Least Squares** 문제. Trivial solution \(\mathbf{h}=\mathbf{0}\)을 배제하기 위해 \(\|\mathbf{h}\| = 1\) 제약 (어차피 up to scale이라 길이는 무관):

\[
\min_{\mathbf{h}} \|A\mathbf{h}\|^2 \quad \text{s.t.} \quad \|\mathbf{h}\| = 1
\]

- 풀이: \(A = UDV^T\)로 **SVD**한 뒤 **\(V\)의 마지막 column**이 최적해. 코드 두 줄.
- Linear한 convex optimization이라 **closed form으로 global optimum 보장** (단, 대응점이 엉망이면 "구해지긴 하지만 원하지 않는" \(H\)가 나올 수 있음).

## 8.5 대응점이 많을수록 좋은 이유 ⭐

- 4개만 있을 때 하나라도 잘못 찍으면 에러가 \(H\)에 그대로 전파 → 불안정.
- 100개를 모두 concat하면 식 > 미지수 (over-constrained) → exact solution이 없을 수 있으나 \(\|A\mathbf{h}\|\)를 최소화하는 **robust한 최적화 해**를 얻음. 측정값(measurement)이 충분하면 모두 leverage하는 것이 에러에 강인 — 연립방정식 손풀이로는 불가능한 장점.

## 8.6 DLT의 일반성 ⭐

이 방식(DLT)은 homography만이 아니라:

- **카메라 캘리브레이션** (카메라 파라미터 추정)
- **3D Reconstruction / Structure from Motion** (도시 복원)

모두 거의 같은 형태로 풀림 — projection도 homography도 결국 **linear model**이기 때문. Homography만 제대로 익히면 나머지는 비슷한 컨셉.

# 9. 대응점은 누가 주는가 — SIFT와 실전 이슈 ⭐

- 실제로는 사람이 클릭하는 게 아니라 **SIFT** 같은 디텍터가 대응점을 자동으로 제공 — 2000년대 초반에 풀린 문제로, 3D 비전 발전의 결정적 토대 (SfM, 파노라마 모두 이 위에서 동작).
- SIFT = **Scale-Invariant Feature Transform**: 같은 장면을 가까이/멀리서 찍으면 패치 크기를 어떻게 맞춰 비교할지 모르는 난제(scale 문제)를 해결한 방법.
- 뉴럴넷 시대에도 **문제의 formulation(인풋/아웃풋 정의)을 이해하는 것**이 중요 — 방법이 클래식이든 러닝이든.
- 실전 이슈: 창문처럼 **반복 패턴**이 있으면 대응점이 셔플되어 **outlier**가 섞임 → 그대로 DLT에 넣으면 엉망인 homography → **inlier만 골라내는 처리**가 필요 (outlier rejection).
- 파노라마 스티칭 파이프라인: SIFT 대응점 → (inlier 선별) → DLT로 \(H\) → 한쪽 이미지의 모든 픽셀을 다른 쪽 공간으로 변환 → 겹치는 부분은 일치, 새 부분은 이어붙음 → 반복하면 긴 파노라마.

## 9.1 Radial Distortion (질문에서 파생) ⭐

- 스티칭 결과에 곡선이 보이는 것은 homography 탓이 아님 — linear 매핑이므로 라인은 반드시 라인으로 보존됨.
- 원인은 **카메라 렌즈의 radial distortion**: 렌즈가 빛을 꺾어(광각일수록 심함) 직선이 곡선으로 찍히는 현상. **Linear한 핀홀 카메라 모델이 커버하지 못하는 비선형 효과**.
- 해결: 디스토션 모델로 **전처리(undistortion)** 를 먼저 해서 아티팩트를 없앤 뒤, 가장 간단한 linear 매핑(핀홀 모델)으로 다룸.

# 10. 다음 시간 예고 — 카메라 모델 ⭐

- 지금까지: 2D↔2D 매핑 (homography, 3×3). 다음: **3D↔2D 매핑 = 카메라 모델**.
- 3D 점의 homogeneous 표현은 4D 벡터 (마찬가지로 1 추가) → 매핑은 **3×4 행렬 \(P\)** (projection matrix):

\[
\mathbf{x} = P\mathbf{X}, \quad P \in \mathbb{R}^{3\times 4}
\]

- \(P\)는 카메라가 보는 각도(포즈)와 내부 파라미터(focal length 등)로 결정. 카메라마다 자기만의 \(P\)가 있음.
- **Calibration**: homography에서 \(H\)를 구했듯 대응관계(3D↔2D)로 \(P\)를 직접 계산 — 역시 DLT류.
- Calibration이 되면: 임의의 3D를 받아 2D로 projection 가능 (3D 모델을 영상 위에 합성하는 응용 등, PnP 문제도 살짝 언급 예정).
- **Triangulation**: 두 카메라가 같은 3D 점을 보고 있다는 대응 정보(\(\mathbf{x}_1 \leftrightarrow \mathbf{x}_2\))가 있으면 그 3D 점을 복원 → 카메라 파라미터를 알면 2D만으로 3D 복원.
- 인터넷에서 받은 이미지는 카메라 파라미터를 모르므로 calibration부터 시작 — 전부 지오메트릭하게 엮여 있어 **러닝 없이도** 대응점만으로 모든 것을 구할 수 있음 (SfM의 원리).

# 11. 오전 핵심 정리 ⭐

- 컴퓨터 비전의 두 축은 **Recognition**(뭐가 어디에)과 **Reconstruction**(2D→3D 복원)이며, 본 강의는 멀티뷰 지오메트리에 기반한 클래식 reconstruction을 다룬다 — **대응점만 있으면 3D 포인트와 카메라 포즈를 지오메트릭하게 계산**할 수 있다.
- 소실점은 **infinity의 점이 카메라 매핑을 통해 finite로 떨어진 것**인데, 유클리디안 기하로는 infinity를 표기할 수 없다 → **Projective Geometry = Euclidean + infinity point**가 필요하다.
- **Homogeneous coordinate**는 차원을 하나 늘리고(1 추가) **up to scale**로 정의한 표현 — DoF는 여전히 2이며, 기하적으로는 점을 원점 통과 **ray**로 매핑한 것이다. 마지막 원소가 0인 점이 **ideal point(무한 점)** 다.
- 점도 선도 3D 벡터로 표현되고, **두 점을 잇는 선 = \(\mathbf{x} \times \mathbf{x}'\)**, **두 선의 교점 = \(\mathbf{l} \times \mathbf{l}'\)** — if-else 없이 평행선의 교점까지 일괄 계산되며, 평행선들은 모두 같은 ideal point에서 만나고(북극성 직관) ideal point들은 **line at infinity \((0,0,1)^T\)** (= 지평선) 위에 있다.
- **Homography**는 collinearity(라인)를 보존하는 \(\mathbb{P}^2 \to \mathbb{P}^2\) 매핑 ⟺ **invertible 3×3 행렬**. 평면↔이미지(central projection), 평면을 보는 두 이미지 사이, **pure rotation**으로 찍은 두 이미지 사이가 모두 homography다 — 파노라마 스티칭, AR 특수효과, 서라운드 뷰의 원리.
- 변환 계층: **Rigid(DoF 3, 길이 보존) ⊂ Similarity(4, 각도 보존) ⊂ Affine(6, 평행성 보존 — 마지막 row 001이라 infinity가 infinity에 머묾) ⊂ Projective(8, collinearity만 보존)**. 변환 종류를 알면 미지수를 줄일 수 있다. 비전에서는 항상 **R 먼저, t 나중**.
- **DLT**: up to scale 등식을 inhomogeneous로 바꿔 대응점당 식 2개 → DoF 8이므로 **최소 대응점 4개** → 모두 concat해 \(A\mathbf{h}=0\)을 \(\|\mathbf{h}\|=1\) 제약 하에 **SVD(V의 마지막 column)** 로 푼다. 대응점이 많을수록 에러에 robust하며, 같은 틀로 캘리브레이션·SfM도 풀린다.
- 대응점은 **SIFT**가 자동으로 제공하지만 반복 패턴 등으로 outlier가 섞이므로 inlier 선별이 필요하고, 직선이 곡선으로 찍히는 것은 **radial distortion**(렌즈, 비선형) 때문이라 전처리로 제거한 뒤 linear 모델을 쓴다.
- 다음 시간: **핀홀 카메라 모델(3×4 projection matrix)**, calibration, triangulation — 2D↔2D의 일반화인 3D↔2D 매핑.
