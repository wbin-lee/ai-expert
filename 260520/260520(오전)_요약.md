# 1. DQN과 Q-learning 복습

## 1.1 RL 문제의 목표

- 사용자는 agent가 달성해야 할 임무를 reward로 설계한다.
- agent의 목표는 expected cumulative reward를 최대화하는 policy를 찾는 것이다.
- MDP에서는 transition probability와 reward model을 알고 있기 때문에 Value Iteration, Policy Iteration으로 policy를 직접 계산할 수 있었다.
- RL에서는 model을 알 수 없으므로 simulation 또는 interaction sample을 이용해 policy를 학습해야 한다.

## 1.2 왜 \(V\)가 아니라 \(Q\)를 학습했는가

- state value \(V(s)\)만 알면 model-free setting에서 optimal action을 바로 고르기 어렵다.
- \(V\)로 policy를 뽑으려면 transition probability \(P(s'|s,a)\)가 필요하다.
- 반면 Q function은 현재 state와 action까지 고정한 뒤의 누적 reward 기대값이다.

\[
Q^*(s,a)
\]

- \(Q^*\)를 알면 model 정보 없이도

\[
\pi^*(s)=\arg\max_a Q^*(s,a)
\]

로 optimal policy를 바로 얻을 수 있다.

## 1.3 Q-learning update

RL에서 sample 한 단위는 보통 다음 형태이다.

\[
(s,a,r,s')
\]

Q-learning은 Bellman equation의 우변을 single-sample estimate로 사용한다.

\[
Q(s,a)\leftarrow (1-\alpha)Q(s,a)+\alpha\left[r+\gamma\max_{a'}Q(s',a')\right]
\]

- 임의의 behavior policy로 데이터를 만들어도 된다.
- 따라서 Q-learning은 off-policy 특성을 가진다.
- 다만 tabular Q-learning은 특정 \((s,a)\) entry만 local하게 업데이트하므로 state/action space가 커지면 scalability 문제가 생긴다.

## 1.4 Approximate Q-learning과 DQN

Approximate Q-learning은 Q function을 함수 근사기로 표현한다.

\[
Q_\phi(s,a)
\]

핵심은 Bellman target과 현재 Q estimate의 차이를 줄이는 regression 문제를 푸는 것이다.

DQN은 Q function을 neural network로 근사한 value-based deep RL 알고리즘이다.

# 2. DQN의 핵심 아이디어 복습

## 2.1 naive neural Q-learning의 문제

단순히 Q function을 neural network로 근사하면 두 문제가 생긴다.

1. sequential sample 사이의 correlation이 크다.
2. target value가 network update에 따라 계속 움직인다.

## 2.2 Experience Replay

Experience Replay는 interaction sample을 replay buffer에 저장한 뒤 random mini-batch로 다시 뽑아 학습하는 방법이다.

- sample correlation을 줄인다.
- 과거 sample을 재사용하므로 sample efficiency가 좋아진다.
- Q-learning의 off-policy 특성 때문에 가능하다.

## 2.3 Target Network

DQN은 두 Q network를 둔다.

| network | 역할 |
|---|---|
| online Q network \(Q_\phi\) | 실제 학습되는 Q network |
| target Q network \(Q_{\phi^-}\) | target 계산에 사용 |

target은 보통 다음과 같이 계산된다.

\[
y=r+\gamma\max_{a'}Q_{\phi^-}(s',a')
\]

- target network를 천천히 업데이트해 moving target 문제를 완화한다.
- hard update 또는 Polyak averaging을 사용할 수 있다.

\[
\phi^- \leftarrow (1-\tau)\phi^-+\tau\phi
\]

## 2.4 epsilon-greedy와 overestimation

- DQN 자체에는 exploration이 없으므로 epsilon-greedy를 붙인다.
- 초기에는 epsilon을 크게 두고 점차 줄이는 것이 합리적이다.
- DQN은 max operator 때문에 overestimation 문제가 생길 수 있다.
- Double DQN은 action selection과 value evaluation에 다른 network를 사용해 이 문제를 완화한다.
- 실전에서는 vanilla DQN보다 Double DQN을 default로 고려하는 것이 자연스럽다.

# 3. Value-Based에서 Policy-Based로

## 3.1 value-based 방법

Q-learning과 DQN은 policy를 직접 학습하지 않는다.

- value space에서 \(Q\)를 학습한다.
- 학습된 \(Q\)에서 argmax로 policy를 추출한다.

대표 알고리즘:

- Q-learning
- DQN
- Double DQN

## 3.2 policy-based 방법

policy-based 방법은 policy 자체를 직접 parameterize하고 학습한다.

stochastic policy는 다음처럼 표현한다.

\[
\pi_\theta(a|s)
\]

의미:

- 현재 state \(s\)에서 action \(a\)를 선택할 확률이다.
- action selection이 확률적으로 이루어진다.
- policy network parameter \(\theta\)를 직접 최적화한다.

---

# 4. Policy Gradient의 Objective

## 4.1 trajectory 정의

state-action trajectory를 다음처럼 둔다.

\[
\tau=(s_0,a_0,s_1,a_1,\dots,s_T,a_T)
\]

MDP의 Markov property와 stochastic policy를 이용하면 trajectory distribution은 다음처럼 factorization된다.

\[
p_\theta(\tau)
=p(s_0)\prod_{t=0}^{T}\pi_\theta(a_t|s_t)P(s_{t+1}|s_t,a_t)
\]

## 4.2 policy parameter 최적화 문제

누적 reward를 \(R(\tau)\)라고 하면 objective는 다음과 같다.

\[
J(\theta)
=\mathbb{E}_{\tau\sim p_\theta(\tau)}[R(\tau)]
\]

목표는 \(J(\theta)\)를 최대화하는 \(\theta\)를 찾는 것이다.

\[
\max_\theta J(\theta)
\]

## 4.3 gradient ascent

policy gradient는 gradient ascent로 parameter를 업데이트한다.

\[
\theta_{k+1}
=\theta_k+\alpha\nabla_\theta J(\theta_k)
\]

- \(\alpha\): step size 또는 learning rate.
- 핵심 문제는 model-free 방식으로 \(\nabla_\theta J(\theta)\)를 계산하는 것이다.

---

# 5. Policy Gradient Theorem의 유도 직관

## 5.1 expectation을 적분으로 표현

\[
J(\theta)=\int R(\tau)p_\theta(\tau)d\tau
\]

이를 \(\theta\)에 대해 미분하면

\[
\nabla_\theta J(\theta)
=\int R(\tau)\nabla_\theta p_\theta(\tau)d\tau
\]

가 된다.

## 5.2 log-derivative trick

로그 미분 공식:

\[
\nabla_\theta \log p_\theta(\tau)
=\frac{\nabla_\theta p_\theta(\tau)}{p_\theta(\tau)}
\]

따라서

\[
\nabla_\theta p_\theta(\tau)
=p_\theta(\tau)\nabla_\theta \log p_\theta(\tau)
\]

이를 넣으면

\[
\nabla_\theta J(\theta)
=\mathbb{E}_{\tau\sim p_\theta}
\left[
R(\tau)\nabla_\theta\log p_\theta(\tau)
\right]
\]

## 5.3 transition probability가 사라지는 이유

\[
\log p_\theta(\tau)
=\log p(s_0)
+\sum_t \log \pi_\theta(a_t|s_t)
+\sum_t \log P(s_{t+1}|s_t,a_t)
\]

이를 \(\theta\)로 미분하면

- initial state distribution은 \(\theta\)와 무관하므로 사라진다.
- transition probability도 policy parameter \(\theta\)와 무관하므로 사라진다.
- 남는 것은 policy term뿐이다.

\[
\nabla_\theta\log p_\theta(\tau)
=\sum_t \nabla_\theta\log \pi_\theta(a_t|s_t)
\]

따라서

\[
\nabla_\theta J(\theta)
=\mathbb{E}_{\tau\sim p_\theta}
\left[
R(\tau)\sum_t \nabla_\theta\log \pi_\theta(a_t|s_t)
\right]
\]

이것이 policy gradient theorem의 핵심 형태이다.

## 5.4 의미

- transition probability를 직접 알 필요가 없다.
- 현재 policy로 trajectory sample을 만들고 sample average를 취하면 gradient를 model-free로 estimate할 수 있다.
- 이 점이 policy gradient 방법의 핵심이다.

# 6. REINFORCE 알고리즘

## 6.1 기본 절차

REINFORCE는 가장 기본적인 policy gradient 알고리즘이다.

1. 현재 policy \(\pi_\theta\)로 여러 trajectory를 생성한다.
2. 각 trajectory에서 누적 reward를 계산한다.
3. 각 trajectory에 대해

\[
R(\tau)\sum_t\nabla_\theta\log\pi_\theta(a_t|s_t)
\]

를 계산한다.
4. 여러 trajectory에 대해 sample average를 취한다.
5. 이 gradient estimate로 \(\theta\)를 업데이트한다.

## 6.2 장점

- 알고리즘이 단순하다.
- 이상적인 setting에서는 unbiased gradient estimate를 얻을 수 있다.
- 충분히 정확한 gradient를 얻으면 local optimum 방향으로 이동할 수 있다.

## 6.3 단점

### high variance

- trajectory sample 수가 적으면 gradient estimate의 variance가 매우 크다.
- 복잡한 문제일수록 많은 trajectory가 필요하다.

### on-policy

- 항상 현재 policy \(\pi_\theta\)로 sample을 생성해야 한다.
- 한 번 gradient 계산에 사용한 sample은 policy update 후 재사용하기 어렵다.
- sample efficiency가 낮다.

## 6.4 왜 current policy가 필요한가

gradient ascent에서 필요한 것은 현재 parameter \(\theta_k\) 위치에서의 gradient이다.

따라서 \(\theta_{k-1}\)로 만든 오래된 policy sample을 쓰면 \(\theta_k\)에서의 gradient를 제대로 estimate하기 어렵다.

이 때문에 vanilla policy gradient는 on-policy 특성을 가진다.

# 7. Gaussian Policy 예시

## 7.1 policy 형태

간단한 stochastic policy로 Gaussian policy를 생각할 수 있다.

\[
\pi_\theta(a|s)=\mathcal{N}(\mu_\theta(s),\Sigma)
\]

- mean vector \(\mu_\theta(s)\)는 neural network가 출력한다.
- covariance matrix \(\Sigma\)는 고정한다.
- parameter \(\theta\)는 mean network에만 들어간다.

## 7.2 gradient 계산

Gaussian log probability에 대해 \(\theta\) gradient를 계산하면 chain rule을 통해 mean network의 gradient가 들어간다.

- 간단한 경우에는 직접 수식으로 구현할 수 있다.
- 복잡한 policy에서는 automatic differentiation을 사용하면 된다.

# 8. Variance를 줄이기 위한 Baseline

## 8.1 baseline 아이디어

policy gradient에서 reward term에 baseline \(B\)를 빼는 것을 생각한다.

\[
\mathbb{E}
\left[
(R(\tau)-B)\nabla_\theta\log p_\theta(\tau)
\right]
\]

이상적으로 \(B\)가 \(\theta\), \(\tau\)와 독립이면 expectation 관점에서 bias를 만들지 않는다.

## 8.2 왜 baseline이 variance를 줄이는가

variance는 대략 reward term의 크기에 영향을 받는다.

\[
R(\tau)-B
\]

가 작아질수록 gradient estimate의 variance가 줄어든다.

따라서 baseline은 누적 reward를 잘 대표하는 값이면 좋다.

## 8.3 value function을 baseline으로 사용

누적 reward의 기대값을 나타내는 대표적 함수가 value function이다.

\[
V^\pi(s)
\]

그래서 baseline으로 value function을 사용하는 것이 자연스럽다.

# 9. Advantage Function

## 9.1 정의

\[
A^\pi(s,a)=Q^\pi(s,a)-V^\pi(s)
\]

## 9.2 직관

현재 state \(s\)에서 action \(a\)를 고정했을 때, 그냥 policy \(\pi\)를 따르는 평균적 선택보다 얼마나 더 좋은지를 나타낸다.

- \(Q^\pi(s,a)\): \(s\)에서 \(a\)를 한 뒤 policy \(\pi\)를 따를 때의 기대 누적 reward.
- \(V^\pi(s)\): \(s\)에서 처음부터 policy \(\pi\)를 따를 때의 기대 누적 reward.
- 둘의 차이가 action \(a\)의 advantage이다.

## 9.3 baseline이 들어간 policy gradient

실전에서는 다음 형태를 많이 사용한다.

\[
\nabla_\theta J(\theta)
\approx
\mathbb{E}
\left[
\nabla_\theta\log\pi_\theta(a|s)
A^{\pi_\theta}(s,a)
\right]
\]

장점:

- variance를 줄인다.

단점:

- value estimate를 사용하므로 bias가 생길 수 있다.
- 그러나 실험적으로 variance 감소의 이득이 더 큰 경우가 많다.

---

# 10. Actor-Critic으로의 확장

## 10.1 critic이 필요한 이유

advantage를 쓰려면 \(Q^{\pi_\theta}\), \(V^{\pi_\theta}\) 또는 \(A^{\pi_\theta}\)를 계산해야 한다.

이 값들을 Monte Carlo로 추정할 수도 있지만 variance가 크다.

따라서 별도 value-based RL 모듈을 두고 현재 policy의 value를 학습한다.

이 모듈을 critic이라고 부른다.

## 10.2 actor와 critic의 역할

| 구성 요소 | 역할 |
|---|---|
| Actor | policy \(\pi_\theta\)를 parameterize하고 policy gradient로 업데이트 |
| Critic | 현재 policy의 \(V^\pi\), \(Q^\pi\), \(A^\pi\)를 평가 |

Actor는 critic이 제공한 평가를 이용해 더 낮은 variance의 gradient를 계산한다.

## 10.3 Policy Iteration과의 비유

Actor-Critic은 Policy Iteration과 구조적으로 유사하다.

| Policy Iteration | Actor-Critic |
|---|---|
| Policy evaluation | Critic |
| Policy improvement | Actor |

# 11. Critic 학습 방법

## 11.1 Monte Carlo target

value network \(V_\phi(s)\)를 학습할 때 target을 실제 누적 reward로 둘 수 있다.

\[
\sum_{t=0}^{T}\gamma^t r_t
\]

장점:

- bias가 작다.

단점:

- variance가 크다.

## 11.2 Bellman target

Bellman equation의 single-sample estimate를 target으로 사용할 수도 있다.

\[
y_t=r_t+\gamma V_\phi(s_{t+1})
\]

장점:

- Monte Carlo target보다 variance가 낮다.

단점:

- 현재 value estimate를 target에 사용하므로 bias가 생길 수 있다.

## 11.3 advantage의 single-sample estimate

\[
A(s_t,a_t)
\approx r_t+\gamma V_\phi(s_{t+1})-V_\phi(s_t)
\]

이 값은 temporal difference error와도 연결된다.

## 11.4 discount factor 주의점

discount factor가 있는 문제에서는 target 계산에 반드시 \(\gamma\)가 들어가야 한다.

Monte Carlo target:

\[
\sum_t \gamma^t r_t
\]

Bellman target:

\[
r_t+\gamma V_\phi(s_{t+1})
\]

강의에서 특히 target 계산 시 discount factor를 빼먹는 실수를 주의해야 한다고 강조했다.

# 12. Actor-Critic의 한계

## 12.1 variance와 bias trade-off

critic을 도입하면 gradient variance는 줄어든다.

하지만 다음 이유로 bias가 생긴다.

- baseline 도입 자체에서 bias가 생길 수 있다.
- critic의 value estimate에도 error가 있다.

즉, Actor-Critic은 high-variance unbiased gradient 대신 lower-variance biased gradient를 사용하는 방식으로 볼 수 있다.

## 12.2 여전히 on-policy

기본 Actor-Critic은 여전히 현재 policy로 sample을 만들어야 한다.

- sample 재사용이 어렵다.
- sample efficiency 문제가 남는다.

## 12.3 Fully incremental 버전

batch를 모으지 않고 single sample이 들어올 때마다 critic과 actor를 바로 업데이트할 수도 있다.

- 현재 policy로 action을 만든다.
- single-sample Bellman target으로 critic을 inexact하게 업데이트한다.
- single-sample advantage로 actor를 stochastic gradient 방식으로 업데이트한다.

단점:

- 학습 안정성이 더 떨어질 수 있다.
- 여전히 on-policy이다.

---

# 13. Off-Policy Actor-Critic 아이디어

## 13.1 behavior policy

on-policy 한계를 줄이기 위해 behavior policy \(\beta\)로 데이터를 만들 수 있다.

그러면 trajectory distribution이 \(p_\theta\)가 아니라 \(p_\beta\)가 된다.

## 13.2 objective 변화

현재 policy로 만들어진 sample이 아니므로 원래 objective와 다른 문제를 풀게 된다.

- sample reuse는 가능해진다.
- 하지만 objective 자체가 바뀌거나 gradient estimator에 bias가 생긴다.

## 13.3 한계

behavior policy를 잘 고르면 exploration 측면에서 이득이 있을 수 있지만, 모든 문제에 안정적으로 적용되는 것은 아니다.

이 문제를 더 정교하게 다루기 위해 이후 TRPO/PPO에서 importance sampling과 trust region 관점을 다룬다고 예고했다.

---

# 14. DDPG로 넘어가기

## 14.1 DDPG의 핵심

DDPG는 Deep Deterministic Policy Gradient의 약자이다.

핵심 특징:

- deep RL 알고리즘이다.
- deterministic policy를 사용한다.
- actor-critic 구조이다.
- critic에는 DQN의 핵심 아이디어를 사용한다.
- continuous control에 적합하다.

## 14.2 stochastic policy와 deterministic policy

stochastic policy:

\[
\pi_\theta(a|s)
\]

- state에서 action에 대한 확률분포를 출력한다.
- exploration이 자연스럽게 포함될 수 있다.
- 하지만 on-policy 성격이 강하고 계산이 복잡할 수 있다.

deterministic policy:

\[
\mu_\theta(s)
\]

- state를 action으로 직접 mapping한다.
- memory와 inference 측면에서 간단하다.
- continuous action을 직접 출력하기 좋다.
- 단독으로는 exploration이 없으므로 noise를 추가해야 한다.

---

# 15. Deterministic Policy Gradient

## 15.1 직관

stochastic policy gradient처럼 trajectory distribution을 직접 미분하는 대신, critic의 Q function을 actor parameter에 대해 미분한다.

현재 policy가 출력한 action을 Q function에 넣는다.

\[
Q^\mu(s,\mu_\theta(s))
\]

이 값을 \(\theta\)에 대해 미분한다.

## 15.2 chain rule

\[
\nabla_\theta Q^\mu(s,\mu_\theta(s))
=
\nabla_a Q^\mu(s,a)\big|_{a=\mu_\theta(s)}
\nabla_\theta \mu_\theta(s)
\]

즉,

- critic이 action 방향 gradient \(\nabla_a Q(s,a)\)를 제공한다.
- actor는 \(\nabla_\theta \mu_\theta(s)\)를 제공한다.
- 두 값을 chain rule로 연결해 actor를 업데이트한다.

## 15.3 discounted state visitation distribution

deterministic policy gradient에서는 \(\mu\)를 따라 방문하는 state distribution이 들어간다.

- 현재 policy로 어떤 state들을 얼마나 자주 방문하는지가 gradient expectation에 반영된다.
- discount factor가 있으면 늦게 도달하는 state에는 더 작은 weight가 붙는다.

---

# 16. Deterministic Actor-Critic

## 16.1 기본 구조

1. 현재 deterministic policy \(\mu_\theta\)로 action을 만든다.
2. environment에서 sample을 얻는다.
3. critic \(Q_\phi^\mu\)를 학습한다.
4. deterministic policy gradient로 actor를 업데이트한다.

## 16.2 exploration

deterministic policy 자체에는 randomness가 없다.

그래서 action에 Gaussian noise 같은 인위적 noise를 더한다.

\[
a_t=\mu_\theta(s_t)+\epsilon_t
\]

이는 DQN의 epsilon-greedy와 비슷한 역할을 하지만, 효과적인 exploration을 보장하지는 않는 heuristic이다.

## 16.3 on-policy 문제

순수 deterministic actor-critic 역시 기본적으로 on-policy 문제가 남는다.

이를 off-policy로 만들기 위해 behavior policy와 replay buffer를 사용하는 방향으로 확장한다.

---

# 17. Off-Policy Deterministic Actor-Critic

## 17.1 핵심 근사

behavior policy \(\beta\)로 데이터를 만들고, 그 데이터를 이용해 deterministic actor gradient를 추정한다.

이 과정에서는 엄밀한 gradient에서 일부 term을 drop하는 근사가 들어간다.

## 17.2 장단점

장점:

- replay buffer의 과거 sample을 재사용할 수 있다.
- sample efficiency가 좋아진다.

단점:

- gradient estimator에 bias가 커질 수 있다.
- behavior policy와 target policy의 차이가 문제를 만들 수 있다.

---

# 18. DDPG: DQN + Off-Policy Deterministic Actor-Critic

## 18.1 DDPG의 구성

DDPG는 다음 두 아이디어를 결합한다.

| 구성 | DDPG에서의 역할 |
|---|---|
| Deterministic Policy Gradient | actor 업데이트 |
| DQN | critic 학습 안정화 |

critic은 optimal Q가 아니라 현재 deterministic policy \(\mu\)의 Q function을 학습한다.

\[
Q^\mu(s,a)
\]

## 18.2 DQN과 다른 점

DQN의 target:

\[
y=r+\gamma\max_{a'}Q_{\phi^-}(s',a')
\]

DDPG critic의 target:

\[
y=r+\gamma Q_{\phi^-}(s',\mu_{\theta^-}(s'))
\]

차이:

- DQN은 optimal Q를 학습하므로 action maximization이 들어간다.
- DDPG는 현재 policy의 Q를 학습하므로 max가 없다.
- max 대신 target actor가 다음 state에서 낼 action을 넣는다.

## 18.3 continuous control에 적합한 이유

continuous action space에서는

\[
\max_a Q(s,a)
\]

를 푸는 것이 어렵다.

DDPG는 target 계산에서 max를 하지 않고 actor output을 바로 넣는다.

따라서 continuous action 문제에 자연스럽다.

또 actor update에서도 \(\nabla_a Q(s,a)\)를 사용하므로 action이 연속공간일 때 gradient가 자연스럽게 정의된다.

---

# 19. DDPG 알고리즘

## 19.1 네트워크

DDPG는 네 개의 network를 사용한다.

| network | 역할 |
|---|---|
| actor \(\mu_\theta\) | deterministic policy |
| critic \(Q_\phi\) | 현재 policy의 Q function |
| target actor \(\mu_{\theta^-}\) | target action 계산 |
| target critic \(Q_{\phi^-}\) | target Q 계산 |

## 19.2 sample 수집

action은 exploration noise를 더해 선택한다.

\[
a_t=\mu_\theta(s_t)+\epsilon_t
\]

transition을 replay buffer에 저장한다.

\[
(s_t,a_t,r_t,s_{t+1},done_t)
\]

## 19.3 critic target

mini-batch sample에 대해 target을 계산한다.

\[
y_i=r_i+\gamma Q_{\phi^-}(s_i',\mu_{\theta^-}(s_i'))
\]

terminal state에서는 masking을 적용해야 한다.

## 19.4 critic update

critic은 regression으로 학습한다.

\[
\min_\phi
\frac{1}{N}\sum_i
\left(
Q_\phi(s_i,a_i)-y_i
\right)^2
\]

## 19.5 actor update

actor는 critic이 평가한 Q를 최대화하도록 업데이트한다.

\[
\max_\theta
\frac{1}{N}\sum_i
Q_\phi(s_i,\mu_\theta(s_i))
\]

gradient descent optimizer를 쓰려면 loss는 보통 다음처럼 둔다.

\[
L_{\text{actor}}
=-\frac{1}{N}\sum_i Q_\phi(s_i,\mu_\theta(s_i))
\]

## 19.6 target network update

critic과 actor target network 모두 Polyak averaging으로 update할 수 있다.

\[
\phi^- \leftarrow (1-\tau)\phi^-+\tau\phi
\]

\[
\theta^- \leftarrow (1-\tau)\theta^-+\tau\theta
\]

---

# 20. DDPG의 장점과 한계

## 20.1 장점

- continuous control을 자연스럽게 다룬다.
- DQN처럼 replay buffer를 사용하므로 sample efficiency가 좋다.
- target network를 통해 critic 학습을 안정화한다.
- action maximization이 없어 continuous action에서 계산 부담이 작다.

## 20.2 한계

- deterministic policy라 exploration이 약하다.
- action noise는 heuristic이며 효율적 exploration을 보장하지 않는다.
- noise scale 조절이 중요하고 tuning이 필요하다.
- off-policy 근사 때문에 bias가 생길 수 있다.

## 20.3 SAC로의 연결

DDPG의 exploration 한계는 이후 Soft Actor-Critic에서 entropy 관점으로 다룬다.

SAC는 stochastic policy와 entropy bonus를 사용해 exploration을 더 체계적으로 설계한다.

---

# 21. DDPG 실험 예시

강의에서는 DDPG가 continuous control 문제를 해결한 사례들을 소개했다.

- Inverted Pendulum balancing
- Half Cheetah locomotion
- physical control task
- joint information을 사용한 low-dimensional feature 기반 학습
- pixel input만 사용한 end-to-end learning

의미:

- DQN이 주로 discrete action game에 강했다면, DDPG는 physical continuous control 문제로 deep RL 적용 범위를 넓혔다.
- 완전한 물리 모델 없이도 data와 neural network만으로 control policy를 학습할 수 있음을 보였다.

# 22. 다음 강의 예고

## 22.1 4일차: TRPO와 PPO

다음날에는 policy gradient 계열의 더 근본적인 문제를 다룬다.

주요 질문:

- gradient 방향으로 그대로 이동하는 것이 정말 좋은가?
- step size를 어떻게 안정적이면서도 공격적으로 정할 것인가?
- policy update가 너무 크게 변하지 않도록 어떻게 제한할 것인가?

TRPO는 trust region과 monotonic improvement를 통해 이 문제를 다룬다.

PPO는 TRPO를 더 구현 친화적으로 단순화한 알고리즘이다.

## 22.2 PPO와 LLM

PPO는 시뮬레이션 비용이 낮은 환경에서 강력하게 쓰이며, LLM fine-tuning 계열에서도 사용된 바 있다.

이후 GRPO도 함께 언급될 예정이라고 했다.

# 23. 오전 핵심 정리

- Q-learning/DQN은 value-based 방법으로, Q function을 학습한 뒤 argmax로 policy를 추출한다.
- policy gradient는 policy 자체를 parameterize하고 expected cumulative reward를 직접 최대화한다.
- log-derivative trick 덕분에 transition probability를 몰라도 model-free gradient estimate가 가능하다.
- REINFORCE는 단순하지만 high variance와 on-policy sample inefficiency가 크다.
- baseline을 도입하면 variance를 줄일 수 있고, value function을 baseline으로 쓰면 advantage function \(A=Q-V\)가 나온다.
- critic은 현재 policy의 value를 평가해 actor가 더 안정적인 gradient를 쓰게 해준다.
- Actor-Critic은 Policy Iteration의 policy evaluation/improvement 구조와 닮아 있다.
- DDPG는 deterministic actor와 DQN-style critic을 결합한 off-policy actor-critic 알고리즘이다.
- DDPG는 continuous action에서 max operator 없이 target을 만들 수 있어 continuous control에 적합하다.
- 다만 deterministic policy의 exploration 문제는 action noise heuristic에 의존하며, 이후 SAC에서 더 체계적으로 다룬다.
