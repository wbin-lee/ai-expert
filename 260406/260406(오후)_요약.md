# 260406 오후 강의 요약 — 지능형 에이전트·탐색·확률 추론

## 1. 지능형 에이전트(Intelligent Agent)

### 1.1 에이전트의 정의
**Agent** : 환경을 지각(Sensing)하고 액추에이터(Actuators)를 통해 행동(Action)하는 자율 시스템.
```
환경 → Sensors → (내부 추론·의사결정) → Actuators → 환경
```

### 1.2 PEAS 프레임워크
에이전트 설계의 4요소:
| 요소 | 의미 | 배달 로봇 예시 |
|------|------|----------------|
| **P**erformance | 성능 척도 | 정확한 전달, 충돌 없음, 짧은 시간 |
| **E**nvironment | 환경 | 사무실 복도·방·사람들·장애물 |
| **A**ctuators | 작동기 | 바퀴, 로봇 팔, 적재함, 스피커 |
| **S**ensors | 센서 | 카메라, 라이다, 위치 센서, 마이크 |

### 1.3 Rational Agent
주어진 정보와 내장된 지식으로 **기대 효용(expected utility)을 최대화** 하는 행동을 선택하는 에이전트. 현실에서는 정보·시간·연산이 제한되어 "Bounded Rationality (Herbert Simon)" 또는 *satisficing* 이 필요하다.

## 2. 탐색(Search) 알고리즘

### 2.1 탐색 문제의 구성
- **상태(State)** : 환경의 구체적 상황
- **액션(Action)** : 상태 전이 연산자
- **경로(Path)** : 상태들의 순열
- **목표(Goal)** / **경로 비용(Path Cost)**

### 2.2 무정보 탐색 (Uninformed Search)
- **BFS (너비 우선)** : 얕은 깊이부터 체계적 탐색
- **DFS (깊이 우선)** : 깊이 우선 스택 구조
- **UCS (균일 비용)** : 누적 비용 최소 경로 우선

### 2.3 정보 탐색 (Informed Search)
- **Greedy Best-First** : 휴리스틱 `h(n)` 만 사용
- **A\*** : 다음 식으로 평가
  ```
  f(n) = g(n) + h(n)
  ```
  - `g(n)` : 시작점에서 n까지의 실제 비용
  - `h(n)` : n에서 목표까지의 휴리스틱 추정값
  - h가 **admissible(이상적 하한)** 이면 A*는 최적해 보장
  - 가장 작은 f(n)부터 확장

### 2.4 게임 트리 탐색
- **Minimax** : 자신(Max)의 최대화 ↔ 상대(Min)의 최소화 가정
- **α-β 가지치기** : 탐색 공간 축소
- 실전에서는 **깊이 제한 + 평가함수** 필요

## 3. 확률 기초와 베이즈 정리

### 3.1 기본 공식
```
조건부 확률   P(A|B) = P(A ∩ B) / P(B)
전확률 법칙   P(A) = Σᵢ P(A|Bᵢ) P(Bᵢ)
베이즈 정리   P(H|E) = P(E|H) P(H) / P(E)
```

### 3.2 베이즈 정리 해석
- `P(H)` : 사전확률(Prior)
- `P(E|H)` : 우도(Likelihood)
- `P(E)` : 증거(Evidence) — 전확률로 분해
- `P(H|E)` : 사후확률(Posterior)

## 4. Bayesian Network (확률 그래프 모델)

### 4.1 정의와 구성
확률 변수들의 **조건부 독립 구조** 를 DAG(유향 비순환 그래프)로 표현.
- 노드 = 변수
- 화살표 = 직접 인과/의존 관계

### 4.2 결합 확률 분해
```
P(X₁, X₂, …, Xₙ) = ∏ᵢ P(Xᵢ | Parents(Xᵢ))
```

### 4.3 추론 방법
- **Variable Elimination (변수 제거)** : 해당 없는 변수를 marginalize
- **Belief Propagation (신념 전파)** : 메시지 패싱
- 대형 네트워크엔 근사추론(MCMC, Variational Inference) 사용

## 5. 기호주의 AI와 지식 표현

### 5.1 논리적 추론
- **명제 논리(Propositional Logic)** : 참/거짓 명제 조합
- **술어 논리(First-Order Logic)** : 변수, 양화사 포함
- **Forward Chaining** : 사실 → 규칙 → 새로운 사실 (bottom-up)
- **Backward Chaining** : 목표 → 필요한 사실 (top-down)

### 5.2 Modus Ponens (전건 긍정)
```
P → Q
P
⊢ Q
```
예시 : `Father(John, Brian)` 과 `∀x,y Father(x,y) → Male(x) ∧ Adult(x)`
- 변수 치환 : x=John, y=Brian
- 적용 규칙 : Modus Ponens
- 중간 결론 : `Male(John) ∧ Adult(John)`
- 최종 결론 : `Adult(John)`

### 5.3 지식 구조
- **Semantic Network (의미망)** : 노드 + 관계
- **Frame (프레임)** : 슬롯 기반 객체 표현
- **Script** : 전형적 상황 시나리오

## 6. 강화학습(Reinforcement Learning) 개요

### 6.1 핵심 요소
- **State s**, **Action a**, **Reward r**, **Policy π(a|s)**
- **MDP(Markov Decision Process)** : (S, A, P, R, γ)
- **Value Function** : `V(s) = E[Σ γᵗ rₜ | s₀ = s]`
- **Policy** : 상태에서 행동을 선택하는 규칙

### 6.2 기본 원리
시행착오(trial-and-error) 로 환경과 상호작용하며 **누적 보상을 최대화**하는 정책을 학습.

## 7. 알파고(AlphaGo) 핵심 기술 (예고편)

바둑처럼 탐색 공간이 엄청난 경우 :
- **정책망(Policy Network)** : 다음 수 후보 축소 (신경망)
- **가치망(Value Network)** : 현재 판세의 가치 추정 (신경망)
- **MCTS (Monte Carlo Tree Search)** : 시뮬레이션 기반 탐색

→ 탐색(기호주의) + 학습(연결주의)의 하이브리드.

## 8. 핵심 정리

1. Agent = Sensor → 의사결정 → Actuator 루프.
2. PEAS 4요소로 에이전트를 기술한다.
3. Rational Agent = 기대 효용 최대화. 현실은 Bounded Rationality.
4. 탐색 : 무정보(BFS/DFS/UCS) / 정보(Greedy / A*).
5. A* 의 `f(n) = g(n) + h(n)` 에서 h가 admissible이어야 최적해 보장.
6. 게임 트리는 Minimax + α-β + 평가함수.
7. 베이즈 정리 `P(H|E) ∝ P(E|H) P(H)` 로 사후확률 갱신.
8. Bayesian Network는 조건부 독립 구조로 결합확률을 분해.
9. 결합확률 `P(X₁…Xₙ) = ∏ P(Xᵢ | Parents(Xᵢ))`.
10. Modus Ponens : `P→Q, P ⊢ Q` (대표적 추론 규칙).
11. 지식 구조 : Semantic Network / Frame / Script.
12. 강화학습은 시행착오의 체계화, 장기 보상 최대화.
13. MDP 4(또는 5)요소 : S, A, P, R (, γ).
14. 불확실성 = 부분 관측성 + 동적 환경 → 확률 추론 필요.
15. AlphaGo = 딥러닝(정책/가치) + MCTS의 하이브리드.

### 강의 중 주요 직관 비유
- **알파고 vs 딥블루** : 탐색(minimax)으로 충분한 체스와, 직관+탐색이 모두 필요한 바둑의 패러다임 차이. 20년 간격이 곧 패러다임 차이.
- **카너만의 제한된 합리성** : 인간도 항상 합리적이지 않다 → AI도 "적당히 좋은 해(satisficing)"를 찾는 것이 현실적.
- **의사결정 게임** : 불확실성 속 제한된 정보로 최선 선택 → Bounded Rationality 의 핵심 예.
