## 0. 한 줄 요약

> **외부 Claude Code의 토큰 한계**와 **사내 AI Agent의 성능 한계**를 동시에 푸는 라우팅 솔루션을 개발하여, 5월 5,000명 / 6월 10,000명 규모로 확대되는 Claude Code 사내 도입의 **비용 효율과 활용도 정착**을 가속화한다.

## 1. 배경 및 문제 정의

### 1.1 현황
- **외부 Claude Code**: 5월 5,000명, 6월 10,000명 사내 오픈. 성능은 우수하나 **사용자당 토큰량이 한정**되어 다수 사용자가 비용·할당량 제약을 체감 중.
- **사내 AI Agent**: 사용량 제한이 없거나 매우 완화되어 있으나, 외부 모델 대비 추론·도구 활용 성능이 부족하여 복잡한 SW 개발 작업에는 단독 사용이 어려움.
- **결과**: 두 도구가 **분리된 채 병존**, 사용자는 작업 유형별로 직접 선택해야 함 → 학습 곡선·전환 비용 증가, 토큰 낭비 발생.

### 1.2 문제 정의
개발자가 하나의 인터페이스에서 작업을 수행할 때, **작업의 난이도·중요도에 따라 외부/사내 모델을 자동 분배**하고, 결과 품질이 보장되도록 검증·재시도 루프를 갖춘 오케스트레이션 계층이 부재하다.

### 1.3 제안의 정합성
- 팀장님 지속 요구 사항인 "**Claude Code를 사내에서 잘 쓰게 하는 도구/방법론**" 직접 대응.
- 부서 내 기존 컨셉인 **AIDLC(AI-Driven Lifecycle)** 의 실체화에 해당 — 워크플로우를 도구로 풀어내는 1차 형태.
- 2026 부서 전략 "**AI4SE 집단 지성 확보**"에서 요구하는 사내 AI 자산 활용도 극대화.

## 2. 제안: 하이브리드 에이전트 오케스트레이션

### 2.1 핵심 아이디어
- **고수준 의사결정**(작업 계획, 아키텍처 결정, 코드 리뷰 판단, 최종 검증) → **외부 Claude Code** 가 직접 수행.
- **저수준 실행 작업**(파일 편집, 보일러플레이트 생성, 단순 lookup, 형식 변환) → **사내 AI Agent** 에 위임.
- 두 모델 사이를 **MCP(Model Context Protocol) 브릿지**가 연결하여, 사용자는 단일 인터페이스(Claude Code)에서 작업하면서 내부적으로는 자동 분배되는 경험을 받는다.

### 2.2 학계·업계 흐름과의 정합
- LLM Cascade / Frugal LLM (Chen et al., 2023) — 저비용 모델 우선 시도, 실패 시 escalate.
- Mixture-of-Agents / Big-Little Routing — 모델 풀에서 작업별 최적 모델 선택.
- 본 제안은 위 두 흐름을 **기업 도입 환경(토큰 할당량 + 사내 자체 모델)** 에 특화한 적용 사례 → 논문성 확보 가능.

### 3.2 라우팅 의사결정 흐름

```
요청 도착
   │
   ▼
[Task Decomposer] ─── 단일 작업으로 분해
   │
   ▼
[Router] ── 난이도·불확실성 점수 계산
   │
   ├─ 고난이도 / 의사결정성  ─────────────▶ 외부 Claude Code 직접 수행
   │
   ├─ 저난이도 / 결정론적     ─────────────▶ 사내 AI Agent 위임
   │                                          │
   │                                          ▼
   │                                    [Verifier]
   │                                          │
   │                                          ├─ 통과 → 결과 반환
   │                                          └─ 실패 → escalate to 외부
   │
   └─ 불확실 (경계 영역)      ─────────────▶ 사내 시도 → 외부 검증 (cascade)
```

### 3.3 주요 컴포넌트 명세

| 컴포넌트 | 역할 | 구현 후보 |
|----------|------|-----------|
| **Planner/Router** | 작업 분해, 라우팅 결정 | 외부 Claude Code 내부에서 system prompt + tool spec으로 구현 |
| **Internal-Agent MCP Bridge** | 사내 모델을 MCP 도구로 노출 | Python 기반 MCP server, 사내 API 어댑터 |
| **Verifier** | 사내 결과의 정합성 검증 | 단위 테스트 실행, 정적 분석, sampling 기반 외부 모델 검증 |
| **Telemetry Store** | 라우팅 결정·성공/실패·비용 로그 | PostgreSQL + 익명화 파이프라인 |
| **Policy Trainer** | 로그 기반 라우팅 정책 개선 | Phase 2 — supervised → contextual bandit → RLHF |

## 4. 핵심 설계 결정 사항

### 4.1 고수준 / 저수준 경계 정의 (가장 중요)
세 가지 안이 있으며, 단계적 진화 전략을 채택한다.

| 단계 | 방식 | 장점 | 단점 |
|------|------|------|------|
| **MVP** | 정적 규칙 (작업 유형 기반) | 즉시 구현, 해석 용이 | 일반화 약함 |
| **v2** | LLM 자체 평가 (난이도·불확실성 self-score) | 적응적, 데이터 불필요 | self-score 신뢰성 검증 필요 |
| **v3** | 학습된 라우터 (사용 로그 기반) | 최적화 가능, 논문성 | 데이터·라벨링 비용 |

### 4.2 오케스트레이션 토폴로지
**채택**: Tool-as-Subagent (사내 모델을 Claude Code의 MCP 도구 또는 subagent로 노출)
- Claude Code 기존 추상화와 호환성 최고.
- 외부 모델이 자연스럽게 위임/검증 루프를 구성할 수 있음.
- 대안(Plan-Execute, Cascade)은 v2/v3에서 부분 도입 검토.

### 4.3 품질 보증
- **1차**: 결정론적 검증(컴파일, 테스트, lint).
- **2차**: 외부 Claude Code의 sampling 검증 (전체의 N%만 검증, 비용 통제).
- **3차**: 사용자 명시 피드백(👍/👎) → reward signal로 라우팅 정책 개선 (RLHF/DPO).

## 9. 부록 — 교육 과정과의 연계

| 교수 | 강의 | 본 과제 활용 지점 |
|------|------|--------------------|
| 황승원 | Agentic LLM (5/12), Code Model (5/11) | Orchestrator·Router 설계, MCP 기반 도구 활용 |
| 여진영 | Prompting / RAG / Distillation (4/27~5/4) | 사내 모델 distillation, RAG로 컨텍스트 보강 |
| 양인순 | RL (PPO/GRPO/DPO) (5/18~22) | 학습 라우터 정책 학습, 사용자 피드백 정렬 |
| 윤성로 | Transformer / 시계열 (4/10~21) | 사용 로그 시계열 분석, 이탈 예측 |

주임교수 후보: **황승원 교수** (1순위) 또는 **여진영 교수** (2순위) — 5월 내 면담 요청 예정.
