## 공통 — 인프라 (vLLM · GPU)

> vLLM, GPU을 모두 인프라 항목으로 넣어두었던 두개 모두 실제 모델을 서빙하는 것까지 고려하고 있는건가요?
>
> 실제로 저희가 GPU 자원을 가지고 있지 않기에 현실적으로 이런 부분은 어려울 것 같다는 생각이 있습니다.
>
> 따라서 실제 모델을 서빙하기 보단 **있는 DS의 LLM 모델을 활용해서 서비스화** 하는 방안이 더 맞을 것 같습니다.

**해석 요약**

- 제안서의 GPU·vLLM 전제는 **사무국 단독으로는 비현실적**.
- 과제 방향은 **자체 모델 서빙·학습 인프라 구축**보다 **DS(데이터사이언스) 조직 보유 LLM API를 활용한 서비스**가 적합.

## 1안 — MemCoder + Code Harness

> 현재 AIDLC에 대해 **메모리사와 협업 과제**를 진행하려고 하고 있습니다. (특정 파일럿 과제에 AIDLC workflow 적용 예정)
>
> 그 단계에서 지금 제안해주신 내용이 **AIDLC와 어떤 연관성**을 가지게 될 지 제가 정확하게 파악이 안되는것 같습니다.

> 실제 사업부의 AIDLC workflow를 적용하기 위해 **현실적인 어려움**들이 많이 있을 것으로 예상됩니다.
>
> 특히 **사내 지식**들을 연계하여 이 workflow를 적용해야하는데, 실제 **claude code**로 workflow를 돌리려면 **사내 정보들을 넣는것에 제약**이 있고, 이런 **문서 데이터를 연계하지 않으면** 실제 요구사항 분석/설계등 aidlc를 각 단계들을 돌려 원하는 수준의 산출물들을 만들어내는데 **한계**가 생깁니다.

> 이런 문제를 풀려면 **사내 도구인 CodeMate**와 같은 사내 coding 도구와 연계하여 AIDLC를 수행하고 **적절한 그 부서내의 암묵지 정보**들을 연계해야할 것 같습니다.

> 이 과정에서 **도움을 주는 AI 도구들이나 방법들**을 진행하는 과제를 진행한다면 좋은 과제가 될 수 있지 않을까합니다.
>
> 다만 아직 이런 문제를 해결하기 위한 **구체적으로 방향이 없고**, 어떤 **서비스나 agent**를 통해 이를 해결해야할지가 **명확하진 않습니다**.
