# 1. 전날 내용 복습

## 1.1 Day 1 핵심

- 강화학습의 기본 프레임워크로 Markov Decision Process(MDP)를 공부했다.
- MDP는 다음 다섯 요소로 구성된다.

| 요소 | 의미 |
|---|---|
| \(S\) | state가 가질 수 있는 값들의 집합 |
| \(A\) | action이 가질 수 있는 값들의 집합 |
| \(P(s' \mid s,a)\) | 현재 state/action에서 다음 state로 갈 transition probability |
| \(R(s,a)\) | 현재 state/action의 reward function |
| \(\gamma\) | 미래 reward를 할인하는 discount factor |

- MDP에서 목표는 expected cumulative reward를 최대화하는 optimal policy를 찾는 것이다.

## 1.2 Bellman equation의 직관

- Bellman equation은 value를 두 부분으로 나눈다.
  - immediate reward
  - long-term benefit
- 좋은 action은 이 두 부분의 합을 최대화하는 action이다.

## 1.3 Optimal value function

- \(V^*(s)\)는 state \(s\)에서 출발했을 때 앞으로 얻을 수 있는 누적 reward 기대값의 최대치이다.
- \(V^*(s)\)를 알면 optimal policy를 구할 수 있다.
- 그래서 MDP에서는 먼저 optimal value function을 계산하고, 그다음 policy를 추출하는 방식이 가능했다.

## 1.4 Bellman operator

- Bellman equation의 우변을 Bellman operator로 정의한다.
- Bellman operator는 contraction 성질을 가진다.
- 임의의 두 value function \(V, V'\)를 operator에 넣으면 거리가 \(\gamma\)배 이하로 줄어든다.
- 이 contraction 성질 때문에 fixed point가 unique하게 존재한다.
- 그 unique fixed point가 \(V^*\)이다.

## 1.5 Value Iteration 복습

Value Iteration은 임의의 \(V_0\)에서 시작해서 Bellman optimality operator를 반복 적용한다.

```text
V_0 -> T V_0 = V_1 -> T V_1 = V_2 -> ... -> V^*
```

- 업데이트는 value space 안에서만 일어난다.
- \(V^*\)에 수렴한 뒤에야 policy space로 넘어가 optimal policy를 뽑는다.
- 각 state마다 action에 대한 maximization이 필요하다.
- 따라서 식은 간단하지만 내부에는 모든 state별 action maximization이 숨어 있다.

# 2. Value Iteration의 관점

## 2.1 Value space만 활용

Value Iteration은 다음 구조이다.

1. value function을 초기화한다.
2. Bellman operator를 반복 적용한다.
3. \(V^*\)에 수렴한다.
4. \(V^*\)에서 optimal policy를 추출한다.

즉, 알고리즘 진행 중에는 policy 정보가 직접 활용되지 않는다.

## 2.2 Optimal policy 추출

\(V^*\)를 얻은 뒤에는 각 state \(s\)에 대해 다음 값을 최대화하는 action을 고른다.

\[
\pi^*(s)=\arg\max_a \left[ R(s,a)+\gamma \sum_{s'}P(s'\mid s,a)V^*(s') \right]
\]

- 모든 state에 대해 action을 assign해야 한다.
- policy는 action sequence가 아니라 state에서 action으로 가는 mapping이다.
- state trajectory가 예상과 다르게 흘러도 현재 state에 맞는 action을 즉시 선택할 수 있다는 장점이 있다.

## 2.3 질문

Value Iteration은 value space 정보만 사용한다.

질문은 다음과 같다.

- value space와 policy space 정보를 동시에 사용할 수는 없을까?
- policy도 업데이트하고 value도 업데이트하는 방식은 없을까?

이 동기에서 Policy Iteration이 나온다.

# 3. Policy Iteration

## 3.1 핵심 아이디어

Policy Iteration은 value space와 policy space 사이를 오가며 업데이트한다.

```text
policy space: pi_0
       ↓ policy evaluation
value space: V^{pi_0}
       ↓ policy improvement
policy space: pi_1
       ↓ policy evaluation
value space: V^{pi_1}
       ↓ policy improvement
policy space: pi_2
...
```

- 먼저 policy를 임의로 초기화한다.
- 현재 policy의 성능을 평가한다.
- 평가된 value function을 바탕으로 policy를 개선한다.
- 이 과정을 반복한다.

## 3.2 두 단계

Policy Iteration은 매 iteration마다 두 단계로 구성된다.

| 단계 | 의미 |
|---|---|
| Policy evaluation | 현재 policy \(\pi_k\)의 value function \(V^{\pi_k}\)를 계산 |
| Policy improvement | \(V^{\pi_k}\)를 이용해 더 좋은 policy \(\pi_{k+1}\)를 생성 |

## 3.3 초기화

- 임의의 deterministic stationary policy로 시작한다.
- stationary policy는 시간에 의존하지 않고 state에만 의존한다.
- deterministic policy는 각 state에서 하나의 action을 정한다.

---

# 4. Policy Evaluation

## 4.1 의미

Policy evaluation은 현재 policy \(\pi_k\)를 사용했을 때 각 state에서 얻을 수 있는 expected cumulative reward를 계산하는 과정이다.

\[
V^{\pi_k}(s)
\]

는 state \(s\)에서 시작해 policy \(\pi_k\)를 따를 때의 value이다.

## 4.2 Bellman operator for fixed policy

policy \(\pi_k\)가 고정되어 있으면 action은 \(\pi_k(s)\)로 정해진다.

\[
(T^{\pi_k}V)(s)
= R(s,\pi_k(s))+\gamma \sum_{s'}P(s'\mid s,\pi_k(s))V(s')
\]

- optimality Bellman operator와 달리 action에 대한 max가 없다.
- policy가 이미 action을 정해주기 때문이다.
- 따라서 policy evaluation의 한 step은 value iteration의 optimality update보다 계산이 가볍다.

## 4.3 Iterative evaluation

임의의 \(V_0\)에서 시작해서 \(T^{\pi_k}\)를 반복 적용하면 \(V^{\pi_k}\)로 수렴한다.

```text
V_0 -> T^{pi_k}V_0 -> T^{pi_k}V_1 -> ... -> V^{pi_k}
```

## 4.4 계산 비용

- policy evaluation 안에도 value iteration 형태의 반복 계산이 들어간다.
- 그래서 겉보기에는 Policy Iteration이 훨씬 무거워 보인다.
- 하지만 policy evaluation은 action maximization을 매번 하지 않는다.
- 이론적으로는 Value Iteration과 같은 complexity class로 분석된다.

---

# 5. Policy Improvement

## 5.1 핵심 식

현재 policy의 value \(V^{\pi_k}\)를 얻었다면, 이를 좋은 long-term value estimate로 믿고 Bellman equation 우변에 넣는다.

\[
\pi_{k+1}(s)
= \arg\max_a \left[ R(s,a)+\gamma \sum_{s'}P(s'\mid s,a)V^{\pi_k}(s') \right]
\]

## 5.2 의미

- immediate reward와 long-term benefit의 합을 최대화하는 action을 고른다.
- 이 작업을 모든 state에 대해 수행한다.
- 이렇게 얻은 policy가 \(\pi_{k+1}\)이다.

## 5.3 action maximization 위치

- Policy evaluation 단계에는 action max가 없다.
- Policy improvement 단계에서 action max가 들어간다.
- 따라서 maximization은 policy를 개선할 때 수행된다.

---

# 6. Policy Improvement가 왜 성능을 높이는가

## 6.1 성능 향상 성질

Policy improvement를 통해 얻은 \(\pi_{k+1}\)은 기존 \(\pi_k\)보다 나쁘지 않다.

\[
V^{\pi_k}(s) \le V^{\pi_{k+1}}(s)
\]

모든 state \(s\)에 대해 위 관계가 성립한다.

## 6.2 수렴 직관

- \(V^{\pi_k}\)는 iteration이 진행될수록 monotonically non-decreasing한다.
- finite state, finite action MDP에서는 value function이 bounded이다.
- bounded monotone sequence는 limit을 갖는다.
- 그 limit이 optimal value이고, 해당 policy는 optimal policy로 수렴한다.

강의에서는 엄밀한 증명은 생략하고 직관만 설명했다.

---

# 7. Value Iteration과 Policy Iteration 비교

## 7.1 Value Iteration

- value space 안에서만 업데이트한다.
- Bellman optimality operator를 반복 적용한다.
- 식은 간단하다.
- 하지만 매 state마다 action maximization이 숨어 있다.

## 7.2 Policy Iteration

- policy evaluation과 policy improvement를 반복한다.
- value space와 policy space 정보를 모두 사용한다.
- 한 iteration은 복잡해 보이지만 iteration 수는 더 적을 수 있다.
- 작은 scale에서는 policy evaluation을 matrix inversion으로 할 수도 있다.
- 큰 scale에서는 iterative evaluation을 쓰는 것이 보통이다.

## 7.3 어느 것이 더 좋은가

- 어떤 문제에서는 Value Iteration이 빠르다.
- 어떤 문제에서는 Policy Iteration이 빠르다.
- 둘 중 하나가 항상 우월하다고 말하기 어렵다.
- 실제 문제에서는 둘 다 돌려보거나 문제 구조에 따라 선택한다.

# 8. MDP 알고리즘의 한계

## 8.1 모델 정보 필요

Value Iteration과 Policy Iteration은 모두 transition probability를 사용한다.

- Bellman operator 계산에 \(P(s'\mid s,a)\)가 필요하다.
- policy evaluation에도 \(P\)가 필요하다.
- policy improvement에도 \(P\)가 필요하다.

## 8.2 강화학습으로 넘어가는 이유

강화학습은 보통 transition probability를 모르는 상황을 다룬다.

- environment model을 모른다.
- reward function도 정확히 모를 수 있다.
- 단지 interaction을 통해 sample을 얻을 수 있다.

따라서 모델을 아는 MDP 알고리즘을 그대로 사용할 수 없다.

## 8.3 두 확장 방향

MDP 알고리즘은 RL 알고리즘의 아이디어를 제공한다.

| MDP 알고리즘 | RL 확장 |
|---|---|
| Value Iteration | Q-learning, DQN |
| Policy Iteration | Actor-Critic, DDPG, PPO 계열 |

오늘은 Value Iteration의 RL 확장인 Q-learning을 중심으로 공부한다.

# 9. Q Function

## 9.1 왜 V가 아니라 Q인가

MDP에서는 \(V^*(s)\)를 계산하면 model을 이용해 policy를 추출할 수 있었다.

하지만 RL에서는 model을 모른다.

\[
\pi^*(s)=\arg\max_a \left[ R(s,a)+\gamma \sum_{s'}P(s'\mid s,a)V^*(s') \right]
\]

위 식에는 \(P(s'\mid s,a)\)가 필요하다.

따라서 \(V^*\)만 학습해도 policy 추출 단계에서 다시 model이 필요하다.

## 9.2 Q function 정의

\[
Q^*(s,a)
\]

는 현재 state \(s\)에서 현재 action \(a\)까지 고정했을 때, 그 이후 최적으로 행동하여 얻을 수 있는 expected cumulative reward의 최대치이다.

## 9.3 V와 Q의 차이

| 함수 | 고정된 것 | 의미 |
|---|---|---|
| \(V^*(s)\) | 현재 state만 고정 | 현재 action을 선택할 자유도가 있음 |
| \(Q^*(s,a)\) | 현재 state와 action 모두 고정 | 현재 action이 이미 정해져 있음 |

## 9.4 V와 Q의 관계

\[
V^*(s)=\max_a Q^*(s,a)
\]

- \(V^*(s)\)는 현재 action을 자유롭게 고를 수 있다.
- 따라서 모든 action의 \(Q^*(s,a)\) 중 가장 큰 값을 고르면 된다.

## 9.5 Q에서 policy 추출

\[
\pi^*(s)=\arg\max_a Q^*(s,a)
\]

- \(Q^*\)만 알면 model 없이 optimal policy를 바로 얻을 수 있다.
- 그래서 RL에서는 \(V^*\)보다 \(Q^*\)를 학습하는 것이 유리하다.

# 10. Q-learning의 목표

## 10.1 한 문장 정의

Q-learning은 **optimal Q function \(Q^*(s,a)\)를 model-free 방식으로 학습하는 알고리즘**이다.

## 10.2 model-free의 의미

- transition probability를 직접 사용하지 않는다.
- reward function의 명시적 형태를 몰라도 된다.
- interaction sample을 이용해 Q를 업데이트한다.

## 10.3 강화학습 데이터 단위

Q-learning에서 하나의 sample은 보통 다음 네 요소로 구성된다.

\[
(s_t, a_t, r_t, s_{t+1})
\]

| 요소 | 의미 |
|---|---|
| \(s_t\) | 현재 state |
| \(a_t\) | 현재 action |
| \(r_t\) | 현재 reward |
| \(s_{t+1}\) | 다음 state |

## 10.4 이 sample이 주는 정보

sample은 두 가지 정보를 준다.

1. \((s_t,a_t,s_{t+1})\): transition probability에 대한 observation
2. \((s_t,a_t,r_t)\): reward function에 대한 observation

즉 model을 직접 알지는 못하지만 sample이 model의 일부 정보를 간접적으로 제공한다.

---

# 11. Q function의 Bellman equation

## 11.1 두 부분으로 분해

Q function도 Bellman equation으로 분해할 수 있다.

\[
Q^*(s,a)
= R(s,a)+\gamma \sum_{s'}P(s'\mid s,a)V^*(s')
\]

여기서

\[
V^*(s')=\max_{a'}Q^*(s',a')
\]

이므로

\[
Q^*(s,a)
= R(s,a)+\gamma \sum_{s'}P(s'\mid s,a)\max_{a'}Q^*(s',a')
\]

## 11.2 직관

- 첫 항은 현재 state/action에서 얻는 immediate reward이다.
- 두 번째 항은 다음 state 이후의 optimal long-term benefit이다.
- 다음 state에 대한 expectation에는 transition probability가 들어간다.

## 11.3 Q Bellman operator

Bellman equation의 우변을 Q function에 대한 Bellman operator로 정의할 수 있다.

- 이 operator도 monotonicity와 contraction 성질을 가진다.
- model을 알고 있다면 Q function도 value iteration 방식으로 계산할 수 있다.
- 하지만 Q를 모든 state-action pair에 대해 업데이트해야 해서 계산량이 크다.

# 12. Model-free Q-learning update

## 12.1 single-sample estimate

모델을 모르면 다음 expectation을 정확히 계산할 수 없다.

\[
\sum_{s'}P(s'\mid s,a)\max_{a'}Q(s',a')
\]

대신 sample \(s_{t+1}\) 하나를 사용해 single-sample estimate를 만든다.

\[
r_t+\gamma \max_{a'}Q(s_{t+1},a')
\]

## 12.2 Q-learning update

\[
Q_{k+1}(s_t,a_t)
= (1-\alpha)Q_k(s_t,a_t)
+ \alpha \left[
r_t+\gamma\max_{a'}Q_k(s_{t+1},a')
\right]
\]

동일하게 쓰면 다음과 같다.

\[
Q_{k+1}(s_t,a_t)
= Q_k(s_t,a_t)
+ \alpha \left[
r_t+\gamma\max_{a'}Q_k(s_{t+1},a')
-Q_k(s_t,a_t)
\right]
\]

## 12.3 old estimate와 new estimate

| 부분 | 의미 |
|---|---|
| \(Q_k(s_t,a_t)\) | old estimate |
| \(r_t+\gamma\max_{a'}Q_k(s_{t+1},a')\) | new estimate |
| \(\alpha\) | step size / learning rate |

## 12.4 local update

- 업데이트는 현재 sample의 \((s_t,a_t)\) 위치에서만 일어난다.
- Q table 전체가 한 번에 업데이트되지 않는다.
- 그래서 tabular Q-learning은 sample이 많이 필요하다.

## 12.5 수렴

- stochastic approximation 이론으로 특정 조건하에 수렴성이 보장된다.
- 실제로는 완벽한 optimal solution까지 기다리지 않는다.
- 충분히 좋은 Q function을 얻으면 중간에 멈춘다.

# 13. Q-learning의 장점

## 13.1 간단함

- update 식이 단순하다.
- 구현이 쉽다.
- model-free이다.

## 13.2 \(Q^*\)에서 policy를 바로 얻음

학습이 끝난 뒤에는 다음처럼 greedy policy를 만들 수 있다.

\[
\pi(s)=\arg\max_a Q(s,a)
\]

이 과정에는 transition probability가 필요하지 않다.

## 13.3 off-policy

Q-learning은 off-policy 알고리즘이다.

- sample을 만들 때 사용한 policy와 학습 목표 policy가 달라도 된다.
- 어떤 policy로 sample을 만들어도 원칙적으로 Q-learning update를 사용할 수 있다.

## 13.4 sample reuse 가능성

off-policy 특성 때문에 과거 sample을 재사용할 수 있다.

- sample을 한 번 만들고 버리지 않아도 된다.
- DQN의 Experience Replay가 가능해지는 기반이다.

---

# 14. Q-learning의 한계

## 14.1 sample efficiency 문제

- local update만 일어난다.
- state-action pair가 많으면 많은 trial이 필요하다.
- Q table이 커질수록 학습이 느려진다.

## 14.2 scalability 문제

Q table의 크기는 대략 다음에 비례한다.

\[
|S| \times |A|
\]

- state space가 크면 table이 커진다.
- action space가 크면 table이 커진다.
- continuous state/action에서는 table 방식이 직접 적용되기 어렵다.

## 14.3 sample correlation

- environment와 순차적으로 interaction하며 sample을 얻는다.
- 순차 sample 사이에는 correlation이 생긴다.
- function approximation과 결합할 때 regression bias나 instability의 원인이 된다.

## 14.4 overestimation

- update에 max operator가 들어간다.
- noisy Q estimate에 max를 취하면 과대추정이 생길 수 있다.
- 이 문제는 오후 강의에서 Double DQN으로 다룬다.

## 14.5 exploration 문제

Q-learning update 자체에는 미지의 영역을 어떻게 탐색할지에 대한 구체적 지침이 없다.

- 어느 action을 시도해야 하는가?
- 안 가본 state를 어떻게 방문할 것인가?
- 현재 estimate를 믿고 exploit할지, 새 정보를 얻기 위해 explore할지 결정해야 한다.

# 15. Epsilon-Greedy Exploration

## 15.1 기본 아이디어

epsilon-greedy는 가장 간단한 exploration 방법이다.

일반적으로 다음처럼 사용한다.

- 확률 \(1-\epsilon\): greedy action 선택
- 확률 \(\epsilon\): random action 선택

강의 중 표현은 문맥에 따라 \(\epsilon\)과 \(1-\epsilon\)이 바뀌어 언급되었지만, 핵심은 greedy 선택과 random exploration을 확률적으로 섞는 것이다.

## 15.2 greedy action

현재 Q estimate를 믿고 가장 Q값이 큰 action을 고른다.

\[
a=\arg\max_a Q(s,a)
\]

## 15.3 random action

무작위 action을 골라 아직 충분히 탐색하지 않은 영역을 방문할 가능성을 만든다.

## 15.4 epsilon scheduling

- 초반에는 \(\epsilon\)을 크게 둔다.
  - Q estimate가 부정확하므로 exploration을 많이 한다.
- 후반에는 \(\epsilon\)을 줄인다.
  - 학습된 Q를 더 믿고 exploitation을 많이 한다.

최적 epsilon schedule은 미리 알 수 없으므로 tuning이 필요하다.

---

# 16. Scalability를 위한 Approximate Q-Learning

## 16.1 tabular 방식의 문제

Tabular Q-learning은 각 \((s,a)\) 위치만 업데이트한다.

- high-dimensional state에서는 table이 너무 커진다.
- continuous state에서는 table 자체가 불가능하다.
- sample이 매우 많이 필요하다.

## 16.2 핵심 아이디어

Q table을 직접 저장하지 않고 parameterized function으로 근사한다.

\[
Q_\theta(s,a)\approx Q^*(s,a)
\]

- \(\theta\)는 weight vector이다.
- sample 하나에서 얻은 정보로 \(\theta\)를 업데이트하면 Q function 전체가 global하게 변한다.
- local update를 global update로 바꾸는 효과가 있다.

## 16.3 정확도와 안정성 trade-off

- 정확한 tabular update를 포기한다.
- 대신 더 큰 문제를 다룰 수 있다.
- function approximation은 instability를 유발할 수 있다.

---

# 17. Approximation의 두 방향

## 17.1 Value space approximation

value function 또는 Q function을 parameterize한다.

\[
V_\theta(s)\approx V^*(s)
\]

\[
Q_\theta(s,a)\approx Q^*(s,a)
\]

RL에서는 model-free policy extraction을 위해 \(Q_\theta\)를 선호한다.

## 17.2 Policy space approximation

policy 자체를 parameterize한다.

\[
\pi_\theta(a\mid s)
\]

또는 deterministic policy:

\[
\mu_\theta(s)
\]

이 방향은 다음 날 policy gradient와 actor-critic에서 다룬다.

## 17.3 두 방향의 차이

- value approximation은 value/Q를 잘 맞춘 뒤 policy를 추출한다.
- policy approximation은 policy 자체를 직접 학습한다.
- 하나가 쉽다고 다른 하나도 쉬운 것은 아니다.

---

# 18. Feature-Based Function Approximation

## 18.1 과거 방식

deep learning이 보편화되기 전에는 feature를 직접 설계했다.

\[
V_\theta(s)=\sum_l \theta_l \phi_l(s)
\]

또는

\[
Q_\theta(s,a)=\sum_l \theta_l \phi_l(s,a)
\]

## 18.2 Piecewise constant approximation

- state space를 여러 구역으로 나눈다.
- 현재 state가 속한 구역의 indicator feature를 1로 둔다.
- 각 구역에서 value가 constant하게 유지된다.

한계:

- state dimension이 커지면 필요한 partition 수가 급증한다.
- 고차원에서는 정확도를 유지하기 어렵다.

## 18.3 Polynomial basis

- Bellman이 1950년대에 제안한 방식 중 하나이다.
- 0차, 1차, 2차, 고차 polynomial term을 basis로 사용한다.

한계:

- 차수를 높이면 overfitting이 쉽게 발생한다.
- 학습 안정성이 떨어질 수 있다.
- 복잡한 문제에는 잘 맞지 않는다.

## 18.4 Radial Basis Function

- 특정 state 주변을 anchor로 두고 radial basis를 만든다.
- Gaussian basis가 대표적이다.
- 비교적 안정성이 좋아 지금도 간단한 문제에서 사용할 수 있다.

## 18.5 Neural network로 전환

- feature engineering은 문제마다 어렵다.
- deep neural network는 feature learning을 자동으로 수행한다.
- 컴퓨터 비전에서 hand-crafted feature에서 CNN으로 넘어간 흐름과 유사하다.

---

# 19. Model-Free Policy Iteration 관점

## 19.1 Q를 이용한 policy evaluation

policy \(\pi\)가 고정되어 있을 때 \(Q^\pi(s,a)\)를 평가할 수 있다.

- 현재 state/action에서 시작한다.
- 다음 시점부터 policy \(\pi\)를 따른다.
- sample trajectory를 여러 개 생성한다.
- cumulative reward 평균을 target으로 삼는다.

## 19.2 regression 문제로 해석

여러 시작점 \((s_i,a_i)\)에 대해 target \(y_i\)를 만들면 다음 regression 문제를 풀 수 있다.

\[
\min_\theta \sum_i \left(Q_\theta(s_i,a_i)-y_i\right)^2
\]

## 19.3 한계

- architecture 선택 문제가 있다.
- exploration이 부족하다.
- convergence와 stability 문제가 있다.
- sample trajectory를 많이 만들어야 하므로 비용이 크다.

---

# 20. Policy Evaluation을 Q-learning 방식으로 하기

## 20.1 fixed policy에 대한 update

policy \(\pi\)가 고정된 경우 다음처럼 update할 수 있다.

\[
Q(s_t,a_t)
\leftarrow
(1-\alpha)Q(s_t,a_t)
+\alpha\left[
r_t+\gamma Q(s_{t+1},\pi(s_{t+1}))
\right]
\]

## 20.2 Q-learning과의 차이

- Q-learning은 다음 state에서 \(\max_{a'}Q(s_{t+1},a')\)를 사용한다.
- fixed policy evaluation은 \(Q(s_{t+1},\pi(s_{t+1}))\)를 사용한다.
- 즉 max가 아니라 고정 policy가 선택하는 action을 넣는다.

## 20.3 여전히 local update

function approximation이 없으면 이 update도 \((s_t,a_t)\) 위치만 바꾼다.

- scalability 문제가 남는다.
- 그래서 Q function을 parameterize해야 한다.

---

# 21. Optimistic Policy Iteration

## 21.1 inexact policy evaluation

정확한 policy evaluation을 끝까지 수행하지 않고, approximate Q function을 조금씩 업데이트한다.

- sample을 얻는다.
- 현재 Q estimate로 greedy action을 고른다.
- target을 만든다.
- stochastic gradient step으로 parameter를 업데이트한다.

## 21.2 optimistic이라고 부르는 이유

policy evaluation이 정확하지 않지만, 그 inexact evaluation을 믿고 policy improvement를 진행하기 때문이다.

## 21.3 epsilon-greedy와 결합 가능

greedy action만 선택하면 exploration이 부족하다.

- 일정 확률로 random action을 선택한다.
- 나머지 확률로 Q estimate에 따른 greedy action을 선택한다.

---

# 22. SARSA

## 22.1 이름의 의미

SARSA는 update에 다음 다섯 요소가 쓰이기 때문에 붙은 이름이다.

\[
(s_t, a_t, r_t, s_{t+1}, a_{t+1})
\]

State, Action, Reward, State, Action의 앞 글자이다.

## 22.2 target

SARSA는 다음 target을 사용한다.

\[
r_t+\gamma Q(s_{t+1},a_{t+1})
\]

## 22.3 특징

- 구현이 간단하다.
- 과거에는 빠른 테스트에 쓰였다.
- function approximation과 결합하면 안정성이 떨어질 수 있다.
- 현재는 주의해서 사용해야 한다.

---

# 23. Approximate Q-Learning

## 23.1 \(Q^*\) 직접 근사

Approximate Q-learning은 policy별 \(Q^\pi\)가 아니라 optimal Q function 자체를 parameterize한다.

\[
Q_\phi(s,a)\approx Q^*(s,a)
\]

## 23.2 target 설정

sample \((s_i,a_i,r_i,s_i')\)에 대해 Bellman equation 우변의 single-sample estimate를 target으로 둔다.

\[
y_i
=r_i+\gamma\max_{a'}Q_\phi(s_i',a')
\]

## 23.3 regression 문제

\[
\min_\phi \sum_i \left(Q_\phi(s_i,a_i)-y_i\right)^2
\]

## 23.4 stochastic gradient 버전

sample 하나 또는 mini-batch를 이용해 gradient step을 밟는다.

- single sample gradient도 가능하다.
- mini-batch gradient도 가능하다.
- off-policy 특성이 유지된다.

---

# 24. Approximate Q-Learning의 남은 문제

## 24.1 function approximation architecture

- 어떤 feature를 쓸 것인가?
- neural network 구조를 어떻게 만들 것인가?

## 24.2 sample correlation

- sample은 순차적으로 들어온다.
- 같은 environment dynamics에서 나온 인접 sample들은 correlated되어 있다.
- regression에서 독립 sample 가정을 깨뜨릴 수 있다.

## 24.3 instability

- target이 Q network 자신에 의해 만들어진다.
- Q estimate가 바뀌면 target도 바뀐다.
- 학습이 unstable하거나 발산할 수 있다.

## 24.4 overestimation

- max operator가 noisy estimate에서 큰 값을 고르는 경향이 있다.
- Q value가 실제보다 부풀려질 수 있다.

## 24.5 exploration

- simple Q-learning update만으로는 exploration 전략이 충분하지 않다.

이 문제들을 해결하며 등장한 것이 DQN이다.

---

# 25. DQN 도입

## 25.1 DQN의 의미

DQN은 Deep Q-Network이다.

- Q-learning과 deep learning을 결합한 deep RL 알고리즘이다.
- Q function을 neural network로 represent한다.
- high-dimensional sensor input, pixel input 등을 처리할 수 있다.

## 25.2 왜 중요한가

- deep RL의 초기 성공 사례 중 하나이다.
- 이후 많은 RL 알고리즘의 기반이 되었다.
- 다음 날 배울 DDPG의 critic 부분도 DQN의 아이디어와 연결된다.

## 25.3 DQN이 해결하려는 문제

| 문제 | DQN의 대응 |
|---|---|
| feature engineering 어려움 | neural network로 feature learning |
| sample correlation | Experience Replay |
| moving target | Target Network |
| large state space | function approximation |

오후 강의에서 DQN의 세부 구조를 다룬다.

---

# 26. DQN 성공 사례

## 26.1 Atari Breakout

- agent는 화면 pixel만 본다.
- 공, 벽돌, paddle에 대한 사전 지식이 없다.
- reward는 game score이다.
- 처음에는 공을 자주 놓친다.
- 학습이 진행되면 공을 거의 놓치지 않는다.
- 더 학습하면 한쪽에 구멍을 뚫고 공을 위로 올려 효율적으로 점수를 얻는 전략을 배운다.

핵심:

- domain knowledge 없이 pixel input에서 end-to-end로 전략을 학습했다.

## 26.2 Robot grasping

- 카메라 pixel, gripper state, joint 정보 등을 state로 사용한다.
- 여러 robot이 동시에 sample을 모은다.
- 다양한 물체를 잡는 task를 학습한다.
- 학습 때 보지 않은 object도 어느 정도 grasp할 수 있다.

## 26.3 disturbance 대응

- 사람이 물체를 건드려 위치가 바뀌면 camera pixel state가 달라진다.
- policy는 현재 state에 맞는 action을 다시 선택한다.
- policy 형태로 학습했기 때문에 disturbance에도 대응할 수 있다.

---

# 27. 오전 핵심 정리

## 27.1 알고리즘 흐름

1. Value Iteration은 value space만 사용한다.
2. Policy Iteration은 value space와 policy space를 번갈아 사용한다.
3. 둘 다 MDP model이 필요하다.
4. RL에서는 model을 모르므로 sample 기반 방법이 필요하다.
5. \(V^*\)만 학습하면 policy 추출에 model이 필요하다.
6. \(Q^*\)를 학습하면 model 없이 greedy policy를 얻을 수 있다.
7. Q-learning은 Bellman equation의 single-sample estimate로 \(Q^*\)를 학습한다.
8. Tabular Q-learning은 간단하지만 scalability가 낮다.
9. Approximate Q-learning은 Q function을 parameterize해 global update를 가능하게 한다.
10. function approximation의 instability를 해결하기 위해 DQN이 등장한다.

## 27.2 꼭 기억할 식

### Q-learning update

\[
Q(s_t,a_t)
\leftarrow
Q(s_t,a_t)
+\alpha
\left[
r_t+\gamma\max_{a'}Q(s_{t+1},a')
-Q(s_t,a_t)
\right]
\]

### Q에서 policy 추출

\[
\pi(s)=\arg\max_a Q(s,a)
\]

### V와 Q 관계

\[
V^*(s)=\max_a Q^*(s,a)
\]

## 27.3 시험 대비 포인트

- Policy Iteration의 두 단계: policy evaluation, policy improvement.
- Value Iteration과 Policy Iteration의 차이.
- 왜 model-free RL에서 \(V\)보다 \(Q\)를 학습하는지.
- Q-learning update 식과 각 항의 의미.
- \((s,a,r,s')\) sample이 transition/reward 정보를 어떻게 제공하는지.
- off-policy의 의미와 sample reuse 가능성.
- epsilon-greedy의 exploration/exploitation 역할.
- tabular Q-learning의 scalability 한계.
- approximate Q-learning에서 function approximator를 쓰는 이유.
- DQN이 필요한 이유: sample correlation, moving target, neural function approximation.
