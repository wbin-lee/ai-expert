# 1. DQN으로 넘어가기 전 문제의식

## 1.1 그냥 Q function을 neural network로 근사하면 왜 안 되는가

Q function을 neural network로 근사하는 시도는 DQN 이전에도 있었다.

- 예: Neural Fitted Q-Iteration.
- 하지만 성능이 만족스럽지 않았다.

주요 문제는 두 가지였다.

1. sample 간 correlation 문제를 충분히 해결하지 못했다.
2. target value가 학습 중 계속 바뀌어 regression target이 불안정했다.

## 1.2 sample correlation

RL sample은 environment와 순차적으로 interaction하며 얻는다.

```text
s_t, a_t, r_t, s_{t+1}
s_{t+1}, a_{t+1}, r_{t+1}, s_{t+2}
...
```

- 인접 sample은 서로 강하게 correlated되어 있다.
- regression은 보통 sample이 독립적이라는 가정에 기대는데, 이 가정이 깨진다.
- 그 결과 Q regression에 bias와 instability가 생긴다.

## 1.3 moving target 문제

DQN류 update의 target은 다음과 같은 형태이다.

\[
y=r+\gamma\max_{a'}Q(s',a')
\]

여기서 \(Q\) 자체가 학습 중 바뀐다.

- network parameter가 바뀌면 target도 바뀐다.
- 내가 맞추려는 정답이 계속 움직인다.
- supervised learning의 fixed label과 다르다.

# 2. DQN의 두 핵심 아이디어

## 2.1 Experience Replay

Experience Replay는 interaction sample을 replay buffer에 저장한 뒤, buffer에서 mini-batch를 random하게 다시 sample하는 방법이다.

```text
environment interaction
      ↓
(s, a, r, s') 저장
      ↓
replay buffer
      ↓
random mini-batch resampling
      ↓
Q network update
```

### 효과

- 순차적으로 들어온 sample을 섞는다.
- mini-batch가 시간 순서대로 뽑히지 않는다.
- sample 간 correlation을 줄인다.
- sample을 재사용하므로 sample efficiency가 좋아진다.

## 2.2 Target Network

DQN은 두 개의 Q network를 사용한다.

| network | 역할 |
|---|---|
| online Q network \(Q_\phi\) | 실제 학습되는 Q function |
| target Q network \(Q_{\phi^-}\) | target value 계산에 사용 |

## 2.3 target network의 목적

- target 계산에 현재 network를 바로 쓰지 않는다.
- target network는 천천히 update한다.
- target이 잠시 고정된 것처럼 만들어 regression을 안정화한다.

즉, moving target 문제를 완화한다.

# 3. DQN 알고리즘 구조

## 3.1 초기화

DQN은 두 parameter vector로 시작한다.

\[
\phi
\]

\[
\phi^-
\]

- \(\phi\): online Q network parameter.
- \(\phi^-\): target Q network parameter.
- 처음에는 두 network를 synchronize한다.

## 3.2 sample 수집

임의의 policy 또는 epsilon-greedy policy로 environment와 interaction한다.

\[
(s_t,a_t,r_t,s_{t+1})
\]

을 얻고 replay buffer에 저장한다.

## 3.3 mini-batch resampling

replay buffer에서 random mini-batch를 뽑는다.

- 바로 직전 sample만 쓰지 않는다.
- 과거 sample도 재사용한다.
- off-policy 특성이 있기 때문에 가능하다.

## 3.4 target 계산

각 mini-batch sample에 대해 다음 target을 계산한다.

\[
y_i
=r_i+\gamma\max_{a'}Q_{\phi^-}(s_i',a')
\]

중요한 점:

- max 안의 Q는 online network가 아니라 target network이다.
- target network를 사용하여 target drift를 줄인다.

## 3.5 Q regression

online Q network가 target에 가까워지도록 regression 문제를 푼다.

\[
\min_\phi \sum_i
\left(
Q_\phi(s_i,a_i)-y_i
\right)^2
\]

실제로는 mini-batch gradient, Adam, AdamW 같은 optimizer를 사용할 수 있다.

## 3.6 target network synchronization

일정 step마다 다음처럼 synchronize한다.

\[
\phi^- \leftarrow \phi
\]

또는 Polyak averaging을 사용한다.

# 4. Polyak Averaging

## 4.1 hard update의 대안

target network를 \(N\) step마다 한 번씩 복사하면 update 순간에 target이 튈 수 있다.

이를 줄이기 위해 매 step 조금씩 섞는 방법을 쓴다.

## 4.2 update 식

\[
\phi^- \leftarrow (1-\tau)\phi^-+\tau\phi
\]

- \(\tau\)는 작은 값이다.
- 예: \(\tau=0.005\).
- target network가 online network를 천천히 따라간다.

## 4.3 의미

- target이 급격하게 변하지 않는다.
- regression target이 더 smooth하게 변한다.
- 학습 안정성을 높인다.

# 5. Experience Replay 자세히 보기

## 5.1 replay buffer

replay buffer는 transition sample의 저장소이다.

\[
(s,a,r,s',done)
\]

을 저장한다.

## 5.2 buffer가 꽉 차면

- 오래된 sample을 버리고 새 sample을 넣을 수 있다.
- 또는 정한 규칙에 따라 sample을 교체한다.

## 5.3 random mini-batch

학습할 때 buffer에서 random하게 mini-batch를 뽑는다.

### 장점

1. sample correlation 감소.
2. 과거 경험 재사용.
3. 더 적은 environment interaction으로 학습 가능.
4. off-policy Q-learning과 잘 맞는다.

# 6. DQN의 장점

## 6.1 간단한 deep RL 구조

- Q-learning update를 neural network regression으로 바꾼다.
- Experience Replay와 Target Network만 추가해도 큰 안정성 개선이 있다.
- 이후 deep RL 알고리즘들의 기본 구성요소가 되었다.

## 6.2 sample efficiency

- replay buffer를 통해 sample을 재사용한다.
- sample을 한 번 쓰고 버리지 않는다.
- Q-learning의 off-policy 특성이 이를 가능하게 한다.

## 6.3 high-dimensional input 처리

- pixel input 같은 비정형 state도 neural network가 처리할 수 있다.
- Atari 같은 게임에서 성공을 보였다.

# 7. DQN의 한계

## 7.1 continuous action space에 약함

DQN target에는 다음 max가 들어간다.

\[
\max_{a'}Q(s',a')
\]

discrete action이면 모든 action의 Q값을 비교하면 된다.

continuous action이면:

- 가능한 action이 무한히 많다.
- max를 계산하려면 별도의 optimization이 필요하다.
- Q function이 action에 대해 concave라는 보장이 없다.
- gradient ascent를 해도 global maximum을 찾는다는 보장이 없다.

따라서 continuous control에서는 DQN 계열을 직접 쓰기 어렵다.

## 7.2 exploration 문제

DQN 자체가 고성능 exploration 전략을 제공하지 않는다.

- 보통 epsilon-greedy를 쓴다.
- 간단하지만 optimal exploration은 아니다.
- Montezuma's Revenge처럼 sparse reward와 복잡한 exploration이 필요한 게임에서는 성능이 낮다.

## 7.3 학습 시간이 길다

RL은 supervised learning보다 시간이 오래 걸리는 경향이 있다.

이유:

- 매 update가 작은 regression 문제를 포함한다.
- environment interaction이 필요하다.
- sample이 correlated되어 있고 noisy하다.
- Q target 자체도 학습 중 변한다.

## 7.4 gradient instability

- gradient가 튈 수 있다.
- clipping이나 Huber loss를 사용할 수 있다.

# 8. Practical Tips for DQN

## 8.1 epsilon scheduling

보통 epsilon을 크게 시작해서 점점 줄인다.

- 초반: Q estimate가 부정확하므로 exploration을 많이 한다.
- 후반: Q estimate가 개선되므로 exploitation을 늘린다.

최적 schedule은 task마다 다르므로 tuning해야 한다.

## 8.2 replay buffer size

- practical한 문제에서는 충분히 큰 replay buffer가 필요하다.
- 이미지 input처럼 data diversity가 큰 경우 buffer가 작으면 안정성이 떨어질 수 있다.
- 단순 실습 문제에서는 작은 buffer로도 동작할 수 있다.

## 8.3 쉬운 문제부터 테스트

강화학습은 debug가 어렵다.

- 처음부터 복잡한 문제에 적용하면 학습 실패 원인을 찾기 어렵다.
- CartPole 같은 쉬운 환경에서 먼저 확인하는 것이 좋다.
- 이후 복잡한 task로 확장한다.

## 8.4 project에서의 병목

강화학습 프로젝트에서 가장 큰 병목은 학습 시간이다.

- 학습이 오래 걸리는 것은 알고리즘 특성상 자연스럽다.
- debugging과 hyperparameter tuning까지 시간이 많이 든다.

# 9. DQN의 Overestimation 문제

## 9.1 관찰

DQN은 Q-value를 실제보다 크게 estimate하는 경향이 있다.

- return의 trend는 맞출 수 있다.
- 하지만 scale이 부풀려질 수 있다.
- overestimated Q value는 exploration과 안정성에 악영향을 준다.

## 9.2 max operator가 문제

DQN target:

\[
y=r+\gamma \max_{a'}Q_{\phi^-}(s',a')
\]

noisy estimate에 max를 취하면 높은 noise를 가진 action이 선택될 가능성이 있다.

## 9.3 수학적 직관

random variable \(X_1, X_2\)에 대해 일반적으로 다음이 성립한다.

\[
E[\max(X_1,X_2)] \ge \max(E[X_1],E[X_2])
\]

즉, max를 먼저 취하고 expectation을 취하면 과대추정 방향으로 bias가 생긴다.

## 9.4 action selection과 value estimation의 coupling

DQN target은 같은 target network로 다음 두 작업을 모두 한다.

1. 어떤 action이 좋은지 선택.
2. 그 action의 value를 평가.

action selection noise와 value estimation noise가 강하게 correlated될 수 있다.

# 10. Double Q-Learning

## 10.1 핵심 아이디어

action selection과 value estimation에 서로 다른 Q network를 사용한다.

- A network로 action을 고르고 B network로 value를 평가한다.
- 또는 B network로 action을 고르고 A network로 value를 평가한다.

## 10.2 원래 Double Q-learning 구조

두 Q function을 동시에 유지한다.

- \(Q_A\) update 시:
  - action selection은 \(Q_A\)
  - value estimation은 \(Q_B\)
- \(Q_B\) update 시:
  - action selection은 \(Q_B\)
  - value estimation은 \(Q_A\)

## 10.3 목적

- max operator가 만드는 overestimation bias를 줄인다.
- action selection noise와 value estimation noise의 correlation을 줄인다.

---

# 11. Double DQN

## 11.1 DQN에는 이미 두 network가 있다

DQN은 이미 online network와 target network를 사용한다.

| network | Double DQN에서의 역할 |
|---|---|
| online network \(Q_\phi\) | action selection |
| target network \(Q_{\phi^-}\) | value estimation |

## 11.2 DQN target

일반 DQN:

\[
y=r+\gamma \max_{a'}Q_{\phi^-}(s',a')
\]

action selection과 value estimation 모두 target network가 한다.

## 11.3 Double DQN target

Double DQN:

\[
a^*=\arg\max_{a'}Q_{\phi}(s',a')
\]

\[
y=r+\gamma Q_{\phi^-}(s',a^*)
\]

즉:

1. online network로 best action을 고른다.
2. target network로 그 action의 value를 평가한다.

## 11.4 구현 차이

DQN에서 Double DQN으로 바꾸는 핵심은 target 계산 한 줄이다.

- 일반 DQN: target network에서 max까지 한 번에 계산.
- Double DQN: online network로 argmax, target network로 value lookup.

## 11.5 효과

- overestimation이 줄어든다.
- 학습 variance가 줄어든다.
- Atari 실험에서 많은 게임에서 DQN보다 안정적이고 높은 성능을 보였다.

---

# 12. DQN과 Double DQN 비교

## 12.1 공통점

- replay buffer를 사용한다.
- target network를 사용한다.
- Q regression을 한다.
- discrete action에 적합하다.
- off-policy 특성이 있다.

## 12.2 차이점

| 항목 | DQN | Double DQN |
|---|---|---|
| action selection | target network | online network |
| value estimation | target network | target network |
| overestimation | 더 큼 | 줄어듦 |
| 구현 난이도 | 기본 | target 한 줄 수정 |

## 12.3 Double DQN의 남은 한계

- continuous action 문제는 여전히 어렵다.
- exploration 문제도 그대로 남는다.
- value-only method의 한계가 있다.
- 복잡한 task에서는 policy 정보를 직접 활용하는 actor-critic 계열이 필요해진다.

# 13. Value-Based Method에서 Actor-Critic으로

## 13.1 DQN 계열의 위치

DQN과 Double DQN은 value-based method이다.

- Q function을 학습한다.
- Q에서 greedy policy를 추출한다.
- policy 자체를 직접 parameterize하지 않는다.

## 13.2 다음 단계

복잡한 문제에서는 value만으로는 한계가 있다.

- continuous control.
- high-dimensional action.
- policy 정보의 직접 활용.

다음 강의에서는 policy gradient와 actor-critic을 배운다.

## 13.3 DDPG와 연결

DDPG는 actor-critic 계열이다.

- critic은 DQN의 Q network 아이디어와 연결된다.
- actor는 policy를 직접 학습한다.

따라서 DQN 실습 이해가 다음날 DDPG 이해에 중요하다.

# 14. 실습 1 — Policy Iteration 마무리

## 14.1 전날 실습 복습

전날에는 다음 알고리즘을 다뤘다.

- backward induction.
- value iteration.
- policy iteration 일부.

오후 실습은 policy iteration을 먼저 마무리했다.

## 14.2 Value Iteration 코드 핵심

Bellman optimality operator \(T\)를 \(V\)에 반복 적용한다.

- 충분히 반복하면 \(V^*\)로 수렴한다.
- 그다음 \(Q^*\)를 계산한다.
- `np.argmax`로 optimal policy를 뽑는다.

주의:

- 이 과정은 reward function과 transition probability를 알고 있어야 가능하다.
- RL에서는 이 정보를 모른다.

## 14.3 Policy Evaluation 구현

policy \(\pi\)가 주어지면 필요한 dynamics만 추출한다.

- \(P_\pi\): policy \(\pi\)가 선택하는 action에 대한 transition matrix.
- \(R_\pi\): policy \(\pi\)가 선택하는 action에 대한 reward vector.

policy evaluation은 다음 선형방정식으로 풀 수 있다.

\[
V^\pi=(I-\gamma P_\pi)^{-1}R_\pi
\]

코드에서는 `numpy.linalg.solve`를 사용한다.

## 14.4 Policy Update 구현

1. \(V^\pi\)에서 \(Q^\pi\)를 계산한다.
2. `argmax`를 적용해 greedy policy를 뽑는다.
3. 이 policy를 다음 iteration의 policy로 사용한다.

## 14.5 stopping condition

update 전 policy와 update 후 policy가 같으면 더 바꿀 것이 없다고 보고 종료한다.

```text
if new_policy == old_policy:
    break
```

## 14.6 toy MDP 결과

- state 2개, action 2개인 단순 MDP.
- Policy Iteration은 한 step 만에 종료되었다.
- Value Iteration과 사실상 같은 결과를 냈다.

## 14.7 GridWorld 결과

- Value Iteration: 153 step.
- Policy Iteration: 6 step.
- 결과 value는 소수점 둘째 자리까지 거의 동일했다.
- 시간은 문제와 구현에 따라 다르다.

## 14.8 왜 MDP 알고리즘을 배우는가

Backward induction, Value Iteration, Policy Iteration은 model 정보가 없으면 사용할 수 없다.

그럼에도 배우는 이유:

- RL 알고리즘의 기반 아이디어이기 때문이다.
- Q function을 학습하고 policy를 추출하는 구조가 그대로 이어진다.

# 16. 실습 3 — Tabular Q-Learning

## 16.1 QTable class

Q function을 table 형태로 저장한다.

\[
Q[s,a]
\]

- discrete state.
- discrete action.
- table로 Q를 표현할 수 있다.

## 16.2 초기화

Q table은 임의 값으로 초기화할 수 있다.

- 예: 모든 값을 \(-300\)으로 초기화.
- 꼭 \(-300\)일 필요는 없다.
- 충분히 학습하면 적절한 값으로 업데이트된다.

## 16.3 update 함수

핵심 update:

\[
Q(s,a)
\leftarrow
Q(s,a)+\alpha
\left[
r+\gamma\max_{a'}Q(s',a')-Q(s,a)
\right]
\]

코드에서 target 또는 error로 부를 수 있다.

## 16.4 `np.max(self.q[next_state])`

- `self.q[next_state]`는 next state에서 가능한 모든 action의 Q값 vector이다.
- `np.max`를 취하면 \(\max_{a'}Q(s',a')\)가 된다.

## 16.5 `act` 함수

state를 입력받아 greedy action을 반환한다.

\[
a=\arg\max_a Q(s,a)
\]

코드에서는 `np.argmax(self.q[state])`를 사용한다.

## 16.6 Q에서 V 계산

\[
V(s)=\max_a Q(s,a)
\]

따라서 Q table에서 state별 max를 취하면 value function을 얻을 수 있다.

# 17. Step Size와 Exploration Schedule

## 17.1 step size

Q-learning에서 \(\alpha\)는 update 크기를 조절한다.

- learning rate.
- step size.

## 17.2 visit count 기반 step size

state-action pair 방문 횟수에 따라 \(\alpha\)를 줄인다.

- 많이 방문한 \((s,a)\): 이미 정보가 많으므로 작게 update.
- 적게 방문한 \((s,a)\): 아직 정보가 부족하므로 크게 update.

## 17.3 epsilon schedule

epsilon-greedy에서 epsilon을 시간에 따라 조절한다.

- linear schedule.
- exponential schedule.

주의:

- final epsilon이 너무 크면 학습이 끝나도 random action을 많이 한다.
- 예: final epsilon 0.5는 너무 크다.
- 보통 0.1, 0.05 등 작은 값으로 둔다.

# 18. Epsilon-Greedy 구현

## 18.1 random number 사용

`u = np.random.rand()`는 0 이상 1 미만의 난수를 뽑는다.

## 18.2 조건

```python
if u < epsilon:
    action = env.action_space.sample()
else:
    action = runner.act(state)
```

- \(u<\epsilon\): random action.
- 그 외: greedy action.

## 18.3 agent-environment interaction

environment에는 action만 전달한다.

```python
next_state, reward, done, info = env.step(action)
```

환경은 next state와 reward를 돌려준다.

## 18.4 transition sample

Q-learning update에는 다음이 들어간다.

\[
(s,a,r,s')
\]

추가로 visit count 기반 \(\alpha\)도 넣는다.

# 20. 실습 4 — DQN 개요

## 20.1 DQN 이름

- D: Deep Learning.
- Q: Q function / Q-learning.
- N: Network.

## 20.2 tabular에서 network로

Tabular Q-learning은 discrete state/action table을 쓴다.

DQN은 neural network로 Q function을 근사한다.

장점:

- high-dimensional observation을 처리할 수 있다.
- table을 직접 만들 필요가 없다.

주의:

- DQN은 보통 discrete action에서 사용한다.
- action output을 finite action 개수만큼 내는 구조가 일반적이다.

## 20.3 DQN의 두 대표 기법

1. Experience Replay.
2. Target Network.

이 두 기법은 이후 많은 RL 알고리즘에서도 기본적으로 사용된다.

# 21. Q Network Architecture

## 21.1 Q network의 두 구조

Q function은 원래 \(Q(s,a)\)이므로 input이 state와 action 두 개이다.

### 구조 1: state와 action 모두 입력

```text
input:  s, a
output: scalar Q(s,a)
```

장점:

- continuous action에도 사용할 수 있다.

단점:

- 여러 action의 Q값을 비교하려면 action마다 network를 여러 번 forward해야 한다.

### 구조 2: state만 입력

```text
input:  s
output: [Q(s,a_1), Q(s,a_2), ..., Q(s,a_m)]
```

장점:

- discrete action에서 효율적이다.
- 한 번 forward로 모든 action의 Q값을 얻는다.
- `argmax`가 쉽다.

단점:

- continuous action에서는 사용할 수 없다.
- action 개수가 유한해야 한다.

## 21.2 실습에서 사용한 구조

CartPole은 action이 discrete이다.

- 왼쪽으로 움직이기.
- 오른쪽으로 움직이기.

따라서 state만 input으로 넣고 action 개수만큼 Q값을 출력하는 구조를 사용한다.

---

# 22. Critic Network 구현

## 22.1 이름이 critic인 이유

Q network는 action의 가치를 평가한다.

actor-critic 용어에서는 value/Q를 평가하는 쪽을 critic이라고 부른다.

## 22.2 3-layer MLP

실습에서는 3 layer MLP를 구현한다.

```text
state -> Linear -> ReLU -> Linear -> ReLU -> Linear -> Q values
```

## 22.3 output layer

마지막 layer는 hidden size 2에서 number of action으로 mapping한다.

```python
self.fc3 = nn.Linear(hidden_size_2, number_of_action)
```

## 22.4 마지막 activation 없음

마지막 output에는 ReLU를 붙이지 않는다.

이유:

- Q value는 음수일 수 있다.
- ReLU를 붙이면 Q value가 0 이상으로 제한된다.

---

# 23. DQNAgent의 action 선택

## 23.1 agent가 가진 것

DQNAgent는 내부에 critic network를 가진다.

```python
self.critic = Critic(...).to(device)
```

## 23.2 epsilon-greedy 포함

`act(state, epsilon)` 함수가 epsilon-greedy를 직접 수행한다.

- 확률 \(\epsilon\): random action.
- 확률 \(1-\epsilon\): greedy action.

## 23.3 random action

action 개수가 \(n\)이면 0부터 \(n-1\) 사이 정수를 random sampling한다.

```python
np.random.randint(self.numberOfAction)
```

## 23.4 greedy action

state를 tensor로 바꾸고 GPU로 보낸 뒤 critic에 통과시킨다.

```python
q_values = self.critic(state)
action = argmax(q_values)
```

GPU에서 계산한 결과를 Gym environment에 넣기 위해 CPU / NumPy로 변환하는 과정이 필요하다.

## 24.1 입력

agent.critic.train()

# 25. Terminal Masking

## 25.1 terminal flag

`done` 또는 `terminal`은 episode가 끝났는지 나타낸다.

- terminal = 1: episode 종료.
- terminal = 0: episode 계속.

## 25.2 mask

\[
mask = 1 - terminal
\]

- episode가 끝났으면 mask = 0.
- episode가 끝나지 않았으면 mask = 1.

## 25.3 왜 필요한가

target:

\[
y=r+\gamma \max_{a'}Q(s',a')
\]

terminal state에서는 다음 state의 Q값을 사용하면 안 된다.

따라서:

\[
y=r+\gamma \cdot mask \cdot \max_{a'}Q(s',a')
\]

terminal이면 mask가 0이 되어 target이 reward만 남는다.

## 25.4 중요성

masking을 빼면 존재하지 않는 next state의 Q값이 들어가 학습이 망가질 수 있다.

---

# 26. Target 계산 코드 의미

## 26.1 next Q 계산

target critic에 next observation을 넣는다.

```python
next_q = target_critic(next_observations).max(...)
```

의미:

\[
\max_{a'}Q_{\phi^-}(s',a')
\]

## 26.2 target

```python
target = rewards + gamma * masks * next_q
```

의미:

\[
y=r+\gamma \cdot mask \cdot \max_{a'}Q_{\phi^-}(s',a')
\]

## 26.3 현재 Q output

online critic에 current observation을 넣고, 실제 수행한 action의 Q값만 가져온다.

```python
out = agent.critic(observations).gather(1, actions)
```

의미:

\[
Q_\phi(s,a)
\]

# 27. Loss와 Optimizer

## 27.1 MSE loss

현재 Q output과 target 사이의 MSE를 줄인다.

\[
loss=\left(Q_\phi(s,a)-y\right)^2
\]

코드:

```python
loss = loss_fn(out, target)
```

## 27.2 gradient descent

일반 PyTorch 학습 흐름과 같다.

```python
optimizer.zero_grad()
loss.backward()
optimizer.step()
```

## 27.3 supervised learning과 차이

겉보기에는 supervised regression처럼 보인다.

하지만 target \(y\)는 dataset에 고정된 label이 아니라 target network와 Bellman equation으로 만든 값이다.

# 28. Target Network Soft Update

## 28.1 target network freeze

target critic은 gradient descent로 직접 업데이트하면 안 된다.

```python
param.requires_grad = False
```

이 설정을 하지 않으면 target network를 사용하는 의미가 사라진다.

## 28.2 soft update

target parameter를 online parameter와 조금씩 섞는다.

\[
\theta^- \leftarrow (1-\tau)\theta^-+\tau\theta
\]

## 28.3 직관

- target network는 천천히 움직인다.
- online network의 변화를 조금씩 따라간다.
- 학습 target의 급격한 변화를 막는다.

---

# 29. DQN Training Loop

## 29.1 초기화

- target critic을 online critic의 copy로 만든다.
- environment를 만든다.
- optimizer를 준비한다.
- replay buffer를 만든다.
- epsilon schedule을 만든다.

## 29.2 init buffer

replay buffer가 비어 있을 때는 update가 의미 없다.

초기에는 random action으로 buffer를 채운다.

```text
if t < init_buffer:
    random action
else:
    epsilon-greedy action
```

## 29.3 interaction

```python
next_observation, reward, done, info = env.step(action)
```

그 결과를 replay buffer에 저장한다.

\[
(s,a,r,s',done)
\]

## 29.4 episode 종료 처리

done이 true이면:

- environment reset.
- count 초기화.
- done flag false로 재설정.

## 29.5 periodic evaluation

evaluation interval마다 현재 policy의 score를 측정하고 출력한다.

## 29.6 periodic update

train interval마다 update 함수를 호출한다.

이 update 함수 안에서:

1. batch sampling.
2. target 계산.
3. current Q 계산.
4. MSE loss 계산.
5. critic update.
6. target critic soft update.

가 수행된다.

# 31. DQN 이후 후속 연구

## 31.1 왜 후속 연구가 필요한가

DQN에는 여전히 문제가 남아 있다.

- 충분히 안정적인가?
- sample efficiency가 충분한가?
- overestimation은 없는가?
- epsilon-greedy로 exploration이 충분한가?

## 31.2 Double DQN

- overestimation 문제를 줄이기 위해 등장했다.
- action selection과 value estimation을 분리한다.

## 31.3 Prioritized Experience Replay

Experience Replay에서 sample을 uniform하게 뽑는 대신, 학습이 잘 안 된 sample을 더 자주 뽑는다.

- TD error가 큰 sample을 더 자주 학습할 수 있다.
- 약점을 더 많이 복습하는 방식이다.

## 31.4 DQN의 의의

DQN은 이후 많은 deep RL 알고리즘의 기반이 되었다.

- Experience Replay.
- Target Network.
- Q network.
- value-based deep RL.

이 구성요소들은 이후 알고리즘에서도 반복적으로 등장한다.

# 32. 오후 핵심 정리

## 32.1 DQN 핵심

1. Q-learning을 neural network와 결합한다.
2. Experience Replay로 sample correlation을 줄인다.
3. Target Network로 moving target 문제를 줄인다.
4. Q regression으로 online network를 학습한다.
5. target network는 hard update 또는 soft update로 천천히 갱신한다.

## 32.2 Double DQN 핵심

1. DQN의 overestimation은 max operator에서 온다.
2. action selection과 value estimation을 같은 network가 하면 noise correlation이 커진다.
3. Double DQN은 online network로 action을 고르고 target network로 value를 평가한다.
4. target 계산 한 줄을 바꾸는 것만으로 안정성과 성능을 높일 수 있다.

## 32.3 실습 핵심

1. Tabular Q-learning은 Q table을 update한다.
2. epsilon-greedy는 random action과 greedy action을 섞는다.
3. evaluation에서는 random action 없이 greedy action만 쓴다.
4. DQN은 critic network를 사용해 state에서 action별 Q값을 출력한다.
5. replay buffer에서 \((s,a,r,s',done)\) batch를 뽑는다.
6. terminal mask를 써서 episode 종료 시 next Q를 제거한다.
7. MSE loss로 current Q와 target Q의 차이를 줄인다.
8. target critic은 gradient로 학습하지 않고 soft update한다.

## 32.4 시험 대비 포인트

- DQN의 두 innovation: Experience Replay, Target Network.
- Experience Replay가 sample correlation을 줄이는 이유.
- Target Network가 moving target 문제를 완화하는 이유.
- DQN target 식.
- terminal masking이 필요한 이유.
- continuous action에서 DQN이 어려운 이유.
- epsilon-greedy schedule의 직관.
- overestimation이 max operator 때문에 생기는 이유.
- Double DQN target 식과 DQN target 식의 차이.
- Q network의 두 architecture와 discrete/continuous action에서의 차이.
- 실습 코드에서 replay buffer, target critic, MSE loss, soft update가 각각 어떤 역할을 하는지.
