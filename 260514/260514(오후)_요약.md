# 2. 최신 음성 이해 기술

## 2.1 Multilingual ASR

- Whisper는 99개 언어와 68만 시간 학습으로 대표된다.
- 최근에는 더 많은 시간, 더 많은 언어로 확장하려는 시도가 있다.
- Meta 같은 회사는 수백~천 개 언어 지원을 목표로 한다.

## 2.2 Speech Translation

- 단순 transcription은 음성을 같은 언어 텍스트로 바꾸는 것이다.
- Speech translation은 입력 음성을 다른 언어 텍스트로 바꾼다.
- 예: 영어 음성 → 한국어 텍스트.
- Meta Seamless 계열 프로젝트처럼 실시간 다국어 번역을 목표로 하는 연구가 활발하다.

## 2.3 회의 transcription과 요약

- Zoom, Teams, Copilot류 서비스에서 회의 음성 전사와 요약을 제공한다.
- 실제 회의는 한 명이 깨끗하게 말하는 상황이 아니다.
- 여러 사람이 겹쳐 말하고, 잡음이 있고, 긴 시간 이어진다.
- 병목은 ASR 자체보다 **화자 분리와 긴 회의 처리**에 있는 경우가 많다.

# 3. Diarization

## 3.1 의미

- Diarization은 “누가 언제 말했는지”를 구분하는 기술이다.
- 여러 화자가 섞인 통짜 오디오를 화자별 구간으로 나눈다.

```text
mixed meeting audio → speaker 1 segments / speaker 2 segments / ...
```

## 3.2 왜 중요한가

- 회의 요약에서 누가 어떤 말을 했는지 알아야 한다.
- 화자 분리가 틀리면 뒤의 ASR과 LLM 요약도 틀어진다.
- 실제 서비스에서 diarization 성능은 아직 만족스럽지 않은 경우가 많다.

## 3.3 일반 파이프라인

1. Diarization으로 화자/구간 분리.
2. 각 구간에 ASR 수행.
3. transcript를 LLM이 요약.

## 3.4 시험/이해 포인트

- 최근 음성 이해에서 ASR만 잘한다고 끝이 아니다.
- real-world 회의에서는 diarization이 큰 병목이다.

# 4. VAD와 Streaming ASR

## 4.1 VAD — Voice Activity Detection

- 사람이 말하고 있는 구간과 침묵 구간을 구분한다.
- 0/1 binary classification처럼 볼 수 있다.

## 4.2 VAD가 중요한 이유

1. 계산량 절약
   - 회의 1시간 전체에 ASR을 계속 켜두면 자원이 낭비된다.
   - 말하는 구간만 ASR을 돌리면 효율적이다.

2. hallucination 방지
   - ASR 모델은 무음 입력에서 이상한 텍스트를 만들 수 있다.
   - VAD로 무음 구간을 제거하면 이런 문제를 줄인다.

## 4.3 Streaming ASR

- 실시간으로 말하는 동안 바로 자막을 생성하는 ASR.
- 유튜브 실시간 자막, 기조연설 자동 자막 등에 사용된다.
- on-device/low-latency 모델과 관련이 깊다.

# 5. Audio/Music Understanding과 Generation

## 5.1 Speech와 Audio의 차이

- Speech는 대응되는 text transcript가 비교적 명확하다.
- 일반 audio는 “정답 텍스트”가 모호하다.
- 따라서 audio captioning, audio question answering 같은 task가 등장한다.

## 5.2 Audio captioning / QA

- 오디오를 듣고 “개 짖는 소리와 배경 음악이 있다” 같은 설명을 생성한다.
- 영화, 게임, 사운드 이펙트 산업과 연결된다.

## 5.3 Music generation

- 텍스트 prompt로 음악을 생성하는 모델들이 발전하고 있다.
- Suno 같은 서비스가 예시로 언급됐다.
- 문제는 오디오/음악에 대한 caption 데이터가 비싸고 모으기 어렵다는 점이다.

## 5.4 데이터의 가치

- speech-text pair는 상대적으로 많다.
- audio/music-caption pair는 직접 annotation해야 하므로 매우 비싸다.
- NVIDIA 등 일부 연구 그룹이 자체 데이터 수집을 강하게 한다.

# 6. 최신 음성 생성 기술

## 6.1 Voice Cloning

- reference voice를 주면 그 사람의 목소리 특성을 흉내 내는 TTS.
- 보이스 피싱과 같은 사회적 문제와도 연결된다.
- reference 음질이 좋고 일반적인 목소리일수록 더 잘 복제된다.

## 6.2 Promptable TTS

- 단순히 텍스트만 읽는 것이 아니라 감정/말투/속도를 prompt로 제어한다.
- 예: 화난 목소리, 과장된 톤, high energy.
- 아직 완벽하게 다이내믹하지는 않지만 빠르게 발전 중이다.

## 6.3 Non-verbal sound

- 웃음, 한숨, 코 훌쩍임 같은 비언어적 표현도 합성 대상이 되고 있다.
- 자연스러운 비언어 음성 합성은 최근에야 잘 되기 시작했다.

## 6.4 Dialogue TTS

- 한 사람이 아니라 두 사람이 대화하는 팟캐스트형 음성 합성도 가능하다.
- Google NotebookLM 같은 예시가 언급됐다.

# 7. Speech Language Model과 Voice Assistant

## 7.1 LLM + TTS 구조

- 최근 voice assistant는 LLM이 텍스트 답변을 만들고, TTS가 음성으로 읽어주는 구조가 많다.

```text
user speech → ASR/audio encoder → LLM → text response → TTS → speech response
```

## 7.2 TTS의 역할

- TTS는 독립 서비스뿐 아니라 LLM 기반 voice assistant의 마지막 출력 모듈로 쓰인다.
- 좋은 TTS를 붙이면 텍스트 LLM도 음성 대화형 서비스처럼 보일 수 있다.

## 7.3 Two-stage 구조의 한계

- 중간에 text를 거치면 음성의 paralinguistic feature가 손실된다.
- 예:
  - 감정
  - 떨림
  - pause
  - 말투
  - backchannel
- 텍스트에는 이런 연속적인 음향 정보가 충분히 담기지 않는다.

## 7.4 End-to-end speech-to-speech

- 오디오 token을 직접 입력/출력하는 end-to-end 모델은 비언어 정보를 더 잘 보존할 가능성이 있다.
- 하지만 구현과 학습이 어렵다.
- 현재는 모듈형 구조가 성능과 안정성 면에서 실용적이다.

---

# 8. Full-duplex와 동시통역

## 8.1 Turn-based의 한계

- 현재 많은 voice assistant는 “내가 말함 → 모델이 말함”의 turn-based 구조다.
- 사람 대화는 중간 끼어들기, 추임새, 겹침이 많다.
- 자연스러운 대화를 위해서는 full-duplex가 필요하다.

## 8.2 Full-duplex

- 사용자와 모델의 stream이 동시에 존재한다.
- 모델이 말하는 중에도 사용자가 끊을 수 있다.
- 모델도 silence token을 내며 대기하거나 적절히 반응할 수 있다.

## 8.3 동시통역

- 순차통역은 발화를 다 들은 뒤 번역한다.
- 동시통역은 입력이 들어오는 동안 다른 언어로 계속 출력한다.
- low-latency ASR/TTS와 full-duplex 모델이 중요하다.

# 9. 청각 구조 영상 — 사람은 어떻게 듣는가

## 9.1 귀의 구조

- 고막 진동이 작은 뼈들을 통해 달팽이관으로 전달된다.
- 달팽이관 내부 fluid가 흔들리고 hair cell이 반응한다.
- auditory nerve를 통해 뇌가 소리로 해석한다.

## 9.2 Frequency별 위치

- 달팽이관의 위치마다 반응하는 frequency가 다르다.
- base 쪽은 높은 pitch.
- apex 쪽은 낮은 pitch.

## 9.3 음성 AI와 연결

- 인간 청각이 frequency별로 다르게 반응한다.
- Mel scale이 인간 청각 특성을 반영한다는 전날 내용과 연결된다.

# 10. 실습 1 — Mel 만들기

## 10.1 데이터 로드

- LJ Speech 같은 영어 음성 데이터셋을 사용했다.
- 원본 waveform을 듣고, mel-spectrogram을 계산해 시각화했다.

## 10.2 STFT/Mel 설정

- sample rate.
- window length.
- hop length.
- n_mels.
- log mel.

## 10.3 Text-to-Mel

- Tacotron 2 acoustic model을 불러온다.
- 입력 text를 tokenizer로 token화한다.
- Tacotron 2가 mel-spectrogram을 생성한다.

## 10.4 punctuation과 pause

- comma, period 같은 문장부호는 mel에서 pause처럼 나타날 수 있다.
- text를 바꿔보며 mel pattern이 어떻게 달라지는지 확인했다.

# 11. Tacotron 2 Attention Map 실습

## 11.1 Alignment 시각화

- Tacotron 2 내부 attention/alignment matrix를 뽑아 시각화했다.
- x축은 text token, y축은 mel frame처럼 볼 수 있다.

## 11.2 정상 attention

- 대각선 형태가 나오면 text와 mel이 순서대로 잘 정렬된 것이다.
- 이 경우 발음이 크게 망가지지 않을 가능성이 높다.

## 11.3 시험 포인트

- Tacotron 2는 attention으로 text-mel alignment를 학습한다.
- attention map이 대각선이면 alignment가 잘 된 것이다.
- attention failure는 반복/누락/발음 오류와 연결된다.

# 12. Vocoder 비교 — Griffin-Lim / WaveGlow / HiFi-GAN

## 12.1 Griffin-Lim

- Neural network가 아니라 수학적 phase reconstruction 방법.
- mel/magnitude에는 phase 정보가 없으므로 phase를 그럴듯하게 추정한다.
- 실습에서 metallic/robotic한 소리가 났다.

## 12.2 Phase 정보

- STFT 결과에는 magnitude와 phase가 있다.
- Mel-spectrogram은 주로 magnitude 정보를 담고 phase는 버린다.
- 사람은 단일 pure tone의 절대 phase 차이를 잘 못 느낄 수 있다.
- 하지만 여러 파형이 합쳐질 때 phase 차이는 중첩/상쇄를 바꿔 waveform 품질에 영향을 준다.

## 12.3 WaveGlow

- WaveNet 계열보다 빠르게 만들려는 neural vocoder 중 하나.
- Griffin-Lim보다 나아질 수 있지만, 실습에서는 HiFi-GAN보다 떨어지는 음질로 들렸다.

## 12.4 HiFi-GAN

- mel을 입력받아 고품질 waveform을 빠르게 생성한다.
- 실습에서 가장 자연스러운 음질을 보였다.
- 현업 TTS에서도 여전히 널리 쓰인다.

## 12.5 핵심 비교

| Vocoder | 특징 | 음질/속도 직관 |
|---|---|---|
| Griffin-Lim | phase 수학적 추정 | robotic, 오래된 방식 |
| WaveGlow | neural vocoder | 중간 |
| HiFi-GAN | NAR GAN vocoder | 빠르고 자연스러움 |

# 13. Acoustic Model과 Vocoder의 독립성

## 13.1 독립 모듈

- Acoustic model과 vocoder는 따로 바꿔 끼울 수 있다.
- Tacotron 2로 만든 mel을 Griffin-Lim, WaveGlow, HiFi-GAN에 각각 넣을 수 있다.
- 원본 audio를 mel로 바꾼 뒤 HiFi-GAN으로 다시 복원할 수도 있다.

## 13.2 의미

- TTS pipeline은 modular하다.
- 좋은 acoustic model과 좋은 vocoder를 조합하면 된다.
- on-device에서는 크기/속도와 품질을 trade-off한다.

# 14. 실습 2 — AR vs NAR Latency

## 14.1 비교 대상

- AR acoustic model: Tacotron 계열.
- NAR acoustic model: FastSpeech 2 계열.

## 14.2 AR의 특징

- 이전 frame/token을 보고 다음 frame을 생성한다.
- 순차 생성이 필요하다.
- 문장이 길어질수록 latency가 누적된다.

## 14.3 NAR의 특징

- 모든 mel frame을 병렬로 생성한다.
- GPU 병렬화가 잘 된다.
- 문장 길이에 따른 latency 증가가 훨씬 작다.

## 14.4 실무적 의미

- 오디오북, 긴 문장 TTS, 동시통역, voice assistant에서는 latency가 중요하다.
- 긴 발화를 합성할수록 AR의 약점이 커진다.
- NAR 모델은 산업적 활용에서 유리하다.

# 15. FastSpeech 2 Control 실습

## 15.1 Pitch control

- pitch를 높이거나 낮춰 목소리 높낮이를 바꿀 수 있다.
- 실습에서 값 조절에 따라 음성이 달라짐을 확인했다.

## 15.2 Duration/speed control

- duration을 조절하면 말 속도가 바뀐다.
- 빠르게/느리게 읽는 TTS를 만들 수 있다.

## 15.3 Energy control

- energy/loudness를 조절해 음성 크기나 강도를 바꿀 수 있다.

## 15.4 의미

- FastSpeech 2는 acoustic feature를 명시적으로 조건으로 넣을 수 있다.
- 최신 promptable TTS는 이런 제어를 자연어 prompt로 확장한다.

# 16. OmniVoice Voice Cloning 실습

## 16.1 모델

- OmniVoice라는 최신 TTS/voice cloning 모델을 사용했다.
- diffusion 기반으로 설명됐다.
- 여러 언어를 지원한다.

## 16.2 Reference voice

- voice cloning에는 reference voice가 필요하다.
- 본인 목소리를 녹음하거나, 데이터셋/유튜브에서 음성을 가져올 수 있다.
- reference 품질이 좋을수록 cloning 품질도 좋아진다.

## 16.3 Cross-lingual voice cloning

- 영어 화자 reference로 한국어 문장을 생성하면 교포 같은 억양이 나올 수 있다.
- reference voice의 발음/언어 특성이 출력에 반영된다.

## 16.4 윤리적 문제

- 보이스 피싱/사칭에 악용될 수 있다.
- 실습은 기술 이해 목적이지만 사회적 위험도 함께 언급됐다.

# 17. Voice Cloning Challenge

## 17.1 과제

- reference real voice와 generated TTS voice를 하나씩 제출한다.
- 다른 사람들이 어느 쪽이 AI 음성인지 맞힌다.
- 목표는 동료들이 구분하기 어렵게 고품질 샘플을 만드는 것.

## 17.2 실습 변수

- reference voice 길이.
- reference voice 품질.
- 합성할 text의 자연스러움.
- 문어체/구어체.
- 배경 잡음.
- 화자 특이성.

## 17.3 관찰

- 좋은 reference를 쓰면 매우 그럴듯한 복제가 가능하다.
- 노트북 마이크의 잡음까지 따라 하는 경우가 있다.
- 짧은 샘플에서는 사람이 구분하기 어려울 수 있다.

# 18. 오후 강의 핵심 정리

## 18.1 반드시 외울 것

1. **Diarization**
   - 여러 화자의 음성을 분리하고 누가 언제 말했는지 구분.
   - 회의 transcription/summary의 병목.

2. **VAD**
   - 말하는 구간과 무음 구간 구분.
   - 계산량 절약과 hallucination 방지.

3. **TTS 최신 방향**
   - voice cloning.
   - promptable TTS.
   - non-verbal sound.
   - dialogue TTS.

4. **Speech language model**
   - LLM + TTS 구조.
   - two-stage는 paralinguistic feature 손실.
   - end-to-end speech-to-speech가 미래 방향.

5. **Tacotron 2 실습**
   - text-to-mel.
   - attention map 대각선 = alignment 성공.

6. **Vocoder 비교**
   - Griffin-Lim은 phase 추정이라 robotic.
   - HiFi-GAN은 빠르고 자연스러움.

7. **AR vs NAR**
   - AR은 순차 생성으로 길어질수록 느림.
   - NAR은 병렬 생성으로 latency 유리.

8. **FastSpeech 2 control**
   - pitch, energy, duration 조절 가능.

9. **Voice cloning**
   - reference voice 기반 화자 복제.
   - 품질이 매우 좋아졌고 윤리적 위험이 큼.

## 18.2 예상 시험 문제 스타일

- Diarization과 VAD가 실제 ASR 서비스에서 왜 중요한지 설명하라.
- Whisper hallucination 방지에 VAD가 왜 도움이 되는지 설명하라.
- Tacotron 2 attention map이 대각선일 때 의미를 설명하라.
- Griffin-Lim과 HiFi-GAN의 차이를 phase/vocoder 관점에서 설명하라.
- AR TTS와 NAR TTS의 latency 차이를 실무 예시와 함께 설명하라.
- FastSpeech 2에서 pitch/duration control이 가능한 이유를 설명하라.
- LLM 뒤에 TTS를 붙이는 two-stage voice assistant의 한계를 설명하라.
