# 260514 오전 강의 요약 — TTS 개요 · Acoustic Model · Tacotron 2 · FastSpeech 2 · Forced Alignment · Vocoder · WaveNet · HiFi-GAN · MOS

> 오전 흐름: **ASR에서 TTS로 전환 → Text-to-Speech 전체 파이프라인 → Acoustic Model과 Vocoder 두 단계 → TTS의 평가 기준(MOS) → Text Normalization과 Phoneme → Tacotron 2 구조와 attention alignment → AR mel frame generation → Post-net과 Stop token → Tacotron 2의 느림/불안정성 → FastSpeech 2 등장 → forced alignment로 duration label 획득 → duration predictor + length regulator → NAR 병렬 생성 → Vocoder로 전환 → Mel에는 phase/high frequency 정보가 빠짐 → WaveNet: causal/dilated convolution, μ-law, AR sample generation → 너무 느림 → HiFi-GAN: NAR GAN vocoder, MPD/MSD, feature matching loss, mel loss → 현업에서도 많이 쓰임**
> 강의 주제: **인공지능 기반 음성합성(TTS)의 기본 구조와 대표 모델** — text를 mel로 만드는 acoustic model과 mel을 waveform으로 바꾸는 vocoder를 분리해 설명.

---

# 1. TTS 전체 그림

## 1.1 TTS의 목표

- Text-to-Speech는 텍스트를 자연스러운 음성으로 바꾸는 기술이다.
- 단순히 읽는 것뿐 아니라 다음 요소가 중요하다.

| 목표 | 설명 |
|---|---|
| Naturalness | 사람이 말하는 것처럼 자연스러운가 |
| Pronunciation | 발음이 맞는가 |
| Prosody | 억양, 속도, 리듬이 자연스러운가 |
| Speaker identity | 특정 화자의 목소리를 유지하는가 |
| Emotion/style | 감정과 말투를 표현하는가 |

## 1.2 현대 TTS의 2단계 구조

현대 TTS는 보통 두 모듈로 나뉜다.

```text
Text / Phoneme → Acoustic Model → Mel-spectrogram → Vocoder → Waveform
```

| 모듈 | 입력 | 출력 | 예시 |
|---|---|---|---|
| Acoustic Model | text 또는 phoneme | mel-spectrogram | Tacotron 2, FastSpeech 2 |
| Vocoder | mel-spectrogram | raw waveform | WaveNet, HiFi-GAN |

## 1.3 시험 포인트

- Acoustic Model과 Vocoder의 입력/출력을 정확히 구분해야 한다.
- Acoustic Model은 “무슨 말을 어떻게 발음할지”를 mel로 만든다.
- Vocoder는 mel을 실제 재생 가능한 waveform으로 바꾼다.

---

# 2. TTS에서의 Alignment Problem

## 2.1 왜 alignment가 필요한가

- 입력 text/phoneme은 짧다.
- 출력 mel-spectrogram은 수백~수천 frame으로 길다.
- 어떤 phoneme이 몇 frame 동안 지속되는지 알아야 한다.
- 사람마다 말 속도와 발음 길이가 다르므로 고정 비율이 없다.

## 2.2 ASR과의 공통점

- ASR: speech frame → text.
- TTS: text → speech/mel frame.
- 방향은 반대지만 둘 다 text와 speech frame 사이 alignment를 풀어야 한다.

## 2.3 Vocoder에는 alignment 문제가 없음

- Mel과 waveform은 STFT hop length로 정해진 고정 비율을 가진다.
- 예: hop length 256이면 mel frame 하나가 waveform 256 sample에 대응한다.
- 따라서 vocoder는 alignment를 새로 찾을 필요가 없다.

---

# 3. TTS 평가 — MOS

## 3.1 MOS의 의미

- MOS = Mean Opinion Score.
- 사람이 음성을 듣고 1~5점으로 평가한 평균.
- TTS 자연스러움은 객관 metric만으로 평가하기 어렵기 때문에 사람 평가가 중요하다.

## 3.2 MOS의 한계

- 평가자마다 기준이 다르다.
- 문장, 시간, 환경에 따라 점수가 달라질 수 있다.
- 비용이 든다.
- 그래도 대안이 부족해 널리 사용된다.

## 3.3 시험 포인트

- TTS는 ASR처럼 WER 같은 명확한 objective metric이 없다.
- 자연스러움은 주관적이라 MOS를 쓴다.

---

# 4. Text Normalization과 Phoneme

## 4.1 Text Normalization

- TTS 입력 텍스트를 모델이 읽기 쉬운 형태로 바꾸는 전처리.
- 숫자, 약어, 기호 등을 실제 발화 형태로 변환한다.

예:

```text
$3.5 → three dollars and fifty cents
Dr. → Doctor
```

## 4.2 왜 필요한가

- 같은 표기가 문맥에 따라 다르게 읽힐 수 있다.
- 모델이 raw text만 보고 모든 경우를 처리하기 어렵다.
- 전처리를 통해 발음 오류를 줄인다.

## 4.3 Phoneme

- 단어를 발음 기호 sequence로 바꾼다.
- 영어 단어는 매우 많지만 phoneme 수는 훨씬 적다.
- 영어는 약 40개 phoneme으로 많은 단어 발음을 표현할 수 있다.

## 4.4 Phoneme을 쓰는 이유

- vocabulary를 줄인다.
- 발음 중심으로 입력을 통일한다.
- 같은 철자라도 다른 발음, 다른 철자라도 같은 발음 문제를 줄인다.

# 5. Tacotron 2

## 5.1 Tacotron 2의 위치

- Tacotron 2는 대표적인 acoustic model.
- text/phoneme을 입력받아 mel-spectrogram을 생성한다.
- 이후 WaveNet 같은 vocoder가 mel을 waveform으로 바꾼다.

## 5.2 구조 직관

```text
text/phoneme → encoder → attention → decoder → mel frame
```

- Encoder는 text/phoneme representation을 만든다.
- Attention은 현재 생성할 mel frame이 입력 text의 어느 위치를 봐야 하는지 결정한다.
- Decoder는 mel frame을 autoregressive하게 생성한다.

## 5.3 Attention alignment

- Tacotron 2는 attention으로 text와 mel 사이 alignment를 직접 학습한다.
- 정상적으로 학습되면 attention map은 대각선 형태가 된다.
- 대각선은 입력 text 진행과 출력 mel 진행이 순서대로 맞는다는 의미다.

## 5.4 Attention failure

- attention이 끊기거나 되돌아가면 문제가 생긴다.
- 같은 단어/구간을 반복할 수 있다.
- 어떤 단어를 건너뛸 수 있다.
- 발음이 흐트러질 수 있다.

---

# 6. Tacotron 2의 학습 세부

## 6.1 Mel은 연속값

- Tacotron 2의 출력은 discrete token이 아니라 mel-spectrogram의 연속값이다.
- 따라서 ASR의 vocabulary softmax + cross entropy와 다르다.
- 보통 MSE/L2 계열의 regression loss를 사용한다.

## 6.2 Post-net

- Decoder가 만든 mel을 한 번 더 보정하는 네트워크.
- 더 자연스럽고 정교한 mel-spectrogram을 만들기 위한 refinement 단계다.

## 6.3 Stop token

- Tacotron 2는 autoregressive하게 mel frame을 계속 생성한다.
- 따라서 언제 멈출지 판단해야 한다.
- stop token을 따로 예측한다.
- stop token은 보통 binary classification으로 보고 BCE loss를 사용한다.

## 6.4 Tacotron 2의 약점

1. **느림**
   - mel frame을 한 장씩 순차 생성한다.
   - 긴 문장일수록 inference가 느리다.

2. **불안정**
   - attention alignment가 실패하면 반복/누락 문제가 생긴다.

# 7. FastSpeech 2

## 7.1 등장 배경

- Tacotron 2는 autoregressive라 느리다.
- attention alignment가 불안정할 수 있다.
- FastSpeech 2는 non-autoregressive 방식으로 모든 mel frame을 병렬 생성한다.

## 7.2 핵심 아이디어

- text/phoneme과 mel frame 사이 duration을 미리 알아낸다.
- 각 phoneme이 몇 frame 동안 지속되는지 알면 alignment를 직접 찾지 않아도 된다.
- phoneme representation을 duration만큼 늘려 mel 길이에 맞춘 뒤 병렬 생성한다.

---

# 8. Forced Alignment

## 8.1 의미

- Forced alignment는 text/phoneme과 실제 speech를 넣어 각 phoneme의 시간 구간/duration을 얻는 과정이다.
- FastSpeech 2 학습 전에 duration label을 만들기 위해 사용한다.

```text
phoneme + speech → forced aligner → phoneme duration
```

## 8.2 비용

- 강의에서는 forced aligner가 CPU 기반이라 느리다고 설명했다.
- GPU로 빠르게 돌리기 어렵다.
- 많은 샘플을 처리하면 매우 오래 걸린다.
- 하지만 학습 전처리로 한 번 수행하면 된다.

## 8.3 추론 때는 forced alignment를 쓸 수 없음

- inference 때는 text만 있고 정답 speech가 없다.
- 따라서 forced aligner를 사용할 수 없다.
- 대신 duration predictor가 각 phoneme의 duration을 예측한다.

---

# 9. Duration Predictor와 Length Regulator

## 9.1 Duration predictor

- 학습 중 forced alignment로 얻은 duration label을 보고 학습한다.
- 추론 때 text/phoneme만 보고 각 phoneme이 몇 frame 지속될지 예측한다.

## 9.2 Length regulator

- phoneme hidden state를 duration만큼 반복한다.
- 이렇게 길이를 늘려 mel frame sequence와 맞춘다.

예:

```text
phoneme A duration 3 → A A A
phoneme B duration 2 → B B
```

## 9.3 FastSpeech 2의 장점

- attention으로 alignment를 매번 찾지 않는다.
- mel frame을 병렬 생성한다.
- Tacotron 2보다 빠르고 안정적이다.

## 9.4 Acoustic feature control

- FastSpeech 2는 pitch, energy, duration 같은 feature를 조건으로 사용할 수 있다.
- 높낮이, 크기, 속도 조절이 가능하다.
- 오후 실습에서 pitch/duration 조절을 직접 확인했다.

---

# 10. Vocoder 개요

## 10.1 Vocoder의 역할

- mel-spectrogram을 waveform으로 바꾼다.
- 사람이 들을 수 있는 실제 오디오 sample sequence를 생성한다.

## 10.2 Mel에서 빠지는 정보

- Mel-spectrogram은 magnitude 중심 representation이다.
- phase 정보가 없다.
- 고주파 정보도 압축/손실된다.
- Vocoder는 이 빠진 정보를 그럴듯하게 보강해 waveform을 만든다.

## 10.3 Mel에 의미 정보가 이미 들어 있음

- 어떤 말을 하는지, 대략 어떤 음색/억양인지 대부분 mel에 들어 있다.
- Vocoder는 새 의미를 만들기보다 “재생 가능한 파형”으로 변환한다.

# 11. WaveNet

## 11.1 WaveNet의 위치

- Google이 발표한 초기 neural vocoder.
- mel-spectrogram을 조건으로 waveform sample을 생성한다.
- 음질은 좋았지만 매우 느렸다.

## 11.2 Autoregressive sample generation

- waveform은 sample rate가 높아 sequence가 매우 길다.
- 16kHz라면 1초에 16,000 sample.
- WaveNet은 sample을 하나씩 순차 생성한다.
- 이전 sample을 보고 다음 sample을 만든다.

## 11.3 Causal convolution

- 미래 sample을 보면 안 된다.
- 현재 sample을 만들 때 과거 sample만 사용한다.
- 이를 causal convolution이라고 한다.

## 11.4 Dilated causal convolution

- receptive field를 넓히기 위해 dilation을 사용한다.
- layer 수를 많이 늘리지 않아도 먼 과거 sample을 볼 수 있다.
- dilation을 지수적으로 키우면 receptive field가 빠르게 커진다.

## 11.5 μ-law quantization

- raw waveform은 연속값이다.
- WaveNet은 waveform 값을 256개 class로 quantize해 softmax + cross entropy로 학습한다.
- μ-law는 0 근처를 촘촘하게, 양 끝을 상대적으로 거칠게 나누는 방식이다.

## 11.6 WaveNet의 한계

- sample 단위 autoregressive라 너무 느리다.
- 실시간/현업 사용이 어렵다.
- 강의에서는 7초 음성 생성에 매우 긴 시간이 걸리는 예시를 들었다.

---

# 12. HiFi-GAN

## 12.1 HiFi-GAN의 위치

- 카카오에서 발표한 GAN 기반 neural vocoder.
- 현재도 산업계/연구에서 많이 쓰인다.
- WaveNet보다 훨씬 빠르다.

## 12.2 Generator

- 입력은 random noise가 아니라 mel-spectrogram이다.
- transposed convolution으로 mel을 waveform 길이까지 upsample한다.
- 내부에 dilated convolution 등을 사용해 local pattern을 만든다.

## 12.3 Non-autoregressive

- HiFi-GAN은 sample을 하나씩 만들지 않는다.
- mel 전체 또는 chunk를 받아 병렬로 waveform을 생성한다.
- GPU 병렬화를 활용할 수 있어 빠르다.

---

# 13. HiFi-GAN Discriminator

## 13.1 MPD — Multi-Period Discriminator

- 음성의 주기성/로컬 패턴을 본다.
- waveform을 특정 period 기준으로 잘라 2D 형태로 본다.
- 성대 진동, 발음의 짧은 반복 구조를 판별한다.

## 13.2 MSD — Multi-Scale Discriminator

- 음성을 여러 scale로 본다.
- downsampling/smoothing을 통해 전체적인 추이와 global pattern을 본다.
- 로컬 주기성만으로 놓칠 수 있는 큰 흐름을 잡는다.

## 13.3 왜 둘 다 필요한가

- 음성은 짧은 주기 패턴도 중요하고 전체적인 음색/에너지 흐름도 중요하다.
- MPD는 local/periodic.
- MSD는 global/multi-scale.

---

# 14. HiFi-GAN Loss

## 14.1 Adversarial loss

- Generator는 discriminator를 속이도록 학습한다.
- Discriminator는 real waveform과 generated waveform을 구분하도록 학습한다.
- LS-GAN 형태가 사용된다.

## 14.2 Feature matching loss

- discriminator의 중간 feature를 비교한다.
- real audio를 넣었을 때와 generated audio를 넣었을 때 중간 표현이 비슷해지도록 한다.
- 음질 안정화에 중요하다.

## 14.3 Mel spectrogram loss

- 생성된 waveform을 다시 mel-spectrogram으로 변환한다.
- 입력 mel과 생성 waveform에서 다시 얻은 mel이 비슷해지도록 한다.
- 원래 조건으로 준 mel을 잘 보존하게 만든다.

## 14.4 시험 포인트

- HiFi-GAN은 adversarial loss만 쓰지 않는다.
- Feature matching loss와 mel spectrogram loss가 중요하다.
- MPD/MSD의 역할을 구분해야 한다.

# 15. 오전 강의 핵심 정리

## 15.1 반드시 외울 것

1. **TTS 2단계**
   - Acoustic Model: text/phoneme → mel.
   - Vocoder: mel → waveform.

2. **Tacotron 2**
   - attention alignment.
   - autoregressive mel frame generation.
   - MSE/L2 mel loss.
   - post-net.
   - stop token + BCE.

3. **FastSpeech 2**
   - non-autoregressive.
   - forced alignment로 duration label.
   - duration predictor.
   - length regulator.

4. **Vocoder**
   - mel → waveform.
   - alignment problem 없음.
   - phase/high frequency 보강.

5. **WaveNet**
   - AR sample generation.
   - causal/dilated convolution.
   - μ-law quantization.
   - 느림.

6. **HiFi-GAN**
   - NAR GAN vocoder.
   - MPD/MSD.
   - adversarial + feature matching + mel loss.

7. **MOS**
   - TTS 자연스러움 평가를 위한 사람 주관 점수.

## 15.2 예상 시험 문제 스타일

- Acoustic Model과 Vocoder의 입력/출력을 설명하라.
- Tacotron 2와 FastSpeech 2의 속도 차이를 설명하라.
- FastSpeech 2에서 forced alignment와 duration predictor가 하는 일을 설명하라.
- Vocoder에는 왜 alignment problem이 없는지 설명하라.
- WaveNet과 HiFi-GAN을 AR/NAR 관점에서 비교하라.
- HiFi-GAN의 MPD/MSD와 loss를 설명하라.
- MOS가 무엇이고 왜 필요한지 설명하라.
