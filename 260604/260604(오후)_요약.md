# 260604 오후 실습 요약 — Transformer 기반 Object Detection 직접 구현 · DETR 모델/Loss/Attention 시각화 · Deformable DETR Attention · RT-DETR 실행

> 오후 흐름: **DETR HuggingFace 인퍼런스로 logits/box 출력 시각화 → 2D Positional Encoding 직접 구현(sine/cosine, x·y) → DETR-style Transformer Encoder/Decoder 직접 구현(key·query에만 positional encoding 더하기, 모든 layer마다) → DETR 전체 모델 조립(ResNet-50 backbone에서 마지막 2 layer 제거, 1×1 input projection, Query/Class/Box embedding) → Forward 함수 구현 → 헝가리안 매칭 + GIoU 직접 구현 → Classification Loss + L1 + GIoU 직접 구현 → DETR Attention 시각화(cross/self-attention, query slot 분포) → Deformable Attention 직접 구현(5단계: offset projection → attention projection + softmax → reference + offset → value projection → weighted sum) → Multi-Scale Deformable Attention(L scales 확장) → RT-DETR 실행 + DETR과 속도 비교**
> 강의 주제: **DETR의 핵심 컴포넌트(Object Query, Hungarian Matching, GIoU, Cross-Attention)를 코드로 만져보며 트랜스포머 디텍터가 어떻게 anchor·NMS 없이 작동하는지, 그리고 Deformable Attention이 어떤 5단계 파이프라인으로 효율을 달성하는지 체감하는 것**.

# 1. DETR 인퍼런스 ⭐

## 1.1 HuggingFace 모델

```python
outputs = detr(image)
# outputs는 dict
logits = outputs['logits']          # (N_queries, num_classes+1)
boxes  = outputs['pred_boxes']      # (N_queries, 4) -- (cx, cy, w, h)
```

## 1.2 후처리 — Logits → Detection

```python
probas = logits.softmax(-1)[:, :-1]   # background 제외
scores, labels = probas.max(-1)        # 각 query의 최대 클래스
keep = scores > threshold

# box 변환: (cx, cy, w, h) → (x1, y1, x2, y2)
boxes_xyxy = cxcywh_to_xyxy(boxes[keep])
```

⭐ DETR은 **NMS 없이** detection 결과가 바로 나옴.

# 2. 2D Positional Encoding 직접 구현 ⭐

## 2.1 1D 트랜스포머와의 차이

- 1D NLP: 토큰의 1D 위치 인코딩 (sequence index).
- **ViT/DETR**: 패치의 **2D 위치 인코딩** (x, y 좌표 각각).

## 2.2 수식

\[
\omega_k = \frac{1}{10000^{2k/d}}
\]

\[
PE(pos, 2k) = \sin(pos \cdot \omega_k), \quad PE(pos, 2k+1) = \cos(pos \cdot \omega_k)
\]

→ x 방향과 y 방향으로 따로 만들어 concat.

## 2.3 시각화

저주파(low-channel)는 y 방향으로 천천히 변하고, 고주파(high-channel)는 빠르게 변함 → 다양한 위치 정보를 인코딩.

# 3. DETR Transformer Encoder/Decoder 직접 구현 ⭐

## 3.1 일반 트랜스포머 vs DETR의 차이 ⭐⭐

| 항목 | 일반 Transformer | DETR |
|---|---|---|
| Positional Encoding 적용 대상 | Q, K, V 모두 | **Q, K만** (V는 그대로) |
| 입력에 더해주는 횟수 | 처음에 한 번 | **모든 layer마다** |

→ 이유: V는 raw feature, attention의 weight 계산(Q, K)에만 위치 정보가 필요. Layer마다 다시 더해주면 deep layer에서도 위치 정보가 흐려지지 않음.

## 3.2 Encoder Forward

```python
q = k = src + pos_embed
src2 = self_attn(q, k, value=src)
src = src + dropout(src2)
src = norm(src)
# FFN
src2 = ffn(src)
src = norm(src + dropout(src2))
```

## 3.3 Decoder Forward (cross-attention 포함)

```python
# Self-attention with object queries
q = k = tgt + query_embed
tgt = tgt + self_attn(q, k, value=tgt)

# Cross-attention with encoder output
q = tgt + query_embed
k = memory + pos_embed
tgt = tgt + cross_attn(q, k, value=memory)

tgt = tgt + ffn(tgt)
```

# 4. DETR 전체 모델 조립 ⭐

## 4.1 컴포넌트

| 컴포넌트 | 구현 |
|---|---|
| **Backbone** | ResNet-50, **마지막 2 layer(AvgPool + FC) 제거** → `nn.Sequential` |
| **Input Projection** | 1×1 Conv2d (backbone의 2048 ch → d_model) |
| **Positional Encoding** | 위에서 만든 2D PE 함수 |
| **Encoder/Decoder** | 위에서 만든 DETR-style Transformer |
| **Query Embedding** | `nn.Embedding(num_queries, d_model)` (학습 가능) |
| **Class Embedding** | `nn.Linear(d_model, num_classes+1)` |
| **Box Embedding** | 3-layer MLP (`Linear → ReLU → Linear → ReLU → Linear`) |

## 4.2 Forward

```python
features = backbone(image)               # (B, 2048, H, W)
src = input_proj(features)               # (B, d_model, H, W)
pos = positional_encoding(src)           # (B, d_model, H, W)

# Flatten & permute for transformer (B, H*W, d_model)
src = src.flatten(2).permute(0, 2, 1)
pos = pos.flatten(2).permute(0, 2, 1)

memory = encoder(src, pos)

# Object queries: (num_queries, d_model)
query_embed = self.query_embed.weight.unsqueeze(0).repeat(B, 1, 1)
tgt = torch.zeros_like(query_embed)

hs = decoder(tgt, memory, query_embed, pos)

# Prediction heads
classes = self.class_embed(hs)           # (B, num_queries, num_classes+1)
boxes   = self.box_embed(hs).sigmoid()   # (B, num_queries, 4)
```

# 5. 헝가리안 매칭 + GIoU 직접 구현 ⭐

## 5.1 Matching Cost

```python
# Classification cost: 정답 클래스 확률에 negative
cost_class = -probs[:, target_classes]

# L1 box cost
cost_bbox = torch.cdist(pred_boxes, target_boxes, p=1)

# GIoU cost
cost_giou = -generalized_box_iou(pred_boxes_xyxy, target_boxes_xyxy)

# Combined
cost = w_class * cost_class + w_bbox * cost_bbox + w_giou * cost_giou
```

## 5.2 GIoU 구현 ⭐

```python
# Intersection
lt = torch.max(boxes1[:, None, :2], boxes2[:, :2])
rb = torch.min(boxes1[:, None, 2:], boxes2[:, 2:])
wh = (rb - lt).clamp(min=0)
inter = wh[..., 0] * wh[..., 1]

# Union
area1 = (boxes1[:, 2] - boxes1[:, 0]) * (boxes1[:, 3] - boxes1[:, 1])
area2 = (boxes2[:, 2] - boxes2[:, 0]) * (boxes2[:, 3] - boxes2[:, 1])
union = area1[:, None] + area2 - inter
iou = inter / union

# Enclosing box (smallest box containing both)
lt = torch.min(boxes1[:, None, :2], boxes2[:, :2])
rb = torch.max(boxes1[:, None, 2:], boxes2[:, 2:])
wh = (rb - lt).clamp(min=0)
enc = wh[..., 0] * wh[..., 1]

giou = iou - (enc - union) / enc
```

## 5.3 Hungarian Matching

```python
from scipy.optimize import linear_sum_assignment
row_ind, col_ind = linear_sum_assignment(cost_matrix.cpu())
```

→ 글로벌하게 cost 최소가 되는 1-to-1 매칭 반환.

# 6. Detection Loss 직접 구현 ⭐

## 6.1 Classification Loss

```python
# Hungarian 매칭 결과로 query → target 매핑
target_classes = torch.full(pred_logits.shape[:2], background_class)
target_classes[batch_idx, query_idx] = matched_target_classes

loss_cls = F.cross_entropy(pred_logits.transpose(1, 2), target_classes)
```

→ 매칭 안 된 query는 background로 분류되어야 함.

## 6.2 Box Loss (실제 오브젝트인 query에만)

```python
loss_l1 = F.l1_loss(pred_boxes_matched, target_boxes, reduction='none')

# Box format: cxcywh → xyxy 변환 필요
loss_giou = 1 - torch.diag(generalized_box_iou(
    cxcywh_to_xyxy(pred_boxes_matched),
    cxcywh_to_xyxy(target_boxes)
))
```

## 6.3 Total Loss

\[
L = \lambda_{cls} L_{cls} + \lambda_{L1} L_{L1} + \lambda_{giou} L_{giou}
\]

→ Dictionary로 각 loss 컴포넌트와 total을 반환.

# 7. DETR Attention 시각화 ⭐

## 7.1 Cross-Attention (Decoder Last Layer)

각 object query slot이 이미지의 어디를 보고 detection을 만들었는지:

- 고양이 detection query → 고양이 위치에 attention 집중.
- 리모컨 query → 리모컨 위치.
- 카우치 query → 배경 가구에 attention.

→ **Query들이 알아서 자기 오브젝트를 찾아간다**는 직관적 증거.

## 7.2 Self-Attention (Encoder)

이미지의 한 위치를 query로 잡았을 때 다른 어떤 위치들과 attention이 강한지:

- 고양이 픽셀 query → 다른 고양이 픽셀들도 높은 attention.
- 리모컨 → 다른 리모컨.

→ **같은 오브젝트의 픽셀끼리 묶이는 grouping**이 자연스럽게 일어남.

## 7.3 Query Slot 분포 분석

COCO 검증셋 50장에서 각 object query가 매칭된 box의 중심을 plot:

→ **각 query slot이 이미지의 특정 영역을 담당**하는 패턴이 보임. 어떤 query는 좌상단, 어떤 query는 우하단의 오브젝트를 잘 잡음.

# 8. Deformable Attention 직접 구현 ⭐⭐

## 8.1 DETR의 한계

- Multi-scale 토큰 폭증 → 학습이 며칠 걸림.
- 작은 오브젝트 약함.

해결: **각 query가 K개의 위치만 샘플링**.

## 8.2 Deformable Attention의 5단계 ⭐

```python
def forward(self, query, reference_points, value_feature):
    # 1. Query → Sampling Offsets (학습 가능한 K개 위치)
    offsets = self.offset_proj(query)  # (B, Nq, M*K*2)
    offsets = offsets.view(B, Nq, M, K, 2)
    
    # 2. Query → Attention Weights (softmax)
    attn = self.attn_proj(query)        # (B, Nq, M*K)
    attn = attn.view(B, Nq, M, K).softmax(-1)
    
    # 3. Reference Point + Offset → Sampling Locations
    locations = reference_points[:, :, None, None, :] + offsets
    # [0, 1] → [-1, 1] for grid_sample
    locations = 2 * locations - 1
    
    # 4. Value Projection
    value = self.value_proj(value_feature)
    
    # 5. Bilinear Sampling + Weighted Sum
    sampled = F.grid_sample(value, locations)
    output = (attn[..., None] * sampled).sum(-2)
    
    return self.output_proj(output)
```

## 8.3 일반 Attention과의 비교

| 항목 | 일반 Attention | Deformable Attention |
|---|---|---|
| 각 query가 보는 key | **모든 key** | **K개의 학습된 offset 위치** |
| 복잡도 | \(O(N_q \cdot N_k \cdot d^2)\) | \(O(N_q \cdot K \cdot d^2)\) |
| 학습 컨버전스 | 느림 (500 epoch) | 빠름 (50 epoch) |

→ Deformable Convolution의 attention 버전이라고 생각하면 됨.

# 9. Multi-Scale Deformable Attention ⭐

## 9.1 확장

여러 스케일의 feature map (L개)에서 동시에 샘플링.

```python
# Offset / Attention 출력 채널이 L배
offsets = self.offset_proj(query)  # (B, Nq, M*L*K*2)
attn    = self.attn_proj(query)    # (B, Nq, M*L*K)
attn    = attn.view(B, Nq, M, L*K).softmax(-1)
```

각 scale의 feature map에서 K개씩, 총 \(L \cdot K\)개 위치를 샘플링하여 weighted sum.

## 9.2 효과

- Feature Pyramid Network 같은 외부 모듈 없이도 multi-scale 정보 결합.
- 작은 오브젝트도 큰 feature map에서 잡힘.
- L=1, K=1로 두면 일반 Deformable Convolution이 됨.

# 13. 오후 핵심 정리 ⭐

- **DETR 후처리**는 단순히 logits softmax → max 클래스 + box 변환(`cxcywh → xyxy`)으로 끝나며, **NMS가 전혀 필요 없다**.
- **2D Positional Encoding**은 sine/cosine으로 x·y 방향을 각각 만들어 concat하며, **DETR은 일반 트랜스포머와 달리 PE를 Q·K에만 더하고 모든 layer마다 다시 더한다** (V에는 절대 안 더함).
- **DETR 모델 조립**의 핵심은 ResNet-50의 마지막 2 layer를 제거하고, 1×1 input projection으로 채널을 줄이며, **Object Query는 `nn.Embedding`으로 학습 가능한 토큰**으로 만드는 것이다.
- **GIoU 구현**의 핵심은 IoU에 추가로 두 박스를 모두 포함하는 **enclosing box \(C\)**를 계산하고, \(\text{GIoU} = \text{IoU} - (|C| - |\text{Union}|)/|C|\)로 박스가 안 겹쳐도 gradient flow를 보장하는 것이다.
- **Hungarian Matching**은 `scipy.optimize.linear_sum_assignment`로 cost matrix에서 글로벌 옵티멈 1-to-1 매칭을 찾으며, cost는 classification 확률 + L1 box + GIoU의 가중합이다.
- **Attention 시각화**로 query slot이 자기 오브젝트를 알아서 찾아간다는 것과 self-attention이 같은 오브젝트 픽셀끼리 grouping함을 직접 확인할 수 있다.
- **Deformable Attention의 5단계**는 (1) Offset Projection, (2) Attention Projection + Softmax, (3) Reference + Offset → 샘플링 좌표, (4) Value Projection, (5) Bilinear Sampling + Weighted Sum이며, Deformable Convolution의 attention 버전이다.
- **Multi-Scale Deformable Attention**은 offset/attention 채널을 L배로 늘려 L개 scale에서 각각 K개 위치를 동시 샘플링하며, FPN 없이도 multi-scale 정보를 결합한다.
- **RT-DETR**은 Colab PyTorch 환경에서도 DETR 대비 2배 이상 빠르며(논문은 TensorRT로 3.86배), Ultralytics 패키지로 쉽게 사용 가능하다.
