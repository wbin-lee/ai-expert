# 260604 오전 강의 요약 — Object Detection 2일차 · Focal Loss · FCOS(Anchor-Free) · YOLO v7 · DETR · Deformable DETR · RT-DETR

> 오전 흐름: **RetinaNet의 Focal Loss(어려운 example에 집중하는 cross-entropy 변형) → Feature Pyramid Network(저레벨 큰 feature + 고레벨 작은 feature를 양방향으로 결합) → Dilated Convolution 복습 → FCOS(앵커/프로포저 없이 픽셀마다 prediction, Centerness Loss 도입) → YOLO v7의 엔지니어링 옵티마이제이션(E-ELAN, Compound Scaling, RepConv, Coarse-to-Fine label assignment) → DETR(트랜스포머 기반, Object Query + Bipartite Matching + GIoU, anchor/NMS 모두 제거) → Deformable DETR(query별로 소수의 key 위치만 샘플링하는 Deformable Attention, multi-scale 확장, two-stage variant) → RT-DETR(YOLO를 능가하는 real-time 트랜스포머 디텍터, Efficient Hybrid Encoder의 AIFI+CCFF, Uncertainty-Minimal Query Selection, Flexible Decoder Layer)**
> 강의 주제: **R-CNN/YOLO/SSD 이후 디텍션 알고리즘들이 어떤 방향으로 진화했는지(Focal Loss로 클래스 불균형, FCOS로 anchor 제거, YOLO v7의 엔지니어링 트릭)와 트랜스포머 기반 디텍터의 등장(DETR → Deformable DETR → RT-DETR)이 어떻게 anchor·NMS 같은 hand-crafted 컴포넌트를 제거하며 발전했는지 파악하는 것**.

# 1. Focal Loss (RetinaNet, 2017) ⭐

## 1.1 Cross-Entropy의 한계 ⭐

Cross-entropy는 결국 **정답 클래스의 negative log probability**:

\[
L_{CE} = -\log p_t
\]

문제: 정답 클래스 score를 1까지 높이려고 노력 → 0.9 → 0.95 → 1 만드는 게 어려운데, **그게 정답 맞추는 데 큰 도움도 안 됨**(어차피 argmax가 결정).

→ 정답을 이미 잘 맞추는 easy example에 신경 쓰느라 **틀리거나 애매한 hard example의 학습이 소홀**해진다.

## 1.2 Focal Loss 정의 ⭐

\[
L_{FL} = -(1 - p_t)^\gamma \log p_t
\]

- \(\gamma\) (hyperparameter, 보통 2): focusing parameter.
- \(p_t \to 1\) (잘 맞춤): \((1-p_t)^\gamma \to 0\) → loss가 **급격히 감소**.
- \(p_t \to 0\) (못 맞춤): \((1-p_t)^\gamma \approx 1\) → loss가 거의 그대로.

| pt | γ=0 (CE) | γ=2 (FL) |
|---|---|---|
| 0.1 | 큼 | 거의 그대로 |
| 0.9 | 약간 줄어듦 | **거의 0** |

→ **잘 맞추는 example의 loss를 깎아 hard example에 집중**시키는 효과.

## 1.3 임팩트

Positive example: focal loss는 score 낮은 애들 비율 ↓, 높은 애들 비율 ↑.
Negative example: 더 극명하게 score가 0 쪽으로 몰림 → 미스 클래시피케이션 risk 감소.

⭐ 최근 대부분의 디텍션 논문에서 Focal Loss를 표준으로 채택.

# 2. Feature Pyramid Network ⭐

CNN feature map의 trade-off:

| Feature Map | 사이즈 | 시멘틱 레벨 |
|---|---|---|
| 저레벨 (입력 근처) | 큼 | 낮음 |
| 고레벨 (출력 근처) | 작음 | 높음 |

작은 오브젝트는 큰 feature map이 좋고, 큰 오브젝트는 작은 feature map이 좋다 → **여러 스케일에서 동시에 detection**.

문제: 큰 feature map은 시멘틱 레벨이 낮아 detection 자체에 약함.

해결: **decoder 레이어를 더 쌓아** feature map을 다시 키운 다음 detection.

\[
\text{Bottom-up CNN} \to \text{Top-down upsampling} \to \text{각 scale에서 detection}
\]

→ 시멘틱 레벨도 살리고 사이즈도 다양해진다.

# 3. Dilated Convolution 복습 ⭐

- Receptive field를 키우는 가장 강력한 수단은 **pooling이 아니라 stride**.
- Dilated Convolution: 3×3 필터를 **rate=2로 띄엄띄엄** 적용 → 같은 파라미터로 receptive field 5×5.
- 장점: **feature map size를 줄이지 않으면서** receptive field 키움.
- Semantic Segmentation의 DeepLab에서 도입.

# 4. FCOS (Fully Convolutional One-Stage, 2019) ⭐

## 4.1 핵심 — Anchor-Free, Proposal-Free

Semantic Segmentation처럼 **feature map의 모든 픽셀에서** detection 수행.

```
Feature map의 픽셀마다 → classification + 4개 박스 regression 파라미터
```

장점: 매우 깔끔, anchor/proposal hyperparameter 없음.
단점: feature map의 모든 픽셀이 prediction을 내므로 중복이 많음 → 처리 필요.

## 4.2 Loss 구성

\[
L = L_{cls}(\text{focal loss}) + L_{box}(\text{IoU-based}) + L_{centerness}
\]

- Classification: focal loss를 모든 위치에 적용.
- Box regression: IoU 기반 loss.
- **Centerness Loss**가 핵심 트릭.

## 4.3 Centerness Loss ⭐⭐

각 픽셀이 자신이 prediction한 bounding box의 **중심에 얼마나 가까운지** 추정.

LRTB(좌·우·상·하 거리)에 대해:

\[
\text{Centerness} = \sqrt{\frac{\min(L,R)}{\max(L,R)} \cdot \frac{\min(T,B)}{\max(T,B)}}
\]

- 픽셀이 박스 정중앙: \(L=R, T=B\) → centerness = 1.
- 한쪽에 치우침: min/max 차이 큼 → centerness → 0.

→ Binary cross-entropy로 학습 → "중앙에서 멀어진 prediction은 페널티".

## 4.4 인퍼런스 — NMS 대체

\[
\text{최종 score} = P(\text{class}) \times \text{Centerness}
\]

NMS 비슷한 효과: **확신 있는 + 중앙에 가까운** prediction이 살아남는다.

# 5. YOLO v7 (2023) ⭐ — 엔지니어링의 정수

## 5.1 컨셉

학습 시 리소스를 더 써도 **인퍼런스 비용은 늘리지 않는** 트릭 모음. YOLO v5 대비 같은 AP에서 120% 빠름.

## 5.2 네 가지 컴포넌트

### (a) E-ELAN (Extended Efficient Layer Aggregation)

여러 레이어의 출력을 채널 방향으로 concat하면서 1×1 conv로 bottleneck. Group Convolution + Shuffle + Merge로 cardinality 확장.

### (b) Compound Scaling

Width(채널)와 Depth(레이어)를 **독립적으로 키우는 게 아니라 함께**. Concatenation 기반 모델에서는 width를 키우면 다음 레이어 채널이 달라지므로 1×1 conv로 조정 필요.

### (c) Planned Re-parameterization (RepConv) ⭐

**학습 시**: 3×3 conv + 1×1 conv + identity를 **non-linearity 없이** 병렬 수행 후 합.
**인퍼런스 시**: 셋 다 linear이므로 **하나의 conv로 collapse** 가능 → 학습은 풍부, 인퍼런스는 빠름.

⚠️ ResNet에 RepConv 붙이면 identity가 중복되어 collapse 불가 → `RepConvN`(identity 없는 버전) 사용.

### (d) Coarse-to-Fine Label Assignment

Auxiliary head를 중간 레이어에 추가:

- Lead head(최종): GT 기반 fine label, 3-nearest neighbor 셀.
- Auxiliary head(중간): Lead head의 soft label로 학습, 5-nearest neighbor 셀.

# 6. DETR (DEtection TRansformer, 2020) ⭐⭐

## 6.1 핵심 변화

| 기존 디텍터 | DETR |
|---|---|
| Anchor box / Default box | **Object Query** (learned tokens) |
| Greedy 매칭 + NMS | **Bipartite Matching** (Hungarian Algorithm) |
| Hand-crafted anchor | End-to-end set prediction |
| NMS 후처리 | **No NMS** |

## 6.2 Object Query ⭐

- 약 100개의 **학습 가능한 더미 토큰**.
- 학습 후 각 query는 "오브젝트가 어디 있을 것 같은지"에 대한 prior를 가짐.
- 트레이닝 시: 더미로 시작해 같이 업데이트.
- 인퍼런스 시: 학습된 query를 모든 이미지에 그대로 사용.

## 6.3 Bipartite Matching (헝가리안 알고리즘) ⭐⭐

100개의 object query 출력과 GT bounding box를 **1-to-1로 매칭**.

**Matching Cost**:

\[
\mathcal{L}_{\text{match}}(y_i, \hat{y}_{\sigma(i)}) = -\mathbb{1}_{\{c_i \neq \emptyset\}} \hat{p}_{\sigma(i)}(c_i) + \mathbb{1}_{\{c_i \neq \emptyset\}} \mathcal{L}_{\text{box}}(b_i, \hat{b}_{\sigma(i)})
\]

\[
\mathcal{L}_{\text{box}} = \lambda_{L1} \| b - \hat{b} \|_1 + \lambda_{\text{iou}} (1 - \text{GIoU}(b, \hat{b}))
\]

- 두 가지 기준: **클래스 확률이 높고** + **박스가 GT와 가까워야**.
- 전체 매칭 cost가 최소가 되도록 글로벌 옵티멈을 찾음(헝가리안 알고리즘).

⚠️ 매칭 단계 자체는 backprop 안 함. 현재 모델 파라미터로 cost 계산 → 매칭 → 그 매칭을 기준으로 detection loss 백프롭.

## 6.4 GIoU (Generalized IoU) ⭐⭐

\[
\text{GIoU}(A, B) = \text{IoU}(A, B) - \frac{|C \setminus (A \cup B)|}{|C|}
\]

여기서 \(C\)는 \(A, B\)를 모두 포함하는 **최소 enclosing box**.

| 상황 | IoU | GIoU |
|---|---|---|
| 박스가 일부 겹침 | 0 < IoU < 1 | 비슷 |
| **박스가 안 겹침, 가까움** | **0** | 약간 음수 |
| **박스가 안 겹침, 멈** | **0** | 더 큰 음수 |

→ IoU=0인 박스 둘 사이의 거리 차이를 학습 시그널로 줄 수 있음. Gradient flow 보장.

## 6.5 Detection Loss

매칭 결과 \(\hat{\sigma}\)에 대해:

\[
\mathcal{L}_{\text{Hungarian}}(y, \hat{y}) = \sum_{i=1}^{N} \left[ -\log \hat{p}_{\hat{\sigma}(i)}(c_i) + \mathbb{1}_{\{c_i \neq \emptyset\}} \mathcal{L}_{\text{box}}(b_i, \hat{b}_{\hat{\sigma}(i)}) \right]
\]

- Matching cost와 거의 같지만 **확률 대신 log probability** 사용.
- Classification은 background 포함 모든 박스에 적용.
- Box loss는 실제 오브젝트인 박스에만 적용.

## 6.6 아키텍처 ⭐

```
입력 이미지
    → CNN backbone (ResNet) → feature map (32× downsample, 2048 ch)
    → 2D Positional Encoding 더하고
    → Transformer Encoder (N개 layer, self-attention)
    → Transformer Decoder (N개 layer, cross-attention with object queries)
    → FFN (Feed Forward Network, 3-layer MLP + ReLU)
    → N개의 prediction (class + bounding box)
```

특징:
- 2D Positional Encoding (sine/cosine, x·y 방향 각각).
- Object queries는 learned positional encoding으로 추가.
- Decoder는 **auto-regressive 아님** — 모든 query를 병렬 처리.

# 7. Deformable DETR (2021) ⭐

## 7.1 DETR의 약점

- 학습 수렴 매우 느림 (500 epoch — Faster R-CNN은 52 epoch).
- 작은 오브젝트 detection 약함.
- Multi-scale feature를 쓰면 토큰 수 폭증 → 더 느림.

## 7.2 Deformable Attention ⭐⭐

기존 attention: 각 query가 **모든 key**와 outer product → \(O(N^2)\).
**Deformable**: 각 query가 **K개의 샘플링된 위치**의 key와만 attention → \(O(NK)\).

\[
\text{DeformAttn}(z_q, p_q, x) = \sum_{m=1}^{M} W_m \left[ \sum_{k=1}^{K} A_{mqk} \cdot W'_m x(p_q + \Delta p_{mqk}) \right]
\]

- \(z_q\): query feature.
- \(p_q\): reference point (현재 query의 위치).
- \(\Delta p_{mqk}\): query feature에서 linear projection으로 학습되는 **offset**.
- \(A_{mqk}\): attention weight (query feature를 linear projection + softmax).

→ Deformable Convolution의 attention 버전. **각 query가 자기 위치에서 어디를 봐야 하는지 학습**.

## 7.3 Multi-Scale Deformable Attention

여러 스케일의 feature map (L개)을 동시에 사용. 총 \(L \cdot K\)개 위치 샘플링 → feature pyramid 없이도 multi-scale 정보 결합.

## 7.4 추가 트릭

| 기법 | 효과 |
|---|---|
| **Iterative Bounding Box Refinement** | 각 decoder layer마다 박스를 점진적으로 refine |
| **Two-Stage Variant** | Encoder에 detection head를 달아 high-score 위치를 object query로 사용 (DETR의 learned query 대신) |

## 7.5 결과

- 학습 epoch 10배 단축.
- 작은 오브젝트 AP 크게 향상.
- Two-stage variant가 추가 성능 향상.

# 8. RT-DETR (Real-Time DETR) ⭐

## 8.1 동기 — YOLO의 약점 공략

YOLO의 NMS 문제:

- Confidence/IoU threshold라는 hyperparameter 필요.
- NMS 시간이 입력마다 다름.
- Anchor-based detector는 anchor-free의 **3배 많은 박스**를 생성 → NMS 시간 증가.

→ DETR의 NMS-free 장점을 살리면서 YOLO보다 빠르게 만들자.

## 8.2 Efficient Hybrid Encoder ⭐

관찰: 저레벨 feature에서 intra-scale interaction은 별 도움 안 됨.

두 모듈로 분리:

| 모듈 | 역할 |
|---|---|
| **AIFI** (Attention-based Intra-scale Feature Interaction) | 가장 high-level feature(S5)에서만 self-attention |
| **CCFF** (CNN-based Cross-scale Feature Fusion) | PaNet 스타일로 S3/S4/F5 결합. 1×1 conv + RepConv 블록 |

→ Encoder 49% GFLOPs 절감, AP 1.1만 떨어짐.

## 8.3 Uncertainty-Minimal Query Selection ⭐

Two-stage DETR의 query selection 개선:

- 단순히 classification score만 보면 localization이 약함.
- **Classification score**와 **Localization score**의 차이를 **uncertainty**로 모델링하고 minimize.
- 두 score가 align되어 있는 (classification 확신 = localization 정확) feature를 우선 선택.

→ High-quality feature가 120% 더 많이 선택, AP 0.8 향상.

## 8.4 Flexible Decoder Layer

Decoder의 모든 layer에 detection head 부착 (auxiliary heads).
인퍼런스 시 layer 수를 줄여도 (예: 6 → 5) AP는 0.1만 하락 → **속도에 따라 유연한 trade-off**.

## 8.5 결과

| 모델 | FPS |
|---|---|
| DETR | 28 |
| **RT-DETR** | **108** (약 4배) |

→ YOLO v5/v6/v7/v8보다 빠르고 정확.

# 9. 오전 핵심 정리 ⭐

- **Focal Loss**(\(L = -(1-p_t)^\gamma \log p_t\))는 cross-entropy에 \((1-p_t)^\gamma\)를 곱해 easy example의 loss를 급격히 줄이고 **hard example에 학습 집중**시키는 표준 기법이 되었다.
- **FCOS**는 anchor/proposal 없이 feature map의 모든 픽셀에서 detection을 하고, **Centerness Loss**(현재 픽셀이 박스 중앙에 얼마나 가까운지)를 추가해 NMS 비슷한 효과를 모델 내부에서 처리한다.
- **YOLO v7**은 새로운 detection 알고리즘이 아니라 **엔지니어링 옵티마이제이션 모음**(E-ELAN, Compound Scaling, RepConv, Coarse-to-Fine label assignment)이며, **학습 비용은 늘려도 인퍼런스 비용은 안 늘린다**는 철학을 따른다.
- **DETR**의 핵심 변화는 (1) **Object Query**(learned tokens)로 anchor 대체, (2) **Hungarian Bipartite Matching**으로 NMS 제거, (3) **GIoU**(non-overlap 박스에도 gradient flow 보장)로 box loss 안정화 — 세 가지가 한 묶음으로 end-to-end set prediction을 가능하게 한다.
- **GIoU**는 IoU에 enclosing box \(C\)에서 union을 뺀 영역의 비율을 페널티로 추가해, IoU=0인 박스 쌍의 거리까지 학습 시그널로 줄 수 있게 한다.
- **Deformable DETR**의 핵심은 각 query가 모든 key 대신 **K개의 학습된 offset 위치만 attention**하는 Deformable Attention이며, 이를 multi-scale로 확장해 작은 오브젝트도 잘 잡고 학습 속도도 10배 향상시킨다.
- **RT-DETR**은 (1) **AIFI**(top-level에서만 self-attention) + **CCFF**(cross-scale fusion)의 Hybrid Encoder, (2) classification/localization uncertainty를 align하는 **Query Selection**, (3) decoder layer 수를 flexibly 줄일 수 있는 구조로 YOLO를 능가하는 real-time 트랜스포머 디텍터가 되었다.
