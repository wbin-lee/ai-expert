# 260522 오전 강의 요약 — Maximum Entropy RL · Exploration vs Exploitation · Soft Q / Soft Value · Soft Policy Iteration · Soft Actor-Critic (SAC) · Reparametrization Trick · Model-Based RL 도입

> 오전 흐름: **TRPO/PPO 복습 → Policy Gradient 계열의 공통 단점 (휴이스틱 Exploration) → Exploration-Exploitation 딜레마와 일상 예시 → 다섯 가지 Exploration 카테고리 (ε-greedy / OFU / Thompson Sampling / Noisy Network / Information Theoretic) → Policy Entropy 정의 → Maximum Entropy MDP의 변형된 objective → Soft Q-function, Soft Value Function 정의 (Soft Max로의 일반화) → Soft Bellman Equation과 Soft Policy Iteration → Information Projection을 통한 Policy Improvement → 모델 프리화: Soft Actor-Critic (SAC) → Critic 설계 (DQN 아이디어 + Entropy 항) → Actor 설계와 Reparametrization Trick → SAC의 Off-Policy 특성과 Sample Efficiency → 실험 결과 (Humanoid, 사족보행 로봇, Robust Visual Policy) → Model-Based RL 도입 (Model-Free와의 차이 / Optimal Control / Trajectory Optimization)**
> 강의 주제: **TRPO·PPO에서 휴이스틱에 머물렀던 Exploration을 Maximum Entropy 원리로 더 체계화하고, 이를 구현한 Off-Policy 알고리즘 SAC를 끝까지 따라가본다. 이어서 Model-Free에서 Model-Based RL로 관점을 전환하여 옵티멀 컨트롤·시스템 식별·MPC의 골격을 잡는다.**

# 1. 전날 내용 복습 — TRPO와 PPO

## 1.1 TRPO 정리

TRPO는 일반 policy gradient를 따라가지 않고, surrogate maximization을 근사적으로 푸는 방식이다.

\[
\theta_{\text{new}} = \theta_{\text{old}} + \sqrt{\frac{2\delta}{g^\top H^{-1}g}} \cdot H^{-1} g
\]

- `H^{-1} g` 가 natural policy gradient 방향.
- 앞의 root term이 effective step size.
- 그러나 full step 그대로 가면 monotonic improvement / trust region constraint가 깨지므로 **inner loop의 line search**로 보정.
- 전체적으로는 on-policy지만, inner loop에서는 importance sampling 덕분에 추가 sampling 불필요.

## 1.2 PPO 정리

TRPO의 가장 큰 bottleneck은 Hessian inverse. PPO는 그것을 피한다.

- 1차 PPO: KL을 다시 penalty로 올린 unconstrained 문제.
- 2차 PPO (실제로 더 많이 쓰임): probability ratio를 직접 clip.

\[
L^{CLIP}(\theta)=\mathbb{E}\left[\min(r_t(\theta)\hat A_t,\ \mathrm{clip}(r_t,1-\epsilon,1+\epsilon)\hat A_t)\right]
\]

핵심 동기: probability ratio가 1로부터 너무 벗어날 incentive를 objective 단계에서 제거 = trust region 효과를 흉내.

## 1.3 advantage 추정 — GAE

advantage 추정에서 GAE를 활용하면 Monte Carlo와 1-step TD 양극단 사이에서 **bias-variance balancing**이 가능하다.

# 2. Policy Gradient 계열의 공통 단점

## 2.1 핵심 문제

TRPO / PPO / GRPO에는 공통적인 두 가지 한계가 있다.

| # | 문제 |
|---|---|
| 1 | On-policy 특성이 여전히 존재 |
| 2 | **Exploration이 휴이스틱하다** |

데이터는 결국 현재 policy로 모은다. randomize policy 덕분에 어느 정도 randomness는 있지만 **정보 취득에 최적화된 noise**는 아니다.

## 2.2 다른 알고리즘들의 휴이스틱

| 알고리즘 | Exploration 방식 |
|---|---|
| DQN | ε-greedy |
| DDPG | deterministic policy에 인위적 Gaussian noise |
| TRPO / PPO | randomize policy의 internal randomness |

⭐ 모두 **provably inefficient**한 단순 휴이스틱.

# 3. Exploration vs Exploitation 딜레마

## 3.1 본질

| 개념 | 의미 |
|---|---|
| **Exploitation** | 지금까지 갖고 있는 정보로 당장 이득이 되는 의사결정 |
| **Exploration** | 당장 이득은 적지만 정보 취득을 위해 새로운 행동 |

> ⭐ RL뿐 아니라 인간의 거의 모든 의사결정이 둘 사이의 밸런싱.

## 3.2 일상 예시

- **식당 선택**: 단골 vs 새 식당.
- **온라인 광고**: 가장 클릭률 높은 광고 vs 다른 결의 광고 — Multi-Armed Bandit 알고리즘이 이 trade-off를 푼다.
- **기름 시추 / 게임 플레이 / 우주 탐사** 등.

## 3.3 알고리즘적으로 해결하려는 5가지 카테고리

| # | 방법 | 핵심 |
|---|---|---|
| 1 | **ε-greedy** | 단순, 휴이스틱, provably inefficient |
| 2 | **Optimism in the Face of Uncertainty (OFU)** | Confidence Set 안에서 가장 긍정적인 모델 선택 |
| 3 | **Thompson Sampling** | Bayesian, posterior에서 MDP 모델을 샘플링 |
| 4 | **Noisy Networks** | 네트워크 weight 자체를 perturb (휴이스틱) |
| 5 | **Information Theoretic Exploration** | 오늘 핵심 — **High Entropy Policy** 활용 |

# 4. 다섯 가지 Exploration 카테고리 상세

## 4.1 ε-greedy

\[
a_t = \begin{cases} \arg\max_a Q(s_t,a) & \text{with prob } 1-\epsilon \\ \text{random} & \text{with prob } \epsilon \end{cases}
\]

- ε 스케줄링: 초반엔 크게, 후반엔 작게.
- 장점: 구현 쉬움, 오버헤드 작음.
- 단점: 정보 취득에 타겟팅이 없음 — 이미 가본 액션도 다시 랜덤 선택 가능.

## 4.2 OFU (Optimism in the Face of Uncertainty)

3 stage 알고리즘:

1. 현재 샘플로 MDP 파라미터의 **Confidence Set** 생성.
2. ⭐ Confidence Set 안에서 **누적 reward를 최대화하는 가장 긍정적인 모델** 선택.
3. 선택된 모델에서 value/policy iteration으로 optimal policy 계산.

- 장점: **Regret-optimal** (수학적으로 후회량을 0으로 수렴시킬 수 있음).
- 단점: **2단계가 non-convex 최적화** → 복잡한 task에서는 풀기 어려움.

## 4.3 Thompson Sampling

Bayesian 방식. 모델 파라미터에 대한 prior → posterior 업데이트 → posterior에서 **MDP 모델을 샘플링** → 그 모델의 optimal policy 계산 → 데이터 추가 수집 → 반복.

- 장점: OFU와 마찬가지로 Regret-optimal, 단 **계산적으로 OFU보다 훨씬 유리**.
- 단점: posterior가 conjugate가 아니면 업데이트 까다로움.
- 활용: 온라인 광고에서 가장 많이 쓰이는 방법.

## 4.4 Noisy Networks

네트워크 weight 자체에 Gaussian noise를 주입.

- 장점: 구현 단순.
- 단점: 휴이스틱, 안 통하는 task도 많음.

## 4.5 Information Theoretic — 오늘의 주제

**High Entropy Policy**를 학습 과정에서 활용해 정보 취득을 극대화. 이론적 Regret 보장은 떨어지지만 실험적으로 매우 강한 성능.

# 5. Entropy의 정의와 Policy Entropy

## 5.1 정보이론적 정의

discrete 확률 분포 \(p\)에 대해:

\[
H(p) = -\sum_i p_i \log p_i = \mathbb{E}_p[-\log p]
\]

물리적 직관: 엔트로피가 낮으면 정돈된 상태, 높으면 disorder.

## 5.2 Policy Entropy

randomize policy \(\pi(\cdot|s)\)의 엔트로피:

\[
H(\pi(\cdot|s)) = -\sum_a \pi(a|s)\log\pi(a|s) = \mathbb{E}_{a\sim\pi}[-\log\pi(a|s)]
\]

- 엔트로피 높음 → 행동 분포가 평평 → 다양한 액션 시도 가능.
- 엔트로피 낮음 → 한 액션에 집중 → exploitation.

> ⭐ high-entropy policy를 학습하면 자연스럽게 exploration이 유도된다.

# 6. Maximum Entropy MDP

## 6.1 변형된 목적함수

일반 MDP는 누적 reward만 최대화한다. Maximum Entropy MDP는 **누적 reward와 policy entropy의 가중합**을 최대화한다.

\[
J(\pi) = \mathbb{E}_\pi\left[\sum_t \gamma^t\left(r_t + \alpha H(\pi(\cdot|s_t))\right)\right]
\]

- α: **temperature**, entropy의 가중치.
- α → 0이면 일반 MDP와 동일.

## 6.2 Soft Q-function과 Soft Value Function

| 함수 | 정의 |
|---|---|
| **Soft Q-function** \(Q^\pi\) | 현재 reward + 다음 스텝부터의 누적 reward + entropy term |
| **Soft Value Function** \(V^\pi\) | \(\alpha \log \int e^{Q^\pi/\alpha}da\) (LogSumExp 형태) |

V의 정의가 LogSumExp이므로:

\[
V^\pi(s) = \alpha \log \int_a \exp\!\left(\frac{Q^\pi(s,a)}{\alpha}\right) da
\]

이는 **soft maximum**이다. α → 0에서 일반 max로 수렴.

## 6.3 Optimal Counterpart

\[
Q^*(s,a),\quad V^*(s) = \alpha\log\int e^{Q^*(s,a)/\alpha}da
\]

# 7. Soft Policy Iteration

## 7.1 Soft Policy Improvement

old policy의 soft Q를 계산했을 때 다음 polices가 **반드시 더 좋다**:

\[
\pi_{\text{new}}(a|s) = \exp\!\left(\frac{Q^{\pi_{\text{old}}}(s,a) - V^{\pi_{\text{old}}}(s)}{\alpha}\right)
\]

> ⭐ 일반 policy improvement는 max 액션을 고르지만, **soft 버전은 max 자체가 빠지고** exponential 형태로 다중 액션에 확률을 분배한다.

## 7.2 Soft Bellman Equation

\[
Q^*(s,a) = r(s,a) + \gamma \mathbb{E}_{s'}[V^*(s')]
\]

여기서 \(V^*\)는 일반 max가 아닌 **soft max (LogSumExp)** 로 계산된다는 점만 차이.

> ⭐ ⭐ ⭐ "Maximum operator를 SoftMax로 교체하면 Maximum Entropy 버전이 된다" — **구조적 유사성**이 핵심.

## 7.3 정책 구축 비교

| 일반 MDP | Maximum Entropy MDP |
|---|---|
| \(\pi^*(s) = \arg\max_a Q^*(s,a)\) | \(\pi^*(a\|s) \propto \exp(Q^*(s,a)/\alpha)\) |
| Deterministic | Stochastic |
| 다른 액션은 절대 못 뽑음 | 차상위 액션도 일정 확률로 뽑힘 → exploration |

## 7.4 Maximum Entropy의 3대 장점

| 장점 | 의미 |
|---|---|
| **계산적** | Bellman equation과 policy construction에서 max operator가 빠져 closed-form |
| **Exploration** ⭐ | 액션 다변화가 자연스럽게 일어남 |
| **구조적 유사성** | 기존 RL 알고리즘의 max를 softmax로 바꾸기만 하면 maximum entropy 변형 알고리즘이 됨 |

# 8. Soft Actor-Critic (SAC)

## 8.1 위치

- 출발점: **Maximum Entropy MDP**.
- 결합: **Off-Policy Actor-Critic** (DDPG의 친척).
- 목표: Exploration + Sample Efficiency + Stability + Hyperparameter 강건성.
- 부제: *"Off-Policy Maximum Entropy Deep Reinforcement Learning with a Stochastic Actor"*.

## 8.2 DDPG와의 차이

| 항목 | DDPG | SAC |
|---|---|---|
| Policy | Deterministic | **Stochastic** (entropy maximization 필요) |
| Critic | Q-function regression | **Soft** Q-function (entropy term 포함) |
| Actor | Deterministic policy gradient | **Information projection** via reparametrization |

## 8.3 Soft Critic 설계

Q-network \(Q_\phi\), Target Q-network \(Q_{\phi^-}\)를 둔다.

- Replay buffer에서 (s, a, r, s')를 sampling.
- Target 계산 시 **반드시 entropy term을 포함**해야 한다.

\[
y = r + \gamma\left( Q_{\phi^-}(s', a') - \alpha \log \pi(a'|s')\right),\quad a'\sim\pi
\]

DQN과 결합 가능한 이유: max entropy 변형에서는 target 계산 시 적분이 자동으로 풀려 stochastic gradient로 처리 가능.

## 8.4 Soft Actor 설계와 Reparametrization Trick

목표는 information projection:

\[
\min_\theta \mathbb{E}_{s\sim D}\,D_{KL}\!\left(\pi_\theta(\cdot|s)\ \Big\|\ \frac{\exp(Q^\pi(s,\cdot)/\alpha)}{Z(s)}\right)
\]

문제: action이 randomize policy로 샘플링되므로 θ에 대한 gradient의 variance가 크다.

해결: **Reparametrization Trick**.

\[
a = f_\theta(\epsilon, s),\quad \epsilon\sim\mathcal{N}(0, I)
\]

즉 action을 (1) deterministic 함수와 (2) 외부 noise의 곱으로 분리해서 표현. θ에 대한 gradient가 깔끔하게 떨어진다.

## 8.5 SAC가 Off-Policy인 이유

- Critic: DQN 기반이라 자명하게 off-policy.
- Actor: 핵심 — replay buffer에서 **state만 재샘플링**하고 **action은 현재 stochastic actor로 새로 뽑는다**. 추가 sampling 없이 자연스러운 off-policy 작동.

## 8.6 Temperature α의 학습 (자동 조절)

원 SAC는 α에 민감하다. 후속 논문에서 α도 학습한다.

\[
\max_\pi \mathbb{E}\left[\sum r_t\right]\quad \text{s.t.}\quad H(\pi) \ge \bar H
\]

Lagrangian으로 풀면 α는 다음 loss로 업데이트:

\[
L(\alpha) = -\alpha\left(\log\pi(a|s) + \bar H\right)
\]

타겟 엔트로피는 보통 \(\bar H = -|\text{action dim}|\)로 설정 (직관: action dim이 클수록 자연스럽게 entropy 값이 작아지므로 target도 같이 작아져야 함).

# 9. SAC 실험 결과

## 9.1 Mujoco 벤치마크

- DDPG, PPO, TD3 모두 능가.
- Humanoid 같은 어려운 환경도 학습 가능 (DDPG는 거의 불가).

## 9.2 Robot Valve Rotation (Visual)

- 픽셀 입력만으로 손가락 로봇이 벨브를 돌리는 task.
- ⭐ Robustness가 강함:
  - 사람이 손으로 벨브를 흔들어도 다시 돌려놓음.
  - 배경에 perturbation이 들어와도 어느 정도 강건.
- 한계: 카메라를 완전히 가리면 generalization 범위를 벗어나 실패.

## 9.3 사족보행 로봇 (Off-Policy의 진가)

- TRPO는 on-policy라 시뮬레이터 필수.
- SAC는 off-policy → **시뮬레이터 없이 실물 로봇에서 직접 try-and-error로 약 2시간 만에 학습 완료**.
- 처음 본 지형(계단·기타)에도 어느 정도 robust.

> ⭐ Robustness의 직관: SAC는 stochastic actor를 학습하므로 Plan A뿐 아니라 Plan B, C도 확률적으로 학습된다. 그래서 상황이 조금 바뀌어도 대응 가능.

# 10. Model-Based RL로의 전환

## 10.1 Model-Free vs Model-Based

| 종류 | 직접 학습하는 대상 | 예시 |
|---|---|---|
| **Model-Free** | policy 또는 value만 학습. transition probability는 학습 X | Q-Learning, DQN, DDPG, TRPO, PPO, SAC |
| **Model-Based** | transition probability(모델)도 학습하고, 그것을 사용해 policy도 학습 | OFU, Thompson Sampling, MPC 계열 |
| **Classical** | 시스템 식별 → 모델 기반 컨트롤러 설계 | 산업 현장 표준 |

> ⭐ Classical Method 단점: 환경이 변하면 시스템 식별과 컨트롤러 설계를 처음부터 다시 해야 함.

## 10.2 Model-Free RL의 장단점

| 측면 | 내용 |
|---|---|
| 장점 | Domain knowledge 없이도 적용 가능, flexible |
| 단점 | 학습이 잘 안 될 때 원인 분석 (debug)이 어려움 |

⭐ End-to-end RL이 자율주행 같은 safety-critical에서 그대로 쓰이지 않는 이유. 사고 발생 시 어디가 문제인지 pinpoint하기 어려움. Tesla FSD조차 일부 모듈만 RL, 대부분은 imitation learning.

## 10.3 Model-Based RL의 우위

- **검증 가능성**: 중간에 모델이 존재 → 사고/실패 시 policy인지 모델인지 분리 분석 가능.
- **Domain knowledge 결합**: 물리법칙으로 transition probability의 구조를 잡고, 파라미터만 학습 → **sample efficiency 향상**.

## 10.4 Deterministic Optimal Control 문제

\[
\max_{a_{0:T-1}} \sum_t r_t,\quad s_{t+1}=f_\theta(s_t,a_t)
\]

deterministic 시스템에서는 **action trajectory 자체를 최적화**해도 된다 (stochastic 시스템에서는 안 됨 — perturbation 때문에).

## 10.5 두 가지 관점

| 관점 | 방식 | 특징 |
|---|---|---|
| **Dynamic Programming** | optimal policy를 직접 학습 | 글로벌 최적, scalability 약함 (curse of dimensionality) |
| **Trajectory Optimization** | action sequence를 직접 최적화 | 로컬 최적, scalable, 그러나 **open-loop** |

Trajectory Optimization은 perturbation에 취약 — 미리 계획한 action sequence가 더 이상 optimal하지 않을 수 있다.

> ⭐ Model-Based RL의 일반적 절차: trajectory optimization으로 데이터를 빠르게 만들되, 최종 산물은 **closed-loop controller (policy)** 형태로 가져가는 것이 이상적.

# 11. 오전 핵심 정리

## 11.1 큰 그림

- Policy Gradient 계열은 강력하지만 **Exploration이 휴이스틱**.
- 다섯 Exploration 카테고리 중 **Maximum Entropy 방식**이 오늘의 주제.
- 일반 MDP에서 **max → soft max (LogSumExp)** 로 바꾸면 Maximum Entropy MDP가 된다.

## 11.2 Maximum Entropy MDP

- Objective: \(\sum r_t + \alpha H(\pi)\).
- Soft Q, Soft V, Soft Bellman equation.
- Soft Policy Iteration이 일반 Policy Iteration의 자연스러운 확장.

## 11.3 SAC 한 줄 요약

> ⭐ **SAC = Soft Policy Iteration + DDPG (Off-Policy Actor-Critic) + Stochastic Actor with Reparametrization Trick**

- Exploration이 알고리즘 내부에서 자연스럽게 발생 (별도 noise 주입 불필요).
- Off-Policy로 sample efficiency 확보.
- Robustness가 강해 시뮬레이터 없이 실물 로봇 학습도 가능.

## 11.4 Model-Based RL 도입

- Model-Free와의 본질적 차이: **transition probability를 학습하느냐**.
- Trajectory Optimization (open-loop) vs Dynamic Programming (closed-loop).
- 산업계 표준 (classical)에서 RL로 가는 가교 역할.

## 11.5 시험 안내 (강사 멘트)

- 답안에 수식 하나도 쓸 필요 없음.
- 각 알고리즘이 **무엇을 학습하고 왜 효율적이며 어떤 상황에 적합한지**가 핵심.
- "max → soft max로 바뀌었다", "off-policy actor-critic이라 sample efficiency가 좋다"처럼 **하이레벨 직관**으로 충분.
