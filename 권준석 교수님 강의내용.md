# 1부. 260624 오전 — 디퓨전 모델의 탄생과 큰 그림

## 1-1. 이 수업에서 다루는 것

권준석 교수님은 중앙대 소프트웨어학부에서 디퓨전 모델을 가르칩니다. 디퓨전은 수학·물리에 깊게 뿌리내린 모델이지만, 이 수업은 **개념 중심**으로 진행됩니다.

강의 로드맵은 대략 이렇습니다.

1. 디퓨전 모델의 역사와 개요
2. 전체 파이프라인(입력 → 과정 → 출력)
3. 아키텍처(UNet 등)
4. Stable Diffusion과 프롬프팅
5. 하이퍼파라미터 조절
6. 최근 변형과 산업 응용(반도체 결함 검출 등)

반도체 현장에서는 **결함(defect) 데이터가 부족**합니다. 정상 웨이퍼는 많지만 불량 샘플은 드물어서, 생성형 AI로 합성 데이터를 만들어 검출기를 학습시키는 것이 실무적 동기가 됩니다.

## 1-2. 생성형 AI의 짧은 역사

2022년 ChatGPT가 등장하면서 생성형 AI가 대중화되었습니다. 이어서 DALL·E, Sora 같은 영상 생성, 그리고 **Stable Diffusion**이 나오면서 이미지 생성이 일상화되었습니다.

Stable Diffusion이 특별했던 이유는 두 가지입니다.

- **경량화**: 이전 디퓨전은 A100·H100급(80GB VRAM) GPU가 필요했는데, Stable Diffusion은 8GB VRAM의 일반 PC GPU에서도 돌아갑니다.
- **오픈소스**: 가중치와 소스 코드를 공개해 누구나 파인튜닝(LoRA, DreamBooth 등)할 수 있게 되었습니다.

> ⭐ **시험 포인트**  
> Stable Diffusion = **VAE(잠재 공간 압축) + 디퓨전(잠재 공간에서 노이즈 제거)**.  
> 핵심 아이디어는 **픽셀 공간이 아니라 레이턴트(latent) 공간**에서 디퓨전을 수행하는 것.

## 1-3. GAN·VAE의 한계 → 디퓨전의 등장

생성 모델의 역사를 짧게 보면:

| 모델 | 특징 | 한계 |
|------|------|------|
| Autoencoder (2006) | 입력을 잠재 벡터 Z로 압축 후 복원 | 생성이 아니라 **복사**에 가깝다 |
| VAE (2013) | Z를 확률분포(가우시안)로 모델링 → 샘플링해 생성 | 이미지가 **흐릿(blur)** 해지기 쉽다 |
| GAN | Generator vs Discriminator 대결 | **Mode collapse** — 다양성 없이 한 종류만 잘 만든다 |
| Diffusion (2020~) | 노이즈를 점진적으로 제거하며 복원 | 느리고 메모리를 많이 쓰지만 **수렴·품질**이 안정적 |

GAN은 “판별자를 속이기 가장 쉬운 데이터 하나”에 집착해 **다양성**을 잃습니다. VAE는 한 번에 복원하지만, 디퓨전은 **여러 단계(T step)** 를 거쳐 복원하므로 더 정밀해질 수 있습니다.

디퓨전의 장점은 GAN과 달리 **물리 법칙(확산 과정)을 따르므로 수렴이 보장**된다는 점입니다. 단점은 **느리다**는 것 — 수백~수천 스텝의 반복 연산이 필요합니다.

## 1-4. 물리 비유: 열역학 제2법칙을 거꾸로

디퓨전 모델의 직관은 물에 물감 한 방울을 떨어뜨리는 것과 같습니다.

- **순방향(Forward)**: 깨끗한 데이터 → 점점 노이즈가 섞여 **엔트로피(무질서도)가 증가** → 완전한 가우시안 노이즈
- **역방향(Backward)**: 노이즈 → 단계마다 노이즈를 조금씩 제거 → **원래 데이터와 유사한 이미지** 복원

자연은 항상 무질서도가 커지는 방향(열역학 제2법칙)으로 흐릅니다. 디퓨전은 이 과정을 **거꾸로 학습**하는 것입니다. 학습 데이터에서 “데이터→노이즈” 경로들을 모아 통계적으로 이해한 뒤, 역으로 “노이즈→데이터” 경로를 따라갑니다.

## 1-5. 레이턴트 스페이스가 모든 것을 바꿨다

Stable Diffusion 이전의 바닐라 디퓨전(DDPM 등)은 **픽셀 공간**에서 모든 연산을 했습니다. 512×512×3 이미지는 약 **78만 차원**의 고차원 공간이고, 여기에 1000 스텝 반복 연산을 하면 비용이 폭발합니다.

Stable Diffusion은 **VAE 인코더**로 이미지를 64×64 크기의 잠재 공간으로 압축합니다(약 **48배** 압축). 디퓨전 연산은 이 저차원 공간에서만 일어나고, 마지막에 **VAE 디코더**가 픽셀 이미지로 복원합니다.

레이턴트 공간의 특징:

- 자연 이미지는 고차원이지만 실제 정보는 **작은 매니폴드(manifold)** 에 살아 있다
- 나뭇잎·구름처럼 **반복 패턴(redundancy)** 이 많아 압축 가능
- 불필요한 배경·노이즈를 제거하고 **형태·구도·색상** 같은 본질만 남긴다
- **다른 모달리티(텍스트, 음성 등)도 같은 잠재 공간으로 옮겨** 크로스 도메인 생성이 가능해진다

> ⭐ **시험 포인트**  
> 레이턴트 스페이스를 쓰면 **프롬프트(텍스트 등)를 주입·조작**하기 쉽다.  
> 텍스트 임베딩과 레이턴트 벡터를 **cross-attention**으로 결합한다.

## 1-6. 블랙박스 문제와 “컨트롤”의 필요성

생성형 AI는 입력→출력은 보이지만 **중간 과정을 이해하기 어렵다**. 그래서:

- **Hallucination**: 논리·물리 법칙을 어기는 결과
- **일관성 붕괴**: 영상에서 얼굴이 바뀌거나 발이 땅을 뚫는 현상
- **통제 불가**: 원하는 포즈·스타일·브랜드 아이덴티티를 맞추기 어렵다

이를 해결하기 위해 **프롬프트 입력**과 **하이퍼파라미터 조절**이라는 두 가지 컨트롤 방법이 발전했습니다. 이후 강의의 핵심 주제가 됩니다.

# 2부. 260624 오후 — 파이프라인, UNet, 픽셀 vs 레이턴트, 조건부 생성

## 2-1. 디퓨전 모델이 하는 일: 노이즈 예측

디퓨전 모델의 목표는 “이미지를 직접 복원”하는 것이 아니라, **각 시점 t에서 제거해야 할 노이즈(에러 맵)를 예측**하는 것입니다.

- **입력**: 현재 시점 t의 노이즈 이미지 $x_t$
- **출력**: 노이즈 맵(에러 맵) — 이미지와 **같은 크기**여야 1:1로 빼기 가능
- **학습**: 예측 노이즈와 실제 노이즈의 **L2(MSE) 차이**를 최소화

> ⭐ **시험 포인트**  
> 이미지 복원보다 **노이즈 예측**이 쉽다.  
> 이미지는 고차원, 노이즈는 저차원·가우시안이라 구조가 단순하다.

## 2-2. UNet: 디퓨전의 백본

노이즈 예측에 쓰이는 아키텍처가 **UNet**입니다. 세 가지 핵심 특징이 있습니다.

### ① U자형 대칭 구조

왼쪽(인코딩)과 오른쪽(디코딩)이 대칭입니다. 디퓨전의 **순방향·역방향 대칭성**과 맞닿아 있습니다.

### ② Time Embedding (시점 임베딩)

“지금 몇 번째 스텝인가”를 UNet에 알려줘야 합니다. 스칼라 t를 **고차원 벡터**로 변환해 주입합니다. t는 순서 없이 랜덤으로 들어가도 학습됩니다.

### ③ Skip Connection

대칭 지점끼리 정보를 직접 연결합니다. 인코더 단의 **픽셀 단위 위치 정보**가 디코더로 전달되어, 세밀한 복원이 가능해집니다.

## 2-3. 백워드 프로세스 파이프라인

추론(생성) 시 전체 흐름:

1. 완전한 가우시안 노이즈 $x_T$에서 시작
2. UNet이 노이즈 맵을 예측
3. $x_t$에서 예측 노이즈를 빼서 $x_{t-1}$ 생성
4. $t = T, T-1, \ldots, 1$ 반복
5. $x_0$이 최종 생성 이미지

```
노이즈 x_T  →  [UNet: 노이즈 예측]  →  x_{T-1}  →  …  →  x_0 (생성 이미지)
```

## 2-4. 픽셀 디퓨전 vs 레이턴트 디퓨전 (LDM)

여기가 시험의 핵심 주제입니다.

### 바닐라(픽셀) 디퓨전 — DDPM 등

- **어디서**: 원본 이미지 **픽셀 도메인**에서 forward·backward 모두 수행
- **크기**: 512×512×3 ≈ 78만 차원
- **문제점**:
  - **연산량·메모리 폭증** — A100/H100급 GPU 필요
  - **시간** — 1000 스텝 반복 × 고차원 연산 = 실용성 낮음
  - **개인 PC에서 불가능**했음
  - 픽셀 단위 1:1 매칭이 필요해 UNet 부담이 큼

### 레이턴트 디퓨전(LDM) — Stable Diffusion

- **어디서**: VAE로 압축한 **잠재(latent) 공간**에서만 디퓨전 수행
- **압축**: 512×512 → 64×64 latent (약 48배, 채널 3→4로 약 48분의 1)
- **장점**:
  - **속도**: 약 48배 빠름 → 1000 스텝이면 약 48,000배 비용 절감 효과
  - **메모리**: 8GB VRAM GPU에서 동작
  - **대중화**: 개인 PC에서 이미지 생성 가능
  - **조작 용이**: 레이턴트에 텍스트·다른 모달리티를 **cross-attention**으로 주입
  - **정보 보존**: VAE가 의미 있는 특징만 압축, 복원 시 디테일 복구

> ⭐ **시험 포인트**  
> **픽셀 도메인 문제점**: 고차원·느림·메모리·GPU 비용  
> **레이턴트 도메인 장점**: 저차원·빠름·저비용·프롬프트 조건 주입·대중화  
> Stable Diffusion = **VAE + 디퓨전(LDM)**

### Stable Diffusion 전체 파이프라인

```
입력 이미지 → [VAE 인코더] → 잠재 벡터 z
                              ↓
                    [디퓨전: 노이즈 추가/제거]
                              ↓
              [VAE 디코더] → 생성 이미지
```

이미지↔레이턴트 변환은 VAE가 담당하고, **디퓨전 모델은 레이턴트 공간에서만** 동작합니다.

## 2-5. Unconditional vs Conditional Generation

### Unconditional (비조건부 생성)

- **랜덤 노이즈**에서 출발
- 학습 데이터가 이루는 **확률 분포 $P(\text{data})$** 방향으로만 복원
- **무엇이 나올지 모른다** — 고양이인지 강아지인지 제어 불가
- 초기 바닐라 디퓨전이 하던 방식

### Conditional (조건부 생성)

- **조건 $C$**(텍스트, 클래스 라벨, 이미지 등)를 추가로 주입
- “고양이”, “노란 꽃”, “스크래치 결함”처럼 **의도에 맞는 데이터** 생성
- 반도체에서는 **원하는 위치·타입·크기의 결함 이미지**를 만들 때 필수

> ⭐ **시험 포인트**  
> - **Unconditional**: 랜덤 생성, 학습 분포 방향  
> - **Conditional**: 정보(텍스트 등) 주입 → 의도 반영  
> 조건부 확률에서 방향 = **데이터 분포 방향 + 조건 C 방향**의 조합

### 조건을 넣는 방법: CLIP + Cross-Attention

텍스트 프롬프트가 들어오면:

1. **CLIP**: 이미지·텍스트를 같은 공간에 임베딩 (contrastive learning으로 사전학습)
2. **Transformer**: 단어 간 문맥·문법 관계 반영
3. **768차원 텍스트 벡터** 생성
4. UNet의 **cross-attention**으로 레이턴트에 주입

CLIP 도입 전후 성능이 26.7 → 96.7로 크게 올랐습니다. 프롬프트 문장을 조금만 바꿔도 벡터 방향이 달라져 **완전히 다른 이미지**가 나오므로, **프롬프트 엔지니어링**이 중요합니다.

## 2-6. Classifier-Free Guidance (CFG)

조건부 생성에서 “프롬프트를 얼마나 강하게 따를지”를 조절하는 핵심 기법입니다.

### 벡터 해석

- **Unconditional 벡터**: 학습 데이터 분포 방향 (조건 없이 가는 방향)
- **Conditional(텍스트) 벡터**: 프롬프트가 가리키는 방향
- **최종 방향** ≈ unconditional + CFG scale × (conditional − unconditional)

CFG scale을 키우면 텍스트 방향 벡터가 **길어져서** 생성 방향이 프롬프트 쪽으로 더 기울어집니다.

### CFG scale 효과 (개략)

| CFG 값 | 효과 |
|--------|------|
| 1~4 (낮음) | 프롬프트 영향 약함, **자유로운·다양한** 이미지 |
| **7~9 (권장)** | 프롬프트 반영 + 품질·다양성 균형 (기본값 **7.5**) |
| 15+ (높음) | 프롬프트에 과도 집착 → **오버새츄레이션**, 색·구조 왜곡, 아티팩트 |

### Negative Prompting

“이런 방향으로는 가지 마라”는 **반대 방향** 가이던스입니다.  
예: `blurry, low quality, mutated` — 흐릿함·저품질·기형 방지.

> ⭐ **시험 포인트**  
> CFG = **Classifier-Free Guidance scale**  
> 프롬프트(조건)의 **영향력(가이던스 강도)** 을 숫자로 조절하는 하이퍼파라미터

# 3부. 260625 오전 — 하이퍼파라미터: 생성 결과를 조율하는 네 개의 손잡이

## 3-1. 생성을 컨트롤하는 두 가지 길

디퓨전 모델은 블랙박스이지만, 결과를 조율하는 방법은 크게 두 가지입니다.

1. **프롬프트 입력** — 텍스트·이미지·음성 등 의도 전달
2. **하이퍼파라미터 조절** — CFG, 스텝, 시드, 샘플러

이 네 파라미터는 **서로 독립이 아닙니다**. 하나를 올리면 다른 것도 같이 조절해야 최적 결과가 나옵니다.

## 3-2. 네 가지 핵심 하이퍼파라미터

| 파라미터 | 역할 | 한 줄 요약 |
|----------|------|------------|
| **CFG scale** | 프롬프트 반영 강도 | 높을수록 의도 강반영, 낮을수록 랜덤에 가까움 |
| **Sampling steps** | 디노이징 반복 횟수 | 많을수록 선명·디테일, 시간↑ |
| **Seed** | 난수 시드 | 고정 → 같은 이미지 재현 / 미고정 → 매번 다름 |
| **Sampler** | 노이즈 제거 알고리즘 | 같은 스텝 수라도 경로·품질·속도가 달라짐 |

## 3-3. Sampling Steps (디노이징 스텝)

원래 디퓨전은 **1000 스텝**을 가정하지만, 실제 추론에서는 20~50 정도로 줄여 씁니다.

### 스텝 수에 따른 현상

| 스텝 | 결과 |
|------|------|
| **적음** (5~10) | 디노이징 미완 → **블러·미완성**, 디테일 없음 |
| **적당** (20~30) | 구조+디테일 균형, **인간 눈으로 충분** |
| **많음** (50+) | 품질 향상 있으나 **포화(saturation)** — 차이 거의 없고 시간만 증가 |

### 생성 순서의 직관

디노이징 초반: **저주파(구조·윤곽)** 먼저 형성  
디노이징 후반: **고주파(디테일·질감)** 복원

### 트레이드오프

- 시간은 스텝에 **선형 비례**
- 품질은 **비선형** — 어느 수준 이상이면 거의 안 좋아짐
- **권장**: 20~50 (보통 20~30이면 충분)

> ⭐ **시험 포인트**  
> 스텝 **↓** → 블러·미완성 / 스텝 **↑** → 선명·디테일, 단 **포화** 후 이득 적음

## 3-4. CFG Scale (가이던스 스케일)

### 값 변화에 따른 현상

| CFG | 현상 |
|-----|------|
| **1~5 (낮음)** | 프롬프트 무시에 가까움, **랜덤·창의적**이지만 의도 반영 안 됨 |
| **7~9 (권장)** | 의도 반영 + 자연스러운 품질 (Hugging Face 기본값 **7.5**) |
| **12~15+** | 프롬프트 과집착 → **오버새츄레이션**: 색 왜곡, 구조 붕괴, AI 티 나는 아티팩트 |

### CFG와 Steps의 연동

CFG를 **높이면** 아티팩트가 늘 수 있습니다. 이때 **스텝 수도 함께 늘리면** 노이즈가 더 정밀히 제거되어 부드러운 이미지를 얻을 수 있습니다.

예: CFG 8 → 12~15로 올릴 때, 스텝 30 → 50+로 올리는 것이 좋다.

> ⭐ **시험 포인트**  
> CFG **↓** → 프롬프트 약함·랜덤 / CFG **↑** → 프롬프트 강반영·오버새츄레이션  
> **CFG↑ 이면 steps↑ 권장** (파라미터 연동)

## 3-5. Seed (시드)

- **고정**: 같은 프롬프트·같은 설정 → **동일 이미지** 재현 (실험·비교용)
- **미고정(-1 등)**: 매번 **다른 랜덤** 결과

## 3-6. Sampler (샘플러)

스텝 수가 정해진 뒤, “노이즈를 **어떤 경로로** 제거할 것인가”를 결정합니다. 도착지(깨끗한 이미지)는 같아도 **가는 길**이 다릅니다.

| 샘플러 | 특성 | 스텝 |
|--------|------|------|
| **DDIM / Euler** | SDE 최적해 → **지름길**, 빠름 | 5~15로도 가능 |
| **DPM** | 확률적 탐색, **왔다갔다** 하며 좋은 길 탐색 | 50+ 필요, **고품질** |

- DDIM: 결정론적, 빠르지만 DPM 대비 품질 다소 낮을 수 있음
- DPM: 확률적, 시간은 걸리지만 **디테일 표현 우수** — 현재 많이 사용

샘플러에 따라 **최적 스텝 수가 달라지므로**, 샘플러를 바꾸면 스텝도 다시 맞춰야 합니다.

## 3-7. 권장 기본값 (교수님 정리)

| 파라미터 | 권장값 |
|----------|--------|
| CFG scale | **7 ~ 9** (기본 7.5) |
| Sampling steps | **20 ~ 30** |
| Sampler | 용도에 따라 DPM(품질) / DDIM(속도) |
| Seed | 재현 필요 시 고정 |

하이퍼파라미터는 **경험적**입니다. try & error로 자신의 프롬프트·목적에 맞는 값을 찾아야 합니다.

## 3-8. 체크포인트와 파인튜닝 (참고)

Stable Diffusion은 **사전학습된 체크포인트**를 공개합니다. 처음부터 학습할 필요 없이 가중치를 불러와 **파인튜닝**하면 됩니다.

- **DreamBooth**: 특정 대상(인물·제품)에 맞춤
- **LoRA**: 적은 파라미터로 빠른 적응

생성형 AI는 여전히 블랙박스이지만, 프롬프트 + 하이퍼파라미터로 **어느 정도 의도한 데이터**를 만들 수 있게 되었습니다.

## 문제 1. 픽셀 도메인 vs 레이턴트 스페이스 디퓨전

**질문 방향**: 기존(바닐라/DDPM) 디퓨전과 Stable Diffusion(LDM)의 **공간 차이**, 각각의 **문제점·장점**

### 픽셀 도메인 디퓨전 — 문제점

- 512×512×3 등 **고차원(≈78만)** 픽셀 공간에서 forward·backward **모두** 수행
- **연산량·메모리** 폭증 → A100/H100급 GPU 필요
- 디노이징 **1000 스텝 반복** × 고차원 = 시간·비용 기하급수적 증가
- **개인 PC에서 실용 불가**, 대중화 어려움

### 레이턴트 스페이스 디퓨전 — 장점

- VAE로 **64×64 잠재 공간**에 압축(약 48배) 후 디퓨전만 latent에서 수행
- **속도·메모리 대폭 절감** → 8GB VRAM 일반 GPU에서 동작
- **정보 유실 없이** 의미적 특징(형태·구도·색) 보존
- 레이턴트에 **텍스트·조건 주입(cross-attention)** 용이 → 조건부 생성·컨트롤
- **오픈소스·대중화** (Stable Diffusion)

**한 줄 답**: Stable Diffusion = **VAE(압축/복원) + LDM(잠재 공간 디퓨전)**

## 문제 2. Unconditional vs Conditional Generation

**질문 방향**: 두 생성 방식의 **정의·차이·왜 조건부가 필요한가**

| | Unconditional | Conditional |
|---|---------------|-------------|
| **정의** | 조건 없이 **랜덤** 노이즈에서 생성 | 텍스트·라벨 등 **조건 C**를 주입해 생성 |
| **방향** | 학습 데이터 분포 $P(\text{data})$ 만 따름 | 데이터 분포 + **조건 C 방향** 조합 |
| **결과** | 무엇이 나올지 **예측 불가** | **의도(프롬프트)** 에 맞는 이미지 |
| **예시** | DDPM 초기 — 그냥 “뭔가” 생성 | “고양이”, “노란 꽃”, “스크래치 결함” |

**구현**: CLIP 텍스트 임베딩 → Transformer → UNet **cross-attention**으로 레이턴트에 주입

---

## 문제 3. 하이퍼파라미터 — CFG Scale

**질문 방향**: CFG(Classifier-Free Guidance) scale 값을 바꾸면 **어떤 현상**이 나타나는가

| CFG 변화 | 현상 |
|----------|------|
| **낮춤** (1~5) | 프롬프트 영향 약함 → **랜덤·자유로운** 이미지, 의도 반영 안 됨 |
| **적당** (7~9) | 프롬프트 반영 + 자연스러운 품질 (기본 **7.5**) |
| **높임** (15+) | 프롬프트 과집착 → **오버새츄레이션**, 색·구조 왜곡, 아티팩트, 데이터 본질 상실 |

**원리**: 최종 방향 = uncond + **CFG ×** (cond − uncond) — scale이 클수록 텍스트 방향 가중

**연동**: CFG를 높이면 **sampling steps도 함께 늘리는 것**이 좋다 (아티팩트 완화)

---

## 문제 4. 하이퍼파라미터 — Sampling Steps (타임스텝)

**질문 방향**: 디노이징 스텝(타임스텝) 수를 바꾸면 **어떤 현상**이 나타나는가

| Steps 변화 | 현상 |
|------------|------|
| **줄임** | 디노이징 불완전 → **블러·미완성**, 디테일 없음 |
| **늘림** | 구조→디테일 순으로 정밀 복원, **선명·고품질** |
| **과다** | **포화(saturation)** — 20~30 이후 눈에 띄는 차이 거의 없고 **시간만 증가** |

**트레이드오프**: 시간은 선형 증가, 품질은 비선형(한계 있음)  
**권장**: 20~50 (실무 20~30)

**연동**: 샘플러 종류(DDIM vs DPM)에 따라 최적 스텝 수가 다름

## 시험 전 체크리스트

### 픽셀 vs 레이턴트

- [ ] 바닐라/DDPM = **픽셀 공간** 디퓨전
- [ ] Stable Diffusion = **레이턴트 공간** LDM = VAE + 디퓨전
- [ ] 픽셀 **문제**: 고차원·느림·메모리·고가 GPU
- [ ] 레이턴트 **장점**: 빠름·저비용·8GB GPU·프롬프트 주입·대중화

### Unconditional vs Conditional

- [ ] Unconditional = 랜덤, 학습 분포만
- [ ] Conditional = 조건(텍스트 등) 주입, **의도 반영**
- [ ] CLIP + cross-attention으로 텍스트→레이턴트 주입

### 하이퍼파라미터

- [ ] **CFG ↓** → 프롬프트 약함·랜덤 / **CFG ↑** → 강반영·오버새츄레이션·아티팩트
- [ ] **Steps ↓** → 블러·미완성 / **Steps ↑** → 선명·디테일, 포화 후 이득 적음
- [ ] CFG↑ 할 때 **steps도↑** 권장
- [ ] 기본값: CFG **7.5**, steps **20~30**
