# 2. IoU 직접 구현 ⭐

\[
\text{IoU} = \frac{|A \cap B|}{|A \cup B|} = \frac{|A \cap B|}{|A| + |B| - |A \cap B|}
\]

코드 핵심:

```python
inter_x1 = max(boxA[0], boxB[0])
inter_y1 = max(boxA[1], boxB[1])
inter_x2 = min(boxA[2], boxB[2])
inter_y2 = min(boxA[3], boxB[3])
inter_area = max(0, inter_x2 - inter_x1) * max(0, inter_y2 - inter_y1)
union_area = areaA + areaB - inter_area
iou = inter_area / union_area
```

→ **합집합 = A+B − 교집합**이라는 식만 코드로 옮기면 끝.

# 3. Fast R-CNN 핵심 구현 ⭐

## 3.1 ROI Pooling

**입력**: 이미지 feature map + proposal (이미지 좌표)
**출력**: 모든 proposal에 대해 동일한 \(C \times 7 \times 7\) feature

핵심 2단계:

### Step 1: Proposal 좌표를 feature map 좌표로 변환

```python
x1' = x1 * spatial_scale
y1' = y1 * spatial_scale
x2' = x2 * spatial_scale
y2' = y2 * spatial_scale
```

\(\text{spatial\_scale} = \frac{\text{feature map size}}{\text{image size}}\) — 이미지보다 feature map이 작으니 좌표를 비례 축소.

### Step 2: 7×7 grid로 나눠 max pool

```python
roi_feat = feature_map[:, y1':y2', x1':x2']  # 슬라이싱
# 7×7 sub-grid로 나눠 각 셀에서 max
pooled = adaptive_max_pool_to_7x7(roi_feat)
```

→ 각 proposal이 어떤 크기든 **\(C \times 7 \times 7\) 고정 텐서**로 출력 → 뒤의 linear head로 안전히 연결.

## 3.2 Multi-Task Loss ⭐

### Head 정의

```python
# Classification head — (num_classes + 1)차원, +1은 background
self.cls_head = nn.Linear(in_features, num_classes + 1)

# Box Regression head — 클래스당 4개 (cx, cy, w, h delta)
self.box_head = nn.Linear(in_features, num_classes * 4)
```

### Loss 계산

```python
loss_cls = F.cross_entropy(pred_cls, gt_cls)

foreground_mask = (gt_cls > 0)  # background(=0) 제외
loss_box = smooth_l1_loss(pred_box[foreground_mask], gt_box[foreground_mask])

loss = loss_cls + lambda * loss_box
```

### Smooth L1 Loss ⭐

\[
\text{smooth}_{L_1}(x) = \begin{cases} 0.5 x^2 & |x| < 1 \\ |x| - 0.5 & \text{otherwise} \end{cases}
\]

- **L1 vs Smooth L1 차이**: 일반 L1은 모든 영역에서 gradient 일정 → 작은 에러도 큰 압력. Smooth L1은 **작은 에러에서 덜 패널티**, **큰 에러에서는 L1처럼**.
- 효과: **현재 모델이 잘 못하는 큰 에러 박스에 집중**하도록 유도.

# 4. SSD 핵심 구현 ⭐

## 4.1 멀티스케일 Feature Map 결합

```python
# VGG 백본 사용 (classification pre-trained)
# 여러 스테이지의 feature map을 모두 사용
features = []
features.append(conv4_3_output)  # 큰 해상도 → 작은 오브젝트
features.append(conv7_output)
features.append(conv8_2_output)
features.append(conv9_2_output)
features.append(conv10_2_output)
features.append(conv11_2_output)  # 작은 해상도 → 큰 오브젝트
```

**왜 멀티스케일?**

- 큰 feature map: 공간 해상도 높음 → 작은 오브젝트가 한 셀에 들어감.
- 작은 feature map: 한 셀이 이미지의 큰 영역을 덮음 → 큰 오브젝트 디텍션에 유리.
- 크기에 무관하게 다양한 오브젝트를 찾을 수 있도록 함께 사용.

## 4.2 Default Box ⭐

### 핵심 직관

한 셀이 하나의 박스를 담당하는 게 아니라 **여러 개의 미리 정의된 박스(Default Box)**를 담당하고, 모델은 그 박스들과 GT의 **delta(변형 파라미터)**만 학습.

### Aspect Ratio별 박스 생성

```python
for ratio in [1, 2, 3, 1/2, 1/3]:
    if ratio == 1:
        # 두 가지 정사각형 (현재 스케일 + 보조 스케일)
        boxes.append(square_at_current_scale)
        boxes.append(square_at_extra_scale)  # 더 큰 정사각형
    else:
        w = scale * sqrt(ratio)
        h = scale * sqrt(1/ratio)
        boxes.append((cx, cy, w, h))
```

| Aspect Ratio | 의미 |
|---|---|
| 1 | 정사각형 (+ 보조 큰 정사각형 1개) |
| 2 | 좌우로 긴 박스 |
| 3 | 좌우로 더 긴 박스 |
| 1/2 | 위아래로 긴 박스 |
| 1/3 | 위아래로 더 긴 박스 |

→ 셀당 최대 **6개의 디폴트 박스**. 전체 약 8732개.

## 4.3 Hard Negative Mining ⭐⭐

### 왜 필요한가

- 8732개 디폴트 박스 중 GT와 매칭되는 positive는 **2~3개 수준**.
- 나머지 거의 다 negative(background) → 그냥 학습하면 모델이 "다 background"로 출력하는 trivial한 해에 빠짐.

### 핵심 아이디어

**Positive : Negative = 1 : 3** 비율로 negative를 줄이되, **어려운 negative**(모델이 positive로 잘못 분류하려는)만 골라 학습.

### 구현

```python
# 1. 모든 박스의 cross-entropy 계산
ce_all = F.cross_entropy(pred, gt, reduction='none')

# 2. Negative 중에서만 cross-entropy 값 추출 (positive는 0으로 마스킹)
ce_negative = ce_all * (1 - positive_mask)

# 3. 내림차순 정렬 (어려운 것 = CE 큰 것이 앞)
rank = ce_negative.argsort(descending=True).argsort()

# 4. 상위 3 × num_positive 만큼만 선택
num_negative = 3 * num_positive
hard_negative_mask = (rank < num_negative)

# 5. positive + hard negative만 confidence loss 계산
final_mask = positive_mask | hard_negative_mask
loss_conf = ce_all[final_mask].sum()
```

→ **랜덤 negative 대신 모델이 어려워하는 negative에 집중** → 안정적 학습.

# 5. YOLO 핵심 구현 ⭐

## 5.1 GT를 셀에 할당

### Step 1: 어느 셀이 GT를 담당?

GT bounding box의 **중심 좌표**가 어느 셀에 속하는지 계산.

```python
cell_x = int(gt_cx * S / image_width)
cell_y = int(gt_cy * S / image_height)  # S=7
```

### Step 2: 셀 내부 상대 위치

```python
rel_x = (gt_cx * S / image_width) - cell_x  # 셀 내부 0~1
rel_y = (gt_cy * S / image_height) - cell_y
```

→ 박스 할당에는 안 쓰이지만 **나중에 location loss 계산에 사용**.

## 5.2 WH-IoU ⭐

### 동기

학습 초기에 박스 **위치**까지 정확히 맞추기는 어려움. → **중심이 일치한다고 가정**하고 **크기(width, height)만으로 IoU를 평가**:

```python
inter_w = min(box1_w, box2_w)
inter_h = min(box1_h, box2_h)
inter_area = inter_w * inter_h

union_area = box1_w * box1_h + box2_w * box2_h - inter_area
wh_iou = inter_area / union_area
```

→ "두 박스의 중심을 일치시킨 상태로 겹친다면 IoU가 얼마일까?"를 계산. **박스 크기가 GT와 잘 맞는지**만 평가하는 메트릭.

## 5.3 Responsible Mask ⭐⭐

### 동기

각 셀은 **2개의 박스를 prediction**. 그중 어느 하나만 GT와 매칭해 loss를 줘야 학습이 안정.

### 구현

```python
# 1. 각 셀에서 2개의 prediction box vs GT의 WH-IoU 계산
wh_ious = compute_wh_iou(pred_boxes, gt_boxes)  # shape: (S, S, 2)

# 2. 더 잘 매칭되는 박스의 인덱스 선택
best_idx = wh_ious.argmax(dim=-1)  # 0 또는 1

# 3. zeros_like로 초기화 후 best 위치만 True
responsible_mask = torch.zeros_like(wh_ious, dtype=torch.bool)
responsible_mask.scatter_(-1, best_idx.unsqueeze(-1), True)

# 4. 오브젝트 있는 셀에서만 적용
object_responsible = responsible_mask & object_mask
```

→ "셀 당 2개의 박스 중 **GT와 크기가 더 비슷한 것**이 그 GT를 책임진다(Responsible)" — 그 박스만 loss 계산에 들어감.

## 5.4 YOLO의 세 가지 장점 정리 ⭐

| 장점 | 의미 |
|---|---|
| **Cell-based Prediction** | SSD가 8732 박스 만드는 동안 YOLO는 7×7×2 = 98개로 끝 → 빠름 |
| **WH-IoU 학습** | 학습 초기 박스 위치까지 다 맞추기 불안정 → 크기부터 먼저 정확히 학습 |
| **Responsible Mask** | 셀 당 두 박스 중 GT와 best match만 책임 → loss 안정화 |

# 7. 오후 핵심 정리 ⭐

- **IoU**는 (교집합 면적) / (A + B − 교집합 면적)으로 구현되며, 두 박스의 좌표만으로 계산 가능하다.
- **Fast R-CNN의 ROI Pooling**은 두 단계로 구현된다: (1) proposal 좌표를 `spatial_scale`로 곱해 feature map 좌표로 변환, (2) 7×7 sub-grid로 나눠 max pool해 고정 디멘전 출력.
- **Multi-Task Loss**의 핵심은 (1) classification head 출력 차원이 **num_classes + 1**(background 포함), (2) box regression head는 **num_classes × 4**, (3) **Smooth L1 Loss**가 작은 에러는 quadratic, 큰 에러는 linear여서 큰 에러 박스에 학습 집중.
- **SSD의 멀티스케일 feature map**은 큰 해상도(작은 오브젝트용)부터 작은 해상도(큰 오브젝트용)까지 list로 결합하며, 각 셀마다 **Aspect Ratio**별로 최대 6개의 Default Box를 미리 정의한다.
- **Hard Negative Mining**은 positive 박스 수의 3배만큼만 negative를 사용하되, **cross-entropy가 큰(=모델이 어려워하는)** negative만 골라 학습 — `argsort(descending=True).argsort()`로 rank를 만들고 상위만 마스킹.
- **YOLO의 GT 셀 할당**은 박스 중심을 \(S\)로 나눠 cell 인덱스로, 그리고 그 셀 내부의 상대 위치를 location loss용으로 따로 저장한다.
- **WH-IoU**는 두 박스의 중심이 같다고 가정하고 width·height만으로 IoU를 계산해 **박스 크기 매칭에만 집중**하는 메트릭이며, 학습 초기 안정성을 준다.
- **Responsible Mask**는 셀 당 2개의 prediction 박스 중 WH-IoU가 더 큰 박스만 True로 만들어, 그 박스만 GT에 대응되도록 loss를 한정한다 — 두 박스가 동시에 학습되며 충돌하는 것을 막는다.
