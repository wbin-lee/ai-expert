# 260602 오전 강의 요약 — Object Detection · IoU/mAP 평가 · Object Proposal(Selective Search/EdgeBox) · R-CNN Family · SSD · YOLO v1/v2/v3

> 오전 흐름: **Object Detection 문제 정의(다중 클래스/다중 인스턴스 바운딩박스) → Image Classification·Localization·Detection·Instance Segmentation 비교 → 평가 메트릭(IoU/Precision/Recall/AP/mAP) → 옛날 exhaustive search 디텍션의 한계 → Object Proposal(Selective Search·EdgeBox) → R-CNN(proposal → CNN 2000번 + SVM + bounding box regression) → Fast R-CNN(이미지 한 번 CNN + ROI Pooling + 멀티태스크 로스) → Faster R-CNN(Region Proposal Network로 proposal까지 통합, 앵커박스) → SSD(멀티스케일 피처맵 + 디폴트박스 + dilated convolution) → YOLO v1(7×7 그리드 셀, 98 bounding box) → YOLO v2(배치노멀/앵커/디멘전 프라이어/시그모이드 파라미터화/패스스루/멀티스케일 트레이닝) → YOLO v3 → ImageNet WordNet 하이라키 활용한 9000 클래스 디텍션**
> 강의 주제: **Object Detection 문제의 평가 지표(특히 mAP/Precision-Recall curve)와 옛날 exhaustive search의 한계를 짚은 뒤, R-CNN 계열(투 스테이지)과 SSD/YOLO(싱글 스테이지)가 어떻게 발전했는지 그 핵심 트레이드오프를 이해하는 것**.

# 1. Object Detection 문제 정의 ⭐

## 1.1 테스크

한 이미지에 **여러 클래스의 여러 인스턴스**가 있을 때 각각의 (클래스, 바운딩박스)를 모두 찾아내기.

- 클로즈셋(미리 정해진 클래스)이 기본. **오픈셋 detection**은 파운데이션 모델(CLIP 등)과 결합해 학습되지 않은 클래스도 디텍션 시도.
- 바운딩박스 컨벤션: 오브젝트가 concave해도 **전체를 포함하는 직사각형**.

## 1.2 유사 문제 비교 ⭐

| 문제 | 입력 / 출력 |
|---|---|
| **Image Classification** | 이미지 1장 → 1 클래스 |
| **Localization** | 이미지 1장 → 1 클래스 + 1 바운딩박스 |
| **Object Detection** | 이미지 1장 → 다중 (클래스, 바운딩박스) |
| **Semantic Segmentation** | 이미지 1장 → 픽셀별 클래스 (인스턴스 구분 X) |
| **Instance Segmentation** | 이미지 1장 → 인스턴스별 픽셀 마스크 (detection의 픽셀 버전) |

# 2. 평가 메트릭 ⭐

## 2.1 IoU (Intersection over Union)

\[
\text{IoU} = \frac{|\text{GT} \cap \text{Pred}|}{|\text{GT} \cup \text{Pred}|}
\]

- 임계치 0.5가 흔히 사용. 0.5만 돼도 시각적으로 상당히 겹친 상태.
- IoU ≥ threshold면 디텍션 성공으로 카운트.

## 2.2 Precision / Recall ⭐

|  | 정의 |
|---|---|
| **Precision** | \(\frac{TP}{TP + FP}\) — 모델이 prediction한 박스 중 진짜 오브젝트 비율 |
| **Recall** | \(\frac{TP}{TP + FN}\) — 씬에 있는 오브젝트 중 모델이 찾은 비율 |

**Trade-off**: 클래시피케이션 confidence threshold를 높이면 precision ↑, recall ↓.

## 2.3 Precision-Recall Curve와 AP ⭐

Detection의 PR 커브는 일반 분류와 달리 **지그재그 형태**:

- 바운딩박스를 score 내림차순 정렬 후 하나씩 보면, 맞은 박스가 연속되면 precision 유지·recall 상승, 틀린 박스가 끼면 precision이 떨어진다.
- 이 curve의 **밑 면적이 Average Precision (AP)**.

\[
\text{AP} = \int_0^1 P(r) \, dr \quad (\text{IoU/threshold별 평균})
\]

- 각 클래스마다 AP 계산 → 클래스 평균이 **mAP (mean AP)**.
- **mAP@0.5**: IoU 0.5 기준. **mAP@[0.5:0.95]**: IoU 0.5부터 0.95까지 0.05 단위로 모두 평균 — 더 엄격한 메트릭.

# 3. 옛날 디텍션과 한계 ⭐

오브젝트 위치/크기/모양 정보가 없으니 **exhaustive search**:

- 모든 위치 × 모든 스케일 × 모든 aspect ratio (+ 가끔 rotation)을 다 슬라이딩.
- 그래서 **클래스마다 따로 디텍터** (face/car/pedestrian detector), 그것도 정면 face만, 뒷면 자동차만, 서 있는 사람(pedestrian)만 등 극도로 제한.
- CPU 빠른 detector(예: Viola-Jones face detection, 2001)는 가능했지만 **CNN을 모든 위치에 돌리면 리얼타임 불가**.

→ **Object Proposal**: 이미지에서 오브젝트일 것 같은 영역만 ~2000개 미리 뽑고 그 영역에 대해서만 CNN.

---

# 4. Object Proposal 알고리즘 ⭐

| 알고리즘 | 핵심 아이디어 |
|---|---|
| **Selective Search** | 다양한 컬러 스페이스/시밀러리티 메트릭으로 **Super Pixel Segmentation**을 하이라키컬하게 반복 → 각 super pixel을 proposal로 |
| **EdgeBox** | 빠른 머신러닝 기반 edge detection 후, 박스 안 edge 픽셀 수 − 박스를 가로지르는 edge 수로 score 매김 (오브젝트는 박스 안에 edge 많고 가로지르는 edge는 적다는 가정) |

성능 비교 (75% recall 기준):

- Selective Search: 1400 박스 필요
- EdgeBox: **800 박스로 충분, 40× 빠름**

다만 구현 편의성으로 **논문에서는 Selective Search가 더 많이 쓰임**.

**Proposal 알고리즘의 핵심 성질**: (1) 클래스 무관(agnostic), (2) **리콜이 가장 중요**(못 뽑은 영역은 영영 못 찾음), (3) 빠른 속도(전처리 단계).

# 5. R-CNN (Region-based CNN, 2014) ⭐

## 5.1 파이프라인

```
이미지 → Selective Search로 ~2000 proposal → 각 영역을 224×224로 리스케일
       → 2000번 CNN(AlexNet) → 4096-d feature → SVM 분류 + Bounding Box Regression
```

## 5.2 가장 큰 컨트리뷰션: Bounding Box Regression ⭐

Proposal 박스 \(p = (p_x, p_y, p_w, p_h)\)를 GT 박스 \(g\)로 변환하는 transformation \(d(p)\)를 회귀.

파라미터화:

\[
g_x = p_w \, d_x(p) + p_x, \quad g_y = p_h \, d_y(p) + p_y
\]
\[
g_w = p_w \exp(d_w(p)), \quad g_h = p_h \exp(d_h(p))
\]

학습:

\[
d_\star(p) = w_\star^T \phi_5(p)
\]

\(\phi_5(p)\)는 conv5 피처. **Ridge regression**으로 \(w_\star\) 학습. 단순 linear projection만으로 충분하다고 가정.

→ 이 bounding box regression은 **모든 후속 디텍션 모델에 표준으로 채택**.

## 5.3 Bottleneck

**CNN을 2000번 돌려야 함** → 너무 느림.

# 6. Fast R-CNN (2015) ⭐

## 6.1 핵심 변화: 이미지를 CNN에 한 번만

- 이미지 전체 → CNN → feature map.
- Proposal을 **feature map 좌표로 매핑**해 그 영역의 feature만 추출.
- CNN 한 번 → 학습 9×, 테스트 200×+ 빨라짐.

## 6.2 ROI Pooling ⭐

문제: proposal 크기가 다 다른데 뒤의 FC layer는 고정 디멘전 입력 필요.

해결:

1. Proposal 좌표를 feature map 그리드에 **라운딩**.
2. 그 영역을 **3×3 sub-grid**로 다시 나눔(버림).
3. 각 sub-cell에서 **max pooling** → 항상 3×3 = 9개 값.

→ 입력 크기 무관하게 고정 디멘전 feature.

## 6.3 멀티태스크 로스 ⭐

SVM 따로 학습 X. **모든 것을 CNN으로**.

\[
L = L_{\text{cls}}(p, u) + \lambda [u \geq 1] \, L_{\text{loc}}(t^u, v)
\]

- \(L_{\text{cls}}\): cross-entropy. **0번 클래스는 background** — 백그라운드면 \([u \geq 1] = 0\)이라 로컬라이제이션 로스 무시.
- \(L_{\text{loc}}\): **Smooth L1 loss** — L1처럼 outlier robust하면서 0에서 미분 가능하도록 quadratic.

\[
\text{smooth}_{L_1}(x) = \begin{cases} 0.5 x^2 & |x| < 1 \\ |x| - 0.5 & \text{otherwise} \end{cases}
\]

# 7. Faster R-CNN (2015) ⭐

## 7.1 마지막 병목: Selective Search

Fast R-CNN까지 빨라지니 **proposal 뽑는 게 가장 느린 단계**. → **네트워크 안에서 proposal까지 해결**.

## 7.2 Region Proposal Network (RPN) ⭐

```
이미지 → CNN → feature map
        → RPN: 각 셀에서 K개의 anchor box마다
                ① 오브젝트 vs 백그라운드 (2K 스코어)
                ② 바운딩박스 회귀 파라미터 (4K 출력)
        → top proposals → ROI Pooling → Classification + Refined regression
```

## 7.3 Anchor Box ⭐

Feature map의 \(m \times n\) 그리드 각 셀에 **K개의 미리 정의된 박스**(다른 스케일 × 다른 aspect ratio) — 총 \(m \cdot n \cdot K\)개 proposal.

→ Selective Search 제거 후 **RPN은 10ms 수준**.

## 7.4 성능 ⭐

| 비교 | Proposal 수 | mAP |
|---|---|---|
| Fast R-CNN + Selective Search | 2000 | 좋음 |
| **Faster R-CNN + RPN** | **300** | **더 좋음** |

# 8. SSD (Single Shot MultiBox Detector, 2016) ⭐

## 8.1 핵심 아이디어

- **Fully Connected 제거** → 전부 convolution.
- **멀티스케일 피처맵**에서 동시에 디텍션 → 다양한 크기 오브젝트 대응.

## 8.2 멀티스케일 디텍션

```
VGG → conv4_3 (38×38) → 디텍션
    → conv 더 → 19×19 → 디텍션
    → conv 더 → 10×10 → 디텍션
    → conv 더 → 5×5 → 디텍션
    → conv 더 → 3×3 → 디텍션
    → conv 더 → 1×1 → 디텍션
```

- 큰 feature map (38×38): **작은 오브젝트** 담당(공간 해상도 높음).
- 작은 feature map (1×1): **큰 오브젝트** 담당.

## 8.3 Default Box ⭐

각 셀마다 최대 6개의 디폴트 박스:

- Aspect ratio: 1, 2, 3, 1/2, 1/3 + 1 한 개의 다른 스케일.
- 각 박스마다 (C+4) 출력: C 클래스 score + 4 박스 regression.
- 총 디폴트 박스 약 **8732개**.

각 piece: 3×3 conv 한 번이 4(또는 6) × (C+4) 개의 output 채널을 produce.

## 8.4 Dilated Convolution ⭐

FC6/FC7에서 사용. Kernel은 그대로 3×3이지만 **rate=2로 한 칸씩 띄어 적용** → 같은 파라미터 수로 receptive field 2배.

\(\rightarrow\) feature map size 줄이지 않고 RF 키우기에 유리. DeepLab에서 처음 도입.

## 8.5 Hard Negative Mining ⭐

- 디폴트 박스 8732개 중 대부분이 background → 그냥 학습하면 모델이 "다 background"로 학습.
- **현재까지 학습된 모델이 positive로 잘못 분류한 어려운 background**만 골라 다시 학습.

## 8.6 성능

성능은 Faster R-CNN > SSD > YOLO, 속도는 그 반대.

# 9. YOLO v1 (You Only Look Once, 2016) ⭐

## 9.1 핵심 아이디어 — Detection을 Regression으로

- 이미지를 **S×S = 7×7 그리드**로 나눔.
- 각 셀이 **B = 2개의 바운딩박스**를 prediction → 총 **98개 박스로 모든 detection 시도**.
- GT 오브젝트의 중심이 속한 셀이 그 오브젝트를 담당.

## 9.2 출력 텐서 ⭐

각 셀 출력: \(B \cdot 5 + C\) 차원

- 각 박스: (x, y, w, h, confidence) — 5개
- 클래스: 20개 (PASCAL VOC)

→ 셀마다 30차원, 7×7×30 = **1470 출력**.

Confidence:

\[
\text{Confidence} = P(\text{Object}) \cdot \text{IoU}
\]

클래스별 스코어:

\[
P(\text{Class}_i \mid \text{Object}) \cdot P(\text{Object}) \cdot \text{IoU} = P(\text{Class}_i) \cdot \text{IoU}
\]

## 9.3 아키텍처

CNN 백본 → 마지막에 **Fully Connected 두 개**로 7×7×30 출력. FC가 들어가 있어 입력 크기 고정.

## 9.4 한계

- **98개로 다 잡으려는 야심찬 계획** → 성능 낮음(mAP 63.4, Faster R-CNN 73.2).
- 하나의 셀이 여러 오브젝트의 중심을 담당할 때 **하나만 디텍션** — 작은 오브젝트 군집에 약함.
- 가장 빠름(45 FPS, Fast YOLO 155 FPS).

# 10. YOLO v2 ⭐ — Better, Faster, Stronger

## 10.1 Better (정확도 향상)

| 기법 | 효과 |
|---|---|
| **Batch Normalization** | 모든 conv 뒤에 추가 |
| **High-Resolution Classifier** | 448×448로 클래시파이어 사전학습 후 detection fine-tune |
| **Convolutional with Anchors** | FC 제거하고 SSD처럼 앵커박스 사용 (mAP −0.3 pt지만 **recall +7 pt**) |
| **Dimension Prior** ⭐ | 트레이닝 데이터의 GT 박스를 **k-means 클러스터링**해 앵커 결정 — 데이터에 맞춤(다소 cheating 논란) |
| **Direct Location Prediction (Sigmoid)** ⭐ | bbox regression 출력을 sigmoid로 [0,1] 바운드 → 학습 안정화 |
| **Passthrough** ⭐ | 큰 feature map(26×26)을 4채널로 reshape해 작은 feature map(13×13)에 concat → 멀티스케일 정보 결합 |
| **Multi-Scale Training** | 학습 중 입력 크기를 다양하게 바꿈 |

## 10.2 Faster (속도)

**Darknet** 백본 — VGG보다 flop 작음.

## 10.3 Stronger — 9000 클래스 Detection ⭐

**WordNet 하이라키** 활용:

- ImageNet에는 detection 레이블 없지만 classification 레이블만 있는 클래스 9000개.
- WordNet은 단어 간 hierarchy(예: Norfolk Terrier → Terrier → Hunting Dog → Dog → Animal → Physical Object).
- Detection 모델을 학습할 때 상위 노드 클래스를 함께 prediction.
- 인퍼런스 시 ImageNet의 leaf 클래스에 대해서도 **체인 룰로 confidence 계산** → 학습 안 한 클래스도 detection 가능.

## 10.4 성능

67 FPS / 76.8 mAP (PASCAL VOC 2007) — SSD와 비등.

# 11. YOLO v3 ⭐

엔지니어링 중심 업그레이드. 학술적 컨트리뷰션보다는 **실용성**.

- **멀티스케일 prediction** (3개 스케일에서 detection — FPN 유사).
- **더 좋은 백본** (Darknet-53, residual 도입).
- 멀티스케일 prediction을 인퍼런스에도 적용해 앙상블 효과.

# 12. 알고리즘 비교 ⭐

| 모델 | 카테고리 | Proposal 처리 | 강점 | 약점 |
|---|---|---|---|---|
| **R-CNN** | Two-stage | Selective Search 2000개 → CNN 2000번 | bounding box regression 도입 | 극도로 느림 |
| **Fast R-CNN** | Two-stage | Selective Search + ROI Pooling | CNN 한 번, 멀티태스크 로스 | Proposal이 병목 |

| **SSD** | Single-stage | 디폴트박스(8732) + 멀티스케일 | 빠르고 정확도 균형 | 작은 오브젝트 약함(보완 가능) |
| **YOLO v1** | Single-stage | 7×7 그리드, 98 박스 | 가장 빠름(155 FPS) | 정확도 낮음 |
| **YOLO v2/v3** | Single-stage | 앵커 + 멀티스케일 | 속도+정확도 균형 | - |

# 13. 오전 핵심 정리 ⭐

- **Detection 평가**의 핵심은 IoU(보통 ≥0.5면 success), Precision-Recall trade-off, 그리고 PR curve의 면적인 **AP**와 클래스 평균인 **mAP**(mAP@0.5 vs mAP@[0.5:0.95]).
- 옛날 exhaustive search는 클래스마다 정면/뒷면 같은 한정된 케이스만 가능했고 CNN과는 호환되지 않아 **Object Proposal**(Selective Search, EdgeBox)이 등장했다 — proposal의 핵심 성질은 **클래스 무관, 리콜 우선, 빠른 속도**.
- **R-CNN**의 진짜 contribution은 파이프라인이 아니라 **bounding box regression** — proposal 박스를 GT로 transform하는 4개 파라미터(\(d_x, d_y, d_w, d_h\))를 conv5 피처의 linear projection으로 회귀하며, 이는 모든 후속 디텍터의 표준이 되었다.
- **Fast R-CNN**은 CNN을 이미지에 한 번만 돌리고 proposal을 feature map 좌표로 매핑한 뒤 **ROI Pooling**(3×3 sub-grid의 max pool)으로 디멘전을 통일하며, classification과 bbox regression을 멀티태스크 로스(cross-entropy + smooth L1)로 동시 학습한다.
- **Faster R-CNN**은 Selective Search 병목을 제거하기 위해 **RPN**(각 셀의 K anchor box마다 obj/no-obj + 4 regression 파라미터)을 도입했고, 300개 proposal로 Fast R-CNN의 2000개를 능가했다.
- **SSD**는 **멀티스케일 feature map**(38×38부터 1×1까지)에서 8732개 디폴트박스로 동시 detection하며, Fully Connected를 완전히 제거하고 dilated convolution과 hard negative mining을 도입했다.
- **YOLO v1**은 detection을 7×7 그리드의 **regression**으로 보고 98 박스로 모든 detection을 시도해 가장 빠르지만 정확도가 낮다.
- **YOLO v2**의 핵심은 **앵커 사용(recall +7pt), k-means dimension prior, sigmoid 파라미터화(학습 안정), passthrough(멀티스케일 결합)**이며, **WordNet 하이라키**로 9000 클래스 detection까지 확장했다.
