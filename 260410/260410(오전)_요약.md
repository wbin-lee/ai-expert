# 260410 오전 강의 요약 — AI 기초 이론·머신러닝 개념·일반화

## 1. AI의 정의와 3세대론

### 1.1 AI의 학문적 정의
- 사람처럼 생각·행동하는 기계, 또는 합리적·이성적으로 행동하는 시스템
- 학계 전통 용어는 "Computational Intelligence", "Rationality"
- 과거 50~60년 저널에는 "AI"가 거의 없었다가 최근 우후죽순 등장

### 1.2 AI의 3세대
1. **1세대 — 규칙 기반(Rule-based)** : 데이터 불필요, 예외 많음
2. **2세대 — 머신러닝(ML)** : 사람이 특징(feature) 설계
3. **3세대 — 딥러닝 / 표현 학습(Representation Learning)** : 기계가 자동 특징 추출

### 1.3 합리성(Rationality)과 효용
- 경제학적 정의 : **효용(Utility)을 최대화** 하는 의사결정
- 불확실성 하에서는 **기대값 기반** 효용
- 문맥 의존적 (로봇 어드바이저 vs 정부 보조금 지급의 기준이 다름)

## 2. AI 성능 측정

### 2.1 Turing Test / Imitation Game
사람과 AI를 구분할 수 없으면 지능 인정. ChatGPT는 현재 통과.

### 2.2 도메인별 성능 차이
| 도메인 | 상태 |
|--------|------|
| 언어 (대화·번역) | 인간 수준 초과 |
| 바둑·체스 | 인간 초월 |
| 영상 분류 | 인간급 |
| 의료 진단 | 일부 초과, 일부 미달 |
| 수술 로봇 | 인간 수준 미달 |

### 2.3 Singularity
기계가 모든 면에서 인간을 압도하는 시점. 폰 노이만이 제안. 도메인별로 점진적 도래 예상.

## 3. 머신러닝의 정의 (Tom Mitchell, 1997)

> "컴퓨터 프로그램이 경험 E로부터 배워, 태스크 T에서 성능 P가 향상될 때 학습하는 것이다."

### 3.1 학습 3패러다임
| 패러다임 | 설명 | 비유 |
|----------|------|------|
| Supervised (지도) | 입력-정답 쌍으로 학습 | 문제집 + 정답지 |
| Unsupervised (비지도) | 구조·분포 자체 학습 | 교과서만 읽기 |
| Reinforcement (강화) | 보상/벌로 학습 | 사랑의 매 + 상장 |

## 4. 데이터 분할과 역할

| 세트 | 용도 | 규칙 |
|------|------|------|
| **Train** | 파라미터 학습 | 자유롭게 사용 |
| **Validation (Dev)** | 하이퍼파라미터·모델 선택 | 여러 번 참조 가능 |
| **Test** | 일반화 성능 평가 | "금고 보관", 오염 금지 |

## 5. 일반화(Generalization)

### 5.1 정의
**학습 시 보지 못한 데이터에서의 성능.** 머신러닝의 궁극 목표. True Error는 알 수 없으므로 Test Error로 근사한다.

### 5.2 VC Generalization Bound
```
P(|E_train - E_test| > ε) ≤ O( N^{DVC} · exp(-N) )
```
- N : 데이터 수
- DVC : VC Dimension (모델 복잡도 측도)
- 의미 : N ↑ 이면 일반화 갭 확률 0 수렴. DVC ↓ 이면 단순한 모델.
- "지옥에 가서라도 데이터를 가져와라" — 빅데이터의 중요성.

### 5.3 Overfitting vs Underfitting
```
모델 복잡도
     │   ┌─ Underfit
   E │  ╱
   r │ ╱      ┌─ Sweet spot
   r │╱       │     ┌─ Overfit
   o │        │    ╱
   r │ ─ ─ ─ ─└─ ─ ╱ ─ ─ ─
     └─────────────────►
```
- **Overfit** : 복잡도 > 데이터 → 노이즈까지 학습
- **Underfit** : 복잡도 < → 신호도 못 학습
- **Sweet spot** : 복잡도 트레이드오프 관리

### 5.4 Occam's Razor
"쓸데없이 곱하지 말라" — 필요 이상 복잡하지 않은 모델을 선호. 아인슈타인의 "Keep It Simple, Stupid".

## 6. 핵심 정리

1. AI 성능은 도메인별 큰 차이 (언어 ≫ 수술).
2. AI 3세대 : 규칙 → ML(특징 공학) → 딥러닝(표현 학습).
3. 머신러닝 = "경험 E로 태스크 T의 성능 P를 향상시키는 프로그램".
4. 3패러다임 : Supervised / Unsupervised / Reinforcement.
5. Train / Val / Test 분리 엄수, Test 는 금고 보관.
6. 일반화 = 학습 미보유 데이터에서의 성능.
7. VC Bound : N ↑, DVC ↓ 일수록 일반화 유리.
8. Overfit vs Underfit = 모델 복잡도 trade-off.
9. Occam's Razor = 단순한 모델이 일반화에 유리.
10. Rationality = 기대 효용 최대화, 문맥 의존적.
11. Turing Test는 여전히 실질적 지표.
12. Singularity = 기계가 모든 면에서 인간 초월. 도메인별 도래.
13. 빅데이터 + 딥러닝이 2012년 이후 폭발의 엔진.
14. "무엇이 중안디?" = 1세대(사람 결정) vs 3세대(자동 결정).
15. AI 겨울은 기대-현실 갭의 반복, 각 회복은 패러다임 시프트.

### 강의 중 주요 직관·일화
- **곡성 "무엇이 중안디?"** : 1세대(규칙)와 3세대(자동 특징) 차이를 한 마디로.
- **Geoffrey Hinton RBM 유레카** : 새벽에 아내 깨우며 계산. (결혼한 분들은 큰일 남을 안다는 농담).
- **Occam's Razor** : 윌리엄 오브 옥캄 "수도원 탈출" 일화.
- **석촌 호수 깊이 측정** : 100 m 단위는 가능, 1 mm 단위는 불가능 — "측정 불가능한 것은 개선 불가".
- **아큐라 레전드 부품 남음** : 아인슈타인의 "Keep It Simple" 과 연결.
