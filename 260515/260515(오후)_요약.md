# 1. 시험 후보 직접 안내

강사가 오후 시작 부분에서 시험에 나올 후보를 직접 좁혀 주었다.  
말하지 않은 것은 시험에 나오지 않고, 말한 후보 중 일부가 나온다고 설명했다.

## 1.1 후보 1 — Cross Entropy와 KL Divergence

### 핵심 답

Cross entropy는 entropy와 KL divergence의 합으로 표현된다.

```text
H(p, p_hat) = H(p) + D_KL(p || p_hat)
```

- 여기서 `p`는 true distribution.
- `p_hat`은 model prediction distribution.
- 시험에서는 유도까지 요구하지는 않을 가능성이 높다고 설명했다.

### Side question

Training database를 one-hot vector로 representation하면 entropy는 0이다.

이유:

- one-hot distribution은 한 클래스 확률이 1이고 나머지는 0이다.
- 불확실성이 없으므로 entropy가 0이다.

## 1.2 후보 2 — Vanilla Encoder-Decoder와 Streaming

### 질문 형태

Vanilla encoder-decoder approach로 ASR을 하면 streaming speech recognition이 안 되는 이유는 무엇인가?

### 핵심 답

- Vanilla encoder-decoder는 encoding이 모두 끝난 뒤 decoder가 동작한다.
- decoder가 output text를 생성한다.
- 따라서 utterance 전체를 다 듣고 encoding이 끝나기 전까지 첫 token도 생성할 수 없다.
- 그러므로 streaming ASR에 적합하지 않다.

## 1.3 후보 2의 Side question — MoChA

Streaming 가능한 encoder-decoder 변형으로 **MoChA**를 언급했다.

- MoChA = Monotonic Chunkwise Attention.
- 삼성전자 DX에서 처음 제품화한 방식이라고 언급.
- encoder-decoder 계열이지만 streaming이 가능하도록 설계된 attention 방식.

## 1.4 후보 3 — Shallow Fusion

### 질문 형태

Domain-specific LM이 있을 때 ASR 성능을 어떻게 높일 수 있는가?

### 핵심 답

- ASR model에서 얻은 log probability와 domain-specific LM에서 얻은 log probability를 linear interpolation한다.
- 이를 통해 domain-specific language model 정보를 decoding에 반영한다.
- 수식까지 적을 필요는 없고, 두 log probability를 섞는다는 점이 핵심이다.

```text
ASR log probability + LM log probability를 선형 결합
```

## 1.5 후보 4 — wav2vec 2.0

CTC는 이미 다른 수업에서 배웠다고 하여 시험 후보에서 제외하는 흐름으로 바뀌었다.  
대신 wav2vec 2.0이 중요한 후보로 지정되었다.

### 질문 형태

wav2vec 2.0은 BERT의 masked language modeling 아이디어와 유사하지만 speech에는 명확한 token이 없다. 이 문제를 어떻게 해결하고 어떤 loss를 사용하는가?

### 핵심 답

- Speech는 text와 달리 token boundary가 명확하지 않고 value가 continuous하다.
- 따라서 BERT처럼 masked token을 cross entropy로 바로 맞히기 어렵다.
- wav2vec 2.0은 vector quantization으로 continuous speech representation을 codebook의 discrete code vector로 매핑한다.
- codebook/code vector를 학습하기 위해 contrastive loss를 사용한다.
- code vector 선택이 일부 code에 쏠리는 것을 막기 위해 diversity loss를 함께 사용한다.
- diversity loss는 negative entropy 형태이며, code vector selection probability가 uniform distribution에 가까워지도록 유도한다.

# 2. Cross Entropy / Entropy / KL

## 2.1 Cross Entropy decomposition

Cross entropy는 다음처럼 entropy와 KL divergence의 합이다.

```text
H(p, q) = H(p) + D_KL(p || q)
```

해석:

- `H(p)`는 true distribution 자체의 불확실성.
- `D_KL(p || q)`는 model distribution이 true distribution과 얼마나 다른지.
- true distribution이 고정되어 있으면 cross entropy를 최소화하는 것은 KL divergence를 최소화하는 것과 같다.

## 2.2 One-hot entropy

- one-hot label은 정답 class 하나의 확률이 1이고 나머지가 0이다.
- 불확실성이 없다.
- 따라서 entropy는 0이다.

## 2.3 시험 포인트

- `Cross entropy = entropy + KL divergence`.
- one-hot distribution의 entropy는 0.
- 유도보다는 관계식과 직관을 우선 외우면 된다.

# 3. Encoder-Decoder와 Streaming

## 3.1 Vanilla Encoder-Decoder

- Encoder가 input sequence 전체를 encoding한다.
- Decoder는 encoder output을 보고 output sequence를 생성한다.
- 기본 구조에서는 encoding이 끝나야 decoding이 시작된다.

## 3.2 Streaming이 안 되는 이유

- Streaming ASR은 말하는 중간에 바로 text를 생성해야 한다.
- Vanilla encoder-decoder는 utterance 전체 encoding이 끝나기 전까지 decoder가 시작되지 않는다.
- 따라서 첫 token도 생성할 수 없다.
- 구조적으로 streaming에 부적합하다.

## 3.3 MoChA

- Monotonic Chunkwise Attention.
- attention을 monotonic하게 진행시키고 chunk 단위로 처리한다.
- 전체 utterance를 기다리지 않고 일부 chunk를 보며 decoding할 수 있다.
- streaming 가능한 encoder-decoder 변형이다.

# 4. Shallow Fusion

## 4.1 문제 상황

- ASR model은 일반적인 speech-to-text 확률을 낸다.
- 특정 domain에서는 domain-specific vocabulary와 표현이 중요하다.
- 예: 의료, 법률, 사내 제품명, 특정 서비스 명령어.

## 4.2 Shallow Fusion 방식

- ASR model log probability.
- Domain-specific LM log probability.
- 두 값을 linear interpolation 또는 weighted sum으로 결합한다.

## 4.3 효과

- 음향적으로 비슷한 후보 중 domain-specific LM이 더 자연스러운 후보에 높은 점수를 준다.
- 특정 도메인의 고유명사, 표현, 명령어를 더 잘 인식하게 된다.

## 4.4 시험 포인트

- 수식보다 “ASR log probability와 LM log probability를 선형 결합한다”는 설명이 중요하다.

# 5. Named Entity Correction

## 5.1 제품화된 ASR의 후처리

- 실제 상용 ASR은 model output만 그대로 쓰지 않는다.
- Named entity recognition/correction을 후처리로 사용한다.

## 5.2 방식

- ASR text에서 named entity에 해당하는 부분을 찾는다.
- 별도 named entity server/list와 matching한다.
- 살짝 잘못 인식된 고유명사를 교정한다.

## 5.3 의미

- 상용 음성 인식 시스템은 neural model 단독이 아니라 여러 후처리/보정 시스템이 결합된 pipeline이다.

# 6. Conformer

## 6.1 Transformer의 장단점

- Transformer는 long-term/global context를 잘 잡는다.
- self-attention은 먼 위치도 직접 참조할 수 있다.
- 하지만 positional encoding 없이는 permutation equivariant 성질 때문에 순서 정보가 약하다.

## 6.2 Speech에서 CNN이 필요한 이유

- 음성 인식은 매우 긴 context보다 짧은 local context가 중요할 때가 많다.
- 발음, 음소 변화, 짧은 구간의 acoustic pattern을 잘 잡아야 한다.
- CNN은 filter 길이 안에서 local variation을 잘 포착한다.

## 6.3 Conformer

- Convolution-augmented Transformer.
- Transformer의 global context와 CNN의 local context modeling을 결합한다.
- 음성 인식에서 Transformer보다 Conformer를 많이 쓰는 경우가 많다.

# 7. CTC와 Transducer 간단 정리

## 7.1 CTC

- Encoder-only 구조에서 ASR을 수행하는 방법.
- output sequence와 target sequence 길이가 다르고 alignment가 없을 때 사용한다.
- 가능한 모든 alignment의 probability를 합산한다.
- blank token을 도입한다.
- blank는 non-speech portion modeling뿐 아니라 real token 사이의 boundary 역할도 한다.
- conditional independence assumption을 사용한다.

## 7.2 CTC의 한계

- 과거 인식 결과가 현재 인식 결과에 영향을 주지 않는다.
- 강한 independence assumption 때문에 성능 손실이 생길 수 있다.

## 7.3 Transducer

- CTC의 conditional independence assumption을 완화한다.
- autoregressive feedback 구조가 있다.
- joint network에서 dimension이 하나 더 커진다.
- training complexity가 증가하고 inference parallelization이 어렵다.

# 8. wav2vec 2.0

## 8.1 BERT와의 유사성

- BERT는 masked language modeling으로 self-supervised pretraining을 한다.
- 입력 token 일부를 masking하고, 원래 token을 맞힌다.
- text는 token boundary와 class가 명확하므로 cross entropy loss를 쓸 수 있다.

## 8.2 Speech에 MLM을 바로 적용하기 어려운 이유

Speech는 text와 다르다.

1. **Token boundary가 명확하지 않다**
   - waveform에서 어디부터 어디까지가 한 token인지 불분명하다.

2. **Value가 continuous하다**
   - speech feature는 연속값이다.
   - text token처럼 discrete class가 아니다.

3. **Cross entropy를 바로 쓰기 어렵다**
   - class가 정의되어 있어야 cross entropy를 쓸 수 있는데 speech에는 명확한 class가 없다.

## 8.3 Vector Quantization

- continuous speech representation을 codebook 안의 code vector 중 가장 가까운 것으로 매핑한다.
- codebook은 code vectors의 리스트다.
- speech feature가 들어오면 가장 가까운 code vector를 찾고, 그 index를 discrete class처럼 사용한다.
- 이를 통해 continuous speech를 discrete representation으로 바꿀 수 있다.

## 8.4 Contrastive Loss

- wav2vec 2.0은 contrastive loss를 사용한다.
- 강의에서는 InfoNCE loss를 언급했다.
- masked position의 true quantized representation을 positive로 두고, 다른 후보들을 negative로 두어 구분하도록 학습한다.
- codebook/code vector를 학습하는 데 사용된다.

## 8.5 Diversity Loss

- contrastive loss만 쓰면 codebook의 일부 code vector만 자주 선택될 수 있다.
- 좋은 codebook은 code vector들이 고르게 사용되어야 한다.
- diversity loss는 code vector selection probability가 uniform distribution에 가까워지도록 한다.
- 강의에서는 diversity loss가 negative entropy라고 설명했다.
- entropy는 uniform distribution에서 최대이고, negative entropy는 uniform distribution에서 최소가 된다.

## 8.6 Gumbel Softmax Trick

- 가장 가까운 code vector를 고르는 argmax는 differentiable하지 않다.
- backpropagation이 어렵다.
- Gumbel Softmax trick을 사용해 argmax/categorical selection을 differentiable하게 근사한다.

## 8.7 wav2vec 2.0의 장점

- unlabeled speech로 pretraining한다.
- 적은 labeled data로 fine-tuning해도 좋은 ASR 성능이 나온다.
- labeled speech transcription 비용을 크게 줄일 수 있다.

# 9. Spoken Language Understanding (SLU)

## 9.1 SLU의 의미

- Spoken Language Understanding은 voice assistant의 핵심이다.
- 사용자의 음성 발화로부터 의도를 이해한다.
- 보통 intent와 slot을 추출한다.

예:

```text
"Play that album"
Intent: PlayMusic
Slot: album = that album
```

## 9.2 JointBERT

- text NLU에서는 BERT를 fine-tuning해 intent와 slot을 동시에 예측할 수 있다.
- `[CLS]` token representation으로 intent를 분류한다.
- token/span 위치에서 slot을 검출한다.

## 9.3 Cascaded Model

가장 단순한 SLU 방식:

```text
speech → ASR → text → NLU → intent/slot
```

- 현재 상용 voice assistant도 이런 cascaded pipeline을 많이 쓴다.

## 9.4 Cascaded Model의 문제

1. **Compounding Error**
   - ASR error가 NLU에서 고쳐지지 않고 오히려 더 큰 error로 이어질 수 있다.

2. **Latency**
   - ASR이 끝나야 NLU가 시작된다.
   - pipeline 단계가 많을수록 latency가 누적된다.

3. **Compute/Memory Cost**
   - 여러 모델을 따로 운영해야 한다.

## 9.5 End-to-End SLU

- waveform에서 바로 intent/slot까지 예측하는 모델.
- error propagation과 latency를 줄일 수 있다.
- 하지만 labeled SLU data가 많지 않아 학습이 어렵다.
- wav2vec 2.0과 BERT처럼 self-supervised pretrained model을 붙이고, adaptation/cross-attention layer를 넣어 fine-tuning하는 방식이 가능하다.

# 13. 오후 강의 핵심 정리

## 13.1 반드시 외울 것

1. **Cross Entropy**
   - `H(p, p_hat) = H(p) + D_KL(p || p_hat)`.
   - one-hot entropy는 0.

2. **Vanilla Encoder-Decoder Streaming 한계**
   - encoding이 끝나야 decoding 시작.
   - 첫 token도 전체 utterance 이후 생성.

3. **MoChA**
   - streaming 가능한 encoder-decoder 변형.

4. **Shallow Fusion**
   - ASR log probability와 domain-specific LM log probability를 linear interpolation.

5. **wav2vec 2.0**
   - speech에는 명확한 discrete token이 없음.
   - vector quantization/codebook.
   - contrastive loss.
   - diversity loss = negative entropy.
   - Gumbel Softmax trick.

6. **SLU**
   - intent + slot.
   - cascaded ASR→NLU는 compounding error와 latency 문제.
   - end-to-end SLU는 데이터 부족이 문제.

## 13.2 예상 시험 문제 스타일

- Cross entropy를 entropy와 KL divergence로 표현하라.
- one-hot distribution의 entropy가 왜 0인지 설명하라.
- vanilla encoder-decoder가 streaming ASR에 부적합한 이유를 설명하라.
- MoChA가 무엇인지 설명하라.
- Shallow Fusion으로 domain-specific LM을 ASR에 결합하는 방식을 설명하라.
- wav2vec 2.0에서 speech MLM을 위해 contrastive loss와 diversity loss를 쓰는 이유를 설명하라.
- cascaded SLU와 end-to-end SLU의 장단점을 설명하라.
