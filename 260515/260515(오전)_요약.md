# 1. CNN을 배우는 이유

## 1.1 Feedforward Network의 한계

- 전날 배운 feedforward neural network는 가장 기본적인 딥러닝 모델이다.
- 작은 MNIST 이미지처럼 28×28 정도는 MLP로 처리할 수 있다.
- 하지만 실제 이미지는 1000×1000 이상인 경우가 많다.
- 1000×1000 이미지는 픽셀 100만 개다.
- 이것을 MLP에 넣고 hidden node 100개만 둬도 첫 레이어 weight가 1억 개 수준으로 커진다.
- 레이어가 깊어질수록 파라미터, 데이터 요구량, 학습 시간이 폭증한다.

## 1.2 Spatial Translation 문제

- 학습 이미지에서는 강아지가 왼쪽 아래에 있었는데, 테스트 이미지에서는 오른쪽에 있을 수 있다.
- MLP는 이미지를 flatten하기 때문에 위치가 바뀌면 완전히 다른 input pattern처럼 보인다.
- 하지만 사람은 위치가 조금 바뀌어도 같은 강아지로 인식한다.
- CNN은 같은 local pattern을 이미지 전체에서 찾도록 설계되어 translation에 강하다.

## 1.3 CNN이 잘 맞는 데이터

- CNN은 grid 형태 데이터에 잘 작동한다.
- 대표적으로 이미지, 비디오, 시계열, 스펙트로그램, CT scan, skeleton motion 등이 있다.

# 2. CNN의 핵심 아이디어

## 2.1 MLP와 CNN의 차이

- MLP의 핵심 연산은 matrix-vector multiplication이다.

```text
h = W x
```

- CNN은 이 연산을 convolution으로 대체한다.
- loss function, backpropagation, gradient descent 같은 학습 원리는 거의 동일하다.
- 핵심 차이는 “fully connected”가 아니라 “local kernel을 공유하며 훑는다”는 점이다.

## 2.2 Convolution의 직관

- Convolution은 가중평균 또는 weighted sum으로 볼 수 있다.
- 센서 노이즈를 줄이기 위해 현재 측정값과 주변 시간 측정값을 가중평균하는 예시로 설명했다.
- 딥러닝에서는 continuous integral보다 discrete sum 형태로 사용한다.

## 2.3 2D Convolution 예시

- input image가 3×4이고 kernel이 2×2라고 하자.
- kernel을 좌측 상단에 올려놓고 대응되는 값끼리 곱해 더한다.
- 그 값을 output feature map의 첫 위치에 저장한다.
- kernel을 오른쪽으로 한 칸씩 이동하며 반복하고, 끝나면 아래로 한 칸 내려간다.
- 이 예시에서는 3×4 input과 2×2 kernel로 2×3 output이 나온다.

## 2.4 Feature map

- convolution 결과로 나오는 output image를 feature map이라고 부른다.
- kernel 하나는 feature map 한 장을 만든다.
- kernel을 여러 개 쓰면 feature map도 여러 channel로 나온다.

# 3. Convolution을 Neural Network 관점에서 보기

## 3.1 Local connectivity

- MLP에서는 output node 하나가 모든 input node와 연결된다.
- CNN에서는 output node 하나가 input의 일부 local patch와만 연결된다.
- 예: 2×2 kernel이면 output 하나는 input 4개와만 연결된다.

## 3.2 Parameter sharing

- CNN은 같은 kernel weight를 모든 위치에서 공유한다.
- 위치마다 다른 weight를 쓰는 것이 아니라 같은 pattern detector를 이미지 전체에 적용한다.
- 이 때문에 파라미터 수가 크게 줄어든다.

## 3.3 효율성

- 일반 MLP는 input 크기와 output 크기를 곱한 만큼 파라미터가 필요하다.
- CNN은 kernel 크기만큼의 파라미터를 공유한다.
- 예시에서 MLP는 수십억 개 파라미터가 필요할 수 있지만, convolution은 kernel 몇 개로 처리 가능하다.

# 4. CNN의 세 가지 메인 아이디어

## 4.1 Sparse Interaction

- 각 output이 모든 input을 보는 것이 아니라 일부 local input만 본다.
- 파라미터 수와 연산량을 줄인다.
- 레이어를 깊게 쌓으면 receptive field가 커져 결국 넓은 영역 정보를 반영할 수 있다.

## 4.2 Parameter Sharing

- 같은 kernel을 여러 위치에서 반복 사용한다.
- 같은 물체나 edge pattern이 이미지 어디에 있든 감지할 수 있다.
- 데이터 효율성과 일반화에 유리하다.

## 4.3 Equivariant Representation

- 입력 이미지가 이동하면 convolution 결과도 같은 방식으로 이동한다.
- 즉, convolution 후 shift한 것과 shift 후 convolution한 것이 대응된다.
- pooling과 결합하면 작은 위치 이동에 robust해진다.

---

# 5. Edge Detection 관점

## 5.1 전통 Computer Vision과 연결

- 딥러닝 이전에도 computer vision에서는 edge detection이 중요했다.
- 물체의 경계선을 찾고, 그 경계선 모양으로 물체를 인식하려 했다.

## 5.2 Kernel이 edge detector 역할

- 예를 들어 `[1, -1]` kernel을 사용하면 좌우 픽셀 차이가 큰 위치에서 큰 값이 나온다.
- 흰색-흰색 또는 검정-검정처럼 변화가 없는 구간은 0에 가깝다.
- 흰색과 검정이 만나는 경계에서는 큰 양수/음수가 나온다.
- CNN에서는 이런 kernel을 사람이 직접 설계하는 것이 아니라 학습으로 자동 획득한다.

# 6. Pooling

## 6.1 Pooling의 의미

- 어떤 영역의 값을 요약해서 하나의 값으로 만든다.
- 대표적으로 max pooling이 있다.
- 4×4 영역에서 max pooling을 하면 그 영역의 최대값 하나만 남긴다.

## 6.2 Pooling의 효과

- output 크기를 줄인다.
- 계산량을 줄인다.
- local translation에 둔감해진다.
- 물체가 조금 이동해도 같은 pooling window 안에 있으면 max 값이 유지될 수 있다.

## 6.3 Pooling 종류

- Max pooling.
- Average pooling.
- L2 norm pooling.
- 중심에 가까울수록 큰 weight를 주는 weighted pooling.

---

# 7. 일반적인 CNN 블록

## 7.1 기본 반복 구조

CNN은 보통 다음 블록을 반복한다.

```text
Convolution → ReLU → Pooling
```

- Convolution: feature extraction.
- ReLU: nonlinearity.
- Pooling: spatial downsampling / robustness.

## 7.2 마지막 분류기

- 여러 convolution/pooling layer 후 feature map을 flatten한다.
- 그 다음 MLP layer와 softmax classifier를 붙인다.
- 즉 CNN 앞부분은 feature extractor, 뒷부분은 classifier로 볼 수 있다.

# 8. Channel과 Multiple Kernel

## 8.1 RGB 이미지

- 실제 컬러 이미지는 height × width × 3 형태다.
- RGB channel 3개가 있기 때문이다.

## 8.2 Kernel depth

- input channel이 3개라면 kernel도 depth가 3이다.
- 예: 2×2 kernel을 RGB 이미지에 적용하면 kernel shape은 2×2×3이다.
- 이 kernel 하나가 훑고 지나가면 output feature map 한 장이 나온다.

## 8.3 Output channel

- output이 256×256×64라면 kernel을 64개 사용했다는 뜻이다.
- kernel 하나가 feature map 한 장을 만들고, 64개 kernel이 64 channel을 만든다.

---

# 9. CNN Architecture 읽는 법

## 9.1 예시 구조

```text
Input: 256 × 256 × 3
Conv + ReLU → 256 × 256 × 64
Pooling stride 4 → 64 × 64 × 64
Conv + ReLU → 64 × 64 × 64
Pooling stride 4 → 16 × 16 × 64
Flatten → 16384
MLP → 1000
Softmax → 1000 classes
```

## 9.2 해석 포인트

- 마지막 dimension은 channel 또는 feature map 개수다.
- convolution 후 channel 수가 커지는 것은 kernel 개수가 많다는 뜻이다.
- pooling은 보통 width/height를 줄이고 channel 수는 유지한다.
- classifier의 output 차원은 class 개수와 맞춘다.

# 10. Stride와 Zero Padding

## 10.1 Stride

- kernel을 몇 칸씩 이동할지 정하는 값.
- stride 1은 한 칸씩 이동.
- stride 2는 두 칸씩 이동.
- stride가 커지면 output spatial size가 작아진다.
- pooling뿐 아니라 convolution에도 stride를 줄 수 있다.

## 10.2 Zero Padding

- convolution을 반복하면 feature map 크기가 계속 줄어든다.
- 크기가 계속 줄면 깊은 네트워크를 쌓기 어렵다.
- zero padding은 input 주변에 가상의 0 값을 붙인다.
- 이를 통해 input과 output의 spatial size를 같게 유지할 수 있다.
- 실제 CNN에서는 기본적으로 많이 사용한다.

---

# 11. 변형 Convolution

## 11.1 Locally Connected Layer

- local patch와만 연결한다는 점은 convolution과 비슷하다.
- 하지만 위치마다 parameter를 공유하지 않는다.
- 즉 unshared convolution처럼 동작한다.
- 특정 위치마다 다른 filter를 쓰고 싶을 때 쓸 수 있지만 일반적으로 많이 쓰지는 않는다.

## 11.2 Partial Connectivity

- 모든 input channel을 모든 output channel에 연결하지 않는다.
- channel 간 독립성이 있다고 가정할 수 있을 때 parameter를 줄이기 위해 쓸 수 있다.
- RGB처럼 channel이 강하게 연관된 경우에는 일반적으로 적합하지 않다.

## 11.3 Tiled Convolution

- 모든 위치에서 같은 parameter를 공유하지 않고, 정해진 pattern으로 parameter를 반복 공유한다.
- full sharing과 no sharing의 중간 형태다.

---

# 12. Transposed Convolution

## 12.1 목적

- 일반 convolution/pooling은 spatial size를 줄인다.
- transposed convolution은 반대로 작은 feature에서 큰 image/feature map을 만든다.
- 이미지 생성 모델, GAN, decoder 구조에서 많이 쓰인다.

## 12.2 직관

- input의 한 값에 kernel을 곱해 output 영역에 뿌린다.
- 여러 input 위치에서 뿌린 값이 output에서 겹치면 더한다.
- stride를 조절해 output 크기를 키울 수 있다.

## 12.3 용어

- Transposed convolution.
- Up convolution.
- Fractionally strided convolution.
- Deconvolution이라고 부르기도 하지만 엄밀히 같은 의미는 아니다.

# 14. 대표 CNN Architecture

## 14.1 AlexNet

- ImageNet에서 deep learning/CNN의 가능성을 크게 보여준 모델.
- 227×227×3 input.
- 큰 kernel, convolution, pooling, fully connected layer 조합.
- 8개 layer 수준.
- 기존 shallow model 대비 error를 크게 낮췄다.

## 14.2 VGG

- 3×3 작은 kernel을 여러 개 깊게 쌓는 아이디어.
- 큰 kernel 하나보다 작은 kernel 여러 층이 더 좋을 수 있음을 보여줬다.
- 구조가 단순하고 직관적이다.
- VGG16은 layer 16개를 의미한다.

## 14.3 GoogLeNet / Inception

- Inception module 사용.
- 여러 branch에서 서로 다른 연산을 수행한 뒤 concatenate한다.
- 1×1 convolution을 적극 사용해 channel dimension과 computation을 줄인다.
- global average pooling으로 마지막 fully connected parameter를 크게 줄인다.

## 14.4 1×1 Convolution

- spatial 위치 하나만 보지만 channel 전체를 본다.
- channel들을 weighted average 또는 mixing한다.
- 정보를 어느 정도 보존하면서 channel 수를 줄이는 데 사용한다.
- 연산량과 파라미터 수를 크게 줄일 수 있다.

## 14.5 ResNet

- residual/shortcut connection 사용.
- block output이 `F(x) + x` 형태가 된다.
- 전체 함수를 직접 학습하기보다 input 대비 residual만 학습하면 된다.
- 매우 깊은 네트워크 학습을 가능하게 했다.
- ResNet-152처럼 깊은 모델이 가능해졌다.

## 14.6 EfficientNet

- depth, width, resolution을 따로따로 조절하지 않고 함께 scaling한다.
- compound scaling 아이디어.
- 모델 크기와 성능의 trade-off를 효율적으로 맞춘다.

# 15. CNN 설계에 대한 강사 코멘트

## 15.1 Trial and Error

- CNN architecture 설계는 이론적으로 딱 정답이 정해져 있지 않다.
- 데이터와 task에 따라 좋은 depth, width, kernel size가 달라진다.
- 실험을 많이 해보며 찾는 경우가 많다.

## 15.2 실무적 접근

- 백지에서 모두 실험하지 않는다.
- 내가 풀 문제와 비슷한 최신 연구/논문에서 사용한 모델을 default로 시작한다.
- 그 모델을 조금씩 바꿔가며 실험한다.
- GPU 리소스가 많을수록 더 많은 trial을 병렬로 해볼 수 있다.

# 16. OH — Homework 1 안내

## 16.1 과제 주제

- Credit card fraud detection.
- 각 transaction이 fraud인지 normal인지 binary classification.
- 입력 feature는 28개 numeric feature.
- `Time`, `Amount` feature가 별도로 존재.
- label `Class`: fraud = 1, normal = 0.

## 16.2 핵심 난점 — Class Imbalance

- 전체 28만 개 중 fraud는 약 500개 수준.
- normal이 압도적으로 많다.
- accuracy만 보면 모두 normal로 예측해도 99% 이상처럼 보일 수 있다.
- 따라서 accuracy는 적절하지 않다.

## 16.3 요구 사항

1. **Imbalance 처리**
   - normal under-sampling.
   - fraud over-sampling / augmentation.
   - class-weighted loss.
   - 데이터 조정 후 loss weighting을 섞어도 됨.

2. **Model structure 고정**
   - MLP/FNN 사용.
   - hidden size: 64 → 32 → 16.
   - ReLU activation.
   - Adam optimizer.
   - Cross entropy 또는 binary classification에 맞는 sigmoid 구조.

3. **Feature preprocessing**
   - feature scale이 다르므로 standardization/normalization 필요.
   - `Amount`, `Time`을 사용할지 drop할지는 판단 가능.

## 16.4 Evaluation

- Accuracy 대신 F1 score 중심.
- Precision과 Recall을 동시에 고려.
- fraud를 얼마나 잘 detect했는지가 중요하다.

# 17. 오전 강의 핵심 정리

## 17.1 반드시 외울 것

1. CNN은 MLP의 matrix multiplication을 convolution으로 대체한다.
2. Convolution은 kernel을 sliding하며 local weighted sum을 계산한다.
3. Sparse interaction은 일부 input만 연결하는 것.
4. Parameter sharing은 같은 kernel을 모든 위치에 공유하는 것.
5. Pooling은 downsampling과 small translation robustness를 준다.
6. Zero padding은 output size를 유지하기 위해 input 주변에 0을 붙인다.
7. 1×1 convolution은 channel mixing/dimension reduction에 사용한다.
8. ResNet의 핵심은 residual connection `F(x)+x`.

## 17.2 예상 시험 문제 스타일

- CNN이 MLP보다 이미지에 적합한 이유를 설명하라.
- Convolution 연산을 간단한 예시로 설명하라.
- Sparse interaction과 parameter sharing의 의미를 설명하라.
- Pooling이 translation robustness에 도움 되는 이유를 설명하라.
- Zero padding과 stride의 역할을 설명하라.
- 1×1 convolution의 목적을 설명하라.
- ResNet의 residual connection이 왜 도움이 되는지 설명하라.
