## 1. 복습 : 생성형 AI가 풀려는 문제

### 1.1 P(x) 추정이라는 근본 문제
생성형 AI의 핵심은 데이터 분포 **P(x)** 를 학습하여 그 분포에서 새 샘플을 뽑는 일이다.
고차원(D가 매우 큼)에서는 임의로 x를 찍어 샘플링하는 것 자체는 쉽지만, 그렇게 뽑은 샘플이 실제로 **의미 있을 확률 P(x)가 0이 아닐 확률이 매우 낮다**. 고차원 P(x)의 직접 추정이 생성형 AI 난이도의 본질이다.

### 1.2 P(x|z) vs P(z|x)
- **P(x|z)** : z가 주어졌을 때 x의 분포 → 생성(Decoder)
- **P(z|x)** : x가 주어졌을 때 z의 분포 → 추론(Encoder)

일반적으로 x가 관측, z가 미관측일 때 **P(z|x)** 가 직접 계산이 더 어려워 **근사(approximation)** 가 필요하다. 어떤 조건부가 더 쉬운지는 상황에 따라 바뀐다.

### 1.3 해결 아이디어 : 뽑기 쉬운 분포에서 출발
x를 직접 뽑기 어렵다면, **뽑기 쉬운 분포 P(z)** (가우시안, 유니폼 등)에서 "먼지" z를 뽑은 뒤, 매우 강력한 비선형 함수 근사기인 **Decoder(DNN)** 에 통과시켜 "손오공"(의미 있는 x)을 만든다.

```
z ~ P(z)    →    Decoder p_θ(x|z)    →    x̃   ("손오공")
```

## 2. Deep Generative Model의 분류 지도

딥러닝으로 P(x)를 근사하는 모델은 크게 둘로 나뉜다.

| 구분 | 특징 | 대표 모델 |
|------|------|-----------|
| **Explicit (Likelihood-based)** | P(x), P(x|z), P(z|x) 를 **명시적으로 모델링** | Autoregressive, **VAE**, Flow, Diffusion |
| **Implicit** | 확률분포를 명시적으로 쓰지 않고 샘플링 능력만 학습 | **GAN** |

### 2.1 Autoregressive Model
다변량 x 의 결합분포를 **연쇄법칙(chain rule)** 으로 분해한다.
```
P(x) = P(x₁) · P(x₂|x₁) · P(x₃|x₁,x₂) · … · P(x_D|x₁,…,x_{D-1})
```
언어 모델(GPT 등)이 대표 사례로, 첫 토큰을 뽑고 그 토큰을 조건으로 다음 토큰을 뽑아 나간다. 퀄리티는 좋지만 **순차적 의존성 때문에 생성이 느리다**. Transformer + Positional Embedding으로 병렬 처리를 해결한 이유이다.

### 2.2 Latent Variable Model (VAE 계열)
체인 룰과 전적분(marginalization)으로 잠재변수 z를 도입한다.
```
P(x) = ∫ P(x|z) · P(z) dz
```
이 한 식을 풀기 위해 해결해야 할 세 가지 문제가 있다.
1. **P(z)** — 사람이 간단한 분포로 설계 (가우시안)
2. **P(x|z)** — 디코더 DNN이 학습
3. **∫ (적분)** — **Variational Inference(VI)** 로 근사

VAE는 이 틀을 "Helmholtz Machine" 계열에서 진화시킨 현대적 구현이다.

### 2.3 GAN, Flow, Diffusion
- **GAN** : 확률분포를 명시적으로 쓰지 않고, Generator vs Discriminator의 minimax 게임으로 암묵적 학습.
- **Flow** : Encoder/Decoder를 **가역(invertible) 함수**로 두고 x ↔ z 간 정확한 왕복.
- **Diffusion** : x에 가우시안 노이즈를 점진적으로 섞은 뒤(forward), 그 **역과정 denoising** 을 학습. 현재 비전 분야의 주류이다.

역사적으로 VAE가 GAN보다 먼저였지만 한동안 GAN 품질에 밀렸고, Diffusion이 등장하면서 GAN의 중요도는 떨어지는 추세이다. 그러나 VAE는 수학적 기초가 튼튼해 **VQ-VAE 등 거대 모델의 컴포넌트**로 여전히 중요하다.

## 3. VAE의 두 가지 Underpinning Concept

### 3.1 Autoencoder
구조 : `x → Encoder → z → Decoder → x̂`
- **주인공은 Encoder**. 학습 후 디코더는 버려지고 인코더만 남아 **입력의 효율적 표현(embedding / feature)** 을 만든다.
- 용도 : downstream task의 feature extractor, manifold learning, 저차원 시각화, 클러스터링.

### 3.2 Latent Variable Generative Model : 왜 z를 도입하는가
고차원 x의 **차원 간 의존성(dependency)** 을 직접 모델링하면 파라미터 수가 지수적으로 폭발한다.
중간에 관찰되지 않는 매개체 **z (latent variable)** 를 도입해 x 차원들이 z와 1-대-1 관계만 맺도록 하면, 파라미터 수가 크게 줄고 문제가 다루기 쉬워진다.

## 4. Probabilistic Encoder / Decoder

실제 VAE의 인코더·디코더는 **x → z 직접 매핑이 아니라 분포의 파라미터를 출력** 하고 거기서 샘플링하는 확률적 모듈이다.

### 4.1 확률적 인코더 q_φ(z|x)
```
x → Encoder(φ) → (μ_z, σ_z)  →  z ~ N(μ_z, σ_z²)
```

### 4.2 확률적 디코더 p_θ(x|z)
```
z → Decoder(θ) → 분포의 파라미터  →  x̃ ~ 그 분포
```
데이터 타입에 따라 디코더가 내놓는 분포의 종류가 달라진다.

| 데이터 타입 | 디코더 출력 분포 | 디코더가 리턴하는 파라미터 |
|-------------|-------------------|----------------------------|
| 연속(continuous) | Gaussian | 평균 μ, (분산 σ²) |
| 이진(binary) | Bernoulli | 앞면 확률 **β** |
| 다중 클래스 | Categorical | 확률 벡터 **γ** |

### 4.3 파라미터 표기 관례
- 디코더(주인공) 파라미터 : **θ** (세타)
- 인코더(조연) 파라미터 : **φ** (파이)

## 5. Variational Inference : 전통 VI vs Amortized VI

### 5.1 핵심 동기 : "똘똘한 z만 남기자"
P(x) = ∫ P(x|z) P(z) dz 를 몬테카를로 샘플링으로 추정한다고 해보자. 개념적으로는 z를 많이 뽑아 평균을 내면 되지만, 고차원에서는 대부분의 z가 **P(x|z) ≈ 0** (디코더에 넣어도 쓸모없는 먼지)이 되어 추정이 엄청나게 느려진다.
따라서 "**현재 관측 x에 대해 의미 있는 z만 골라 쓰자**"는 아이디어가 필요하고, 그것이 바로 **P(z|x)** 다. 그러나 P(z|x)는 직접 계산이 불가능하므로 근사(q) 한다. 이것이 Variational Inference의 출발점이다.

### 5.2 진짜 posterior ≈ q
```
q*(z) = argmin_{q ∈ Q}  D_KL( q(z) || P(z|x) )
```
거리로는 **KL Divergence** 를 쓴다. ("**계란 전체 크기 = 크로스 엔트로피, 흰자 = KL, 노른자 = 엔트로피**" 비유로 설명되었고, VAE는 흰자(KL) 관점을 사용.)

### 5.3 전통 VI vs Amortized VI

| 구분 | Traditional VI | **Amortized VI** (VAE 채택) |
|------|----------------|------------------------------|
| 대상 | q(z) 를 **x 하나마다 따로** 최적화 | q_φ(z|x) 를 **신경망**으로 학습 |
| 파라미터 | 각 x마다 별도 | **모든 x가 공유** (Parameter Sharing) |
| 적합 상황 | 데이터가 매우 적은 전통 통계/물리 문제 | 데이터가 수천~수억 개인 딥러닝 환경 |
| 확장성 | 낮음 | 높음 (Scalable) |

**Amortized** 는 "분할 상환"의 의미. 3억원을 한 번에 내지 말고 300만원씩 100번 내듯, 여러 x에 대해 로스를 조금씩 쌓아가며 공통 파라미터 φ를 갱신한다는 관점과 정합한다. 그래서 q(z) 가 아니라 **q_φ(z|x)** 처럼 x에 대한 조건부로 적는다.

## 6. ELBO (Evidence Lower Bound) 유도와 해석

### 6.1 왜 ELBO인가
log P(x) 를 직접 최대화하고 싶지만 ∫ P(x|z) P(z) dz 는 intractable. 그래서 **log P(x) 의 하한(Lower Bound)인 ELBO** 를 대신 최대화한다.

```
log P(x) = ELBO + D_KL( q_φ(z|x) ∥ P(z|x) )
         ≥ ELBO     (KL ≥ 0)
```
좌변 log P(x)는 데이터가 주어지면 상수이므로, ELBO 최대화 = 우변 KL 최소화 = 근사 품질 향상.

### 6.2 ELBO의 두 항 분해
```
ELBO = E_{z~q_φ(z|x)} [ log p_θ(x|z) ]   −   D_KL( q_φ(z|x) ∥ p(z) )
         └─────────── ① ───────────┘       └─────────── ② ──────────┘
           Reconstruction term                Regularization term
```

1. **Reconstruction 항** — z를 디코더에 넣었을 때 원본 x가 얼마나 그럴듯하게 복원되는가. 최대화. 부호를 뒤집으면 **Training Error** 와 같다.
2. **KL 항** — 인코더 분포 q_φ(z|x) 를 간단한 prior p(z)=N(0, I) 에 가깝게 끌어당기는 **정규화** 역할. 잠재 공간이 연속적이고 부드럽게 채워지도록 한다.

### 6.3 지도학습의 "Augmented Error" 구조와 대응
VAE의 목적함수는 지도학습의 **Augmented Error = Training Error + Regularizer** 와 정확히 같은 구조이다.
- log p_θ(x|z) 최대화 ↔ Training Error 최소화
- KL(q_φ(z|x) ∥ p(z)) 최소화 ↔ Regularization

## 7. 데이터 타입별 Reconstruction Loss

```
L_VAE = L_KL + L_recon
```

### 7.1 KL 항 (Closed form)
인코더가 가우시안, prior가 N(0, I) 일 때 KL은 **닫힌 형식(closed form)** 으로 계산된다 (공학수학 표준 결과). **별도 샘플링 없이 미분 가능** 하므로 백프롭에 바로 쓸 수 있다.

### 7.2 Reconstruction 항 : 데이터 타입에 따라 구체 형태가 바뀜

| 데이터 타입 | 디코더 출력 분포 | Reconstruction Loss | 유도 근거 |
|-------------|-------------------|----------------------|-----------|
| 연속 | Gaussian N(μ, σ²I) | **MSE** (Squared Error) | log p_θ(x|z) 의 가우시안 전개에서 도출 |
| 이진 (0/1) | Bernoulli(β) | **BCE** (Binary Cross-Entropy) | −[x log β + (1−x) log(1−β)] |
| 다중 클래스 | Categorical(γ) | **Cross-Entropy** | −Σ_k x_k log γ_k |

### 7.3 Isotropic Gaussian 가정과 MSE 도출
실제 유도 시 디코더가 **isotropic Gaussian** (공분산 σ²I) 을 내놓는다고 가정하면 PDF가 간단해지고, σ²를 상수로 간주하면 log-likelihood 최대화가 정확히 **MSE 최소화** 와 같아진다. 실습에서 "VAE의 reconstruction loss가 왜 MSE/BCE/CE인가?" 의 이론적 근거이다.

### 7.4 Training 시 샘플링 vs 평균
디코더가 분포를 내놓지만 **학습 시에는 평균(μ, β, γ)만 써서 deterministic reconstruction** 을 한다 (샘플링하면 백프롭이 불안정). "분산 쪽으로는 그래디언트가 안 가나?" 라는 질문이 자주 나오는데, σ 텀이 ELBO의 정규화 상수/분산 항에 남아 있어 **미분이 여전히 σ로도 흐른다**.

## 8. Reparameterization Trick (시험 반드시 출제)

### 8.1 문제
ELBO 계산 시 z ~ q_φ(z|x) 의 **샘플링은 미분 불가능** 하다. 역전파가 막힌다 → 인코더 φ를 학습할 수 없다.

### 8.2 아이디어 : 무작위성을 입력으로 밀어내기
샘플링을 **미분 가능한 계산(deterministic computation)** 으로 치환한다.
```
원래  (미분 불가) :   z ~ N(μ_φ(x), σ_φ(x)²)
변경  (미분 가능) :   ε ~ N(0, I)   (고정된 입력 노이즈)
                       z = μ_φ(x) + σ_φ(x) ⊙ ε      (elementwise 곱)
```
두 형태가 같은 분포를 내는 건 자명하지만, 후자는 랜덤성이 **입력층(ε)** 으로 밀려 있어서 μ, σ 경로를 통해 그래디언트가 흐른다.

### 8.3 그래디언트 경로 시각화
- 원래 : 인코더 → 분포 파라미터(μ,σ) → **[랜덤 노드 z]** → 디코더 → loss
  - 랜덤 노드가 길을 막아 백프롭 불가.
- Reparam : 인코더 → (μ,σ) → **z = μ + σ ⊙ ε** → 디코더 → loss
  - ε는 외부 입력으로 처리, z까지 전부 미분 가능. 그래디언트가 인코더 파라미터 φ까지 흐른다.

### 8.4 일반성
이 트릭은 VAE 전용이 아니라 **Diffusion Model 등 다른 확률 생성 모델에서도 반복해서 등장** 하는 표준 기법이다.

## 9. Training vs Inference

### 9.1 Training
```
x → Encoder(φ) → μ, σ
ε ~ N(0, I);  z = μ + σ ⊙ ε              (Reparameterization)
z → Decoder(θ) → 분포 파라미터
L = L_KL(φ) + L_recon(θ, φ)              (데이터 타입별 MSE/BCE/CE)
SGD + Backprop → φ, θ 동시 업데이트
```

### 9.2 Inference (샘플 생성)
```
(Encoder는 버림 — "조연 퇴장")
z ~ N(0, I)  →  Decoder(θ)  →  분포에서 샘플링 → 새 샘플 x̃
```
생성 시에는 **분포에서 실제 샘플링** 을 해서 **다양성(diversity)** 을 확보한다.

## 10. Prior 설계 : 왜 N(0, I) 인가

VAE는 prior p(z) 로 **표준 가우시안 N(0, I) + diagonal covariance** 를 쓴다. 이유:

1. **샘플링이 쉽다** — 평균 근처에서 뽑으면 됨.
2. **해석 가능성 (포르쉐 비유)** — 비싼 포르쉐도 스위치 하나에 기능 하나씩. 레이턴트 차원들이 독립(uncorrelated)이어야 "이 축은 각도, 저 축은 웃음 정도" 같은 해석이 가능하다.
3. **정규화 효과** — 학습된 q_φ(z|x) 들을 N(0, I) 쪽으로 모아 레이턴트 공간을 **연속적이고 완전(continuity + completeness)** 하게 채운다. → 인퍼런스 때 임의의 N(0, I) 샘플을 넣어도 그럴듯한 x̃가 나온다.

주의 : "KL을 N(0, I) 에 거는 건 **지향점** 일 뿐, 실제로 모든 x가 그대로 N(0, I) 로 수렴하지는 않는다." 각 x마다 다른 μ, σ를 학습하므로 샘플들은 충분히 퍼져 있다.

## 11. AE vs VAE : 기하학적·개념적 차이

| 항목 | Standard Autoencoder | VAE |
|------|----------------------|-----|
| 매핑 | 결정적 **점 → 점** | 확률적 **점 → 분포** |
| 주목적 | 효율적 표현 / 압축 | **생성 모델** |
| 학습 후 활용 | Encoder만 사용 (feature extractor) | **Decoder만 사용** (generator) |
| 잠재 공간 | 클래스별로 분리·밀집, **공백 많음** | **연속(Continuity) + 완전(Completeness)** |
| 보간(interpolation) | 불안정 (공백 지나감) | 자연스러움 |

비유 : AE는 그림판의 **핀(pin)**, VAE는 **스프레이(spray)**. 스프레이처럼 분포로 영역을 채워야 임의 지점에서 뽑아도 의미 있는 출력이 나온다.

## 12. 초기 VAE의 한계와 후속 Variants

### 12.1 Quality-Diversity Trade-off
- 초기 **VAE** : 다양성(diversity)은 좋지만 이미지가 **흐릿(blurry)**.
- 초기 **GAN** : 이미지 선명하지만 다양성 낮음(mode collapse).
- 오늘날엔 양쪽 다 크게 개선됐지만, **VAE의 전통적 drawback = blurry sample** 이다.

### 12.2 Posterior Collapse (조심해야 할 문제)
모든 x에 대해 **q_φ(z|x) → p(z) 로 붕괴** 해 버리는 현상.
- 인코더가 x의 정보를 z에 싣는 걸 포기 → z가 x와 독립이 됨.
- 결과 : 어떤 x를 넣어도 비슷한 출력. "넷플릭스 <지옥> 도깨비" 같은 흑백·평균화된 얼굴만 나오는 병리적 예시가 강의에서 소개됨(CelebA 학습 시 관찰).
- 디코더가 너무 강력하거나 KL 항이 과도하게 강할 때 발생.

### 12.3 NVAE (Nouveau VAE)
- "Make VAE's Great Again" 슬로건, NVIDIA 연구자(Arash Vahdat) 주도.
- **Deep hierarchical latent codes** — z 공간을 여러 층으로 계층화해 정교하게 매핑.
- 초기 VAE 대비 품질을 대폭 끌어올려 VAE 재부흥.

### 12.4 VQ-VAE (Vector Quantized VAE)
- 배경 : 가우시안 z는 **연속적**이어서 언어·심볼처럼 **이산적 성질(discrete nature)** 이 필요한 데이터엔 부적합.
- 해결 : 레이턴트를 **이산 코드북**으로 양자화(vector quantization).
- VQ = Vector Quantization = 통신공학의 **k-means clustering / Lloyd 알고리즘** (Robert Gray 교수의 개념).
- **VQ-VAE-2** 는 여기에 NVAE처럼 hierarchy를 얹은 버전.
- LLM·이미지 토크나이저 등 현대 대형 모델의 핵심 컴포넌트로 사용된다.

### 12.5 β-VAE
- 목표 : **Disentanglement** — 잠재 차원을 해석 가능한 독립 요소(예: 얼굴 각도, 웃음 정도)로 분리.
- 로스 :
  ```
  L = L_recon + β · L_KL     (β ≥ 1)
  ```
- β > 1 : KL 항 강조 → 잠재 공간이 단순·정규화되어 disentangle 됨.
- β = 1 : 표준 VAE.
- β 를 너무 키우면 정보 손실·블러 증가. 균형이 관건.
- 얽힌(entangled) 개념을 "실이 꼬였다(entangle) / 풀었다(disentangle)" 라는 비유로 설명.

## 13. 핵심 정리 (한 페이지 요약)

1. **동기** : 고차원 P(x) 를 직접 모델링하기 어렵 → 잠재변수 z 도입, P(x) = ∫ P(x|z) P(z) dz.
2. **세 가지 문제** : ① P(z) 는 인간이 N(0, I) 로 설계, ② P(x|z) 는 디코더 DNN, ③ ∫ 는 Amortized VI 로 근사.
3. **아키텍처** : 확률적 인코더 q_φ(z|x) + 확률적 디코더 p_θ(x|z). 둘 다 DNN.
4. **목적함수** : log P(x) 의 하한 ELBO 최대화. **ELBO = Reconstruction − KL**.
5. **로스 구현** : L = L_KL (closed-form) + L_recon (MSE/BCE/CE, 데이터 타입에 따라).
6. **Reparameterization Trick** : z = μ + σ ⊙ ε, ε ~ N(0, I). 샘플링을 계산으로 치환해 백프롭 가능.
7. **Training** : 평균 기반 deterministic reconstruction. **Inference** : 샘플링 기반 stochastic generation.
8. **문제점** : Blurry samples, Posterior Collapse (도깨비 얼굴).
9. **Variants** : NVAE(hierarchy), VQ-VAE(이산화), β-VAE(disentanglement).
10. **현대적 가치** : 단독 생성 품질에서는 Diffusion/GAN에 밀려도 **VQ-VAE 등 거대 모델의 컴포넌트** 로 필수.

### 강의 중 주요 직관 비유 모음
- **먼지 / 손오공** : 간단한 z (먼지)로부터 복잡한 x (손오공)를 만든다.
- **호랑이 / 늑대 / 가축** : ELBO 유도에서 계산 불가능한 항(호랑이), 신경망으로 근사 가능한 항(늑대), 쉬운 항(가축).
- **흰자 / 노른자 / 계란** : 크로스 엔트로피 = 전체 계란, KL = 흰자, 엔트로피 = 노른자.
- **포르쉐 스위치** : diagonal prior 가 왜 해석 가능성을 주는지 — 스위치 하나에 기능 하나.
- **대리인 / 고객** : 고차원 dependency 를 직접 모델링하지 말고 잠재 대리인 z와 1-대-1 관계로 단순화.
- **분할 상환 (amortized)** : 3억을 한 번에 내지 말고 300만원씩 100번 나눠 내듯, 많은 x에 로스를 나눠 쌓아 공통 φ 학습.
- **지옥 도깨비** : Posterior Collapse 의 시각적 결과 — 모든 입력이 평균 얼굴로 뭉개짐.
- **스프레이 / 핀** : VAE 는 점을 분포로 스프레이, AE 는 점을 점으로 핀으로 꽂음.
