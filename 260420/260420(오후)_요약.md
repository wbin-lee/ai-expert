## 1. 복습 : Generative Models의 대분류

| 구분 | 특징 | 대표 모델 |
|------|------|-----------|
| **Explicit (Rule-based)** | P(x)를 직접 계산 / 근사 | Gaussian, **GMM**, Normalizing Flow |
| **Latent Variable Model** | z를 매개로 P(x) 모델링 | **VAE** |
| **Denoising** | 노이즈 역과정 학습 | Diffusion, Score Matching |
| **Autoregressive** | 다음 토큰 예측 | GPT, Llama (Decoder-only Transformer) |

- 이미지/비디오의 **현재 표준 = Diffusion / Score matching**.
- 언어/오디오 등 sequential 데이터의 **현재 표준 = Autoregressive**.
- 이 실습에서는 Explicit → VAE → Decoder-only Transformer 순으로 다룬다.

## 2. Explicit Density Estimation : MCMC vs VI

### 2.1 MCMC (Markov Chain Monte Carlo)
- **아이디어** : 뽑기 쉬운 분포(예: 2D 유니폼)에서 시작한 점이, 타겟 분포 P(x)의 확률이 높은 곳으로 이동하도록 iterative하게 optimize.
- **알고리즘** (간단 버전)
  1. 현재 점 z에서 랜덤 방향의 후보 z' 를 샘플링.
  2. log-ratio = log P(z') − log P(z) 를 계산.
  3. log-ratio > torch.rand() 의 조건을 만족하면 z'로 이동, 아니면 머무름.
- **특징** : 충분히 오래 돌리면 어떤 복잡한 posterior도 근사 가능. 하지만 느리고, burn-in / mixing 진단이 필요. 가끔 랜덤하게 먼 곳으로도 이동하는 이유는 **도넛·non-linear 분포**를 탐색하려면 stochasticity가 필요하기 때문.

### 2.2 VI (Variational Inference)
- **아이디어** : 다루기 쉬운 분포 q(z) (예: Gaussian)를 정해 놓고, 이 q(z)를 posterior에 가장 가깝게 **fitting/optimize**.
- **장단점** : 단순하지만, 단일 Gaussian은 여러 mode를 가진 분포를 표현할 수 없다 → GMM으로 확장.

## 3. GMM (Gaussian Mixture Model) + EM 알고리즘

### 3.1 구조
여러 개의 Gaussian(평균 + 공분산 + 가중치)을 합쳐 복잡한 target distribution을 근사한다.

### 3.2 EM 알고리즘
- **E-step (Expectation)** : 각 데이터 포인트가 어떤 Gaussian 군집에 속할지에 대한 기댓값 계산.
- **M-step (Maximization)** : 계산된 기댓값 기준으로 각 Gaussian의 파라미터를 최대가능도로 업데이트.
- 20 iteration 정도 반복하면 타겟 분포에 잘 맞춰진다.

### 3.3 한계점
1. **K(가우시안 개수) 선택은 heuristic** — 15, 20, 30 개 돌려보며 튜닝해야 함.
2. **초기값에 매우 민감** (unstable initialization).
3. 복잡한 spiral 형태의 분포 등에서는 K를 크게 늘려야 하고, 크기 차이에 의한 **가중치 bias** 발생 가능.

### 3.4 샘플링 해석 (중요 정정)
질문자 지적으로 정정된 내용 : 3개의 Gaussian 중 하나를 고르고 뽑는 것이 아니라, **각 점에서의 Gaussian 값을 모두 더해 normalize** 한 뒤 그 혼합 분포 전체에서 확률적으로 샘플링한다. 즉 각 가우시안의 **mixture weight는 해당 군집의 데이터 포인트 수**에 의해 결정된다.

### 3.5 현역 용도
- 비록 옛날 알고리즘이지만 **클러스터링(unsupervised)** 에서는 여전히 현역.
- decision boundary와 결합해 샘플링 분포를 표현하는 데 사용됨.

## 4. 핵심 정리 (Generative Model)

> **생성 모델은 데이터를 직접 만들도록 배우는 것이 아니라, 데이터의 distribution을 학습하고 그 분포에서 샘플링하는 모델이다.**

- Discriminative 모델 : 은행나무 이미지를 보고 "은행나무"라는 하나의 라벨만 출력 → 다양성 없음.
- Generative 모델 : "은행나무"라는 분포에서 다양한 은행나무 이미지를 샘플링해 생성할 수 있음.

## 5. VAE 실습 : Fashion-MNIST

### 5.1 데이터셋
- **Fashion-MNIST** : 1 × 28 × 28 흑백 의류 이미지 (T-shirt, dress, coat, sneaker, bag 등 10 class).
- 원래 ImageNet으로 하려 했지만 MLP 몇 층으로는 학습이 안 돼서 Fashion-MNIST로 교체.
- **Pixel 값 ∈ [0, 1]** → Bernoulli 가정 + BCE loss가 가능.

### 5.2 AE vs VAE 리캡
| 구분 | Autoencoder | VAE |
|------|-------------|-----|
| 인코더 출력 | 단일 벡터 h | **μ, log σ²** (분포 파라미터) |
| 중간 레이턴트 | 분포 없음 (deterministic) | q_φ(z\|x) 로 **분포 학습** |
| 디코더 입력 | 같은 x → 같은 h → 같은 x̂ | reparameterized z → stochastic x̂ |
| 사용 목적 | 압축·복원 | **생성** (prior에서 z 뽑으면 x 생성 가능) |

### 5.3 VAE 모델 구조 (MLP 버전)
```
latent_dim = 16, hidden_dim = 400, input = 28*28 = 784

x (28×28)  →  flatten
          →  Linear(784 → 400)
          →  Linear(400 → 400)
          →  Linear(400 → 16)  → μ   (shape: 16)
          →  Linear(400 → 16)  → log σ²  (shape: 16)

z = reparameterize(μ, log σ²)

z (16)    →  Linear(16 → 400)
          →  Linear(400 → 400)
          →  Linear(400 → 784)
          →  reshape → reconstructed image
```

### 5.4 왜 log variance 인가
- 모델 출력은 (−∞, +∞) 이지만, **분산은 반드시 양수**여야 함.
- 그래서 모델이 내뱉는 값을 **log σ²** 로 해석하고, `σ = exp(0.5 * log σ²)` 로 되돌린다.
- 0.5 계수는 log σ² 에서 log σ 로 바꾸기 위한 것 (σ = exp((1/2) · log σ²)).

### 5.5 Reparameterization
```
σ = exp(0.5 * logvar)
ε ~ N(0, I)                     # 16차원 랜덤 노이즈
z = μ + σ * ε                   # 16차원 latent sample
```
`torch.rand_like` 기반 노이즈가 들어가서 샘플링을 **미분 가능한 연산**으로 치환. 16차원 레이턴트 = 16개의 독립 Gaussian 각각에서 한 번씩 샘플.

### 5.6 Loss 구성
```python
# KL divergence (closed form)
kl_loss = -0.5 * sum(1 + logvar - mu**2 - exp(logvar))

# Reconstruction (BCE with logits, 흑백 이미지 → 0/1 Bernoulli 가정)
recon_loss = F.binary_cross_entropy_with_logits(recon_logit, x)

# Total
total_loss = recon_loss + beta * kl_loss
```
- BCE가 **엄밀하진 않지만 (실제로는 grayscale)** 0/1 근사로 단순화. 연속 가정이면 MSE가 맞음.
- RGB 이미지였다면 별다른 선택지 없이 MSE.

## 6. β (Beta) 하이퍼파라미터와 β-VAE

### 6.1 의미
- β = 0 → reconstruction만 학습 (AE와 비슷, latent 분포는 우리가 원하는 N(0, I)에서 멀어짐).
- β = 1 → 표준 VAE.
- β > 1 → latent 분포가 N(0, I) 에 강하게 끌림 (disentanglement ↑, reconstruction ↓).
- 일반적 실무값 : **0.05 ~ 0.5** 정도로 작게 설정하는 경우가 많음.

### 6.2 Posterior Collapse
- β가 지나치게 크면 **모델이 reconstruction을 포기**하고 latent를 N(0, I)에 그대로 맞추는 현상.
- 결과 : 이미지가 찰흙처럼 뭉개지거나 원본 형체를 잃음.

### 6.3 Beta Scheduling (Warm-up)
- **학습 초반에는 β = 0** 에서 시작해, 점진적으로 목표값(예: 0.1)까지 올리는 warm-up.
- 초반에 KL 압력이 강하면 reconstruction 경로 자체를 못 찾는 문제를 피하기 위함.
- posterior collapse가 관찰되면 warm-up stage를 1000 step 등으로 늘려 더 천천히 올린다.

## 7. VAE 학습 & 시각화

### 7.1 Training Loop
1. Get batch (Fashion-MNIST)
2. Forward : x → μ, logvar, z → x̂ (recon_logit)
3. loss = recon_loss + β_now · kl_loss
4. optimizer.zero_grad() → loss.backward() → **grad norm clipping** → optimizer.step()

- 3000 step 학습까지 돌려야 사람이 알아볼 수 있는 품질이 나옴. 1000/1200/1800으로는 부족.
- Reconstruction loss는 숫자상으로 크게 안 떨어져도, **눈으로 확인한 품질**이 중요 판단 기준.
- KL loss가 완전 0이면 **코드 버그** (완전한 N(0, I)가 되는 것은 지향점일 뿐 실제로는 도달 불가).

### 7.2 시각화 4종

1. **Reconstruction** : x → encoder → z → decoder → x̂. 원본과 비교.
2. **Sample from Prior** : z ~ N(0, I) 에서 직접 뽑아 decoder에 넣음 ← **진짜 생성 테스트**.
3. **Sample from Posterior** : x → encoder → μ, σ → 거기서 여러 z 샘플링 → decoder → 해당 x에 가까운 다양한 변형이 나와야 정상 (학습이 잘 됐다는 증거).
4. **Latent Interpolation** : z_A와 z_B 사이를 (0.1, 0.9), (0.2, 0.8) … 로 보간 → 자연스러운 그라데이션이 나오면 좋은 latent space.

### 7.3 Pixel Interpolation vs Latent Interpolation
- Pixel 보간 : 두 이미지를 단순 가중합 → 형체가 페이드-아웃 되며 사라짐.
- Latent 보간 : 레이턴트 공간에서 보간 → 두 이미지 사이의 **의미 있는 중간 형태**가 나타나야 이상적.
- 단, "의미 있는 중간이 나와야 학습이 잘 됐다"는 직관적 주장일 뿐, 실제 P(x)를 볼 수 없어 엄밀한 증명은 어렵다. "어느 정도 약팔이"라는 솔직한 코멘트.

## 8. Blurry Sample 문제

### 8.1 원인
- VAE loss는 **Expectation over q(z|x)** 형태로 학습 → 확률적 평균을 최소화.
- MSE / BCE가 배치/스텝 단위로 평균을 학습해버리는 결과, 픽셀 하나하나가 모호해짐.
- 비유 : 시험에서 정답이 6인데 0과 헷갈릴 때 6과 0 사이의 애매한 글씨로 쓰는 것과 같다. 모델도 loss를 줄이기 위해 expectation 형태의 blurry output을 낸다.

### 8.2 완화 방법
- 학습을 더 오래, 데이터셋을 양질로, 모델을 더 크게.
- GAN loss / Diffusion loss를 곁들여 쓰는 변형 모델들이 제안됨.

## 9. Latent Space 시각화 : PCA vs t-SNE

| 방법 | 원리 | 결과 |
|------|------|------|
| **PCA** | Linear transformation으로 **variance 최대화** 방향 찾기 | 16D → 2D 압축 시 군집이 불명확하게 보일 수 있음 |
| **t-SNE** | **Non-linear**, 가까운 애는 가깝게·먼 애는 멀게 (Contrastive 성격) | 군집이 훨씬 뚜렷하게 보임 |

- Fashion-MNIST t-SNE에서 **ankle boot / sneaker / sandal은 서로 가까이**, coat / pullover / shirt 끼리도 가까이 모인다 → 시각적으로 유사한 클래스는 latent에서도 가까이 배치됨.
- 차원 축소 기법으로 보통 **t-SNE 우선**, 그 다음 PCA.

## 10. VQ-VAE (Vector Quantized VAE)

- 최근 업계 표준으로 자리잡은 VAE 계열.
- 일반 VAE는 latent를 **float16** 으로 두고 KL로만 정규화 → 압축률이 원하는 만큼 안 나옴.
- VQ-VAE는 **코드북** 기반 vector quantization을 적용해 latent를 **int8 / int4 / BF16** 등 이산 단위로 양자화 → 압축률 ↑ + 이산적 구조.
- 2D 이미지·3D 모델 생성 분야의 AutoEncoder는 **요즘도 Convolutional VQ-VAE** 가 주류 (Transformer 기반 대체 시도는 대부분 실패 — grid topology의 강력함).

## 11. Convolutional VAE (옵션, solution 폴더)

### 11.1 구조 아이디어
- MLP 레이어를 **Conv2D / ConvTranspose2D** 로 대체.
- 이미지 도메인은 CNN이 훨씬 강력하므로 자연스러운 선택.
- 인코더와 디코더는 **symmetric** 으로 설계 (32 → 64 → 128 → … ↔ 128 → 64 → 32 → …).

### 11.2 Conv2D 파라미터 (예)
- `kernel=4, stride=2, padding=1` → spatial size 절반.
- 28 → 14 → 7 → flatten → μ, logvar (latent_dim=32).

### 11.3 커널 크기를 4 (짝수) 로 한 이유
- 원칙적으로 커널은 **홀수가 표준** (padding 비대칭 방지).
- 여기선 28 × 28 → 7 × 7 로 딱 떨어지게 하려고 짝수 4 선택. 7에 이르러서는 다시 홀수로 전환.

### 11.4 실습 결과
- MLP 버전보다 오히려 성능이 떨어짐. 이유:
  - **Learnable parameter 수가 훨씬 적음** (MLP의 Linear layer는 (784 × 400) 등 수십만 개 파라미터 vs Conv는 (kernel² × in_ch × out_ch) 수준).
  - Layer를 깊게 쌓지 않음 → receptive field와 표현력 부족.
- 일반적으로 **최신 Convolutional VAE는 수백 개(400~500개) 레이어** + 양자화·병렬학습 등 트릭 다수. A100 수 장 단위 GPU 필요.

### 11.5 실무 조언
- 타겟 도메인(예: 삼성 휴대폰 이미지)이 있다면 natural image pretrained보다 **스크래치 학습**이 더 깔끔 (자원 있을 때).
- 스케일러빌리티 시대 : 모델도 데이터도 **크면 클수록 좋다**.

## 12. VAE의 역할 포지셔닝
- 생성 모델로서는 Diffusion/GAN에 밀리지만, **latent로 압축하는 서브 모듈**로 현대적 가치 유지.
- **Latent Diffusion**, **Latent Generative Model** 등에서 VAE가 먼저 이미지 → latent로 바꾸고, 그 latent 공간에서 Diffusion/AR 모델이 생성을 수행.
- Autoregressive 모델에서는 Tokenizer가 VAE 역할을 대신하기 때문에 VAE를 거의 안 씀.
- **CNN / Transformer = 빌딩블럭**, **VAE / GAN / Diffusion = 그 블럭으로 지은 건물(목적성 있는 설계)**. 둘은 큰 개념과 작은 개념의 관계가 아니라 **분류 축이 다른 개념**.

## 13. Decoder-Only Transformer (보너스 세션)

### 13.1 Motivation
- 앞 강의에서 Encoder + Decoder 기반 번역 모델을 구현했다면, 여기서는 **Encoder를 제거한 Decoder-only 구조**를 소개.
- 현대 LLM(ChatGPT, Llama, Gemini, Claude 등) 대부분이 Decoder-only.

### 13.2 번역 태스크의 표현 방식
- Encoder-Decoder : encoder에 영어 문장, decoder에서 한국어로 token-by-token 생성.
- Decoder-only : **prefix로 "I love you를 한국어로 번역해봐"** 같은 프롬프트를 통째로 decoder에 넣고, 그 다음에 올 토큰을 계속 next-token prediction.
- 직관적으로는 어색해 보이지만, **대규모 데이터를 때려 넣으면 압도적으로 효율적**이라는 것이 경험적 결론.

### 13.3 실습 데이터 & 토크나이저
- 짧은 문장 6줄을 데이터로 사용.
- **캐릭터 단위 tokenizer** (알파벳 + 기타 기호) → vocab size 27.
- 모델은 next-token prediction으로 길이 27의 logit을 출력 → softmax → 다음 캐릭터 선택.

### 13.4 Shifting 방식
- 번역의 경우 `<BOS>` 토큰을 추가해 shift 했지만, 여기서는 문장 중간을 임의로 잘라 학습하므로 **입력을 한 칸 shift한 것이 정답**.
- 예 : `small m` → `small mo`, `small mo` → `small mod`, … token by token.

### 13.5 구조
- Encoder 완전 제거 → **Cross Attention도 없다**.
- Transformer block = LayerNorm → **(Masked) Self-Attention** → LayerNorm → Feed-Forward.
- Causal mask 적용 (미래 토큰 못 봄).

### 13.6 학습 & 샘플링
- Training loss는 단조 감소, Validation loss는 오히려 증가 → **double descent 이전의 첫 descent** 구간.
- Evaluation : prompt로 "T" 하나만 주고, 그 뒤의 캐릭터를 next-token prediction 으로 쭉 생성.
- 결과가 놀랍게도 `the next token, attention lets each token look back, usable context` 처럼 학습 데이터의 문장을 자연스럽게 재구성.

### 13.7 Temperature
```
p = softmax(logit / T)
```
- **T ↓ 0** : 분포가 뾰족해져 **deterministic** → 항상 같은 문장.
- **T ↑** : 분포가 평탄해져 **확률적 다양성 ↑** → 원래 확률이 낮던 토큰도 뽑힘.
- 대신 T를 너무 올리면 학습된 적 없는 이상한 단어가 나옴 (데이터셋이 작아서 특히 심함).
- 이 **stochastic 샘플링이 가능하기 때문에** decoder-only transformer도 **generative model**로 분류.

## 14. 핵심 정리 (한 페이지 요약)

1. **Generative Model의 정의** : 데이터 자체가 아니라 **데이터의 distribution P(x)** 를 학습하고, 그 분포에서 샘플링.
2. **Explicit density** : MCMC (iterative, 느림, 정확), VI (간단, 다중 mode 취약) → GMM + EM은 여전히 클러스터링 현역.
3. **VAE 실습의 핵심 플로우** :
   - x → Encoder → (μ, log σ²)
   - z = μ + σ · ε (reparameterization, ε ~ N(0, I))
   - z → Decoder → x̂
   - Loss = recon (BCE/MSE) + β · KL
4. **β 조절** : 너무 크면 posterior collapse, 너무 작으면 latent가 N(0, I) 에서 이탈. **Beta Scheduling (warm-up)** 이 실무 기법.
5. **시각화 4종** : Reconstruction, Prior Sampling, Posterior Sampling, Latent Interpolation.
6. **Blurry 현상** : Expectation 기반 학습이 만든 필연적 결과 → 학습량·데이터·모델 크기로 완화.
7. **Latent 분석** : 차원 축소는 보통 t-SNE가 PCA보다 깔끔. Fashion-MNIST에서 시각적으로 유사한 의류끼리 latent 공간에서도 가까이 모임.
8. **Convolutional VAE** : 이미지 도메인에선 CNN이 우위지만, **깊이와 파라미터 수를 충분히 확보해야** 제대로 학습됨.
9. **현대적 위치** : VAE는 단독 생성 품질보다 **Latent Diffusion 등의 압축 컴포넌트**로 사용되며, VQ-VAE가 업계 표준.
10. **Decoder-only Transformer** : Cross-attention 제거, 프롬프트 기반 next-token prediction. Temperature로 quality-diversity trade-off 조절.

### 강의 중 주요 비유 / 표현 모음
- **CNN·Transformer = 콘크리트/철판, VAE·GAN·Diffusion = 그걸로 지은 목적성 있는 건물**.
- **시험에서 6·0이 헷갈리면 중간 글씨로 쓴다** : Expectation 기반 학습이 만든 blurry output 비유.
- **약팔이** : latent interpolation이 의미 있는 중간을 내놓으면 학습이 잘 된 것이라는 주장에 대한 교수님의 솔직한 표현.
- **찰흙 덩어리 / 종이가 찢어진 듯** : Posterior Collapse / Model Collapse 시 prior 샘플링 결과 모습.
- **건물 안을 무엇으로 채우는가** : VAE라는 "건물"을 CNN으로 채우면 Convolutional VAE, Transformer로 채우면 Transformer VAE.
