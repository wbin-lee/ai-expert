# 1. 전날 내용 복습 — 음성 입력과 Alignment

## 1.1 Mel-spectrogram을 쓰는 이유

- 음성 raw waveform은 길이가 너무 길다.
- 16kHz라면 1초에 16,000개 sample.
- 모델이 raw waveform을 직접 다루면 sequence length가 너무 길어 학습이 어렵다.
- STFT와 Mel filterbank를 거쳐 mel-spectrogram으로 만들면 시간축/주파수축이 모두 압축된다.
- 인간 청각은 저주파에 민감하고 고주파에 둔감하므로 mel scale이 음성에 적합하다.

## 1.2 Alignment problem 재확인

- ASR은 음성 frame sequence를 짧은 텍스트 sequence로 바꾼다.
- 입력 길이와 출력 길이가 다르고, 어느 음성 구간이 어느 글자/단어에 해당하는지 정답 alignment가 없다.
- 같은 문장도 사람마다 말 속도, 억양, 발음 길이가 다르다.
- 따라서 ASR 모델의 핵심은 **음성과 텍스트 사이의 가변 길이 정렬을 어떻게 학습하느냐**다.

# 2. ASR 평가 지표 — WER / CER

## 2.1 WER 정의

- WER = Word Error Rate.
- ASR 모델 출력이 정답 문장과 얼마나 다른지 단어 단위로 측정한다.
- 세 가지 오류를 센다.

| 항목 | 의미 |
|---|---|
| Substitution (S) | 정답 단어가 다른 단어로 바뀜 |
| Deletion (D) | 정답에 있던 단어가 빠짐 |
| Insertion (I) | 정답에 없는 단어가 추가됨 |

공식:

$$
WER = \frac{S + D + I}{N_{\text{reference}}}
$$

- 작을수록 좋다.
- 0이면 완벽한 전사.

## 2.2 CER

- 한국어/일본어처럼 띄어쓰기 기준이 모호한 언어에서는 WER이 불안정할 수 있다.
- 예: 띄어쓰기 하나 때문에 단어 단위 오류가 크게 계산될 수 있음.
- 이 경우 Character Error Rate, 즉 CER을 사용한다.
- CER은 글자 단위로 substitution/deletion/insertion을 계산한다.

## 2.3 시험 포인트

- WER 공식은 반드시 외워야 한다.
- S, D, I의 의미를 예시와 함께 설명할 수 있어야 한다.
- 한국어/일본어는 CER이 더 적합하다는 점이 중요하다.

# 3. CTC — Connectionist Temporal Classification

## 3.1 CTC가 필요한 이유

- ASR에서 입력 mel-spectrogram은 길고, 출력 텍스트는 짧다.
- frame마다 정답 글자를 직접 붙인 label이 없다.
- 즉, frame-level alignment label 없이 학습해야 한다.
- CTC는 이 문제를 end-to-end로 풀기 위해 등장했다.

## 3.2 CTC의 구조

- 입력 mel-spectrogram을 encoder에 넣는다.
- encoder가 각 time step마다 hidden feature를 만든다.
- 각 time step마다 softmax를 통해 가능한 문자/token 확률을 예측한다.
- 기본 구조는 비교적 단순하다.

```text
mel-spectrogram → Encoder → frame-wise softmax → token probability
```

# 4. CTC의 3대 키워드 — Blank / Collapse / Marginalize

## 4.1 Blank

- CTC는 일반 문자 외에 특수 token인 blank를 추가한다.
- blank는 보통 `ε`로 표시한다.
- blank의 역할은 여러 가지다.

### 4.1.1 길이 차이 채우기

- 음성 frame은 텍스트보다 훨씬 길다.
- 모든 frame에 실제 글자가 대응되지는 않는다.
- blank는 글자가 없는 구간, 늘어진 발음 구간, 전환 구간을 채울 수 있다.

### 4.1.2 같은 글자 반복 보존

- Collapse 과정에서 연속된 같은 글자는 하나로 합쳐진다.
- 예를 들어 `ll`처럼 같은 글자가 실제로 두 번 나와야 하는 경우 문제가 생긴다.
- 이때 중간에 blank를 넣으면 두 글자를 구분할 수 있다.

```text
l l        → collapse → l
l ε l      → collapse/remove blank → ll
```

### 4.1.3 발음 전환 구간

- 한 phoneme/글자에서 다음 글자로 넘어갈 때 어느 쪽에도 명확히 속하지 않는 frame이 있다.
- blank는 이런 transition buffer 역할도 한다.
- blank가 반드시 silence만 의미하는 것은 아니다.

## 4.2 Collapse

- CTC inference에서는 frame마다 가장 확률 높은 token을 고른 뒤 collapse rule을 적용한다.
- 과정은 중요하다.

1. 연속해서 반복되는 같은 token을 하나로 합친다.
2. blank를 제거한다.

예:

```text
π = [A, A, ε, B, B, ε]
collapse duplicates → [A, ε, B, ε]
remove blank → AB
```

## 4.3 Marginalize

- 학습 시에는 정답 텍스트에 대응되는 정확한 path를 모른다.
- 하지만 같은 정답 텍스트로 collapse되는 path는 여러 개가 있다.
- CTC는 가능한 모든 path의 확률을 더한다.
- 이것이 marginalization이다.

```text
P(y|x) = sum over all paths π that collapse to y of P(π|x)
```

## 4.4 시험 포인트

- CTC를 설명하라는 문제는 거의 항상 **Blank / Collapse / Marginalize** 세 단어가 핵심이다.
- blank를 silence로만 쓰면 부정확하다.
- collapse는 “중복 제거 후 blank 제거” 순서가 중요하다.
- training은 한 path를 고르는 것이 아니라 모든 가능한 path를 합산한다.

# 5. CTC의 Conditional Independence 가정

## 5.1 가정

- CTC는 각 time step의 출력이 서로 독립이라고 가정한다.

$$
P(\pi|x) = \prod_t P(\pi_t|x)
$$

- 즉, 현재 frame의 token 예측이 이전에 어떤 token을 냈는지 직접 보지 않는다.

## 5.2 한계

- 언어 문맥을 직접 반영하기 어렵다.
- 예를 들어 “I want some ice cream”과 “I want some I scream”은 음성적으로 비슷하다.
- 사람은 앞뒤 문맥을 보고 “ice cream”이 자연스럽다고 판단하지만, CTC는 문맥 정보를 약하게 쓴다.

## 5.3 완화 방법

- 외부 Language Model을 붙여 문장 자연스러움을 보정한다.
- Beam search에서 LM score를 함께 고려한다.
- 하지만 모델 자체가 과거 token을 내부적으로 보는 구조는 아니다.

# 6. CTC Training — Forward Algorithm

## 6.1 왜 DP가 필요한가

- 가능한 path 수가 너무 많다.
- 모든 path를 직접 나열해 확률을 더하는 것은 불가능하다.
- 따라서 dynamic programming을 사용한다.

## 6.2 Extended label

- 정답 label 사이사이에 blank를 삽입한다.
- 예를 들어 정답이 `AB`라면 extended label은 다음처럼 만든다.

```text
Z = [ε, A, ε, B, ε]
```

- 길이는 보통 `2U + 1`.

## 6.3 Forward algorithm 직관

- 시간축과 extended label축으로 2D grid를 만든다.
- 각 칸은 “이 시점까지 이 label 위치까지 도달할 확률”을 의미한다.
- 이전 칸들에서 올 수 있는 확률을 더하며 누적한다.
- 마지막 위치까지 도달하는 확률을 합치면 `P(y|x)`가 된다.

## 6.4 시험 포인트

- 세부 recurrence를 외우는 것보다 “모든 가능한 path를 DP로 효율적으로 합산한다”는 직관이 중요하다.
- extended label에 blank를 끼워 넣는다는 점을 기억해야 한다.

# 7. CTC Inference — Greedy vs Beam Search

## 7.1 Greedy decoding

- 각 time step에서 가장 확률 높은 token을 고른다.
- 그 결과 path에 collapse rule을 적용한다.
- 구현은 쉽고 빠르다.

## 7.2 Greedy의 한계

- 학습은 모든 path의 확률 합을 최대화한다.
- 그런데 greedy는 path 하나만 고른다.
- 가장 확률 높은 path 하나가 가장 확률 높은 텍스트를 보장하지 않는다.

## 7.3 Beam search

- 여러 후보 path/prefix를 유지하며 탐색한다.
- CTC에서는 prefix beam search를 많이 쓴다.
- 외부 LM과 결합하기도 쉽다.

# 8. RNN-Transducer (RNN-T)

## 8.1 등장 배경

- CTC의 가장 큰 한계는 conditional independence다.
- 즉, 과거에 생성한 텍스트를 보지 못한다.
- RNN-T는 음성 정보와 과거 텍스트 정보를 함께 보고 다음 token을 예측하도록 설계되었다.

## 8.2 RNN-T의 3개 모듈

| 모듈 | 역할 |
|---|---|
| Encoder | mel-spectrogram을 음성 representation으로 변환 |
| Prediction Network | 지금까지 생성한 token sequence를 요약 |
| Joint Network | encoder 출력과 prediction 출력 결합 후 다음 token 예측 |

```text
speech → Encoder ┐
                 ├→ Joint Network → next token / blank
past text → Prediction Network ┘
```

## 8.3 CTC와의 핵심 차이

- CTC는 speech만 보고 각 frame token을 독립적으로 예측한다.
- RNN-T는 speech와 이전 text를 함께 본다.
- 따라서 문맥 의존성이 생긴다.

## 8.4 On-device / streaming

- RNN-T는 온디바이스 ASR에서 많이 쓰인다.
- 스마트폰 같은 제한된 환경에서도 비교적 실용적이다.
- streaming 처리에 적합하다.
- 큰 서버 모델보다는 작고 빠르게 운영하기 좋다.

---

# 9. RNN-T의 Blank 의미

## 9.1 CTC blank와 다름

| 모델 | Blank 의미 |
|---|---|
| CTC | collapse 방지, 길이 채움, 전환 구간 |
| RNN-T | 현재 speech frame에서 더 낼 token이 없으니 다음 time frame으로 이동 |

## 9.2 RNN-T lattice

- RNN-T는 time축과 label축을 가진 grid/lattice로 볼 수 있다.
- token을 출력하면 label 방향으로 이동한다.
- blank를 출력하면 time 방향으로 이동한다.
- 한 time frame에서 여러 token을 출력할 수도 있다.

## 9.3 Training

- RNN-T도 가능한 alignment/path를 하나로 정하지 않는다.
- 가능한 path들의 확률을 marginalize한다.
- DP로 효율적으로 계산한다.
- 다만 CTC와 path 구조와 blank 의미가 다르다.

# 10. Conformer Encoder

## 10.1 Conformer의 의미

- Conformer = Convolution + Transformer.
- 음성 인코더에서 사실상 표준처럼 쓰인다.

## 10.2 왜 결합하는가

| 구성 | 역할 |
|---|---|
| Convolution | 짧은 발음, 지역적 음향 패턴 포착 |
| Attention/Transformer | 긴 문맥, 전체 utterance 관계 포착 |

- 음성은 local pattern도 중요하고 global context도 중요하다.
- Conformer는 두 장점을 결합한다.

## 10.3 시험 포인트

- 세부 block 순서보다 “convolution은 local, attention은 global”이라는 직관이 중요하다.
- CTC/RNN-T/AED 모두 encoder로 Conformer를 쓸 수 있다.

# 11. 오전 강의 핵심 정리

## 11.1 반드시 외울 것

1. **WER 공식**
   - `WER = (S + D + I) / N`
   - 한국어/일본어는 CER 고려.

2. **CTC 3키워드**
   - Blank
   - Collapse
   - Marginalize

3. **CTC blank의 역할**
   - 반복 글자 보존
   - 길이 채움
   - transition buffer

4. **CTC의 한계**
   - Conditional independence.
   - 문맥을 직접 못 봄.
   - 외부 LM으로 보완 가능.

5. **RNN-T 구조**
   - Encoder
   - Prediction Network
   - Joint Network

6. **RNN-T blank**
   - CTC blank와 다르다.
   - 현재 frame 종료 후 다음 time frame 이동 신호.

7. **Conformer**
   - convolution local + attention global.

## 11.2 예상 시험 문제 스타일

- CTC의 Blank/Collapse/Marginalize를 설명하라.
- CTC가 alignment problem을 어떻게 해결하는지 설명하라.
- CTC의 conditional independence가 왜 문제인지 예시와 함께 설명하라.
- RNN-T가 CTC와 다른 점을 설명하라.
- RNN-T의 Prediction Network와 Joint Network 역할을 설명하라.
- CTC blank와 RNN-T blank의 차이를 설명하라.
- WER을 계산하라.
