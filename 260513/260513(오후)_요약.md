# 1. 오전 복습 — CTC와 RNN-T의 위치

## 1.1 CTC

- Encoder가 mel-spectrogram을 frame별 hidden state로 변환한다.
- 각 frame에서 token probability를 낸다.
- Blank / Collapse / Marginalize로 alignment label 없이 학습한다.
- 한계는 conditional independence.
- 과거 text context를 직접 보지 못한다.

## 1.2 RNN-T

- CTC의 conditional independence 한계를 보완한다.
- Encoder는 speech 정보를 만든다.
- Prediction Network는 이전까지 생성한 text 정보를 요약한다.
- Joint Network가 두 정보를 결합한다.
- On-device, streaming ASR에서 실용적이다.

## 1.3 AED로 넘어가는 이유

- 서버처럼 계산 자원이 충분하면 더 큰 모델을 쓸 수 있다.
- 큰 모델과 attention 구조를 쓰면 성능이 좋아진다.
- RNN-T보다 더 강한 text modeling과 alignment modeling이 가능하다.

---

# 2. AED — Attention-based Encoder-Decoder

## 2.1 기본 구조

- AED는 encoder와 decoder로 구성된다.
- 대표 초기 모델은 Listen, Attend, Spell.

```text
speech/mel → Encoder → hidden states
previous text tokens → Decoder + Attention → next token
```

## 2.2 Encoder

- 입력 speech feature를 hidden representation으로 바꾼다.
- Conformer나 Transformer encoder를 사용할 수 있다.
- 전체 음성 정보를 담은 sequence를 만든다.

## 2.3 Decoder

- 이전에 생성한 text token들을 보고 다음 token을 autoregressive하게 생성한다.
- RNN-T의 prediction network와 joint network 역할을 더 강력한 decoder가 수행한다고 볼 수 있다.
- 일반적으로 큰 decoder를 쓰면 언어 문맥을 더 잘 모델링한다.

## 2.4 Chain rule 관점

- AED는 다음과 같은 형태로 텍스트를 생성한다.

$$
P(y|x) = \prod_u P(y_u | y_{<u}, x)
$$

- 현재 token은 이전 token들과 전체 speech input에 의존한다.
- CTC의 conditional independence와 대비된다.

---

# 3. AED 학습 방식

## 3.1 Teacher forcing

- 학습할 때는 이전 step의 모델 예측이 아니라 정답 token을 decoder 입력으로 넣는다.
- 이를 teacher forcing이라고 한다.
- 다음 token을 맞히도록 cross entropy로 학습한다.

## 3.2 CTC/RNN-T와의 차이

| 항목 | CTC/RNN-T | AED |
|---|---|---|
| Alignment 처리 | 가능한 path를 marginalize | attention으로 soft alignment 학습 |
| 학습 objective | path probability 합산 | next-token cross entropy |
| Decoder | CTC는 없음, RNN-T는 prediction network | autoregressive decoder |

## 3.3 시험 포인트

- AED는 path marginalization이 아니라 attention으로 alignment를 푼다.
- Teacher forcing과 next-token prediction 구조를 이해해야 한다.

# 4. Attention 구조 — Self/Cross × Causal/Non-causal

## 4.1 Attention의 직관

- Query가 Key를 보고 관련도를 계산한다.
- softmax로 attention weight를 만든다.
- 그 weight로 Value를 가중합한다.

```text
Query: 지금 필요한 정보
Key: 검색 대상의 주소/색인
Value: 실제 가져올 내용
```

## 4.2 Self-attention

- 같은 sequence 안에서 query/key/value가 모두 나온다.
- 외부 정보 없이 자기 자신 안에서 필요한 정보를 찾는다.

## 4.3 Cross-attention

- query는 decoder 쪽에서 오고, key/value는 encoder 쪽에서 온다.
- decoder가 speech encoder output 중 어디를 볼지 결정한다.
- AED에서 alignment를 푸는 핵심이다.

## 4.4 Causal vs Non-causal

| 구분 | 의미 | 사용 위치 |
|---|---|---|
| Causal | 미래 token을 볼 수 없음 | Decoder self-attention |
| Non-causal | 전체 sequence를 볼 수 있음 | Encoder self-attention, decoder cross-attention |

## 4.5 AED에서의 세 attention

1. **Encoder self-attention**
   - speech feature끼리 서로 참고한다.
   - 전체 음성이 주어져 있으므로 non-causal.

2. **Decoder self-attention**
   - 이전 text token들끼리 참고한다.
   - 미래 token을 보면 안 되므로 causal.

3. **Decoder cross-attention**
   - text decoder가 speech encoder output을 참고한다.
   - 전체 speech는 이미 주어져 있으므로 non-causal.

## 4.6 시험 포인트

- “Self/Cross”와 “Causal/Non-causal”을 섞어서 설명할 수 있어야 한다.
- 특히 decoder cross-attention이 alignment를 담당한다는 점이 중요하다.

# 5. Cross-Attention과 Alignment

## 5.1 Soft alignment

- AED는 hard alignment label 없이도 attention weight로 alignment를 학습한다.
- 현재 생성할 token이 speech의 어느 부분을 봐야 하는지 attention이 나타낸다.
- 잘 학습된 모델의 attention map은 대각선 형태를 보인다.

## 5.2 대각선이 의미하는 것

- 음성은 시간 순서대로 진행된다.
- 텍스트도 보통 순서대로 생성된다.
- 따라서 정상 alignment는 왼쪽 위에서 오른쪽 아래로 이동하는 대각선 모양이다.

## 5.3 Attention failure

- attention이 흐트러지면 같은 구간을 반복해서 보거나 건너뛸 수 있다.
- ASR에서는 잘못된 transcription으로 나타난다.
- TTS에서는 반복 발화, skip, 발음 누락 등으로 나타난다.

# 6. Whisper

## 6.1 Whisper의 정체

- Whisper는 OpenAI가 공개한 대규모 speech model.
- 구조적으로는 Transformer 기반 AED 계열이다.
- 68만 시간 규모의 speech-caption 데이터를 사용했다.
- 99개 언어를 지원한다.

## 6.2 Multitask 모델

- Whisper는 단순 ASR만 하지 않는다.
- 여러 task를 token 생성 문제로 통합한다.

| Task | 설명 |
|---|---|
| Transcription | 음성을 같은 언어 텍스트로 전사 |
| Translation | 음성을 영어 텍스트로 번역 |
| Language Detection | 입력 음성 언어 감지 |
| Timestamp prediction | 음성과 단어/구간 시간 연결 |

## 6.3 Language/task tag

- Whisper는 language tag와 task tag를 사용한다.
- 어떤 언어로 처리할지, 전사할지 번역할지 등을 token으로 지시한다.
- tag가 잘못되면 엉뚱한 언어 출력이나 번역처럼 보이는 결과가 나올 수 있다.

## 6.4 대규모 weak supervision

- Whisper는 깨끗한 실험실 데이터만 학습한 것이 아니다.
- 웹에서 얻은 noisy caption 데이터도 대규모로 사용했다.
- 이 때문에 잡음, OOD, 다양한 언어/환경에 강하다.
- 반대로 데이터가 noisy하므로 hallucination 문제가 생길 수 있다.

# 7. Whisper Hallucination

## 7.1 Hallucination이란

- 실제 음성에 없는 내용을 모델이 만들어내는 현상.
- 무음 구간에서 “you” 같은 단어를 출력하거나, 방송 outro 문구를 생성할 수 있다.
- autoregressive decoder가 그럴듯한 텍스트를 계속 이어가려는 성향과 관련이 있다.

## 7.2 원인

1. **Noisy web caption 학습**
   - 실제 음성과 정확히 맞지 않는 자막도 학습에 포함될 수 있다.

2. **Autoregressive decoder**
   - 이전 출력에 조건부로 다음 token을 계속 생성한다.
   - 한 번 잘못된 흐름에 들어가면 반복/소설 생성이 생긴다.

3. **Text prior가 강함**
   - 음성이 불명확하면 음향 정보보다 언어 모델 prior가 강하게 작동한다.

## 7.3 완화 방법

- `initial_prompt`
  - 고유명사, 도메인, 문맥 힌트를 미리 준다.
- `no_speech_threshold`
  - 무음 구간에서 전사를 억제한다.
- VAD
  - 사람이 말하는 구간만 ASR에 넣는다.
- `compression_ratio_threshold`
  - 반복 루프처럼 압축률이 이상한 출력 감지.
- `temperature`
  - decoding randomness 조절.
- `condition_on_previous_text`
  - 이전 segment text를 계속 조건으로 넣을지 조절.

## 7.4 시험 포인트

- Whisper는 강력하지만 hallucination이 있다.
- 원인은 weak-supervised noisy data + autoregressive text generation.
- initial prompt, VAD, no_speech threshold 같은 완화책을 말할 수 있어야 한다.

# 8. 실습 — 모델 비교

## 8.1 비교 대상

- CTC 기반 모델
- RNN-T 기반 모델
- AED/Whisper 기반 모델

## 8.2 속도

- CTC는 구조가 단순해 빠르다.
- RNN-T는 streaming/on-device에 적합하다.
- AED/Whisper는 큰 decoder와 attention 때문에 상대적으로 느릴 수 있다.
- 하지만 큰 모델과 대규모 학습 덕분에 정확도와 robust가 좋다.

## 8.3 WER 계산 시 주의

- punctuation, 대소문자, 띄어쓰기 normalization을 하지 않으면 WER이 과장될 수 있다.
- 실습에서는 같은 문장을 의미상 맞게 인식해도 표기 차이 때문에 점수가 나빠질 수 있음을 확인한다.

# 9. Stress Test — Speed / Noise / OOD

## 9.1 말 빠르기 변화

- 음성을 빠르게 만들면 ASR이 어려워진다.
- 모델마다 robustness 차이가 나타난다.

## 9.2 Noise

- 배경 잡음이 섞이면 attention이 퍼지거나 잘못된 구간을 볼 수 있다.
- Whisper는 대규모 noisy 데이터 덕분에 비교적 강하지만 완벽하지 않다.

## 9.3 OOD

- 학습 분포와 다른 발음, 언어, 환경에서는 성능이 떨어진다.
- 작은 특화 모델이 특정 도메인에서는 Whisper base보다 나을 수도 있다.
- 반대로 Whisper medium/large처럼 큰 모델은 general robustness가 강하다.

# 10. Cross-Attention 시각화 실습

## 10.1 Hook 사용

- Whisper 내부 attention weight를 hook으로 뽑아 시각화한다.
- decoder cross-attention이 speech frame을 어떻게 보는지 확인한다.

## 10.2 정상 사례

- attention map이 대각선 형태를 보인다.
- 이는 text token 생성 순서와 speech time frame 진행이 잘 맞는다는 의미다.

## 10.3 Noise 사례

- 잡음이 있거나 모델이 헷갈리면 attention이 넓게 퍼진다.
- 특정 token이 음성의 명확한 구간에 집중하지 못한다.

## 10.4 Whisper window

- Whisper는 고정 길이 window로 음성을 처리한다.
- 실습에서는 30초 window, 1500 frame 같은 구조를 확인한다.

# 11. Silence / Repetition Hallucination

## 11.1 Silence hallucination

- 무음인데도 텍스트를 생성할 수 있다.
- ASR 모델은 “사람이 말할 때 텍스트로 바꾸는 것”에 특화되어 있으므로, 무음 입력에서 이상 행동을 할 수 있다.
- VAD나 no_speech threshold가 중요하다.

## 11.2 Repetition loop

- autoregressive decoder가 같은 문구를 반복 생성할 수 있다.
- 반복 문장은 compression ratio가 비정상적으로 낮거나 높게 나타날 수 있다.
- threshold로 감지해 decoding을 다시 하거나 중단할 수 있다.

# 12. Whisper의 강점과 한계

## 12.1 강점

- 다국어 지원.
- noise/OOD robustness.
- ASR, translation, language detection, timestamp를 하나의 모델로 처리.
- open-source로 쉽게 실습 가능.

## 12.2 한계

- hallucination.
- 큰 모델은 느리고 무겁다.
- streaming/on-device에서는 RNN-T가 더 실용적일 수 있다.
- 언어/task tag가 잘못되면 결과가 망가질 수 있다.

# 13. 오후 강의 핵심 정리

## 13.1 반드시 외울 것

1. **AED 구조**
   - Encoder + autoregressive Decoder + Cross-attention.

2. **Attention 4분류**
   - Self vs Cross.
   - Causal vs Non-causal.

3. **AED alignment**
   - Cross-attention이 soft alignment 역할.
   - 정상 attention map은 대각선.

4. **Whisper**
   - Transformer AED 계열.
   - 68만 시간.
   - 99개 언어.
   - ASR/translation/language detection/timestamp multitask.

5. **Hallucination**
   - noisy weak-supervised data + autoregressive decoder + text prior.
   - initial prompt, VAD, no_speech threshold 등으로 완화.

6. **모델 선택**
   - RNN-T: on-device/streaming.
   - AED/Whisper: server/large-scale/high accuracy.

## 13.2 예상 시험 문제 스타일

- AED가 CTC/RNN-T와 alignment를 푸는 방식이 어떻게 다른지 설명하라.
- Encoder self-attention, decoder self-attention, decoder cross-attention을 causal/non-causal 관점에서 설명하라.
- Whisper가 어떤 task를 수행하는지 설명하라.
- Whisper hallucination의 원인과 완화책을 설명하라.
- RNN-T와 AED가 각각 어떤 환경에서 주로 쓰이는지 비교하라.
