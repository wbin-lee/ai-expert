# 1부. 260706 오전 — Weakly Supervised Learning의 출발

## 1-1. 왜 Weakly SL을 배우는가

최종현 교수님은 **Weakly Supervised Learning**을 이렇게 정의합니다.  
데이터는 있는데 **슈퍼비전(라벨)이 적은** 상황에서 지도학습을 하는 방법입니다.

- 전체 데이터 양이 적을 수도 있고
- 데이터는 많은데 **라벨이 붙은 샘플만 적을** 수도 있습니다

요즘은 대부분 **언라벨 데이터로 pre-training** → **SFT(Supervised Fine-tuning)** 순서로 가지만, 마지막 단계의 SFT 자체가 사실상 Weakly SL인 경우가 많습니다. 데이터가 적은 태스크에서 “어떻게든 잘해야 한다”는 문제는 여전히 현실입니다.

대표 기법: **Few-shot**, **Semi-supervised**, **Active Learning**, **Self-supervised**(엄밀히는 unsupervised에 가깝지만 pre-training 경로로 연결).

> ⭐ **시험 포인트**  
> 머신러닝에서 **용어를 정확히 이해하는 것**이 절반 이상입니다. 같은 개념을 다른 이름으로 부르는 경우가 많습니다(MLP = FC = FFN 등). 용어의 뜻을 알면 외울 필요가 줄어듭니다.

## 1-2. 도메인 어댑테이션과 일반화

**도메인 어댑테이션** = 입력 공간 X가 조금 바뀌었을 때, 함수 F가 새 X에 잘 적응하도록 하는 연구.  
예: 원래 강아지 사진 분포 → 새 카메라·새 환경에서 찍은 강아지 사진.

이건 결국 **Knowledge Transfer**이고, 한 글자로 줄이면 **Generalization(일반화)**입니다.

| 방식 | 하는 일 | 대표 서비스 |
|------|---------|------------|
| **Information Retrieval** | 이미 있는 데이터(X, Y) 중에서 쿼리에 맞는 것을 **찾아줌** | 구글 검색 |
| **Generation** | 본 적 없는 X에 대해서도 **새 Y를 만들어** 답함 | ChatGPT 등 생성형 AI |

검색은 “있는 것을 긁어오는” 일이고, 생성은 “없는 것을 상상해서 채우는” 일입니다. AI가 구글의 비즈니스 모델을 흔들 수 있는 이유가 바로 여기 있습니다.

> ⭐ **시험 포인트**  
> **도메인 어댑테이션**은 Weakly SL의 핵심 맥락입니다. X_old → X_new로 바뀔 때 F가 적응하는 문제 = Knowledge Transfer = Generalization.

## 1-3. Weakly SL의 두 가지 전략

라벨이 부족할 때 해법은 크게 **두 가지**뿐입니다.

1. **Inductive Bias(귀납적 편향)를 잘 설계한다**  
   - 도메인 지식이 필요합니다.  
   - 예: CNN(패치 단위 인코딩), L1/L2 regularizer, 트리 깊이 제한, 아키텍처 설계  
   - Loss = Likelihood(데이터) + Regularizer(= Prior = Inductive Bias)

2. **Pre-training을 잘 한다**  
   - 언라벨 데이터를 대량 활용해 모델을 “예열”  
   - 요즘의 주류: self-supervised / unsupervised pre-training → 소량 라벨로 fine-tuning

> ⭐ **시험 포인트**  
> **Weakly SL 두 전략 = Inductive Bias + Pre-training**. 시험에서 하이레벨로 이 그림을 그리면 풀 수 있는 문제로 출제되었습니다.

### Inductive Bias 예시

- **Maximum Margin** → SVM  
- **Minimum Description Length** → L1/L2 regularizer  
- **Maximum Conditional Independence** → Naive Bayes  
- **Neural Network Architecture** → CNN, Transformer 등

데이터 외의 모든 설계 선택은 Inductive Bias입니다. Bayesian 관점에서 **Likelihood = 데이터**, **Prior = Inductive Bias**로 보면 됩니다.

## 1-4. Bias–Variance Trade-off

머신러닝에서 가장 중요한 개념 중 하나입니다.

- **Bias(편향)**: 모델에 강하게 걸린 가정 → 리지드한 모델, underfitting 경향  
- **Variance(분산)**: 데이터 다양성을 모델이 얼마나 반영하는가 → overfitting 경향

둘은 **상충**합니다. 과한 bias도, 순수 likelihood만 쫓는 것도 좋지 않습니다.

- 데이터가 **적을수록** inductive bias가 더 중요해집니다.  
- 데이터가 **많을수록** 노이즈가 평균 0으로 수렴(CLT) → benign overfitting 가능.

> ⭐ **시험 포인트**  
> **Bias–Variance trade-off**는 “어떻게 잘하냐?”의 답이 “잘 해야 한다”로 끝나는, 레시피의 영역이지만 개념 자체는 반드시 알아야 합니다.

## 1-5. MLP vs CNN — 컴퓨터 비전 40년의 교훈

MLP는 픽셀 **순서와 무관**하게 학습합니다(곱셈·덧셈의 교환법칙). UAT(Universal Approximation Theorem) 덕분에 이론상 어떤 함수든 근사 가능하지만, **데이터가 엄청나게 많이** 필요합니다.

컴퓨터 비전 40년 연구의 결론:

> 이미지는 **픽셀 하나**가 아니라 **주변 픽셀 묶음(패치)**으로 봐야 의미가 생긴다.

그래서 **패치 단위 인코딩** = CNN. 같은 성능에 MLP는 10억 장, CNN은 1,000장 수준이면 된다는 식의 데이터 효율 차이가 납니다. CNN 구조 자체가 넓은 의미의 **Inductive Bias**입니다.

## 1-6. Zero-shot vs Unsupervised — 헷갈리기 쉬운 독립 개념

| | Zero-shot Learning | Unsupervised Learning |
|--|-------------------|----------------------|
| **핵심 질문** | 해당 **데이터를 학습에 보여줬느냐** | **슈퍼비전(라벨)을 줬느냐** |
| **예시** | “하늘 나는 자동차” 샘플을 아예 안 줌 | 자동차 사진은 주되, 뭔지 라벨은 안 줌 |

두 개념은 **완전히 독립**입니다.  
- Unsupervised인데 zero-shot이 아닐 수 있음(데이터는 봤지만 라벨 없음)  
- Zero-shot인데 supervised pre-training을 거쳤을 수도 있음(논란 여지)

Few-shot의 “few” 기준(10~20개)도 엄밀한 정의는 없고 주관적입니다.

> ⭐ **시험 포인트**  
> Zero-shot vs Unsupervised는 “시험에 내기 좋은” 주제지만, 교수님은 **직접 이 문제로는 내지 않았고** 더 어려운 응용 문제를 냈다고 하셨습니다. 개념은 알아두되, 시험 예상 1~3번의 핵심은 아닙니다.

# 2부. 260706 오후 — Self-supervised, CLIP, Instruction FT, RLHF

## 2-1. Self-supervised Learning과 Representation

**Self-supervised learning** = 라벨 없이 **스스로 만든 proxy task**로 representation을 학습.

비전에서의 pretext task 예:
- **Relative patch location**: 패치 A가 B의 어디에 있었는지 맞추기  
- **Jigsaw puzzle**: 셔플된 패치 순서 복원  
- **Inpainting / Outpainting**: 잘린 부분 채우기  
- **Colorization**: 흑백 → 색 예측

NLP에서는 **Next Word Prediction(Language Modeling)**이 사실상 모든 NLP 태스크의 메타 태스크가 되었고, 이게 LLM의 기반입니다.

## 2-2. Contrastive Learning과 CLIP

**Contrastive learning**: positive pair는 가깝게, negative pair는 멀게.

**CLIP** = 이미지 인코더 + 텍스트 인코더. 배치 안에서 **맞는 (이미지, 텍스트) 쌍의 similarity를 maximize**, 나머지는 minimize.

**Cosine similarity** = 두 벡터 사이 각도 θ의 cos 값. 방향이 같을수록(θ→0) 값이 커짐.  
내적(dot product)은 magnitude가 1에 가깝다고 가정하면 cosine similarity와 같아집니다.  
`1 - cosine_similarity`로 distance 형태로 바꿔 부르는 경우도 있습니다(용어 혼용 주의).

**SigLIP**: softmax 대신 **sigmoid**로 바꿔 배치 병렬화를 쉽게 함.  
**SigLIP2**: Local loss(더 세밀한 visual understanding) + EMA image encoder(천천히 움직이는 teacher로 안정적 embedding).

> ⭐ **시험 포인트**  
> **Embedding** = topology-preserving map(구조를 보존하는 사상). CLIP은 cosine similarity로 이미지·텍스트를 같은 공간에 배치합니다.

## 2-3. Meta-learning과 Instruction Fine-tuning

**Meta-learning / MAML**: 여러 태스크에서 “빨리 적응하는” 초기값을 학습.  
**Instruction Fine-tuning**의 정신적 원형입니다.

논문 제목 그대로:  
*Fine-Tuning Language Models on a Collection of Tasks Described via Instructions Improves the Zero-Shot Performance on Unseen Tasks*

- GPT-3 → 다양한 instruction으로 fine-tune → **InstructGPT**  
- 여전히 챗봇으로 쓰기엔 헛소리·컨텍스트 문제 → **Human Preference** 반영 → **RLHF** → **ChatGPT**

## 2-4. Instruction Fine-tuning의 한계 (→ RLHF로 이어지는 이유)

교수님이 강조한 **세 가지 한계**:

### ① QA Pair 수집 비용
Instruction–Answer pair = Ground Truth( CV에서는 Ground Truth, NLP에서는 Gold data).  
수집에 **시간 + Human cost**가 많이 듭니다.

### ② Creative Generation의 어려움
Instruction tuning 데이터는 **명확한 정답**이 있는 QA 형태.  
소설·창작처럼 “좋다/나쁘다”가 주관적인 open-ended 생성은 학습하기 어렵습니다.

### ③ 토큰 레벨 동등 penalty (가장 중요할 수 있음)

Language modeling objective는 **모든 토큰 오류를 동일하게** penalize합니다.

예: “2026년 미국 대통령은?” → 정답은 Trump인데 “조 바이든”이라고 하면  
- “조 바이든” 토큰이 핵심 오류  
- “2022년”은 맞고 “2025년”은 틀림 — **중요도가 다른데** LM은 전부 똑같이 벌점

Instruction fine-tuning만으로는 **어떤 토큰이 human preference에 더 중요한지** 차등 반영이 안 됩니다.  
→ **Human preference 데이터**로 추가 학습 = **RLHF**

> ⭐ **시험 포인트**  
> **Instruction FT 한계 + RLHF가 극복하려는 것**이 시험 문제에 스며들었을 가능성이 높습니다. 특히 **토큰 레벨 동등 penalty**와 **human preference 인코딩**을 연결해서 설명할 수 있어야 합니다.

## 2-5. RLHF에서 왜 Reinforcement Learning을 쓰는가

RL이 맞는 **한 가지 시나리오**:  
**지금 당장** 각 스텝이 잘됐는지 판단하기 어렵고, **시퀀스가 끝난 뒤**에야 좋은지 나쁜지 알 수 있을 때.

| 도메인 | 왜 RL? |
|--------|--------|
| **로봇 보행** | 매 순간 모터 각도를 프로그래밍하기 불가 → 끝에 “넘어지지 않음” reward |
| **챗봇** | 각 턴마다 좋은지 애매 → 대화 전체가 끝난 뒤 good/bad feedback |
| **바둑(AlphaGo)** | 한 수의 승률은 불명확 → 수십~수백 수 뒤 승패로 역산 |

챗봇 학습: “시퀀스로 대화해라 → 마지막에 이 대화는 좋았다/나빴다” = **RL만이 자연스러운 프레임워크**.

> ⭐ **시험 포인트**  
> RLHF에서 RL을 쓰는 이유 = **시퀀스 끝에서만 reliable reward를 줄 수 있기 때문**. 중간 토큰별 supervised label이 어렵다.

# 3부. 260707 오전 — Continual Learning

## 3-1. Continual Learning이란

**Continual Learning(= Lifelong Learning = Continuous Learning)**  
이미 학습된 모델이 있고, **새 데이터가 순차적으로 들어올 때** 모델을 업데이트하는 패러다임.

**Non-stationarity(비정상성)**: 데이터 분포가 끊임없이 바뀌는 세상에서 모델도 계속 바뀌어야 한다.

Weakly SL과의 공통점:

> 이미 학습한 지식을 **transfer**해서 새 데이터에 적용한다. 처음부터 다시 학습할 이유가 없다. (오렌지에서 배운 것으로 자몽을 배우듯)

## 3-2. Stability–Plasticity Dilemma

뇌·신경망 모두 facing하는 **트레이드오프**:

| | 의미 | Continual Learning에서 |
|--|------|------------------------|
| **Stability(안정성)** | 기존 지식 유지, 안 까먹음 | Catastrophic Forgetting 방지 |
| **Plasticity(가소성)** | 새 지식 수용, 말랑말랑함 | Adaptability |

- 안 까먹으려 하면 → 새 것을 안 배우게 됨  
- 새 것을 배우려 하면 → 옛것을 까먹음  

**둘을 밸런스**하는 것이 Continual Learning의 목표입니다.

> ⭐ **시험 포인트**  
> Continual Learning의 핵심 = **Catastrophic Forgetting vs Adaptability 밸런싱**. 클래스 incremental 연구가 왜 catastrophic forgetting에 집중했는지 이 맥락에서 설명할 수 있어야 합니다.

## 3-3. Catastrophic Forgetting

**Catastrophic Forgetting**: 새 지식을 배우다 **기존 지식이 통째로 사라지는** 현상.  
1989년 McCloskey & Cohen이 발견.

왜 일어나는가:
- **파라미터 수 < 저장해야 할 지식의 양**  
- Continual learning(특히 1 epoch 순차 학습)은 multi-epoch 복습보다 forgetting에 **더 취약**

완화 아이디어:
- Multi-epoch으로 **복습**  
- **파라미터 수 증가**  
- **Episodic memory**(버퍼에 과거 샘플 저장 후 replay)

## 3-4. Class Incremental Learning과 셋업 비판

**Class Incremental Learning(CIL)** = Continual Learning의 대표 인스턴스이지만, **전부는 아님**.

학계 5~6년간 CIL이 “대세 셋업”이었지만, 교수님은 **너무 포커스할 필요 없다**고 강조합니다.

### Disjoint setup
- Task마다 클래스가 예쁘게 묶여 들어옴 (Task1: 클래스 1,2,3 / Task2: 4,5,6 …)  
- 현실과 거리가 있음

### Blurry setup (Oxford, ~2020)
- 모든 클래스가 처음부터 등장, 빈도만 다름 (major / minor class)  
- “새 클래스 추가”라는 incremental의 본질과 어긋날 수 있음

### Incremental Blurry (교수님 연구실 제안, 2022)
- 일부 클래스는 disjoint하게 **reserve**했다가 순차 추가  
- 나머지는 blurry하게 공유 → disjoint↔blurry 일반화  
- (인기는 없었음)

## 3-5. Continual Learning 방법론 3가지

| 카테고리 | 아이디어 | 예시 |
|----------|---------|------|
| **Replay-based** | 과거 데이터 일부(1~5%)를 버퍼에 저장, 새 학습 시 함께 replay | Episodic memory, generative replay |
| **Regularization-based** | Task1에서 배운 지식을 Task2 학습의 regularizer로 | EWC, SI, HAT (prior/data focus) |
| **Parameter isolation** | 새 지식용 별도 네트워크 → merge 또는 계속 추가 | Fixed/Dynamic architecture |

**Dynamic architecture**의 문제: inference 시 어떤 서브네트워크를 쓸지 모름, 모델 크기 계속 증가.

## 3-6. GDUM — “바보 같이 단순한데 SOTA를 이긴다”

**GDUM** (Greedily balanced sampler + Dumb learner):

1. Episodic memory 버퍼에 지금까지 본 샘플을 **클래스별 균형** 맞춰 저장 (greedy balancing)  
2. 매 step **기존 모델은 버리고**, 버퍼 데이터만으로 **from scratch** 재학습  
3. 복잡한 continual learner가 아니지만 blurry 등에서 **기존 SOTA(EWC, SI, HAT 등)를 능가**

교수님 해석: “니네 다 잘못하고 있었어” — continual learning 방법들이 과도하게 복잡했을 수 있다.

(실습에서 GDUM을 다루었으나, 본 문서에서는 실습 제외)

## 3-7. Online vs Offline Continual Learning

| | Online CL | Offline CL |
|--|-----------|------------|
| **학습 방식** | 스트림 데이터를 **1 epoch**만 보고 지나감 | 버퍼에 모아 **multi-epoch** 학습 |
| **특징** | Forgetting 심함, 빠른 적응 필요 | 상대적으로 안정, 비현실적일 수 있음 |

데이터가 스트림으로 들어온다면 **online이 더 자연스러운** 시나리오.  
그런데 벤치마크는 often offline으로 비교 → **공정 비교 이슈**.

# 4부. 260707 오후 — Large Model CL, CoLoRA, 메모리, 시험 가이드

## 4-1. Large Model에서 Forgetting vs Adaptability

클래스 incremental 연구는 **작은 모델 + catastrophic forgetting**에 최적화되어 왔습니다.

**Large model(7B+)** 에서 실험하면:
- Forgetting은 상대적으로 **작아지는** 경향  
- **Adaptability(새 태스크 적응)** 가 더 큰 이슈

결론: **Large model continual learning에서는 forgetting보다 adaptation이 더 중요**합니다.

- 쉬운 태스크 → adaptability ↑  
- 어려운 태스크 → adaptability ↓  
- **Curriculum learning**(쉬운 것부터)이 forgetting도 줄이고 adaptability도 높임  
- 그러나 continual learning에서 task 순서를 임의로 조작하면 **비현실적**이라는 비판도 있음

> ⭐ **시험 포인트**  
> Large model CL: **forgetting ↓, adaptability가 dominant issue**. CIL만 연구할 필요 없다는 논지와 연결됩니다.

## 4-2. Adaptive Layer Freezing (ELSA)

**문제**: 어떤 파라미터를 freeze할지 2^N 경우의 수 → 탐색 불가.

**아이디어**: 레이어 단위로 freeze. 뒤쪽(semantic) 레이어가 classification adaptation에 더 중요 → **뒤에서부터** 몇 개 레이어만 업데이트.

**Batch Informativeness** = Fisher Information Matrix의 trace  
→ 정보 gain / compute cost 비율이 최대인 레이어까지만 업데이트, 나머지 freeze  
→ **성능 유지 + FLOPs 대폭 절감**

## 4-3. Federated Heterogeneous CL과 CoLoRA

**Heterogeneous Continual Learning**: Task마다 **아키텍처가 다름** (LeNet → ResNet → Transformer).

**Federated Learning**: 서버 + 여러 클라이언트가 파라미터를 주고받으며 학습.  
전통적 FedAvg는 **같은 아키텍처** 가정 → 서버는 큰 모델, 클라이언트는 작은 edge 디바이스일 때 비효율.

**CoLoRA (Collaborative LoRA)** — 교수님 최근 연구:

- 각 클라이언트·서버의 **pretrained network + LoRA adapter**  
- LoRA: \(W \approx AB\), rank R로 low-rank 근사  
- 클라이언트마다 input/output dimension이 달라 **그냥 average 불가**

**핵심 아이디어**: LoRA를 **A · Core(R×R) · B** 세 부분으로 factorize  
- **Core R×R matrix**에 대부분 정보 저장 (rank R은 hyperparameter로 통일 가능)  
- Core를 **full rank**로 학습시키는 constraint → 정보 maximization  
- 서로 다른 task/client의 Core를 **comparable**하게 alignment (CCL, L2 norm minimization)  
- Federated aggregation 후 서버에서 reconstruction

## 4-4. Memory Agent — 뇌와 AI의 공통 철학

Continual learning agent를 **메모리 구조**로 보면:

| 구성 | 역할 | AI 구현 예 |
|------|------|-----------|
| **Core model** | 기본 추론 엔진 | 신경망 본체 |
| **Working memory** | 단기 작업 기억 | 현재 컨텍스트 |
| **Long-term memory** | 장기 저장 | **RAG** — 인터넷 문서를 external long-term memory로 |
| **External world** | 환경과 상호작용 | 인터넷·실세계 피드백 |

모델은 **internal + external**에 걸쳐 있고, RAG는 “웹에 있는 long-term memory를 retrieve해서 generation”하는 구현입니다.

## 4-5. 두 강의의 마무리 메시지

**Weakly SL + Continual Learning 공통 철학**:

1. 이미 학습한 knowledge를 **transfer**해서 쓴다  
2. 처음부터 다시 학습할 필요가 없다  
3. 현재 training paradigm이 이를 제대로 반영하는지 **의문**  
4. Continual learning은 **non-stationarity**를 다루는 방법이며, 러닝 패러다임·아키텍처가 계속 바뀌는 시대에 더 필요해짐

**Large model 시대**: CIL 중심 연구가 통하지 않을 가능성. **Adaptation + realistic setup**이 더 의미 있을 수 있음.
