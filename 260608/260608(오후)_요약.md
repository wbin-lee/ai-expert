## 0. 오프닝 및 실습 개요

조교는 출석부 서명 확인 후, 본인 소개를 시작했다. "저는 오늘 실습을 맡은 카이스트 홍수원(홍승훈) 교수님 연구실의 박사과정 유재훈"이라고 인사를 하며 오후 실습을 시작했다.

오늘 실습할 내용은 **지난주 금요일에 실습했던 세그멘테이션 모델을 컨볼루셔널 뉴럴 네트워크(CNN)가 아니라 트랜스포머 기반의 네트워크로 구현해 보는 것**이다. 오전 수업에서 다룬 **"트랜스포머 디코더 폴(for) 세그멘테이션" 구조**를 직접 구현해 본다.

전체 코드 구성은 금요일 FCN 실습과 크게 다르지 않아 큰 어려움은 없을 것이라고 안내했다. 실습 자료에서 "세그멘터(Segmenter) 폴더"의 자료를 사본으로 떠서 본인 드라이브에 셋업하라고 지시했다.

### 오늘 실습의 전체 구조

조교는 그림을 보여주며 설명했다.

- **인코더(Encoder)** 가 있고, 인코더의 출력을 받아 **디코더(Decoder)** 가 세그멘테이션을 수행한다.
- 금요일에 실습했던 것은 이 인코더-디코더 구조의 **컨볼루션 버전**이었다.
- 오늘은 **이 둘을 모두 트랜스포머로 구현** 한다.

### 실습 파트 구성 (총 3파트)

1. **P1 — Attention Layer 구현**: 트랜스포머의 기본 구성 블락인 attention 구현
2. **P2 — Mask Transformer 구현**: attention으로 만든 트랜스포머 블락을 활용해 decoding을 수행하는 마스크 트랜스포머 구현
3. **P3 — Segmenter 결합**: 앞에서 만든 트랜스포머 인코더와 마스크 트랜스포머 디코더를 결합해 최종 세그멘터 완성

조교는 "오늘 내용 자체는 금요일 실습과 크게 다르지 않기 때문에 조금 빠르게 2시간 내로 끝을 내고, 마지막 한 시간은 학습 돌리시면서 자습 내지는 질문 받는 시간으로 진행"하겠다고 안내했다.

### 사전 준비

콜랩 노트북 사본을 만든 뒤, 지난주 금요일과 동일한 방식으로:
- 라이브러리 import
- 데이터로더 셋업
- **텐서보드(TensorBoard)** 도 함께 불러옴
- **CRF(Conditional Random Field)라는 후처리 작업**도 할 수 있는 코드를 불러옴

이 부분은 금요일과 동일하므로 그냥 쭉 실행하면 된다. 그 다음 "P1 Implement Attention Layer"부터 본격적인 실습 시작.

## 1. P1 — Attention Layer 구현

### 1-1. Attention 연산의 기본 원리

조교는 오전에 비전 트랜스포머(ViT)와 트랜스포머 기반 세그멘테이션에 대해 배웠을 것이라고 상기시키며, 트랜스포머의 핵심인 attention 연산을 설명했다.

**Attention 연산의 흐름:**

1. **Query, Key, Value(Q, K, V)** 라는 세 가지 컨셉으로 수행된다.
2. Input image에 해당하는 **X**가 들어오면, X가 Q, K, V 세 가지 타입으로 나눠진다.
3. 이 분기는 각각 `self.q`, `self.k`, `self.v`에 해당하는 **리니어 레이어를 통과**시켜 변환된다.
4. 변환된 Q, K, V 토큰들이 attention 연산에 들어간다.
5. **Attention 연산**: 쿼리(Q)와 키(K) 사이의 유사도를 **inner product** 로 계산한 뒤, 이렇게 연산된 유사도를 기반으로 **밸류(V)를 다시 모아주는 연산**을 수행해 아웃풋을 뱉는다.
6. "각 패치 간의 유사도를 기반으로 패치 간의 정보를 모아준다"고 보면 된다.
7. 이 출력이 다시 원래 모양으로 나오게 만드는 것이 전체 attention layer.

### 1-2. 구현해야 할 부분

뚫어놓은 부분(빈칸):
- `self.q`, `self.k`, `self.v`, 뒤쪽의 드롭아웃(dropout) 및 프로젝션(projection) 레이어를 일단 initialize
- Forward 단계: Q, K, V를 리니어 레이어에 태우고, 트랜스포즈로 shape를 맞추고, Q-K 사이의 similarity 기반 attention 계산, 마지막으로 드롭아웃과 프로젝션을 먹여서 최종 출력

조교는 "55분까지 진행해 보시면 될 것 같다"고 안내했고, "솔루션 파일도 올려놨기 때문에 헷갈리면 솔루션을 봐도 상관없다"고 말했다.

> 🔥**조교의 관찰**🔥
> "사실 요즘은 이게 코딩을 직접 하는 경우가 생각보다는 좀 많이 줄어든 것 같기는 해서, 오히려 뭔가 그 본인 구현한 거 혹은 솔루션을 보고 이해하고 검증하는 게 더 중요한 것 같기도 하거든요."

### 1-3. 솔루션 설명 (Initialization)

- **QKV 매핑**: 각각 **리니어 레이어(Linear Layer)** 로 구현
- **Attention drop**: Drop-out 레이어 활용
- **Projection layer**: QKV 사이 attention 연산이 끝난 후 다시 한 번 transformation 하기 위해 **하나의 리니어 레이어**가 들어감

### 1-4. 솔루션 설명 (Forward — Multi-Head Attention)

**핵심 포인트 — Multi-Head Attention:**

기본적으로는 QKV 연산을 하면 되는데, 신경 써야 하는 부분은 **Multi-Head Attention**을 구현하기 때문이다.

- 각 채널을 **헤드 갯수로 나눠준다**.
- 각 헤드별로 **서로 다른 Attention**, 즉 쿼리와 키 사이의 similarity를 **다양한 방면에서 보기 위해서** 헤드를 나눠 놓은 것.
- 각 헤드에 따라 서로 다르게 보는 유사도 개념으로 헤드별 연산을 처리한 뒤 **마지막에 모아주는 방식**.

**Reshape & Permute:**
- `reshape` 부분 → 각 헤드별로 다르게 분리하기 위함
- `permute` 부분 → 뒤쪽에서 유사도를 계산할 때 matrix multiplication을 이용하는데, 이게 **특정한 shape 순서를 요구하기 때문에** 이를 맞춰주려고 넣음

### 1-5. self.scale의 역할 (매우 중요)

🔥**시험포인트** — 조교가 "왜" 위주의 시험문제를 강조하면서 직접 언급한 개념🔥

> 조교 원문:
> "attention 보시면 보통 일반적으로 저 **self.scale**이라는 게 항상 들어가게 되는데, 저 친구의 역할은 저희가 softmax를 수행을 하게 되면 이제 값이 너무 크게 되면 attention이 너무 한쪽으로 치우쳐버릴 수 있어요. 그래서 저희가 attention을 보통 쓰는 목적은 글로벌하게 보고 이 사이에서의 similarity를 좀 잘 활용하길 원하는 건데, 이게 너무 샤프해지면 특정한 토큰만 보는 식으로 좀 안 좋게 배워질 수가 있거든요. 그래서 이거를 해결하기 위해서 self.scale이 보통은 이제 **디멘션의 루트 값으로 나눠주는 방식**으로 구현이 됩니다."

- 정규화된 값을 softmax에 넣어 similarity로 변환
- 그 뒤의 **드롭아웃**은 과적합 방지를 위해 들어가는 일반적인 드롭아웃

### 1-6. Forward 마지막 단계

- attention weight(유사도)를 다 계산 → 이 유사도를 기반으로 **밸류(V)를 다시 모아줌** (matrix multiplication으로 구현됨)
- 헤드별로 분리된 것을 **다시 묶어주는** 작업: "헤드별로 서로 다른 관점으로 유사도를 보고 정보를 처리한 다음에 다시 정보를 모아서 transformation을 완성해야 되기 때문에"
- 다시 `B, N, C` 원래 모양으로 바꿔주고, 마지막에 한 번 더 **리니어 레이어**를 얹어주는 식으로 attention 구현 완성

## 2. P2 — Mask Transformer 구현

### 2-1. 마스크 트랜스포머 리캡 (오전 수업 내용)

P1에서 만든 attention 블락을 활용해 마스크 트랜스포머를 구현한다. 트랜스포머라는 레이어 자체는 어텐션 블락을 여러 개 쌓아서 만들어주는 것이기 때문에, P1에서 구현한 어텐션 블락이 마스크 트랜스포머 안쪽에 들어간다.

### 2-2. 마스크 트랜스포머의 두 입력

마스크 트랜스포머는 **두 가지 입력**을 받는다:

1. **Contextualized tokens (xenc)**: `self.encoder`의 output들. 즉, 인코더가 이미지를 처리해서 나온 결과.
2. **xcls (클래스 임베딩)**: 각각의 클래스가 어떤 특성을 가지고 있는지 **모델이 학습하면서 미리 저장해 둘 수 있도록** 하는 입력.

> 조교의 설명 예시:
> "트리에 해당하는 임베딩이 초록색, 사이드워크에 해당하는 임베딩이 노란색, 펄슨(person)에 해당하는 임베딩이 빨간색, 이런 식으로 표시가 되어 있는 걸 보실 수가 있고요."

### 2-3. 마스크 트랜스포머의 동작

- 이미지를 보고 정보를 처리한 **XENC** 와, 클래스에 대해서 우리가 갖고 있는 정보 **XCLS** 간의 정보를 서로 연산.
- X인코더의 패치들 중에 **어떤 게 어떤 XCLS에 가까운가**를 최종적으로 출력.
- 기본 attention 연산을 통해 상호작용 가능하게 만든 뒤, 최종적으로:
  - "이미지를 보고 우리가 클래스를 파악한 것" (P)
  - "xenc, cls를 보고 이미지를 파악한 것" (C)
- 이 둘 사이 유사도를 **matrix multiplication**으로 연산해 **최종적으로 클래스 마스크가 나오게 됨**.

### 2-4. 구현 흐름 (Forward)

조교가 구현 순서를 풀어 설명했다:

1. **두 개의 입력 받기**:
   - input X = `xenc` (인코더 출력)
   - `self.cls_emb` = 모델이 내부적으로 CLS에 대한 정보를 학습하며 축적해 놓는 **클래스 임베딩**
2. **배치 처리**: `cls_emb`에 대한 카피를 만들어 둠 (배치 개수만큼)
3. **Concatenate (concat)**: 두 입력을 하나의 트랜스포머에 넣기 위해 **토큰의 length 방향, 즉 N 방향으로 concat**해서 트랜스포머에 넣음 (디멘션 방향이 아니라 토큰 길이 방향임을 강조)
4. **Transformer Block 통과**: 앞서 P1에서 구현한 어텐션이 들어가 있는 블락을 쌓아 마스크 트랜스포머 구성
5. **분리**: 최종 출력을 → P (이미지 패치 토큰) 와 C (클래스 토큰)로 분리
6. **유사도 계산**: P와 C 사이의 유사도를 **matrix multiplication**으로 연산 → "이 패치는 어느 클래스에 가깝구나"를 기반으로 최종 세그멘테이션

### 2-5. 구현 진행 안내

이거는 2시 20분까지 진행, 2시 20분에 쉬는 시간 10분, 필요하면 뒤에 시간 더 제공.

### 2-6. 솔루션 상세 설명

**블락(Block) 구성:**
- 마스크 트랜스포머를 구성하기 위한 트랜스포머 블락을 쌓기 위해 **`ModuleList` 활용**
- 어텐션이 들어가 있는 블락을 여러 개 구성

**CLS 임베딩:**
- 클래스에 대한 정보를 담을 수 있도록 **클래스 개수만큼의 d-dimension 벡터**를 둠

**프로젝터 디코더 (`proj_dec`):**
- 모델 디멘션을 맞춰주기 위함
- 인코더에서 온 input embedding (xenc)을 d_model로 맞춰주는 역할

**나머지:**
- 출력 쪽에 달려있는 projection 레이어, **LayerNorm** 레이어를 구성

**Forward 단계 상세:**

1. 디코더에 쓰는 input dimension과 디코더 내부 dimension, 그리고 인코더에서 온 피처벡터 dimension이 맞지 않을 수 있어서, 정의한 `proj_dec` 레이어로 **x의 디멘션을 d_model로 맞춰줌**.
2. CLS 임베딩은 배치 개수만큼 들고 있지 않고, **각각의 클래스에 대한 정보만 들고 있음**. 다만 input은 배치 개수만큼 들어올 수 있으므로 **`Expand` 활용**해서 shape 맞춰줌.
3. 두 입력을 트랜스포머에 넣기 위해 **토큰 방향(length N 방향)으로 붙임**.
4. 정의한 Transformer Block을 여러 번 반복.
5. **LayerNorm** 태워서 transformation.
6. 마지막 출력을 → 이미지에 해당했던 토큰들(patches)과 CLS에 해당했던 토큰들(클래스를 나타내는 정보)로 **분리**.
7. 각 패치와 각 클래스 사이의 유사도를 계산 → 각 패치가 어느 클래스에 해당하는지 **마스크를 계산**.
8. 최종적으로 **MaskNorm 레이어** 태우고, shape를 `[B, 클래스 갯수, 패치 갯수(GS), 패치 갯수(GS)]` 로 맞춰서 출력.

### 2-7. 테스트 관련 주의사항

테스트를 돌렸을 때 "4번 같은 경우에는 시드가 고정이 안 되는 문제가 있어서 **3번까지만 패스됐는지 확인**해 주시면 될 것 같다"고 안내.

## 3. P3 — Segmenter (최종 결합)

### 3-1. P3 개요

앞서 어텐션을 구현 → 그 블락을 활용해 마스크 포머 디코더를 구현 → 이제 **인코더와 붙여서 세그멘터를 완성**.

조교는 "P3는 아까 P1, P2보다는 훨씬 간단하고, 이게 전부예요"라고 표현.

### 3-2. 구현 흐름

**인코더:**
- **Pretrained encoder를 사용** 할 것이므로 **TIMM 모듈을 활용** 해서 인코더를 가져옴.

**CLS 토큰 제거 (Remove CLS Token):**

🔥**시험포인트 가능성 — 컨벤션과 관련된 개념 설명**🔥

- 보통 ViT는 오전 수업에서 설명했듯이 **classification task에 학습되어 있는 경우가 많음**.
- 이 convention 때문에 보통은 **CLS 토큰이라는 것이 달려 있음**.
- CLS 토큰은 원래 **classification 할 때 쓰는 토큰**인데, 우리는 classification을 인코더가 할 게 아니라 디코더가 할 것이므로 **CLS 토큰을 없애줘야 함**.
- 그래서 `Remove CLS token` 파트가 있음.

**디코더 통과:**
- CLS 토큰을 없애 `H, W` 계의 토큰 (`H × W` 개) 을 얻은 후, P2에서 구현한 디코더에 넣어 마스크 생성.

**Resize:**
- 디코더 출력은 `[GS, GS]` 계수로 나오는데, 이걸 이미지에 다시 덮어 씌우기 위해 **이미지 사이즈로 interpolate**.
- 이렇게 하면 세그멘터 구현 완성.

### 3-3. Forward 상세

```
1. 이미지 입력 → 이미지 사이즈를 H, W로 저장 (마지막에 마스크를 이미지 사이즈로 되돌리기 위해)
2. 이미지 → 인코더 → 피처 추출
3. CLS 토큰 제거
4. 마스크 트랜스포머에 넣기
5. 마스크를 원래 이미지 사이즈로 resize/interpolate
```

### 3-4. 학습 및 시각화

- Training, encoder 부분은 금요일 실습과 동일하므로 설명 생략.
- 학습을 돌리고 inference 점수 확인.
- 텐서보드에 학습 결과물, 세그멘테이션 마스크가 나오는지 확인.
- 텐서보드에 로스 그래프, 세그멘테이션 점수, 세그멘테이션 결과가 같이 나옴 → 실제 이미지 기준으로 얼마나 잘 나오는지, 학습에 따라 얼마나 개선되는지 확인 가능.

## 부록: 오늘 실습에서 다룬 모델 컴포넌트 한눈에

| 컴포넌트 | 핵심 |
|---------|------|
| Attention | Q, K, V 분기 → similarity → V 모으기 |
| Multi-Head | 채널을 헤드 수로 split → 여러 관점의 유사도 |
| self.scale | softmax가 sharp해지지 않도록 dim의 sqrt로 나눔 |
| Mask Transformer | xenc + xcls를 N 방향 concat → Transformer Block → P, C 분리 → P·C similarity → mask |
| cls_emb | 클래스별 정보를 담는 학습가능한 임베딩 |
| Expand | cls_emb를 배치 크기에 맞게 복제 |
| Segmenter | TIMM pretrained ViT (인코더) → CLS 토큰 제거 → Mask Transformer → 원본 이미지 사이즈 interpolate |
| TIMM | Pretrained 인코더를 불러오는 라이브러리 |
| CLS 토큰 | ViT의 classification 토큰 — 세그멘테이션 시 제거 필요 |
| TensorBoard | 학습 로스, 세그멘테이션 점수, 시각화 결과 확인 |
| CRF | 후처리 (post-processing) 옵션 |
