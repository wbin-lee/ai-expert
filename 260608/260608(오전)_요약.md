# 260608 오전 강의 요약 — Vision Transformer · Self-Attention as Hypernetwork · Context Encoding · Segmenter(Convolution Decoder/Transformer Decoder) · Open-Vocabulary Segmentation

> 오전 흐름: **Receptive Field 관점에서 신경망 일반화(Element-wise → Local Context → Global Context) → Global Context의 두 극단(파라미터 폭증의 FC vs 효율적 표현력 문제) → **하이퍼네트워크** 개념(가중치를 다른 신경망이 출력) → Self-Attention = 다른 형태의 하이퍼네트워크(Dot-Product Attention, Softmax-Normalized Attention) → CNN에 Self-Attention 레이어를 1×1 Conv로 끼워넣어 receptive field 데이터 전체로 → 비전 트랜스포머(ViT)와 CLS Token → Segmentation을 위한 디코더 설계(Convolution Decoder/Skip Connection 포함, Transformer Decoder/CLS Embedding으로 클래스 표현 정제) → Hybrid(DETR-like) Mask Transformer → Open-Vocabulary Segmentation(언어 모델의 임베딩을 클래스 표현으로 사용해 학습 안 본 클래스도 일반화)**
> 강의 주제: **트랜스포머가 NLP에서 출발했지만 비전에서도 dominant한 이유를 Receptive Field와 Hypernetwork 관점에서 이해하고, 이미지→이미지 변환 문제(segmentation/depth/detection)의 디코더 설계를 Convolution/Transformer 두 방향으로 일반화하는 것**.

# 1. Receptive Field 관점에서 신경망 일반화 ⭐

## 1.1 기본 골

입력 시퀀스 \(x_1, \dots, x_n \in \mathbb{R}^d\) → 출력 시퀀스 \(y_1, \dots, y_n \in \mathbb{R}^{d'}\)로 변환하는 함수.

가장 일반적인 형태:

\[
y_i = \sum_j w_{ij} \cdot f(x_j)
\]

- \(f\)는 원소별 변환 (element-wise).
- \(w_{ij}\)는 \(j\)번째 입력이 \(i\)번째 출력에 기여하는 가중치.

## 1.2 가중치 설계에 따른 연산 분류 ⭐

| 설계 | 연산 |
|---|---|
| \(w_{ij} = 0\) (\(i \neq j\)일 때) | **Element-wise mapping** — RF = 1 |
| \(w_{ij} = 1/K\) (인접 K개에서) | **Average Pooling** — RF = K |
| \(w_{ij} = W_{j-i}\) (인접 영역에서, 학습 가중치) | **Convolution** — RF = K, 트랜슬레이션 이퀴베리언트 |
| 모든 \(i, j\)에 대해 정의 | **Fully Connected** — RF = 전체, 파라미터 \(n^2\) |

→ 같은 일반 골에서 출발해 가중치 설계만으로 모든 기본 연산이 도출된다.

# 2. Self-Attention as Hypernetwork ⭐⭐

## 2.1 FC Layer의 문제

Global context를 위해 FC layer를 쓰면:

- 파라미터 = \(n^2\) → 시퀀스 길이에 제곱 비례.
- 가중치 \(w_{ij}\)가 **입력에 무관하게 고정** → 같은 모델이 다른 이미지(예: 다른 스케일)에서 다르게 작동 못함 → 표현력 떨어짐.

## 2.2 Hypernetwork 아이디어 ⭐

\(w_{ij}\)를 그냥 학습 파라미터가 아니라 **다른 신경망의 출력**으로 만든다:

\[
w_{ij} = g(x_i, x_j)
\]

→ 신경망의 가중치를 출력하는 신경망 = **하이퍼네트워크**.

핵심 변화: 가중치가 **입력 데이터에 따라 동적으로 바뀐다**.

## 2.3 Dot-Product Attention 설계

\(g\)를 다음과 같이 설계:

\[
w_{ij} = (W_Q x_i)^T (W_K x_j) / \sqrt{d'}
\]

→ 학습 가능한 \(W_Q, W_K\)로 \(x_i, x_j\)를 각각 query/key로 변환 후 dot product.

## 2.4 Softmax Attention ⭐

추가로 softmax 정규화:

\[
w_{ij} = \frac{\exp(\text{dot})}{\sum_k \exp(\text{dot})}
\]

성질:

- \(\sum_j w_{ij} = 1\) (합이 1)
- \(w_{ij} \geq 0\) (양수)
- 한 곳을 키우면 다른 곳은 작아짐 → 사람의 **attention** 직관과 일치.

## 2.5 Self-Attention 정리

\[
y_i = \sum_j \text{softmax}\left(\frac{(W_Q x_i)^T (W_K x_j)}{\sqrt{d'}}\right) (W_V x_j)
\]

| 항목 | 값 |
|---|---|
| 파라미터 | **3개의 행렬** \(W_Q, W_K, W_V\) (\(d \times d'\)) — 시퀀스 길이 무관 |
| Receptive Field | **전체** (입력 시퀀스 전체) |
| 가중치 | 입력에 따라 **동적**으로 변함 |

→ FC의 단점(파라미터 폭증, 정적 가중치)을 모두 해결한 **하이퍼네트워크 형태의 효율적 global context layer**.

# 3. CNN에 Self-Attention 끼워넣기 ⭐

## 3.1 1×1 Conv로 구현

Feature map \(X \in \mathbb{R}^{C \times H \times W}\)에서 \(N = H \cdot W\)로 두면 \(N \times C\) 시퀀스.

\(W_Q, W_K, W_V\)는 **1×1 conv**로 구현 가능:

```
Q = 1x1_Conv_Q(X)
K = 1x1_Conv_K(X)
V = 1x1_Conv_V(X)
attention = softmax(Q @ K^T / sqrt(d'))
output = attention @ V
```

## 3.2 효과 ⭐

- **파라미터**: 1×1 conv 3개만 추가 (소량).
- **Receptive Field**: CNN의 작은 RF를 **이미지 전체로 확장**.
- **Feature map 크기 변화 없음** — 해상도 손실 없음.

→ FCN의 두 한계(작은 RF, 낮은 해상도) 중 **RF 문제를 가장 효율적으로 해결**.

실험적으로 FCN에 Self-Attention 하나만 추가해도 mIoU 대폭 향상.

# 4. 트랜스포머 = Attention Only 신경망 ⭐

## 4.1 구조

```
입력 토큰 시퀀스 → Element-wise FC + Self-Attention → 반복 → 출력
```

- Convolution이 전혀 없음 — 오직 FC와 Self-Attention만.
- FC는 element-wise 인코딩, Self-Attention은 원소 간 interaction.

## 4.2 Positional Encoding ⭐

Self-Attention은 본질적으로 **permutation equivariant** (순서 무관).

→ 순서/위치 정보를 주려면 **positional encoding**을 더해줘야 함.

핵심 관찰: 트랜스포머 자체는 **데이터에 특화된 컴포넌트가 전혀 없음**.

| 데이터 | 토크나이즈/PE 방법 |
|---|---|
| 텍스트 | 단어 토큰 + 1D PE |
| 이미지 | 패치 + 2D PE |
| 비디오 | 패치 + 3D PE |
| 그래프 | 노드 임베딩 + 구조 정보 |

→ "신경망 자체는 universal, 어떻게 토크나이즈할지만 데이터별로 바꿈" — 통합 패러다임.

## 4.3 Scaling Law

트랜스포머는 데이터/파라미터를 키울수록 **predictable하게 성능이 좋아짐**(CNN이나 FC에서는 보기 어려운 성질).

가설 중 하나: **하이퍼네트워크 구조 자체가 in-context learning의 토대**.

- In-context learning: 프롬프트의 예시만 주면 신경망이 적응하듯 작동.
- 메커니즘: 프롬프트가 내부적으로 **파라미터 업데이트와 같은 효과**를 만들어냄 (cross-attention이 fine-tuning 같이 동작).

# 5. Vision Transformer (ViT) ⭐

## 5.1 구조

```
이미지 → 16×16 패치로 분할 (총 N개)
     → 각 패치를 Linear projection으로 임베딩 → N개 토큰
     → Learnable CLS Token 1개 추가 (총 N+1 토큰)
     → Positional Embedding 더함
     → Transformer (Self-Attention + FC 반복)
     → CLS Token 위치의 출력 → Classification Head
```

## 5.2 CLS Token의 역할 ⭐

- 이미지에서 나온 게 아닌 **학습 가능한 빈 토큰**.
- Self-Attention을 통해 모든 패치와 상호작용하며 **이미지 전체의 요약 표현**을 학습.
- 최종적으로 이 토큰만 뽑아 분류에 사용.

## 5.3 데이터 스케일별 성능

| 데이터 크기 | 결과 |
|---|---|
| 작은 데이터 | CNN이 더 잘함 (inductive bias 이득) |
| 매우 큰 데이터 | **ViT가 더 잘함** (편향 없이 데이터에서 모두 학습) |

→ "데이터 적으면 CNN, 많으면 Transformer."

# 6. Segmenter — Convolution Decoder ⭐

## 6.1 문제

ViT는 패치 단위(예: 16×16) 토큰 출력 → segmentation은 **픽셀 단위** 마스크 필요.

H×W 이미지를 패치로 쪼개면 \((H/p) \times (W/p)\) 토큰 → \(p^2\)배 업샘플 필요.

## 6.2 가장 간단한 방법

```
Trans 출력 (N개의 d-dim 토큰)
    → 1×1 Linear → N개의 k-dim 토큰 (k=num_class)
    → 격자 형태로 재배열 → 작은 score map
    → Bilinear upsampling으로 원본 크기로
```

문제: 패치가 크면 upsampling이 너무 거칠어 마스크 디테일 없음.

## 6.3 U-Net 스타일 디코더 ⭐

각 디코딩 레이어에 **Skip Connection** 추가 — 트랜스포머 인코더의 여러 layer에서 feature를 뽑아 디코더에 전달:

```
디코딩 레이어 입력 = 이전 디코더 출력 + Skip Connection 피처
디코딩 레이어 출력 = Transposed Convolution으로 점진적 업샘플
```

`Reassemble` 연산으로 시퀀스를 2D 격자로 복원, `Resample(s)`로 해상도 조정.

→ FCN-8과 같은 멀티스케일 결합 원리를 트랜스포머에 적용.

## 6.4 컨볼루션 디코더의 추가 장점

- 패치 경계의 불연속성을 conv가 자연스럽게 smoothing.
- 패치 사이즈가 커도 conv가 픽셀 단위 디테일 살림.

→ 메모리 허용 한도 내에서 **패치 사이즈는 작을수록 좋다** (1×1이 이상적이나 메모리 폭증).

# 7. Segmenter — Transformer Decoder (Mask Transformer) ⭐⭐

## 7.1 동기

기존 FC classifier:

\[
\text{score}_c = \langle z_{\text{image}}, z_c \rangle
\]

\(z_c\)는 클래스별 대표 벡터로, **이미지와 무관하게 고정**.

문제: 한 클래스 안에서도 다양한 변종 존재 (자동차 = 스포츠카/버스/트럭). 하나의 고정 \(z_c\)로는 표현 부족.

## 7.2 Mask Transformer 아이디어 ⭐

클래스 임베딩 \(z_c\)를 **이미지에 맞춰 동적으로 정제**:

```
입력: 패치 토큰들 + 클래스 토큰들 (Learnable, num_class 개)
   → Transformer Decoder (Self-Attention)
   → 출력: 정제된 패치 토큰들 + 정제된 클래스 토큰들
   → 최종 score = (정제된 패치 토큰) · (정제된 클래스 토큰)
```

직관:

- "나무" 클래스가 한국 풍경에서는 소나무 풍 임베딩으로, 미국에서는 다른 나무로 정제됨.
- 겨울 사진에서는 잎 없는 나무 특징으로 정제.

→ Classification head 자체가 이미지에 맞게 동적으로 변하는 효과.

## 7.3 구조

```
Encoder: ViT (이미지 패치 → contextualized 패치 토큰)
                ↓
Decoder: Mask Transformer
   - 입력: 패치 토큰 N개 + 클래스 토큰 K개 (learnable)
   - Self-Attention으로 둘 사이 상호 정제
   - 출력 패치 토큰 P (N개) + 출력 클래스 토큰 C (K개)
   - 최종 마스크 = P · C^T (N×K matrix of similarities)
```

## 7.4 시각화

Attention map을 보면:

- 강아지 패치를 기준으로 보면 다른 강아지 영역의 패치들에 attention 집중.
- 같은 클래스에 속하는 패치끼리 자동으로 grouping.

# 8. Hybrid (DETR-like) — Mask2Former 류 ⭐

DETR의 object query 아이디어를 segmentation에 적용:

```
이미지 → CNN U-Net → multi-scale feature maps
                                    ↓
Learnable Object Queries (N개) ── Transformer Decoder (Cross-Attention)
                                    ↓
각 Query → (Class Prediction, Mask Proposal)
                                    ↓
GT와 매칭 (헝가리안 알고리즘과 유사)
```

핵심 모티베이션: N개의 마스크 후보를 먼저 생성하고, 각 마스크에 클래스를 부여.

→ 마스크가 instance level까지 자연스럽게 분리됨 → instance segmentation도 같은 구조로 가능.

# 9. Open-Vocabulary Segmentation ⭐

## 9.1 동기

학습된 클래스에만 한정되는 게 일반적인 segmentation의 한계. 학습 안 본 클래스도 다룰 수 있을까?

## 9.2 핵심 아이디어

**Class Embedding을 언어 모델의 임베딩으로 대체**:

```
이전: z_c = learnable parameter (학습 시 고정)
새로운: z_c = TextEncoder("강아지")  ← CLIP 등 vision-language 모델 활용
```

## 9.3 일반화 메커니즘 ⭐

- CLIP은 (이미지, 텍스트) 쌍을 같은 임베딩 공간에 정렬하도록 학습됨.
- "강아지"와 "개"는 다른 단어지만 비슷한 텍스트 임베딩을 가짐.
- 학습 시 "강아지" 클래스로만 segmentation을 배웠어도, "개" 단어 임베딩으로 query하면 유사한 임베딩 → 자동으로 일반화.

→ 학습 안 본 클래스도 detection/segmentation 가능 — 정확도는 떨어질 수 있지만 출발점 제공. 필요하면 fine-tuning.

# 10. 큰 그림 — Pixel Labeling은 다 같은 문제 ⭐

이미지→이미지 변환:

| 테스크 | 출력 |
|---|---|
| Semantic Segmentation | 클래스 ID |
| Instance Segmentation | 인스턴스 ID |
| Depth Estimation | depth value |
| 3D Reconstruction | 3D coordinate |
| Detection | bbox + class |

→ 다 픽셀별 label 추정. 같은 백본으로 푼다.

**나아가**: analogy-making으로 일반화 가능 — "이런 입력에서 이런 출력이 나왔다, 그러면 새 입력에서는?" 모델을 5~10장으로 학습. 한 모델로 모든 pixel labeling 문제 처리.

→ 발표자 연구실의 연구 방향.

# 11. 오전 핵심 정리 ⭐

- 신경망의 모든 기본 연산(element-wise, conv, average pool, FC)은 **\(y_i = \sum_j w_{ij} f(x_j)\)**의 골에서 가중치 설계 차이로 도출된다.
- **Fully Connected**는 global context를 잡지만 파라미터 \(n^2\)에 정적 가중치라는 두 단점이 있다.
- **Self-Attention**은 \(w_{ij} = g(x_i, x_j)\)로 가중치를 **하이퍼네트워크의 출력**으로 만들어, 파라미터 3개 행렬만으로 입력에 따라 동적으로 변하는 global context를 구현한다.
- **CNN에 Self-Attention을 1×1 Conv 3개로 끼워넣으면** receptive field가 즉시 이미지 전체로 확장되며, 파라미터 추가는 미미하다.
- **트랜스포머는 데이터 무관 신경망**이며, 데이터별로 토크나이즈/PE만 다르게 해주면 텍스트/이미지/비디오/그래프 모두 처리 가능 — universal architecture.
- **ViT**는 이미지를 패치로 토크나이즈하고 CLS Token으로 전체 이미지를 요약하며, **충분히 큰 데이터에서는 CNN을 능가**한다.
- **Segmenter의 Convolution Decoder**는 트랜스포머 인코더의 패치 출력을 U-Net 스타일 skip connection으로 받아 점진적 upsampling해 픽셀 단위 마스크를 만든다.
- **Mask Transformer**는 learnable 클래스 토큰을 패치 토큰과 함께 self-attention해 **클래스 임베딩 자체를 이미지에 맞게 동적으로 정제**한다 — 한 클래스 안의 다양성을 표현.
- **Open-Vocabulary Segmentation**은 클래스 임베딩을 CLIP 같은 언어 모델의 텍스트 임베딩으로 대체해 학습 안 본 클래스에도 일반화한다.
