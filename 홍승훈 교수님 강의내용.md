# 1부. 260622 오전 — 생성 모델의 출발, 오토리그레시브, Attention, Transformer

## 1-1. 분류에서 생성으로: 왜 확률 이야기가 나오나

비전 수업에서 배운 대부분의 문제는 **입력 → 출력**이 **결정적(deterministic)** 이었습니다. 같은 이미지를 넣으면 항상 같은 답이 나옵니다.

분류는 이를 **카테고리(레이블)** 로 바꿔 풀었습니다. 그런데 분류에는 한계가 있습니다.

- 새로운 인지 작업마다 **사람이 레이블**을 달아야 해서 스케일링이 어렵다.
- “배경”처럼 퉁치는 레이블은 **편향된 지식**을 배우게 한다.
- 맥락이 바뀌면 레이블 체계 자체를 다시 짜야 한다.

**생성 모델**은 관점이 다릅니다. “입력이 무엇을 출력할지”가 아니라, **데이터 자체를 만들어내는 것**이 학습 목표입니다.

> ⭐ **시험 포인트**  
> 생성 = **데이터가 따르는 확률 분포를 추정**하고, 그 분포에서 **샘플링**하는 것.  
> **샘플링 = 생성**이다.

1D 예시로 보면, 회색 점(학습 데이터)을 보고 분포(유니폼·가우시안 등)를 추정한 뒤, 그 분포에서 파란 점을 뽑으면 **학습에 없던 새 데이터**가 나옵니다. 고차원 이미지 $X$도 마찬가지로, 모델 $G_\theta$가 만든 샘플들의 분포가 실제 $P(X)$에 가까워지길 원합니다.

분류와의 큰 차이: 생성 모델의 출력은 **불확실**합니다. 분포가 뾰족하면 확실하고, 평탄하면 불확실합니다.

## 1-2. 목적 함수: KL → Cross-Entropy → Likelihood

두 분포 $P^*$(실제)와 $P_\theta$(모델) 사이 거리를 **KL divergence**로 잡으면, $\theta$에 대해 최적화할 때 첫 항은 상수이고 **cross-entropy 최소화**와 같아집니다.

분류 때는 GT 확률(원핫 벡터)을 **알고** 있었습니다. 생성에서는 $P^*(X)$, 즉 **데이터가 존재할 확률**을 **아무도 모릅니다**. 게다가 $X$에 대한 **적분**이라 직접 계산도 불가능합니다.

해결책은 **Monte Carlo 샘플링**입니다.

- 관측 데이터 $X_i$는 이미 $P^*$에서 샘플된 것 → $P^*$를 직접 알 필요 없음
- 적분 대신 유한 샘플의 평균으로 근사 → 계산 가능

이 근사식을 정리하면 **likelihood maximization** (= negative log-likelihood minimization)이 됩니다.

> ⭐ **시험 포인트**  
> - 분류와 생성: **같은 cross-entropy 꼴**, 차이는 GT 확률을 **알 수 있느냐**  
> - 생성 모델 공통 목표: **$P(X)$ likelihood 최대화** (NLL 최소화)  
> - 확률 총량이 1이므로, 학습 데이터 확률을 **높이면** 나머지 영역 확률은 **낮아짐** → 노이즈·비데이터 구분 학습

고차원일수록 joint probability 추정 **분산**이 커집니다. 이벤트가 동시에 일어날 확률은 차원이 늘수록 거의 0에 가깝기 때문입니다.

## 1-3. Chain Rule과 오토리그레시브 모델

고차원 joint $P(X_1,\ldots,X_T)$를 한 번에 추정하기 어렵습니다. **Chain rule**로 쪼갭니다.

$$P(X_1,\ldots,X_T) = \prod_{t=1}^{T} P(X_t \mid X_1,\ldots,X_{t-1})$$

각 **조건부 확률**은 분산이 작아 학습이 안정적입니다. 이것이 **autoregressive(오토리그레시브)** 모델의 핵심입니다.

RNN으로 next-word prediction을 하면, 겉보기엔 분류(cross-entropy)지만 실제로는 **생성 목적 함수**를 factorization한 것입니다.

> ⭐ **시험 포인트**  
> - Factorization **순서**는 바꿔도 joint는 같지만, 각 항의 **분산(불확실성)** 은 달라짐  
> - 언어는 보통 **왼쪽→오른쪽**(previous word prediction)이 불확실성이 적음

## 1-4. RNN의 병목과 Transformer의 등장

오토리그레시브의 원형 과제는 **기계 번역**(sequence-to-sequence)입니다. 초기 encoder-decoder RNN 구조에는 두 가지 구조적 문제가 있었습니다.

### 문제 1: 정보 병목 (long-term dependency)

입력 문장 전체를 **고정 크기 벡터 $C$** 하나에 꾸겨 넣습니다. 교수님 비유: **A4 치트시트**에 수업 전체를 요약해 시험(디코딩)을 보는 것 — 빠진 정보는 복원 불가.

### 문제 2: 병렬화 불가

$h_t$를 계산하려면 $h_{t-1}$이 필요합니다. 학습·추론 모두 **직렬** 처리.

### Attention이 해결하는 것

디코딩할 때 **모든 인코더 타임스텝에 직접 접근**합니다. 치트시트 대신 **강의 녹화본**을 들고 가는 느낌입니다.

- 가중치 $\alpha_{it}$: softmax로 0~1, 합이 1 → **확률 벡터** = attention
- query(현재 hidden)와 key(인코더 hidden) **내적**으로 유사도 계산
- attention은 **skip connection**처럼 gradient가 멀리까지 흘러 **long-term modeling·optimization**에 유리

> ⭐ **시험 포인트**  
> - RNN 병목: (1) **고정 벡터 압축** (2) **직렬 의존**  
> - Attention: 인코더 전 시점 **직접 참조**, 확률적 가중합

### Transformer: recurrence 제거

Attention만으로 과거 정보를 볼 수 있으니 **재귀 연결을 제거**합니다. 토큰 간 attention은 **병렬** 계산 가능.

| | RNN+Attention | Transformer |
|---|-------------|-------------|
| Long-range | attention으로 완화 | self-attention으로 전역 |
| 학습 병렬화 | 여전히 직렬 | **teacher forcing**으로 병렬 |
| 추론 | 직렬 | 여전히 **한 토큰씩** 생성 |

> ⭐ **시험 포인트**  
> - Transformer **학습**은 병렬 가능 (teacher forcing으로 정답 토큰을 미리 넣음)  
> - **추론(생성)** 은 여전히 직렬 — KV caching으로 비용 완화  
> - Scaling law가 Transformer에서 두드러진 이유: 병목 제거 + 하드웨어 활용

## 1-5. AR 모델의 한계와 Flow Matching 예고

Transformer도 완벽하지 않습니다.

- 컨텍스트가 길어질수록 attention 비용 증가 ($O(N^2)$)
- 생성 시 **병렬화 불가** (autoregressive conditioning)
- RNN 계열(linear attention, state space model)은 **상수 시간·메모리**가 장점이나 정보 압축 trade-off

병렬 생성을 노리는 방향으로 **flow matching** 등 비-AR 생성 모델이 주목받고, 이미지·비디오 생성의 표준이 되고 있습니다 (260623에서 이어짐).

# 3부. 260623 오전 — VAE, Beta-VAE, GAN 비교, Diffusion 유도

## 3-1. Latent Variable Model과 VAE 복습

고차원 데이터(이미지·문장)는 전체 공간의 극히 일부 **저차원 manifold** 위에만 존재합니다. **Latent $Z$**가 compact representation이라 가정합니다.

$$P(X) = \int P(X|Z)\,P(Z)\,dZ$$

$P(X,Z)$의 적분은 불가능 → **ELBO (Evidence Lower Bound)** 를 최대화합니다.

VAE 구조:

- **Encoder** $q_\phi(Z|X)$: $X$→가우시안(평균·분산)
- **Decoder** $p_\theta(X|Z)$: $Z$→가우시안(이미지 복원)
- **Prior** $P(Z)$: 보통 $\mathcal{N}(0,I)$
- Loss: **reconstruction** + **KL$(q_\phi(Z|X)\,\|\,P(Z))$**

오토인코더와 차이: VAE는 **확률적** — $Z$는 점이 아니라 분포에서 샘플.

> ⭐ **시험 포인트 (교수님 직접 언급)**  
> **Reparameterization trick**: $Z = \mu + \sigma \odot \epsilon$, $\epsilon \sim \mathcal{N}(0,I)$  
> - 샘플링은 미분 불가 → $\mu,\sigma$로 gradient 흐르게 재표현  
> - 시험에 **여쭤볼 수 있다**고 명시

추론 시: encoder 없이 **prior에서 $Z$ 샘플** → decoder로 생성.

이미지 VAE: encoder CNN, decoder **transposed conv**(업샘플링).

## 3-2. VAE의 뿌연 이미지와 GAN과의 대비

VAE 생성 결과가 **뿌옇다**는 것은 잘 알려진 특성입니다.

| | GAN | VAE |
|---|-----|-----|
| 학습 안정성 | 불안정(min-max, 로스 지그재그) | **ELBO**로 안정 |
| 모드 커버리지 | **mode collapse** 쉬움 (일부 모드만) | KL 때문에 0 확률 **큰 페널티** → 모든 모드에 조금씩 확률 |
| 생성 품질 | sharp | **blur** (모드 사이 평균 효과) |
| 확률 해석 | 암시적 | **명시적**(가우시안, ELBO 항 분해) |

> ⭐ **시험 포인트**  
> VAE blur 원인: multimodal $P(X)$에서 KL이 **모든 곳에 확률을 조금씩** assign → 인접 모드 **평균**  
> GAN은 mode collapse를 **페널티 없이** 할 수 있어 sharp하지만 coverage 약함

## 3-3. Disentanglement와 Beta-VAE

$Z$ 각 차원이 독립 factor(나이, 각도 등)를 담기길 바라지만, 기본 VAE는 **distributed(entangled) representation**이 기본입니다.

**Beta-VAE**: KL 항에 $\beta > 1$ 가중치 → prior(identity covariance, **차원 독립** 가우시안)에 가깝게 → **disentanglement** 경향.

KL을 정보이론적으로 분해하면 3항:

1. **Mutual information** $I(Z;N)$ — $Z$가 데이터 정보 담는 정도 → **높아야** 함
2. **Total correlation** — $Z$ 차원 간 correlation → **낮아야** disentangle
3. **Prior matching** — prior와 가까움 → **낮아야** 함 (생성 시 prior 샘플 가능)

$\beta$를 **전체 KL**에 곱하면 reconstruction까지 망가짐 → **total correlation 항만** 가중치 조절하는 것이 정밀 (β-TC-VAE 아이디어).

## 3-4. Manifold와 latent interpolation

픽셀 공간에서 두 이미지를 선형 보간하면 **manifold 밖**으로 떨어져 이상한 이미지가 됩니다. Latent에서 보간 후 decoder에 넣으면 **valid한 데이터**가 나옵니다.

단, $Z$ 차원은 **해석 불가**(게임 캐릭터 슬라이더처럼 각 축 의미 없음).

## 3-5. Hierarchical VAE → Diffusion으로

VAE의 한계를 **언더피팅** 관점으로 보면: **단순 prior(가우시안)** → **복잡한 데이터 분포**로 가는 변환이 너무 어렵다. 한 번에 던지지 말고 **여러 단계**로 나누자.

**Hierarchical (latent) VAE**: $Z_1, Z_2, \ldots, Z_T$를 **Markov chain**으로 쌓고, 각 단계마다 encoder/decoder가 조금씩 변환.

핵심 아이디어로 diffusion을 유도:

1. **인코딩(노이징) 경로는 손으로 설계** — 학습 안 함
2. **디코딩(디노이징)만 학습**
3. 한 스텝: $X_{t-1}$에 스케일링 + 가우시안 노이즈 추가 → $X_t$
4. 충분히 반복하면 **순수 가우시안 노이즈**에 수렴

```
데이터 X₀ ──(노이즈+)──> X₁ ──> … ──> X_T ≈ 𝒩(0,I)
              포워드(고정)                    ↑
                                              │
         생성: X_T에서 시작해 디노이징 반복 ──┘
              백워드(학습)
```

> ⭐ **시험 포인트**  
> - Diffusion = **Hierarchical VAE의 특수형** (인코딩 경로 **고정**)  
> - VAE: 저차원 $Z$ | Diffusion: **매 단계 차원 = 데이터 차원** (픽셀 공간에서 노이징)  
> - 복잡한 변환을 **작은 step**으로 쪼개 언더피팅 완화

### Forward process (한 스텝)

$$X_t = \sqrt{\bar\alpha_t}\,X_{t-1} + \sqrt{1-\bar\alpha_t}\,\epsilon$$

- $\alpha_t \in (0,1)$: 노이즈 스케줄 (하이퍼파라미터)
- 가우시안끼리 연산 → **analytic하게 $X_t$로 점프** 가능 (전체 chain backprop 불필요)

### ELBO 3항 (직관)

1. **Reconstruction**: $X_0$ 복원
2. **Prior matching**: $X_T$가 가우시안 — **설계상 0**, 최적화에서 **사라짐**
3. **Denoising matching**: $X_t$에서 $X_{t-1}$로 가는 분포 맞추기 — **핵심**

학습 트릭: teacher forcing과 유사하게, 임의 타임스텝 $t$를 샘플해 **한 스텝 디노이징만** 병렬 학습.

### Noise prediction 형태

정답 평균 꼴을 맞추기 위해 decoder를 **residual 형태**로 파라미터화하면, 최종적으로:

> **$X_t$와 $t$가 주어졌을 때, 섞인 노이즈 $\epsilon$을 예측**하면 학습 완료

이것이 DDPM류 **noise prediction**의 핵심입니다.

## 3-6. Flow Matching (맛보기)

Diffusion은 prior가 **가우시안**이어야 analytic jump가 편합니다. 더 일반적인 **임의 분포 → 임의 분포** 변환은 **Flow Matching**이 직관적입니다.

- $t \in [0,1]$: $Z_0$(노이즈) → $Z_1$(데이터)
- 중간 경로(보간)를 **손으로 정의** → 미분하면 **velocity field(속도장)**
- 신경망이 $v_\theta(Z_t, t)$를 학습 → 실제 속도와 **regression**
- 생성: ODE solver로 적분

Diffusion·score matching·flow matching은 **한 패밀리** — 분포를 다른 분포로 옮기는 **연속 과정**의 한 스텝을 배우는 것.

> ⭐ **시험 포인트**  
> - Flow matching: **velocity field** 회귀, prior 제약이 diffusion보다 느슨  
> - Diffusion denoising ≈ **score(∇log P)** 따라 gradient ascent — 내일 수업에서 심화

조건부 생성(텍스트→이미지): $P(X|text)$ 추정 — 항에 text 넣거나 **classifier-free guidance** 등 (다음 수업).
