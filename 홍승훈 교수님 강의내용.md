# 1장. 6월 5일 오전 — Segmentation의 기초와 FCN/U-Net

## 1.1 CNN 기본 연산자 복습

### 1.1.1 컨볼루션 연산자
교수님은 컨볼루션을 *"이미지를 이미지로 변환하는 가장 기본적인 연산자"* 라고 정의하셨다. 3×3 컨볼루션은 3×3 영역 안에서 각 픽셀들의 weighted sum을 해서 값 하나를 내주는 것. 슬라이딩 윈도우로 모든 위치에서 반복하면 입력보다 작은 출력 이미지가 나온다.

### 1.1.2 패딩과 스트라이드
- 패딩 없으면 입출력 사이즈가 필연적으로 다르다. *"바운더리 픽셀들에서는 컨볼루션을 할 수 없으니까."*
- 스트라이드: 컨볼루션을 건너뛰어가며 수행.

### 1.1.3 입출력 사이즈 계산식

> 🔥**시험포인트**🔥
>
> 입력 n×n×c, 커널 k×k, 스트라이드 s, 패딩 p, 필터 개수 d일 때:
> - 커널이 커지면 출력 사이즈가 작아진다.
> - 패딩을 양쪽에 잘 넣으면 보정 가능.
> - 스트라이드의 반비례로 출력 사이즈가 작아진다.
> - 출력 채널 = 필터 개수.
>
> 예시: 입력 52×52×128, 커널 3, stride 1, padding 1 → 공간 그대로, 채널만 바꾸는 연산. stride 2 → 공간 1/2.

### 1.1.4 Receptive Field (리셉티브 필드)

> 🔥🔥**시험포인트 — 교수님이 가장 강조한 정의**🔥🔥
>
> *"리셉티브 필드는 항상 **입력 데이터**에 대해서 정의됩니다. 제가 리셉티브 필드를 두 번째 레이어에서 계산하더라도, 리셉티브 필드는 여기 있는 한 액티베이션이 **입력 데이터에서 얼마만큼의 영역을 관측하고 있냐**로 정의가 되거든요. **항상 입력으로 정의되거든요**."*

직관: *"어떤 한 액티베이션의 값이 입력 픽셀들에 대해서 모든 입력 픽셀에 다 연관된 건 아니거든요. 어떤 픽셀 값을 바꿔도 그게 액티베이션한테 영향을 안 줄 수 있다 — 그러면 그 픽셀이 리셉티브 필드 바깥에 있다는 얘기."*

- 커널 3×3이면 첫 레이어 RF는 3×3. 두 번째 레이어에서는 누적해 커진다.
- 재귀적 계산. 커널 사이즈 > 1이면 RF가 항상 커진다.
- 스트라이드가 중간에 붙으면 RF는 비례해서 커진다.

### 1.1.5 풀링 — 파라미터 없이 RF 키우기
- 컨볼루션 스트라이드를 키워 RF를 늘리면 파라미터가 늘어남.
- **풀링은 학습 파라미터 없는 컨볼루션**. Max/Mean pooling은 단순 통계.
- 그래서 CNN 안에 풀링이 중간중간 들어간다.

### 1.1.6 왜 RF가 중요한가

> 🔥**시험포인트**🔥
>
> *"이게 차인지 오토바이인지 보려면, **바퀴만 봐서는 알 수 없잖아요**. 전체를 다 봐야 알 수 있잖아요."*
>
> Classification CNN 마지막에 FC layer를 두는 것 = **RF를 이미지 전체로 만들기 위함**.
>
> 교수님: *"오늘은 이 얘기를 많이 할 겁니다. 리셉티브 필드 얘기를."*

## 1.2 Semantic Segmentation 문제 소개

### 1.2.1 Classification vs Segmentation
- Classification: 이미지 단위 클래스.
- Segmentation: **픽셀 단위 클래스 레이블**.

### 1.2.2 학습 데이터의 어려움 — 반도체 일화
> *"반도체 공정 같은 거를 하는데, 거기서 웨이퍼 같은 거 사진으로 찍어가지고 여기 분량이 있냐 없냐 검출하려고 하면 세그멘테이션 마스크를 사람이 손으로 그려줘야 되거든요. 근데 어떤 반도체 웨이퍼 보고 여기 분량이 있는지 없는지 판단하는 거는 전문가들만 할 수 있잖아요. **그래서 엄청 비싼 반도체 전문가들이 마스크 칠하고 있거든요**."*

### 1.2.3 Segmentation이 전통적으로 어려웠던 두 이유

> 🔥**시험포인트**🔥
>
> 1. **모든 픽셀의 특징을 다 학습해야 함**: classification은 얼굴만 잘 인식해도 OK. 그러나 segmentation은 *"제 몸 같은 경우에는 별로 특징이 없잖아요. 그냥 단색의 옷을 입고 있다면. 그런데 그 모든 몸에 해당하는 픽셀들에도 클래스 레이블을 줘야 되기 때문에 훨씬 더 찾아야 되는 물체들의 특징이 많은 편이다."*
> 2. **해 공간이 훨씬 큼**: 20 클래스 classification은 찍어도 5%, segmentation은 32×32 이미지여도 가능한 해의 가짓수가 어마어마함. *"신경망 처음 배울 때는 예측이 거의 찍는 거거든요. 거기서부터 학습 신호를 받아서 시작해야 되는데 쉽지가 않습니다."*

### 1.2.4 연구 역사의 변곡점

> 🔥**시험포인트**🔥
> - **2012년 전**: MSRC라는 단순 데이터셋 하나만 사용. *"엄청 단순한 데이터셋. 푸른 초원 위에 양 한 마리 딱 놓여져 있는 이런 데서 양 세그멘테이션 하는 거."*
> - **2012년**: AlexNet 등장.
> - **2014년**: 본격적인 segmentation 모델들이 폭발적으로 나옴 → 오늘 다룰 FCN이 그 시발점.
> - 이후 발전: *"그 논문들이 다루고자 하는 내용이 궁극적으로는 **리셉티브 필드를 어떻게 다룰 건지**에 대한 내용으로 귀결한다."*

## 1.3 초기 시도 — Segmentation Proposal 기반

교수님 코멘트: *"이거 처음에 나온 모델들일수록 복잡한 거 아시나요? 뭔가 덕지덕지 붙여가지고 어떻게든지 되게 만들려고 해서. 이해하실 필요 전혀 없고요, 직관적으로만 설명 드릴 테니까 이해 안 되면 그냥 넘어가셔도 상관없습니다."*

### 1.3.1 작동 원리
- 학습된 CNN으로 피처 추출 (이전엔 SIFT/HOG 같은 hand-designed feature).
- **Segmentation proposal** 뽑기: 색이 비슷한 픽셀들 뭉치기 → 마스크 후보 약 2000개.
- 가정: 그중 하나가 진짜 물체에 해당.
- 마스크 안 피처를 풀링 → classifier로 분류.

### 1.3.2 문제점
- (1) Region proposal 알고리즘이 중요. 정확한 마스크가 후보에 없으면 망함.
- (2) **End-to-end training이 안됨** — feature extractor는 classification용으로 학습, segmentation에 최적화 X.
- → **end-to-end framework**가 필요. *"이걸 누가 제일 먼저 하냐, 요게 요 당시에 화두였고요."*

## 1.4 Fully Convolutional Network (FCN, 2014)

### 1.4.1 직관적 설계 (naive)
- 원래 224×224 받는 CNN을, 더 큰 이미지에서 224×224 패치 슬라이딩.
- 각 위치에서 classification → score map.
- 작은 score map을 업샘플 → 픽셀 단위 cross-entropy loss.

### 1.4.2 효율성 문제와 컨볼루션 해석
*"중복해서 겹치는 영역이 많은데 재사용을 안 하잖아요. 이걸 기가 막히게 하는 연산자가 있거든요. 그게 컨볼루션입니다."*

### 1.4.3 FC layer를 컨볼루션으로 "해석"

> 🔥**시험포인트 — 핵심 발상**🔥
>
> **FC layer를 컨볼루션으로 새로 정의하는 게 아니라, FC layer를 컨볼루션으로 "해석"만 한다.**
>
> 예: 마지막 conv 출력이 7×7×512, 첫 FC가 4096-dim → 가중치 매트릭스 7×7×512 → 4096. 이걸 **7×7 커널, 512 채널, 4096 필터의 컨볼루션** 으로 해석. 입력이 더 크면(22×22×512), 슬라이딩 → 16×16×4096 출력. 두 번째 FC는 1×1 컨볼루션으로.
>
> *"전체가 컨볼루션 연산자가 되니까 전체가 하나의 그냥 비선형 필터처럼 동작하는 거거든요."*

### 1.4.4 FCN 전체 구조
- 입력: 더 크게(500×500).
- 출력: 채널 = 클래스 수(21), 공간 사이즈는 다운샘플 때문에 작음(16×16 등).
- 업샘플링: 인접 픽셀 값으로 보간.
- 픽셀 단위 cross-entropy → backprop. **End-to-end 학습**.

### 1.4.5 FCN의 두 가지 문제

> 🔥**시험포인트**🔥
>
> 1. **Output score map resolution이 너무 낮음** — 다운샘플 때문. 억지로 키워도 *"없던 디테일이 살아나지 않거든요."* 마스크가 뭉뚝.
> 2. **Receptive Field가 작음** — 원래 224×224 받는 CNN이므로 큰 이미지를 넣어도 RF는 그대로. *"오토바이 바퀴만 보고 차다, 버스다 분류하게 되는 거. 전역적인 컨텍스트를 보지 못하고."*

### 1.4.6 실패 예시 — 말 사진
> *"말의 얼굴이 있는 데서는 정확하게 말이라고 예측하는데, 다른 데서는 막 틀려요. 얼굴 가리고 몸만 보여드리면 말인지, 개인지, 고양이인지 모르잖아요."*

### 1.4.7 해상도-RF Trade-off

> 🔥**시험포인트**🔥
> *"아웃풋 레졸루션을 높이고 싶으면 이미지를 많이 키워야 돼요. 근데 입력 이미지를 많이 키우게 되면 더 작은 영역을 보고 클래시피케이션을 하는 게 돼버리기 때문에 클래시피케이션 결과는 점점 부정확해집니다. 이 두 개가 트레이드 오프인 관계처럼 보이거든요. **근데 이 두 개를 둘 다 잘 하려는 게 지금 하려는 겁니다**."*

## 1.5 일반화 관점 — 모든 image-to-image 문제

> 🔥**시험포인트**🔥
> *"이 문제를 일반화하면 이미지를 입력으로 받아서 이미지를 출력으로 내는 문제가 되거든요. 이미지를 이미지로 변환하는 모든 문제에 사실은 쓸 수 있는 신경망입니다."*
>
> *"여기서 유넷, 스킵 커넥션 이런 거 배울 건데, 이것들이 예를 들면 지금 랭귀지 모델에도 들어가 있고요. **나노바나나**에도 들어가 있고요. 모든 이미지를 이미지로 변환하거나 시퀀스를 시퀀스로 변환하거나 ... 언어모델에도 들어가고 생성모델에도 들어가고."*

## 1.7 Encoder-Decoder Network

### 1.7.1 인코더/디코더 개념
- **인코더**: 정보를 추상화 (mp4 인코더와 같은 개념).
- **디코더**: 추상화된 정보를 원래 관측으로 복원.
- 이상적으로는 역함수 관계. 단, 추상화 과정에서 정보 손실 → 디코더는 역함수의 **근사**.

### 1.7.2 Pooling의 역함수
- **Max pooling 역함수**: 선택된 픽셀이 풀링 윈도우 사이즈로 다시 커지는 연산.
- **Average pooling 역함수**: 평균에서 원래 샘플 값 복원.

### 1.7.3 Transposed Convolution

> 🔥**시험포인트 — 디코더의 핵심 연산자**🔥
>
> $x \in \mathbb{R}^d$, $y = Wx$ ($W$는 $d' \times d$). 역연산자 $W'$를 어떻게? 이상적으로는 $W^{-1}$. 그런데 **$W^T$도 역할을 할 수 있음** ($d \times d'$ 사이즈).
>
> **인코더의 핵심 = 컨볼루션 / 디코더의 핵심 = transposed convolution.**
>
> 컨볼루션 stride → 사이즈 작아짐. **Transposed conv stride → 사이즈 커짐**.
>
> 시각화: 컨볼루션은 커널을 이미지에 곱해 하나의 값. **Transposed conv는 거꾸로, 입력 하나에 필터를 곱해서 갖다 놓는 것**.

### 1.7.4 나쁜 답 — 중간 피처 가져다 쓰기 (FCN의 보완)
- 중간 레이어 피처는 다운샘플 덜 됨 → 공간 정보 보존.
- 단점: 🔥**시험포인트**🔥
  1. RF가 작다.
  2. 모델 depth가 얕다 → 피처가 풍부하지 않다.

> 🔥**시험포인트 — Deep을 쓰는 이유 (본질)**🔥
> 학생: 왜 shallow가 아니라 deep?
> *"신경망은 레이어 두 개만 있어도 이론상 완벽하거든요. 비선형성 때문에 쓰는 거잖아요. ReLU 레이어면 공간을 한번 접는 거잖아요. 많이 접히면 접힐수록 피처가 더 복잡하게 변화할 수 있어요."*
>
> *"클래시피케이션 마지막 레이어는 항상 선형 레이어거든요. 공간을 직선으로 나누는 거예요. 복잡하게 분포된 데이터를 비선형적으로 틀어가지고 결국에는 선형적으로 분리 가능하도록 만들어 놓고 싶은 거거든요."*

**중간 피처 vs 마지막 피처 trade-off**:
- 중간: 공간정보 ↑, 모양 세밀 ↑, 분류 정확도 ↓
- 마지막: 분류 정확도 ↑, 공간정보 ↓
- FCN 방식: 그냥 sum. *"완벽한 해결책은 아니잖아요."*

### 1.7.5 U-Net — Encoder-Decoder의 결정판

> 🔥**시험포인트 — 핵심 구조**🔥
>
> *"공간 정보는 최대한 보존하되 depth를 희생하고 싶지 않은 거예요."*
>
> - **인코더**: 컨볼루션 + 다운샘플링 반복 → 레졸루션 ↓, 채널 ↑
> - **디코더**: transposed conv stride로 레졸루션 키움
> - **Skip Connection**: 인코더 대응 레이어 피처를 디코더의 같은 레졸루션 레이어로 직접 전달
>   - 채널 방향 concatenation 후 컨볼루션
>   - *"디스크리미네티브하지만 정보가 손실된 피처(밑에서 올라오는)와 손실되기 전 공간 정보를 가진 피처(인코더에서 바로 오는)를 같이 합쳐서 어떻게 섞을지는 컨볼루션을 통해 배워라."*

### 1.7.6 Skip Connection의 두 번째 역할 — 학습 안정성

> 🔥**시험포인트**🔥
> Skip이 없으면 백프롭이 깊은 경로를 거치면서 그래디언트가 약해질 수 있음. Skip은 *"그래디언트가 흐를 수 있는 **고속도로**를 깔아주는 느낌."* **ResNet의 residual connection도 같은 원리**.

### 1.7.7 U-Net의 위상

> 🔥**시험포인트**🔥
> *"여기저기서 많이 쓰는 모델입니다. 지금 시대에도 relevant하고 ... transformer에서도 U-Net 구조가 있습니다. **VAE에도 들어가고, 디퓨전 모델에도 들어가고, 언어 모델에도 약간 들어가고요. 트랜스포머가 인코더랑 디코더로 구성되어 있다고?** 마찬가지거든요."*

### 1.7.8 또 다른 인코더-디코더 (SegNet 류)
- 224×224 → 1×1까지 줄였다가 다시 224×224로 복원하는 극단적 디자인.
- **Unpooling을 통한 간접 Skip Connection**: max pooling 시 어떤 위치가 선택됐는지 저장 → 디코더에서 그 위치에 갖다 놓기.
- 단점: 풀링 안 된 위치는 빈 값 → transposed conv 한 번 더 돌려 채움.
- 장점: 모양 잘 남고 **RF가 이미지 전체**.

### 1.7.9 후처리에 대한 코멘트

> 🔥**시험포인트**🔥
> *"옛날엔 픽셀 단위 후처리(비슷한 픽셀값은 같은 레이블)를 많이 했는데, 인코더-디코더가 출력이 세밀해져서 후처리가 빠짐. 모델 학습 바깥쪽에서 돌아가는 거니까 end-to-end learning이 안 되니까 게를 최대한 제거하려는 목적에서도 후처리는 빠지는 게 좋고요. 요즘에는 후처리를 거의 안 합니다."*

## 1.8 Atrous (Dilated) Convolution

### 1.8.1 RF를 키우는 다른 방법들의 한계
- 다운샘플 많이 → 공간 정보 ↓.
- 커널 키우기 → **파라미터 증가 → 과적합 위험**.

### 1.8.2 Atrous Convolution의 아이디어

> 🔥**시험포인트**🔥
> *"원래 컨볼루션 연산자가 이렇게 생겼잖아요. 그럼 여기 컨볼루션 연산자의 공간을 비워주는 겁니다. 이게 끝입니다."*
>
> - 3×3 커널, 파라미터 9개 그대로지만 **effective 커널 5×5**.
> - **커널 키우되 파라미터는 안 늘리는 방법**.
> - **Atrous rate**: 파라미터들 사이 간격.
> - 주의: 너무 크면 *"패턴이 안 보이잖아요"* (로컬리티 손실).

## 1.9 Spatial Pyramid Pooling

(디텍션에서 이미 다뤘다고 빠르게)
- 풀링 윈도우 사이즈를 여러 개로 두고 동시에 풀링.
- 글로벌 average pooling → 벡터 하나, 더 세밀히 쪼개면 구역별 벡터.
- 극단적으론 RF = 이미지 전체.
- **Atrous convolution으로도 구현 가능** (풀링이 학습 가능한 컨볼루션의 특수형이므로).

## 1.10 Segmentation 종합 정리

> 🔥**시험포인트 — 교수님 직접 정리**🔥
> *"세그멘테이션을 위한 신경망의 가장 기본적인 특징 하나는 **모든 연산이 컨볼루션으로 구성되어 있다**. 그렇기 때문에 속도가 엄청나게 빠릅니다. Fully convolutional network 자체는 리셉티브 필드가 작고 score map의 사이즈도 작은 문제가 있어서 그거를 해결하기 위한 모델들을 몇 개 봤고, **가장 널리 사용되는 모델은 인코더-디코더 아키텍처입니다**."*

## 1.11 다른 비전 문제로의 일반화

### 1.11.1 Depth Estimation
- 픽셀마다 카메라 렌즈와의 거리.
- 로봇/3D 응용 필수.
- 이미지 → 이미지 변환 (출력 = depth label).

### 1.11.2 Detection — YOLO도 사실 FCN

> *"YOLO 잘 보시면 FCN이랑 똑같이 생기지 않았나요? 가만히 보면? 한번 복습을 해보시면, 어? FCN인데? 이렇게 생각하실 거거든요."*

### 1.11.3 Pose Estimation
- 포즈 = 관절(joint)의 위치 집합. 인접 관절 이으면 **skeleton**.
- 응용 풍부: 메타 오큘러스 손 인터페이스, **위성의 회전 각도**(우주공학), 모션캡쳐(영화/게임), **원숭이 행동 분석**(생명과학).
- 출력 = 관절 위치마다 점이 찍힌 이미지. **각 관절을 클래스로 보면 segmentation과 동일**.

> 🔥**시험포인트**🔥
> *"제 오른 다리 무릎 관절을 찾으라고 하면 어렵죠. 다른 관절을 통해서 추정 가능. 사람 좌우대칭이라 왼쪽 팔꿈치는 오른쪽 팔꿈치랑 똑같이 생겼거든요. 그 팔꿈치만 보고 왼쪽인지 오른쪽인지 구분하려고 하면 안 돼요. 그래서 **전역적인 맥락을 봐야 되거든요.**"*
>
> 따라서 같은 모델(인코더-디코더) 그대로 사용.

### 1.11.4 Depth의 본질적 어려움 (질의응답)
> *"이미지 한 장에서 깊이를 추정하는 거는 풀 수 없는 문제를 푸는 거랑 같거든요. 깊이는 한 장만 가지고는 정의가 안 되고 두 장이 있어야 정의됩니다. ray가 만나야 됩니다."* 멀티뷰여도 *"이미지 여러 장을 이미지 여러 장으로 변환하는 문제. 원리는 똑같습니다."*

## 1.12 마무리 — 기계학습의 방향성

> 🔥🔥**시험포인트 — 교수님 마무리 메시지**🔥🔥
> *"기계학습이 엄청 빨리 바뀌고 있잖아요. 그런데 방향은 있거든요. ... depth 추정, 세그멘테이션 이런 것들이 다 별도의 연구 주제였는데, 얘네가 지금 다 하나로 합쳐졌잖아요. **U-Net이 다 쓰일 수 있거든요**."*
>
> *"비전 안에서만 일어나는 게 아니고 도메인이 바뀌어도 생기거든요. 비전 모델과 언어 모델은 다 하나의 모델로 표현할 수 있거든요. **나노바나나** 같은 이미지 생성 모델은 GPT와 같은 텍스트 생성 모델이랑 다르게 생겼어요. 근데 다르게 생길 이유가 별로 없거든요. 다 합쳐질 거에요. 결국에 사람 뇌에 하나 있는 것처럼 모델도 그냥 하나 있는 형태로 갈 거거든요."*

학생들에게 머릿속 실험 제안: *"한 가지 비전 문제를 생각해보세요. 걔를 이미지 투 이미지로 표현할 수 있는지 한번 생각해보시고요. 그걸 위한 백본을 오늘 배운 모델이랑 껴맞춰볼 수 있는지 한번 생각해보세요. 엄청 높은 확률로 아마 그럴 겁니다."*

# 2장. 6월 8일 오전 — Vision Transformer와 Segmentation Decoder

## 2.1 도입 — 비전 관점에서 Transformer 재방문

지난 시간 segmentation을 이어 트랜스포머 기반 세그멘테이션으로 진행. 학생들이 트랜스포머를 RNN/언어 맥락에서 배웠다고 해서, *"그러면 비전 관점에서 한 번 재방문해보는 게 나쁘지 않으니..."* 라며 시작.

## 2.2 Receptive Field 복습

> 🔥**시험포인트 — 1장과 직결**🔥
>
> - **정의**: 신경망의 특정 액티베이션 하나가 이미지의 어느 정도 영역을 관측하는지 정량화한 지표.
> - **이상**: RF가 이미지 전체로 정의되는 것이 가장 좋다.
> - 어깨 비유: *"제가 어깨만 딱 봤을 때는 어깨는 뭔가 사람의 어떤 특징적인 피처가 별로 없잖아요."* → 어깨 픽셀의 RF가 얼굴까지 커버해야 한다.

### 2.2.1 RF를 키우는 기존 방법들
1. **Fully Connected Layer** — RF 전체. 단점: 파라미터 ↑↑.
2. **컨볼루션 커널 크기 키우기** — 극단적으로 키우면 FC와 동일.
3. **Stride/Pooling** — 피처맵 사이즈 ↓ → 공간 정보 손실.

지난 강의 핵심: *"리셉트 필드를 키우려고 막 시도했잖아요. 저 피쳐맵 사이즈는 안 줄이면서."*

## 2.3 시퀀스-투-시퀀스 함수의 일반적 골

### 2.3.1 Setup
- 입력 시퀀스: n개 원소, 각 d-dim 벡터.
- 출력 시퀀스: 길이 같음, 차원 d'.
- 이미지로 보면 원소 = 픽셀, n = 픽셀 수, d = 3.

### 2.3.2 Case 1: Element-wise 매핑 (가장 단순)
- $f(x_i) = W x_i$ ($W$는 $d' \times d$, 원소 간 파라미터 공유).
- 장점: 파라미터 시퀀스 길이 무관, $d \times d'$만. *"이것보다 더 파라미터 효율적일 수는 없거든요."*
- 단점: **RF = 1**. *"픽셀 값 하나 보고 클래스 레이블을 찾는건데 말이 안되잖아요."*

### 2.3.3 Case 2: 모든 입력 원소의 Weighted Sum (가장 복잡)
- $y_i = \sum_j w_{ij} \cdot g(x_j)$
- **RF = 데이터 전체**.
- weight 설계에 따라 여러 신경망 레이어 다 표현 가능.

## 2.4 Local Context Encoding — Average Pooling과 Convolution 통합 표현

### 2.4.1 Average Pooling 도출
$w_{ij}$를 $|j-i| \leq \lfloor k/2 \rfloor$ 일 때 $1/k$, 아니면 0으로 정의 → **average pooling**.

### 2.4.2 Convolution 도출
$w_{ij}$를 위치 $j-i$에 따라 다른 weight $w_{j-i+\lfloor k/2 \rfloor}$로 정의 → 슬라이딩 윈도우로 같은 weight들이 적용 → **convolution**.

> 🔥**시험포인트 — 일반 골에서 도출**🔥
> 같은 weighted sum 골에서 $w$를 어떻게 설정하느냐로 average pooling / convolution / FC / self-attention이 모두 표현됨.
>
> **Local context encoding**: 출력 원소 근처 입력만 고려. 잘 설계하면 **translation invariance** 만족. 단점: RF가 데이터 전체가 아님.

## 2.5 Global Context Encoding과 Fully Connected Layer의 약점

> 🔥**시험포인트 — FC layer의 두 약점**🔥
> 1. **파라미터 $n^2$** (시퀀스 길이의 제곱). 시퀀스 길어지면 비효율.
> 2. **$w_{ij}$가 입력 $x$와 무관하게 고정**.

### 2.5.1 "교수님 3배 확대" 비유
> *"이미지 한 장은 교수님이 그냥 서 있는 것, 다른 한 장은 교수님이 3배 확대돼서 있는 것. 어깨 픽셀(i)의 레이블을 추정할 때, 얼굴 픽셀(j)이 도움 되므로 $w_{ij}$가 크게 설정될 수 있는데... 이미지가 바뀌어 교수님이 확대되면 어깨/얼굴 위치 자체가 바뀌는데, $w_{ij}$는 고정되어 있어 표현력이 떨어진다."*
>
> → *"플리코넥틸 레이어가 약간 안 좋은 이유를 좀 아시겠나요? **파라미터는 겁나 많이 필요한데, 표현할 수 있는 함수는 되게 제한적**."*

## 2.6 해결책 — Hypernetwork와 Self-Attention 도출

### 2.6.1 아이디어
$w_{ij}$를 다른 모델의 출력 $g(x_i, x_j)$로 만들자 → **입력 데이터에 따라 동적으로 weight 변화**.

### 2.6.2 Hypernetwork

> 🔥**시험포인트 — 핵심 프레임**🔥
> *"신경망의 가중치를 출력으로 내는 신경망을 뭐라고 부르는지 배우셨나요? **하이퍼네트워크**."*
>
> - 일반 신경망: 입력에 따라 출력만 바뀜, 가중치 고정.
> - Hypernetwork: 입력에 따라 **가중치도 동적**.
>
> *"트랜스포머를 배울 때 **하이퍼네트워크적인 관점**을 갖는 게 중요합니다. 왜 이게 동작하는지 이해를 하려면."*

### 2.6.3 Self-Attention 도출
- $g$를 dot-product kernel로: $x_i$에 $W_Q$ 곱해 query, $x_j$에 $W_K$ 곱해 key, inner product.
- 분모에 normalization (sum to 1), exponential 씌우면 → **softmax**.
- 합이 1, 모든 원소 양수 → **Attention의 정의 만족**.
- → 이것이 바로 **Self-Attention**.

### 2.6.4 Self-Attention의 파라미터

> 🔥**시험포인트**🔥
> - $W_K, W_Q, W_V$ 3개 매트릭스, 각 $D \times D'$.
> - **총 파라미터: $3 \times D \times D'$, 시퀀스 길이와 무관**.
> - 그런데도 **RF = 데이터 전체**.

### 2.6.5 한 줄 요약

> 🔥🔥**시험포인트 — Self-Attention 3관점**🔥🔥
>
> Self-Attention은:
> 1. **전역 맥락을 참고하는 레이어**
> 2. **Hypernetwork**
> 3. **FC layer의 효율적 대체**

## 2.7 CNN에 Self-Attention 끼워넣기

- $W_Q, W_K, W_V$를 **1×1 convolution**으로 구현.
- *"파라미터를 엄청 많이 늘린 것도 아니고 그냥 1×1 컨볼루션 3개. 근데 그렇게 셀프어텐션 레이어로 구현하면 그 아웃풋의 RF는 데이터 전체."*

### 효과 — "싱크대" 예시
> FCN에 self-attention 하나 넣으면 AQC 확 향상. 변기 옆 세면대 예측: *"변기 옆에 붙어있는게 보통 세면대니까."* FCN은 RF 작아서 못 보지만 self-attention 들어가면 잘 맞춤.

## 2.8 Transformer의 비전적 관점

### 2.8.1 RNN vs CNN 관점 비교
- **RNN 관점**: RF는 원래도 전체였음, 트랜스포머로 바꿔도 변화 없음.
- **CNN 관점**: RF를 데이터 전체로 확장 + hypernetwork → 강점이 명확.

### 2.8.2 In-Context Learning과 Hypernetwork
> *"In-Context Learning은 프롬프트만 바꿔주면 신경망이 막 다른 짓을 하잖아요. 매지컬하지 않나요?"*
> ICL은 내부적으로 파라미터 업데이트하는 효과 → fine-tuning 없이 fine-tuning한 것처럼 → **hypernetwork 관점에서 설명 가능**.

### 2.8.3 카이밍 헤(Kaiming He) 일화
> 🔥**시험포인트**🔥
> *"엄밀히 따지면 이 코스로 따라왔거든요. 제가 아까 얘를 논 로컬 네트워크라고 불렀잖아요. **카이밍 헤** 혹시 아시나요? 레지듀얼넷 만든 사람이요. 이 개념도 만들었거든요. **논 로컬 컨텍스처 인코딩**. 이 개념으로 발전돼서 트랜스포머가 나온 거예요."*
>
> → 트랜스포머는 시퀀스 모델이 아니라 **non-local context encoding** 맥락에서 발전.

## 2.9 Vision Transformer (ViT)

### 2.9.1 구조
- 입력 이미지를 패치들로 쪼개기.
- 각 패치 → linear projection → 토큰.
- **Self-attention + FC layer 반복**. 컨볼루션 X.
  - FC = 원소별 인코딩
  - Self-attention = 원소 간 interaction
- **Positional encoding** 추가.

### 2.9.2 Global Token (Classification Token)
- Image classification 같은 단일 prediction이 필요할 때.
- 이미지에서 안 오는, **학습 가능한 벡터**를 0번 토큰으로.
- Self-attention으로 모든 패치 정보를 가중평균 흡수.
- 마지막에 classification head.

### 2.9.3 데이터 양과 Convolution의 운명

> 🔥**시험포인트**🔥
> *"처음에는 사람들이 섞어서 썼어요. 근데 알고 봤더니 **데이터가 엄청 많아지면 점점 컨볼루션을 안 써도 성능이 똑같은 거예요**. → CNN은 데이터 적을 때 효율적인 아키텍처. 데이터 많으면 굳이 필요 없음."*

### 2.9.4 Transformer의 Scaling Law
- 데이터/파라미터 커질수록 성능 좋아지는 scaling law 만족.
- CNN, FC는 scaling 안 되고 트랜스포머만 됨 → 교수님 추정: **"hypernetwork 이라 그런 것 같다"**.

## 2.10 비전 도메인의 "통일" 이야기

> 🔥**시험포인트 — 큰 그림**🔥
>
> ### 비전과 NLP의 핑퐁
> - 2012 AlexNet (비전 부활) → 비전이 CNN/RNN을 NLP에 넘김 → NLP가 Transformer 개발 → Transformer가 다시 비전으로 → *"비전이 트랜스포머에 먹혔다."*
>
> ### Transformer는 데이터 비특화
> *"트랜스포머 자체는 시퀀스 모델이 아닙니다. 사실 데이터 특성에 대한 가정이 없음. 시퀀스로 다루려면 1D positional encoding, 이미지면 2D, 비디오면 3D. **데이터 종류는 토크나이제이션 방식으로 결정**."*
>
> → *"기계학습 분야에서 어떻게 보면 처음으로 데이터의 특성과 상관없이 잘 동작하는 신경망이 생겼다."*
>
> ### 물리학 "통일 이론" 비유
> *"물리학에서 통일 이론 뭐 이런 거 있잖아요. 그런 것처럼 이제 한 모델과 한 학습 파이프라인이 이제 나오게 될 거라고 저희 연구하는 사람들은 기대하고 있습니다. 이미지든, 언어든, 비디오든, 음성이든, 그래프든, 화학식이든, 물리방정식이든 다 한 모델, 한 학습 파이프라인으로 통합하려고 하고요."*
>
> 농담: *"그러면 이제 저희 코스가 훨씬 짧아지겠죠."*

## 2.11 (오전 2부) Transformer Decoder for Segmentation

ViT는 사실상 **encoder**. 이제 디코더 설계 = **패치 단위 표현을 픽셀 단위 클래스 레이블로 변환**.

> 🔥🔥**시험포인트 — 반복 강조**🔥🔥
> *"세그멘테이션이라는 게 모든 컴퓨터 비전 문제들의 **90%를 대변하는 문제**이거든요."*

### 2.11.1 Setup (디코더 문제 정의)
- 입력 H×W, 패치 P×P, 패치 수 $N = (H/P)(W/P)$.
- 트랜스포머 출력: N개 패치, 각 D-dim.
- 원하는 출력: H×W 픽셀별 K개 클래스 스코어.
- → N → H×W로 $P^2$배 업샘플 + D → K 차원 변환.

세 가지 디코더 아이디어: **Convolution Decoder / Transformer Decoder / 하이브리드**.

## 2.12 Convolution Decoder

### 2.12.1 가장 단순한 형태
1. N개 토큰에 MLP (D × K) → N개 패치마다 K-dim 클래스 스코어.
2. 업샘플로 픽셀 단위로.

### 2.12.2 패치마다 독립 classification이 왜 괜찮은가?
> *"제가 계속 말했는데 어떤 클래스 레이블을 추정할 때는 맥락을 참고해야 된다고 했잖아요. 근데 패치마다 독립적으로 클래시피케이션을 하고 있는 거거든요. 이거 왜 괜찮죠?"*
>
> → 답: **이미 self-attention 단계에서 RF가 전체로 확장되어 맥락이 인코딩됨**.

### 2.12.3 패치 크기가 크면 — 인코더-디코더 + Skip Connection
- 1장의 U-Net처럼 점진적 디코더.
- 트랜스포머 블록들 사이에서 중간 피처를 뽑아 디코더로 전달.

### 2.12.4 "Reassemble" 연산
- 트랜스포머 출력(1D 시퀀스)을 2D 격자로 재배치.
- Reassemble_S: 출력이 원본 대비 1/S 사이즈가 되도록.
- 다운/업샘플 통합 처리.

### 2.12.5 Attention 시각화
- 강아지 위 패치 잡으면 **다른 모든 강아지 영역에서 attention 높음** → 같은 의미 영역끼리 정보 교환.

## 2.13 Q&A 모음 — 5가지 학생 질문

### Q1: 패치 사이즈를 줄이면 안 되나?
> 🔥**시험포인트 — 단골**🔥
> *"맞다. 패치를 줄이면 된다. 다만 **토큰 개수 N이 늘어나면서 메모리 폭증**. Self-attention 계산 복잡도는 N². 예: N이 10에서 100으로 가면 메모리는 100→10,000으로 100배. 패치 사이즈 16→1이면 N이 16² 배 증가. **결충 지점이 존재합니다. 메모리가 허용하는 가장 작은 패치를 쓰겠다고 됩니다**."*
>
> 실험적으로 **패치 작을수록 성능이 좋아지는 경향**.

### Q2: 패치 경계의 불연속?
> 안 겹치게 자르면 모자이크처럼 불연속. 해결:
> (1) **패치를 겹치게(overlap)** 자르기 — CNN sliding window처럼.
> (2) **Convolution decoder** 사용 — conv가 경계에서 스무딩.

### Q3: 패치 순서를 어떻게 정의?
> 🔥**시험포인트**🔥
> 동일한 순서로 일관되게 지정하면 OK. raster scan order, 거꾸로, 가운데부터 시계방향... 다 같음.
>
> **15퍼즐 비유**: 어릴 적 15퍼즐 — 같은 이미지인데 순서가 섞이는 것. *"순서를 안 정해주는 것 = 이 모든 섞인 상태를 같은 이미지로 보라고 하는 것. 모델 표현력이 무한하면 가능하지만 현실적으론 어려움. **하나 정해서 외우게 하는 게 더 간단**."*
>
> **순서를 정하면 안 되는 케이스**:
> - **LiDAR / Point Cloud** (자율주행 라이다)
> - **Graph** (노드 1번을 누구로 정하느냐에 따라 인접행렬 달라짐)
>
> **Relative Positional Encoding**: 절대 위치 대신 상대 위치. *"줄여서 **ROF**(RoPE). 최근 트랜스포머들은 대체로 ROF 기반."*

### Q4: 작은 물체 마스크가 부정확
> 트랜스포머 자체는 사이즈 못 잡는 문제 없음. 원인: **패치가 너무 큼**. 패치 안 픽셀 구분은 트랜스포머 바깥(업샘플링)에서 일어남. 해결: 패치 줄이기, 또는 convolution decoder.

### Q5: Open-vocabulary (미리 못 본 클래스)
> 🔥**시험포인트 — Open-Vocabulary 원리**🔥
>
> 핵심: **언어 모델과 시각 모델을 정렬**. 같은 개념이라면 시각 표현과 언어 표현이 일치해야 함. **CLIP** 같은 모델.
>
> 적용: 클래스마다 학습 가능한 벡터 대신 **언어 모델에서 추출한 단어 임베딩** 사용. 100개 클래스로 학습할 때 그 클래스 단어 임베딩을 클래스 벡터로 두고 학습 (**언어 모델은 고정**, 이미지 인코더만 학습).
>
> 일반화 이유: 언어 모델에서 "강아지"와 "개"는 비슷한 임베딩 → 강아지로 학습해도 "개"라는 단어에 일반화.

## 2.14 Transformer Decoder

### 2.14.1 동기 — 클래스 표현도 동적으로

Classification의 일반 형태 재해석:
- 피처 $Z$ (D-dim) × $W$ ($D \times K$) = 스코어 (K-dim).
- **$W$의 각 row vector $Z_C$ = 그 클래스 대표 D-dim 벡터**.
- 이미지마다 $Z$는 바뀌지만 $Z_C$는 고정.

**문제 제기 — 자동차 예시**:
- 공통 특징: 바퀴 4개, 운전대.
- **버스, 스포츠카, 화물차는 완전 다름** → 하나의 대표값으로 묶기 어려움.

### 2.14.2 해결 — 클래스 표현을 이미지에 맞춰 정제

> 🔥🔥**시험포인트 — 핵심 동기**🔥🔥
> *"이 트랜스포머 디코더가 하는 건 뭐냐면은 **클래스의 표현을 데이터에 맞게 정제**하는 거거든요."*
>
> - 자동차 클래스 토큰이 "F1 경기장 이미지"면 스포츠카에 맞게, "공사 현장"이면 화물차에 맞게 정제.
> - **겨울 나무 vs 여름 나무 비유**: *"겨울에는 초록색 피처를 나무에 사용할 수 없잖아요. 나뭇잎이 없으니까."*

### 2.14.3 구조
- **인코더**: 일반 ViT, 패치 토큰 출력.
- **디코더 입력**:
  - 인코더에서 온 패치 토큰들
  - **클래스마다 하나씩 있는 학습 가능한 벡터** (D-dim, backprop으로 학습)
- 디코더 안 **self-attention**으로 둘이 정보 교환:
  - 패치 토큰도 클래스 토큰 보고 업데이트
  - 클래스 토큰도 패치들 보고 업데이트
- 출력: 정제된 K개 클래스 토큰.
- 정제된 클래스 토큰 ↔ 패치 피처 inner product → 픽셀별 클래스 스코어.

### 2.14.4 핵심 비교
- 정제 없이(=일반 MLP) inner product = 기존 classification.
- 디코더 한 번 거친 후 inner product = 컨텍스츄얼라이즈된 클래스 표현 → **성능 향상**.

## 2.15 하이브리드 디코더 (DETR-like)

교수님 코멘트: *"복잡해지는데, 그렇게 엄청 중요하지 않다."*
- **DETR**(Detection Transformer) 알면 익숙.
- 인코더: Convolution U-Net.
- 디코더: Transformer decoder.
- 학습 가능한 query 토큰 N개 → N개의 "마스크 proposal" + 각 마스크의 클래스 prediction.
- GT 마스크와 매칭하도록 학습.

## 2.16 마무리 — Analogy Making

오전 마무리에 교수님 연구실 방향 소개:

> 🔥**시험포인트 — 미래의 방향**🔥
> *"세그멘테이션이 중요해서 배우는 게 아니라, 다른 모든 비전 문제가 다 이걸로 풀리기 때문. 디텍션, 세그멘테이션, 인스턴스 세그멘테이션, depth 추정, 3D 구조 추정 — 다 **픽셀의 레이블을 찾는 문제**."*
>
> 더 발전시키면 푸는 원리도 비슷 → **못 본 픽셀 레이블링 문제도 5장, 10장 정도로 학습** 가능 (**Analogy Making**).
>
> 구조: 이미지 + 예시 레이블을 함께 입력. *"A는 B일 때 C는 무엇이 되어야 하는가"* 식 analogy. depth든 segmentation이든 logic이 비슷.

> 본 부록은 실습 세션의 코드 내용이 아니라, 조교가 직접 풀어준 **시험 메타 정보**다. 시험 준비에 직접적 도움이 되므로 별도 정리한다.

## A.1 출제 스타일
- **"왜(Why)" 중심, 암기보다 이해**.
- "이게 빠지면 어떻게 될까" 식의 **어블레이션(ablation) 패턴**.

## A.2 예상 출제 영역 (조교가 언급한 패턴)
1. **Multi-Head Attention의 이유** — 왜 head를 여러 개로 나누는가
2. **Positional Embedding 없을 때 어떻게 되는가**
3. **FCN vs U-Net 비교** — 무엇이 좋아졌는가
4. **Atrous Convolution의 의미와 효과**
5. **Spatial Pyramid Pooling의 동기**
6. **1×1 Conv의 역할**
7. **FCN vs Deconvolution Network 차이**
8. **Skip Connection의 수식 항을 제거하면 어떻게 되는가**

## A.3 잘 안 나올 유형
- Receptive Field 사이즈 정확 계산
- 파라미터 수 계산
- sin/cos positional encoding 디테일

## A.4 시험 문제 수
- 학생 발언 기반 — **4문제** 정도

## A.5 한계 명시
- 조교가 3년째 조교 중이지만 **시험 문제를 직접 공유받지는 못한다**고 솔직히 말했음.

# 부록 B. 시험 전 빠른 복습 — 한 페이지 정리

## 6월 5일 오전 (Segmentation 기초)

### CNN 복습
- 출력 사이즈 계산식 (n, k, s, p, d).
- **RF의 정의: 항상 입력 데이터에 대해 정의됨**.
- 풀링 = 파라미터 없는 컨볼루션, RF 키우기용.

### Segmentation의 어려움
1. 모든 픽셀 특징 학습 (몸 같은 단조로운 영역도)
2. 해 공간이 기하급수적

### 역사적 변곡점
- 2012 AlexNet / 2014 FCN 등장.
- 모든 후속 모델 = **RF를 어떻게 다룰지의 문제**.

### FCN
- **FC layer를 컨볼루션으로 "해석"** — 재정의가 아님.
- 두 문제: **(1) score map 해상도 낮음, (2) RF 작음**.
- **해상도-RF Trade-off**: 입력 키우면 출력 해상도 ↑, RF 상대적으로 ↓.

### 해결책 3가지 (중요도 순)
1. **Encoder-Decoder (U-Net)** ← 가장 중요
2. **Atrous Convolution**
3. **Spatial Pyramid Pooling**

### U-Net 핵심
- 인코더-디코더 + **Skip Connection**.
- Skip의 두 역할: (1) 정보 보존, (2) **그래디언트 고속도로** (ResNet residual과 같은 원리).
- **Transposed Convolution** = 디코더 핵심 ($W^T$).
- 위상: **VAE/디퓨전/언어모델/나노바나나 어디에나**.

### Deep을 쓰는 이유 (본질)
- 이론상 2 layer면 완벽.
- **비선형성**(ReLU)으로 공간을 접어 복잡한 분포를 선형 분리 가능하게 만들기 위함.

### Atrous (Dilated) Convolution
- 커널 사이의 공간을 비움 → 파라미터 안 늘리고 RF 확장.
- Atrous rate 너무 크면 로컬리티 손실.

### 일반화 (Image-to-Image)
- **비전 90%가 image-to-image 문제** — YOLO도 FCN의 변형.
- Depth, Pose Estimation 모두 같은 모델로 풀린다.
- **Depth 본질적 어려움**: 한 장으로는 정의 안 됨, 두 장 ray가 만나야 함.

### 마무리 방향성
- 세그멘테이션/depth/디텍션 → 다 U-Net으로 통일.
- 비전/언어/생성모델 → 결국 **한 모델로 통일**.

## 6월 8일 오전 (Vision Transformer)

### Self-Attention 도출
- 일반 골: $y_i = \sum_j w_{ij} g(x_j)$.
- 도출: Average pooling / Convolution / FC / Self-Attention 모두 같은 골에서 weight 정의 방식만 다름.

### FC layer 약점
1. 파라미터 $n^2$
2. **$w_{ij}$가 입력에 무관하게 고정** (3배 확대 비유)

### Hypernetwork 관점 (핵심 프레임)
- Self-Attention = **가중치가 입력에 따라 동적으로 바뀌는 hypernetwork**.
- 트랜스포머 = 시퀀스 모델이 아니라 **non-local context encoding의 발전** (Kaiming He 계보).

### Self-Attention 3관점
1. 전역 맥락 인코딩
2. Hypernetwork
3. FC의 효율적 대체 (파라미터 $3DD'$, 시퀀스 길이 무관)

### 복잡도와 trade-off
- Self-attention **O(N²)**.
- 패치 크기 ↓ → 토큰 ↑ → 메모리 폭증. *"10² = 100, 100² = 10,000."*
- "메모리가 허용하는 가장 작은 패치"가 최적.

### ViT의 발견
- 데이터 많아지면 **CNN inductive bias 불필요**.
- Transformer = **데이터 비특화** → 토크나이제이션만 바꾸면 모든 도메인 적용.
- Scaling law 만족하는 이유: hypernetwork 추정.

### Segmentation Decoder
- **Convolution Decoder**: MLP + 업샘플. 인코더-디코더 + skip 가능.
- **Transformer Decoder**: 클래스 토큰을 이미지 맥락에 맞게 정제 (자동차/나무 비유).
- **하이브리드 (DETR-like)**: N개 mask proposal + class prediction.

### Positional Encoding
- 동일한 순서면 어떤 순서든 OK.
- **Set/Graph는 순서 정의 불가** (LiDAR, 인접행렬).
- **RoPE (Relative Positional Encoding)** — 최근 트랜스포머 표준.

### Open-Vocabulary
- 클래스 벡터 대신 **언어 모델 단어 임베딩** 사용 (**CLIP** 원리).
- 언어 모델 고정, 이미지 인코더만 학습 → 강아지/개로 일반화.

### Analogy Making (교수님 연구실 방향)
- 못 본 픽셀 레이블링도 예시 5~10장으로 학습.
- "A:B = C:?" 식 추론 → 모든 픽셀 레이블링 통합.
