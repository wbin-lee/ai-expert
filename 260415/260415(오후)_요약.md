# 260415 오후 강의 요약 — Seq2Seq · Attention · Transformer 등장 배경

## 1. Sequence-to-Sequence (Seq2Seq)

### 1.1 정의
입력 시퀀스와 출력 시퀀스의 길이가 **다른** 매핑 (Many-to-Many 비동기).
- 예 : 기계 번역, 요약, 대화 생성

### 1.2 Encoder-Decoder 구조
```
입력 시퀀스 ──▶ Encoder (Many-to-One) ──▶ Context Vector
                                              │
                                              ▼
출력 시퀀스 ◀── Decoder (One-to-Many) ◀──────────
```

### 1.3 Context Vector
Encoder RNN의 마지막 히든 상태.
- 입력 전체의 정보를 **단일 벡터** 로 요약
- "농축된 엑기스 / 에스프레소"에 비유

## 2. Seq2Seq의 한계

### 2.1 Information Bottleneck
긴 시퀀스를 단 하나의 벡터로 압축 → 정보 손실.
```
문장이 길수록 → 초반 단어의 정보가 후반에 희석
→ 후반 생성에서 부정확한 번역
```

### 2.2 해결 : Attention
출력 단어마다 **입력의 모든 단어를 다시 참고** 하되, **각 단어에 얼마나 집중할지** 를 동적으로 결정.

## 3. Attention 메커니즘

### 3.1 정렬(Alignment) 개념
출력의 각 단어가 입력의 어느 단어에서 왔는지. 기계 번역의 정렬을 학습.

### 3.2 어텐션 가중치
- 0~1 사이 값 (softmax 결과)
- 모든 입력 위치에 대한 가중치 합 = 1

### 3.3 어텐션 행렬(Attention Matrix)
```
        Output_1  Output_2  Output_3 …
Input_1   0.7      0.1       0.05
Input_2   0.2      0.6       0.10
Input_3   0.1      0.3       0.85
```
- 행 = 입력, 열 = 출력
- 셀 (i, j) = 입력 i가 출력 j에 기여한 정도

### 3.4 언어별 패턴
- **SVO 언어** (한국어, 일본어) : 대각선 패턴
- **불어·아랍어** : off-diagonal, 순서 재배치

## 4. 동작 예시 (번역)

```
Input  : The agreement on the European Economic Area …
Output : L'accord sur la zone économique européenne …

"zone"을 생성할 때:
   "area" 와 연관 0.7 → 크게 참고
   "agreement" 와 0.3 → 부분 참고
   나머지는 거의 0
```

## 5. 두 가지 Attention

### 5.1 Cross-Attention
- 서로 다른 두 시퀀스 간 관계 (입력 ↔ 출력)
- Seq2Seq 기계 번역의 원래 Attention

### 5.2 Self-Attention (Transformer의 핵심)
- 같은 시퀀스 내부 토큰 간 관계
- 문장 내 단어들의 상호 의존성 파악
- Transformer 의 주 메커니즘

## 6. Transformer 등장 배경

### 6.1 CNN의 비전 성공 → NLP의 자극
비전 분야가 CNN으로 급성장, NLP 진영이 새로운 돌파구 필요.

### 6.2 RNN의 구조적 한계
- 순차 의존 → 병렬화 불가
- 긴 시퀀스 학습 어려움 (vanishing)

### 6.3 Transformer(2017)의 해결
- **Attention만으로** 시퀀스 처리 → RNN 제거
- **완전 병렬화** 가능
- **Scalability** — 데이터·모델 크기 ↑ 에도 포화 없음
- → Foundation Model 의 기반

## 7. Transformer의 철학

> "Attention Is All You Need" (Vaswani et al., 2017)

- Attention 메커니즘 만으로 시퀀스 처리 가능
- Scalability 가 딥러닝 시대의 새로운 축이 됨
- CNN/RNN의 제약(포화·순차 의존)을 넘어선 범용 구조

## 8. 핵심 정리

1. Seq2Seq = 입력·출력 길이가 다른 매핑.
2. Encoder(많→1) + Decoder(1→많) 구조.
3. Context Vector = Encoder 마지막 히든 상태.
4. 단일 벡터 압축은 긴 문장에서 Information Bottleneck.
5. Attention으로 입력의 관련 부분만 골라서 참고.
6. Attention 가중치는 softmax → 합 1.
7. Attention 행렬로 입력-출력 관계 시각화.
8. SVO 언어는 대각선, 그 외는 재배치 패턴.
9. Cross-Attention : 두 다른 시퀀스.
10. Self-Attention : 같은 시퀀스 내부.
11. RNN 제거 → **완전 병렬화** 가능.
12. Transformer의 Scalability가 Foundation Model 가능케.
13. 2017 "Attention is All You Need" 가 패러다임 전환.
14. Vision(ViT), Multimodal 로도 확장.
15. 현대 모든 LLM·DiT·Flamingo 등의 공통 기반.

### 강의 중 주요 직관·일화
- **커피 비유** : 입력 = 물(긴 시퀀스), Encoder = 에스프레소 추출, Decoder = 물타기(출력 길이 조절).
- **의사 호칭 변화** : Cross-Attention만 있던 시대 → Self-Attention 등장으로 구분 필요해진 개념적 유비.
- **Slido 실시간 질문** : 160자 제한으로 핵심 질문 유도.
- **Transformer 저자들** : 모두 동등 기여(First Author) — 협업 문화의 한 단면.
