# 260415 오전 강의 요약 — RNN·LSTM·동적 시스템 관점

## 1. RNN의 동기

### 1.1 순차 데이터의 특성
텍스트·음성·시계열 등은 **순서**가 본질. 각 시점의 정보가 다음 시점에 영향.

### 1.2 RNN의 핵심
- **메모리 (Memory)** : 히든 상태 h_t 로 과거 요약
- **피드백 루프** : 이전 상태를 현재 계산에 사용
- **파라미터 공유** : 모든 t에서 동일한 θ

## 2. 동적 시스템(Dynamical System) 관점

### 2.1 두 가지 형태
```
① s_t = f(s_{t-1})              # 자율계 (입력 없음)
② s_t = f(s_{t-1}, x_t)         # 제어계 (외부 입력)
```
RNN은 형태 ②에 해당. Moore Machine / Mealy Machine 과도 개념적 연결.

### 2.2 상태 벡터 해석
- s_t = 과거 입력의 **손실 압축(Lossy Compression)**
- 무한 시퀀스도 유한 차원 벡터로 요약
- **Task-relevant summary** : 과제에 따라 다른 요약이 됨

## 3. RNN의 기본식

```
h_t = tanh( W_h · h_{t-1} + W_x · x_t + b )
y_t = W_y · h_t + b_y
```

## 4. Unfolding (시간축 전개)

순환식을 시간축으로 펼치면 깊은 네트워크 형태:
```
x₁ → h₁ → h₂ → h₃ → …
     ↑     ↑     ↑
     x₂    x₃   …   (동일 W 재사용)
```
- 시간이 무한이면 이론적으로 무한 깊이
- **시간적 Parameter Sharing** (CNN의 공간적 공유와 대응)
- 학습은 **BPTT (Backpropagation Through Time)**

## 5. RNN의 5가지 매핑 형태

| 유형 | 입력 | 출력 | 예시 |
|------|------|------|------|
| One-to-One | 1 | 1 | 이미지 분류 (일반 MLP) |
| One-to-Many | 1 | N | 이미지 캡셔닝 |
| Many-to-One | N | 1 | 감성 분석 |
| Many-to-Many 동기 | N | N (길이 같음) | 비디오 프레임 분류 |
| **Many-to-Many 비동기** | N | M | **기계 번역 (가장 어려움)** |

## 6. Bidirectional RNN

```
→ h_t = f(h_{t-1}, x_t)          (과거 요약)
← g_t = f(g_{t+1}, x_t)          (미래 요약)
출력 = [h_t ; g_t] 또는 합산
```
언어 태스크에서 효과적 (예: NER, 문맥 이해).

## 7. RNN의 한계 : Vanishing / Exploding Gradient

### 7.1 수식 직관
```
∂h_t/∂h_0 = ∏ (∂h_k/∂h_{k-1})
```
- 각 미분값이 < 1 : `0.9999^T → 0` (Vanishing)
- 각 미분값이 > 1 : `1.00001^T → ∞` (Exploding)

### 7.2 해결책
- Gradient Clipping (Exploding)
- **LSTM / GRU** (Vanishing)

## 8. LSTM (Long Short-Term Memory, 1997)

### 8.1 개요
Sepp Hochreiter & Jürgen Schmidhuber 가 제안. 명시적 **Cell State c_t** 와 **Gating Mechanism** 으로 장기 의존성을 유지.

### 8.2 3개 게이트
```
Forget Gate : f_t = σ( W_f · [h_{t-1}, x_t] + b_f )
Input Gate  : i_t = σ( W_i · [h_{t-1}, x_t] + b_i )
Candidate   : c̃_t = tanh( W_c · [h_{t-1}, x_t] + b_c )
Cell State  : c_t = f_t ⊙ c_{t-1} + i_t ⊙ c̃_t
Output Gate : o_t = σ( W_o · [h_{t-1}, x_t] + b_o )
Hidden      : h_t = o_t ⊙ tanh( c_t )
```

### 8.3 게이트의 역할
- **Forget** : 셀 상태에서 버릴 정보 (SSD 의 flash 지우기 과정에 비유)
- **Input** : 새로 저장할 정보
- **Output** : 셀 상태 → 히든 상태로의 출력

### 8.4 셀 상태의 의미
- **장기 메모리** = 셀 상태 c_t
- **단기 메모리** = 히든 상태 h_t
- 셀 상태는 "곱하기 1" 경로로 흘러 그래디언트 보존

## 9. GRU (Gated Recurrent Unit)

조경현 교수가 제안. LSTM 간소화.
- **Reset Gate**, **Update Gate** 의 2개 게이트만 사용
- 셀/히든 분리 없이 **h_t 하나로 통합**
- LSTM 대비 계산 효율, 유사 성능

## 10. 핵심 정리

1. RNN = 메모리를 가진 신경망 (루프/피드백).
2. 모든 t에 동일한 θ 사용 (Parameter Sharing).
3. Unfolding으로 Forward/Backward 가능 (BPTT).
4. 상태 = 과거 입력의 Lossy Compression.
5. Vanishing(0.9999^T) / Exploding(1.00001^T).
6. LSTM은 명시적 셀 상태로 기울기 흐름 개선.
7. LSTM 3게이트 : Forget / Input / Output.
8. Candidate는 tanh(-1~1), 게이트는 sigmoid(0~1).
9. GRU = 경량화, 2 게이트, 성능 유사.
10. Bidirectional로 좌·우 문맥 모두 활용.
11. Many-to-Many 비동기가 seq2seq (번역 등).
12. 상태 해석은 task-dependent.
13. 시간적 Parameter Sharing ↔ CNN 공간적 Sharing 의 쌍.
14. RNN은 기계 번역 시대 ~ Transformer 등장 전의 표준.
15. Transformer 등장 후에도 저자원 환경에서는 여전히 유용.

### 강의 중 주요 직관·일화
- **"시간 = 절대 질서"** : 면접 질문 "세상의 질서가 뭐냐" → 쌍둥이도 0.1초 차로 태어남. RNN의 시간 처리 본질에 연결.
- **변기 공유 비유** : Parameter Sharing 의 경제성.
- **SSD vs HDD** : 인터페이스(SATA)는 같지만 내부가 다름 → LSTM도 RNN 호환 인터페이스 + 내부만 업그레이드.
- **German Engineering(LSTM) vs Korean Engineering(GRU)** : 정교함 vs 실용성.
- **의사 호칭 변화 (한의사 vs 의사)** : 초기에 크로스 어텐션만 있었다가 셀프도 등장하며 구분된 맥락 비유.
