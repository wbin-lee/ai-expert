# 2. RL을 왜 배우는가 — 자율주행 예시

## 2.1 자율주행의 두 단계

1. 센서 데이터에서 주변 차량, 보행자, 차선 등 유의미한 정보를 추출.
2. 그 정보를 바탕으로 steering, acceleration 같은 의사결정을 수행.

## 2.2 RL의 관점

- RL은 중간 feature engineering 없이 sensor data에서 action까지 end-to-end로 policy를 학습할 수 있다.
- 특히 불확실성이 있는 상황에서 의사결정 policy를 학습하는 formalism을 제공한다.

## 2.3 불확실성

- 주변 차량 움직임.
- 보행자 행동.
- 날씨.
- 센서 노이즈.
- 도로 상황.

이런 불확실성 때문에 단순 supervised learning만으로는 sequential decision making을 처리하기 어렵다.

# 3. RL의 두 뿌리

## 3.1 Optimal Control

- 공학적 뿌리.
- 시스템 성능을 최대화하는 controller를 설계하는 문제.
- finite horizon / infinite horizon 문제로 나뉜다.
- Richard Bellman의 dynamic programming이 중요한 역할을 했다.

### 한계

1. 시스템/environment model을 완벽히 알아야 한다.
2. dynamic programming은 state dimension이 커질수록 계산량이 지수적으로 증가한다.
   - Curse of dimensionality.

## 3.2 Trial-and-Error Learning

- 심리학적 뿌리.
- 동물은 만족스러운 결과를 가져오는 행동을 더 자주 하고, 불편한 결과를 가져오는 행동은 줄인다.
- 시행착오를 통해 행동 방식을 학습한다.

## 3.3 RL의 한 문장 정의

RL은 **optimal control 문제를 trial-and-error learning으로 푸는 방법**이다.

# 4. RL의 기본 구조

## 4.1 Agent와 Environment

- Agent는 action을 선택한다.
- Environment는 action을 받고 state를 update하며 reward를 준다.

```text
Agent observes state s_t
Agent chooses action a_t
Environment returns reward r_t and next state s_{t+1}
```

## 4.2 세 핵심 요소

| 요소 | 의미 |
|---|---|
| State | 환경/시스템의 현재 상태 |
| Action | agent가 취하는 행동/control input |
| Reward | 현재 state-action이 얼마나 좋은지 나타내는 scalar feedback |

## 4.3 Supervised Learning과 차이

- Supervisor/label이 없다.
- 정답 action을 알려주지 않는다.
- state, action, reward feedback만으로 학습한다.
- reward는 즉각적인 성능만 측정하고, long-term behavior는 reward 누적합으로 평가해야 한다.

---

# 5. Reward

## 5.1 Reward의 의미

- 현재 state-action이 현재 시점에서 얼마나 이득인지 나타내는 scalar.
- 강화학습 사용자가 직접 설계한다.
- 딥러닝의 loss function 설계와 비슷하게 중요하다.

## 5.2 목표

- 한 순간의 reward가 아니라 expected cumulative reward를 최대화한다.

```text
maximize expected cumulative reward
```

## 5.3 Reward 예시

| 문제 | Positive reward | Negative reward |
|---|---|---|
| Drone control | 목표 경로를 잘 따라감 | 벽/다른 드론과 충돌 |
| Humanoid walking | 앞으로 잘 걸음 | 넘어짐 |
| Portfolio management | 돈을 범 | 돈을 잃음 |
| Game | 점수 획득 | 점수 손실 |

## 5.4 Reward 설계의 어려움

- 물리 세계에서는 reward 설계가 어렵다.
- reward weight tuning이 중요하다.
- 잘못 설계하면 의도하지 않은 behavior를 학습할 수 있다.

# 6. Sequential Decision Making

## 6.1 Action의 long-term effect

- 현재 action은 다음 state뿐 아니라 앞으로의 trajectory 전체를 바꾼다.
- 갈림길에서 직진할지 우회전할지 결정하면 이후 가능한 state path가 달라진다.

## 6.2 핵심 직관

- 지금 당장 reward가 조금 낮더라도 long-term reward가 크면 그 action을 선택해야 한다.
- Bellman equation은 이 직관을 수식화한 것이다.

## 6.3 Exploration vs Exploitation

- Exploration: 미지의 영역을 탐색해 정보를 얻는 action.
- Exploitation: 현재 알고 있는 정보를 이용해 당장 reward를 높이는 action.
- RL 알고리즘은 둘 사이 balance를 맞춰야 한다.

---

# 7. Policy

## 7.1 Policy의 의미

- Policy는 state에서 action으로 가는 mapping이다.

```text
π: state → action
```

- 공학적으로는 controller로 볼 수 있다.
- 게임에서는 플레이 방식으로 볼 수 있다.

## 7.2 Deterministic Policy

- state 하나가 들어오면 action 하나를 고정적으로 출력한다.

```text
a = π(s)
```

## 7.3 Stochastic / Randomized Policy

- state가 주어졌을 때 action에 대한 probability distribution을 출력한다.

```text
π(a | s) = state s에서 action a를 선택할 확률
```

## 7.4 Action sequence가 아니라 policy를 학습하는 이유

- 미리 action sequence를 최적화하면 불확실성에 취약하다.
- 실제 state trajectory가 예상과 달라지면 미리 정한 action이 맞지 않는다.
- policy는 현재 state에 맞는 action을 즉시 출력하므로 disturbance/state perturbation에 대응할 수 있다.

# 8. MDP — Markov Decision Process

## 8.1 MDP의 5요소

MDP는 다음 5개 요소로 정의된다.

```text
(S, A, P, R, γ)
```

| 요소 | 의미 |
|---|---|
| S | state space |
| A | action space |
| P | transition probability |
| R | reward function |
| γ | discount factor |

## 8.2 Transition Probability

```text
P(s' | s, a)
```

- 현재 state가 `s`, 현재 action이 `a`일 때 다음 state가 `s'`가 될 확률.
- stochastic dynamics를 표현한다.

## 8.3 Discount Factor γ

- 0과 1 사이 값.
- 미래 reward를 얼마나 중요하게 볼지 조절한다.
- γ가 0에 가까우면 미래를 강하게 discount한다.
- γ가 1에 가까우면 미래 reward를 거의 동일하게 본다.

## 8.4 MDP 목표

- expected discounted cumulative reward를 최대화하는 policy를 찾는 것.

```text
find π maximizing E[Σ γ^t r_t]
```

# 9. Value Function

## 9.1 V-function

`V^π(s)`는 state `s`에서 출발해 policy `π`를 따랐을 때 앞으로 얻을 cumulative reward의 기대값이다.

```text
V^π(s) = expected return starting from s under policy π
```

## 9.2 Q-function

`Q^π(s, a)`는 state `s`에서 action `a`를 먼저 취하고, 이후 policy `π`를 따랐을 때 앞으로 얻을 cumulative reward의 기대값이다.

```text
Q^π(s, a) = expected return after taking action a at state s, then following π
```

## 9.3 차이

| 함수 | 입력 | 의미 |
|---|---|---|
| V-function | state | 이 state 자체의 가치 |
| Q-function | state, action | 이 state에서 이 action을 선택하는 가치 |

---

# 10. Bellman Equation 직관

## 10.1 Dynamic Programming 핵심

Value function은 두 부분으로 쪼갤 수 있다.

```text
현재 reward + 다음 state부터의 discounted future value
```

즉:

```text
V(s) = immediate reward + γ × future value
```

## 10.2 Policy Evaluation

- policy `π`가 고정되어 있을 때 그 policy의 value function을 계산하는 문제.
- Monte Carlo simulation으로도 가능하지만 variance가 클 수 있다.
- Bellman equation을 이용하면 dynamic programming으로 더 체계적으로 계산할 수 있다.

## 10.3 Matrix / Operator Form

- finite state/action에서는 Bellman equation을 matrix equation으로 쓸 수 있다.
- 또는 operator fixed point 문제로 볼 수 있다.

---

# 11. Optimal Value Function

## 11.1 Optimal V-function

`V*(s)`는 state `s`에서 출발했을 때 가능한 모든 policy 중 얻을 수 있는 최대 expected cumulative reward다.

```text
V*(s) = max_π V^π(s)
```

## 11.2 Optimal Q-function

`Q*(s, a)`는 state `s`에서 action `a`를 먼저 취한 뒤, 이후 최적으로 행동했을 때 얻을 수 있는 최대 expected cumulative reward다.

```text
Q*(s, a) = max_π Q^π(s, a)
```

## 11.3 MDP를 푼다는 의미

- `V*` 또는 `Q*`를 구하면 optimal policy를 얻을 수 있다.
- 그래서 optimal value function만 구해도 MDP를 풀었다고 말할 수 있다.

---

# 12. Value Iteration / Policy Iteration Preview

## 12.1 Value Iteration

- Bellman optimality operator를 반복 적용한다.
- 각 iteration마다 value function이 optimal value에 가까워진다.
- gridworld 예시에서 goal 근처 value가 높고 trap 근처 value가 낮아지는 모습을 설명했다.

## 12.2 Policy Iteration

- policy evaluation과 policy improvement를 번갈아 수행한다.
- value space와 policy space를 오가며 optimal value/policy로 수렴한다.

# 13. RL 문제의 정의

## 13.1 MDP와 RL의 차이

- MDP theory는 transition probability `P`와 reward function `R`을 알고 있다고 가정할 수 있다.
- RL은 MDP model을 모르는 상태에서 data/experience로 optimal policy를 학습한다.

## 13.2 Model-free RL

- transition probability를 직접 identify하지 않는다.
- black-box environment와 interaction하면서 optimal value/policy를 바로 학습한다.
- 1~4일차는 주로 model-free RL을 다룬다.

## 13.3 Model-based RL

- environment model도 함께 학습하거나 활용한다.
- 5일차에 다룰 예정.

# 14. Classical RL Algorithms

## 14.1 Q-learning

- optimal Q-function을 데이터로 학습한다.
- Bellman equation을 이용해 Q estimate를 업데이트한다.
- 데이터 단위는 `(s_t, a_t, r_t, s_{t+1})`.

## 14.2 Policy Gradient

- policy를 직접 parameterize한다.
- policy parameter θ를 gradient ascent로 업데이트한다.
- model-free하게 gradient를 추정하기 위해 policy gradient theorem을 사용한다.

## 14.3 Actor-Critic

- Actor는 policy를 업데이트한다.
- Critic은 value/Q function으로 현재 policy/action을 평가한다.
- modern deep RL 알고리즘은 대부분 actor-critic 계열이다.

# 16. Finite Horizon MDP

## 16.1 정의

- 시간 범위가 `0`부터 유한한 `T`까지인 MDP.
- 마지막 시간에는 action 이후 미래가 없으므로 terminal reward를 따로 둔다.

## 16.2 Infinite Horizon과 차이

- finite horizon에서는 시작 시간이 중요하다.
- value function에 시간 index가 붙는다.

```text
V_t^π(s)
```

- decision rule도 시간마다 달라질 수 있다.
- infinite horizon에서는 stationary policy가 성립하는 경우가 많지만 finite horizon에서는 일반적으로 time-dependent policy가 필요하다.

## 16.3 Inventory Control 예시

- state: 현재 inventory level.
- action: 주문량.
- demand: random variable.
- next inventory:

```text
s_{t+1} = max(s_t + a_t - d_t, 0)
```

- demand distribution을 이용해 transition probability를 계산할 수 있다.

## 16.4 Backward Induction

- finite horizon에서는 terminal value를 알고 시작한다.
- 마지막 시간 value는 terminal reward와 같다.
- 마지막 시간부터 시간을 거꾸로 줄이며 Bellman equation을 적용한다.
- 이를 backward induction이라고 볼 수 있다.

# 17. 오전 강의 핵심 정리

## 17.1 반드시 외울 것

1. RL은 optimal control을 trial-and-error learning으로 푸는 방법.
2. RL의 핵심 요소는 state, action, reward, policy.
3. 목표는 expected cumulative reward maximization.
4. policy는 action sequence가 아니라 state→action mapping.
5. MDP는 `(S, A, P, R, γ)`로 구성된다.
6. `P(s'|s,a)`는 transition probability.
7. `V^π(s)`는 state value, `Q^π(s,a)`는 state-action value.
8. Bellman equation의 핵심은 immediate reward + discounted future value.
9. RL은 model을 모르는 상태에서 MDP를 푸는 문제.
10. finite horizon에서는 value/policy가 time-dependent하다.
