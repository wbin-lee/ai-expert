# 1. Finite Horizon Bellman Optimality

## 1.1 핵심 직관

Optimal value function도 두 부분으로 쪼갤 수 있다.

```text
현재 immediate reward + 다음 state부터의 future optimal value
```

- 지금 당장 reward만 보는 것이 아니다.
- long-term benefit까지 고려해 action을 선택해야 한다.
- 이 직관이 Bellman optimality equation에 들어 있다.

## 1.2 Terminal Condition

- finite horizon에서는 마지막 시간의 value function을 알고 시작한다.
- 마지막 시간 이후에는 더 이상 action/future가 없으므로 terminal reward가 value가 된다.

```text
V_T*(s) = R_T(s)
```

## 1.3 Backward 계산

- `T`에서 시작해 `T-1`, `T-2`, ... , `0` 순서로 거꾸로 계산한다.
- 이것이 backward induction / finite-horizon dynamic programming의 기본 구조다.

# 2. Optimal Policy Construct

## 2.1 Optimal value에서 policy 구하기

Optimal value function `V*`를 구했다면, 현재 state에서 다음 값을 최대화하는 action을 고르면 된다.

```text
π*(s) = argmax_a [ immediate reward + γ × expected future value ]
```

## 2.2 의미

- Bellman equation 안의 action maximization에서 선택된 action이 optimal action이다.
- 각 state마다 따로 argmax를 해야 한다.
- state가 100개라면 state마다 maximization 문제를 풀어야 한다.

## 2.3 Deterministic Markov Policy

- finite state/action MDP에서는 deterministic Markov policy만으로도 optimality를 얻을 수 있다.
- history-dependent policy나 randomized policy가 꼭 필요하지 않다.

## 2.4 Randomized Optimal Policy

- maximizer action이 여러 개라면 그 action들 사이에서 randomization해도 optimality가 유지된다.
- 예: `a1*`, `a2*`, `a3*`가 모두 optimal이면 0.5/0.3/0.2로 섞어도 optimal.

# 3. Continuous MDP

## 3.1 State Update Equation

Continuous system은 다음처럼 dynamics로 표현할 수 있다.

```text
s' = f(s, a, w)
```

- `w`는 random noise.
- noise distribution을 알고 있다고 가정할 수 있다.

## 3.2 Transition Probability로 해석

- state update equation도 transition probability로 바꿔 생각할 수 있다.
- `s' = f(s,a,w)`를 만족하는 noise에 대해 probability를 적분한다.
- discrete sum 대신 continuous integral이 등장한다.

## 3.3 Continuous case의 어려움

- state/action이 연속이면 모든 state에 대해 Bellman update를 exact하게 할 수 없다.
- discretization을 해도 dimension이 커지면 grid point 수가 지수적으로 늘어난다.

## 3.4 Curse of Dimensionality

- 1D에서 각 축을 3개 grid로 자르면 3개 point.
- 2D는 3² = 9개.
- 3D는 3³ = 27개.
- nD는 3ⁿ개.
- exact dynamic programming은 high-dimensional continuous problem에서 계산량 문제가 크다.

# 4. Infinite Horizon MDP

## 4.1 정의

- 시간 범위가 `0`부터 무한대까지.
- discounted cumulative reward를 고려한다.

```text
E[Σ_{t=0}^∞ γ^t r_t]
```

## 4.2 추가 가정

- reward function은 시간에 따라 변하지 않는다.
- transition probability도 시간에 따라 변하지 않는다.
- state/action set은 finite라고 가정한다.
- discount factor `γ`는 0과 1 사이.

## 4.3 Stationary Policy

- infinite horizon에서는 모든 시간에 같은 decision rule을 쓰는 stationary policy가 자연스럽다.
- finite horizon과 달리 value function/policy에 time index가 필요하지 않은 경우가 많다.

# 5. Policy Evaluation — Vector Form

## 5.1 Vπ Vector

state가 `n`개라면 value function을 n-dimensional vector로 쓸 수 있다.

```text
Vπ = [Vπ(s1), Vπ(s2), ..., Vπ(sn)]
```

## 5.2 Reward Vector

policy `π`를 따를 때 각 state에서의 expected immediate reward.

- deterministic policy이면 action 자리에 `π(s)`를 넣으면 된다.
- stochastic policy이면 action probability로 weighted average한다.

## 5.3 Transition Matrix Pπ

`Pπ[i, j]`는 state `i`에서 policy `π`를 썼을 때 다음 state가 `j`가 될 확률이다.

## 5.4 Bellman Equation Vector Form

```text
Vπ = Rπ + γ Pπ Vπ
```

이를 풀면:

```text
Vπ = (I - γPπ)^(-1) Rπ
```

## 5.5 Matrix Inversion의 한계

- matrix inversion은 계산량이 크다.
- 대규모 state space에서는 직접 inversion하지 않는다.
- operator/fixed point 형태가 더 실용적이다.

---

# 6. Bellman Operator와 Fixed Point

## 6.1 Bellman Operator

Policy evaluation operator `Tπ`를 다음처럼 정의한다.

```text
Tπ(V) = Rπ + γ Pπ V
```

Bellman equation은 다음 fixed point 문제로 쓸 수 있다.

```text
Vπ = Tπ(Vπ)
```

## 6.2 Fixed Point

- 어떤 operator를 적용해도 자기 자신이 나오는 점.
- `T(V) = V`를 만족하는 `V`.

## 6.3 Contraction Mapping

Operator `T`가 contraction이면 operator 적용 후 두 점 사이 거리가 줄어든다.

```text
||T(V) - T(V')|| ≤ γ ||V - V'||
```

- `γ`는 0과 1 사이.
- Bellman operator `Tπ`는 γ-contraction이다.

## 6.4 Banach Fixed Point Theorem

Contraction mapping은:

1. unique fixed point를 가진다.
2. 임의의 초기값에서 operator를 반복 적용하면 그 fixed point로 수렴한다.

## 6.5 Policy Evaluation Value Iteration

임의의 `V0`에서 시작해 반복한다.

```text
V_{k+1} = Tπ(V_k)
```

그러면 `V_k → Vπ`.

---

# 7. Bellman Optimality Operator

## 7.1 Optimality Operator

Optimal value를 위한 operator는 action maximization이 들어간다.

```text
T(V)(s) = max_a [ R(s,a) + γ Σ_{s'} P(s'|s,a)V(s') ]
```

## 7.2 성질

- Bellman optimality operator도 γ-contraction이다.
- monotonicity도 가진다.

## 7.3 Unique Fixed Point

Bellman optimality equation:

```text
V = T(V)
```

- unique solution을 가진다.
- 그 unique solution이 optimal value function `V*`다.

---

# 8. Value Iteration

## 8.1 알고리즘

임의의 `V0`에서 시작해 Bellman optimality operator를 반복 적용한다.

```text
V_{k+1} = T(V_k)
```

충분히 반복하면:

```text
V_k → V*
```

## 8.2 Hidden Complexity

- 식은 간단해 보이지만 각 iteration에서 모든 state에 대해 action maximization을 해야 한다.
- state가 만 개면 maximization도 state마다 수행해야 한다.
- exact DP는 state space가 커지면 여전히 비싸다.

## 8.3 Optimal Policy 추출

`V*`를 구한 뒤:

```text
π*(s) = argmax_a [ R(s,a) + γ Σ P(s'|s,a)V*(s') ]
```

- infinite horizon에서는 stationary optimal policy가 나온다.
- finite horizon에서는 time-dependent policy가 나온다.

# 11. Tabular MDP 실습

## 11.1 Tabular의 의미

- state/action space가 discrete하고 작다.
- value function, policy, reward, transition probability를 table/array로 표현할 수 있다.

## 11.2 Reward Array

Reward function `R(s,a)`는 2D array로 표현한다.

```text
R[state, action]
```

## 11.3 Transition Probability Tensor

Transition probability `P(s'|s,a)`는 3D array/tensor로 표현한다.

```text
P[action, state, next_state]
```

강의 코드에서는 indexing convention을 정해 사용한다. convention은 바꿀 수 있지만 코드 전체 index를 일관되게 맞춰야 한다.

## 11.4 Model을 아는 알고리즘

Backward induction, value iteration, policy iteration은 transition probability와 reward function을 알고 있다고 가정한다.

따라서 엄밀히 말하면 강화학습 알고리즘이 아니라 MDP planning / dynamic programming 알고리즘이다.

RL은 `P`와 `R`을 모르는 상태에서 interaction data로 학습한다.

# 12. Backward Induction 실습

## 12.1 입력

- transition probability `P`.
- reward array `R`.
- discount factor `γ`.
- finite horizon length.

## 12.2 알고리즘 구조

- 마지막 time step부터 거꾸로 계산한다.
- 각 state, action에 대해 Q-value를 계산한다.
- 각 state에서 Q-value가 최대인 action을 policy로 저장한다.
- 최대 Q-value를 value function으로 저장한다.

## 12.3 핵심 코드 직관

```text
Q(s,a) = R(s,a) + γ Σ P(s'|s,a) V_{t+1}(s')
V_t(s) = max_a Q(s,a)
π_t(s) = argmax_a Q(s,a)
```

## 12.4 실습 결과 해석

- horizon 100의 간단한 2-state/2-action MDP를 풀었다.
- backward induction으로 각 time step의 value와 policy가 출력된다.
- `t=100`은 episode 종료 상태라 value가 0.
- `t=0`은 시작 시점에서 앞으로 받을 cumulative reward의 기대값.

# 13. Q-function과 Greedy Policy

## 13.1 Q-function

`Q(s,a)`는 state `s`에서 action `a`를 먼저 선택했을 때의 가치다.

```text
Q(s,a) = R(s,a) + γ Σ P(s'|s,a)V(s')
```

## 13.2 Greedy Policy

Q-value가 가장 큰 action을 선택한다.

```text
π(s) = argmax_a Q(s,a)
```

## 13.3 코드 구현 포인트

- NumPy matrix multiplication / `@` operator를 사용해 `Σ P V`를 계산한다.
- axis를 잘 맞춰야 한다.
- argmax는 action axis에 대해 수행해야 한다.

# 14. Value Iteration 실습

## 14.1 알고리즘

1. `V`를 0으로 초기화.
2. Bellman update를 적용해 `V_next` 계산.
3. `||V_next - V||`가 threshold `eps`보다 작으면 종료.
4. 아니면 `V = V_next`로 갱신하고 반복.
5. 수렴한 `V*`에서 greedy policy를 추출.

## 14.2 Epsilon threshold

- 작게 잡으면 더 정확하지만 오래 걸린다.
- 크게 잡으면 빨리 끝나지만 부정확하다.
- 예: `1e-3`, `1e-6`.

## 14.3 실습 관찰

- `eps=1e-6`에서는 약 129 step 만에 수렴.
- `eps`를 크게 하면 더 빨리 끝나지만 value가 덜 정확하다.
- `eps`를 매우 작게 해도 simple MDP에서는 값이 거의 변하지 않았다.

## 14.4 Backward Induction과 결과 비교

- backward induction과 value iteration이 같은 optimal policy를 냈다.
- 예시에서는:

```text
state 0 → action 1
state 1 → action 0
```

# 15. Value Iteration vs Policy Iteration

## 15.1 Value Iteration

- value function만 반복 업데이트한다.
- value가 수렴한 뒤 마지막에 policy를 한 번 추출한다.

```text
V0 → V1 → V2 → ... → V*
then extract π*
```

## 15.2 Policy Iteration

- policy evaluation과 policy improvement를 번갈아 수행한다.
- value와 policy를 함께 업데이트한다.

```text
V1, π1 → V2, π2 → V3, π3 → ...
```

## 15.3 어느 것이 더 좋은가

- 무조건 어느 쪽이 빠르다고 말하기 어렵다.
- stopping criterion, MDP 구조, state/action size에 따라 달라진다.
