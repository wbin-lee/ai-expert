# 1. 오전 내용 복습

## 1.1 PPO의 핵심

오전에는 TRPO와 PPO를 공부했다.

PPO에서 가장 많이 쓰는 버전은 clipped objective이다.

policy ratio:

\[
r_t(\theta)
=
\frac{\pi_\theta(a_t|s_t)}
{\pi_{\theta_{\text{old}}}(a_t|s_t)}
\]

이 ratio는 old policy 대비 current policy가 해당 action의 확률을 얼마나 바꿨는지 나타낸다.

PPO는 ratio를 다음 구간으로 clipping한다.

\[
[1-\epsilon,1+\epsilon]
\]

clipped objective:

\[
L^{CLIP}(\theta)
=
\mathbb{E}
\left[
\min
\left(
r_t(\theta)\hat A_t,
\text{clip}(r_t(\theta),1-\epsilon,1+\epsilon)\hat A_t
\right)
\right]
\]

## 1.2 PPO의 장점

- TRPO와 달리 Hessian을 쓰지 않는다.
- conjugate gradient가 필요 없다.
- line search도 보통 필요 없다.
- Adam 같은 first-order optimizer로 구현할 수 있다.

## 1.3 남은 문제

PPO objective에는 advantage estimate \(\hat A_t\)가 필요하다.

따라서 PPO를 잘 쓰려면 advantage를 어떻게 안정적으로 estimate할지가 중요하다.

---

# 2. PPO에서 DQN Critic을 쓰지 않는 이유

## 2.1 DDPG에서는 DQN critic이 자연스러웠다

DDPG는 off-policy deterministic actor-critic이다.

- replay buffer를 사용한다.
- critic도 off-policy Q-learning 계열로 학습할 수 있다.
- deterministic policy이므로 target에서 next action을 하나만 넣으면 된다.

DDPG target:

\[
y=r+\gamma Q_{\phi^-}(s',\mu_{\theta^-}(s'))
\]

## 2.2 PPO는 on-policy이다

PPO는 global하게 on-policy이다.

- 현재 policy로 trajectory를 모은다.
- replay buffer에 과거 데이터를 오래 저장해 재사용하는 구조가 아니다.

따라서 DQN-style off-policy critic을 붙이는 것이 구조적으로 자연스럽지 않다.

## 2.3 PPO는 stochastic policy를 쓴다

PPO는 보통 randomized policy를 사용한다.

\[
\pi_\theta(a|s)
\]

stochastic policy의 value evaluation에 DQN을 쓰려면 target 계산에서 action에 대한 expectation 또는 integral이 필요해질 수 있다.

continuous action에서는 이 적분이 계산 병목이 된다.

따라서 PPO에서는 DQN critic보다 value function 기반 advantage estimation을 주로 사용한다.

---

# 3. GAE의 문제의식

GAE는 Generalized Advantage Estimation이다.

목표:

- advantage estimate의 bias와 variance를 적절히 trade-off한다.

극단적인 두 방식:

| 방식 | 장점 | 단점 |
|---|---|---|
| Monte Carlo return | bias가 작다 | variance가 크다 |
| 1-step TD residual | variance가 작다 | value estimate bias에 민감 |

GAE는 이 두 방식 사이를 \(\lambda\)로 연결한다.

---

# 4. Policy Gradient에서 \(\Psi_t\)의 역할

policy gradient는 일반적으로 다음 형태로 볼 수 있다.

\[
\nabla_\theta J(\theta)
=
\mathbb{E}
\left[
\nabla_\theta\log\pi_\theta(a_t|s_t)\Psi_t
\right]
\]

\(\Psi_t\)에는 여러 선택지가 들어갈 수 있다.

- 전체 누적 reward.
- action 이후의 return.
- baseline을 뺀 return.
- \(Q^\pi(s_t,a_t)\).
- \(A^\pi(s_t,a_t)\).
- TD residual.

실전에서는 variance 감소를 위해 advantage를 사용하는 것이 중요하다.

---

# 5. Discounted Value와 Advantage

discount factor \(\gamma\)가 있는 setting에서 value와 Q는 다음처럼 정의된다.

\[
V_\gamma^\pi(s)
=
\mathbb{E}
\left[
\sum_{l=0}^{\infty}
\gamma^l r_{t+l}
\mid s_t=s
\right]
\]

\[
Q_\gamma^\pi(s,a)
=
\mathbb{E}
\left[
\sum_{l=0}^{\infty}
\gamma^l r_{t+l}
\mid s_t=s,a_t=a
\right]
\]

advantage:

\[
A_\gamma^\pi(s,a)
=
Q_\gamma^\pi(s,a)-V_\gamma^\pi(s)
\]

PPO와 policy gradient에서는 이 advantage estimate가 update 방향을 결정한다.

---

# 6. TD Residual

## 6.1 Approximate value function

value function은 보통 정확하지 않고 critic으로 근사한다.

\[
V(s)
\approx
V^\pi(s)
\]

## 6.2 TD residual 정의

TD residual은 다음과 같다.

\[
\delta_t^V
=
r_t+\gamma V(s_{t+1})-V(s_t)
\]

이는 1-step Bellman target과 현재 value의 차이이다.

## 6.3 advantage와의 연결

만약 \(V\)가 정확한 \(V^\pi\)라면, TD residual의 conditional expectation은 advantage가 된다.

직관적으로:

- \(r_t+\gamma V(s_{t+1})\)는 \(Q(s_t,a_t)\)의 1-step estimate이다.
- \(V(s_t)\)를 빼면 \(Q-V\), 즉 advantage가 된다.

따라서 \(\delta_t^V\)를 가장 기본적인 advantage estimate로 사용할 수 있다.

---

# 7. Multi-Step Advantage Estimate

## 7.1 1-step estimate

\[
\hat A_t^{(1)}
=
\delta_t
=
r_t+\gamma V(s_{t+1})-V(s_t)
\]

이 방식은 variance가 낮지만 value function bias에 크게 의존한다.

## 7.2 2-step estimate

\[
\hat A_t^{(2)}
=
\delta_t+\gamma\delta_{t+1}
\]

전개하면:

\[
\hat A_t^{(2)}
=
r_t+\gamma r_{t+1}
\gamma^2 V(s_{t+2})
-V(s_t)
\]

## 7.3 k-step estimate

\[
\hat A_t^{(k)}
=
\sum_{l=0}^{k-1}
\gamma^l\delta_{t+l}
\]

전개하면:

\[
\hat A_t^{(k)}
=
\sum_{l=0}^{k-1}\gamma^l r_{t+l}
\gamma^k V(s_{t+k})
-V(s_t)
\]

## 7.4 k가 커질수록 생기는 효과

k가 커질수록:

- 더 긴 미래 reward를 직접 사용한다.
- value bootstrap에 의존하는 정도가 줄어든다.
- bias는 줄어든다.
- sample reward가 더 많이 들어와 variance는 커진다.

k가 무한대에 가까워지면 Monte Carlo return에서 baseline \(V(s_t)\)를 뺀 형태에 가까워진다.

---

# 8. GAE: Generalized Advantage Estimator

## 8.1 \(\lambda\) 도입

GAE는 여러 k-step advantage estimate를 가중합한다.

새 hyperparameter \(\lambda\)를 도입한다.

\[
0\le \lambda \le 1
\]

주의:

- \(\gamma\)는 문제에 정의된 discount factor이다.
- \(\lambda\)는 GAE를 위한 새 hyperparameter이다.

## 8.2 GAE 정의

GAE는 다음과 같이 정의된다.

\[
\hat A_t^{GAE(\gamma,\lambda)}
=
\sum_{l=0}^{\infty}
(\gamma\lambda)^l
\delta_{t+l}
\]

즉 TD residual의 discounted sum이다.

discount factor가 \(\gamma\lambda\) 형태로 들어간다.

## 8.3 \(\lambda=0\)

\[
\lambda=0
\]

이면

\[
\hat A_t^{GAE}
=
\delta_t
\]

즉 1-step TD residual만 사용한다.

특징:

- variance가 낮다.
- bias가 클 수 있다.

## 8.4 \(\lambda=1\)

\[
\lambda=1
\]

이면 Monte Carlo return 기반 advantage에 가까워진다.

특징:

- bias가 작다.
- variance가 크다.

## 8.5 중간값

\[
0<\lambda<1
\]

은 bias와 variance 사이의 절충이다.

실전에서는 보통 \(\lambda\)를 0과 1 사이에서 조정해 안정적인 advantage estimate를 만든다.

---

# 9. GAE의 Recursion 구현

GAE는 뒤에서 앞으로 계산하면 효율적이다.

TD residual:

\[
\delta_t=r_t+\gamma V(s_{t+1})-V(s_t)
\]

recursion:

\[
\hat A_t
=
\delta_t+\gamma\lambda \hat A_{t+1}
\]

value target은 return 형태로 backward recursion할 수 있다.

\[
R_t=r_t+\gamma R_{t+1}
\]

실습에서는 trajectory 마지막부터 앞으로 오면서:

- target value
- advantage

를 함께 계산한다.

---

# 10. PPO와 GAE

PPO objective에는 advantage가 필요하다.

\[
L^{CLIP}(\theta)
=
\mathbb{E}
\left[
\min
\left(
r_t(\theta)\hat A_t,
\text{clip}(r_t(\theta),1-\epsilon,1+\epsilon)\hat A_t
\right)
\right]
\]

여기서 \(\hat A_t\)를 GAE로 계산하면:

- Monte Carlo보다 variance가 낮다.
- 1-step TD보다 bias가 낮다.
- PPO 학습이 더 안정적이다.

GAE는 PPO만을 위해 만들어진 것은 아니지만, PPO와 결합했을 때 매우 많이 사용된다.

---

# 11. PPO와 LLM Fine-Tuning

## 11.1 LLM을 policy로 보기

LLM도 강화학습 관점에서 policy로 볼 수 있다.

| RL | LLM |
|---|---|
| state | prompt, conversation history, question |
| action | generated output, next token, answer |
| policy | \(p_\theta(o|q)\) |
| reward | human preference, reward model, verifiable reward |

LLM은 prompt \(q\)가 주어졌을 때 output \(o\)를 생성하는 policy이다.

## 11.2 RLHF

LLM을 human preference에 맞게 align하기 위해 RLHF가 사용된다.

기본 흐름:

1. pretrained 또는 SFT model이 있다.
2. human feedback 또는 reward model이 있다.
3. PPO 같은 RL 알고리즘으로 model을 fine-tune한다.

## 11.3 PPO의 value model 부담

PPO에는 advantage estimate가 필요하다.

이를 위해 value model이 필요하다.

LLM setting에서는:

- policy model 자체가 이미 크다.
- value model까지 추가하면 memory와 computation 부담이 커진다.
- value model이 부정확하면 policy update도 불안정해질 수 있다.

이 문제의식에서 GRPO가 등장한다.

---

# 12. GRPO 개요

GRPO는 Group Relative Policy Optimization이다.

핵심 아이디어:

- PPO와 매우 비슷한 objective를 사용한다.
- 하지만 value model을 쓰지 않는다.
- 같은 question에 대해 여러 output을 sampling해 group을 만든다.
- group 안에서 reward를 상대평가해 advantage를 만든다.

즉, value model 기반 절대평가를 버리고 group 내 상대평가를 사용한다.

---

# 13. Group Relative Advantage

## 13.1 여러 답변 생성

같은 question \(q\)에 대해 \(G\)개의 answer를 sampling한다.

\[
o_1,o_2,\dots,o_G
\]

각 output에 대해 reward를 계산한다.

\[
r_1,r_2,\dots,r_G
\]

## 13.2 상대 advantage

GRPO는 group 평균과 표준편차로 reward를 normalize한다.

\[
\hat A_i
=
\frac{r_i-\text{mean}(r_1,\dots,r_G)}
{\text{std}(r_1,\dots,r_G)}
\]

의미:

- group 평균보다 잘한 답변은 양의 advantage.
- group 평균보다 못한 답변은 음의 advantage.
- scale은 group standard deviation으로 normalize한다.

## 13.3 장점

- value network가 필요 없다.
- value model 학습 불안정성이 사라진다.
- LLM fine-tuning에서 memory/computation을 줄일 수 있다.

---

# 14. GRPO Objective

GRPO는 PPO-style clipped objective를 사용한다.

policy ratio:

\[
r_i(\theta)
=
\frac{\pi_\theta(o_i|q)}
{\pi_{\theta_{\text{old}}}(o_i|q)}
\]

clipped objective:

\[
\min
\left(
r_i(\theta)\hat A_i,
\text{clip}(r_i(\theta),1-\epsilon,1+\epsilon)\hat A_i
\right)
\]

여기에 reference policy와의 KL penalty를 추가한다.

## 14.1 Reference policy

reference policy는 보통 SFT가 끝난 model이다.

\[
\pi_{\text{ref}}
\]

GRPO는 현재 policy가 reference model에서 너무 멀어지지 않도록 KL penalty를 둔다.

## 14.2 KL penalty

실제 구현에서는 KL term의 estimator로 다음 형태가 사용될 수 있다.

\[
\rho-\log\rho-1
\]

여기서 \(\rho\)는 reference/current probability ratio 형태로 계산된다.

---

# 15. TRPO, PPO, GRPO 비교

| 알고리즘 | 핵심 아이디어 | 장점 | 단점 |
|---|---|---|---|
| TRPO | trust region, natural policy gradient, line search | 안정적이고 이론적으로 정교 | 구현 복잡, 계산량 큼 |
| PPO | clipped objective 또는 adaptive KL penalty | 구현 쉬움, first-order optimizer 사용 | strict trust region 보장은 약함 |
| GRPO | group-relative advantage로 value model 제거 | LLM fine-tuning에 경량 | group sampling 필요, physical control에는 덜 자연스러움 |

---

# 16. GRPO의 적용 범위

GRPO는 특히 LLM에 특화되어 있다.

이유:

- 같은 prompt에 대해 여러 답변을 sampling하기 쉽다.
- 답변들끼리 reward를 비교할 수 있다.
- value model 없이도 group relative score를 만들 수 있다.

물리 control task에서는 같은 state에서 여러 action을 동시에 실행해 비교하기 어렵다.

따라서 GRPO는 physical world보다는 language model fine-tuning에 더 자연스럽다.

# 17. TRPO 실습 개요

오후 실습은 TRPO/PPO/GRPO를 다뤘다.

TRPO 실습의 목표:

- TRPO의 최종 loss와 constraint가 코드에서 어떻게 구현되는지 확인한다.
- GAE로 advantage를 계산한다.
- Fisher vector product, conjugate gradient, backtracking line search가 어떤 역할을 하는지 이해한다.

---

# 18. TRPO 실습: Policy와 Critic

## 18.1 Policy network

TRPO policy network는 observation을 받아 action distribution을 출력한다.

continuous action policy에서는 Gaussian distribution을 사용할 수 있다.

- mean network
- log sigma parameter

log sigma를 쓰는 이유:

- network output은 실수 전체 범위를 가질 수 있다.
- sigma는 양수여야 한다.
- log sigma를 parameterize한 뒤 exponentiation하면 양수 scale을 만들 수 있다.

## 18.2 Critic network

critic은 observation을 받아 value를 출력한다.

\[
V_\phi(s)
\]

이 value는 GAE와 value target 계산에 사용된다.

---

# 19. TRPO 실습: Trajectory와 종료 처리

trajectory를 수집할 때 episode가 끝나는 방식에 따라 마지막 value 처리가 달라진다.

## 19.1 실제 terminal

환경이 물리적으로 episode 종료를 알리면 마지막 value를 0으로 둔다.

\[
V(s_T)=0
\]

## 19.2 truncation

episode가 실제 terminal이 아니라 time limit 등으로 잘린 경우에는 마지막 state의 value estimate를 bootstrap한다.

\[
V(s_T)
\]

이 차이를 구분해야 value target과 advantage 계산이 왜곡되지 않는다.

---

# 20. TRPO 실습: GAE 계산

실습의 `compute_value` 또는 유사 함수에서는 뒤에서 앞으로 계산한다.

TD residual:

\[
\delta_t
=
r_t+\gamma V(s_{t+1})-V(s_t)
\]

advantage recursion:

\[
A_t
=
\delta_t+\gamma\lambda A_{t+1}
\]

value target recursion:

\[
R_t
=
r_t+\gamma R_{t+1}
\]

구현상:

- `delta`는 episode 길이만큼의 vector이다.
- `advantage`와 `target_value`를 backward loop로 채운다.
- \(\gamma\lambda\)가 GAE recursion의 핵심이다.

---

# 21. TRPO 실습: Policy Update 근사

TRPO는 constrained optimization을 직접 풀지 않는다.

근사:

- objective는 first-order approximation.
- KL constraint는 second-order approximation.

결과:

\[
\max_\theta g^\top(\theta-\theta_{\text{old}})
\]

\[
\text{s.t.}
\quad
\frac{1}{2}
(\theta-\theta_{\text{old}})^\top
H
(\theta-\theta_{\text{old}})
\le \delta
\]

solution:

\[
\theta_{\text{new}}
=
\theta_{\text{old}}
+
\sqrt{
\frac{2\delta}{g^\top H^{-1}g}
}
H^{-1}g
\]

---

# 22. Fisher Vector Product

## 22.1 왜 필요한가

TRPO에서는 Hessian \(H\)를 직접 만들거나 저장하기 어렵다.

conjugate gradient는 \(H^{-1}g\)를 구하기 위해 반복적으로 \(Hv\) 형태의 product가 필요하다.

## 22.2 Fisher vector product

Fisher vector product는 \(Hv\)를 Hessian matrix를 명시적으로 만들지 않고 계산하는 방법이다.

개념적으로:

- KL gradient와 vector product를 이용한다.
- automatic differentiation으로 second-order product를 효율적으로 계산한다.

실습에서는 util 함수로 제공되어 있고, 내부 구현은 자세히 다루지 않았다.

---

# 23. Conjugate Gradient

## 23.1 목적

직접 \(H^{-1}\)를 구하지 않고 다음 linear system을 푼다.

\[
Hs=g
\]

그러면 \(s\approx H^{-1}g\)가 된다.

## 23.2 직관

conjugate gradient는 quadratic problem을 효율적으로 푸는 iterative method이다.

TRPO에서는:

- natural policy gradient direction을 구하기 위해 사용한다.
- Hessian inverse를 직접 계산하지 않는다.
- Fisher vector product를 반복적으로 호출한다.

---

# 24. Backtracking Line Search

## 24.1 왜 필요한가

TRPO는 많은 근사를 사용한다.

따라서 closed form step을 그대로 쓰면:

- KL constraint를 넘을 수 있다.
- 실제 return improvement가 안 될 수 있다.

## 24.2 line search parameter

실습에서 언급된 주요 parameter:

- backtracking coefficient: 처음 full step의 몇 배부터 시도할지.
- \(\beta\): step을 줄이는 비율.
- \(\alpha\): improvement condition을 얼마나 엄격하게 볼지.

## 24.3 절차

1. full step을 시도한다.
2. expected improvement와 KL constraint를 검사한다.
3. 조건을 만족하면 accept한다.
4. 실패하면 step을 줄여 다시 시도한다.

---

# 25. PPO 실습

## 25.1 PPO 목표

TRPO의 safe update 효과를 first-order 방식으로 흉내 낸다.

PPO는 KL constrained optimization 대신 clipped objective를 사용한다.

## 25.2 PPO update 구현

policy ratio:

\[
r_t
=
\frac{\pi_\theta(a_t|s_t)}
{\pi_{\theta_{\text{old}}}(a_t|s_t)}
\]

clipped ratio:

\[
\text{clip}(r_t,1-\epsilon,1+\epsilon)
\]

loss는 maximize objective를 minimize 형태로 바꾸기 위해 negative를 붙인다.

\[
L_\pi
=
-
\mathbb{E}
\left[
\min
\left(
r_t A_t,
\text{clip}(r_t,1-\epsilon,1+\epsilon)A_t
\right)
\right]
\]

## 25.3 실습 결과와 주의

PPO는 TRPO보다 구현이 간단하고 대부분의 task에서 고르게 잘 작동한다.

TRPO는 hyperparameter와 engineering trick이 많이 필요하다.

강의에서는 실제 프로젝트에서 내부 구현을 꼭 만질 필요가 없다면 Stable-Baselines3 같은 검증된 라이브러리 사용도 합리적이라고 설명했다.

TRPO/PPO를 scratch부터 구현하면 오류 가능성이 높다.

# 27. GRPO 실습 개요

마지막 실습은 GRPO였다.

컨트롤 task에서 벗어나 LLM fine-tuning을 toy example로 모사했다.

목표:

- 작은 MLP language-policy model을 만든다.
- 질문에 대해 여러 답변 후보 중 하나를 고르게 한다.
- reward를 통해 올바른 답변 확률을 높인다.
- PPO-style loss와 group-relative advantage를 구현한다.

---

# 28. LLM을 강화학습으로 학습하는 이유

## 28.1 Supervised learning의 한계

LLM은 대규모 text corpus로 next-token likelihood를 maximize하며 학습된다.

하지만 supervised learning만으로는 human preference에 완전히 align되기 어렵다.

## 28.2 RLHF

RLHF는 human feedback을 reward signal로 사용해 LLM을 fine-tune한다.

예:

- 두 답변 중 사람이 더 선호하는 답변을 선택한다.
- reward model을 학습한다.
- PPO 등으로 language model을 preference에 맞게 업데이트한다.

## 28.3 from scratch가 아니라 fine-tuning

LLM을 강화학습만으로 처음부터 학습시키지는 않는다.

일반적인 흐름:

1. 대규모 supervised pretraining.
2. SFT.
3. RLHF 또는 verifiable reward 기반 RL fine-tuning.

강화학습은 보통 이미 어느 정도 대화가 가능한 model을 human preference에 맞게 조정하는 단계에서 사용된다.

---

# 29. PPO의 LLM 적용 한계

PPO로 LLM을 fine-tune하려면 value model이 필요하다.

문제:

- policy LLM 자체가 매우 크다.
- value model도 별도로 필요해 memory/computation 부담이 커진다.
- value model이 부정확하면 policy가 잘못된 방향으로 update될 수 있다.

GRPO는 이 부담을 줄이기 위해 value model을 제거한다.

---

# 30. GRPO 실습: Toy Language Model

## 30.1 tokenizer와 embedding

실습에서는 pretrained BERT tokenizer/model을 사용해 text를 embedding한다.

예시 질문:

```text
What is the capital of Brazil?
```

답변 후보:

1. Brasilia
2. Rio de Janeiro
3. Sao Paulo
4. Fortaleza

정답 index는 Brasilia에 해당하는 0이다.

## 30.2 policy model

실제 LLM 대신 3개 linear layer를 가진 작은 MLP를 사용한다.

입력:

- question text embedding

출력:

- 각 답변 후보를 선택할 확률

마지막에는 softmax를 사용한다.

\[
\text{probs}=\text{softmax}(\text{logits})
\]

## 30.3 value model 없음

GRPO는 value model을 사용하지 않는다.

실습에서도 PPO와 달리 별도 value network를 구현하지 않는다.

# 31. GRPO 실습: 초기 policy 확인

초기화 직후 policy는 거의 random이다.

Brazil 수도 질문에 대해 각 답변 확률은 대략 0.25 근처였다.

예:

- Brasilia: 약 0.23
- Rio de Janeiro: 약 0.25
- Sao Paulo: 약 0.26
- Fortaleza: 약 0.26

초기 model은 정답이 아닌 Sao Paulo를 선택했다.

이는 model이 아직 reward로 학습되지 않았기 때문이다.

# 32. GRPO Loss 구현

## 32.1 old policy에서 group sample 생성

old policy \(\pi_{\theta_{\text{old}}}\)로 같은 question에 대해 \(G\)개의 action을 sampling한다.

구현에서는 `torch.multinomial`을 사용해 probability distribution에서 여러 답변을 뽑는다.

## 32.2 current policy probability

current policy \(\pi_\theta\)에도 같은 question을 넣어 각 action probability를 구한다.

## 32.3 policy ratio

PPO-style ratio:

\[
r_i
=
\frac{\pi_\theta(a_i|q)}
{\pi_{\theta_{\text{old}}}(a_i|q)}
\]

이 ratio가 clipped objective에 들어간다.

---

# 33. GRPO 실습: Reward와 Advantage

## 33.1 reward 정의

toy task에서는 사람이 직접 reward를 정의한다.

Brazil 수도 예시:

- 정답 Brasilia를 고르면 \(+1.0\).
- 오답을 고르면 \(-0.1\).

이는 실제 LLM fine-tuning의 reward model 또는 human feedback을 단순화한 것이다.

## 33.2 group-relative advantage

reward tensor에서 평균과 표준편차를 계산한다.

\[
\hat A_i
=
\frac{r_i-\bar r}{\text{std}(r)}
\]

이 advantage는 reward와 같은 shape의 tensor이다.

---

# 34. GRPO 실습: Clipped Policy Loss

clipped ratio:

\[
\tilde r_i
=
\text{clip}(r_i,1-\epsilon,1+\epsilon)
\]

policy objective:

\[
\mathbb{E}
\left[
\min(r_i\hat A_i,\tilde r_i\hat A_i)
\right]
\]

gradient descent optimizer를 사용하기 위해 loss는 negative로 둔다.

\[
L_{\text{policy}}
=
-
\mathbb{E}
\left[
\min(r_i\hat A_i,\tilde r_i\hat A_i)
\right]
\]

즉 PPO loss와 거의 같고, advantage 계산 방식만 group-relative로 바뀐다.

---

# 35. GRPO 실습: KL Penalty

## 35.1 reference policy와 old policy 차이

논문에서 KL penalty는 reference policy \(\pi_{\text{ref}}\)와 current policy 사이에 둔다.

\(\pi_{\text{ref}}\)는 보통 SFT가 끝난 model이다.

실습에서는 별도 reference policy가 없으므로 old policy를 대신 사용했다.

주의:

- 실제 GRPO에서는 \(\pi_{\text{old}}\)와 \(\pi_{\text{ref}}\)가 같은 개념이 아니다.
- 실습에서는 toy setting 때문에 단순화했다.

## 35.2 KL estimator

KL penalty는 ratio 기반 estimator로 계산했다.

\[
\rho-\log\rho-1
\]

여기서 \(\rho\)는 current/reference 또는 old/current probability ratio 형태이다.

## 35.3 total loss

total loss:

\[
L
=
L_{\text{policy}}
+
\beta L_{KL}
\]

\(\beta\)는 KL penalty weight이다.

강의에서는 예시로 매우 작은 값, 예를 들어 0.0005 정도를 언급했다.

학습 전:

- Brasilia 확률은 약 0.23 수준이었다.

GRPO 100 epoch 정도 학습 후:

- 나머지 오답 확률은 감소했다.


답변 후보:

Apple 질문에 대해서도 GRPO로 학습한 뒤:

- Red 확률이 상승했다.

# 37. 실습에서 실제 GRPO와 다른 점

## 37.1 실제 LLM은 SFT 후 RL fine-tuning

실습에서는 random 초기화된 작은 MLP를 바로 GRPO로 학습했다.

실제 LLM에서는:

- pretrained/SFT model이 먼저 있다.
- 그 model을 reference로 두고 RL fine-tuning을 한다.

## 37.2 reference policy 단순화

실제 GRPO:

\[
\pi_{\text{ref}}
\]

는 SFT model이다.

실습:

- 별도 SFT reference가 없으므로 old policy를 reference처럼 사용했다.

## 37.3 toy task

실습은 실제 LLM fine-tuning이 아니라 toy language-policy example이다.

하지만 다음 흐름을 이해하기에는 충분하다.

- prompt를 state로 본다.
- 답변 선택을 action으로 본다.
- reward로 선호 답변을 강화한다.
- group reward normalization으로 advantage를 만든다.
- PPO-style clipped loss로 policy를 업데이트한다.

# 38. 오후 핵심 정리

- PPO에는 advantage estimate가 필요하고, 이를 안정적으로 계산하는 대표 방법이 GAE이다.
- GAE는 TD residual들의 \((\gamma\lambda)\)-discounted sum이다.
- \(\lambda=0\)이면 1-step TD residual이 되어 variance는 낮지만 bias가 크다.
- \(\lambda=1\)이면 Monte Carlo return에 가까워져 bias는 낮지만 variance가 크다.
- PPO는 DQN critic보다 value function과 GAE를 사용하는 방식이 더 자연스럽다.
- LLM fine-tuning에서 PPO는 value model이 필요해 memory/computation 부담이 크다.
- GRPO는 같은 prompt에 대한 여러 답변을 group으로 만들고, reward를 group 평균/표준편차로 normalize해 advantage를 만든다.
- GRPO는 value model을 제거해 LLM fine-tuning을 경량화한다.
- TRPO 실습에서는 GAE, Fisher vector product, conjugate gradient, backtracking line search가 핵심 구현 요소였다.
- PPO 실습에서는 ratio clipping과 negative clipped objective loss가 핵심이다.
- GRPO 실습에서는 toy language model을 policy로 보고, group-relative advantage와 KL penalty를 이용해 정답 답변 확률을 높였다.
