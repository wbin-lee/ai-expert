## 2. ⭐⭐ DeepSeek 쇼크 — 큰 그림

### 2.1 ⭐ 시작점
- ⭐ ⭐ 강사 본인이 ⭐ **딥식 페이퍼 나왔을 때 미국에서 연구년 중** — 미국 회사들이 다 같이 읽고 구현하던 날을 생생하게 기억.

### 2.2 ⭐⭐⭐ 왜 쇼크였나 — 시험 출제 포인트
- ⭐ ⭐ ⭐ 그동안 ⭐ **빅테크 + OpenAI 주도** → 우리에겐 시장 없는 줄 알았음.

- ⭐ ⭐ 한국 → "우리도 딥특 필요" → ⭐ 독자적 파운데이션 모델 경쟁 시작.

### 2.3 ⭐ 국내 파장
- ⭐ ⭐ 네이버 ⭐ **탈락** — Open Weight를 일부 썼다는 게 이슈.
- ⭐ 업스테이지 — Weight 갖다 썼다고 ⭐ **타사가 이의 제기** → ⭐ 대표 설명: ⭐ "우리는 ⭐ **웨이트 움직인 과정을 다 가지고 있다**".
- ⭐ ⭐ 아키텍처 자체는 ⭐ **트랜스포머 + SwiGLU 류**라 비슷한 게 당연.

## 3. ⭐⭐ SFT (Supervised Fine-Tuning) 복습

### 3.1 ⭐ 의미
- ⭐ 우리가 했던 Instruction Tuning이 ⭐ **SFT**의 한 형태.
- ⭐ Translate this from English to Korean → I love you → 사랑해 — 이런 페어 ⭐ 다양한 인스트럭션 변형으로 ⭐ **제너럴라이즈**.
- "in 3 words" / "5 words" 같은 변형도 학습시켜 ⭐ **본 적 없는 변형도 가능**.

### 3.2 ⭐⭐ Post-training 두 가지 — 시험 출제 포인트
| 종류 | 데이터 | 형식 |
|------|-------|------|
| **Continual Pre-training** | 삼성전자 ⭐ 내부 문서 등 ⭐ **unsupervised 문서** | ⭐ Pre-training과 동일 (next token) |
| **Instruction Tuning (SFT)** | ⭐ Task의 input/output 페어 | ⭐ Inference 형식과 같이 |

### 3.3 ⭐ 새 트렌드 — Reasoning
- ⭐ ⭐ Output이 한 방에 안 나오고 ⭐ **긴 보고서 / Reasoning chain**으로 바뀜.
- → ⭐ "긴 리즈닝 잘하게 LLM을 post-train" = 요새 핫.

## 4. ⭐⭐ DeepSeek 족보 + MoE

### 4.1 ⭐⭐ DeepSeek 족보 — 시험 출제 포인트
- ⭐ ⭐ ⭐ R1은 작다는 평가에 가려지지만 ⭐ **V3라는 큰 베이스 모델**이 ⭐ 든든한 ⭐ **티처 모델**.
- ⭐ ⭐ 강력한 티처 모델 없이는 ⭐ **R1을 만들기 어려움**.

### 4.2 ⭐⭐ MoE (Mixture of Experts)
- ⭐ ⭐ 어댑터 / 로라 / Forgetting 맥락:
  - ⭐ 새 테스크 학습 시 ⭐ 전체 파라미터 스페이스가 변하면 → ⭐ 원래 테스크 영향.
  - ⭐ ⭐ 지역 메모리 두고 ⭐ **그 영역만 업데이트** = MoE 아이디어.
- ⭐ ⭐ ⭐ **뇌 영역 비유** — 논리 / 언어 / 수학을 ⭐ 영역 분리하듯 ⭐ Expert 분리.

### 4.3 ⭐ Router + Top-K
- ⭐ Router가 ⭐ **어느 Expert를 깰지** 결정.
- ⭐ 보통 ⭐ **Top-1 / Top-2** 백업으로 같이 고려.
- ⭐ 강사 평: ⭐ MoE는 ⭐ **자원 부족 → 궁여지책**에 가까웠는데 핫해짐.
- ⭐ 1~2년 전엔 Expert 몇 개 → 요즘 ⭐ **수백 개**로 효율화 연구.

## 6. ⭐⭐ RL 기초 + 비유 모음 — 시험 출제 포인트

### 6.1 ⭐⭐⭐ 강아지 훈련 비유
- ⭐ ⭐ ⭐ RL은 ⭐ **강아지 훈련을 본떠 만들어진 것**.
- ⭐ 캣넬에 넣고 싶다 → ⭐ 스낵 넣어줘서 "들어가면 좋은 일" 학습 → ⭐ 다음엔 문만 열어도 들어감.
- ⭐ 소파에 올라가면 안 됨 → 강하게 밀침 → ⭐ 올라가지 않는 곳으로 배움.
- ⭐ ⭐ ⭐ 룰 베이스 / 훈련 데이터 / RL의 ⭐ **무제한 공급 환경** 비교.

### 6.2 ⭐⭐ 주식시장 비유 — 시험 출제 포인트
| 개념 | 주식 비유 |
|------|---------|
| **State** | 시장 상황, 호가창 |
| **Action** | 매도 / 매수 / 홀드 |
| **Reward** | ⭐ 익절 / 손절 |
| **Policy (π)** | "8시면 사고, 종가에 판다" 식 함수 |
| **Value** | 증권사가 알려주는 ⭐ **목표가** = ⭐ 현재 state의 ⭐ 미래 기대 |

- ⭐ "이거 사면 바보" 신문 헤드라인 = 즉각 reward 신호 → ⭐ 가치 활용.
- ⭐ 안 팔아봐야 진짜 reward 모름 (10년 후 익절일 수도) → ⭐ ⭐ **Value 신호로 대체**.

### 6.3 ⭐⭐ 바둑 / 알파고 비유
- ⭐ ⭐ ⭐ 바둑 = 정말 정말 ⭐ **long-horizon reward** — 끝에 이기면 1, 지면 0.
- ⭐ 4시간 후 몇천 개 액션 후에야 reward.
- ⭐ ⭐ State = 흑돌/백돌/빈칸 / Action = 좌표에 놓기 / Action 스페이스 ⭐ **엄청 큼**.
- ⭐ ⭐ 강사 본인이 알파고 보면서 ⭐ **"지금 이세돌이 이기고 있냐?"** = ⭐ **Value 궁금** — 직감적으로 RL 이해.

### 6.4 ⭐⭐ Sparse Reward & Long-Horizon
- ⭐ 강아지 소파 = 즉시 음수 reward → ⭐ 쉬운 RL.
- ⭐ ⭐ ⭐ 보통 reward는 ⭐ **롤아웃 끝나야** 나옴 = ⭐ **Sparse Reward**.
- ⭐ 과자 먹을지 말지 — 즉시 reward는 "맛있다", ⭐ 장기 reward는 "체중 유지" — 한참 후에야 알 수 있음.
- ⭐ ⭐ ⭐ Value를 잘 잡으면 ⭐ **즉각적 reward 신호처럼 활용** 가능.

### 6.5 ⭐⭐ LLM은 Policy다 — 시험 출제 포인트
- ⭐ ⭐ ⭐ **바둑판 = 여태까지 생성한 토큰들**.
- ⭐ ⭐ ⭐ **다음 토큰 분포 생성 = Action**.
- ⭐ ⭐ "Shanghai is in" → "China" 출력 시 ⭐ **1, 0 reward**.
- ⭐ 코드 생성: 다 만들고 ⭐ 테스트 통과해야 reward = ⭐ Long-horizon.

## 7. ⭐⭐ Credit Assignment + 정신승리

### 7.1 ⭐⭐⭐ Credit Assignment — 시험 출제 포인트
- ⭐ ⭐ ⭐ Reward가 멀리 있을 때 ⭐ **이때 이 액션에 의미가 있었던 것 같아** 라고 해석.
- ⭐ ⭐ ⭐ **잘 안 됨** — 훈련 시도는 하지만 언제나 잘 안 됨.

### 7.2 ⭐⭐ 자서전 / 정신승리 비유
- ⭐ ⭐ ⭐ "어떤 사람이 일생 다 살고 ⭐ **꽤 성공적이었던 것 같아**" → 막 설명하면서 자서전 쓰는 것 = ⭐ Credit Assignment.
- ⭐ Reward가 sparse → 사람은 ⭐ 서사를 ⭐ **설명해야** 함 → ⭐ "그때 삼성전자 AI 엑스퍼트 들은 것이 결정적이었다" → 그 액션에 ⭐ **의미 부여**.
- ⭐ ⭐ ⭐ 사실은 ⭐ **causation 아니고 correlation** — AB 테스트해야 진짜 아는 것.
- ⭐ ⭐ ⭐ "지금 엄청 잘하고 있어, 이렇게 롤아웃될 거야" 라고 ⭐ **정신승리**해야 함 — 그게 ⭐ 강아지 훈련 / 인생 / RL의 본질.

### 7.3 ⭐ 강사 어록
- ⭐ ⭐ ⭐ **"강아지가 없지만 강영욱(=강형욱)의 모든 프로를 보고 있고 / 애가 없지만 오은영의 모든 프로를 보고 있다"** — 본인이 RL을 그렇게 이해.

## 8. ⭐⭐ PPO / GRPO / DPO — 시험 출제 포인트

### 8.1 ⭐ 세 요소
- ⭐ ⭐ 원래 Policy (Reference) — ⭐ **버리면 안 됨**.
- ⭐ ⭐ Reward Model.
- ⭐ ⭐ Value Model (PPO에서).

### 8.2 ⭐⭐ Forgetting 막기
- ⭐ ⭐ ⭐ Reward 따라 ⭐ **홀딱홀딱 policy 바꾸면** 죽도 밥도 안 됨.
- ⭐ ⭐ ⭐ 강사 주식 비유: ⭐ "개장에 샀다 물려서 다음엔 개장에 팔거야" → ⭐ 왔다 갔다 ⭐ 탕진.
- ⭐ ⭐ 처방: ⭐ **온고지신** — 새 거 배우되 원래 너무 변하지 않게.
- ⭐ Reference KL divergence + Clipping이 그 역할.

### 8.3 ⭐⭐⭐ PPO → GRPO 비교 — 시험 출제 포인트
| 항목 | PPO | GRPO |
|------|-----|------|
| **Value Model** | ⭐ ⭐ ⭐ **별도 러닝** | ⭐ ⭐ ⭐ **제거** |
| **Advantage** | Value 기반 | ⭐ ⭐ **여러 샘플 평균을 Value로 침** |
| **취지** | 리워드 분산 줄임 | Value learning이 ⭐ **방해** → 빼버림 |

- ⭐ ⭐ ⭐ GRPO 아이디어: ⭐ **여러 번 생성한 reward의 ⭐ 중간(평균)을 현재 Value로** 침.
- ⭐ ⭐ ⭐ 평균보다 높으면 잘했다 / 아니면 못했다.
- ⭐ Value function 학습 ⭐ **불필요** + 결과도 좋음.

### 8.4 ⭐⭐ DPO — Preference 기반
- ⭐ ⭐ "옷 고르는 취향을 ⭐ 점수로 매기는 모델은 어렵지만"
- ⭐ ⭐ ⭐ "이 옷이 저 옷보다 좋아 / B는 D보다 좋아 / D는 E보다 좋아" — ⭐ 페어와이즈 비교는 ⭐ **쉽다**.
- ⭐ ⭐ ⭐ Reward Model을 ⭐ **걔로 대체** = DPO.
- ⭐ 그런 데이터는 ⭐ 널려 있음.

### 8.5 ⭐⭐ 강사 평가 — 시험 출제 포인트
- ⭐ ⭐ ⭐ Value 예측은 ⭐ **항상 어려움** — "은퇴할 때 석학이 되길 원한다면 지금 얼마나 잘하고 있나? 몇 십 년 후에야 알 수 있는 것."
- ⭐ ⭐ ⭐ → Value 빼는 게 ⭐ **결과적으로 좋은 시도**였음.
- ⭐ ⭐ 반면 ⭐ **코드 / 추론**은 reward 쉬움 (컴파일됨/안됨) → Value 빼고 다른 데 노력.

## 9. ⭐⭐ R1의 간단한 Reward — 시험 출제 포인트

### 9.1 ⭐ Reward 구성
- ⭐ ⭐ ⭐ R1의 reward는 ⭐ **놀랍게 간단**:
  - ⭐ 수학 답 맞음 = ⭐ 1.
  - ⭐ ⭐ 그게 없을 수도 있으니 ⭐ **포맷만 주면 1점** — "생각하면서 답하면" 즉시 1점.
- ⭐ ⭐ 이렇게 ⭐ **즉시 즉시 reward** 가능 → 학습 ⭐ 잘 진행.

### 9.2 ⭐⭐⭐ 결과의 놀라움 — 시험 출제 포인트
- ⭐ ⭐ ⭐ 답을 ⭐ **단답형으로 안 주고** 점점 ⭐ **토큰 많이 쓰면서 되짚어 봄**.
- ⭐ ⭐ 중간 과정 / 메모 작성 → 성능 ⭐ 많이 올라감.
- ⭐ ⭐ R1 페이퍼 테이블: ⭐ OpenAI보다 좋더라.
- ⭐ ⭐ ⭐ 진짜로 ⭐ **추론하는 순간** 발견 — "여기 이게 좀 중요한 것 같다" 류.
- ⭐ ⭐ ⭐ 단, ⭐ **체리피킹** — 실제 돌려보면 ⭐ **중국말 막 나오고** 헷갈림.

### 9.3 ⭐ R1 (vs R1-Zero) 정리
- ⭐ ⭐ R1-Zero = ⭐ **이런 것도 가능하다** 보임 (SFT를 RL로 치환).
- ⭐ ⭐ R1 = ⭐ **데이터 사용**, ⭐ Rejection Sampling, ⭐ 사람 데이터, ⭐ 디스틸레이션.
- ⭐ ⭐ ⭐ Distillation 데이터를 ⭐ **다른 모델의 SFT에도 사용 가능** 보여줌.

## 10. ⭐ 조교 RL 노트북 — 액션의 세 가지 단위

### 10.1 ⭐⭐ 세 가지 관점 — 시험 출제 포인트
| 관점 | 액션 단위 | 등장 시기 |
|------|---------|---------|
| **Token Level** | 토큰 하나 | ⭐ 가장 먼저 / 익숙 |
| **Contextual Bandit** | 한 번 답함 | ⭐ Single-step / Multi-step |
| **Tool Use** | 툴 호출 1번 | ⭐ 최근 |

### 10.2 ⭐⭐ BLEU + REINFORCE 시절 — 토큰 레벨 액션
- ⭐ ⭐ 옛날 번역은 ⭐ **Supervised**: 소스 → 타겟 페어 보여줘서 정확히 따라가게.
- ⭐ ⭐ ⭐ 불만: ⭐ **좋은 번역은 다양**한데 ⭐ 하나만 따라가도록 학습됨.
- ⭐ ⭐ 그래서 RL처럼 학습 시도:
  - ⭐ Action = 토큰 하나 생성.
  - ⭐ State = 지금까지 만든 prefix.
  - ⭐ Reward = ⭐ **BLEU** (참조 번역과의 n-gram precision).
- ⭐ Reinforce 알고리즘 = Policy Gradient 그대로 적용.

### 10.3 ⭐⭐⭐ Sparse Reward 문제 — 시험 출제 포인트
- ⭐ ⭐ ⭐ Atari 벽돌깨기 = 부술 때마다 reward → ⭐ Dense.
- ⭐ ⭐ ⭐ 번역 = 마지막 끝나야만 점수 → ⭐ Sparse + Long.
- ⭐ ⭐ ⭐ 강사 비유: ⭐ **"대학원 들어왔는데 교수님이 아무 피드백 안 주고 1년에 한 번씩 평가만 한다"** = 어떻게 배우나?
- ⭐ ⭐ ⭐ 중간 시그널 있어야 ⭐ **배우기 좋음**.

## 11. ⭐⭐ Reward Shaping — 시험 출제 포인트

### 11.1 ⭐⭐ 자동차 시뮬레이터 비유
- ⭐ ⭐ ⭐ 굽이친 도로 자동차 시뮬:
  - ⭐ 완주 시간 = 진짜 reward.
  - ⭐ 바깥으로 나가면 ⭐ **큰 음수 reward** (죽음).
- ⭐ ⭐ ⭐ 학습 어려우면 ⭐ **shaped reward**:
  - ⭐ ⭐ 가운데 잘 가면 ⭐ +0.1.
  - ⭐ ⭐ 오른쪽 거의 붙었는데 더 꺾으려 하면 ⭐ −0.1.
- ⭐ ⭐ **미시적 가이던스** = ⭐ Reward Shaping.

### 11.2 ⭐⭐⭐ Reward Hacking — 시험 출제 포인트
- ⭐ ⭐ ⭐ ⭐ Shaped Reward 함부로 주면 ⭐ **Optimal Policy 변경 위험**.
- ⭐ ⭐ ⭐ ⭐ 비유: ⭐ **부모님이 "책상 3시간 앉으면 +0.1"** 주면 → 자녀는 ⭐ **공부 안 하고 앉기만** → 시험 점수 망함.
- ⭐ ⭐ Reward Hacking과 연결.

### 11.3 ⭐⭐⭐ Potential-based Reward Shaping — 시험 출제 포인트
- ⭐ ⭐ ⭐ 물리 시간 위치 에너지 / ⭐ 태양-지구 ⭐ 무한 원점에서 0 / 가까울수록 ⭐ 음수.
- ⭐ ⭐ ⭐ **포텐셜은 state에만 의존**.
- ⭐ ⭐ ⭐ Shaped reward = ⭐ Φ(s') − Φ(s) (다음 - 현재 포텐셜 차).
- ⭐ ⭐ ⭐ ⭐ **장점: Optimal Policy 보존** — 어디로 가는지는 안 바뀜.
- ⭐ ⭐ 유클리드 미로 + ⭐ A* 휴리스틱 (목적지까지 남은 거리)과 유사 개념.

### 11.4 ⭐⭐ BLEU의 Potential
- ⭐ ⭐ ⭐ BLEU는 ⭐ 완성된 번역에만 정의되는 게 원래지만 ⭐ **n-gram precision이라 prefix만으로도 점수 가능** (덜 완성된 거 그냥 완성된 것처럼).
- ⭐ ⭐ ⭐ 그걸 ⭐ **Potential**로 정의 → ⭐ state에만 의존 OK.
- ⭐ ⭐ ⭐ **Telescoping Sum** — 중간 reward 다 더하면 ⭐ **마지막 BLEU − 처음 BLEU** = ⭐ **0**부터 시작이니 ⭐ **최종 BLEU**.
- ⭐ 결국 같은 최종 BLEU에 ⭐ **중간 시그널만 더 줌**.

### 11.5 ⭐⭐ 토큰 레벨 RL의 한계 — 시험 출제 포인트
- ⭐ ⭐ ⭐ **State Space 폭발** — 모든 prefix가 다른 state → ⭐ **exponential**.
- ⭐ ⭐ ⭐ **Action Space ⭐ vocabulary 크기만큼**.
- ⭐ ⭐ ⭐ ⭐ 사실 ⭐ **대부분은 말이 안 되는 액션** — "I'm easy" 식.
- ⭐ ⭐ 가능한 action 중 ⭐ **불가능한 게 대부분** → 학습 어려움.

## 12. ⭐⭐ 출석 결석 토이 예시 — Outcome vs Process Reward

### 12.1 ⭐⭐ 설정 — 시험 출제 포인트
- ⭐ ⭐ ⭐ 학부 수업의 ⭐ **실제 attendance policy**:
  - ⭐ ⭐ **3번까지는 무료** (이유 불문 빠짐 OK).
  - ⭐ ⭐ **4번째부터 −1점**.
  - ⭐ ⭐ **cap −5** (무한히 안 내려감).

### 12.2 ⭐⭐ Outcome / Terminal / Trajectory Reward
- ⭐ 학기 끝에야 ⭐ **총 결석 수 → 점수**.
- ⭐ = ⭐ Outcome Reward = ⭐ Terminal Reward.

### 12.3 ⭐⭐ Process Reward / Dense / Shaped — 시험 출제 포인트
- ⭐ ⭐ ⭐ 사실 사람은 ⭐ **미리 reward 줄 수 있음**:
  - ⭐ 3번째까지는 ⭐ **0 reward**.
  - ⭐ 4번째부터는 ⭐ **−1**.
- ⭐ ⭐ ⭐ 이게 ⭐ **Shaped Reward** / Dense / Process Reward.
- ⭐ ⭐ Process는 보통 ⭐ 수학 풀이 중간 단계 채점처럼 ⭐ **중간 단계 ⭐ 잘/틀**.

## 13. ⭐⭐ Reward Model의 Credit Assignment — 시험 출제 포인트

### 13.1 ⭐ Reward Model 학습
- ⭐ ⭐ ⭐ Outcome Label만 있어도 ⭐ **Process Reward 같은 거 돌려주는 RM 학습 가능**.
- ⭐ 트랜스포머 기반 RM의 ⭐ **Parameterization 트릭**:
  - ⭐ 각 토큰마다 점수 → ⭐ **다 더해서 totally**.
  - ⭐ 70점 레이블이면 ⭐ 10, 20, 40 식 합이 70 되도록.

### 13.2 ⭐⭐ 출석 토이로 시연
- ⭐ ⭐ ⭐ 6번 결석 (true outcome = -3) → ⭐ Trained RM은 ⭐ **각 결석에 점수 분배**.
  - ⭐ 출석 = 작은 positive.
  - ⭐ 첫 결석 = 모더레이트 페널티.
  - ⭐ 세 번째 결석 = ⭐ **오히려 plus**.
  - ⭐ 네 번째부터 = ⭐ **거의 −1**.
- ⭐ ⭐ → RM이 ⭐ 실제 규칙을 ⭐ **거의 리커버**!

### 13.3 ⭐⭐⭐ Credit Assignment의 진짜 의미 — 시험 출제 포인트
- ⭐ ⭐ ⭐ 학기 끝나고 학생 입장에서 보면:
  - ⭐ 6번 결석 = 총 −3.
  - ⭐ ⭐ 진짜로 ⭐ **앞 3번은 무료 / 뒤 3번이 −1씩**이 진짜 설명일까?
  - ⭐ ⭐ ⭐ ⭐ 또는 ⭐ **모든 결석이 균등하게 −0.5씩 기여**?
- ⭐ ⭐ ⭐ ⭐ 이게 ⭐ **Credit Assignment 의미의 모호함**.
- ⭐ ⭐ ⭐ RM은 ⭐ **prefix만 보면서** 점수 주니까 ⭐ **앞으로 뭐가 일어날지 모름** → 균등 분배 식으로 학습.
- ⭐ ⭐ ⭐ ⭐ 실제 의미는 ⭐ **"이 결석 봤을 때 향후 기대값이 얼마나 움직였는지"** = ⭐ retrospect 컨트리뷰션과 다름.

## 14. ⭐ 툴 유즈 액션 — Search Aron 토이

### 14.1 ⭐ 세팅
- ⭐ ⭐ Toy 환경: 질문 ⭐ 3종류:
  1. ⭐ A가 명백한 경우.
  2. ⭐ B가 명백한 경우.
  3. ⭐ ⭐ **Ambiguous** — 검색해야 정답 알 수 있음.

### 14.2 ⭐ Reward 구성

- ⭐ 답 틀림 = 큰 페널티.
- ⭐ ⭐ Search = ⭐ **−0.15** (작은 비용).
- ⭐ Optimal: ⭐ 명백하면 ⭐ **바로 답**, Ambiguous면 ⭐ **한 번만 Search**.

### 14.3 ⭐ 결과
- ⭐ ⭐ 작은 세팅이라 ⭐ **금방 Optimal로 수렴**.
- ⭐ ⭐ Outcome reward만 줘도 ⭐ **언제 툴 호출할지 잘 배움**.

### 14.4 ⭐ 실세계
- ⭐ ⭐ 1B 정도 작은 모델은:
  - ⭐ ⭐ 언제 검색해야 할지 ⭐ **결정 못함**.
  - ⭐ ⭐ ⭐ **어떤 쿼리 던질지** 모름.
- ⭐ ⭐ 이런 거 ⭐ RL로 학습 → ⭐ 검색 호출 패턴 개선.
- ⭐ ⭐ 다양한 기법:
  - ⭐ ⭐ 결과만 알려주기.
  - ⭐ ⭐ ⭐ ⭐ "이 문서는 중요한 문서야" — 검색 시 그 문서 받아오면 ⭐ positive reward.

### 17.3 ⭐ 비유 / 직관 모음

| 개념 | 비유 / 직관 |
|------|-----------|
| **MoE 영역 분리** | 뇌의 언어/논리/수학 영역처럼 Expert를 분리 |
| **강아지 훈련 = RL** | 캣넬 스낵으로 들어가게 / 소파 밀침으로 안 올라가게 |
| **주식 = RL** | State = 시장 / Action = 매수매도 / Reward = 익절 / Value = 증권사 목표가 |
| **바둑 = RL** | Long-horizon, 4시간 후 0/1, 액션 스페이스 거대 |
| **"이세돌이 이기고 있나?"** | 그게 정확히 Value 개념 |
| **자서전 = Credit Assignment** | 일생 살고 "이 액션이 결정적이었어" 의미 부여, 사실 correlation |
| **정신승리** | 잘 굴러가고 있다고 믿어야 RL이 됨, 인생도 그러함 |
| **온고지신 = KL Constraint** | 새 거 배우되 원래 거 너무 변하지 않게 |
| **주식 팔리시 변동** | 개장에 샀다 물려서 다음엔 개장 판다 → 탕진 |
| **DPO 옷 고르기** | 점수 매기긴 어렵지만 "B가 D보다 좋아"는 쉽다 |
| **Value 예측 어려움** | "은퇴할 때 석학 되려면 지금 잘하고 있나?" 알 수 없음 |
| **R1 추론** | 단답 안 주고 토큰 더 쓰면서 되짚어 봄 |
| **R1 체리피킹** | 중국말 막 나오고 헷갈리는 게 실제 |
| **벽돌깨기 vs 번역** | Dense vs Sparse — 매 부숨 vs 끝에서야 점수 |
| **대학원 방목** | 1년에 한 번씩 평가만 하면 어떻게 배우나 = Sparse |
| **자동차 시뮬레이터** | 가운데 +0.1 / 바깥 −0.1 = Reward Shaping |
| **부모님 책상 3시간** | Reward Hacking — 공부 안 하고 앉기만 |
| **포텐셜 = 위치 에너지** | 무한 원점 0 / 가까울수록 음수 |
| **A* 휴리스틱** | 목적지 남은 거리, Potential과 유사 |
| **Telescoping Sum** | 중간 다 캔슬되고 끝-처음만 남음 |
| **"I'm easy"** | 가능한 action 대부분이 말 안 됨 |
| **출석 3번 무료** | 학부 실제 정책, Outcome Reward 사례 |
| **6번 결석 = −3** | 균등 −0.5 vs 뒤 3개 −1 — Credit 모호 |
| **Chess Elo 400** | 400 차이 = 90% 승률 ≈ Bradley-Terry |
| **non-transitive** | 가위바위보처럼 A>B>C>A 가능 → Elo 불가 |

### 17.4 ⭐ 강사 어록
- "어제 만들어 주신 스킬들을 합치니까 어긋나는 부분이 있는데, 인스트럭션을 잘 줘도 모델이 놓치니 ⭐ Validation 루프가 구조적으로 필요하다."
- "에이전트는 백 번 듣는 것보다 ⭐ 자기 일을 한 번 오토메이트해 보는 게 진짜 배움이다."
- "딥식 페이퍼가 나왔을 때 미국에서 연구년 중이었는데, 미국 회사들이 다 같이 읽고 구현해 보던 그날을 너무나 생생하게 기억한다."
- "네이버가 탈락한 이유는 Open Weight을 일부 썼다는 게 이슈가 됐고, 업스테이지는 ⭐ 웨이트 움직인 과정을 다 가지고 있다고 해명했다."
- "반도체 수출 제약이 있는 중국이 만든 LLM이 어느 정도 경쟁력을 갖춘다는 걸 보여준 모먼트였다."
- "R1을 이해하려면 ⭐ R1-Zero가 먼저고, R1-Zero는 출시한 모델이 아니라 ⭐ thought experiment에 가깝다."
- "RL은 정확히 ⭐ 강아지 훈련을 본떠 만든 것 — 잘하면 간식, 못하면 밀침."
- "오은영의 모든 프로를 보고 있는 강아지가 없지만, 알고리즘으로 본뜬다."
- "주식시장이 RL이라면, ⭐ 증권사 목표가가 Value, ⭐ 매도/매수/홀드가 Action."
- "바둑판이 ⭐ LLM에서는 여태까지 생성한 토큰들이고, ⭐ 다음 토큰 분포 = Action, ⭐ 정답 = Reward."
- "Reward 따라 홀딱홀딱 policy 바꾸면 ⭐ 죽도 밥도 안 된다 — 마치 내 주식 팔리시 같다."
- "GRPO 아이디어 — ⭐ 여러 번 생성한 reward의 평균을 현재 Value라 치고, ⭐ Value model 자체를 안 배운다."
- "DPO — ⭐ 옷에 점수 매기긴 어렵지만 '이 옷이 저 옷보다 좋다'는 쉽다."
- "Value 예측은 항상 어렵다 — ⭐ 은퇴할 때 석학이 되길 원한다면 지금 얼마나 잘하고 있나 어떻게 알겠나."
- "R1은 ⭐ 포맷만 줘도 1점 줬다 — 그렇게 간단한 reward로도 토큰 길게 쓰면서 성능 올라간다."
- "R1 페이퍼 테이블에 나온 추론 순간은 ⭐ 체리피킹이다 — 실제 돌리면 중국말 막 나온다."
- "Atari 벽돌깨기처럼 매번 부술 때마다 reward 주면 좋겠는데, 번역은 ⭐ 마지막 끝나야 알 수 있다."
- "Reward Shaping을 잘못 주면 ⭐ 부모님이 책상 3시간 앉아 있으면 +0.1 줘서 공부 안 하고 망하는 식이다."
- "Potential-based Reward Shaping은 ⭐ Optimal Policy를 변경하지 않는다 — 어디로 가는지는 보존."
- "BLEU + REINFORCE는 이론상 Potential-based shaping이 좋은데 ⭐ 실제로 별 게인 없었다 — 초반 다양성 억제."
- "Credit Assignment의 의미는 단순한 retrospect 분배가 아니라 ⭐ 향후 기대값이 얼마나 움직였느냐다."
- "1빌리언 모델은 ⭐ 언제 검색을 호출해야 하는지도 모르고 어떤 쿼리를 던질지도 모른다."
- "MCTS / Continuation 기반이 ⭐ 다음 SQL 실습에서 더 깊게 다뤄질 것이다."
