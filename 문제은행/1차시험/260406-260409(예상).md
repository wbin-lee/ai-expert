# 예상 문제지

## 예상문제 1. CNN 출력 크기 및 파라미터 수 계산 (260408 오전 예상 / 계산형)

다음과 같은 CNN 모델에 MNIST 이미지를 입력한다.

```
● 입력 이미지 크기 : 1 × 28 × 28 (채널=1, 28×28 픽셀)
● Conv2D : 필터 32개, 커널 3×3, stride=1, padding=0
● MaxPool2D : 커널 2×2, stride=2
● Conv2D : 필터 64개, 커널 3×3, stride=1, padding=0
● MaxPool2D : 커널 2×2, stride=2
● Flatten 후 FC → 10개 클래스 출력
```

다음을 순서대로 계산하시오. (출력 크기는 정수부 `⌊ ⌋` 로 내림한다.)

1. 첫 번째 Conv2D 직후의 출력 텐서의 공간 크기(H×W)
2. 첫 번째 MaxPool2D 직후의 출력 텐서의 공간 크기(H×W)
3. 두 번째 Conv2D 직후의 출력 텐서의 공간 크기(H×W)
4. 두 번째 MaxPool2D 직후의 출력 텐서의 공간 크기(H×W)
5. Flatten 직후의 벡터 차원 수 (채널 × H × W)
6. 전체 FC 계층의 파라미터 수(가중치 + 바이어스). 단, 직전 Flatten 차원과 10개 출력에 대한 fully-connected 계산이다.

### 정답

출력 크기 공식 : `Output = ⌊(Input − Kernel) / Stride⌋ + 1`

1. Conv2D(3×3, stride=1, padding=0) : (28 − 3)/1 + 1 = **26 × 26** (채널 32)
2. MaxPool2D(2×2, stride=2) : (26 − 2)/2 + 1 = **13 × 13** (채널 32)
3. Conv2D(3×3, stride=1, padding=0) : (13 − 3)/1 + 1 = **11 × 11** (채널 64)
4. MaxPool2D(2×2, stride=2) : (11 − 2)/2 + 1 = **5 × 5** (채널 64) (11/2 내림 = 5)
5. Flatten 차원 = 64 × 5 × 5 = **1600**
6. FC 파라미터 수 = (입력 차원 × 출력 차원) + bias = 1600 × 10 + 10 = **16,010**

채점 포인트
```
● 출력 크기 공식 `⌊(I − K)/S⌋ + 1` 적용 ✔
● 각 레이어의 채널 수와 공간 크기 정확 ✔
● Flatten 차원 (채널 × H × W) ✔
● FC 파라미터 수 (weight + bias) ✔
```

## 예상문제 2. RNN의 한계와 LSTM 게이트 구조 (260408 오전 예상 / 구조·개념형)

강의에서는 시퀀스 데이터를 처리하는 모델로 RNN과 그 개선 모델인 LSTM을 소개하였다.

1. 기본 RNN이 긴 시퀀스를 처리할 때 발생하는 대표적인 학습 문제를 1개 쓰고, 왜 발생하는지 한 줄로 설명하시오.
2. LSTM이 도입한 3개의 게이트(gate) 이름을 모두 쓰고, 각 게이트가 수행하는 역할을 한 줄씩 쓰시오.
3. LSTM이 기본 RNN 대비 갖는 이점을 "장기 의존성(long-term dependency)" 개념과 연결하여 1~2줄로 서술하시오.

### 정답

1. **기울기 소실(Vanishing Gradient) 문제**
   - 시간 방향으로 체인 룰이 반복적으로 곱해지면서, 시퀀스가 길어질수록 초기 시점의 그래디언트가 0에 가깝게 수렴하여 앞쪽 입력의 정보가 학습에 반영되지 않기 때문이다.
   - (또는 정보 병목(Information Bottleneck) 문제 : 고정 크기 은닉 상태에 모든 과거 정보를 압축해야 하므로 초기 정보가 소실됨)

2. LSTM의 3개 게이트
   - **Forget Gate (망각 게이트)** : 이전 셀 상태(cell state)에서 어떤 정보를 버릴지 결정한다.
   - **Input Gate (입력 게이트)** : 현재 입력에서 어떤 새로운 정보를 셀 상태에 저장할지 결정한다.
   - **Output Gate (출력 게이트)** : 셀 상태로부터 어떤 정보를 은닉 상태(출력)로 내보낼지 결정한다.

3. LSTM은 셀 상태(cell state)를 별도로 두고 게이트로 정보의 흐름을 조절함으로써, 중요한 정보를 오랜 시점에 걸쳐 유지하고 불필요한 정보는 잊을 수 있다. 따라서 기본 RNN이 다루지 못하는 **장기 의존성**을 효과적으로 학습할 수 있다.

채점 포인트
```
● 기울기 소실(또는 정보 병목) 언급 ✔
● Forget / Input / Output 게이트 이름 정확히 ✔
● 각 게이트 기능 1줄 이상 설명 ✔
● 장기 의존성 해결 언급 ✔
```

## 예상문제 3. Scaled Dot-Product Attention 계산 (260409 오전 예상 / 계산형)

Transformer의 핵심 연산인 Scaled Dot-Product Attention은 다음과 같이 정의된다.

```
Attention(Q, K, V) = softmax( Q·Kᵀ / √d_k ) · V
```

한 개의 쿼리(Query) 벡터와 2개의 키/값(Key, Value) 벡터가 다음과 같이 주어졌다. (키와 값의 차원 `d_k = 2`)

```
● Q  = [1, 0]
● K₁ = [1, 0],   V₁ = [1, 2]
● K₂ = [0, 1],   V₂ = [3, 4]
```

다음을 순서대로 구하시오. (softmax 계산 시 `e ≈ 2.718` 을 사용하고, 최종 값은 소수 둘째 자리까지 반올림하여 쓰시오.)

1. Q·K₁ 및 Q·K₂ 의 내적 점수
2. √d_k 로 나눈 후의 스케일된 점수
3. 두 점수에 대한 softmax 가중치 α₁, α₂
4. 최종 Attention 출력 벡터 (= α₁·V₁ + α₂·V₂)
5. 위 식에서 √d_k 로 나누어 "스케일"하는 이유를 한 줄로 서술하시오.

### 정답

1. 내적 점수
   - Q·K₁ = 1·1 + 0·0 = **1**
   - Q·K₂ = 1·0 + 0·1 = **0**

2. 스케일된 점수 (√2 ≈ 1.414 로 나눔)
   - s₁ = 1 / √2 ≈ **0.707**
   - s₂ = 0 / √2 = **0.000**

3. Softmax 가중치
   - e^0.707 ≈ 2.028, e^0 = 1
   - 합 = 2.028 + 1 = 3.028
   - α₁ = 2.028 / 3.028 ≈ **0.67**
   - α₂ = 1.000 / 3.028 ≈ **0.33**

4. Attention 출력
   - output = 0.67 × [1, 2] + 0.33 × [3, 4]
   - = [0.67, 1.34] + [0.99, 1.32]
   - ≈ **[1.66, 2.66]**

5. **√d_k 로 나누는 이유** : 차원 d_k가 커질수록 내적 Q·Kᵀ의 분산이 커져 softmax 입력값이 매우 커지면, 출력 분포가 한쪽으로 지나치게 치우쳐(saturate) 기울기가 소실된다. 이를 방지하기 위해 스케일링하여 softmax의 분포를 안정화한다.

채점 포인트
```
● Q·K 내적 계산 정확 ✔
● √d_k 스케일링 적용 ✔
● Softmax 가중치 합 = 1 만족, 계산 정확 ✔
● 최종 출력 벡터 (가중합) 정확 ✔
● 스케일링의 목적(분산·gradient saturation 방지) 언급 ✔
```

## 예상문제 4. BERT vs GPT 구조·학습 비교와 In-Context Learning (260409 오후 예상 / 비교·개념형)

대표적인 사전학습 언어모델인 **BERT**와 **GPT**는 모두 Transformer를 기반으로 하지만 구조와 학습 방식에 차이가 있다.

1. BERT와 GPT는 각각 Transformer의 어떤 부분(Encoder / Decoder)을 사용하는지 쓰시오.
2. 두 모델의 사전학습 목표(pre-training objective)를 각각 쓰시오.
3. 두 모델의 문맥 파악 방향(Bidirectional / Unidirectional)을 각각 쓰시오.
4. 다음 두 태스크는 BERT와 GPT 중 어느 쪽에 더 적합한지 각각 이유와 함께 쓰시오.
   - (가) 문장 빈칸 채우기 / 문장 분류 / 개체명 인식
   - (나) 주어진 프롬프트를 이어 받아 긴 글을 생성하기
5. GPT-3 이후 LLM에서 강조된 **In-Context Learning (Few-shot Learning)** 의 개념을 한두 줄로 서술하시오.

### 정답

1. 사용하는 구조
   - **BERT** : Transformer **Encoder** 만 사용
   - **GPT** : Transformer **Decoder** 만 사용

2. 사전학습 목표
   - **BERT** : Masked Language Modeling (MLM). 입력 토큰 중 일부를 [MASK]로 가리고, 그 자리의 원래 토큰을 예측하도록 학습한다. (+ 경우에 따라 Next Sentence Prediction)
   - **GPT** : Next Token Prediction (Causal Language Modeling). 이전 토큰들로부터 다음 토큰을 예측하도록 학습한다.

3. 문맥 파악 방향
   - **BERT** : **Bidirectional** (양방향). 좌우 문맥을 동시에 본다.
   - **GPT** : **Unidirectional** (일방향). 왼쪽(과거) 문맥만 본다. 미래 토큰은 마스킹된다.

4. 태스크별 적합성
   - **(가) 빈칸 채우기 / 분류 / 개체명 인식 → BERT가 적합**
     - 이유 : 이 태스크들은 대상 토큰의 좌우 문맥을 모두 활용해야 풍부한 의미 표현을 얻을 수 있고, BERT의 MLM / 양방향 인코더가 이에 최적화되어 있기 때문이다.
   - **(나) 프롬프트 이어받아 긴 글 생성 → GPT가 적합**
     - 이유 : 텍스트 생성은 왼쪽에서 오른쪽으로 다음 토큰을 순차적으로 예측하는 문제이며, GPT의 일방향 디코더가 Next Token Prediction으로 이 문제에 자연스럽게 정합하기 때문이다.

5. **In-Context Learning (Few-shot Learning)**
   - 사전학습된 대규모 언어모델이 **별도의 파라미터 업데이트(Fine-tuning) 없이**, 프롬프트에 주어진 몇 개의 예시(zero/one/few-shot)와 지시만으로 새로운 태스크를 수행할 수 있는 능력을 말한다. 이는 모델의 규모(파라미터·데이터·연산)가 커지면 자연스럽게 나타나는 **창발적(emergent) 능력**으로, Scaling Law와 함께 LLM의 핵심 특성이다.

채점 포인트
```
● Encoder vs Decoder 정확히 구분 ✔
● MLM / Next Token Prediction 목표 정확히 ✔
● Bidirectional / Unidirectional 정확히 ✔
● 태스크별 적합성과 이유 연결 ✔
● In-Context Learning 개념 + fine-tuning 없이 수행 가능 언급 ✔
```
