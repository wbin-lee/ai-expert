# 4. 고확률 예측 문제 4개

## 예측문제 1. Scaled Dot-Product Attention 계산

Transformer의 attention은 다음 식으로 계산된다.

```
Attention(Q, K, V) = softmax(QK^T / sqrt(d_k))V
```

다음 벡터가 주어져 있다.

```
Q  = [1, 0]
K1 = [1, 0],   V1 = [2, 0]
K2 = [0, 1],   V2 = [0, 2]
d_k = 2
```

다음을 계산하시오. 단, `sqrt(2)=1.414`, `e^0.707≈2.03`, `e^0=1`로 계산하고, 최종 답은 소수 둘째 자리까지 쓰시오.

1. `Q·K1`, `Q·K2` 내적 점수
2. `sqrt(d_k)`로 나눈 scaled score
3. softmax attention weight `alpha1`, `alpha2`
4. 최종 Attention 출력 벡터
5. `sqrt(d_k)`로 나누는 이유를 한 줄로 설명하시오.

### 정답

1. 내적 점수

```
Q·K1 = 1 x 1 + 0 x 0 = 1
Q·K2 = 1 x 0 + 0 x 1 = 0
```

2. scaled score

```
s1 = 1 / sqrt(2) = 0.707
s2 = 0 / sqrt(2) = 0
```

3. softmax attention weight

```
alpha1 = e^0.707 / (e^0.707 + e^0)
       = 2.03 / (2.03 + 1)
       = 0.67

alpha2 = e^0 / (e^0.707 + e^0)
       = 1 / (2.03 + 1)
       = 0.33
```

4. 최종 Attention 출력 벡터

```
output = 0.67 x [2,0] + 0.33 x [0,2]
       = [1.34, 0] + [0, 0.66]
       = [1.34, 0.66]
```

5. `sqrt(d_k)`로 나누는 이유

차원이 커지면 `QK^T` 내적값의 크기가 커져 softmax가 한쪽으로 치우칠 수 있다. 이를 완화하고 attention 분포와 gradient를 안정화하기 위해 `sqrt(d_k)`로 나눈다.

채점 포인트

```
● Q와 K의 내적 계산
● sqrt(d_k) scaling 적용
● softmax 확률 계산
● V의 weighted sum 계산
● scaling 이유 설명
```

## 예측문제 2. BERT와 GPT 비교

BERT와 GPT는 모두 Transformer 기반 모델이지만 구조와 학습 방식이 다르다. 다음 물음에 답하시오.

1. BERT와 GPT가 각각 Transformer의 어느 부분을 사용하는지 쓰시오.
2. BERT와 GPT의 사전학습 목표를 각각 쓰시오.
3. BERT와 GPT의 문맥 처리 방향을 각각 쓰시오.
4. 다음 태스크에 더 적합한 모델을 BERT/GPT 중에서 고르고 이유를 쓰시오.
   - 문장 분류 또는 개체명 인식
   - 주어진 문장을 이어서 글 생성
5. GPT 계열 모델에서 말하는 In-Context Learning이 무엇인지 설명하시오.

### 정답

1. 사용하는 Transformer 구조

```
● BERT: Transformer Encoder-only
● GPT: Transformer Decoder-only
```

2. 사전학습 목표

```
● BERT: Masked Language Modeling(MLM). 문장 중 일부 토큰을 가리고 원래 토큰을 예측한다.
● GPT: Next Token Prediction. 앞의 토큰들을 보고 다음 토큰을 예측한다.
```

3. 문맥 처리 방향

```
● BERT: Bidirectional. 앞뒤 문맥을 모두 본다.
● GPT: Unidirectional/Causal. 앞 문맥만 보고 미래 토큰은 보지 않는다.
```

4. 태스크별 적합성

```
● 문장 분류 또는 개체명 인식: BERT
```

BERT는 앞뒤 문맥을 모두 활용해 문장 전체의 의미 표현을 만들기 좋으므로 이해 중심 태스크에 적합하다.

```
● 주어진 문장을 이어서 글 생성: GPT
```

GPT는 이전 토큰을 바탕으로 다음 토큰을 순차적으로 생성하도록 학습되었으므로 생성 태스크에 적합하다.

5. In-Context Learning

모델 파라미터를 fine-tuning하지 않고, 프롬프트 안에 지시문이나 몇 개의 예시를 함께 넣어 새로운 문제를 풀게 하는 방식이다. 예시가 없으면 zero-shot, 하나면 one-shot, 몇 개면 few-shot이라고 부른다.

채점 포인트

```
● Encoder-only / Decoder-only 구분
● MLM / Next Token Prediction 구분
● Bidirectional / Unidirectional 구분
● 이해 태스크와 생성 태스크의 적합성 설명
● fine-tuning 없이 프롬프트 예시로 수행한다는 설명
```

## 예측문제 3. CNN 출력 크기와 Flatten 차원 계산

다음 CNN 모델에 `1 x 28 x 28` 크기의 MNIST 이미지가 입력된다.

```
입력: 1 x 28 x 28
Conv2D: 필터 16개, kernel size 5 x 5, stride=1, padding=0
MaxPool2D: kernel size 2 x 2, stride=2
Conv2D: 필터 32개, kernel size 3 x 3, stride=1, padding=1
MaxPool2D: kernel size 2 x 2, stride=2
Flatten
```

다음을 순서대로 계산하시오.

1. 첫 번째 Conv2D 후 출력 크기
2. 첫 번째 MaxPool2D 후 출력 크기
3. 두 번째 Conv2D 후 출력 크기
4. 두 번째 MaxPool2D 후 출력 크기
5. Flatten 후 벡터 차원

### 정답

CNN 출력 크기 공식은 다음과 같다.

```
H_out = floor((H_in - K + 2P) / S) + 1
```

1. 첫 번째 Conv2D

```
H_out = (28 - 5 + 2 x 0) / 1 + 1 = 24
```

출력 크기: `16 x 24 x 24`

2. 첫 번째 MaxPool2D

```
H_out = (24 - 2) / 2 + 1 = 12
```

출력 크기: `16 x 12 x 12`

3. 두 번째 Conv2D

```
H_out = (12 - 3 + 2 x 1) / 1 + 1 = 12
```

출력 크기: `32 x 12 x 12`

4. 두 번째 MaxPool2D

```
H_out = (12 - 2) / 2 + 1 = 6
```

출력 크기: `32 x 6 x 6`

5. Flatten 차원

```
32 x 6 x 6 = 1152
```

따라서 Flatten 후 벡터 차원은 `1152`이다.

채점 포인트

```
● 출력 크기 공식 적용
● kernel, stride, padding 의미 이해
● pooling 후 크기 계산
● Flatten = 채널 x 높이 x 너비
```

## 예측문제 4. RNN의 한계와 LSTM 게이트

RNN과 LSTM에 대해 다음 물음에 답하시오.

1. RNN이 sequence 데이터를 처리할 수 있는 이유를 hidden state 개념으로 설명하시오.
2. 기본 RNN이 긴 sequence에서 겪는 대표적인 학습 문제를 쓰고, 왜 발생하는지 설명하시오.
3. LSTM의 3가지 gate 이름을 쓰고, 각 역할을 한 줄씩 설명하시오.
4. LSTM 또는 GRU가 기본 RNN보다 긴 context를 더 잘 다룰 수 있는 이유를 설명하시오.

### 정답

1. RNN은 현재 입력 `x_t`와 이전 hidden state `h_{t-1}`를 함께 사용하여 현재 hidden state `h_t`를 계산한다. 따라서 이전 시점의 정보가 hidden state를 통해 다음 시점으로 전달된다.

```
h_t = tanh(W_h h_{t-1} + W_x x_t + b)
```

2. 대표적인 문제는 **기울기 소실(Vanishing Gradient)** 이다.

긴 sequence에서는 시간 방향으로 chain rule이 반복 적용된다. 이때 작은 미분값이 계속 곱해지면 앞쪽 시점까지 전달되는 gradient가 거의 0이 되어 초기 입력 정보가 학습에 잘 반영되지 않는다.

3. LSTM의 3가지 gate

```
● Forget Gate: 이전 cell state에서 어떤 정보를 잊을지 결정한다.
● Input Gate: 현재 입력에서 어떤 정보를 새롭게 저장할지 결정한다.
● Output Gate: cell state의 정보 중 어떤 것을 hidden state로 출력할지 결정한다.
```

4. LSTM은 hidden state와 별도로 cell state를 두고 gate로 정보의 삭제, 저장, 출력을 조절한다. 그래서 중요한 정보를 더 오래 유지할 수 있고, 기본 RNN보다 장기 의존성 문제를 완화할 수 있다. GRU는 LSTM을 더 단순화한 gate 기반 모델로, RNN보다 gradient 흐름을 더 잘 유지하도록 설계되었다.

채점 포인트

```
● hidden state가 과거 정보를 전달한다는 설명
● vanishing gradient와 chain rule 반복 언급
● Forget/Input/Output gate 이름과 역할
● cell state 또는 gate를 통한 장기 정보 유지 설명
```
