## 🚨 출제 확률 최상위 근거 — "반드시 출제" 로 지목된 내용

### ① 요약서에 "시험 반드시 출제" 로 명시된 항목 → **Q15 (Reparameterization Trick)**

`260420(오전)_요약.md` 의 섹션 제목에 다음과 같이 명시되어 있다.

> **## 8. Reparameterization Trick (시험 반드시 출제)**

- 윤성로 교수님이 VAE 수업에서 **"이것은 시험에 반드시 출제된다"** 고 직접 공지한 유일한 항목이다.
- 핵심 포인트: **샘플링은 미분 불가능 → 랜덤성을 외부 입력 ε으로 분리 → `z = μ + σ ⊙ ε` 로 결정론적 계산 치환 → 그래디언트가 인코더 파라미터 φ까지 흐름**.
- 이 트릭은 VAE 뿐만 아니라 Diffusion 등 다른 확률적 생성 모델에서도 반복적으로 등장하는 표준 기법이기에 개념형 출제 확률이 사실상 100%에 가깝다.

### ② 교수님이 수업에서 반복 강조한 "DGZ" → **Q17, Q18 (GAN Generator·Discriminator, Non-saturating Loss)**

`260421(오전)_요약.md` 의 GAN 설명 전반에 걸쳐 교수님이 **"오늘의 핵심은 DGZ"** 라고 여러 차례 반복 강조한 대목이 있다.

- 교수님 어록 발췌 (요약서 기록):
  > *"오늘의 핵심은 DGZ라는 거 기억해 주시고 가격이 DGZ가 될 거다라는 거 꼭 기억을 해 주시고 … DGZ 기억하세요. 그 다음에 이 네 가지 확률만 이해하시면 뒤지지 않으시고 서바이브 하실 수 있어요."*
  > *"DGZ, 기억하세요."* (수업 중 최소 5회 이상 반복)

- **DGZ = D(G(z))** 로, "가짜 샘플을 Discriminator가 진짜라고 판단할 확률" = GAN의 "가격 흥정 대상".
- 교수님이 강조한 네 가지 확률 프레임:

  | 기호 | 의미 |
  |------|------|
  | `D(x)` | 진짜를 진짜라고 하는 확률 |
  | `1 − D(x)` | 진짜를 가짜라고 하는 확률 |
  | **`D(G(z))`** | **가짜를 진짜라고 하는 확률 = "가격"** |
  | `1 − D(G(z))` | 가짜를 가짜라고 하는 확률 (발각 확률) |

- 이 DGZ 프레임을 기반으로 **Q17 (G/D 역할)** 과 **Q18 (Non-saturating loss — `log D(G(z))` 를 최대화)** 이 자연스럽게 파생되므로 두 문제의 출제 확률이 매우 높다.

### ③ 기타 교수님이 수업에서 강조한 반복 포인트

- **Nash Equilibrium에서 `D = 0.5`** — "완벽히 구별할 때가 아니라 **구별을 못할 때**가 수렴점이다" 라고 수업 중 두 번 바로잡음 → 개념 혼동 방지용 출제 가능.
- **Mode Collapse (Q19)** — VAE의 blurry 문제와 대비되는 GAN 고유의 한계로 반복 언급 → "Quality-Diversity Trade-off" 프레임과 함께 출제 단골.
- **"고해성사" 비유** — 원 Minimax → Non-saturating Loss로 바꾼 역사적 배경을 Goodfellow의 "이실직고(고해성사)"로 설명. 비유 자체가 학습 안정성과 gradient 포화 문제의 이해를 돕는 핵심 장치로 강조됨 (Q18 배경).

따라서 **Q15 / Q17 / Q18 세 문제** 는 교수님의 직접 강조 발언에 의해 출제 확률이 사실상 확정적이라고 판단한다.

# 딥러닝 기초 답안

### Q13. [생성형 AI 기초]
생성형 모델(Generative Model)과 판별형 모델(Discriminative Model)의 차이를 설명하시오.

> **모범 답안**
>
> 판별형 모델은 조건부 확률 P(y\|x)를 학습하여 입력 x를 하나의 정답 y로 분류·예측하는 반면, 생성형 모델은 데이터 분포 P(x) 또는 결합분포 P(x, y)를 학습하여 그 분포에서 새로운 샘플을 만들어낼 수 있다. 즉 판별형은 결정 경계를 배우고, 생성형은 데이터의 분포 자체를 배워 **샘플링**이 가능하다는 점이 핵심 차이이다.

### Q14. [VAE (Variational Autoencoder)]
VAE와 일반 Autoencoder(AE)의 구조적 차이를 설명하시오.

> **모범 답안**
>
> 일반 AE의 인코더는 입력 x를 단일 잠재 벡터 z로 결정론적(deterministic) 매핑하여 압축·복원에 집중하는 반면, VAE의 인코더는 잠재 분포 q_φ(z\|x)의 파라미터(μ, σ²)를 출력하고 그 분포에서 z를 샘플링한다. 이로 인해 VAE는 학습 후 디코더만 사용하여 prior에서 뽑은 새로운 z로 다양한 샘플을 생성할 수 있는 **확률적 생성 모델**로 동작한다.

### Q15. [VAE (Variational Autoencoder)]
VAE에서 Reparameterization Trick이 왜 필요한지 설명하시오.

> **모범 답안**
>
> 인코더가 내놓은 분포에서 직접 z를 샘플링하면 샘플링 연산이 미분 불가능하여 역전파로 인코더 파라미터를 학습할 수 없다. 이를 해결하기 위해 `z = μ + σ ⊙ ε (ε ~ N(0, I))` 형태로 무작위성을 외부 입력 노이즈 ε로 분리하여, μ와 σ를 거쳐 그래디언트가 정상적으로 흐르도록 만드는 기법이다.

### Q16. [VAE (Variational Autoencoder)]
VAE의 목적함수인 ELBO(Evidence Lower Bound)를 구성하는 두 항의 역할을 각각 설명하시오.

> **모범 답안**
>
> ELBO는 **Reconstruction 항** `E_{z~q_φ(z|x)}[log p_θ(x|z)]` 과 **KL 항** `−D_KL(q_φ(z|x) ∥ p(z))` 로 구성된다. Reconstruction 항은 샘플링된 z를 디코더에 통과시켰을 때 원본 x가 얼마나 잘 복원되는지를 평가하는 훈련 손실 역할을 하고, KL 항은 인코더 분포를 prior(일반적으로 N(0, I))에 가깝게 끌어당기는 **정규화(regularizer)** 역할을 하여 잠재 공간이 연속적이고 부드럽게 채워지게 한다.

### Q17. [GAN (Generative Adversarial Network)]
GAN에서 Generator와 Discriminator의 역할을 각각 설명하시오.

> **모범 답안**
>
> Generator는 무작위 노이즈 z를 입력받아 진짜처럼 보이는 가짜 샘플 G(z)를 만들어 Discriminator를 속이는 것을 목표로 하고, Discriminator는 입력이 진짜 데이터인지 Generator가 만든 가짜인지를 0~1 확률로 구분하도록 학습한다. 두 네트워크는 Minimax 목적함수 하에서 **적대적(adversarial)**으로 경쟁하며 함께 성능이 향상된다.

### Q18. [GAN (Generative Adversarial Network)]
GAN의 Minimax 목적함수에서 Generator에 대해 `log(1 − D(G(z)))` 를 최소화하는 대신 `log D(G(z))` 를 최대화하는 Non-saturating Loss를 사용하는 이유를 설명하시오.

> **모범 답안**
>
> 학습 초반에는 Generator가 만든 샘플이 조잡하여 Discriminator가 쉽게 구분해 `D(G(z)) ≈ 0` 이 되는데, 이 영역에서 `log(1 − D(G(z)))`의 기울기가 매우 작아(포화되어) Generator에 충분한 학습 신호가 전달되지 않는다. `log D(G(z))`를 최대화하는 형태로 목적을 뒤집으면 같은 영역에서 **기울기가 크게 유지**되어 학습 초반에도 Generator가 안정적으로 업데이트될 수 있기 때문이다.

### Q19. [GAN (Generative Adversarial Network)]
GAN 학습에서 발생하는 Mode Collapse(모드 붕괴) 현상이 무엇인지 설명하시오.

> **모범 답안**
>
> Generator가 데이터 분포의 다양한 모드(유형) 중 일부만 반복적으로 생성하여 **다양성(diversity)이 사라지는** 현상이다. 예를 들어 MNIST 학습 시 0~9 중 특정 숫자만 계속 만들어내는 경우로, Minimax 학습의 불안정성과 Generator가 Discriminator를 속이기 쉬운 모드로 수렴해 버리는 데서 발생한다.

### Q20. [생성 모델 평가]
FID(Fréchet Inception Distance)가 무엇을 측정하는 지표이며 값이 낮을수록 좋은 이유를 설명하시오.

> **모범 답안**
>
> FID는 사전학습된 Inception v3 네트워크의 중간 feature 공간에서 **진짜 이미지 분포와 가짜 이미지 분포를 각각 다변량 Gaussian으로 근사** 한 뒤, 두 분포의 평균 μ와 공분산 Σ 간 Fréchet distance(Wasserstein-2 distance)를 계산한 값이다. 값이 낮을수록 두 분포의 통계량이 가까워 생성 이미지가 진짜 이미지와 feature 수준에서 구분되지 않음을 의미하므로 생성 품질이 좋다고 판단한다.
