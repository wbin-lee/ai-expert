## 1. RNN 모델 파라미터 차원 계산

다음과 같은 조건을 가진 RNN 모델을 구축하고 있습니다:

- 학습에 사용되는 말뭉치에는 총 **12,000개의 고유 단어**가 존재함
- 각 단어는 **one-hot encoding**으로 표현됨
- RNN의 hidden state의 크기(hidden size)는 **100**으로 설정됨
- 출력은 **3개의 감성 클래스 (긍정, 부정, 중립)** 중 하나로 분류됨
- RNN의 수식은 다음과 같은 형태를 따름:

$$
h_t = \tanh(W_{xh} x_t + W_{hh} h_{t-1} + b_h)
$$

$$
y_t = W_{hy} h_t
$$

위 조건을 바탕으로 다음 **파라미터 행렬과 벡터의 차원**을 각각 구하시오. (row, column의 순서는 바뀌어도 됨)

| 파라미터 이름 | 차원 |
|---------------|------|
| $W_{xh}$ |  |
| $W_{hh}$ |  |
| $W_{hy}$ |  |
| $b_h$ |  |

### 정답

> row, column 이 바뀌어도 정답임. 예: 100 × 12000 과 12000 × 100은 동일함

| 파라미터 이름 | 차원 (Dimensionality) | 설명 |
|---------------|------------------------|------|
| $W_{xh}$ | **100 × 12,000** | input $x_t$는 12,000차원 (one-hot), hidden state는 100차원 |
| $W_{hh}$ | **100 × 100** | 이전 hidden state에서 다음 hidden state로 |
| $W_{hy}$ | **3 × 100** | hidden state → 3개 클래스 출력 |
| $b_h$ | **100 × 1 또는 100** | hidden state에 더해지는 bias 벡터 |

## 3. Attention 메커니즘

Attention 메커니즘이 기본 Encoder-Decoder 구조의 한계를 **어떻게 해결**하는지 설명하시오.

### (1) 기본 Encoder-Decoder 구조에서 발생하는 bottleneck 문제에 대해 설명하시오.

### (2) Attention 메커니즘이 디코딩 단계에서 어떻게 진행되는지 구체적으로 설명하시오.

### 정답

**(1)**

기본 Encoder-Decoder 구조에서는 입력 시퀀스 전체 정보를 **고정된 길이의 하나의 벡터(context vector)** 로 압축하여 디코더에 전달함.
입력 문장이 길거나 복잡할 경우, 이 하나의 벡터만으로 모든 정보를 담기 어렵고, 결과적으로 정보 손실이 발생함 → **성능 저하**

**(2)**

Attention은 **입력 시퀀스의 모든 hidden state**를 참조하며, 각 디코딩 스텝마다 현재 디코더 상태와 입력 시퀀스의 각 hidden state 간의 유사도(attention score)를 계산함.
이 유사도를 기반으로 가중합(context vector)을 동적으로 계산하여, 매 스텝마다 중요한 입력 정보를 선택적으로 반영함 → **정보 손실 없이 더 정확한 출력 가능**

## 5. Transformer 인코더 블록 파라미터 개수 계산

다음은 Transformer 모델의 **단일 인코더 블록(encoder block)** 구성입니다:

- 임베딩 차원 (hidden dimension, $d_{\text{model}}$) = **10**
- 어텐션 헤드 수 ($h$) = **2**
- 각 헤드의 query, key, value 차원: $d_q = d_k = d_v = 5$
- **Feedforward network**는 두 개의 선형 계층으로 구성되며,
  - 첫 번째 계층은 차원을 $d_{\text{ff}} = 30$으로 확장하고,
  - 두 번째 계층은 다시 $d_{\text{model}}$ 차원으로 축소함
- 모든 선형 계층에는 **bias가 없음**

이때, 다음 각 항목별로 **파라미터 개수**를 구하시오:

### (1) Self-Attention 블록 내의 파라미터 수

- $W_q$: Query weight matrix (전체)
- $W_k$: Key weight matrix (전체)
- $W_v$: Value weight matrix (전체)
- $W_o$: Output projection matrix

### (2) Feedforward 네트워크의 파라미터 수

- 첫 번째 선형 계층: $\text{Linear}(d_{\text{model}} \rightarrow d_{\text{ff}})$
- 두 번째 선형 계층: $\text{Linear}(d_{\text{ff}} \rightarrow d_{\text{model}})$

### (3) 총 파라미터 수는?

### 정답

| 항목 | 파라미터 수 | 차원 설명 |
|------|--------------|-----------|
| $W_q$ | 100 | $2 \times (5 \times 10)$ |
| $W_k$ | 100 | $2 \times (5 \times 10)$ |
| $W_v$ | 100 | $2 \times (5 \times 10)$ |
| $W_o$ | 100 | $10 \times 10$ |

## 7. Pairwise Preference와 RLHF / DPO

다음은 하나의 입력 $x$에 대해 언어모델이 생성한 세 개의 답변이다.

- $y_1$: 정확하지만 너무 짧고 설명이 부족한 답변
- $y_2$: 정확하고 사용자의 의도에 맞게 친절하게 설명한 답변
- $y_3$: 그럴듯하지만 사실과 다른 내용을 포함한 답변

사람 annotator는 세 답변의 선호도를 다음과 같이 평가하였다.

$$
y_2 > y_1 > y_3
$$

### (1) 이 선호도 순위로부터 만들 수 있는 pairwise preference label을 모두 쓰시오.

### (2) RLHF에서는 이 데이터를 어떻게 활용하는지 Reward Model과 연결하여 설명하시오.

### (3) DPO에서는 별도의 Reward Model 없이 이 데이터를 어떻게 활용하는지 설명하시오.

### 정답

**(1)** 만들 수 있는 pairwise preference label은 다음과 같다.

$$
y_2 > y_1, \quad y_1 > y_3, \quad y_2 > y_3
$$

**(2)** RLHF에서는 위 pairwise preference 데이터를 이용해 Reward Model을 학습한다.
Reward Model은 같은 입력 $x$에 대해 사람이 더 선호한 답변에는 더 높은 reward를, 덜 선호한 답변에는 더 낮은 reward를 주도록 학습된다.

그 후 언어모델은 이 Reward Model의 점수가 높은 답변을 생성하도록 **PPO와 같은 강화학습 방식**으로 최적화된다.

**(3)** DPO에서는 **별도의 Reward Model을 학습하지 않는다.**
대신 선호된 답변 $y^+$의 생성 확률은 높이고, 거부된 답변 $y^-$의 생성 확률은 낮추도록 **언어모델을 직접 학습**한다.

## 8. RLHF의 PPO 최적화 — Reward Model & KL Penalty

RLHF에서 PPO 최적화 시 다음 두 가지 요소가 함께 고려된다.

### (1) Reward Model의 역할은 무엇인지 서술하시오.

### (2) KL divergence penalty를 추가하는 이유를 서술하시오.

### 정답

**(1)**

Reward Model은 **사람이 선호하는 답변일수록 높은 점수를 주도록** 학습되며, 언어모델이 더 좋은 답변을 생성하도록 **학습 방향을 제공**한다.

**(2)**

KL divergence penalty는 학습된 모델이 **기존 사전학습 모델의 분포에서 너무 크게 벗어나지 않도록 제한**한다. 이를 통해 **학습 안정성을 유지**하고 **비정상적인 출력이 생성되는 것을 방지**한다.
