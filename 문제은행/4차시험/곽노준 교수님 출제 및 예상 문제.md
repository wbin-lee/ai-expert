## 1.5 확정문제별 심층 분석

### Q1 — Local / Global + Saliency vs Concept-based

**시험이 검증하는 것**
- 해석 단위가 **한 샘플(Where)** 인지 **모델 전체 컨셉(What)** 인지
- saliency(위치 강조)와 concept-based(인간 이해 가능 개념)의 **설명 깊이 차이**

**답의 뼈대** (PDF 정답 축)

| 축 | Local | Global |
|----|-------|--------|
| **초점** | 개별 입력 사례의 예측 근거 | 모델 전반의 개념·규칙·패턴 |
| **대표 방법** | Grad-CAM, CAM, SHAP | TCAV, ACE, CRAFT |
| **산출** | heatmap, feature contribution | concept sensitivity score |

| | Saliency map | Concept-based |
|--|--------------|---------------|
| **강점** | 시각적·직관적 | 인간 이해 가능 개념 단위 |
| **약점** | **Why** 부족 — 병변인지 마커/촬영조건인지 불명 | (상대적으로) 전반 신뢰·편향 평가에 유리 |

**강의 보강**: “Explanation Can Be Manipulated” — 클래스만 바꿔도 설명이 그럴듯해 보일 수 있음 → 로컬 한계 = Q1 후반의 핵심.

### Q2 — Mechanistic vs Feature Attribution / Saliency XAI

**시험이 검증하는 것**
- 기존 XAI = **입력↔출력** 관계의 외부 관찰
- Mechanistic = **내부 계산 경로**(뉴런, attention head, circuit, layer)

**답의 3축**

1. **일반 XAI**: saliency, Grad-CAM, SHAP → “어디를 봤나 / 어떤 feature가 기여했나”
2. **Mechanistic**: 내부 representation이 **어떤 기능을 하고 어떻게 결합**되어 출력이 되는지 (인과·구조적)
3. **동기·한계**: LLM/ViT 등 복잡 모델의 hallucination·bias·shortcut 분석에 중요하나, 고차원 표현 전면 해석은 현실적으로 어려움

**강의 보강**: SAE(Sparse Autoencoder) — 차원을 늘린 뒤 sparsity로 **컨셉 분리**. 확정 답안은 SAE를 직접 묻지 않으나, Mechanistic 예시로 쓸 수 있다.

### Q3 — Aleatoric vs Epistemic + XAI·신뢰성

**시험이 검증하는 것**
- 불확실성 **원인** 구분 (데이터 vs 모델 지식)
- 불확실성 추정과 XAI가 **서로를 대체하지 않고 보완**한다는 점

| | Aleatoric | Epistemic |
|--|-----------|-----------|
| **원인** | 데이터 noise·겹침·라벨 불일치 | 데이터 부족·분포 미학습 |
| **줄일 수 있나** | 단순 데이터 추가로 **완전 제거 어려움** (irreducible) | **더 많은·다양한 데이터**·모델 개선으로 완화 |
| **예** | 센서 노이즈, 흐림, 가림 | 타 병원 장비 domain shift |

**관계 논술 키워드**: high confidence + Grad-CAM이 **spurious correlation**(로고·가장자리)을 보면 신뢰 불가 → **불확실성 = 얼마나 확신 / XAI = 근거의 타당성**.

### Q4 — Targeted vs Untargeted + Robustness·Interpretability

**시험이 검증하는 것**
- 공격 **목표**와 **loss 방향**
- 공격이 **예측뿐 아니라 설명도 왜곡**할 수 있음

| | Untargeted | Targeted |
|--|------------|----------|
| **목표** | 정답에서만 벗어나면 성공 | **지정 클래스**로 오분류 |
| **loss** | 정답 클래스 loss **증가**(maximize) | 목표 클래스 loss **감소**(minimize) |
| **난이도** | 상대적으로 쉬움 | decision boundary를 **특정 방향**으로 → 더 어려움 |

**관계 논술 키워드**
- Adv. perturbation → saliency/Grad-CAM/SHAP도 왜곡 가능 (같은 gradient·activation 의존)
- XAI로 vulnerability 분석 가능하나 **완전한 방어책 아님**
- 방어: adversarial training, robust feature learning, uncertainty estimation, XAI 검증 **병행**

### Q5 — SmoothQuant vs QuaRot

**시험이 검증하는 것**
- 공통 문제: LLM **activation outlier**
- 해법 철학: **부담 이전(scaling)** vs **에너지 분산(rotation)**

| | SmoothQuant | QuaRot |
|--|-------------|--------|
| **핵심 연산** | diagonal **scaling** \(S\) | **orthogonal rotation** (random Hadamard \(H\)) |
| **직관** | \(Y=XW=(XS^{-1})(SW)\) — outlier 부담을 **weight로 이동** | \(Y=XW=(XH)(H^{T}W)\) — outlier를 **전 차원에 분산** |
| **보존** | matmul 결과 동등 (재배치) | \(H^{T}H=I\) → 길이·내적 보존 |
| **목표 bit** | **W8A8** (메모리+INT8 GEMM 속도) | **W4A4 + KV4** (더 낮은 bit) |

**한 줄**: SmoothQuant = “어려운 activation을 weight에 넘긴다” / QuaRot = “튀는 에너지를 펴서 낮춘다”.

## 문제 1 (확정)

설명가능한 인공지능에서 local interpretability와 global interpretability의 차이를 설명하고, saliency map 기반 설명과 concept-based explanation의 차이를 논하라.

### 답안

**Local interpretability**는 개별 입력 사례에 대해 모델이 어떤 근거로 특정 예측을 했는지를 설명하는 데 초점을 둔다. 예를 들어 의료영상 분류 모델에서 특정 X-ray 이미지가 폐렴으로 분류되었을 때, Grad-CAM이나 CAM은 모델이 영상의 어느 영역을 중요하게 보았는지를 heatmap 형태로 보여준다. SHAP은 입력 feature들이 해당 예측에 얼마나 기여했는지를 정량적으로 분석한다. 따라서 local interpretability는 개별 사례에 대한 직관적인 설명을 제공한다.

**Global interpretability**는 모델이 전체적으로 어떤 개념, 규칙, 패턴에 의존하여 예측하는지를 설명하려는 접근이다. TCAV는 모델이 “줄무늬”, “질감”, “병변 형태” 같은 인간 친화적 개념에 얼마나 민감하게 반응하는지를 분석한다. ACE나 CRAFT는 모델 내부 표현에서 의미 있는 개념을 발견·분해하여 전반적인 판단 구조를 설명하려 한다.

**Saliency map** 기반 설명은 시각적으로 직관적이지만, 모델이 실제로 **왜** 그런 판단을 했는지에 대한 개념적 설명은 부족할 수 있다. heatmap이 병변을 강조하더라도 병변 형태인지, 주변 마커·촬영 조건인지 불명확할 수 있다. 반면 **concept-based explanation**은 인간이 이해할 수 있는 개념 단위 의존도를 보여 주어, 모델의 전반적 신뢰성과 편향 여부를 평가하는 데 더 유용할 수 있다.

## 문제 2 (확정)

Mechanistic interpretability가 일반적인 feature attribution 또는 saliency 기반 XAI와 구별되는 점을 설명하라.

### 답안

일반적인 XAI 방법은 주로 **입력과 출력 사이의 관계**를 설명하는 데 초점을 둔다. saliency map·Grad-CAM은 특정 입력에서 모델이 어느 영역을 중요하게 보았는지를 보여 주고, SHAP은 각 입력 feature의 최종 예측 기여도를 계산한다. 모델의 판단 근거를 **외부에서 관찰 가능한 형태**로 설명하는 데 유용하다.

반면 **mechanistic interpretability**는 모델 **내부에서 실제로 어떤 계산이 일어나는지**를 밝히는 것을 목표로 한다. 단순히 “이 입력이 출력에 어떤 영향을 주었다”가 아니라, 뉴런·attention head·layer·circuit·representation이 **어떤 기능을 수행하며 어떻게 결합**되어 최종 출력을 만드는지 분석한다.

saliency map은 “어느 부분을 보았는가”에 가깝고, mechanistic은 “특정 뉴런/attention head가 어떤 개념을 감지하고, 그 정보가 **어떤 경로**로 최종 판단에 영향을 주었는가”에 가깝다. 따라서 더 **인과적이고 구조적인** 설명을 추구한다. LLM·ViT처럼 복잡한 모델의 hallucination, bias, shortcut learning, unsafe behavior 분석에 중요하나, 규모·고차원 표현 때문에 전면 해석은 현실적으로 어렵다는 한계가 있다.

## 문제 3 (확정)

머신러닝에서 aleatoric uncertainty와 epistemic uncertainty의 차이를 설명하고, 설명가능성과 모델 신뢰성 평가의 관계를 논하라.

### 답안

**Aleatoric uncertainty**는 데이터 자체에 내재된 불확실성에서 비롯된다. 센서 노이즈, 영상 흐림·가림, 낮은 해상도, 전문가 간 라벨 불일치 등이 예이다. 같은 종류의 데이터를 단순히 더 많이 수집한다고 완전히 제거되기 어려워 **irreducible uncertainty**라고 불린다.

**Epistemic uncertainty**는 모델의 지식 부족에서 발생한다. 학습 데이터가 부족하거나 특정 도메인·분포를 충분히 학습하지 못했을 때 커진다. 한 병원 데이터만으로 학습한 모델이 다른 병원 장비 영상을 만나 높은 불확실성을 보이는 경우가 해당하며, **더 다양한 데이터 수집·모델 개선**으로 줄일 수 있다.

XAI는 불확실성을 직접 제거하지는 않지만, 예측이 신뢰할 만한지 판단하는 데 도움을 준다. 높은 confidence로 암을 예측했는데 Grad-CAM이 병변이 아니라 **병원 로고·영상 가장자리**를 강조한다면, **spurious correlation**에 의존했을 가능성이 있어 신뢰하기 어렵다.

따라서 **불확실성 추정**과 **XAI**는 상호보완적이다. 불확실성 추정은 “얼마나 확신하는가”를, XAI는 “그 확신의 근거가 타당한가”를 보여 준다. 의료·자율주행처럼 안전이 중요한 분야에서는 정확도만으로 부족하고, **언제 불확실해지는지**와 **어떤 근거로 판단하는지**를 함께 분석해야 한다.

## 문제 4 (확정)

Adversarial attack에서 targeted attack과 untargeted attack의 차이를 설명하고, adversarial robustness와 interpretability의 관계를 논하라.

### 답안

Adversarial attack은 입력에 사람이 거의 인식하기 어려운 작은 **perturbation**을 더해 모델 예측을 바꾸는 공격이다.

**Untargeted attack**은 원래 정답을 맞히지 못하게 만드는 것이 목적이다. 어떤 잘못된 클래스로 가든 정답에서 벗어나면 성공한다. **Targeted attack**은 공격자가 미리 지정한 **특정 클래스**로 오분류하도록 유도한다. (예: 고양이 → 아무 오분류 = untargeted, 반드시 “개” = targeted). Targeted는 특정 방향으로 decision boundary를 넘어야 하므로 일반적으로 더 어렵다.

목적 함수 관점에서 untargeted는 정답 클래스에 대한 **loss를 증가**시키는 방향으로, targeted는 목표 클래스에 대한 **loss를 감소**(또는 목표 확률 증가)시키는 방향으로 perturbation을 찾는다.

Adversarial perturbation은 최종 예측뿐 아니라 saliency map, Grad-CAM, SHAP 같은 **설명 결과도 왜곡**할 수 있다. XAI 역시 gradient·내부 activation에 의존하는 경우가 많기 때문이다. 공격자는 예측을 유지한 채 설명만 바꾸거나, 예측과 설명을 동시에 조작할 수도 있다.

XAI는 adversarial vulnerability 분석에 도움이 된다. adversarial example에서 attention/saliency가 객체에서 배경으로 이동하면 취약한 feature 의존을 알 수 있다. 그러나 XAI 자체가 완전한 방어책은 아니며, heatmap이 그럴듯해도 robust를 보장하지 않는다. robustness를 높이려면 **adversarial training, robust feature learning, uncertainty estimation, XAI 기반 검증** 등을 함께 활용해야 한다.

## 문제 5 (확정)

SmoothQuant와 QuaRot은 모두 LLM 양자화에서 activation outlier 문제를 해결하려는 방법이다. 두 방법이 activation outlier를 처리하는 방식의 차이를 설명하라.

### 답안

두 방법 모두 activation outlier가 LLM 양자화 성능 저하의 중요한 원인이라고 보지만, 처리 방식이 다르다.

**SmoothQuant**는 activation outlier를 직접 없애기보다, 그 부담을 **weight 쪽으로 옮긴다**. activation은 입력에 따라 동적으로 변하고 outlier가 심하지만, weight는 고정·상대적으로 안정적이다. diagonal scaling matrix \(S\)로

\[
Y = XW = (XS^{-1})(SW)
\]

처럼 변환한다. \(XS^{-1}\)은 activation의 큰 outlier를 줄여 **INT8 양자화를 쉽게** 하고, \(SW\)는 줄어든 scale을 weight로 옮긴다. 핵심은 **activation 양자화 난이도를 weight로 이전**하는 것이다.

**QuaRot**은 outlier를 weight로 옮기지 않고, **orthogonal rotation**으로 outlier 에너지를 여러 차원에 분산시킨다. randomized Hadamard 등 회전 행렬 \(H\)로

\[
Y = XW = (XH)(H^{T}W)
\]

와 같이 쓴다. \(H\)는 orthogonal이므로 벡터 길이·내적을 보존하면서, 특정 channel에 몰린 outlier를 전체 feature dimension으로 퍼뜨려 **4-bit 수준 양자화**도 더 안정적으로 만든다.

**정리**: SmoothQuant = **scaling**으로 outlier 부담을 weight로 이동 → **W8A8**에 적합. QuaRot = **rotation**으로 outlier 에너지를 전 차원에 분산 → **W4A4KV4**처럼 더 낮은 bit-width를 목표.

## ★★★ ① GPTQ vs AWQ (Weight-only PTQ)

**출제 포인트**: 3일차 문제 2  
**우선순위 근거**

- 확정 Q5가 **activation+weight** 계열(SmoothQuant/QuaRot)만 다루고, 강의 출제 포인트의 다른 축인 **weight-only PTQ 쌍**은 비어 있음

| | **GPTQ** | **AWQ** |
|--|----------|---------|
| **핵심 아이디어** | **Hessian**(\(X^{T}X\)) 기반 **column-wise greedy** weight 양자화 | **Activation-aware** diagonal scaling — outlier activation 채널의 weight를 더 정밀 격자 |
| **절차 직관** | 민감한 weight부터 quantize, 오차를 남은 weight에 전파 (Cholesky로 효율 업데이트) | activation 통계로 중요 채널 파악 → \(S\)로 weight 채널 스케일 조정 후 quantize |
| **Activation** | 보통 **FP16 유지** (W4A16) | **FP16 유지**, weight만 4-bit |
| **목표** | W4A16, block-wise (OBQ element-wise 대비 효율) | W4, perplexity 저하 ~1% 미만 |
| **한 줄** | “오차 전파를 고려한 **최적에 가까운 weight 양자화**” | “**activation이 센 채널**의 weight를 아껴 quantize” |

**AWQ vs SmoothQuant (보조)**: AWQ는 activation을 **quantize하지 않고** 메모리(weight)만 줄임. SmoothQuant는 activation도 INT8로 줄여 **연산 속도**까지 노림. 6번째가 GPTQ/AWQ면 한 줄 대비로 쓰면 가산점.

## ★★★ ② CAM vs Grad-CAM

**출제 포인트**: 1일차 문제 2  
**우선순위 근거**

- 강의에서 ⭐로 **재학습 여부·gradient weight**를 시험 포인트로 명시
- 확정 Q1·Q2는 CAM/Grad-CAM을 **예시 이름**으로만 쓰고, **둘의 차이**는 묻지 않음
- “전부 설명하는 문제” 철학에 맞는 **짧고 명확한 대비형**

| | **CAM** (2016) | **Grad-CAM** (2017) |
|--|----------------|---------------------|
| **채널 weight** | GAP 후 클래스별 **학습된 W** | logit \(y^{c}\)에 대한 feature map **gradient** 평균 |
| **재학습** | 마지막 레이어를 **따로 다시 학습** 필요 | **재학습 불필요** — pretrained 모델에 backprop만 |
| **핵심 식 직관** | activation map의 weighted sum | \(w_k^c = \frac{1}{Z}\sum_{i,j}\frac{\partial y^c}{\partial A_{ij}^k}\) |
| **한계/보완** | 구조 제약(GAP 연결 필요) | **feature value × gradient** — gradient만으로는 부족, 입력(activation) 값도 중요 |

**한 줄**: CAM = “클래스 weight를 **배워서** 히트맵” / Grad-CAM = “gradient로 weight를 **즉시 추정** → 재학습 없음”.

## 예상문제 1 — GPTQ vs AWQ (★ 최우선)

LLM 양자화에서 weight-only PTQ 방법인 GPTQ와 AWQ의 차이를 설명하시오. 각 방법이 weight를 양자화할 때 무엇을 고려하는지, activation 처리와 목표 비트 설정(예: W4A16) 관점을 포함하시오.

### 모범 답안

**GPTQ**와 **AWQ**는 모두 학습 후(PTQ) **weight를 낮은 비트(주로 4-bit)** 로 줄여 메모리를 절감하려는 방법이다. activation은 보통 FP16으로 두고 weight만 양자화한다(**W4A16** 계열). 그러나 “어떤 weight를 어떻게 반올림·보정할지”를 정하는 기준이 다르다.

**GPTQ**는 입력 activation으로 만든 **Hessian 정보**(\(X^{T}X\))를 이용해, weight를 **column-wise greedy**하게 양자화한다. 민감한(오차에 큰 영향을 주는) weight부터 quantize하고, 그 오차를 **아직 양자화하지 않은 나머지 weight에 전파·보정**한다. Cholesky factorization 등으로 이 업데이트를 효율적으로 수행하며, element-wise 최적에 가까운 OBQ보다 **block-wise**로 연산·메모리를 줄인 실용형이다. 핵심 한 줄: **“양자화 오차의 2차 근사를 보고, 남은 weight로 보상한다.”**

**AWQ(Activation-Aware Weight Quantization)** 는 weight 오차 전파보다, **activation outlier 채널**을 본다. 특정 채널의 activation이 크면 그 채널과 곱해지는 weight가 결과에 더 중요하므로, diagonal scaling \(S\)로 해당 weight 채널을 **더 촘촘한 격자**로 quantize할 수 있게 만든다. \(S\)는 RMSNorm의 \(\gamma\) 등에 fuse 가능하고, activation 자체는 FP16을 유지한다. 목표 역시 weight 4-bit이며 perplexity 저하를 매우 작게 유지하는 것이 보고된다. 핵심 한 줄: **“activation이 센 채널의 weight를 더 정밀하게 살린다.”**

| | GPTQ | AWQ |
|--|------|-----|
| 판단 기준 | Hessian 기반 weight 민감도 | Activation 크기(중요 채널) |
| 대표 목표 | W4A16, greedy+오차 전파 | W4, activation-aware scaling |
| Activation | FP16 유지 | FP16 유지 |

**SmoothQuant와의 한 줄 구분**(여유 시): SmoothQuant는 activation **outlier 부담을 weight로 옮겨 둘 다(W8A8) quantize**하고, AWQ/GPTQ는 activation을 **quantize하지 않은 채 weight 메모리**를 줄인다.

## 예상문제 2 — CAM vs Grad-CAM (★ 차순위·동급 유력)

Class Activation Mapping(CAM)과 Grad-CAM의 차이를 설명하시오. 특히 채널 가중치를 구하는 방식과, 사전 학습된 모델에 적용할 때의 재학습 필요 여부 관점에서 서술하시오.

### 모범 답안

**CAM**과 **Grad-CAM**은 모두 CNN 등에서 **특정 클래스 예측에 중요한 공간 위치**를 heatmap으로 보여 주는 **local attribution** 방법이다. 마지막 convolution feature map의 채널별 중요도를 구한 뒤 weighted sum으로 히트맵을 만든다는 점은 같다. 차이는 **그 채널 중요도(weight)를 어떻게 얻는가**에 있다.

**CAM**은 마지막 conv 직후 **Global Average Pooling(GAP)** 으로 채널별 스칼라를 만들고, 분류기에 있는 **클래스별 학습 weight \(W\)** 를 곱해 채널을 가중합한다. 이 구조를 쓰려면 네트워크가 GAP+선형 분류 형태여야 하고, 이미 학습된 CNN 위에 **마지막 레이어를 목적에 맞게 다시 학습**해야 하는 경우가 많다. 이것이 CAM의 **가장 큰 실무 단점**이다.

**Grad-CAM**은 클래스 logit \(y^{c}\)를 feature map \(A^{k}\)로 미분한 **gradient**를 공간 평균하여 채널 weight를 만든다.

\[
w_k^c = \frac{1}{Z}\sum_{i,j}\frac{\partial y^c}{\partial A_{ij}^k}
\]

미리 학습된 모델에 **backpropagation만** 하면 되므로 **재학습이 필요 없다**. CAM이 요구하던 구조 변경·재학습 문제를 해소한다. 강의에서 강조한 직관은 gradient만 보는 것이 아니라 **feature(activation) 값 자체와 gradient를 함께** 쓰는 것이 중요하다는 점이다(값이 큰 위치×기여도).

| | CAM | Grad-CAM |
|--|-----|----------|
| 채널 weight | 학습된 class weight (+GAP) | \(\partial y^c / \partial A^k\) 의 공간 평균 |
| 재학습 | **필요**한 경우가 많음 | **불필요** |
| 적용 용이성 | 구조 제약 | pretrained 모델에 바로 적용 |

**한 줄**: CAM은 “**배운 weight**로 어디를 볼지 정한다”, Grad-CAM은 “**gradient로 weight를 즉석 추정**하여 재학습 없이 어디를 볼지 정한다”.

## 예상문제 3 — White-box vs Black-box 및 FGSM vs PGD

Adversarial attack에서 white-box attack과 black-box attack의 차이를 설명하고, white-box 공격의 대표 방법인 FGSM과 PGD의 차이를 설명하시오.

### 모범 답안

Adversarial attack은 입력에 작은 perturbation을 더해 모델 예측을 바꾸는 공격이다. **접근 권한**에 따라 white-box와 black-box로 나눈다.

**White-box**는 공격자가 모델 구조, 가중치, 그리고 **gradient**에 접근할 수 있다. loss를 입력에 대해 미분하여 perturbation 방향을 정확히 알 수 있으므로 FGSM, PGD 같은 방법이 여기에 해당한다. **Black-box**는 모델 내부를 모르고 **입·출력만** 관찰한다. finite difference 등으로 gradient를 **추정**하거나 쿼리 기반 공격을 쓰며, 일반적으로 **시간과 쿼리 비용이 크다**.

같은 white-box 안에서도 **한 번에 칠지, 여러 번 칠며 범위를 지킬지**가 FGSM과 PGD의 차이다.

**FGSM(Fast Gradient Sign Method)** 은 한 스텝 공격이다.

\[
x_{adv} = x + \epsilon \cdot \mathrm{sign}(\nabla_x L(x,y))
\]

gradient sign 방향으로 \(\epsilon\)만큼 한 번만 더하므로 **매우 빠르고** closed-form에 가깝다. 단, 사실상 모든 픽셀이 동시에 바뀌어 salt-and-pepper성 노이즈처럼 보일 수 있고, 한 스텝이라 **최적 공격 강도는 약할 수 있다**.

**PGD(Projected Gradient Descent)** 는 FGSM을 **여러 번 반복**(iterative)하고, 매 스텝 또는 누적 후 결과를 원본 \(x\) 중심의 **\(\epsilon\)-ball 안으로 clipping(투영)** 한다. 특정 픽셀 perturbation이 과도하게 쌓이지 않게 제약을 지키면서, 더 강한(벤치마크에서 기본으로 쓰이는) white-box **untargeted** 공격을 만든다. PPO의 clipping과 같이 **제약 최적화(Trust Region)** 관점으로도 이해할 수 있다.

| | FGSM | PGD |
|--|------|-----|
| 스텝 | **1회** | **다회(iterative)** |
| 범위 유지 | \(\epsilon\) 한 번 적용 | **ε-ball projection/clip** |
| 특징 | 빠름, 약할 수 있음 | 더 강함, 벤치마크 표준 |

**확정 Q4와의 연결**: untargeted는 정답 loss maximize, targeted는 목표 loss minimize 방향 — FGSM/PGD 모두 이 목적 함수 설정에 따라 targeted/untargeted로 쓸 수 있다.

### 확정 5문제 (반드시 서술 연습)

- [ ] **Local**: 한 샘플, Where — CAM/Grad-CAM/SHAP / **Global**: 모델 전체, What — TCAV·concept
- [ ] **Saliency**: 직관적 but Why 부족 / **Concept-based**: 인간 개념 단위, 편향·신뢰 평가
- [ ] **Mechanistic**: 내부 뉴런·head·circuit·경로 (입력 중요도와 구별)
- [ ] **Aleatoric** = 데이터 내재·irreducible / **Epistemic** = 지식 부족·데이터로 완화
- [ ] 불확실성(확신 정도) + XAI(근거 타당성) = **상호보완**; spurious correlation 예시
- [ ] **Untargeted** = loss↑ 아무 오분류 / **Targeted** = 특정 클래스, loss↓, 더 어려움
- [ ] Adv. → 설명도 왜곡 가능; XAI는 분석 도구이지 완전 방어 아님
- [ ] **SmoothQuant**: \(XS^{-1}, SW\), outlier → weight, **W8A8**
- [ ] **QuaRot**: Hadamard rotation, outlier 분산, **W4A4KV4**
