## 1.4 확정문제별 심층 분석

### Q1 — Metric Learning · Subspace Alignment · Distribution Matching

**세 접근의 정렬 기준**

| 접근 | 무엇을 맞추나 | 핵심 메커니즘 |
|------|---------------|---------------|
| **Metric Learning** | **거리 관계** | 같은 클래스 가깝게, 다른 클래스 멀게 — source·target **쌍**에 동일 제약 |
| **Subspace Alignment** | **Subspace(기저)** | 각 도메인 subspace를 **공통 subspace**로 정렬 — 분포가 아니라 **공간 구조** |
| **Distribution Matching** | **분포 자체** | MMD 등으로 source·target 분포 차이 **직접 최소화** |

**차이 한 줄**: Metric = **레이블 있는 거리** / Subspace = **간접·기하** / Distribution = **통계·가장 직접**

### Q2 — DA-Fusion (DG, diffusion augmentation)

**DG에서 도움이 되는 이유**
- Diffusion으로 style·background·texture 등을 바꾼 **synthetic sample** 생성
- Source domain을 **다양한 appearance** 도메인으로 **확장**
- 특정 source distribution에 **과적합 방지** → **domain-invariant** 특징 학습 → **unseen target** 성능↑

**한계**
- 생성 도메인이 **실제 target distribution과 일치한다는 보장 없음**
- 생성 품질·**pretrained 편향**에 의존
- Augmentation 과정에서 **semantic 정보 훼손** → **label noise** 위험

### Q3 — TCA vs DIP

| | **TCA** | **DIP** |
|---|---------|---------|
| **순서** | $W\phi(x)$ — **Hilbert 먼저 → 사영 W** | $\phi(Wx)$ — **사영 W 먼저 → Hilbert** |
| **MMD 계산** | 사영된 latent에서 **근사** | Hilbert space에서 **온전히** 계산 → **더 정확** |
| **최적화** | 고유값 분해 형태로 **효율적** | $W$가 커널 내부 → **non-convex**, **어려움** |

**트레이드오프**: TCA = **풀기 쉬움 · MMD 근사** / DIP = **MMD 정확 · 최적화 난이**

## 문제 1 (확정) — Domain Adaptation: 3대 접근

Domain Adaptation의 대표적인 세 가지 원리로 **Metric Learning**, **Subspace Alignment**, **Distribution Matching**이 있습니다. 각 원리가 무엇인지 설명하고, 세 접근 방식의 차이를 서술하세요.

### 답안

**Metric Learning**은 데이터 간의 **distance function**을 학습하는 방식이다. 같은 클래스 샘플은 가깝게, 다른 클래스는 멀게 위치하도록 거리를 학습하며, DA에서는 이 제약을 source와 target 도메인 간 샘플 쌍에 적용한다. 도메인이 달라도 같은 클래스면 거리가 작아지도록 **변환 행렬**을 학습해, 학습된 거리 공간에서 **도메인 간극**이 줄어든다.

**Subspace Alignment**는 source와 target 데이터가 각각 **서로 다른 subspace**에 있다고 보고, 두 도메인의 subspace를 **하나의 공통 subspace**로 정렬하는 방식이다. 분포 자체가 아니라 데이터가 놓인 **공간의 기저**를 맞춤으로써 도메인 차이를 줄이며, 정렬된 공통 공간에서 모델이 두 도메인에 걸쳐 일반화할 수 있다.

**Distribution Matching**은 source와 target의 **분포 차이를 직접** 측정하고 최소화하는 접근이다. 대표적으로 두 분포를 Hilbert space로 매핑한 뒤 평균 간 차이로 분포 거리를 재는 **MMD**를 사용하며, 이 값이 최소가 되는 표현을 학습하면 두 도메인이 같은 분포처럼 정렬된다.

**세 방식의 차이**: domain gap을 **무엇을 기준으로 줄이느냐**에 차이가 있다. Metric Learning은 **샘플 쌍 단위 거리 관계**를 기준으로 정렬하므로 **클래스 레이블**을 활용한다. Subspace Alignment는 개별 샘플·분포가 아니라 도메인의 **기하학적 subspace**를 맞추는 **간접** 방식이다. Distribution Matching은 **distribution mismatch** 자체를 통계적 거리(MMD 등)로 **직접 최소화**한다.

## 문제 2 (확정) — Domain Generalization: DA-Fusion

위 그림은 domain generalization 기법 중 domain augmentation 방식을 활용한 **DA-Fusion**의 모식도이다. 이 네트워크는 diffusion model을 활용해 source image의 style, background, texture 등을 바꾸어 synthetic sample을 생성한다. 이러한 방식이 **DG 관점에서 target domain 성능 향상에 도움이 되는 이유**와 이 방식이 가진 **한계점**을 서술하세요.

### 답안

**도움이 되는 이유**

- Source domain을 **다양한 appearance**를 가지는 도메인으로 **확장**할 수 있다.
- 모델이 특정 source distribution에 **과적합되지 않고** 더 넓은 분포를 경험하게 한다.
- 따라서 **domain-invariant**한 특징을 학습하게 되어 **unseen target domain** 성능 향상에 도움이 된다.

**한계점**

- 생성된 domain이 **실제 target distribution과 일치한다는 보장이 없다**.
- **생성 품질**, diffusion **사전학습 편향**에 성능이 의존할 수 있다.
- Augmentation 과정에서 클래스 판별에 필요한 **semantic 정보를 훼손**할 수 있어 **label noise** 발생 위험이 있다.

## 문제 3 (확정) — Domain Adaptation: TCA vs DIP

위 두 그림은 MMD 기반 domain adaptation 기법인 **TCA**(Transfer Component Analysis)와 **DIP**(Domain Invariant Projection)의 모식도이다. 두 방식 모두 source와 target 분포 간 **MMD를 최소화**하는 표현을 학습하지만, 커널 매핑 $\phi$와 사영 $W$의 **적용 순서**가 다르다. 두 방식의 연산 순서 차이를 설명하고, 이 차이가 **MMD 계산의 정확성**과 **최적화 난이도** 관점에서 각각 어떤 장단점을 만드는지 서술하세요.

### 답안

**연산 순서 차이**

- **TCA**: 입력을 먼저 커널 $\phi$로 Hilbert space에 매핑한 뒤 사영 $W$를 적용 — **$W\phi(x)$** 구조
- **DIP**: 입력에 먼저 사영 $W$를 적용해 latent space를 만든 뒤 커널 적용 — **$\phi(Wx)$** 구조

**TCA**

- **장점**: empirical kernel map에 대한 사영으로 formulation → **고유값 분해** 형태로 **효율적으로** 풀 수 있음
- **단점**: 사영이 커널 매핑 **이후**에 적용 → MMD가 실제 Hilbert space가 아니라 **사영된 latent에서 근사** 계산

**DIP**

- **장점**: Latent representation을 먼저 학습하고 그 위에서 커널 적용 → MMD가 **온전히 Hilbert space**에서 계산 → 분포 차이를 **더 정확히** 측정·최소화
- **단점**: $W$가 커널 **내부**에 들어가 **non-convex** → 고유값 분해 같은 깔끔한 해 없음 → **최적화 어려움**

## ★★★ ② DA vs DG — 정의·차이

**출제 포인트**: DG 핵심 #1

| | DA | DG |
|---|-----|-----|

## ★★☆ ③ Group DRO — Worst-Case vs ERM

**출제 포인트**: DG 핵심 #5

- **ERM**: **평균** 손실 최소화 → 한 도메인만 잘 되고 다른 도메인 망할 수 있음
- **Group DRO**: 도메인(그룹)별 손실 중 **최악(worst)** 그룹 손실 최소화 → 모든 도메인 **최소 성능** 보장

**왜 유력한가**: Learning strategy 대표 기법. MLDG와 짝을 이루는 **DG 심화** 후보.

- Phase 기반 feature → DG에 도움 (Q2 diffusion augmentation과 **주파수 관점** 연결)

## 예상문제 1 — MLDG (★ 최우선)

**MLDG(Meta-Learning for Domain Generalization)** 가 domain generalization 문제를 어떻게 풀려 하는지 설명하시오. Meta-train과 meta-test를 나누는 **self-simulation**이 왜 unseen target domain 일반화에 도움이 되는지 서술하시오.

### 모범 답안

**MLDG의 아이디어**

- 여러 **source domain**이 있을 때, 일부를 **meta-train**, 나머지를 **meta-test**로 나눈다.
- Meta-train domain으로 모델을 업데이트한 뒤, **meta-test domain**(학습 중 보지 않은 도메인 역할)에서도 loss가 작아지도록 **meta-parameter**를 학습한다.
- 즉 학습 과정 안에 “**가상의 새 도메인**에서 테스트한다”는 경험을 넣는 **self-simulation**이다.

**Unseen target에 도움이 되는 이유**

- 실제 테스트 시 target domain은 **미지정·미접근**이지만, MLDG는 meta-test로 **그 상황을 모의**한다.
- Meta-learning으로 “한 도메인에서 학습하고 **다른 도메인에서 망하지 않도록**” 하는 능력을 **직접 최적화**한다.
- 결과적으로 **domain-invariant**하면서 **task-discriminative**한 representation이 강화되어, 학습에 없던 target에서도 성능이 유지되기 쉽다.

## 예상문제 2 — Domain Adaptation vs Domain Generalization

**Domain Adaptation(DA)**과 **Domain Generalization(DG)**의 문제 설정 차이를 설명하시오. Target domain의 **지정 여부**, **target 데이터 사용 여부**, **학습 목표**를 중심으로 비교하시오.

### 모범 답안

| | **Domain Adaptation (DA)** | **Domain Generalization (DG)** |
|---|---------------------------|--------------------------------|
| **Target domain** | **특정 target이 정해져 있음** | Target이 **미지정** (어떤 domain이 올지 모름) |
| **Target 데이터** | UDA 등에서 target **비라벨 데이터 접근 가능** | Target 데이터를 **사실상 사용하지 않음** |
| **학습 목표** | Source와 target 간 **distribution discrepancy를 줄임** (align) | **여러 source**로 **domain-invariant + class discriminative** feature 학습 |
| **테스트** | **정해진 target** domain에서 평가 | **보지 못한** 새 domain에서 평가 |

**한 줄 요약**

- **DA**: “**이 target**에 맞추자” — target을 알고(데이터로) 정렬한다.
- **DG**: “**어떤 target**이 와도 견디자” — target 없이 일반화한다.

**공통 가정**: Covariate shift 등 — $P(X)$는 도메인마다 다르지만 $P(Y|X)$는 같다는 설정이 자주 쓰인다.

## 예상문제 3 — Group DRO

**ERM(Empirical Risk Minimization)**과 **Group DRO(Distributionally Robust Optimization)**의 학습 목표 차이를 설명하시오. Domain generalization에서 Group DRO가 여러 source domain 환경에 왜 유리할 수 있는지 서술하시오.

### 모범 답안

**ERM**

- 전체(또는 배치) 샘플에 대한 손실의 **평균**을 최소화한다.
- **평균** 성능은 좋아도, **특정 도메인(그룹)** 에서만 성능이 나쁠 수 있다 — 한 도메인에 과적합.

**Group DRO**

- 데이터를 **그룹(도메인)** 별로 나누고, 그룹별 손실 중 **최악(worst-case)** 인 그룹의 손실을 최소화한다.
- **평균**이 아니라 **최악 그룹**을 끌어올리는 것이 목표이다.

**DG에서 유리한 이유**

- DG는 **여러 source domain**에서 학습하고 **unseen target**에서 평가한다.
- ERM만 쓰면 “대부분 도메인에서 잘 되는” 모델이 **일부 도메인에서 크게 실패**할 수 있다.
- Group DRO는 **가장 어려운 도메인** 성능을 보장하려 하므로, domain 간 **성능 편차**를 줄이고 **worst-case generalization**에 가깝게 학습한다.
