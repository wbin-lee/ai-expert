# 2부. 확정문제 (공개 PDF 원문)

## 문제 1 (확정)

VQA(Visual Question Answering) task가 무엇인지 설명하고, 일반적인 이미지 분류와 비교했을 때 어떤 점에서 더 복합적인 문제인지와 VQA 모델의 기본 구성요소를 함께 서술하시오.

### 답안

- **VQA**는 이미지와 질문을 함께 입력받아 질문에 맞는 답을 출력하는 **Vision-Language task**이다.
- 일반적인 이미지 분류와 달리, VQA는 **같은 이미지라도 질문에 따라** 필요한 정보와 추론 방식이 달라지므로 **이미지와 언어를 함께** 이해해야 한다.
- 기본 구성: **이미지 인코더**, **텍스트 인코더**, 두 정보를 결합하는 **멀티모달 모듈(fusion)**, **답변 디코더**.

---

## 문제 2 (확정)

Image captioning 모델은 일반적으로 MLE(Maximum Likelihood Estimation)로 학습되지만, 생성된 캡션이 사람에게는 부자연스럽게 느껴지거나 이미지의 핵심을 충분히 잘 설명하지 못하는 경우가 있다. 이러한 현상이 발생하는 이유를 간단히 설명하고, 이를 완화하기 위해 사용되는 학습 방법을 서술하시오.

### 답안

- **MLE**는 정답 캡션의 단어를 순서대로 잘 맞히도록 학습 → **문장 전체의 자연스러움·의미적 적절성**보다 **단어 수준 예측 정확도**에 초점.
- 한 이미지에 **가능한 정답 캡션이 여러 개**인데 학습은 일부만 기준으로 하고, 평가는 **문장 전체 품질·사람 판단** → **학습 목표와 평가 기준의 차이(misalignment)**.
- 완화: **문장 단위 보상을 직접 최적화**하는 강화학습 — **policy gradient**, **SCST(Self-Critical Sequence Training)** 등.

---

## 문제 3 (확정)

멀티모달 학습에서 cross-modal supervision과 fusion은 각각 어떤 목적에서 사용되는지 설명하고, 두 방식이 활용하는 데이터의 성질을 비교하시오.

### 답안

**Cross-modal supervision**
- 한 modality를 **다른 modality의 학습 신호(supervision)** 로 활용.
- 예: 텍스트 설명으로 **이미지 표현** 학습.
- 활용 성질: modality 간 **공통 정보(redundancy)**.

**Fusion**
- 여러 modality를 **함께 입력**하여 예측 **강건성** 향상.
- 예: 음성 인식에서 오디오+영상 → noisy audio에서도 안정적.
- 활용 성질: 각 modality의 **보완적 정보(complementarity)**.

**비교**
- Cross-modal supervision: 한 modality가 다른 modality의 supervision 역할을 할 수 있을 때 적합.
- Fusion: 여러 modality를 **함께** 써야 더 정확한 판단이 가능한 상황에 적합.

# 3부. 남은 1문제 — 출제 예상 개념 정리

강의 「시험 출제 포인트」 기준, 확정 3문제가 담지 못한 영역을 **우선순위 순**으로 정리한다.

---

## ★★★ ① Foundation Model — Adaptiveness + CLIP

**출제 포인트**: E-23, E-26, E-24

| 개념 | 핵심 |
|------|------|
| **Foundation model** | 대규모 사전학습 large model. 핵심 특성 = **Adaptiveness** — 다양한 downstream·프롬프팅으로 적응 |
| 적응 방식 변화 | task-specific **fine-tuning** → **prompting**(튜닝 없이) |
| **CLIP** | 웹 스케일 image-text pair(**alt-text**, ~400M)로 **contrastive learning** |
| Contrastive | 배치 내 matched pair = positive, 나머지 = negative → embedding **같은 방향** |
| **Zero-shot classification** | `"a photo of a {class}"` 템플릿 embedding ↔ 이미지 **similarity 최대** 클래스 |
| 부가 특성 | **Domain robustness**(sketch 등), **linear probing**으로도 strong baseline |

**왜 4번째 문제에 가장 유력한가**
- 강의 후반(260630 오전) **핵심 주제**이나 확정 3문제 어디에도 없음.
- Q3(supervision vs fusion)과 자연스럽게 이어지는 “다음 단계 = 웹 스케일 foundation model” 흐름.
- 확정문제와 동일한 **“정의 + 비교/활용”** 서술형으로 내기 좋음.

---

## ★★★ ② VQA v2 — Language Bias

**출제 포인트**: B-8

| 개념 | 핵심 |
|------|------|
| VQA v1 문제 | Yes/No에 **Yes 편향** — 이미지 없이 Yes만 찍어도 60~70% |
| VQA v2 동기 | **language bias** 제거, **이미지 의존성** 검증 |
| **Balanced eval set** | 답 분포 균형 |
| **Complementary images** | **같은 질문**에 답이 **바뀌는** 이미지 쌍 → 진짜 비전 이해 필요 |
| 예시 | “바나나 색?” — 텍스트만 Yellow, 실제 초록 바나나는 Green |

**왜 유력한가**: Q1이 VQA **기본**만 다룸. v2는 강의에서 ⭐로 강조된 **대표적 “왜 v2가 필요한가”** 주제.

---

## ★★☆ ③ Visual Attention — Bottom-up / Top-down + DeVQA

**출제 포인트**: B-10~12

| 개념 | 핵심 |
|------|------|
| VQA attention | **Reference resolution** — 질문이 가리키는 객체·영역 찾기 |
| **Bottom-up** | object detector로 **의미 단위 region** 추출 |
| **Top-down** | 질문 관련 region에 **soft attention** |
| **DeVQA** | (1) image+question, (2) **question-only** dual path |
| DeVQA reweighting | question-only로도 맞음 → **language bias** → 가중치 **축소** / question-only는 틀림 → 이미지 필요 → 가중치 **강화** |

**왜 유력한가**: Q1의 “멀티모달 모듈”을 **구체 방법**으로 심화하는 전형적 4번째 문제 패턴.

---

## ★★☆ ④ Captioning — Show&Tell vs Show Attend&Tell + 평가 지표

**출제 포인트**: C-15~17

| 개념 | 핵심 |
|------|------|
| **Show and Tell** | CNN → single vector → RNN. 이미지 정보 **처음 한 번만** → 후반 hallucination |
| **Show, Attend and Tell** | 매 스텝 **soft attention**으로 관련 영역 aggregate |
| **Hard attention** | location **샘플링** → gradient 끊김 → **Policy Gradient(REINFORCE)** |
| **CIDEr** | n-gram + **TF-IDF** — content word에 높은 가중 |
| **SPICE** | **scene graph** 유사도 — 이미지 내용 반영 |

**왜 중간 우선순위인가**: Q2가 MLE·RL을 이미 다뤄 **중복 가능성** 있으나, attention·평가 지표는 아직 미출제.

---

## ★★☆ ⑤ SCST 상세 (Q2 보강형)

**출제 포인트**: D-21

- Reward = **CIDEr**(또는 BLEU 등 — objective에 따라 해당 metric 상승)
- **Baseline** = greedy/beam으로 만든 **자기 캡션(self-critical)**
- 샘플 CIDEr > baseline → encourage, 아니면 discourage → **variance 감소**

Q2에서 SCST를 **이름만** 언급했으므로, 4번째가 SCST **메커니즘 상세**일 가능성은 있으나, CLIP·language bias보다는 낮음.

## 예상문제 1 — Foundation Model & CLIP (★ 최우선)

Foundation model을 정의하고, foundation model의 핵심 특성이 무엇인지 설명하시오. 이어서 CLIP이 어떤 방식으로 image-text representation을 학습하는지, 그리고 학습된 CLIP을 이용한 zero-shot image classification이 어떻게 동작하는지 서술하시오.

### 모범 답안

**Foundation model 정의**
- **대규모 데이터**로 사전학습된 **large model**이다.
- 다양한 downstream task에 **적응(adapt)** 할 수 있도록 설계된다.

**핵심 특성 — Adaptiveness**
- Foundation model의 본질은 “크기”나 “self-supervised” 자체가 아니라 **Adaptiveness**이다.
- 하나의 모델이 fine-tuning, prompting 등 다양한 방식으로 **여러 task·도메인에 적응**할 수 있다.
- 최근 흐름: task-specific fine-tuning → **prompting**(추가 튜닝 없이 입력만으로 task 수행). Vision에서도 SAM의 point/box prompt 등 **visual prompting**이 같은 계열이다.

**CLIP 학습 방식**
- **웹 스케일 image-text pair**(alt-text 기반, 수억 쌍)를 사용한다.
- **Contrastive learning**: 한 배치에서 **matched**(이미지·캡션 쌍)는 positive, **unmatched**는 negative로 취급.
- 이미지·텍스트를 각각 encoder로 embedding한 뒤, matched pair의 dot product(similarity)가 크도록, unmatched는 작도록 **symmetric cross-entropy**로 학습한다.
- 결과: 의미적으로 대응하는 이미지·텍스트가 embedding space에서 **같은 방향**에 놓인다.

**Zero-shot image classification**
- 각 클래스 이름에 대해 `"a photo of a {class}"` 같은 **프롬프트 템플릿**으로 텍스트 embedding을 만든다.
- 분류할 이미지 embedding과 모든 클래스 텍스트 embedding의 **similarity**를 계산한다.
- **가장 similarity가 높은** 클래스를 예측 — 별도 classifier 학습 없이 수행(zero-shot).
- ImageNet supervised model 대비 sketch 등 **domain shift**에 강한 **domain robustness**도 보고된다.

## 예상문제 2 — VQA v2 Language Bias

VQA v1 데이터셋에서 나타나는 language bias 문제가 무엇인지 설명하고, VQA v2가 이 문제를 완화하기 위해 도입한 방법을 서술하시오.

### 모범 답안

**VQA v1의 language bias**
- VQA v1은 MS COCO 이미지에 crowdsourcing QA를 수집했다.
- 특히 **Yes/No 질문**에서 **Yes로 답하는 편향**이 심했다.
- **이미지 없이** 질문 텍스트만 보고 “Yes”를 출력해도 **60~70%** 정도 맞출 수 있었다.
- 즉 모델이 **비전을 보지 않고 언어 통계만**으로도 높은 점수를 낼 수 있어, 진짜 이미지 이해를 검증하기 어렵다.

**VQA v2의 완화 방법**

1. **Balanced evaluation set**  
   - 평가 집합에서 **답 분포를 균형** 있게 구성해, 특정 답(Yes 등)만 찍는 전략이 통하지 않게 한다.

2. **Complementary image pairs**  
   - **같은 질문**에 대해 **답이 바뀌는** 이미지 쌍을 의도적으로 구성한다.  
   - 예: “What color is the banana?” — 일반 바나나 이미지는 Yellow, **초록 바나나** 이미지는 Green.  
   - 텍스트만으로는 항상 Yellow를 출력하는 모델은 한쪽에서 반드시 틀린다.  
   - → **이미지를 반드시 봐야만** 맞출 수 있는 샘플로 **language bias를 제거**하고 비전 의존성을 검증한다.

**한 줄 요약**: VQA v2 = “언어 편향으로 우회하지 못하게” **답 균형 + 같은 질문·다른 이미지** 쌍으로 진짜 visual reasoning을 요구한다.

## 예상문제 3 — Bottom-up / Top-down Attention & DeVQA

VQA에서 bottom-up attention과 top-down attention이 각각 어떤 역할을 하는지 설명하고, DeVQA가 language bias를 완화하기 위해 사용하는 dual-path 구조와 sample reweighting 방식을 서술하시오.

### 모범 답안

**VQA에서 attention의 목적**
- VQA의 핵심 어려움 중 하나는 **reference problem** — 질문이 가리키는 객체·영역을 찾는 것이다.
- Visual attention은 질문(Query)과 이미지 피처(Key/Value)를 맞춰 **관련 영역을 선택(reference resolution)** 한다.

**Bottom-up attention**
- **Object detector** 등으로 이미지에서 **의미 있는 region proposal**(objectness 기반, class-agnostic 포함)을 먼저 추출한다.
- “픽셀 grid 전체”가 아니라 **객체·영역 단위 feature**를 후보로 제공 — **bottom-up** = 이미지에서 올라오는 후보 생성.

**Top-down attention**
- 질문(또는 상위 task signal)을 기준으로, bottom-up이 준 region들 중 **질문과 관련 있는 영역**에 **soft attention** 가중치를 부여한다.
- 예: “What color are the cows?” → cow region에 먼저 attention, 그 안에서 color 속성 추론.
- VQA·Image Captioning 모두에서 성능 향상에 기여했다.

**DeVQA — dual path**
- **Path 1**: image + question → 답 (정상 멀티모달 경로)
- **Path 2**: **question-only** → 답 (이미지 없이 질문만)

**Sample reweighting**
- **Question-only로도 맞는** 샘플: 텍스트 **language bias**로 풀린 경우 → 학습 시 가중치 **축소(down-weight)**.
- **Question-only로는 틀리는** 샘플: 이미지를 봐야만 맞출 수 있음 → 가중치 **강화(up-weight)**.
- 두 경로의 출력 차이를 이용해, 모델이 **비전에 의존해야 하는 샘플**에 더 집중하도록 유도한다.

# 5부. 시험 직전 체크리스트

### 확정 3문제 (반드시 서술 연습)

- [ ] VQA = image + question → answer / 질문마다 추론 달라짐
- [ ] VQA 구성: image encoder, text encoder, fusion, answer decoder
- [ ] Captioning MLE = 단어 수준 / GT 비유일 / 학습↔평가 misalignment
- [ ] RL 완화: policy gradient, **SCST**
- [ ] Cross-modal supervision = **redundancy** (한 모달이 다른 모달의 신호)
- [ ] Fusion = **complementarity** (다 모달 동시 입력, 강건성)

### 예상 4번째 (우선 대비)

- [ ] Foundation model 핵심 = **Adaptiveness** (fine-tune → prompting)
- [ ] CLIP = contrastive + 웹 스케일 alt-text / zero-shot = `a photo of a {class}` + max similarity
- [ ] VQA v2 = Yes 편향 / balanced eval / **complementary images**
- [ ] Bottom-up = detector region / Top-down = 질문 기반 attention
- [ ] DeVQA = question-only dual path + reweighting

### 보조 대비

- [ ] CLEVR = compositional generalization (빨간 개)
- [ ] CIDEr = TF-IDF n-gram / SPICE = scene graph
- [ ] Show&Tell vs Show Attend&Tell / hard attention → REINFORCE
- [ ] Flamingo = frozen LLM + Perceiver Resampler + zero-init gating
