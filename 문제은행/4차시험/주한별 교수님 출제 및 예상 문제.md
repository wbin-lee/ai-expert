## 1.3 확정문제별 심층 분석

### Q1 — Visual Hull (Shape from Silhouette)

**문제가 주는 전제**
- 3D 점 $X$
- 여러 대의 **calibrated camera** (각 뷰의 projection matrix 알려짐)
- 각 뷰의 **silhouette mask** (foreground vs background)

**판별 절차 (답의 뼈대)**

```
3D 점 X
  → 각 카메라로 project (projection matrix)
  → 2D 픽셀 위치
  → 해당 뷰의 silhouette mask에서 foreground/background 확인
  → 모든 뷰에서 foreground(실루엣 안)이면 → OCCUPIED
     하나라도 background이면 → VACANT (Visual Hull 밖)
```

**핵심 한 줄**: Visual Hull = **모든 뷰의 실루엣 교집합**에 해당하는 3D 영역. 한 뷰에서라도 실루엣 밖이면 그 점은 물체 내부가 아니다.

**시험 팁**
- “**all views**” / “**inside the silhouette in all views**”를 반드시 쓴다.
- Pose estimation 맥락: 여러 뷰 실루엣으로 **3D human volume**을 근사하는 Shape-from-Silhouette 계열.

### Q2 — Point Cloud / Voxel / Mesh

| 표현 | Description | Advantage | Disadvantage |
|------|-------------|-----------|--------------|
| **Point Cloud** | 순서 없는 3D 점 집합 $\{(x_i,y_i,z_i)\}$ | 센서(stereo/depth)에서 **직접 획득** 쉬움; 임의 형상 자유 표현 | **이웃·연결성 없음** → adjacency/surface/topology 정의 어려움; **두 cloud 비교** 어려움 |
| **Voxel** | 3D 공간을 **규칙 grid**로 이산화 | **이웃 관계 명확**; 2D image의 3D 확장; **3D convolution**에 적합 | 메모리 **$O(n^3)$**; **Manhattan-world**(축 정렬) 편향 |
| **Mesh** | **Vertices + faces**(connectivity)로 표면 정의 | **표면을 직접** 표현; **렌더링**에 이상적 (graphics 표준) | **고정 topology**로 reconstruction 어려움; self-intersection; 고해상도 시 vertex↑ |

**배점 구조**: 표현당 6점 = Description 2 + Advantage 2 + Disadvantage 2 (추정)

## 문제 1 (확정) — Visual Hull [Difficulty: Medium, 20 pts]

**Visual Hull (Shape from Silhouette).** Given a 3D point $X$, along with several calibrated camera views and their input image silhouette masks (foreground vs. background), explain how to decide whether $X$ is **occupied** or **vacant**. (Equations are not required.)

### 답안

1. 3D 점 $X$를 각 카메라 이미지에 그 카메라의 known calibration(**projection matrix**)을 사용해 **투영**하여, 뷰마다 **2D 픽셀 위치**를 구한다.
2. 각 뷰에서 해당 픽셀 위치의 **silhouette mask**를 확인한다. 픽셀이 **foreground(실루엣 안)** 에 있거나 **background**에 있다.
3. $X$는 투영이 **모든 뷰**에서 실루엣 **안(foreground)** 에 들어올 때만 **occupied**로 선언한다. 한 뷰라도 background이면 vacant(Visual Hull 밖)이다.

## 문제 2 (확정) — 3D Shape Representations [Difficulty: Hard, 18 pts]

For the three 3D shape representations covered in the lecture, give a simple **description** and state one key **advantage** and one key **disadvantage** of each. (Point Cloud 6 pts, Voxel 6 pts, Mesh 6 pts)

### 답안

#### (a) Point Cloud

- **Description**: 순서 없는 3D 점들의 집합 $\{(x_i, y_i, z_i)\}$.
- **Advantage**: 센서(stereo/depth)에서 **직접 얻기 쉬움**; 임의 형상을 **자유롭게** 표현.
- **Disadvantage**: **이웃·연결 관계 없음** → adjacency·surface·topology 정의 어려움; **두 point cloud 비교** 어려움.

#### (b) Voxel

- **Description**: 3D 공간을 **규칙적인 grid**의 부피 요소(volume element)로 이산화.
- **Advantage**: **이웃 관계가 명확**; 2D image의 3D 확장; **3D convolution**에 적합.
- **Disadvantage**: 메모리 **$O(n^3)$**; **Manhattan-world**(축 정렬) 편향.

#### (c) Mesh

- **Description**: **vertices + faces**(connectivity)로 표면을 정의.
- **Advantage**: **표면을 직접** 표현; **렌더링**에 이상적 (graphics 표준).
- **Disadvantage**: **고정 topology**로 reconstruction 어려움; self-intersection; 고해상도에 **vertex 수 증가** 필요.

# 3부. 시험 직전 체크리스트

### 문제 1 — Visual Hull

- [ ] Calibrated camera → **projection matrix**로 3D→2D 투영
- [ ] 각 뷰 **silhouette mask**에서 foreground/background 확인
- [ ] **모든 뷰**에서 실루엣 **안** → **occupied**; 하나라도 밖 → vacant
- [ ] 수식 없이 **절차**만 서술해도 됨

### 문제 2 — 3D 표현 비교

- [ ] **Point Cloud**: unordered points / 센서 획득 쉬움 / connectivity·비교 어려움
- [ ] **Voxel**: regular grid / neighbors·3D conv / $O(n^3)$ memory·axis bias
- [ ] **Mesh**: vertices+faces / surface·rendering / topology·self-intersection
