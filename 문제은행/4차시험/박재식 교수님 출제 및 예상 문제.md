## 1.5 확정문제별 심층 분석

### Q1 — Point-based vs Voxel-based

**시험이 검증하는 것**: 3D 표현의 **트레이드오프** 이해 — 정확도 vs 속도 vs 메모리.

| 관점 | Point-based | Voxel-based |
|------|-------------|-------------|
| **Accuracy** | 원래 좌표 보존 → **geometry 정확** | voxel quantization → **세밀 좌표 손실** |
| **Inference speed** | kNN/grouping 필요 → **느림** | grid conv/hash → **빠름** |
| **Memory** | 많은 point 직접 처리 → **큼** | sparse voxel만 저장 → **상대적 가벼움** |

### Q2 — Sparse Convolution

**문제**: 3D point cloud/LiDAR는 **대부분 빈 공간** → dense 3D conv는 empty voxel까지 연산 → 메모리·연산 폭증.

**해결**: **Occupied voxel만** sparse tensor로 저장 → hash/coordinate map으로 이웃 탐색 → **존재하는 voxel에만** kernel 적용.

**강의 연결**: Hash **O(N)** vs PointNet KNN **O(N log N)**. MinkowskiNet = sparse conv 표준 구현.

### Q3 — Per-scene MLP (grayscale image)

**시험이 검증하는 것**: Neural Implicit의 **Input/Output 정의** — 강의에서 “가장 중요”하다고 한 항목.

- **Input**: 픽셀 **(x, y) coordinate**
- **Output**: 해당 위치의 **pixel intensity**

→ 한 장의 grayscale 이미지를 MLP가 **함수로 memorization**(per-scene optimization).

### Q4 — Sine vs ReLU

**ReLU 한계**: **저주파**만 잘 표현 — 자연 이미지·3D는 고주파 디테일 많음.

**Sine 장점**:
- $\sin(Wx+b)$ 형태로 **주기·바이어스**를 자연스럽게 학습
- 층을 쌓으면 **basis function 조합**으로 복잡한 함수 표현
- 음성·이미지·비디오 등 **real-world 고주파 데이터** fitting에 유리

**강의 보강**: NeRF는 SIREN 대신 **Positional Encoding**으로 고주파 주입.

### Q5 — 왜 Gaussian인가?

| 이유 | 설명 |
|------|------|
| **Photorealistic rendering** | 사실적 렌더링 품질 |
| **가벼움·빠름** | NeRF/implicit/MLP 대비 **용량·속도** 우위 |
| **Explicit** | 파라미터가 **그대로 저장**·export (.ply), 합성 용이 |

**Q5가 안 묻는 것**: differentiable rasterization, clone/split/prune — **6번째·예상문제 후보**.

## 문제 1 (확정)

3D point cloud를 처리하는 대표적인 방식인 Point-based representation과 Voxel-based representation을 비교하시오. 각 방식의 특징을 **inference speed, accuracy, memory usage** 관점에서 서술하시오.

### 답안

**Point-based representation**은 point cloud의 원래 좌표를 직접 사용하여 각 point의 feature를 처리하는 방식이다. 점의 세밀한 위치 정보를 보존할 수 있기 때문에 상대적으로 정확한 geometry 표현이 가능하고, **높은 accuracy**를 기대할 수 있다. 하지만 point 간 이웃을 찾기 위해 kNN search나 grouping 연산이 필요하므로 **inference speed가 느릴** 수 있으며, 많은 point를 직접 처리해야 하므로 **memory usage도 큰** 편이다.

**Voxel-based representation**은 3D 공간을 일정한 grid 또는 voxel로 나눈 뒤, point들을 해당 voxel에 할당하여 처리하는 방식이다. voxel grid를 사용하면 convolution이나 hash-based neighbor search를 효율적으로 적용할 수 있어 **inference speed가 빠르고**, sparse voxel만 저장하면 **memory usage도 상대적으로 가볍다**. 하지만 여러 point가 하나의 voxel로 quantization되면서 세밀한 좌표 정보가 손실될 수 있으므로 point-based 방식에 비해 **accuracy가 낮아질** 수 있다.

## 문제 2 (확정)

3D 데이터에서 일반적인 dense convolution을 적용하기 어려운 이유를 설명하고, 이를 해결하기 위한 **Sparse Convolution**의 기본 원리와 수행 방식을 서술하시오.

### 답안

3D point cloud나 LiDAR 데이터는 대부분의 공간이 비어 있는 **sparse**한 데이터이다. 따라서 전체 3D 공간을 dense voxel grid로 만든 뒤 일반적인 3D convolution을 적용하면, 실제 point가 존재하지 않는 빈 공간까지 모두 계산해야 하므로 **메모리 사용량과 연산량이 매우 커진다**. 특히 3D convolution은 해상도가 증가할수록 계산량이 급격히 증가하기 때문에, 대규모 3D scene에 그대로 적용하기 어렵다.

**Sparse Convolution**은 이러한 문제를 해결하기 위해, 전체 voxel grid가 아니라 **point가 존재하는 occupied voxel에 대해서만** convolution을 수행한다. 먼저 3D point들을 voxel coordinate로 quantization하고, occupied voxel의 좌표와 feature만 **sparse tensor** 형태로 저장한다. 이후 hash table 또는 coordinate map을 이용해 각 occupied voxel 주변의 이웃 voxel을 빠르게 찾고, **실제로 존재하는 voxel feature에 대해서만** convolution kernel을 적용한다.

## 문제 4 (확정)

아래 그림에서 sine 함수를 ReLU 함수 대신 사용하면 좋은 점이 무엇인지 설명하시오.

### 답안

$\sin(Wx+b)$의 형태이기 때문에 자연스럽게 **다차원 함수의 주기와 바이어스**를 학습하게 된다. 여러 레이어를 쌓게 되면 데이터를 최적으로 표현할 수 있는 **basis function들의 조합**으로 함수를 표현할 수 있다. 따라서 real-world data(음성, 이미지, 비디오 등)의 처리가 훨씬 용이해진다.

(ReLU MLP는 **저주파**만 잘 표현하는 반면, sin activation은 **고주파 디테일** 표현에 유리하다는 점을 보충하면 좋다.)

## 예상문제 1 — NeRF (★ 최우선)

NeRF(Neural Radiance Fields)가 수행하는 task가 무엇인지 설명하고, **한 장의 이미지만으로는** 3D 복원이 어려운 이유와 **multi-view**가 필요한 이유를 서술하시오. NeRF의 기본 **input/output**과 학습·렌더링 파이프라인을 함께 설명하시오.

### 모범 답안

**Task — Novel View Synthesis**

- 여러 각도에서 촬영한 2D 이미지들로부터, **학습에 없던 새 카메라 시점**에서의 이미지를 합성하는 것이 목표이다.

**한 장만으로 어려운 이유**

- 단일 이미지에서 카메라 ray 위 **어디에 물체가 있는지(depth)** 가 모호하다.
- **Multi-view triangulation**으로 여러 시점의 일관된 기하를 추정해야 3D를 복원할 수 있다.

**Input / Output**

- **Input**: 3D 좌표 $(x,y,z)$ + **시선 방향(view direction)**
- **Output**: 해당 위치의 **RGB** + **density(σ)**

**파이프라인**

1. MLP가 공간상 각 점의 RGB·density를 예측한다.
2. **Volume rendering**(ray marching + alpha blending)으로 2D 이미지를 합성한다.
3. 합성 이미지와 GT 이미지의 **MSE loss**로 MLP를 업데이트한다.
4. 학습은 **per-scene optimization** — 하나의 씬을 네트워크에 **memorization**하는 방식이다.

**한계 (Q5와 연결)**

- 렌더링이 **느리고**(수 초/프레임), 표현이 **implicit**하여 mesh·point로 **export가 어렵다** → 이후 3D Gaussian Splatting으로 발전.

## 예상문제 2 — Explicit vs Implicit Representation

3D scene을 표현하는 **Explicit representation**과 **Implicit representation**을 비교하시오. 각각의 저장 방식, 장단점, 대표 예시를 서술하고, 3D Gaussian Splatting이 **Explicit**에 해당하는 이유를 설명하시오.

### 모범 답안

| | Explicit | Implicit |
|---|----------|----------|
| **저장 방식** | Point cloud, voxel, **가우시안 파라미터** 등 **배열·직접 값** | **신경망(MLP)** 이 좌표→값 함수를 학습 |
| **장점** | 바로 로드·렌더·**export**(.ply 등), 조작·합성 용이 | 연속 query, **압축**(weights만 저장) |
| **단점** | 점·가우시안 수에 따라 **메모리↑** | 사용하려면 **network inference** 또는 후처리 필요 |
| **대표 예** | Point Cloud, Voxel, **3DGS** | **NeRF**, SDF MLP |

**3DGS가 Explicit인 이유**

- 각 가우시안의 position, rotation, scale, color(SH), opacity가 **학습 가능한 파라미터**로 **직접 저장**된다.
- `.ply` 등으로 **즉시 export** 가능하고, 가우시안끼리 **union**으로 씬 합성이 쉽다.
- NeRF처럼 “네트워크 안에만 존재”하는 것이 아니라 **명시적 primitive**이다.

## 예상문제 3 — 3D Gaussian Splatting 학습 원리 (Q5 심화)

3D Gaussian Splatting(3DGS)의 **학습 파이프라인**을 설명하시오. Differentiable rasterization의 역할과, **Adaptive density control**(clone, split, prune)이 각각 어떤 상황에서 수행되는지 서술하시오.

### 모범 답안

**학습 파이프라인**

1. **SfM/COLMAP** 포인트(또는 랜덤)로 가우시안 **초기화**
2. **Differentiable rasterizer**로 현재 가우시안들을 2D 이미지로 렌더링
3. GT 멀티뷰 이미지와 **photometric loss**(MSE) 계산
4. **Adam**으로 모든 가우시안 파라미터(position, rotation, scale, SH, opacity) 업데이트
5. **Adaptive density control**로 가우시안 개수·밀도 조절

**Differentiable rasterization**

- GPU **rasterization·alpha blending**을 미분 가능하게 만들어, 렌더링 오차가 각 가우시안 파라미터로 **역전파**되게 한다.
- NeRF의 느린 ray marching + MLP inference 대신, GPU가 잘 하는 연산을 활용해 **실시간(100+ FPS)** 에 가깝게 렌더링한다.

**Adaptive density control**

| 연산 | 조건 | 목적 |
|------|------|------|
| **Clone** | gradient 큼 + **scale 작음** | 디테일 부족 영역에 가우시안 **복제** |
| **Split** | gradient 큼 + **scale 큼** | 큰 가우시안을 **분할**해 표현력↑ |
| **Prune** | opacity 낮음·불필요 | **제거**해 floater·노이즈 감소 |

**보충**: SfM 초기화가 랜덤보다 **배경·원거리** 복원에 유리하다.

# 5부. 시험 직전 체크리스트

### 확정 5문제 (반드시 서술 연습)

- [ ] Point: **accuracy↑** speed↓ memory↑ / Voxel: **speed↑** memory↓ accuracy↓(quantization)
- [ ] 3D **sparse** → dense conv 비효율 / Sparse conv = **occupied voxel만**
- [ ] Image implicit: input **(x,y)**, output **intensity**
- [ ] **sin vs ReLU**: 고주파·basis / ReLU = 저주파
- [ ] GS why Gaussian: **photorealistic · 가벼움·빠름 · explicit**

### 예상 6번째 (우선 대비)

- [ ] NeRF = **novel view synthesis** · multi-view **triangulation**
- [ ] NeRF I/O: **(x,y,z,view_dir) → RGB+density** · **volume rendering**
- [ ] **Per-scene optimization** = memorization
- [ ] Explicit vs Implicit · 3DGS = explicit **.ply export**
- [ ] 3DGS: **differentiable rasterization** + clone/split/prune

### 보조 대비

- [ ] **2DGS** = 2D disk, 표면/mesh, floater↓
- [ ] **7 DoF** = XYZ + WDH + yaw
- [ ] **PointPillars** = pillar → pseudo-image → 2D detector
- [ ] SDF: 표면=0, **Eikonal** |∇SDF|=1
- [ ] Fitting 목적 = **overfitting이 의도**, validation 불필요
