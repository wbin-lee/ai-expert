# 1부. 확정문제 4선 — 깊은 분석

## 1.1 확정문제 한눈에

| 문제 | 주제 | 강의 출처 | 유형 |
|------|------|-----------|------|
| **Q1** | Single RGB CNN 한계 + Two-Stream 보완 | 260618 오전 (3-4절) | 개념 서술 — **문제 정의·한계·해결 구조** |
| **Q2** | Optical flow 정의, $2L$ 채널, RGB 대비 정보 | 260618 오전 (3-2, 3-4절) | 개념 서술 — **정의 → 수치(채널) → 의미** |
| **Q3** | CNN vs ViT — 데이터 규모에 따른 성능 차이의 구조적 이유 | 260601 오전 (1-2, 1-6절) | 개념 서술 — **양쪽 아키텍처 대비 + 규모** |

## 1.2 확정문제가 말해 주는 출제 철학

### (1) 계산이 아니라 **“왜 그렇게 설계했는가”**

4문제 모두 수치 계산 없이, **구조적 동기(motivation)** 와 **정보의 종류(appearance vs motion, local bias vs data scale)** 를 묻는다. 답안도 “한계 → 보완 메커니즘 → 결과”의 인과 체인으로 쓰는 것이 핵심이다.

### (2) 강의의 ⭐ **시험 포인트** 와 1:1 대응

| 확정 Q | 강의 문서의 시험 포인트 표기 | 교수님이 요구하는 답의 뼈대 |
|--------|------------------------------|----------------------------|
| Q1 | 3-4절 "Single frame 한계 → Two-Stream" | appearance only → motion 부재 → spatial+temporal **complementary** |
| Q2 | 3-2절 "Optical flow & $2L$" | $(u,v)$ 정의 → $L$장 스택 → 배경 억제·움직임 강조 |
| Q3 | 1-6절 "CNN vs ViT" | locality·weight sharing·equivariance vs positional encoding·약한 inductive bias·**data scale** |
| Q4 | 1-2·1-4절 "VGG & ResNet" | effective RF·파라미터·non-linearity / degradation ≠ overfitting·$H(x)=F(x)+x$ |

→ **교수님은 “강의에서 ⭐로 표시한 문장”을 거의 그대로 서술형으로 변환한다.**

### (3) 커버리지 — 확정 4문제가 이미 소진한 영역

| 영역 | 확정에서 다룬 내용 | **아직 미출제** |
|------|-------------------|----------------|
| CNN·ViT 기초 | FC 한계, VGG, ResNet, ViT vs CNN | Pooling, GoogLeNet 1×1, Swin/PVT 개념, FC↔Conv |
| Action Recognition 기초 | Single frame, Two-Stream, optical flow | 3D CNN·R(2+1)D, SlowFast, TSM, Non-local, TimeSformer/ViViT |
| 데이터셋 | (Q1·Q2 맥락에서 간접) | Something-Something, trimmed/untrimmed, ActivityNet |
| 비디오 테스크 | — | **TAD, VOS/Tracking, Localization vs Tracking 구분** |
| Representation | — | Video MAE, ST-Adapter, LoRA, UniFormer/UniFormer2 |

### (4) 260619 오후 교수님 코멘트 — **남은 2문제의 최강 단서**

행정실 요청으로 구체 수치(“100문항” 등)는 피하라 했지만, 교수님이 직접 강조한 방향은 명확하다.

> *“테스크들의 집중을 해주셔도 좋지 않을까”*  
> *“테스크의 목적이 있잖아요. 거기서 중요한 포인트들을 이해하시면 좋지 않을까”*

교수님이 예시로 든 **3가지 축**:

| 테스크 유형 | 입력·타겟의 특성 | 방법론에 반영되는 점 |
|-------------|------------------|----------------------|
| **Action Localization (TAD 등)** | 비디오 전체가 입력, **찾을 액션 클래스는 사전에 정해짐** | 2D object detection을 **시간 1축**으로 옮긴 formulation → DETR류 learnable query, actionness |
| **Tracking / VOS** | **초기 target(mask/box)이 주어지면** 그때부터 테스크가 정의됨. 쫓을 대상이 **영상마다 다름** | memory로 이전 mask·feature 전달, (과거) online adaptation, appearance drift 대응 |
| **Action Recognition** | (확정 Q1·Q2) trimmed clip 전체 → 클래스 1개 | appearance+motion 분리(Two-Stream), flow $2L$ 채널 |

**핵심 메시지**: 같은 “비디오”라도 **테스크가 무엇을 조건으로 주느냐**(전체 탐색 vs 초기 target vs 전체 분류)에 따라 문제 formulation과 기법이 달라진다. 남은 2문제는 이 축에서 나올 가능성이 가장 높다.

# 2부. 출제 예상 개념 정리 (출제 포인트 중심)

## ★★★ ① Tracking·VOS vs Action Localization — 테스크 특성 비교

**출제 포인트 (강의 260618 오전, 260619 오후)**  
교수님이 시험 직전에 **가장 구체적으로** 언급한 비교 주제.

| 구분 | Action Localization / TAD | Tracking / VOS |
|------|---------------------------|----------------|
| 관심 대상 | **어떤 액션**이 **언제**(·어디서) 일어났는가 | **어떤 객체**가 시간에 따라 **어떻게 움직이는가** |
| 타겟 정의 | 찾을 **액션 종류가 데이터셋·테스크에 고정** | **초기 프레임의 box/mask**가 주어지면 그 객체가 타겟 |
| 입력 역할 | 비디오를 **그대로 스캔**해 이벤트 구간·위치 출력 | 초기 조건 + 이후 프레임 → **조건부** 추적 |
| 출력 | 시간 구간 $[t, t+d]$ + class (TAD) 또는 spatio-temporal tube | 프레임별 box 또는 **mask 시퀀스** |
| STAD vs Tracking | STAD는 “어디서 액션이 일어났는지”를 **탐색** | Tracking은 “**이미 정해진** 객체의 위치 변화”를 **따라감** |

**왜 이게 시험에 나오기 좋은가**  
- 표면적으로 STAD와 Tracking이 비슷해 보이지만, 교수님이 **“두 가지가 좀 다르다”** 고 반복 강조.  
- 서술형으로 “테스크 정의 → 방법론 차이”를 묻기에 최적 (확정 Q1과 동일한 **한계·보완** 서술 패턴).

## ★★★ ② Temporal Action Detection (TAD) ≈ 1D Object Detection

**출제 포인트 (강의 260619 오전 5-4절)**

- TAD: untrimmed 비디오에서 **어느 time interval에 어떤 액션**이 있는지 찾기.
- **문제 formulation = 2D object detection을 1D(time)로 치환**한 것 (교수님: “거의 같은 문제”).
- 최근 주류: **DETR/TubeR** 스타일  
  - **Learnable action query** + encoder feature와 **cross-attention**  
  - 출력: **시작 시각 $t$**, **duration $d$**, **class**  
  - **Actionness score**: 배경·비액션 proposal 필터 (DETR의 objectness와 유사)  
  - 고정 개수 proposal → 실제 액션 수와 불일치 → actionness로 걸러냄

**Encoder–Decoder 공통 패턴** (강의 전반)

```
입력 → Encoder backbone → Task-specific Decoder → 구조화된 출력
```

| 2D | 비디오 테스크 |
|----|---------------|
| Image classification → class 1개 | Action Recognition |
| Object detection → box + class | TAD → segment + class |
| Segmentation → mask | VOS → mask 시퀀스 |

## ★★★ ③ Video Object Segmentation (VOS) — XMem과 Memory

**출제 포인트 (강의 260619 오전 5-4절)**

- **초기 프레임 mask**가 주어지면 이후 프레임에서 **같은 객체 segmentation 유지**.
- Tracking 특성 반영: **“내가 받은 mask가 무엇이냐”** 가 핵심 정보 → **memory**로 전달.
- **XMem 3단계 memory**:
  - **Sensory memory**: RNN/LSTM으로 **직전 hidden state** 전달 (암묵적·즉각적)
  - **Working memory**: 최근 프레임 **key–value** (value = **mask representation**, feature가 아닌 mask 인코딩)
  - **Long-term memory**: working memory overflow 시 **prototype으로 압축** 저장
- 과거 **online lightweight adapter**: feature encoder가 약해 test-time에 비디오별 적응 필요했음.  
  요즘은 강한 encoder + **memory 기반 mask 전달**이 핵심 (online 학습으로 training sample 쌓지 않음).

**시험에 나올 때의 답 방향**  
- “왜 memory가 필요한가?” → 초기 target 조건 + appearance drift(포즈·각도 변화)  
- “value에 mask를 넣는 이유?” → appearance만 비교하면 drift에 취약, **mask representation**으로 aggregation

## ★★☆ ④ UniFormer2 — Pretrained ViT 확장

**출제 포인트 (강의 260619 오후 6-1절)**

기존 UniFormer video는 이미지 UniFormer **전체**를 temporal inflation해야 해서 **BERT-style ViT 생태계**를 못 씀.

**UniFormer2**:
1. **Pretrained ViT** encoder 활용 (freeze 또는 기반)
2. Temporal adapter 자리에 **Local MHRA** (depthwise conv = 고정 local aggregation pattern)
3. **Global Uni-Block**: frame feature를 query로 **전체 spatio-temporal feature**와 global cross-attention → adapter만으로 부족한 **통합 contextualization**

→ 확정 Q3(CNN vs ViT)과 연결: **대규모 pretrained ViT를 비디오에 효율 이전**하는 설계 철학.

---

## ★★☆ ⑤ Action Recognition 후속 — SlowFast, TSM, 3D CNN

**출제 포인트 (강의 260618 오전 3-5~3-6절)** — 확정 Q1·Q2의 **자연스러운 확장**

| 방법 | 핵심 | Two-Stream과의 관계 |
|------|------|---------------------|
| **3D CNN (C3D)** | spatial+temporal 동시 conv | 연산·데이터 heavy, spatio-temporal 패턴 공간 커버 한계 |
| **R(2+1)D** | 3D conv → **2D spatial + 1D temporal** factorization | data-efficient |
| **SlowFast** | **Slow**(저FPS·고채널)=appearance, **Fast**(고FPS·저채널)=motion, 중간 fusion | Two-Stream을 **한 네트워크·다른 시간 해상도**로 구현 |
| **TSM** | 2D CNN 유지, **채널별 temporal shift**로 이웃 프레임 정보 교환 | 거의 추가 연산 없이 temporal modeling |
| **Non-local** | 모든 위치 쌍 관계 ≈ **self-attention** | |

**Something-Something** (데이터셋 포인트): single frame으로 불가능한 **모션·방향** 클래스 → “비디오가 정말 필요한가?” 검증 벤치마크.

---

## ★★☆ ⑥ Representation Learning — Video MAE, ST-Adapter, LoRA

**출제 포인트 (강의 260619 오전 5-5~5-6절)** — 우선순위는 테스크 특성보다 낮으나 서술형 가능

- **Video MAE**: 패치 마스킹 후 복원 → 맥락 이해. 픽셀 복원보다 **feature(motion 등) 예측**이 downstream에 유리.
- **ST-Adapter**: frozen ViT 사이 **lightweight adapter** (spatial → temporal). full fine-tune보다 적은 파라미터.
- **LoRA**: $W' = W + BA$, low-rank update만 학습.

## 예상문제 1

**Spatio-Temporal Action Detection**과 **Video Object Segmentation(VOS)** 은 모두 비디오에서 시간에 따른 변화를 다루지만, 교수님 강의에서 강조한 **테스크 정의**와 **입력 조건**의 관점에서 두 테스크가 어떻게 다른지 설명하고, 그 차이가 각 테스크의 방법론 선택에 어떻게 반영되는지 서술하시오.

### 모범 답안

**테스크 정의의 차이**

- **Spatio-Temporal Action Detection (또는 Temporal Action Localization)**  
  - 입력: **전체 untrimmed 비디오**.  
  - 목표: 데이터셋에 정의된 **액션 클래스** 중 어떤 것이 **어느 시공간 구간**에서 일어났는지 **탐색(search)**.  
  - 타겟 액션의 **종류는 테스크·데이터셋에 의해 미리 정해져 있음** (교수님: “타겟으로 찾고 있는 액션들의 종류가 정해져 있다”).  
  - 관심 대상은 **액션(event)**.

- **Video Object Segmentation (VOS)**  
  - 입력: 비디오 + **초기 프레임의 segmentation mask**(또는 box).  
  - 목표: **주어진 그 객체**의 mask를 이후 모든 프레임에서 **유지·추적(track)**.  
  - **쫓아야 할 객체는 영상마다 다르며**, 초기 조건으로 그때서야 테스크가 정의됨 (교수님: “이니셜하게 뭔가 주어지면 인풋에서 테스크가 정의된다”).  
  - 관심 대상은 **특정 object**의 위치·형태 변화.

**방법론에의 반영**

- **Localization / Detection 계열**  
  - 비디오 전체를 스캔하는 **detection formulation**이 자연스럽다.  
  - Temporal Action Detection은 2D object detection을 **시간 1축**으로 옮긴 문제로 보며, **learnable action query + cross-attention**(DETR/TubeR류)으로 time segment $(t, d)$와 class를 예측한다.  
  - 고정 개수 proposal 중 배경 구간을 걸러내기 위해 **actionness score**를 사용한다.

- **Tracking / VOS 계열**  
  - “어떤 객체를 쫓을지”가 이미 주어졌으므로, 전역 탐색보다 **초기 mask 정보를 후속 프레임에 전달**하는 것이 핵심이다.  
  - **Memory** 구조(sensory / working / long-term)로 이전 프레임의 **mask representation**과 feature를 저장·조회한다.  
  - 객체 포즈·외형이 바뀌면(initial appearance만 비교하면) **drift**가 발생하므로, working memory의 **value에 mask 인코딩**을 넣어 appearance 변화에 robust하게 aggregation한다.  
  - (과거) feature encoder가 약할 때는 **online adapter**로 비디오별 적응을 했으나, 현재는 강한 encoder + **memory 기반 mask 전달**이 주류이다.

**한 줄 요약**: Localization은 “**정해진 액션 클래스를 비디오에서 찾기**”, VOS/Tracking은 “**주어진 객체를 시간 따라가기**”이며, 이 입력 조건의 차이가 detection-style decoder vs memory-style 전달 구조로 이어진다.

## 예상문제 2

**Temporal Action Detection(TAD)** 을 2D object detection과 비교하여, (1) 문제 formulation이 어떻게 대응되는지, (2) DETR 기반 TAD에서 action query·time segment regression·actionness score가 각각 어떤 역할을 하는지 설명하시오.

### 모범 답안

**(1) 2D Object Detection과의 formulation 대응**

| 2D Object Detection | Temporal Action Detection |
|---------------------|---------------------------|
| 입력: 이미지 | 입력: 비디오(또는 프레임 시퀀스 feature) |
| 공간 축에서 **위치** 탐색 | **시간 축**에서 **구간(interval)** 탐색 |
| 출력: bounding box $(x, y, w, h)$ + class | 출력: **시작 시각 $t$** + **duration $d$** + class |
| “어디에 무엇이 있는가” | “**언제부터 얼마나** 어떤 액션이 있는가” |

교수님 설명대로 TAD는 spatial dimension의 box detection을 **1D time segment detection**으로 치환한 것이다. 템포럴 차원이 추가되었을 뿐, 본질적으로 **detection = 후보 proposal + 클래스·위치(구간) 회귀 + 배경 제거**라는 구조는 동일하다.

**(2) DETR 기반 TAD 구성 요소**

- **Encoder**: 비디오(프레임)를 feature로 인코딩. 프레임 단위 feature + **positional encoding**으로 temporal 정보 제공.

- **Action query (learnable query)**  
  - DETR의 object query와 같이, 학습 가능한 query가 encoder feature와 **cross-attention**을 반복하며 **액션 후보(segment) 표현**을 refine한다.  
  - 고정 개수의 query가 **여러 time segment proposal**을 동시에 담당한다.

- **Time segment regression ($t$, $d$)**  
  - 각 query에서 **시작 시각 $t$** 와 **지속 시간 $d$** 를 예측 → time interval proposal.  
  - 2D detection의 bounding box regression $(x,y,w,h)$ 에 대응한다.  
  - 초기 $t$는 랜덤에 가깝게 주어지고, decoder layer를 거치며 점진적으로 refine된다.

- **Classification head**  
  - 각 segment proposal에 **액션 클래스 score** 부여. box의 class prediction과 동일한 역할.

- **Actionness score**  
  - DETR이 고정 개수 query를 출력하므로, 실제 비디오의 액션 수와 proposal 수가 맞지 않을 수 있다.  
  - 일부 proposal은 **배경·비액션 구간**일 수 있음 → **actionness**(DETR의 objectness와 유사)로 “이 구간에 실제 액션이 있는가”를 판별·필터링한다.  
  - 제안된 time interval에 해당하는 feature를 **RoI-align(1D)** 등으로 모아 regression한다.

**핵심**: TAD는 “비디오 이해”라는 포장 아래 **1D detection**이며, DETR의 query–cross-attention–set prediction 구조가 그대로 이식된다.

## 예상문제 3

**Video Object Segmentation(VOS)** 에서 초기 프레임의 mask가 주어졌을 때, XMem이 이후 프레임에서 segmentation을 유지하기 위해 사용하는 **sensory memory, working memory, long-term memory**의 역할을 각각 설명하고, working memory에서 **key와 value가 서로 다른 정보**(feature vs mask representation)를 담는 이유를 설명하시오.

### 모범 답안

**VOS 테스크 전제**

- 초기 프레임에 **어떤 객체의 mask**가 주어지면, 이후 프레임마다 **동일 객체의 mask**를 예측해야 한다.  
- Tracking 특성상 “**받은 mask가 무엇이냐**”가 핵심 조건이며, 단순히 매 프레임 독립 분류하는 recognition과 다르다.

**3단계 memory의 역할**

| Memory | 역할 | 직관 |
|--------|------|------|
| **Sensory memory** | **직전 스텝** 정보를 **RNN/LSTM hidden state**로 전달 | 방금 전 프레임 맥락의 **즉각적·암묵적** 전달. LSTM cell state처럼 모델 내부에 녹아 있는 단기 기억 |
| **Working memory** | **최근 몇 프레임**의 **key–value** 쌍을 명시적으로 저장 | 현재 query frame feature가 memory의 key들과 **cross-attention** → 관련 과거 정보 aggregation |
| **Long-term memory** | working memory가 **overflow**할 때, 과거 정보를 **prototype 등으로 압축**해 장기 보관 | 긴 비디오에서도 **메모리 한도**를 넘지 않으면서 먼 과거 참조 |

**Working memory에서 key ≠ value인 이유**

- **Key**: 해당 프레임의 **visual feature** (query와 유사도를 잴 대상).  
  “지금 프레임과 **어느 과거 프레임이 관련 있는가**”를 찾기 위함.

- **Value**: 해당 프레임의 **mask representation**(mask를 인코딩한 표현).  
  단순히 initial frame feature와 **appearance 유사도**만 비교하면, 객체 **포즈·각도·가림**이 바뀔 때(appearance drift) 추적이 실패한다.  
  따라서 attention으로 선택된 과거 위치에서 가져올 정보는 **“그때 그 객체가 어디였는지(mask)”** 이어야 한다.

- 디코더는 query feature로 memory를 조회한 뒤, aggregate된 **mask representation**을 읽어(readout) 현재 프레임 mask를 예측한다.  
  → **feature로 매칭, mask로 전달**이라는 분업이 drift에 robust한 tracking을 가능하게 한다.

**과거 vs 현재 접근**

- 예전: feature encoder가 약해 **online lightweight adapter**로 비디오마다 test-time 적응.  
- 현재(XMem 등): **강한 pretrained encoder** + **memory로 mask 정보 전달**. memory에 쌓은 것을 training sample로 재학습하지는 않지만, **이전 히스토리를 조회**해 예측에 활용한다.

# 4부. 시험 직전 체크 — 확정 4문제 + 예상 3문제

### 확정 (반드시 암기·서술 연습)

- [ ] Single RGB: appearance only, motion 없음 → Two-Stream complementary fusion  
- [ ] Optical flow: $(u,v)$, $L$장 → $2L$ ch, 배경 억제·움직임 강조  
- [ ] CNN: locality, weight sharing, translation equivariance, 작은 데이터 안정  
- [ ] ViT: patch token, positional encoding, 약한 inductive bias, 대규모 data·pretrain  
- [ ] VGG: $3\times3$ stack → effective RF, 적은 파라미터, 층간 non-linearity  
- [ ] ResNet: degradation ≠ overfitting, $H(x)=F(x)+x$, identity mapping 용이  

### 예상 (남은 2슬롯 — 우선 대비)

- [ ] **Localization vs Tracking**: 액션 클래스 고정·전역 탐색 vs 초기 mask/box·조건부 추적  
- [ ] **TAD = 1D detection**: box ↔ $(t,d)$, action query, actionness  
- [ ] **XMem memory**: sensory(RNN) / working(KV) / long-term(압축), value=mask representation  
- [ ] (보조) UniFormer2: pretrained ViT + local MHRA + global uni-block  
- [ ] (보조) SlowFast·TSM: appearance/motion 경로, temporal shift
