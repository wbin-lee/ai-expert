## 1. 확정문제 + 작년 기출 분석

### 1.1 확정문제 스타일

| 확정문제 | 유형 | 출제 포인트 |
|---|---|---|
| 문제 1. CNN → FCN 변환 | **원리 서술형** ("왜 가능한지 + 과정 설명") | FC layer를 컨볼루션으로 "해석"하는 발상 → 임의 크기 입력 + per-pixel argmax |
| 문제 2. FCN의 한계와 해결책 | **한계 + 해결책 매핑형** (사실상 ablation) | 한계 2가지(해상도↓, RF 고정)를 쓰고, 각 기법이 **어떤 문제를** 해결하는지 연결 |
| 문제 3. Conv vs Self-attention | **다관점 비교형** | 지정된 3가지 관점(receptive field / 입력 의존성 / parameter sharing)에서 비교 |

### 1.2 작년(2025) 기출과의 대응 — 결정적 단서

작년 시험은 3문제였고, 올해 확정문제와 다음과 같이 대응된다.

| 2025 기출 | 내용 | 2026 확정문제 대응 |
|---|---|---|
| 기출 1 | FCN의 두 한계(A: small RF / B: coarse upsampling)에 적합한 해결책 선택 — A: ②SPP·④Atrous, B: ①Deconv network·③Skip connection | **확정문제 2와 동일 주제** (선택형 → 서술형으로 변환) |
| 기출 2 | **U-Net gradient ablation**: (1) 모든 skip connection(A)의 gradient가 0 / (2) 디코더 깊은 경로 지점(B)의 gradient가 0 / (3) 둘 다 — 일 때 예상 출력을 보기에서 선택 (힌트: 학습 데이터 대부분 픽셀은 background) | **대응 없음 ← 4번째 문제 최유력** |
| 기출 3 | Global RF를 가지면서 파라미터가 입력 크기와 무관한 연산은? → **Self-attention** | **확정문제 3과 동일 주제** (선택형 → 비교 서술형으로 변환) |

**작년 기출 2번의 정답 로직** (그대로 다시 나올 수 있으므로 암기 수준으로):

- **Case 1 (skip만 죽음)** → 깊은 경로(전역 맥락)는 학습됨 → **클래스는 맞지만 경계가 뭉뚝한 블롭 마스크** (작년 보기 2: "dog" 클래스 정답, 모양 뭉뚝).
- **Case 2 (깊은 경로만 죽음)** → 얕은 skip 경로만 학습 → **모양·경계는 비교적 살아있지만 RF가 작아 클래스를 오분류** (작년 보기 1: 모양은 있되 "cat" 오답).
- **Case 3 (둘 다 죽음)** → 유효한 피처가 출력단에 도달하지 못함 → 모델은 prior(다수 클래스)로 수렴 → **전부 background** (작년 보기 4: 전부 검정).

**패턴 정리:**

1. **계산 문제가 하나도 없다.** 확정 3문제 모두 서술형이고, 작년 기출도 "이게 빠지면/죽으면 어떻게 되는가"식 선택형. 조교 출제 가이드("왜 중심, 암기보다 이해", "어블레이션 패턴", "RF·파라미터 수 계산은 안 나옴")와 정확히 일치한다.
2. **공통 뼈대 = "무엇이 문제였고, 어떤 구조가 왜 그것을 해결하는가"** + **"그 구조가 고장나면 출력이 어떻게 되는가"** (작년 기출 2).
3. **작년 3문제 중 2문제가 올해 확정문제로 재활용**되었다(주제 기준). 남은 작년 기출 = **U-Net skip connection gradient ablation** → 올해 4번째 문제의 최유력 후보.
4. 작년은 보기 선택형이 섞여 있었으므로, 4번째 문제가 **선택형(고르고 이유 설명)** 으로 나올 가능성도 대비할 것.

## 2. 출제 예상 개념 정리 (우선순위 순)

### ① U-Net Skip Connection — 역할과 고장(ablation) 시 출력 ★★★ (작년 기출 직접 근거)

작년 기출 2번이 정확히 이 주제이고, 올해 확정문제에 대응이 없으며, 조교 출제 가이드에도 **"FCN vs U-Net 비교"(#3)와 "Skip Connection 항을 제거하면 어떻게 되는가"(#8)** 두 항목으로 겹쳐 등장한다. 모든 신호가 이 주제를 가리킨다.

- **역할 1 — 정보 결합**: 인코더의 같은 해상도 레이어 피처를 디코더로 직접 전달. *"디스크리미네티브하지만 정보가 손실된 피처(아래에서 올라오는)와, 손실되기 전 공간 정보를 가진 피처(인코더에서 바로 오는)를 채널 방향 concat 후 컨볼루션으로 어떻게 섞을지 배워라."*
- **역할 2 — 학습 안정성**: 백프롭 시 그래디언트가 흐를 수 있는 **"고속도로"**. ResNet residual connection과 같은 원리.
- **고장 시나리오별 출력** (작년 기출 정답 로직): skip 죽음 → 클래스 정답 + 경계 뭉뚝 / 깊은 경로 죽음 → 모양 보존 + 클래스 오답 / 둘 다 죽음 → 전부 background.
- 관련: **Transposed convolution** = 디코더의 핵심 연산자 ($y=Wx$의 역연산을 $W^T$로 근사, stride를 주면 사이즈가 커짐).

### ② Positional Embedding — 없으면 어떻게 되는가 ★★★

조교 가이드 **#2 그대로**. 확정문제 3이 self-attention을 다뤘으므로 ViT 쪽에서 한 문제가 더 나온다면 가장 유력한 ablation.

- Self-attention은 모든 토큰 쌍의 유사도(query·key)만 보므로 **토큰의 순서/위치 정보가 전혀 없음** — 입력을 섞어도 출력이 같이 섞일 뿐(permutation equivariant).
- PE가 없으면 이미지가 **15퍼즐처럼 섞인 패치 집합(set)** 으로 취급됨 → "어깨 옆에 얼굴이 있다" 같은 공간 관계를 사용할 수 없음.
- 순서 자체는 임의여도 됨(raster scan, 거꾸로, 시계방향 모두 OK) — **일관되게 하나를 정해 외우게 하는 것**이 핵심.
- 순서를 정하면 안 되는 데이터: **LiDAR/Point Cloud, Graph** → 이 경우 절대 위치 대신 **상대 위치(RoPE)** 또는 순서 없는 처리.

### ③ Transformer Decoder의 동기 — 클래스 표현의 정제 ★★☆

교수님이 🔥🔥로 강조한 핵심 동기이며, 확정문제 3(인코더 비교)과 짝을 이루는 **디코더 쪽 비교 문제**로 변환하기 좋은 주제.

- 일반 classification head: 피처 $Z$ × 가중치 $W$($D\times K$) → $W$의 각 row = **클래스 대표 벡터 $Z_C$, 모든 이미지에 고정**.
- 문제 — **자동차 예시**: 버스/스포츠카/화물차는 완전히 다르게 생겨 하나의 고정 대표 벡터로 묶기 어려움. **겨울 나무 vs 여름 나무**: 겨울엔 "초록색" 피처를 쓸 수 없음.
- 해결: 클래스마다 학습 가능한 토큰을 디코더에 넣고 **self-attention으로 패치 토큰들과 정보 교환** → 클래스 토큰이 **이미지 맥락에 맞게 정제**됨 → 정제된 클래스 토큰과 패치 피처의 inner product로 픽셀별 스코어.
- F1 경기장 이미지면 자동차 토큰이 스포츠카 쪽으로, 공사 현장이면 화물차 쪽으로 정제.

### ④ ViT 패치 크기 trade-off ★★☆

교수님이 Q&A에서 "단골"로 다룬 주제. RF-trade-off라는 강의 관통 메시지의 트랜스포머 버전.

- 패치를 줄이면 성능은 좋아지지만 **토큰 수 N 증가 → self-attention 복잡도/메모리 O(N²) 폭증** (N이 10→100이면 메모리 100→10,000).
- 결론: **"메모리가 허용하는 가장 작은 패치"** 가 최적.
- 패치가 크면 작은 물체 마스크가 부정확 — 패치 내부 픽셀 구분은 트랜스포머 바깥(업샘플링)에서 일어나기 때문. 해결: 패치 줄이기 or convolution decoder.

### ⑤ Self-Attention = Hypernetwork 관점 ★★☆

확정문제 3의 "입력 의존성"을 한 단계 깊게 묻는 형태로 나올 수 있음.

- FC layer의 두 약점: **파라미터 $n^2$** + **$w_{ij}$가 입력과 무관하게 고정** (교수님 3배 확대 비유).
- Self-attention: $w_{ij}$를 다른 모델의 출력 $g(x_i, x_j)$로 — **가중치를 출력으로 내는 신경망 = hypernetwork**.
- 파라미터는 $W_Q, W_K, W_V$ 합쳐 $3DD'$ 뿐, **시퀀스 길이와 무관**한데 RF는 데이터 전체. (작년 기출 3번의 정답 근거가 정확히 이것.)
- Self-Attention 3관점: (1) 전역 맥락 인코딩 레이어, (2) hypernetwork, (3) FC의 효율적 대체.
- 트랜스포머만 scaling law를 만족하는 이유(교수님 추정): hypernetwork라서. In-context learning도 같은 관점으로 설명.

### ⑥ 기타 가능 포인트 ★☆☆

- **FCN 한계-해결책 매칭 정답 암기** (작년 기출 1 그대로): A. small receptive field → **Spatial pyramid pooling, Atrous convolution** / B. coarse score map 업샘플로 인한 shape 정보 손실 → **Deconvolution network, Skip connection**.
- **1×1 Convolution의 역할** (조교 가이드 #6): 공간 사이즈를 바꾸지 않고 채널 차원만 변환. FCN의 두 번째 FC 해석, ViT에서 $W_Q,W_K,W_V$ 구현 등.
- **Spatial Pyramid Pooling의 동기** (조교 가이드 #5): 풀링 윈도우 여러 개를 동시에 → 다양한 크기의 문맥, 극단적으로 RF = 이미지 전체.
- **Multi-Head Attention의 이유** (조교 가이드 #1): head마다 다른 관점(다른 유사도 기준)으로 attention → 한 종류의 attention map으로는 다양한 관계를 동시에 못 잡으므로 병렬로 나눠 표현력 확보.
- **Open-Vocabulary** (CLIP 원리): 클래스 벡터 대신 언어 모델 단어 임베딩 사용, 언어 모델은 고정 — "강아지"로 학습해도 "개"에 일반화.
- **후처리를 안 쓰는 이유**: end-to-end 학습 바깥에 있기 때문 + encoder-decoder로 출력이 충분히 세밀해짐.

## 3. 예상문제

### 예상문제 1 — U-Net Skip Connection 고장 시나리오 분석 (작년 기출 2번의 2026 버전, 최우선 대비)

아래는 semantic segmentation을 위한 U-Net 아키텍처이다. 인코더와 디코더 사이에는 여러 skip connection(A)이 있고, 인코더 최심부(bottleneck)를 지나 올라오는 디코더의 깊은 경로 위에 지점 (B)가 있다. 학습 과정에서 다음 세 가지 문제 상황을 고려하자.

- Case 1: 모든 skip connection (A)으로 흐르는 gradient가 0이 됨.
- Case 2: 디코더의 (B) 지점에서 gradient가 0이 됨 (깊은 경로로 학습 신호가 흐르지 않음).
- Case 3: (1)과 (2)가 동시에 발생함.

각 경우에 대해, 학습 종료 후 입력 이미지를 넣었을 때 예상되는 segmentation 출력이 어떤 특성을 보일지 설명하고 그 이유를 서술하시오. (힌트: 모든 학습 이미지에서 대부분의 픽셀은 background로 레이블되어 있다고 가정한다.)

#### 답안 (정답 예시)

- **Case 1 (skip만 죽음)**: skip connection 경로의 파라미터는 학습되지 않고, **인코더 최심부를 지나는 깊은 경로만 학습**된다. 깊은 경로는 RF가 커서 전역 맥락(semantics)을 충분히 인코딩하므로 **물체의 클래스는 올바르게 예측**한다. 그러나 다운샘플링으로 손실된 고해상도 공간 정보를 skip으로 보충받지 못하므로, 업샘플을 해도 없던 디테일이 살아나지 않아 **경계가 뭉뚝하고 세부 형태가 손실된 블롭(blob) 형태의 마스크**가 나온다. (작년 기출 보기 중 "클래스는 맞고 모양만 뭉뚝한 마스크"가 정답이었다.)
- **Case 2 (깊은 경로만 죽음)**: 이번에는 **얕은 인코더 레이어에서 skip으로 바로 건너오는 경로만 학습**된다. 얕은 피처는 다운샘플이 덜 되어 공간 정보가 풍부하므로 **물체의 윤곽·경계는 비교적 세밀하게** 잡힌다. 그러나 얕은 경로는 RF가 작고 depth가 얕아 semantic 수준이 낮으므로(말 얼굴을 가리고 몸만 보는 상황), **클래스를 오분류**할 수 있다. (작년 기출 보기 중 "모양은 살아있지만 클래스가 틀린 마스크"가 정답이었다.)
- **Case 3 (둘 다 죽음)**: 인코더에서 디코더로 유효한 피처가 전달되는 두 경로가 모두 학습되지 않으므로, 모델은 입력과 무관하게 loss를 줄이는 방향 — 즉 **학습 데이터의 prior로 수렴**한다. 대부분의 픽셀이 background로 레이블되어 있으므로, **모든 픽셀을 background로 예측하는 출력**이 나온다. (작년 기출 보기 중 "전부 배경"이 정답이었다.)

(보충 — 정상 학습 시 skip connection의 두 역할: ① 손실되기 전의 공간 정보를 디코더에 직접 전달해 경계를 정교하게 만들고(concat 후 conv로 섞는 법을 학습), ② 그래디언트가 흐르는 "고속도로"를 깔아 깊은 네트워크의 학습을 안정화한다. ResNet residual connection과 같은 원리.)

### 예상문제 2 — Positional Embedding 제거 ablation (조교 가이드 #2, "없으면 어떻게 되는가" 스타일)

Vision Transformer는 이미지를 패치 토큰으로 만든 뒤 positional embedding을 더해 self-attention 레이어에 입력한다.

(1) Positional embedding을 제거하면 모델이 입력 이미지를 어떻게 "보게" 되는지, self-attention 연산의 성질에 근거해 설명하시오.
(2) 패치의 순서(번호 부여 방식)는 어떻게 정해야 하는지 설명하시오.
(3) 순서를 일관되게 정하는 것이 불가능하거나 부적절한 데이터의 예를 들고, 그 경우의 대안을 설명하시오.

#### 답안 (정답 예시)

**(1)** Self-attention은 모든 토큰 쌍에 대해 query와 key의 유사도만으로 weight를 계산하므로, **토큰이 어느 위치에서 왔는지에 대한 정보를 전혀 사용하지 않는다**(입력 순서를 섞으면 출력도 똑같이 섞여 나올 뿐 결과 내용이 같다 — permutation equivariant). 따라서 positional embedding이 없으면 이미지는 공간 배치가 사라진 **"섞인 퍼즐 조각들의 집합(set)"** 으로 취급된다. "어깨 픽셀의 레이블을 정할 때 그 위에 얼굴이 있다" 같은 공간적 맥락을 쓸 수 없게 되어, 패치 배치가 중요한 segmentation 성능이 크게 떨어진다.

**(2)** 어떤 순서든 상관없지만 **항상 동일한 순서로 일관되게** 부여하면 된다. Raster scan이든, 역순이든, 가운데부터 시계방향이든 모델 입장에서는 동등하다. 순서를 정해주지 않는 것은 15퍼즐처럼 섞인 모든 상태를 같은 이미지로 인식하라고 요구하는 것과 같아서, 하나의 순서를 정해 모델이 외우게 하는 것이 훨씬 간단하다.

**(3)** **LiDAR/Point Cloud, Graph**처럼 원소에 자연스러운 순서가 없는 데이터는 일관된 순서를 정의하기 어렵다(그래프는 노드 1번을 누구로 정하느냐에 따라 인접행렬 자체가 달라진다). 이런 경우 절대 위치 인코딩 대신 **상대적 위치 관계(relative positional encoding, RoPE 계열)** 를 쓰거나 순서에 의존하지 않는 처리를 한다. 최근 트랜스포머들은 대체로 상대 위치 기반을 사용한다.

### 예상문제 3 — 고정 클래스 벡터 vs Transformer Decoder (확정문제 3의 비교형 스타일)

Segmentation에서 픽셀(패치)별 클래스 스코어를 만드는 두 방식을 비교하려 한다.

- 방식 A: 패치 피처에 고정된 가중치 행렬 $W$ ($D \times K$)를 곱하는 일반적인 classification head (MLP)
- 방식 B: 클래스마다 학습 가능한 토큰을 두고, Transformer decoder에서 패치 토큰들과 self-attention으로 정보를 교환한 뒤, 정제된 클래스 토큰과 패치 피처의 inner product로 스코어를 계산

(1) 방식 A에서 $W$의 각 row vector가 갖는 의미를 설명하고, 이 방식의 표현력 한계를 구체적 예와 함께 설명하시오.
(2) 방식 B가 이 한계를 어떻게 해결하는지 설명하시오.

#### 답안 (정답 예시)

**(1)** 피처 $Z$($D$-dim)에 $W$를 곱해 $K$개 클래스 스코어를 내는 것은, $W$의 각 row vector를 **해당 클래스를 대표하는 $D$차원 벡터 $Z_C$** 로 보고 피처와의 inner product(유사도)로 분류하는 것과 같다. 문제는 이미지가 바뀌어도 $Z_C$는 **모든 이미지에 대해 고정**이라는 점이다. 그러나 한 클래스의 외형은 이미지마다 크게 다르다 — 예컨대 "자동차" 클래스에는 버스, 스포츠카, 화물차처럼 서로 완전히 다르게 생긴 인스턴스가 섞여 있고, "나무"는 여름에는 초록 잎이 특징이지만 겨울에는 잎이 없어 그 피처를 쓸 수 없다. 이렇게 다양한 외형을 하나의 고정 대표 벡터로 묶어야 하므로 표현력이 제한된다.

**(2)** 방식 B는 클래스 토큰이 decoder의 self-attention을 통해 **현재 이미지의 패치 토큰들을 직접 참조하며 업데이트**되도록 한다. 즉 클래스의 표현을 **이미지(데이터) 맥락에 맞게 정제**한다. F1 경기장 이미지가 들어오면 자동차 클래스 토큰이 스포츠카에 맞는 표현으로, 공사 현장 이미지면 화물차에 맞는 표현으로 바뀌고, 겨울 풍경이면 나무 토큰이 잎 없는 나무에 맞게 조정된다. 이렇게 contextualize된 클래스 토큰과 패치 피처의 inner product를 쓰면, 고정 벡터로 분류할 때보다 이미지마다 달라지는 클래스 외형을 더 잘 구분할 수 있어 성능이 향상된다.

## 4. 보너스 — 추가로 준비해두면 좋은 단답 후보

4번째 문제가 위 3개를 빗나갈 경우를 대비한 한 줄 답안 (조교 출제 가이드 항목 위주):

1. **FCN 한계-해결책 매칭** (작년 기출 1 정답): small RF → **SPP, Atrous conv** / coarse upsampling의 shape 손실 → **Deconvolution network, Skip connection**.
2. **Global RF + 파라미터가 입력 크기와 무관한 연산은?** (작년 기출 3 정답): **Self-attention** — k×k conv는 RF가 local, 전체 포지션 MLP는 파라미터가 $n^2$로 증가, 포지션별 독립 MLP는 RF=1.
3. **Multi-Head Attention을 쓰는 이유는?** → 하나의 attention map은 한 종류의 관계(유사도 기준)만 포착함. head를 여러 개로 나누면 각 head가 서로 다른 관점의 관계(위치, 의미, 형태 등)를 병렬로 학습해 표현력이 풍부해짐.
4. **1×1 Convolution의 역할은?** → 공간 구조는 건드리지 않고 **채널 차원만 변환**(채널 간 weighted sum). FCN에서 두 번째 FC layer의 컨볼루션 해석이 1×1 conv이고, self-attention의 $W_Q, W_K, W_V$도 1×1 conv 3개로 구현 가능.
5. **Spatial Pyramid Pooling의 동기는?** → 풀링 윈도우 크기를 여러 개 두고 동시에 풀링해 **여러 스케일의 문맥**을 한 번에 인코딩. 글로벌 풀링까지 쓰면 RF가 이미지 전체가 됨. 풀링은 파라미터 없는 컨볼루션이므로 atrous conv로도 구현 가능.
6. **FCN vs Deconvolution Network(U-Net류)의 차이는?** → FCN은 낮은 해상도 score map을 단순 보간으로 업샘플(+중간 피처 sum)하지만, deconvolution network는 **학습되는 디코더**(transposed conv, unpooling)로 해상도를 점진적으로 복원해 경계가 세밀함.
7. **ViT에서 패치를 무작정 줄이지 못하는 이유는?** → 토큰 수 N이 늘면 self-attention 메모리/계산이 **O(N²)** 으로 폭증 (패치 16→1이면 N이 16²배). "메모리가 허용하는 가장 작은 패치"가 실용적 최적.
8. **데이터가 많아지면 convolution이 필요 없어지는 이유는?** → conv의 locality/translation invariance는 데이터가 적을 때 유리한 inductive bias일 뿐, 데이터가 충분하면 트랜스포머가 그 구조를 데이터에서 직접 배우므로 성능 차이가 사라짐.
9. **Open-vocabulary segmentation의 원리는?** → 클래스마다 학습하는 벡터 대신 **고정된 언어 모델의 단어 임베딩**을 클래스 벡터로 사용(CLIP 방식: 시각-언어 표현 정렬). "강아지"로 학습해도 임베딩이 비슷한 "개"에 일반화됨.
10. **요즘 후처리(CRF 등)를 안 쓰는 이유는?** → encoder-decoder로 출력이 충분히 세밀해졌고, 후처리는 모델 학습 바깥에서 돌아 **end-to-end learning을 깨기 때문**.
