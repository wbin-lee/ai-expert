# 1부. 확정문제 + 작년 기출 분석 — 출제 스타일의 패턴

## 1.1 확정 3문제의 공통점

| 문제 | 주제 | 출처(강의) | 유형 | 강의 내 출제 신호 |
|---|---|---|---|---|
| 1 | Iron Law of CPU Performance | 5/26 (1.5절) | 정량 계산 (평균 CPI → CPU Time → 최적화 후 speedup) | 🔥×4 "이 수업의 핵심 공식. 매년 단골" |
| 2 | TPU Systolic Array | 5/27, 5/29 (2.10, 4.3.4절) | 그림 기반 동작 추적 (입력 배치, 출력 방향, 마지막 출력) | 🔥 "옛날에 냈던 문제 중에 하나" — 2×3 × 3×4 예제 그대로 |
| 3 | Roofline Model | 5/26 (1.9~1.13절) | 정량 계산 (ridge point → bound 판별 → BW 변화 효과) | 🔥 "8번 중 6번 출제" (75%+) |

**패턴**: 세 문제 모두 (a) 교수님이 강의에서 "단골"이라고 직접 못 박은 주제, (b) 강의 중 직접 시연한 예제의 숫자만 바꾼 형태, (c) 소문항 2~3개로 "기본 정의 → 계산 → 조건 변화 후 재계산" 구조의 **정량 계산형**이다.

## 1.2 작년(2025) 기출 분석 — 결정적 단서

작년 시험은 4문제였고, 올해 확정문제와 다음과 같이 대응된다.

| 2025 기출 | 내용 | 2026 확정문제 대응 |
|---|---|---|
| 기출 1 | Iron Law — 프로세서 A/B의 CPU Time 계산 (A: 2.5GHz·1.2×10⁹·CPI 1.0 → 0.48초 / B: 3.0GHz·1.5×10⁹·CPI 0.8 → 0.40초) | **확정문제 1과 동일 주제** (숫자·구성만 변경: 올해는 명령어 클래스별 CPI 표) |
| 기출 2 | Roofline — TPUv1 플롯에서 LSTM0·MLP0(memory-bound)/CNN0(compute-bound) 판별 + BW 2배 시 사선이 위로 이동, Peak TOPS 불변 | **확정문제 3과 동일 주제** (플롯 해석형 → 수치 계산형으로 변경) |
| 기출 3 | Systolic Array — 입력 X 배치, 출력 방향(y1x→(a), y2x→(b)), 마지막 출력(y23) | **확정문제 2와 완전 동일** (그림·숫자·답까지 그대로 재활용!) |
| 기출 4 | **RISC-V — RV64GC 명명 규칙 해석 + C 확장의 목적과 임베디드 이점** | **대응 없음 ← 4번째 문제 최유력** |

**핵심 발견 — 교수님은 문제를 거의 그대로 재활용한다:**

1. 작년 4문제 중 3문제(Iron Law, Roofline, Systolic)의 주제가 올해 확정 3문제로 그대로 이어졌고, 특히 **Systolic Array는 그림과 정답까지 완전히 동일**하다.
2. 따라서 남은 4번째 슬롯도 **작년 4번 문제(RISC-V RV64GC)가 재출제될 가능성이 가장 높다**.
3. 보정할 점: 강의에서 "매년 출제되는 국룰"이라던 **평균 내기 문제는 작년 시험에는 실제로 출제되지 않았다.** 신호 발언만으로는 부족하고, 실제 기출 슬롯 구성이 더 강한 근거다.
4. 작년 기출 4번도 "정의 해석(소문항 1) + 목적·이점 서술(소문항 2)" 구조 — 계산 없는 **암기·서술형**이었다. 4번째 문제는 계산형이 아닐 수 있음에 유의.

## 1.35 📊 난이도 분포 — 4번째(예상)문제는 '상' (우선순위 조정 신호)

추가 정보: 출제 4문제의 난이도는 **상 2 + 중 2**이며, **예상문제(4번째)는 '상'으로 출제**된다. → 확정문제 3개는 **중 2 + 상 1**로 구성된다.

확정문제 3개를 난이도로 보면, 공간 추론이 까다로운 **Systolic Array(문제 2)가 '상'**, 공식 대입 계산인 Iron Law(문제 1)·Roofline(문제 3)이 **'중'** 에 해당할 가능성이 높다(상 1 + 중 2). 따라서 **남은 1문제는 '상'**.

⚠️ 이 신호는 위 우선순위를 **일부 조정**한다. '상' 난이도는 보통 **계산이 까다롭거나 트리키한 유형**에서 나오는데:
- **↑ 상향**: **평균 내기의 국룰(후보 2)** — arithmetic/harmonic/geometric을 구분해 **올바른 평균을 골라 계산**하는, 틀리기 쉬운 트리키 계산 → '상'에 가장 부합. **NPU v1→v2 PE 개수 결정(후보 3, Roofline 응용)** 도 계산형 '상' 후보.
- **↓ 하향(난이도 관점)**: **RISC-V Naming(후보 1)** 은 *"작년 4번 기출 그대로"* 라는 재활용 신호는 여전히 최강이지만, 본질이 **암기·서술형**이라 전형적인 '상'과는 거리가 있다. 단, "C 확장의 임베디드 이점 2가지"처럼 **다단계 서술**을 요구하면 '상'으로 출제될 여지는 있다.

**종합 판단**: 재활용 패턴만 보면 RISC-V가 1순위지만, **"4번째 = 상"** 신호까지 함께 보면 **평균 내기(트리키 계산)** 가 '상' 슬롯의 강력한 경쟁 후보로 올라온다. → **RISC-V(재활용)와 평균 내기(난이도 부합) 둘 다 최우선으로 대비**할 것.

## 1.4 핵심 개념 정리 (예상문제 대비)

### ① RISC-V Naming Convention — 작년 기출 4번, 미출제 1순위

- **RV64GC 해석** (작년 기출 정답):
  - **RV**: RISC-V 아키텍처
  - **64**: 64비트 정수 레지스터 파일과 64비트 사용자 주소 공간
  - **G** = "General-purpose" = **I + M + A + F + D**:
    - **I**: 기본 정수 명령어 (47개)
    - **M**: 정수 곱셈/나눗셈
    - **A**: 원자적(Atomic) 명령어
    - **F**: 단정밀도 부동소수점
    - **D**: 배정밀도 부동소수점
  - **C**: 16비트 압축 명령어(Compressed Instructions) 확장
- **C 확장의 목적**: 자주 쓰는 32비트 명령어를 16비트로 압축해 **코드 크기 축소**.
- **임베디드 이점** (작년 기출 정답 2가지):
  1. **비용 절감**: 메모리(Flash/ROM/SRAM) 제약이 큰 임베디드에서 같은 프로그램을 더 적은 메모리에 저장 → 하드웨어 비용 직접 절감.
  2. **성능 향상·전력 감소**: I-Cache에 더 많은 명령어 적재 → hit rate 상승 → 실행 속도 향상 + 전력 소모가 큰 외부 메모리(DRAM/Flash) 접근 횟수 감소 → 시스템 전력 절감.
- 변형 대비: RV32IMAC 같은 다른 조합 해석, "G에 포함되는 5개 확장을 쓰라" 형태.

### ② 평균 내기의 국룰

| 평균 | 사용 대상 | 공식 |
|---|---|---|
| **Arithmetic Mean (산술평균)** | 시간 (Execution Time) | $\frac{1}{n}\sum_i t_i$ |
| **Harmonic Mean (조화평균)** | 처리율 (Rate: MIPS, FLOPS, QPS) | $\frac{n}{\sum_i \frac{1}{x_i}}$ |
| **Geometric Mean (기하평균)** | 비율 (Speedup, Ratio) | $\sqrt[n]{\prod_i r_i}$ |

- 교수님 원문: *"이건 텍스트북에 있는 그대로 썼으니까"* — 그대로 출제 가능 신호.
- 강의 시연 예제: 머신 A/B × 벤치마크 X/Y/Z. **Execution Time은 산술평균, MIPS는 조화평균**으로 각각 계산했을 때 둘 다 A가 2.66배 우위 — 같은 결론이 나오는지 확인하는 것이 평가 포인트.
- Rate에 산술평균을 쓰면 안 되는 이유: MIPS는 시간의 역수이므로, 산술평균을 취하면 짧은(빠른) 벤치마크에 과대 가중치가 걸린다. 전체 작업량 ÷ 전체 시간과 일치하는 것은 조화평균이다.
- 함정 보정: 벤치마크 길이 차가 클 때는 **Weighted Arithmetic Mean** (베이스라인 wall-clock의 역수를 weight로).

### ③ NPU PE 개수 결정 — Roofline 응용 (킬러 후보)

강의 시연 예제 (1.15절):
- NPU v1: 500 MHz, PE 2,000개, DRAM BW 10 GB/s → v2에서 BW 10배(100 GB/s), 메모리 액세스 패턴 동일(AI 불변).
- 풀이 골격: ① AI 불변 → ② BW k배면 사선이 log k만큼 평행이동 → ③ 같은 AI에서 attainable이 k배 → ④ 이를 받아내려면 Peak FLOPS도 k배 → ⑤ PE 1개 = 1 FLOP/cycle × 클럭이므로 필요한 PE 수 환산.
- 핵심 공식: $\text{Peak FLOPS} = N_{PE} \times \text{FLOPs/cycle} \times \text{Clock}$, $\text{Ridge} = \frac{\text{Peak FLOPS}}{\text{Peak BW}}$.

### ④ 메모리 계층 에너지 코스트 + 데이터 리유즈

| 동작 | Normalized Energy Cost |
|---|---:|
| ALU operation | 1X |
| Local Register File | 1X |
| NOC (Network-on-Chip) | 2X |
| Global Buffer | 6X |
| **DRAM** | **200X** |

- 32-bit DRAM read vs 8-bit add = **약 2만 배** (0.03 pJ vs 640 pJ).
- AlexNet: **724M MAC, 2,896M DRAM access** → convolutional reuse만으로 1/500, partial sum local accumulation까지 쓰면 **61M (1/50)**.
- CNN의 3가지 reuse 패턴: **Convolutional Reuse**(슬라이딩 윈도우 겹침), **Input Feature Map Reuse**(여러 output channel이 같은 입력 공유), **Filter Reuse**(batch > 1일 때 같은 필터 재사용).
- 3-level memory hierarchy: Local RF (per PE) → Global Shared Buffer (on-chip) → DRAM (off-chip).

### ⑤ 그 외 단답·서술 대비 한 줄 요약

- **bf16** = 1(sign) + 8(exponent) + 7(mantissa). FP32와 **range 동일**, 변환은 truncation/zero-padding만으로. 트레이닝에서는 정밀도보다 range가 중요하다는 것이 채택 근거.
- **4가지 챌린지**: ① Data movement 에너지(2만 배) ② 메모리 BW/용량 폭증(KV cache, 에이전트 4×/15×) ③ NPU underutilization(CNN=compute bound, NLP=memory bound → co-scheduling) ④ Sustainability(Meta 전력의 inference 70%).
- **PD Disaggregation**: Prefill은 AI 높음 → compute-centric 최신 GPU / Decode는 AI 낮음 → memory-centric 레거시 GPU. "토큰 퍼 달러 = 메모리 BW 퍼 달러".
- **TPU v1 CISC 5명령어**: Read Host Memory / Write Host Memory / Load Weights / Matrix Multiply·Convolve / Activation Function. 호스트 CPU가 fetch/dispatch.
- **Roofline 플롯 해석** (작년 기출 2 형태 대비): TPUv1에서 LSTM0·MLP0는 memory-bound(사선 아래 위치), CNN0는 compute-bound(평평한 지붕 근처). BW 2배 → **사선의 Y절편이 log₁₀2만큼 위로 이동, Peak TOPS(지붕)는 불변**.

# 2부. 예상문제

## 예상문제 1 — RISC-V Naming Convention (작년 기출 4번의 재출제 대비, 출제 확률 최상)

RISC-V는 개방형 표준 ISA로, 모듈화된 구조를 통해 다양한 컴퓨팅 환경에 맞게 확장할 수 있다. ISA의 이름은 지원하는 기능에 따라 표준화된 명명 규칙을 따른다.

1. `RV64GC`라는 ISA 이름이 의미하는 바를 각 구성요소(`RV`, `64`, `G`, `C`)로 나누어 설명하시오.
2. `G`가 포함하는 5개의 표준 확장을 각각의 알파벳과 함께 쓰시오.
3. `C` 확장(Compressed Instructions)의 주된 목적은 무엇이며, 이 확장이 특히 임베디드 시스템에서 어떤 이점을 제공하는지 설명하시오.
4. (변형 대비) 어떤 IoT 마이크로컨트롤러가 `RV32IMAC`을 채택했다. 이 ISA가 지원하는 기능과 지원하지 않는 기능을 설명하시오.

### 답안

1. 구성요소별 의미:
   - **RV**: RISC-V 아키텍처임을 나타낸다.
   - **64**: 64비트 정수 레지스터 파일과 64비트 사용자 주소 공간을 지원한다.
   - **G**: "General-purpose"의 약어로, 기본 정수(I), 정수 곱셈/나눗셈(M), 원자적 명령어(A), 단정밀도 부동소수점(F), 배정밀도 부동소수점(D) 확장을 모두 포함한다.
   - **C**: 16비트 길이의 압축 명령어(Compressed Instructions) 확장을 지원한다.

2. **G = I + M + A + F + D**:
   - **I** (Integer): 기본 정수 명령어 (47개)
   - **M** (Multiply/Divide): 정수 곱셈·나눗셈
   - **A** (Atomic): 원자적 메모리 연산
   - **F** (Float): 단정밀도(32-bit) 부동소수점
   - **D** (Double): 배정밀도(64-bit) 부동소수점

3. **목적**: 자주 사용되는 32비트 명령어들을 16비트 형태로 압축 표현하여 전체 프로그램의 **코드 크기(code size)를 줄이는 것**.
   **임베디드 이점**:
   1. **비용 절감**: 임베디드 시스템은 메모리(Flash, ROM, SRAM) 용량 제약이 크다. 코드 크기가 작아지면 같은 프로그램을 더 적은 메모리에 저장할 수 있어 하드웨어 비용을 직접 절감한다.
   2. **성능 향상 및 전력 감소**: 명령어 캐시(I-Cache)에 더 많은 명령어를 담을 수 있어 hit rate가 높아진다. 이는 메모리 접근 횟수를 줄여 실행 속도를 높이고, DRAM·Flash 같은 전력 소모가 큰 외부 메모리와의 데이터 전송을 줄여 시스템 전체 전력 소모를 감소시킨다.

4. RV32IMAC = 32비트 주소 공간/레지스터 + 기본 정수(I) + 곱셈·나눗셈(M) + 원자적 명령어(A) + 압축 명령어(C)를 지원한다. **F와 D가 빠져 있으므로 부동소수점 연산을 하드웨어로 지원하지 않는다** (필요 시 소프트웨어 에뮬레이션). 부동소수점이 불필요한 저비용·저전력 IoT/MCU 환경에 맞춰 면적과 전력을 아낀 구성이며, C 확장으로 코드 사이즈를 줄여 작은 Flash/SRAM에 최적화했다.

## 예상문제 2 — 평균 내기의 국룰 (작년 미출제였지만 발언 신호 최강)

두 머신 A, B에서 세 개의 벤치마크 X, Y, Z를 실행하였다. 각 벤치마크의 명령어 수(IC)와 실행 시간은 아래 표와 같다.

| 벤치마크 | 명령어 수 (IC) | 머신 A 실행 시간 | 머신 B 실행 시간 |
|---|---:|---:|---:|
| X | 2 × 10^9 | 10초 | 20초 |
| Y | 4 × 10^9 | 40초 | 80초 |
| Z | 8 × 10^9 | 100초 | 200초 |

1. 실행 시간(Execution Time) 기준으로 두 머신의 평균 성능을 비교하고자 한다. 어떤 평균을 사용해야 하는지 쓰고, 각 머신의 평균 실행 시간과 두 머신의 성능 비를 구하시오.
2. 각 벤치마크의 MIPS를 구하고, MIPS 기준으로 두 머신의 평균 성능을 비교하고자 한다. 어떤 평균을 사용해야 하는지 쓰고, 각 머신의 평균 MIPS와 성능 비를 구하시오.
3. 벤치마크별 Speedup(A over B)의 평균을 구하고자 한다. 어떤 평균을 사용해야 하는지 쓰고 그 값을 구하시오.
4. 처리율(Rate) 매트릭에 산술평균을 사용하면 안 되는 이유를 간략히 설명하시오.

### 답안

1. 시간에는 **Arithmetic Mean (산술평균)** 을 사용한다.

$$\bar{T}_A = \frac{10 + 40 + 100}{3} = 50\ \text{초}, \quad \bar{T}_B = \frac{20 + 80 + 200}{3} = 100\ \text{초}$$

$$\text{성능 비} = \frac{\bar{T}_B}{\bar{T}_A} = \frac{100}{50} = 2.0 \Rightarrow \text{A가 B보다 2배 빠름}$$

2. 처리율에는 **Harmonic Mean (조화평균)** 을 사용한다. 각 벤치마크의 MIPS = IC / (실행 시간 × 10^6):

| 벤치마크 | 머신 A MIPS | 머신 B MIPS |
|---|---:|---:|
| X | `2×10^9 / 10초 = 200` | `2×10^9 / 20초 = 100` |
| Y | `4×10^9 / 40초 = 100` | `4×10^9 / 80초 = 50` |
| Z | `8×10^9 / 100초 = 80` | `8×10^9 / 200초 = 40` |

$$H_A = \frac{3}{\frac{1}{200} + \frac{1}{100} + \frac{1}{80}} = \frac{3}{0.0275} \approx 109.1\ \text{MIPS}$$

$$H_B = \frac{3}{\frac{1}{100} + \frac{1}{50} + \frac{1}{40}} = \frac{3}{0.055} \approx 54.5\ \text{MIPS}$$

$$\text{성능 비} = \frac{109.1}{54.5} = 2.0$$

산술평균(시간)으로 구한 결과와 **일관되게 A가 2배 우위** — 올바른 평균을 골랐을 때 두 매트릭의 결론이 일치하는 것이 핵심 포인트.

3. 비율(Speedup)에는 **Geometric Mean (기하평균)** 을 사용한다. 벤치마크별 Speedup = 20/10, 80/40, 200/100 = 2, 2, 2.

$$G = \sqrt[3]{2 \times 2 \times 2} = 2.0$$

4. MIPS는 시간의 **역수** 성격의 매트릭이므로, 산술평균을 취하면 실행 시간이 짧은 벤치마크(MIPS가 높게 나오는 쪽)에 과대한 가중치가 걸려 전체 작업 관점의 처리율을 왜곡한다. "전체 명령어 수 ÷ 전체 실행 시간"과 일치하는 평균은 조화평균이다.

## 예상문제 3 — NPU 차세대 설계: Roofline 기반 PE 개수 결정 (킬러 문제 후보)

S사의 NPU v1은 다음과 같은 스펙을 가진다.

- 클럭: `1 GHz`
- PE(Processing Element) 개수: `4,096`개 — PE 1개는 매 사이클 1개의 FP 연산 수행
- DRAM Bandwidth: `8 GB/s`

이 NPU의 주력 워크로드 커널의 arithmetic intensity는 `512 FLOP/byte`이며, NPU v2에서도 메모리 액세스 패턴은 변하지 않는다(AI 불변).

1. NPU v1의 peak compute(GFLOP/s)와 ridge point(FLOP/byte)를 구하고, 주력 워크로드가 compute-bound인지 memory-bound인지 판별하시오.
2. NPU v2에서 DRAM bandwidth를 4배(`32 GB/s`)로 늘렸다. PE 개수를 v1과 동일하게 유지하면 주력 워크로드의 attainable performance는 어떻게 되는가? 이때 어떤 자원이 놀게 되는지 설명하시오.
3. NPU v2에서 늘어난 메모리 대역폭을 낭비 없이 활용하여 주력 워크로드의 최대 성능을 끌어내려면 최소 몇 개의 PE가 필요한가? (클럭은 `1 GHz`로 동일하다고 가정)

### 답안

1. Peak compute:

$$4{,}096\ \text{PE} \times 1\ \text{FLOP/cycle} \times 1\ \text{GHz} = 4{,}096\ \text{GFLOP/s}$$

Ridge point:

$$\frac{4{,}096\ \text{GFLOP/s}}{8\ \text{GB/s}} = 512\ \text{FLOP/byte}$$

주력 워크로드의 AI(512)가 ridge point(512)와 **정확히 일치** — 가장 적은 엔지니어링 노력으로 peak에 도달하는 **이상적인(ideal) 지점**이며, compute와 memory 자원이 모두 100% 활용된다.

2. BW가 4배가 되면 Roofline의 사선이 log 4만큼 위로 평행 이동하고, ridge point는 다음과 같이 커진다.

$$\frac{4{,}096}{32} = 128\ \text{FLOP/byte}$$

워크로드 AI(512) > 새 ridge point(128)이므로 이제 **compute-bound**가 된다.

$$\text{Attainable performance} = \min(4{,}096,\ 32 \times 512) = 4{,}096\ \text{GFLOP/s}$$

성능은 v1과 **변화 없음**. 사선 기준 도달 가능 성능은 `32 × 512 = 16,384 GFLOP/s`인데 peak compute가 4,096에서 머리를 치므로, **늘어난 메모리 대역폭의 3/4이 놀게 된다** (compute가 병목).

3. 늘어난 BW를 전부 받아내려면 peak compute가 사선과 만나는 지점까지 올라와야 한다.

$$\text{필요한 Peak FLOPS} = \text{Peak BW} \times AI = 32\ \text{GB/s} \times 512\ \text{FLOP/byte} = 16{,}384\ \text{GFLOP/s}$$

PE 1개 = 1 GHz × 1 FLOP/cycle = 1 GFLOP/s이므로:

$$N_{PE} \geq \frac{16{,}384\ \text{GFLOP/s}}{1\ \text{GFLOP/s}} = 16{,}384\ \text{개}$$

즉 BW를 4배 늘렸으므로 PE도 **4배(16,384개)** 로 늘려야 ridge point가 다시 512에 위치하여 두 자원이 모두 낭비 없이 활용된다.

## 예상문제 4 — 메모리 계층 에너지 코스트와 데이터 리유즈

DNN 가속기에서 각 동작의 정규화된 에너지 코스트는 아래 표와 같다.

| 동작 | Normalized Energy Cost |
|---|---:|
| ALU operation (MAC 1회) | 1X |
| Local Register File 접근 | 1X |
| Global Buffer 접근 | 6X |
| DRAM 접근 | 200X |

하나의 MAC 연산은 **weight 읽기, input activation 읽기, partial sum 읽기** 의 3회 read와 연산 후 **partial sum 쓰기** 1회, 총 4회의 메모리 접근을 수반한다고 가정한다.

1. ScratchPad/캐시가 전혀 없어 4회의 메모리 접근이 모두 DRAM에서 일어나는 worst case에서, MAC 1회당 총 에너지 코스트를 구하시오.
2. 데이터 리유즈를 극대화하여 4회의 접근이 모두 Local Register File에서 일어난다면 MAC 1회당 총 에너지 코스트는 얼마이며, worst case 대비 몇 배 절감되는가?
3. CNN convolution layer에서 활용할 수 있는 **3가지 데이터 리유즈 패턴**의 명칭을 쓰고 각각 한 문장으로 설명하시오.
4. AlexNet은 724M번의 MAC 연산에 2,896M번의 DRAM 접근이 필요하지만, 데이터 리유즈와 partial sum local accumulation을 모두 적용하면 DRAM 접근을 61M번으로 줄일 수 있다. 이때 DRAM 접근 에너지는 약 몇 분의 1로 줄어드는가?

### 답안

1. Worst case 에너지:

$$1X\ (\text{MAC}) + 4 \times 200X\ (\text{DRAM}) = 801X$$

2. 모든 접근이 Local RF인 경우:

$$1X\ (\text{MAC}) + 4 \times 1X\ (\text{RF}) = 5X$$

절감 배수:

$$\frac{801X}{5X} \approx 160\ \text{배}$$

(메모리 접근 에너지만 비교하면 `800X / 4X = 200배` — DRAM과 RF의 에너지 비와 동일)

3. CNN convolution layer의 3가지 리유즈 패턴:

| 패턴 | 설명 |
|---|---|
| **Convolutional Reuse** | 슬라이딩 윈도우가 stride 1로 이동하면 이전 위치와 입력이 겹치므로, input feature map과 filter를 반복 재사용 |
| **Input Feature Map Reuse** | output channel이 여러 개일 때, 동일한 input feature map에 서로 다른 필터들을 적용하므로 입력을 재사용 |
| **Filter Reuse** | batch size > 1일 때, 같은 필터를 한 번만 로딩하여 여러 input feature map에 반복 적용 |

4. DRAM 접근 횟수가 줄어든 비율만큼 DRAM 에너지가 절감된다.

$$\frac{2{,}896\text{M}}{61\text{M}} \approx 47.5 \Rightarrow \text{약 } \frac{1}{50}\text{로 절감}$$

# 3부. 추가 대비 — 서술형으로 변형될 경우의 답안 골격

킬러 문제가 계산형이 아니라 **개념 서술형**으로 나올 경우를 대비한 모범답안 골격.

**Q. Roofline 플롯이 주어지고 커널 위치로 병목을 판별하라. (작년 기출 2 형태)**
→ ridge point보다 왼쪽(AI 낮음)에 있는 커널은 **memory-bound** (사선이 천장), 오른쪽(AI 높음)은 **compute-bound** (평평한 지붕이 천장). TPUv1 예: LSTM0·MLP0 = memory-bound, CNN0 = compute-bound. 메모리 BW가 2배가 되면 **사선의 Y절편이 log₁₀2만큼 위로 평행 이동**하고 ridge point는 절반으로 줄며, **Peak TOPS(지붕 높이)는 변하지 않는다**.

**Q. bf16이 트레이닝에서 표준 FP16보다 유리한 이유를 비트 구성과 함께 설명하라.**
→ bf16 = sign 1 + exponent 8 + mantissa 7. exponent를 FP32와 동일하게 유지하여 **dynamic range가 FP32와 같다** (표준 FP16은 exponent 5비트로 range가 좁아 gradient overflow/underflow 위험). 트레이닝에서는 gradient의 정밀도보다 range가 중요하며, FP32 ↔ bf16 변환이 truncation/zero-padding만으로 가능해 하드웨어 코스트도 낮다. mantissa 축소는 곱셈기 면적·전력 절감 효과가 크다.

**Q. AI 컴퓨팅의 4가지 챌린지를 쓰고 각각의 해결 방향을 한 가지씩 들어라.**
→ ① **Data movement 비용** (32-bit DRAM read는 8-bit add 대비 약 2만 배 에너지) — PIM/FIM으로 메모리 근처 연산. ② **메모리 BW/용량 수요 폭증** (KV cache, 멀티 에이전트 토큰 15배) — Tiered memory (HBM + CXL/LPDDR). ③ **NPU 자원 underutilization** (CNN=compute bound, NLP=memory bound로 Roofline 미스매치) — multi-model co-scheduling으로 composite AI를 ridge point 근처로. ④ **Sustainability** (CAPEX 연 수백조, 전력 한계) — 전성비 중심 설계, inference 효율화 (Meta 전력의 70%가 inference).

**Q. Prefill/Decode Disaggregation이 왜 표준이 되었는가?**
→ Prefill은 입력 토큰을 한 번에 처리해 arithmetic intensity가 높아 **compute-bound** → 최신 compute-centric GPU가 적합. Decode는 토큰을 하나씩 auto-regressive로 생성하며 KV cache를 반복 읽어 AI가 낮은 **memory-bound** → 메모리 BW 대비 가격이 싼 레거시 GPU가 적합. "토큰 스루풋 ∝ 메모리 BW"이므로 "토큰 퍼 달러 = 메모리 BW 퍼 달러"가 되어, phase별로 다른 하드웨어에 배치하면 전체 토큰 단가가 내려간다.
