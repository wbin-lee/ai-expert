## 0. 핵심 분석 — 세 파트 / 네 문제 vs 확정문제 3개

교수님/조교 공지: 시험은 **세 파트에서 총 4문제**, *"무거운 암기는 필요 없고 이해만 하면 된다. 간단한 계산 문제는 나올 수 있다."* (투뷰 지오메트리·삼각측량·PnP 같은 7~8장 어려운 내용은 **출제 안 됨**.)

**세 파트:**
- **파트 1**: Homogeneous 좌표 변환 & Projective Geometry (본문 2장)
- **파트 2**: Homography — 개념과 "어떻게 구했는지"(DLT) (본문 3·4장)
- **파트 3**: 카메라 모델 — Projection Matrix `P = K[R|t]` (본문 5·6장)

**확정문제 3개를 파트에 매핑:**

| 확정문제 | 내용 | 파트 |
|---|---|---|
| **문제 1** | Homogeneous ↔ Euclidean 좌표 변환 (양방향) | **파트 1** |
| **문제 2** | Camera projection — `PX` 계산, inhomogeneous 변환 | **파트 3** (계산) |
| **문제 3** | `P = K[R\|t]` 분해 + K/R/t/P 각각의 의미와 DoF | **파트 3** (개념·DoF) |

→ 확정문제는 **파트 1 ×1, 파트 3 ×2**. **파트 2 (Homography)는 단 한 문제도 없음.**

### 🎯 결론: 남은 1문제는 **파트 2 — Homography (특히 DLT 유도)** 일 확률이 압도적으로 높음

근거:
1. **세 파트 × 4문제** 구조인데 **파트 2(Homography)만 확정문제에 전혀 없음** → 4문제 균형상 여기서 1문제가 나오는 게 자연스러움.
2. 교수님이 Homography의 **DLT 유도("어떻게 구했는지")를 "🔥 출제 명시 포인트"** 로 강의록에 직접 표기 (강의내용 501행).
3. *"미지수 몇 개? 대응점 하나가 식 몇 개? 그래서 최소 몇 쌍?"* (8 / 2 / 4) 류의 **카운팅 계산**과, 변환 계층(rigid/affine/projective) 분류를 *"나올 만한 간단 계산"* 으로 명시.

→ **예상문제 1번(Homography DLT 카운팅·유도)을 최우선으로 대비**하세요.

### 📊 난이도 분포 — 4번째(예상)문제는 '상'

추가 정보: 출제 4문제의 난이도는 **상 2 + 중 2**이며, **예상문제(4번째)는 '상'으로 출제**된다. → 확정문제 3개는 **중 2 + 상 1**로 구성된다.

이 신호는 위 예측을 강화한다. 남은 1문제가 '상'이라는 것은 단순 좌표 변환(확정 문제 1, 하~중)이나 단일 행렬곱 대입(확정 문제 2, 중)이 아니라, **유도·카운팅 논리**가 필요한 문제임을 시사한다. **Homography DLT(예상문제 1: DoF 8 / 대응점당 식 2개 / 최소 4쌍 카운팅 + `Ah=0`·SVD)** 가 정확히 그 '상' 유형이다. 확정문제 중에서는 **`P=K[R\|t]` 분해·DoF(문제 3)가 '상'**, 좌표 변환(문제 1)과 projection 계산(문제 2)이 **'중'(또는 하)** 에 해당할 가능성이 높다.

## 1. 핵심 개념 정리 — Homography (파트 2)

### 1.1 정의
- **Homography**: 2D 프로젝티브 공간 → 2D 프로젝티브 공간의 가역 매핑 중 **직선을 직선으로 보존**하는 것 = **3×3 가역 행렬** `H`.
- `x′ = H x` (up to scale) — homogeneous 좌표에서 성립.

### 1.2 언제 성립하는가 (세 경우)
1. 3D 공간의 **한 평면(plane)** 과 이미지 사이 (central projection — 그 평면 위 점들만).
2. **같은 평면**을 본 두 이미지 사이.
3. **퓨어 로테이션**(카메라 센터 고정, 회전만)으로 찍은 두 이미지 사이 — depth 무관 → **파노라마의 원리**. (단 이 경우 3D 복원은 불가.)

### 1.3 변환 계층(Transformation Hierarchy)과 DoF
포함 관계: **rigid(3) ⊂ similarity(4) ⊂ affine(6) ⊂ projective(8)**

| 변환 | DoF | 특징 | 보존하는 것 |
|---|---|---|---|
| **Rigid (Euclidean)** | 3 | 회전 + 이동 | 거리, 각도, 평행 |
| **Similarity** | 4 | + 균일 스케일 | 각도, 평행, 비율 |
| **Affine** | 6 | 마지막 행 `(0,0,1)` | **평행(parallelism)** |
| **Projective (Homography)** | 8 | 마지막 행 자유 | 직선(만) — **평행 깨짐 → 소실점 생김** |

- **affine이 평행을 보존하는 이유**: 마지막 행이 `(0,0,1)`이라 **무한대 점(ideal point)이 무한대에 머묾** → 평행선은 여전히 무한대에서 만남 → 평행 유지.
- **projective는 평행을 깸**: 무한대 점을 유한한 점(소실점)으로 보내기 때문.

### 1.4 ⭐ 구하는 법 (DLT) — "어떻게 구했는지"가 출제 명시 포인트
1. `H`의 DoF = `9 − 1`(up to scale) = **8** → 식 **8개** 필요.
2. 대응점 **1쌍** = 식 **2개** (`x′ = Hx`의 up to scale 등식을 양변 inhomogeneous로 바꿔 진짜 등호 2개로).
3. ∴ **최소 대응점 4쌍.**
4. 대응점들을 정리해 **`A h = 0`** 꼴로 만든다 (대응점 n쌍 → `2n × 9` 행렬 A, 모두 같은 H를 공유하므로 행을 concat). `‖h‖ = 1` 제약 하에 `‖Ah‖` 최소화 → **SVD 후 V의 마지막 열(가장 작은 singular value에 해당)** 이 `h`.
5. 대응점이 많을수록(over-complete) 노이즈에 강건 → 연립방정식 대신 DLT를 쓰는 이유.

### 1.5 Algebraic vs Geometric Error
- DLT가 최소화하는 `‖Ah‖`는 **algebraic error** (선형, closed-form으로 한 방에 풀림).
- 진짜 원하는 것은 **geometric error** `Σ d(x′, Hx)²` (reprojection 거리, 비선형).
- 패턴: **DLT 해를 초기값으로 gradient descent** 로 geometric error를 다듬는다 — 모든 3D 비전 파이프라인의 공통 구조.

### 1.6 (보조) RANSAC
- 실제 대응점(SIFT 매칭)엔 아웃라이어가 섞임 → 매 반복 **최소 샘플(H는 4쌍)** 로 가설을 세우고 인라이어 투표 → 최다 지지 가설의 인라이어로 재피팅.
- 최소 샘플 수가 클수록 반복이 더 필요 (DoF/최소대응점 수에 민감).

### 1.7 한 장 요약 (2D↔2D vs 3D→2D 대비 — 강의 전체의 핵심)
| | 2D → 2D | 3D → 2D |
|---|---|---|
| 매핑 | **호모그래피 H (3×3)** | **카메라 행렬 P = K[R\|t] (3×4)** |
| DoF | **8** | 11 (K 5 + R 3 + t 3) |
| 대응점당 식 | 2개 | 2개 |
| 최소 대응점 | **4쌍** | 6쌍 |
| 구하는 법 | DLT: `Ah=0` → SVD, V 마지막 열 | 동일 |

## 2. 예상문제

### 🔥 예상문제 1 (최우선 — Homography DLT, 남은 1문제로 가장 유력)

두 이미지 사이의 homography `H` (`x′ = Hx`, up to scale)를 대응점들로부터 구하려고 한다.

(a) `H`를 결정하는 데 필요한 자유도(DoF)는 몇 개이며, 그 이유는 무엇인가?
(b) 대응점 한 쌍은 몇 개의 방정식을 주는가? 따라서 `H`를 구하기 위한 **최소 대응점 수**는 몇 쌍인가?
(c) 대응점이 여러 쌍 주어졌을 때 `H`를 구하는 **DLT(Direct Linear Transform)** 절차를 `Ah = 0` 형태와 SVD를 사용하여 설명하시오.
(d) 대응점이 최소 개수보다 많을 때 DLT를 쓰는 것이 단순 연립방정식 풀이보다 좋은 이유는 무엇인가?

#### 답안

**(a)** `H`는 3×3 행렬로 원소가 9개지만, **up to scale**(임의의 0이 아닌 상수를 곱해도 같은 변환)이므로 1을 빼서 **DoF = 8**.

**(b)** 한 대응점 `(x, y) ↔ (x′, y′)`는 `x′ = Hx`의 up to scale 등식을 inhomogeneous로 풀면 **2개의 방정식**을 준다. DoF가 8이므로 식 8개가 필요 → **최소 대응점 = 8/2 = 4쌍.**

**(c)** 각 대응점이 주는 2개 식을 `H`의 원소 벡터 `h`(9차원)에 대해 선형으로 정리하면, 대응점 n쌍에 대해 `2n × 9` 행렬 `A`가 만들어지고 **`A h = 0`** 형태가 된다 (모든 대응점이 같은 `H`를 공유하므로 행을 쌓는다). 자명해 `h = 0`을 피하려고 `‖h‖ = 1` 제약을 두고 `‖Ah‖`를 최소화하면, 해는 **`A`의 SVD에서 가장 작은 singular value에 대응하는 우특이벡터 — 즉 `V`의 마지막 열**이 된다. 이를 3×3로 reshape하면 `H`.

**(d)** 대응점에는 노이즈가 섞여 있어 정확히 `Ah = 0`을 만족하는 해가 없다. 대응점을 최소 개수보다 많이(over-complete) 쌓으면 **최소제곱 의미로 노이즈에 강건한** 해를 얻을 수 있다. (단 DLT가 최소화하는 것은 algebraic error이므로, 정확도가 더 필요하면 이 해를 초기값으로 geometric error를 비선형 최적화한다.)

### 🔥 예상문제 2 (변형 대비 — 변환 계층과 DoF)

2D 변환은 rigid ⊂ similarity ⊂ affine ⊂ projective의 계층 구조를 이룬다.

(a) 네 변환의 자유도(DoF)를 각각 쓰시오.
(b) **affine 변환은 평행선을 평행하게 보존**하지만 **projective(homography)는 평행을 깬다.** 그 이유를 **무한대 점(ideal point, 마지막 원소가 0인 점)** 과 각 변환 행렬의 **마지막 행** 을 이용해 설명하시오.

#### 답안

**(a)** Rigid **3** (회전 2D는 1 + 이동 2 = 3), Similarity **4** (+ 균일 스케일), Affine **6**, Projective(Homography) **8**.

**(b)** Affine 변환 행렬의 마지막 행은 `(0, 0, 1)`이다. 이 때문에 무한대 점 `(x, y, 0)`을 변환해도 마지막 원소가 여전히 0으로 유지되어 **무한대 점이 무한대에 그대로 머문다.** 평행선들은 무한대의 ideal point에서 만나는 선들인데, 그 교점이 무한대에 남아 있으므로 변환 후에도 여전히 평행하다 → **평행 보존.**
반면 projective(homography)는 마지막 행이 자유롭기 때문에 무한대 점 `(x, y, 0)`을 마지막 원소가 0이 아닌 **유한한 점(소실점)** 으로 보낼 수 있다. 그 결과 평행선들이 유한한 소실점에서 만나게 되어 **평행이 깨진다.**

### 🔥 예상문제 3 (변형 대비 — Homography가 성립하는 조건 + 2D/3D 매핑 대비)

(a) 두 이미지(또는 평면과 이미지) 사이의 관계가 **하나의 homography로 표현될 수 있는 경우**를 세 가지 쓰시오.
(b) Homography(2D↔2D)와 카메라 projection matrix(3D→2D)를 **행렬 크기, DoF, 최소 대응점 수** 관점에서 비교하시오.

#### 답안

**(a)**
1. 3D 공간의 **한 평면** 과 그것을 찍은 이미지 사이 (그 평면 위의 점들에 한해).
2. **같은 평면** 을 서로 다른 위치에서 본 두 이미지 사이.
3. 카메라 센터를 고정하고 **순수 회전(pure rotation)** 만으로 찍은 두 이미지 사이 (depth와 무관 → 파노라마의 원리. 단 이 경우 3D 복원은 불가능).

**(b)**

| | Homography (2D→2D) | Camera matrix P (3D→2D) |
|---|---|---|
| 행렬 크기 | **3×3** | **3×4** |
| DoF | **8** (= 9 − 1) | **11** (= 12 − 1, K 5 + R 3 + t 3) |
| 대응점당 식 | 2개 | 2개 |
| 최소 대응점 | **4쌍** (2D↔2D) | **6쌍** (3D↔2D) |
| 구하는 법 | DLT (`Ah=0` → SVD) | 동일 (calibration) |

> 핵심 한 줄: **2D↔2D는 3×3 호모그래피, 3D→2D는 3×4 카메라 행렬.** 둘 다 대응점만 있으면 DLT(`Ah=0`, SVD)로 풀린다.
