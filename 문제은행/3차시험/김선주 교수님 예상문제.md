## 1. 핵심 개념 정리 — AI로 ISP를 대체하는 2가지 부류 (출제 포인트 ⑤)

ISP는 원래 10여 단계의 **모듈러 파이프라인**(디모자이크 → 노이즈 제거 → WB → 색변환 → 톤매핑 → …)입니다. 이를 AI(DNN)로 대체하려는 시도는 크게 **두 부류**로 나뉩니다.

### (1) End-to-End / One-Stage (단일 DNN이 ISP 전체 대체)
- **방식**: `raw → 하나의 큰 DNN → 최종 sRGB`. 파이프라인 전체를 모델 하나가 통으로 처리.
- **대표 사례**: 교수님 박사 시절 연구(원-스테이지 ISP 대체, forward raw→sRGB + inverse sRGB→raw), *"Learning to See in the Dark"* (저조도 디노이징 + ISP 렌더링을 한 번에).
- **장점**
  - 파이프라인 전체를 한 번에 최적화 → **글로벌 옵티멀** 도달 가능
  - 모듈 간 **에러 누적 없음**
  - **데이터 페어가 단순** — `(raw, 최종 sRGB)` 한 쌍만 있으면 학습 가능
  - 모델 하나만 관리
- **단점**
  - **블랙박스라 미세 컨트롤·튜닝이 거의 불가능 (치명적 단점)**
  - 결과가 마음에 안 들 때 어디를 손봐야 할지 모름 → 제조사가 원하는 특정 색감/스타일 통제 어려움
  - 학습 데이터에 강하게 의존, 해석 가능성 낮음 → 회사가 채택하기 어려움

### (2) Modular DNN Replacement (모듈별로 AI 교체)
- **방식**: 각 ISP 모듈(또는 2~3개 묶음)을 sub-DNN으로 모델링, 단계별 진행. **실제 카메라 회사가 선호하는 방향.**
- **장점**
  - 각 모듈 역할이 명확 → **디버깅·튜닝 쉬움 (튜너빌리티)**
  - 기존 ISP 노하우 그대로 활용, **문제 모듈만 교체 가능** (WB만, 디노이징만)
  - 해석 가능
- **단점**
  - **글로벌 옵티멀 보장 안 됨** (각 모듈이 로컬 옵티멀), 모듈 간 에러 누적 가능
  - **각 모듈마다 데이터 페어를 따로 준비**해야 함 → 데이터 수집·정제 번거로움
  - 여러 모델 관리 부담, 시스템 복잡도 ↑, 효율 저하 가능

### 한 줄 비교 (암기용)
| 항목 | End-to-End | Modular |
|---|---|---|
| 최적화 | **글로벌 옵티멀** | 로컬 옵티멀(에러 누적) |
| 데이터 | (raw, sRGB) **페어 1쌍만** | **모듈마다** 페어 필요 |
| 튜닝/제어 | **불가능(블랙박스)** | **쉬움(튜너빌리티)** |
| 관리 | 모델 1개 | 모델 여러 개 |

> **교수님의 비전**: 두 진영의 한계를 모두 풀 길로 **디퓨전 기반 단일 파운데이션 모델(프롬프트로 튜닝 가능)** 제시 — *"end-to-end의 단순함 + 모듈러의 튜너빌리티를 동시에"*.

## 2. 예상문제

### 🔥 예상문제 1 (출제 포인트 ⑤, 남은 1문제로 가장 유력)

전통적인 ISP는 여러 모듈로 구성된 파이프라인이다. 이를 AI(딥러닝)로 대체하려는 접근은 크게 **두 가지 부류**로 나눌 수 있다.

(a) 이 두 가지 부류가 각각 무엇인지 방식과 함께 간단히 설명하시오.
(b) 두 접근의 **장점과 단점**을 각각 서술하시오. 특히 **데이터(학습 페어) 측면**과 **튜닝/제어 가능성 측면**에서의 차이를 반드시 포함하시오.

#### 답안

**(a) 두 가지 부류**

1. **End-to-End (One-Stage) 방식**: 하나의 큰 DNN이 `raw 입력 → 최종 sRGB 출력`을 통째로 매핑하여 ISP 파이프라인 전체를 단일 모델로 대체.
2. **Modular DNN Replacement 방식**: ISP의 각 모듈(또는 2~3개 묶음)을 개별 sub-DNN으로 대체하여 단계적으로 처리. 실제 카메라 회사가 선호하는 방향.

**(b) 장단점**

| | **End-to-End** | **Modular** |
|---|---|---|
| **장점** | 파이프라인 전체를 한 번에 최적화 → **글로벌 옵티멀** 가능, 모듈 간 에러 누적 없음, **데이터 페어가 (raw, sRGB) 한 쌍으로 단순**, 모델 1개만 관리 | 각 모듈 역할이 명확해 **디버깅·튜닝 쉬움(튜너빌리티)**, 문제 모듈만 교체 가능, 기존 ISP 노하우 활용, 해석 가능 |
| **단점** | **블랙박스라 미세 튜닝·색감 제어가 거의 불가능(치명적)**, 결과가 나쁠 때 손볼 곳을 모름, 데이터 의존·해석성 낮음 | 글로벌 옵티멀 보장 안 됨(로컬 옵티멀·에러 누적), **각 모듈마다 데이터 페어를 따로 준비**해야 함, 여러 모델 관리 부담·복잡도 ↑ |

- **데이터 측면**: End-to-End는 (raw, 최종 sRGB) 페어 한 쌍만 있으면 되어 단순하지만, Modular는 모듈마다 입력·출력 페어를 별도로 수집·정제해야 해 번거롭다.
- **튜닝/제어 측면**: End-to-End는 블랙박스라 특정 색감·스타일을 손보기 어렵지만, Modular는 모듈 단위로 디버깅·교체가 가능해 제어가 쉽다. (이 튜너빌리티 때문에 카메라 회사들은 Modular를 선호한다.)

> 보너스: 교수님은 두 진영의 한계를 모두 극복할 방향으로 **프롬프트로 튜닝 가능한 디퓨전 기반 단일 파운데이션 모델**을 본인의 비전으로 제시했다(End-to-End의 단순함 + Modular의 튜너빌리티를 동시에).
