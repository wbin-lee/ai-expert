## 1. 확정문제 스타일 분석

| 확정문제 | 유형 | 출제 포인트 |
|---|---|---|
| 문제 1. R-CNN vs Fast R-CNN | **서술형** ("~한 이유를 서술하시오") | 두 방법의 **구조적 차이 → 효과(속도)** 의 인과관계. 강의에서 교수님이 명시적으로 강조한 비교 포인트 |
| 문제 2. Focal Loss | **수식 해석형** | 수식을 주고 **특수 케이스(γ=0)와 파라미터의 효과(γ>0)** 를 해석. "되게 중요하니까 기억하라"고 직접 언급한 주제 |
| 문제 3. DETR matching cost | **수치 계산형** | 강의에서 다룬 수식(L1, GIoU, 매칭 코스트)에 **구체적 숫자를 대입해 직접 계산**. 표로 데이터 제공, 답이 깔끔한 수치 |

**패턴 정리:**

1. **세 문제 모두 강의록의 🔥시험포인트🔥에서 그대로 출제됨.** 교수님이 "기억하라", "정확히 이해하라"고 말한 부분이 곧 문제다.
2. **유형 배분**: 개념 서술 1 + 수식 해석 1 + 계산 1. 4번째 문제는 이 중 어느 유형이든 가능하나, 확정 3문제가 다루지 않은 **새 주제**에서 나올 가능성이 높다.
3. **주제 커버리지**: 확정 3문제는 R-CNN family(1장), RetinaNet/Focal Loss(2장), DETR(2장)을 커버. **아직 출제되지 않은 대형 시험포인트**는 ↓

### 📊 난이도 분포 — 4번째(예상)문제는 '상'

추가 정보: 출제 4문제의 난이도는 **상 2 + 중 2**이며, **예상문제(4번째)는 '상'으로 출제**된다. → 확정문제 3개는 **중 2 + 상 1**로 구성된다.

이 신호는 예측의 **우선순위를 조정**해준다. 남은 1문제가 '상'이라는 것은 한 줄 개념 서술보다 **계산이 까다롭거나 트리키한 유형**일 가능성이 높다는 뜻이다. 따라서:
- **↑ 우선순위 상승**: 교수님이 *"트리키해서 정확히 이해하라"* 고 강조한 **Evaluation(IoU/Precision/Recall/PR curve/AP) 계산(예상문제 2)** 과 **FCOS Centerness 계산(예상문제 1)** — 둘 다 다단계 계산이 필요한 명백한 '상' 유형.
- **↓ 우선순위 하락**: 한 줄 개념 서술인 **Hard Negative Mining(예상문제 3)** 은 단독 출제 시 오히려 '중' 난이도에 가까워, 4번째('상') 자리보다는 확정문제의 '중' 슬롯 성격에 가깝다.

(확정문제 중에서는 DETR matching cost 계산(문제 3)이 '상', R-CNN 비교(문제 1)와 Focal Loss 해석(문제 2)이 '중'에 해당할 가능성이 높다.)

## 2. 출제 예상 개념 정리 (우선순위 순)

### ① Evaluation — IoU / Precision / Recall / PR Curve / AP·mAP ★★★

교수님이 *"트리키한 면이 많아서 정확히 이해하고 넘어가는 게 낫다"* 고 명시적으로 강조한 섹션. 확정 3문제가 전혀 건드리지 않은 유일한 대형 주제.

- **IoU** = Intersection / Union. **0.5 이상**이면 디텍션 성공으로 간주 (생각보다 큰 오버랩).
- **Precision** = TP / (TP + FP) — 모델이 뽑은 박스 중 진짜 비율.
- **Recall** = TP / (TP + FN) — 실제 오브젝트 중 찾아낸 비율.
- Threshold ↓ → Precision ↓, Recall ↑ (trade-off → 둘 다 봐야 함).
- **디텍션 PR curve는 지그재그**: score 순으로 정렬해 내려가며 보면, FP가 나올 때 precision이 뚝 떨어졌다가 TP가 나오면 다시 올라가기 때문.
- **AP** = PR curve 아래 면적. **mAP** = 클래스별 AP의 평균. **mAP@0.5**(IoU 0.5 고정) vs **일반 mAP**(IoU 0.5~0.95를 0.05 단위로 평균).

### ② FCOS — Centerness ★★★

수식이 명확하고 숫자 대입 계산이 가능해서 **확정문제 2·3번과 동일한 스타일로 변환하기 가장 좋은 주제**.

- 앵커·proposal 없이 **모든 location마다 prediction** (세그멘테이션처럼) → 교수님: *"이게 정말로 원스테이지스럽다."*
- 픽셀에서 박스 left/right/top/bottom까지 거리 L, R, T, B일 때:

$$\text{Centerness} = \sqrt{\frac{\min(L,R)}{\max(L,R)} \times \frac{\min(T,B)}{\max(T,B)}}$$

- 센터면 1, 가장자리로 갈수록 0. BCE로 학습.
- **인퍼런스 최종 score = classification score × centerness score** (교수님이 직접 강조).
- Limitation: 오버랩된 오브젝트.

### ③ SSD — Hard Negative Mining ★★★

교수님이 *"디텍션 할 때 사실은 **가장 중요한 건** 하드 네거티브 마이닝"* 이라고 **"가장 중요"를 명시한 유일한 항목**. 서술형 단골감.

- 아무데서나 negative를 뽑으면 너무 쉬워서 학습 효과가 낮음.
- **현재까지 학습된 디텍터가 positive로 잘못 판단한 샘플(false positive)** 을 negative로 계속 공급 → 어려운 negative를 정확히 구분하게 됨.

### ④ DETR의 NMS-free / Hungarian 매칭 구조 ★★☆

확정문제 3이 계산을 다뤘으므로, 같은 주제의 **개념 서술** 이 추가로 나올 수 있음.

- DETR = 트랜스포머 기반, **anchor 없음, NMS 없음**, set prediction + **1-to-1 bipartite matching (Hungarian)**.
- 기존 NMS의 문제: threshold에 민감 + 겹친 서로 다른 오브젝트 중 하나가 지워짐.
- **매칭 단계는 백프롭 안 됨** — 현재 모델 출력으로 매칭(σ)만 정하고, 그 후 detection loss를 계산해 백프롭.
- GIoU가 필요한 이유: IoU는 오버랩 없으면 항상 0 → **그래디언트 없음**. GIoU는 enclosing box 페널티로 거리 정보를 줌.

### ⑤ Deformable DETR — DETR 약점 보완 ★★☆

- DETR 약점: **수렴 느림(500 epoch vs Faster R-CNN 50)**, 작은 오브젝트 약함, 인퍼런스 느림.
- **Deformable Attention**: query마다 sampling offset Δp를 학습해 **K개 key만** attention → encoder 복잡도가 spatial resolution에 linear.
- Multi-scale deformable attention → 별도 FPN 불필요. Training epoch 약 10배 감소.

### ⑥ 기타 가능 포인트 ★☆☆

- **RoI Pooling 동작 3단계**: 좌표 rounding → 7×7 분할 → 셀별 max pooling (교수님이 "한 번 더 읽어보라" 강조). — 단, 확정문제 1이 Fast R-CNN을 이미 다뤄 중복 가능성 있음.
- **Dilated Convolution**: 파라미터 수 동일, receptive field ↑, feature map 사이즈 유지 (DeepLab 출처).
- **YOLO V1**: 7×7×2 = 98개 박스, "한 놈만 패는" 한계, 출력 차원 7×7×30.
- **박스 수 비교**: R-CNN 2000 / Faster R-CNN 수백 / SSD 8,732 / YOLO 98.
- **RT-DETR**: YOLO NMS 비판(하이퍼파라미터 민감, 입력 따라 시간 변동), AIFI(S5만) + CCFF.

## 3. 예상문제

### 예상문제 1 — FCOS Centerness 계산 (계산형, 확정문제 3 스타일)

FCOS에서 feature map의 각 location은 자신이 속한 ground-truth 박스의 네 변까지의 거리 $(L, R, T, B)$ 로부터 다음의 centerness를 예측하도록 학습된다.

$$\text{Centerness} = \sqrt{\frac{\min(L,R)}{\max(L,R)} \times \frac{\min(T,B)}{\max(T,B)}}$$

인퍼런스 시 최종 score는 **classification score × centerness score** 로 계산된다.

GT 박스가 corners 기준 `(x1, y1, x2, y2) = (0, 0, 8, 8)` 일 때, 박스 내부의 세 location P1, P2, P3에 대해 (1) 각 location의 centerness를 구하고, (2) 최종 score가 가장 높아 대표 prediction으로 선택되는 location을 고르시오.

| location | 좌표 (x, y) | classification score |
|---|---|---:|
| P1 | (4, 4) | 0.8 |
| P2 | (2, 2) | 0.9 |
| P3 | (4, 2) | 0.7 |

#### 답안

각 location의 거리 (L, R, T, B) — L: 왼쪽 변(x=0), R: 오른쪽 변(x=8), T: 위 변(y=0), B: 아래 변(y=8)까지의 거리.

- **P1 (4, 4)**: (L,R,T,B) = (4,4,4,4) → $\sqrt{\frac{4}{4}\times\frac{4}{4}} = \mathbf{1.0}$
- **P2 (2, 2)**: (L,R,T,B) = (2,6,2,6) → $\sqrt{\frac{2}{6}\times\frac{2}{6}} = \frac{1}{3} \approx \mathbf{0.333}$
- **P3 (4, 2)**: (L,R,T,B) = (4,4,2,6) → $\sqrt{1\times\frac{2}{6}} = \frac{1}{\sqrt{3}} \approx \mathbf{0.577}$

최종 score = classification × centerness:

- P1: $0.8 \times 1.0 = \mathbf{0.8}$
- P2: $0.9 \times 0.333 = 0.3$
- P3: $0.7 \times 0.577 \approx 0.404$

→ **P1이 선택된다.** (classification score가 가장 높은 P2가 아니라, 박스 중심에 있어 centerness가 1인 P1이 선택됨 — centerness가 가장자리 location의 낮은 품질 박스를 억제하는 역할을 함을 보여줌.)

### 예상문제 2 — Precision/Recall/AP 계산과 PR curve의 지그재그 (계산+서술형)

어떤 단일 클래스에 대해 이미지에 ground-truth 오브젝트가 총 **4개** 있고, 디텍터가 5개의 박스를 출력했다. confidence score 내림차순으로 정렬한 결과와 정답 여부(IoU ≥ 0.5 기준)는 다음과 같다.

| 순위 | confidence | 판정 |
|---|---:|---|
| 1 | 0.95 | TP |
| 2 | 0.85 | TP |
| 3 | 0.70 | FP |
| 4 | 0.60 | TP |
| 5 | 0.50 | FP |

(1) 순위 1부터 누적으로 박스를 채택할 때, 각 순위에서의 precision과 recall을 구하시오.
(2) 이 결과로 그린 PR curve가 일반적인 ML의 PR curve와 달리 지그재그 형태가 되는 이유를 서술하시오.
(3) 각 recall 구간에서 그 지점 이후의 최대 precision으로 보간(all-point interpolation)하여 AP를 구하시오.

#### 답안

**(1)** 누적 TP/FP로 계산 (GT 총 4개이므로 recall 분모는 4):

| 순위 | 누적 TP | 누적 FP | Precision | Recall |
|---|---:|---:|---:|---:|
| 1 | 1 | 0 | 1/1 = 1.000 | 1/4 = 0.25 |
| 2 | 2 | 0 | 2/2 = 1.000 | 2/4 = 0.50 |
| 3 | 2 | 1 | 2/3 ≈ 0.667 | 2/4 = 0.50 |
| 4 | 3 | 1 | 3/4 = 0.750 | 3/4 = 0.75 |
| 5 | 3 | 2 | 3/5 = 0.600 | 3/4 = 0.75 |

**(2)** score 순으로 박스를 하나씩 추가하며 곡선을 그리는데, **FP가 나오는 순간 precision이 뚝 떨어졌다가(recall은 그대로), 다음 TP가 나오면 precision이 다시 올라가기 때문**이다. 위 표에서도 순위 3(FP)에서 precision이 1.0 → 0.667로 떨어졌다가 순위 4(TP)에서 0.75로 회복된다. 일반 ML의 PR curve는 단조 감소하지만 디텍션은 이런 등락이 반복되어 지그재그가 된다.

**(3)** recall 구간별 보간 precision ($p_{interp}(r) = \max_{\tilde r \geq r} p(\tilde r)$):

- recall 0 ~ 0.25 구간: max precision = 1.0
- recall 0.25 ~ 0.50 구간: max precision = 1.0
- recall 0.50 ~ 0.75 구간: max precision = 0.75
- recall 0.75 ~ 1.00 구간: 4번째 GT를 끝내 못 찾았으므로(FN) precision = 0

$$AP = 0.25(1.0) + 0.25(1.0) + 0.25(0.75) + 0.25(0) = \mathbf{0.6875}$$

(참고: 이 AP를 모든 클래스에 대해 평균 내면 mAP, IoU threshold를 0.5~0.95까지 0.05 단위로 바꿔가며 평균 내면 일반 mAP가 된다.)

### 예상문제 3 — Hard Negative Mining (서술형, 확정문제 1 스타일)

SSD를 비롯한 객체 탐지기 학습에서 사용되는 **Hard Negative Mining**이 무엇인지 설명하고, 이를 사용하지 않고 임의의 위치에서 negative 샘플을 뽑아 학습할 경우 발생하는 문제를 서술하시오.

#### 답안

- **Hard Negative Mining**: 현재까지 학습된 디텍터가 **positive(오브젝트)로 잘못 판단한 샘플(false positive)** 을 골라 negative 학습 샘플로 계속 공급하는 기법이다. 즉, 모델이 "어려워하는" negative를 집중적으로 학습시킨다.
- **사용하지 않을 경우의 문제**: 이미지의 대부분은 배경이므로 임의로 뽑은 negative는 대부분 오브젝트와 전혀 닮지 않은 **너무 쉬운 샘플**이다. 쉬운 negative는 이미 잘 분류되어 loss 기여가 거의 없으므로 학습 효과가 낮고, 정작 오브젝트와 혼동되는 어려운 배경 패턴을 구분하지 못해 **false positive(false alarm)가 많은 디텍터**가 된다. Hard negative mining을 쓰면 어려운 negative를 정확히 negative로 분류하게 되어 precision이 향상된다.

## 4. 보너스 — 추가로 준비해두면 좋은 단답 후보

4번째 문제가 위 3개를 빗나갈 경우를 대비한 한 줄 답안:

1. **DETR이 NMS가 필요 없는 이유는?** → Hungarian algorithm 기반 bipartite matching으로 object query와 GT를 **1대1로 매칭**해 학습하므로, 하나의 오브젝트에 하나의 prediction만 나오도록 set prediction으로 학습되기 때문 (중복 prediction 자체가 억제됨).
2. **DETR 매칭 코스트에서 IoU 대신 GIoU를 쓰는 이유는?** → 두 박스가 겹치지 않으면 IoU는 항상 0이라 **그래디언트(거리 정보)가 없음**. GIoU는 enclosing box C의 빈 영역을 페널티로 줘서 멀리 있을수록 값이 작아지므로 겹치지 않는 박스끼리도 구별 가능.
3. **DETR의 약점과 Deformable DETR의 해결책은?** → 수렴 느림(500 vs 50 epoch)·작은 오브젝트 약함 → query마다 학습된 offset으로 **K개 key만 샘플링**하는 deformable attention + multi-scale attention으로 해결, epoch 약 10배 감소.
4. **RoI Pooling의 동작은?** → proposal 좌표를 integer로 **rounding** → 영역을 **7×7로 분할** → 각 셀에서 **max pooling** → 어떤 크기의 proposal이든 같은 차원의 feature 출력.
5. **Dilated convolution의 장점은?** → 필터 파라미터 수는 같지만 띄엄띄엄 보므로 **receptive field가 커지고**, pooling(stride)과 달리 **feature map 사이즈가 유지**됨 (dense prediction에 유리).
6. **YOLO V1의 한계는?** → 셀당 박스 2개, 총 7×7×2 = **98개**로 박스가 극단적으로 적고, 한 셀에 여러 오브젝트의 중심이 떨어지면 **하나만 검출**("한 놈만 패는" 문제). localization error가 주된 오류.
