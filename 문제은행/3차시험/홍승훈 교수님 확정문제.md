# Samsung AI Expert 2026 과정 문제

과목명: **Semantic Segmentation**  
담당교수: **홍승훈**

---

## 문제 1

이미지 분류용 CNN을 FCN으로 변환하면 왜 임의 크기의 입력 이미지에 대해 spatial score map을 출력할 수 있는지 설명하고, 이 score map으로부터 최종 semantic segmentation label map을 얻는 과정을 설명하시오.

### 답안 (정답 예시)

Fully-connected layer는 고정된 크기의 입력 feature 전체에 대해 하나의 vector를 출력하므로 입력 크기가 고정되어야 한다. 하지만 이를 convolution layer로 바꾸면 같은 연산을 spatial location마다 sliding 방식으로 적용할 수 있다.

Semantic segmentation에서는 각 위치마다 class별 score가 생성되므로, 최종적으로 각 pixel 또는 feature 위치에서 class score가 가장 큰 class를 선택하는 **per-pixel argmax**를 수행하여 label map을 얻는다. 필요한 경우 score map을 원본 이미지 해상도로 upsampling한다.

## 문제 2

기본 FCN의 대표적인 한계 두 가지를 설명하고, 이를 개선하기 위해 사용되는 다음 두 방법이 각각 어떤 문제를 해결하는지 설명하시오:

1. Encoder-decoder network
2. Atrous convolution

### 답안 (정답 예시)

기본 FCN의 첫 번째 한계는 여러 번의 downsampling으로 인해 score map의 해상도가 낮아지고, 그 결과 물체의 경계나 세부 형태가 손실될 수 있다는 점이다. 두 번째 한계는 receptive field가 고정되어 있어 작은 물체와 큰 물체, 또는 주변 문맥이 중요한 경우를 모두 잘 처리하기 어렵다는 점이다.

1. **Encoder-decoder network**는 encoder에서 손실된 공간 정보를 decoder에서 복원하여 더 정교한 segmentation map을 만들기 위한 구조이다. 특히 skip connection이나 deconvolution/upsampling을 통해 낮은 수준의 위치 정보와 높은 수준의 semantic 정보를 결합할 수 있다.

2. **Atrous convolution**은 convolution filter 사이에 hole을 두어 parameter 수를 늘리지 않고 receptive field를 넓히는 방법이다. 따라서 더 넓은 문맥을 보면서도 feature map 해상도를 지나치게 낮추지 않을 수 있다.

## 문제 3

Semantic segmentation 관점에서 convolution 기반 encoder와 self-attention 기반 Transformer encoder의 차이를 **receptive field**, **입력 의존성**, **parameter sharing** 측면에서 비교하시오.

### 답안 (정답 예시)

**Convolution**은 주로 local neighborhood에 대해서만 정보를 집계한다. 따라서 한 layer의 receptive field는 kernel size에 의해 제한되며, 넓은 문맥을 보려면 layer를 많이 쌓거나 pooling/downsampling, atrous convolution 등을 사용해야 한다. Convolution filter는 위치마다 공유되므로 parameter-efficient하고 translation invariance를 잘 활용하지만, 각 위치에서 어떤 주변 정보를 중요하게 볼지는 입력에 따라 동적으로 크게 달라지지 않는다.

**Self-attention**은 각 token이 모든 다른 token과 pairwise attention을 계산할 수 있으므로 한 layer에서도 전체 입력을 receptive field로 가질 수 있다. Attention weight는 query와 key의 유사도에 의해 입력마다 동적으로 결정되므로, 같은 위치 관계라도 이미지 내용에 따라 중요한 context를 다르게 선택할 수 있다.
