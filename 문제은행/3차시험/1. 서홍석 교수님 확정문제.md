## 문제 1

단일 RGB 프레임만 입력받는 일반 CNN이 action recognition에서 갖는 근본적 한계를 쓰고, two-stream 구조가 이를 어떤 방식으로 보완하는지 설명하시오.

### 답안

- 한 프레임은 외형(appearance)만 담고 시간/움직임 정보가 없음 → 정지 화면이 비슷한 행동을 구분하기 어려움
- two-stream은 spatial(RGB = 외형) + temporal(optical flow = 움직임) 두 경로로 분리
- "무엇이 보이는가"와 "어떻게 움직이는가"라는 상보적(complementary) 단서를 따로 학습한 뒤 융합

## 문제 2

Temporal stream의 입력으로 쓰이는 optical flow가 무엇을 나타내는지, 입력 채널 수가 왜 $2L$이 되는지, 그리고 그것이 한 장의 RGB 프레임이 줄 수 없는 어떤 정보를 제공하는지 설명하시오.

### 답안

- optical flow = 연속 두 프레임 사이의 픽셀별 변위 $(u, v)$ (수평·수직 성분)
- flow 한 장 = 2 채널, $L$장을 쌓으므로 $2L$ 채널
- 정지 외형이 아닌 단기 움직임(방향·크기)을 명시적으로 제공 → 배경(정지)은 억제되고 움직이는 객체가 강조됨

## 문제 3

작은 규모의 image classification dataset에서는 CNN 계열 모델이 ViT보다 안정적으로 학습되는 경우가 많지만, 대규모 dataset 또는 대규모 pretraining 환경에서는 ViT 계열 모델이 강한 성능을 보일 수 있다. 이 현상이 발생하는 구조적 이유를 설명하시오.

### 답안

CNN은 이미지를 2D feature map으로 유지하면서 local convolution을 적용한다. Convolution은 가까운 픽셀들이 중요한 패턴을 만든다는 locality를 활용하고, 같은 filter를 여러 spatial location에 공유하여 local pattern을 검출한다. 또한 convolution layer는 입력이 이동하면 feature map도 그에 맞게 이동하는 translation equivariance 성질을 가진다. 따라서 CNN은 이미지의 local structure를 구조적으로 잘 활용할 수 있어, 비교적 작은 dataset에서도 안정적으로 feature를 학습할 수 있다.

반면 ViT는 이미지를 patch로 나누고 각 patch를 token으로 변환한 뒤 self-attention을 적용한다. Self-attention은 patch token들 사이의 관계를 직접 학습할 수 있어, 이미지의 여러 영역 사이의 관계를 유연하게 모델링할 수 있다. 그러나 self-attention 자체는 token의 순서나 위치를 알지 못하므로, positional encoding을 통해 각 patch의 spatial position을 알려주어야 한다.

따라서 작은 dataset에서는 이미지의 local structure를 구조적으로 활용하는 CNN 계열 모델이 더 안정적으로 학습될 수 있다. 하지만 dataset이나 pretraining 규모가 충분히 커지면, ViT는 patch token들 사이의 관계를 self-attention으로 학습하면서 더 유연한 image representation을 만들 수 있다. 그래서 대규모 환경에서는 ViT 계열 모델이 강한 image classification 성능을 보일 수 있다.

## 문제 4

VGGNet과 ResNet은 ImageNet classification 성능을 높이기 위해 더 깊은 CNN 구조를 사용하였다. VGGNet이 작은 $3 \times 3$ convolution을 반복적으로 사용한 이유를 설명하고, ResNet이 residual connection을 도입한 배경을 설명하시오.

### 답안

**VGGNet**

VGGNet은 큰 convolution filter를 사용하는 대신 작은 $3 \times 3$ convolution을 여러 층 반복해서 사용하였다. 작은 filter를 여러 번 쌓으면 큰 filter와 같은 effective receptive field를 얻을 수 있다. 예를 들어 $3 \times 3$ convolution 세 개를 연속으로 쌓으면 $7 \times 7$ convolution과 같은 범위의 정보를 볼 수 있다.

이 방식의 장점은 두 가지이다. 첫째, 하나의 큰 convolution을 쓰는 것보다 parameter 수를 줄일 수 있다. 둘째, convolution 사이마다 ReLU 같은 non-linearity가 들어가기 때문에 더 복잡한 feature를 표현할 수 있다. 따라서 VGGNet은 작은 filter를 깊게 쌓아 receptive field와 표현력을 확보한 classification 모델이다.

**ResNet**

ResNet은 단순히 CNN을 깊게 쌓는 것만으로는 성능이 계속 좋아지지 않는다는 문제에서 출발한다. plain CNN에서는 layer 수가 증가했는데도 training error와 test error가 모두 나빠질 수 있다. 이는 overfitting 문제가 아니라, 깊은 네트워크를 최적화하기 어려워지는 **degradation problem**이다.

ResNet은 이를 해결하기 위해 residual connection을 사용한다. 일반적인 block이 원하는 변환 $H(x)$를 직접 학습한다면, ResNet은 $H(x) = F(x) + x$ 와 같이 입력 $x$에 residual $F(x)$를 더하는 형태를 학습한다. 이 구조에서는 residual branch가 $F(x) = 0$에 가까운 출력을 내면 identity mapping $H(x) = x$를 쉽게 만들 수 있다. 따라서 깊은 모델이 적어도 얕은 모델의 동작을 유지하는 방향으로 학습될 수 있다.

결과적으로 ResNet은 layer를 매우 깊게 쌓았을 때 발생하는 optimization difficulty를 residual mapping으로 완화하여, 깊은 image classifier를 효과적으로 학습할 수 있게 만든 구조이다.
