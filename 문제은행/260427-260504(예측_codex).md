## 예측 문제 1. Language Model과 Autoregressive Generation

다음 질문에 답하시오.

### (1) 언어 모델(Language Model)의 원래 정의를 확률 관점에서 설명하시오.

### (2) 문장 확률 $P(w_1, w_2, \dots, w_T)$를 next-token probability의 곱으로 분해하여 표현하시오.

### (3) Autoregressive generation이 무엇인지 설명하고, BERT와 GPT 중 어느 모델이 이에 더 직접적으로 해당하는지 쓰시오.

### 예상 정답

**(1)** 언어 모델은 주어진 맥락(context) 다음에 어떤 단어/token이 등장할 확률을 모델링하는 모델이다. 즉 $P(w_t \mid w_1, \dots, w_{t-1})$를 예측한다.

**(2)**

$$
P(w_1, w_2, \dots, w_T)
= \prod_{t=1}^{T} P(w_t \mid w_1, \dots, w_{t-1})
$$

**(3)** Autoregressive generation은 지금까지 생성한 token들을 다시 context로 넣고 다음 token을 하나씩 순차적으로 생성하는 방식이다. GPT가 decoder-only causal language model이므로 이에 더 직접적으로 해당한다. BERT는 입력 전체를 보고 masked token을 예측하는 encoder-only 모델이므로 일반적인 autoregressive generation 모델은 아니다.

### 출제 가능성이 높은 이유

- `260504(오전)`에서 교수님이 8문제 복습 후 “무엇이 language model인가”, “맥락 다음 단어 발생 확률”, “autoregressive model”을 다시 설명했다.
- 기존 4번 문제에 next-token prediction이 나오지만, language model의 원래 정의와 문장 확률 분해 자체는 빠져 있다.
- 객관식/단답형/짧은 수식형으로 내기 좋다.

## 예측 문제 2. Pre-training, Instruction Tuning, RLHF의 차이

LLM이 사용자의 지시를 잘 따르는 assistant가 되기까지의 학습 과정을 다음 세 단계로 나누어 설명하시오.

### (1) Pre-training

### (2) Instruction Tuning 또는 SFT

### (3) RLHF

또한 pre-training만 수행한 GPT-3류 모델이 사용자의 질문에 바로 답하지 못하고 text completion처럼 동작할 수 있는 이유를 설명하시오.

### 예상 정답

**(1) Pre-training**은 대량의 일반 텍스트로 next-token prediction을 학습하는 단계이다. 언어 패턴, 문법, 지식, 상식을 배운다.

**(2) Instruction Tuning/SFT**는 instruction-output 쌍을 이용해 “이런 명령이 오면 이런 답변을 해야 한다”는 행동을 학습시키는 단계이다. 모델을 instruction-following assistant에 가깝게 만든다.

**(3) RLHF**는 사람의 선호 데이터를 이용해 reward model을 학습하고, PPO 같은 강화학습으로 사람이 더 선호할 답변을 생성하도록 조정하는 단계이다.

Pre-training만 한 모델은 기본적으로 “사용자 요청에 답하는 모델”이 아니라 “주어진 텍스트 뒤에 올 법한 텍스트를 이어 쓰는 모델”이다. 그래서 질문에 답하는 대신 비슷한 질문 패턴이나 뒤따를 법한 문장을 계속 생성할 수 있다.

### 출제 가능성이 높은 이유

- `260430(오후)`와 `260504(오전)`에서 pre-training, instruction tuning, RLHF의 연결이 반복된다.
- 기존 7, 8번은 RLHF/DPO/PPO에 집중하지만 전체 post-training pipeline 비교는 빠져 있다.
- 교수님이 “선행학습, Instruction Fine Tuning, RLHF 세 페이지로 나누어 배웠다”고 직접 언급했다.

## 예측 문제 3. T5의 Text-to-Text Framework와 Prefix

T5(Text-to-Text Transfer Transformer)에 대해 다음 질문에 답하시오.

### (1) T5는 Transformer 기반 모델 분류 A) Encoder-only, B) Decoder-only, C) Encoder-Decoder 중 어디에 속하는가?

### (2) T5의 text-to-text framework가 무엇인지 예시와 함께 설명하시오.

### (3) T5에서 task prefix를 붙이는 이유를 설명하시오.

### 예상 정답

**(1)** T5는 **C) Encoder-Decoder** 모델이다.

**(2)** Text-to-text framework는 번역, 요약뿐 아니라 분류나 회귀 문제까지 모두 “텍스트 입력 → 텍스트 출력” 문제로 바꾸는 방식이다. 예를 들어 문법성 판단도 class 확률을 직접 출력하는 대신 `acceptable` 또는 `not acceptable`이라는 문자열을 출력하게 만들 수 있다.

**(3)** Prefix는 입력이 어떤 task인지 모델에게 알려주는 표시이다. 예를 들어 `translate English to German: ...`, `summarize: ...`, `cola sentence: ...`처럼 prefix를 붙이면 같은 encoder-decoder 모델과 같은 decoder를 사용하더라도 task를 구분할 수 있다.

### 출제 가능성이 높은 이유

- 기존 4번 문제는 BERT와 GPT만 묻고, 교수님은 `260504(오전)`에서 BERT/GPT/T5를 architecture, data, learning objective의 세 축으로 볼 수 있다고 말했다.
- `260430(오전)`의 큰 주제가 T5였지만 기존 8문제에는 T5가 거의 없다.
- 모델 분류형 문제 스타일과 잘 맞는다.

## 예측 문제 4. T5 Span Corruption과 일반 Language Modeling 비교

T5의 pre-training objective와 관련하여 다음 질문에 답하시오.

### (1) Span corruption이 무엇인지 설명하시오.

### (2) 일반적인 autoregressive language modeling과 span corruption의 차이를 설명하시오.

### (3) 왜 span corruption이 encoder-decoder 모델 학습에 도움이 될 수 있는지 설명하시오.

### 예상 정답

**(1)** Span corruption은 입력 문장에서 하나의 token이 아니라 연속된 여러 token span을 가리고, decoder가 그 가려진 span을 생성하도록 학습하는 방식이다.

**(2)** 일반 autoregressive language modeling은 앞쪽 context를 보고 뒤쪽 token을 순서대로 예측한다. 반면 span corruption은 문장의 앞, 중간, 뒤 어디든 일부 span을 가릴 수 있고, 모델은 손상된 입력 전체를 이해해서 빠진 span을 복원해야 한다.

**(3)** Span corruption은 encoder가 입력 전체의 정보를 더 고르게 이해하도록 만든다. 단순히 바로 앞 context만 잘 보면 되는 next-token prediction보다 어려운 denoising task이므로, encoder-decoder 모델이 입력 전체를 압축하고 활용하는 능력을 기르는 데 도움이 된다.

### 출제 가능성이 높은 이유

- T5 pre-training 설명 중 가장 구체적이고 시험화하기 쉬운 개념이다.
- “MLM은 token을 가리고, span corruption은 연속된 span을 가린다”는 식의 비교형 단답으로 출제하기 좋다.
- 기존 문제 3의 attention bottleneck처럼 “왜 이 방식이 필요한가”를 묻는 스타일과 맞는다.
