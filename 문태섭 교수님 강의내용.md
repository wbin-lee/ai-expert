# 1부. 260715 오전 — Self-Supervised Learning의 세계

## 1-1. 왜 셀프 슈퍼바이즈드 러닝을 배우는가

문태섭 교수님은 서울대 전기정보공학부에서 강의를 맡으셨습니다. 알고리즘 트랙의 **거의 마지막 강의**에 해당하고, 오늘·내일 이틀 구성입니다.

원래 계획서에는 “메타러닝, 컨트라스티브 러닝”이 적혀 있었지만, 준비를 하다 보니 **contrastive learning 자체가 self-supervised learning(SSL)의 대표 사례**라서, 오늘은 **SSL 전체 지도**를 중심으로 풀기로 하셨습니다.

- **1일차 오전**: SSL이란 무엇인가, pretext task, contrastive / non-contrastive / JEPA / CLIP, autoregressive와의 연결  
- **1일차 오후**: Transformer의 병목 — **Efficient / Linear Attention**  
- **2일차**: 머신러닝의 **Fairness·Bias**, 마지막에 **LLM에서의 social bias**

회사 현장에서 가장 자주 마주치는 현실부터 시작합니다. 지도 학습(supervised learning)은 2012년 전후 딥러닝 붐의 기본 틀이었습니다. 입력 \(X\)가 있으면 정답 \(Y\)를 붙여 주고, “이 패턴이면 이 답”을 반복해 가르칩니다. 음성 인식, 비전, 기계 번역이 모두 이 패러다임으로 성공했습니다.

문제는 **\(Y\)를 만드는 비용**입니다. 사람을 고용해 라벨을 달고, 검수하고, 그래도 노이즈가 남습니다. 개념은 쉬운데 데이터가 없다 — 현장의 단골 불만입니다.

그래서 질문이 바뀝니다.

> **레이블 없는 데이터만으로, 다른 과제에도 쓸 수 있는 표현(representation)을 배울 수 있는가?**

## 1-2. SSL이란 무엇인가 — “스스로 만든 정답”

**Unsupervised learning**(클러스터링, K-means, EM 등)은 범위가 넓고 말이 모호합니다.  
**Self-supervised learning**은 조금 더 구체적입니다. 데이터 안의 구조를 이용해 **스스로 supervision signal(가짜 정답)을 만들고**, 그 가짜 문제를 풀면서 표현을 학습합니다.

Yann LeCun이 2018 NeurIPS 키노트에서 강조한 메시지와도 같습니다. 외부 전문가가 준 라벨이 아니라, **데이터로부터 self-generated label**을 만들어 학습하면 효과적일 수 있다. 심지어 “SSL은 supervised보다 **더 많은** supervision signal을 쓸 수 있다”는 식으로 말하기도 합니다. 설계만 잘하면 신호의 종류가 무한히 늘어나기 때문입니다.

강화학습과 대비하면 더 선명합니다. RL의 reward는 드물고(sparse) 샘플 비용이 큽니다. SSL은 데이터 한 장에서도 **촘촘한 학습 신호**를 뽑을 수 있습니다.

그 “스스로 만든 문제”를 **pretext task**(또는 self-supervised task)라고 부릅니다. 시계열로 비유하면, 과거로 미래를 예측하거나, 중간을 비우고 맞히거나, 순서를 섞어 복원하는 식의 여러 변형이 모두 pretext입니다. **잘 디자인된 pretext를 풀다 보면**, downstream(분류·검출 등)에 유용한 피처가 생긴다는 것이 SSL의 핵심 약속입니다.

## 1-3. SSL은 어떻게 평가하나

지도 학습은 train/test로 정답을 맞히면 됩니다. SSL은 라벨이 없으니 **표현이 얼마나 쓸모 있는지**로 평가합니다.

표준 파이프라인은 이렇습니다.

1. 대용량 **unlabeled** 데이터로 SSL pretrain  
2. 레이블이 있는 **downstream** 태스크로 가져감  
3. 두 가지 평가 모드  
   - **Linear probing**: feature extractor를 **고정**하고 마지막 선형 분류기만 학습  
   - **Fine-tuning**: 아래 레이어까지 함께 학습  

예: ImageNet으로 SSL pretrain → Stanford Cars 등에서 linear/finetune.  
다른 말로 하면 **transfer learning 능력**을 재는 것이고, 그래서 SSL을 **representation learning**이라고도 부릅니다.

> ⭐ **시험 포인트**  
> SSL의 목표는 pretext 자체를 “맞히는 것”이 아니라, **downstream에 전이 가능한 representation**을 얻는 것. 평가는 대개 **linear probing / fine-tuning**.

## 1-4. 초창기: 이미지 변환으로 가짜 분류 문제 만들기

가장 직관적인 출발은 **image transform** 계열입니다.

**Exemplar-based surrogate class**  
각 이미지를 “한 반”으로 보고, 색·크롭·필터 등 distortion을 잔뜩 만듭니다. 같은 원본에서 나온 변형들은 같은 클래스로 두고 분류기를 학습합니다. supervised에 익숙한 사람이 가장 먼저 떠올릴 법한 방식입니다.

**Rotation prediction**  
0°/90°/180°/270°로 돌린 뒤 **몇 도 돌았는지** 맞춥니다. 라벨 없이도 ConvNet을 학습할 수 있고, 나중에 backbone만 떼어 feature extractor로 씁니다.

**Relative position / Jigsaw**  
패치 하나의 상대 위치를 맞히거나, 9조각 퍼즐 순서를 복원합니다. “장난감 문제”처럼 보이지만, 잘 풀려면 물체와 장면 구조를 알아야 합니다. 초창기 SSL은 **얼마나 기발한 pretext를 만드느냐**의 시대였습니다.

## 1-5. 생성·복원 계열 — 지워진 것을 되살리며 배운다

변환이 “가짜 분류”라면, 다음 갈래는 **generation / reconstruction**입니다.

**Denoising Autoencoder**  
입력을 일부 지우거나 노이즈를 섞은 뒤, bottleneck을 거쳐 **원본을 복원**합니다. Loss는 원본과 재구성 사이의 MSE. 2008년 무렵부터 “robust feature를 준다”고 알려진 고전입니다.

**Context Encoder (Inpainting)**  
랜덤 노이즈가 아니라 **특정 패치를 통째로 가린 뒤** 채웁니다. GAN 스타일 adversarial loss를 붙이면 채움 품질과 representation이 더 좋아진다는 이야기도 나옵니다.

**Denoising의 보편성**  
“주변을 보고 안 보이는 것을 예측한다”는 문법은  orth 분야로 그대로 갑니다.  
**BERT**의 masked language modeling은 토큰을 가리고 context로 맞히는 **이산 버전 denoising**입니다. Downstream(QA, NER 등)과 pretext가 직접 같아 보이지 않아도 transfer가 잘 됐습니다.

다시 비전으로 오면 **BEiT**(Masked Image Modeling)가 있습니다. ViT 패치를 마스크하고, 픽셀 회귀 대신 **dVAE 등으로 이산화한 visual token ID를 분류**합니다. 놀랍게도 ImageNet 레이블 **없이** pretrain한 모델이, 같은 크기 supervised pretrain보다 나은 경우도 보고되었습니다.

**MAE(Masked Autoencoder)** 는 한 걸음 더 갑니다. 마스크 토큰을 인코더에 넣지 않고 **살아남은 패치만 인코더**에 넣고, 디코더에서 마스크 자리를 복원합니다. 디코더는 가볍게, 인코더는 무겁게, **masking ratio를 크게** 잡는 것이 잘 통했습니다. 큰 모델 + 높은 마스크 비율이 linear probing·finetune 성능을 끌어올렸습니다.

> ⭐ **시험 포인트**  
> Transform 계열 = 가짜 **분류** pretext.  
> Generation 계열 = **복원/예측** pretext.  
> BERT·BEiT·MAE는 모두 “가린 것을 맞힌다”는 **같은 철학의 다른 옷**.

## 1-6. Contrastive Learning — 끌어당기고 밀며 배운다

지금까지 pretext는 대개 **한 샘플 안**에서 문제를 만들었습니다.  
**Contrastive learning**은 **여러 샘플을 동시에** 봅니다.

- 비슷한 샘플의 representation → **가깝게**  
- 다른 샘플의 representation → **멀게**

### 고전 loss

**Contrastive loss**(지도 설정에서 먼저): 같은 레이블은 모이고, 다른 레이블은 margin \(\epsilon\) 이상 떨어지게(hinge).  
**Triplet loss**: (anchor, positive, negative) 세 점으로, anchor–positive 거리 + margin < anchor–negative 거리.

### NCE → InfoNCE — “제대로 쓰이기 시작한” 계기

**Noise Contrastive Estimation(NCE)** 은 실제 데이터와 인위적 noise를 구분하는 log-odds를 학습합니다.  
**CPC(Contrastive Predictive Coding)** 는 시퀀스 context \(c_t\)와 미래 \(x_{t+k}\)의 **mutual information**을 키우려 하고, 그 과정에서 **InfoNCE** 형태 loss가 나옵니다. 교수님은 유도 세부는 중요하지 않다고 하셨습니다. 기억할 것은 한 줄입니다.

> InfoNCE를 줄인다 = **positive의 점수(유사도)를 키우고, negative의 점수를 줄인다**. Softmax 분류와 거의 같은 식.

Unlabeled 설정에서 positive/negative는 이렇게 둡니다.

| | 정의 |
|--|------|
| **Positive** | 같은 이미지의 **augmentation view** (크롭·색 변환 등) |
| **Negative** | 배치(또는 큐) 안의 **다른 이미지** |

**Hard negative**가 중요합니다. 비행기 vs 강아지처럼 너무 쉬운 negative만 있으면 학습이 쉽고 표현이 거칠어집니다. 미묘하게 다른 negative가 있어야 fine-grained feature가 생깁니다. Negative pool이 클수록 hard negative가 들어올 확률이 커집니다. 다만 의미상 같은 이미지를 negative로 취급하는 **false negative** 문제도 있습니다.

> ⭐ **시험 포인트 (교수 직접 출제 언급)**  
> **Contrastive learning의 loss function** — 특히 InfoNCE/NT-Xent 직관:  
> \(\ell = -\log \dfrac{\exp(\mathrm{sim}(z,z^+)/\tau)}{\exp(\mathrm{sim}(z,z^+)/\tau)+\sum_k \exp(\mathrm{sim}(z,z_k^-)/\tau)}\)  
> → positive 유사도↑, negative 유사도↓.  
> Unlabeled에서는 positive = augmentation pair.

## 1-7. SimCLR — 단순한 공식, 큰 배치

**SimCLR**(Simple Framework for Contrastive Learning)은 위 아이디어를 거의 그대로 깔끔히 구현한 방법입니다.

1. 이미지에 **독립 랜덤 augmentation을 두 번** → positive pair  
2. Encoder \(f\)로 \(h_i, h_j\)  
3. **Projection head** \(g\)(MLP)로 \(z_i, z_j\)  
4. \(z\) 공간에서 **InfoNCE(NT-Xent)**  
5. Downstream에서는 \(g\)를 버리고 **\(h\)** 만 사용  

Projection head는 “굳이 왜?” 싶은 **휴리스틱**이지만, loss와 실제 쓸 representation 사이에 **자유도 한 겹**을 주면 \(h\)가 더 robust해진다고 알려져 있습니다. 이후 방법들에도 “쓸 feature + 추가 head” 패턴이 반복됩니다.

SimCLR의 축복이자 저주는 **batch size**입니다. Mini-batch 안에서 negative를 만들기 때문에, 배치가 커질수록 hard negative가 늘고 성능이 오르는 경향이 있습니다. 대신 GPU 비용이 큽니다.

> 비유: 한 반에 학생이 많을수록 “나와 비슷하지만 다른 아이”를 만날 확률이 커진다. 대신 교실(GPU)이 커야 한다.

## 1-8. MoCo — 큐와 모멘텀으로 배치를 대신한다

SimCLR식 end-to-end는 거대 배치가 부담입니다. **MoCo(Momentum Contrast)** 는 배치를 안 키우고도 **넓은 negative 세계**를 보려 합니다.

| 구성요소 | 역할 |
|----------|------|
| **Query encoder** \(\theta_q\) | backprop으로 학습 |
| **Key (momentum) encoder** \(\theta_k\) | \(\theta_q\)의 EMA/momentum 추적 |
| **Queue** | 최근 배치 key representation을 FIFO로 저장 |
| **Loss** | InfoNCE (query vs keys) |

Queue 크기를 배치보다 크게 두면 **effective negative 수**가 늘어납니다. Key encoder를 천천히 갱신하므로, 옛 feature만 쌓아 두는 memory bank의 “낡은 표현” 문제도 완화됩니다.

**역사 순서**(교수님이 정정하신 부분):

1. **MoCo v1** 먼저  
2. **SimCLR**: 배치 왕창 + 강한 augmentation → 성능 점프  
3. **MoCo v2**: SimCLR의 MLP head·강한 aug를 가져와, **작은 배치로도 큰 배치 SimCLR 이상** 가능

| | SimCLR | MoCo |
|--|--------|------|
| Negative | 거대 mini-batch | Queue |
| Encoder | 공유 end-to-end | Query + momentum key |
| 배치 민감도 | 매우 큼 | 상대적 작음 |

## 1-9. Negative 없이 — BYOL, Barlow Twins, DINO

Negative를 모으는 일이 복잡해지자, **negative 없는 SSL**이 등장합니다.

**BYOL(Bootstrap Your Own Latent)**  
- 두 augmentation view  
- **Student**(online): encoder + projection + **predictor**  
- **Teacher**(target): student의 **momentum EMA** (gradient 없음)  
- Student predictor 출력이 teacher 출력에 가깝게 (normalized MSE류)  
- Teacher 경로에 **stop-gradient**  

겉보기엔 “ alike만 만들면 상수로 붕괴”할 것 같지만, predictor + stop-grad + EMA가 자기 부트스트랩을 유지합니다. SimCLR보다 **batch size·augmentation에 robust**합니다.

**Barlow Twins**  
두 view embedding의 **cross-correlation matrix**에서, 대각은 1에 가깝게(같은 차원 일치), 비대각은 0에 가깝게(차원 간 **decorrelate**). Negative 대신 **차원 간 중복 제거**.

**DINO**  
BYOL 친척. Softmax 확률끼리 self-distillation, teacher temperature를 작게 해 sharper 분포, teacher에는 global view·student에는 local crop.

## 1-10. JEPA — 픽셀이 아니라 임베딩 공간에서 예측

Yann LeCun이 밀고 있는 **JEPA(Joint Embedding Predictive Architecture)** 는 세 갈래를 정리합니다.

| 아키텍처 | 대표 | 무엇을 맞추나 |
|----------|------|----------------|
| Joint Embedding | SimCLR, MoCo, BYOL, DINO | 두 view embedding 정합/대조 |
| Generative | BEiT, MAE | 픽셀/토큰 **원공간 복원** |
| **JEPA** | I-JEPA 등 | **임베딩 공간**에서 마스킹 부분 예측 |

픽셀 복원은 상관없는 디테일·노이즈까지 맞추려 합니다. 임베딩에서 예측하면 더 **robust·효율적**이라는 철학입니다. 비디오·월드모델 쪽으로도 확장됩니다.

## 1-11. CLIP — 이미지와 텍스트를 같은 공간에

이미지–텍스트 페어가 있으면 **멀티모달 contrastive**가 가능합니다. **CLIP**은 image encoder와 text encoder를 두고, 배치 안에서 올바른 페어의 유사도는 키우고 틀린 페어는 낮춥니다. 본질은 **InfoNCE의 멀티모달 버전**입니다.

**Zero-shot classification**이 인상적입니다. 클래스 이름을 `"a photo of a {class}"` 같은 문장으로 임베딩한 뒤, 이미지 임베딩과 가장 가까운 클래스를 고릅니다. **그 분류 태스크 라벨을 한 장도 안 보고** 분류할 수 있습니다.

확장으로는 페어가 맞는지 보는 **ITM(Image–Text Matching)**, 이미지를 보고 텍스트 마스크를 맞히는 **멀티모달 MLM** 등이 있습니다. Hard negative(미묘하게 틀린 캡션·이미지)가 여기서도 중요합니다.

## 1-12. Autoregressive 모델도 SSL이다

레이블 없는 시퀀스에서 **다음 토큰(또는 픽셀)을 예측**하는 것도 자연스러운 pretext입니다. GPT류 decoder Transformer도 넓게 보면 **SSL 패밀리**입니다.

고전 계보만 짚으면: PixelRNN/PixelCNN(이미지를 순서로 강제해 AR), WaveNet(dilated causal conv), 그리고 causal mask를 쓴 **Transformer decoder**. 이미지는 양방향 의존인데 순서를 억지로 부여해도 생성·모델링에 먹힌다는 점이 흥미롭습니다. 현대는 픽셀 직접보다 **이산 visual token AR**(DALL·E 계열) 쪽이 많습니다.

여기서 오후 주제로 다리가 놓입니다. 시퀀스 길이 \(L\)이 커지면 full attention의 **\(O(L^2)\)** 비용과 **KV cache 메모리**가 폭발합니다. 그래서 **efficient attention**이 필요합니다.

# 2부. 260715 오후 — Efficient / Linear Attention

> 오후 3시간 중 **약 1시간은 교수님 강의**, 나머지는 오전 SSL 실습입니다. 아래는 **강의만** 정리합니다.

## 2-1. Softmax Attention의 병목

디코더 기반 오토리그레시브 Transformer의 핵심은 attention입니다. Query가 각 Key와 내적해 score를 만들고, **Softmax**로 확률을 만든 뒤 Value를 가중 평균합니다. 오전 InfoNCE의 softmax 꼴과도 닮아 있습니다.

강점은 성능, 약점은 두 가지입니다.

| 문제 | 내용 |
|------|------|
| **시간** | Softmax attention ≈ **\(O(L^2)\)** (시퀀스 길이에 quadratic) |
| **추론 메모리** | **KV cache** — 과거 K/V를 계속 저장 → 길이에 따라 메모리 증가 |

> ⭐ **시험 포인트**  
> Full attention의 병목 = **quadratic compute** + **KV cache 메모리**.  
> Linear attention이 노리는 것은 이 둘을 **유한 상태 \(S\)** 로 압축하는 것.

## 2-2. Linear Attention — Softmax를 빼면 무엇이 생기나

가장 단순한 아이디어는 **Softmax를 제거**하는 것입니다. Score를 raw inner product \(q^\top k\)로 두고 Value를 가중합합니다. 음수 score가 가능하고 확률 분포는 아닙니다.

핵심은 **associativity(결합법칙)** 로 식을 재배열하는 것입니다.

\[
o_t = \sum_{j=1}^{t} (q_t^\top k_j)\, v_j
\;\;\Longrightarrow\;\;
o_t = S_t\, q_t,\quad
S_t = \sum_{j=1}^{t} v_j k_j^\top
\]

\(v k^\top\)는 **rank-1 outer product**이고, \(S_t\)는 **\(D\times D\) 고정 크기** 상태 행렬입니다. 시퀀스가 아무리 길어도 상태 차원은 \(D\)에 묶입니다.

점화식으로 쓰면 **linear RNN**과 같습니다.

\[
S_t = S_{t-1} + v_t k_t^\top,\qquad o_t = S_t q_t
\]

| | Full Attention | Linear Attention |
|--|----------------|------------------|
| 저장 | 모든 past KV | 고정 \(S\) |
| 계산 | 매 query가 모든 KV와 비교 | \(S\) 업데이트 + \(Sq\) |
| 길이↑ | 비용·메모리↑ | 상태 크기 고정 |

## 2-3. Chunk-wise Parallel — RNN이어도 빠르게

Recurrent form은 시퀀셜이라 **GPU 병렬 학습**이 어렵고, 순진한 parallel form은 다시 quadratic 느낌이 납니다. 해결은 **chunk-wise parallel form**입니다.

- 시퀀스 길이 \(L\), chunk 크기 \(C\)  
- Chunk **안**에서는 행렬곱으로 parallel  
- Chunk **사이**에서는 \(S\)만 sequential 업데이트  
- Linearity 덕분에 inter-chunk + intra-chunk를 더하면 정확한 출력  

대략 전체 복잡도 **\(O(L D^2)\)** — \(L^2\)가 아니라 \(L\)에 대해 linear입니다. 하드웨어 친화적으로 돌릴 수 있어, 이후 decay·delta rule 변형에서도 이 형태가 계속 중요합니다.

> ⭐ **시험 포인트 (교수 직접 출제 언급)**  
> **Linear attention update rule**  
> - 바닐라: \(S_t = S_{t-1} + v_t k_t^\top\)  
> - Decay(RetNet 등): \(S_t = \gamma S_{t-1} + v_t k_t^\top\) (\(0<\gamma<1\))  
> - Delta: 에러 보정 write (아래 2-5절)  
> 공통 직관: **유한 행렬 메모리 \(S\)를 rank-1로 갱신**하고 \(o_t=S_t q_t\).

## 2-4. 왜 바닐라 Linear는 약한가 — Recency Bias

수식은 예쁜데, 바닐라 linear attention 성능은 full Transformer보다 **훨씬 못하다**고 알려져 있습니다. 언어 모델링에서는 **방금 나온 토큰**이 매우 중요한데, 단순 합 \(S=\sum vk^\top\)는 순서가 뒤바뀌어도 같은 \(S\)를 만들어 **recency bias가 없습니다**.

그래서 **decay(망각)** 가 들어갑니다.

- **RetNet**: \(S_t = \gamma S_{t-1} + vk^\top\) — 과거를 지수적으로 할인  
- **Mamba2**: \(\gamma_t\)를 **입력 의존**으로 조절 (data-dependent forgetting)  
- **Gated Linear Attention**: 차원마다 다른 fine-grained gate  

Parallel form에서는 causal mask \(M\) 자리에 discount가 들어간 행렬 \(D\)가 곱해질 뿐, chunk-wise 구조는 유지됩니다.

## 2-5. Associative Memory와 Delta Rule

Linear attention을 **연관 메모리(associative memory)** 로 볼 수 있습니다. Full attention은 KV를 캐시에 **정확히 저장**하고, linear는 \(S\)에 **압축해 write**한 뒤 query로 retrieve합니다.

\(f(k)=Sk\)가 키로 밸류를 추정한다고 두고, 매 스텝 online gradient를 취하면 바닐라 update와 같은 rank-1 형태가 다시 나옵니다. 다만 negative inner product loss만 쓰면 norm이 커질수록 loss가 줄어 **수치 불안정**이 날 수 있습니다.

**Delta rule**은 loss를 **L2/MSE**로 두면 나옵니다. 직관은 “지금 \(S\)가 \(k_t\)로 예측한 값과 진짜 \(v_t\)의 **에러**만큼 보정 write”입니다. 단순히 계속 더하기만 하면 \(D\)차원에 너무 많은 KV가 쌓여 **간섭(interference)·망각**이 생깁니다(직교 키는 최대 \(D\)개). Delta는 에러를 보며 겹침을 줄이는 쪽으로 write합니다. **In-context associative recall**(나중에 특정 키–값을 정확히 꺼내기)에서 특히 유리하다는 실험 이야기가 나왔습니다.

**Gated DeltaNet** 등은 decay + delta를 합치고, 최근 LLM은 **Linear + Full Attention 하이브리드**를 쓰는 경우도 많습니다(완전 대체가 아니라 혼합). TTT(Test-Time Training), Titan 등은 메모리를 비선형·optimizer 쪽으로 확장하는 연구 흐름입니다.

> 한 줄 스토리: Softmax 병목 → Softmax 제거와 \(S\) → chunk-wise로 빠르게 → recency 위해 decay → 용량·간섭 위해 **delta update** → 실전은 하이브리드.

# 3부. 260716 오전 — Fairness와 Debiasing

> 오전에는 비전·테이블 데이터를 기준으로 **spurious correlation, Group DRO, Fairness 지표(DP·EO·EOpp)** 를 다루셨고, 오후 실습에서 COMPAS 등으로 연결됩니다. 반도체 현장에서도 “라벨과 상관이 있는 배경/조건” 때문에 비슷한 문제가 날 수 있다고 교수님께서 언급하셨습니다.

## 3-1. Spurious Correlation — 정답이 아닌 단서에 붙잡히다

모델이 높은 정확도를 내도, **진짜 인과 특징**이 아니라 **우연히 라벨과 상관된 단서(spurious feature)** 에 의존할 수 있습니다.

대표 예시로 자주 쓰이는 것이 **Waterbirds** 계열 설정입니다. “물새 vs 육지새”를 맞히는데, 학습 데이터에서는 물새가 물 배경, 육지새가 숲 배경에 **대부분** 붙어 있습니다. ERM(평균 리스크 최소화)은 overall accuracy만 보면 배경만 보고도 잘 맞힙니다. 문제는 **소수 그룹** — 물새가 숲에, 육지새가 물 위에 있는 경우 — 에서 성능이 무너진다는 것입니다.

현장 비유: 공정 조건·장비·조명 같은 것이 불량 라벨과 섞여 있으면, 모델이 “결함”이 아니라 “그 조건”을 외울 수 있습니다.

## 3-2. ERM vs Group DRO

**ERM(Empirical Risk Minimization)**  
전체 샘플의 평균 loss(보통 cross-entropy)만 줄입니다. 다수 그룹에 맞춰지면 overall accuracy는 높아도 **worst group**이 희생될 수 있습니다.

**Group DRO(Distributionally Robust Optimization)**  
그룹 \(g\)마다 loss \(L_g\)를 재고, 그중 **가장 나쁜 그룹**에 더 무게를 둡니다. 직관적으로는 매 배치에서 그룹별 loss를 보고, 어려운 그룹의 weight \(q_g\)를 키워 그쪽 gradient가 더 잘 흐르게 합니다.

결과 패턴(개념):

- Overall accuracy는 조금 떨어질 수 있음  
- **Worst-group accuracy**는 올라감  
- 그룹 간 격차가 줄어드는 경우가 많음  

그룹은 보통 **sensitive attribute × 라벨 \(Y\)** 조합으로 잡습니다. 예: 인종 2값 × 재범 여부 2값 → 4개 서브그룹.

## 3-3. Fairness 지표 세 가지 — DP, EOpp, EO

교수님이 **시험에 넣었다고 직접 언급**하신 핵심입니다. 이진 그룹 \(A\)와 이진 예측 \(\hat{Y}\), 정답 \(Y\)를 기준으로 합니다.

### Demographic Parity (DP)

\[
P(\hat{Y}=1 \mid A=0) = P(\hat{Y}=1 \mid A=1)
\]

**정답 \(Y\)를 보지 않고**, 그룹마다 “긍정 예측 비율”이 같아야 합니다.  
채용에서 “합격률이 성별과 무관”처럼 들리면 공정해 보이지만, **실제 자격(\(Y\)) 분포가 다를 때** 예측률을 억지로 같게 하면 **역차별** 논쟁이 생깁니다. 교수·조교 모두 이 한계를 강조했습니다.

### Equal Opportunity (EOpp)

\[
P(\hat{Y}=1 \mid Y=1, A=0) = P(\hat{Y}=1 \mid Y=1, A=1)
\]

**자격이 있는 사람들(\(Y=1\))** 안에서만, 그룹 간 기회(TPR)가 같아야 합니다. “진짜 좋은 사람을 놓치지 않는 비율”의 평등.

### Equalized Odds (EO)

\(Y=1\)뿐 아니라 **\(Y=0\)** 에서도 예측 분포가 맞아야 합니다. 즉 TPR과 FPR(또는 해당 조건부 확률)을 함께 맞추는, 더 강한 조건입니다.

| 지표 | 조건에 \(Y\) | 한 줄 의미 |
|------|-------------|------------|
| **DP** | 없음 | 그룹별 **긍정 예측률** 동일 |
| **EOpp** | \(Y=1\)만 | 참양성률(기회) 동일 |
| **EO** | \(Y=0,1\) 모두 | 참·거짓 양성 쪽 모두 맞춤 |

> ⭐ **시험 포인트 (교수 직접 출제 언급)**  
> **Fairness metric 정의** — DP / EOpp / EO를 **수식 또는 말로 정확히** 구분할 것.  
> DP는 \(Y\) 무시 → base rate가 다를 때 위험.  
> EOpp·EO는 \(Y\)를 조건에 넣어 “자격 있는 사람” 관점의 공정성.

## 3-4. COMPAS와 Fair DRO

**COMPAS**는 재범 예측 등 법·형사 맥락의 테이블 데이터로, fairness 연구에서 오래 쓰인 벤치마크입니다(ProPublica 계열 전처리 버전). 인종 등 sensitive attribute와 재범 라벨이 있어, ERM vs Group DRO를 비교하기 좋습니다.

**Fair DRO**는 Group DRO를 fairness notion 쪽으로 변형한 계열입니다. 단순히 worst group accuracy만이 아니라, 선택한 fairness 지표를 목표에 넣고, 클래스별 weighting 등이 들어갑니다. Accuracy와 fairness 사이 **trade-off 하이퍼파라미터**(\(\rho\) 등)로 파레토 곡선을 볼 수 있습니다.

중요한 메시지는 기술만이 아닙니다.

- 어떤 fairness notion을 쓸지는 **정책·가치 선택**에 가깝다  
- Group fairness와 individual fairness가 충돌할 수 있다  
- 학습 중 라벨·그룹 정보가 항상 있는 것도 아니다

# 4부. 260716 오후 — LLM Social Bias와 Fairness

> 오후 앞부분은 교수님 강의, 이후는 TA 실습입니다. 아래는 **교수님 강의 + 시험 안내** 중심입니다.

## 4-1. 가벼운 마음으로, 그러나 현실적인 문제

교수님은 “조금 가벼운 마음으로 들으셔도 된다”고 하시면서도, LLM 시대에 **소셜 바이어스·불공정**이 실제로 불거지고 있고, 아직 완전히 정립된 분야는 아니라고 하셨습니다. 오전 분류 fairness와 **같은 뿌리**, 다른 측정 방식입니다.

## 4-2. Representational Harm vs Allocational Harm

| 유형 | 의미 | 예 |
|------|------|-----|
| **Representational Harm** | 특정 소셜 그룹을 **잘못 표상** | 여성을 비하하는 표현, “I’m an autistic dad”에 “I’m sorry to hear that”처럼 부정적 뉘앙스 |
| **Allocational Harm** | sensitive attribute에 따라 **결정·자원 배분**이 달라짐 | 분류 fairness의 의사결정 차별과 유사 |

## 4-3. 편향은 어디서 스며드나

사람은 편향되어 있고, 그 사회를 반영한 데이터로 만든 LLM도 비슷한 문제를 물려받습니다.

- **Pretraining 데이터**의 majority viewpoint  
- **큐레이션·toxicity filter**가 minority 관련 텍스트를 과도하게 제거  
- **Annotation / SFT** 시 annotator 편향  
- **RLHF·DPO**의 preference 라벨에 사람 편향  
- **System prompt**에도 스며들 수 있음  

## 4-4. 분류 fairness vs LLM — 재는 방식이 다르다

오전처럼 이진 그룹·이진 라벨이면 **DP·EO·EOpp** 같은 수식 메트릭으로 “얼마나 위반했는지”를 잴 수 있습니다.  
LLM은 출력이 열려 있어서, 아직은 **벤치마크를 만들고 그 점수**로 보는 empirical 평가가 대부분입니다.

### BBQ (Bias Benchmark for QA)

대표적인 QA 편향 벤치마크입니다. 같은 상황에 두 가지 컨텍스트를 둡니다.

1. **Ambiguous**: 정보 부족 — 정답은 보통 **unknown**. 스테레오타입으로 찍으면 편향.  
2. **Disambiguated**: 추가 정보로 정답이 정해짐 — 스테레오와 **반대로** 답해야 하는 경우도 있음.

예: 폭탄 사건 후 Christian과 Muslim이 함께 도망친 장면에서, 모호할 때는 unknown이어야 하고, “Christian이 자백했다”는 정보가 있으면 그에 맞게 답해야 합니다. 무슬림=테러 같은 연상이 있으면 틀린 답이 나옵니다.

Age, Disability, Gender, Nationality, Race, Religion 등 축으로 질문하고, **accuracy**와 **bias score**(unknown 대신 편향 쪽으로 대답하는 비율 등)를 봅니다. OpenAI·Anthropic 등 frontier 모델의 **safety report / system card**에도 BBQ가 자주 실립니다.

### Discriminate Eval 계열

인구통계 속성만 바꾼 **counterfactual** 질문에서 high-stakes Yes/No가 어떻게 달라지는지 봅니다. 특정 속성이 유리/불리하게 작용하는지를 계수로 요약하는 식입니다(TA 실습에서도 연결).

## 4-5. LLM Debiasing — FairSteer와 프롬프트

학습 전체를 다시 돌리기 어려우니, **추론 시** 개입하는 방법도 연구됩니다.  
한 예로 **FairSteer** 계열: biased/unbiased 답의 activation을 구분해 linear probe로 탐지하고, unbiased 쪽으로 가는 **steering vector**를 activation에 더합니다. 실무에서는 **system prompt 엔지니어링**이 더 흔히 쓰이기도 합니다.

## 4-6. 텍스트–이미지 생성과 문화 편향

Text-to-Image에서도 “CEO → 백인 남성”, “간호사 → 여성”처럼 스테레오가 증폭됩니다. 한 속성만 50:50으로 맞춰도 **교차 그룹**에서 왜곡이 남을 수 있어, 사용자 정의 목표 분포와 생성 분포의 갭(NPR 등)으로 fairness를 재는 연구도 있습니다.

영어·서구 중심(WEIRD) pretrain 데이터는 **한국어·한국 문화** 답과 어긋날 수 있습니다. **KO-BBQ**처럼 BBQ를 한국 맥락으로 옮긴 벤치마크, multi-agent **debate**(+ self-reflection)로 문화 시나리오를 교정하려는 시도도 소개되었습니다. Self-reflection만으로는 효과가 약하고, debate를 섞으면 나아지는 실험 이야기가 나왔습니다.

강의 말미, 쉬는 시간에 조교에게 시험 관련 질문이 있었다는 말씀을 하시며 교수님이 **출제 범위를 직접** 정리해 주셨습니다. 거의 원문에 가깝게 옮깁니다.

> 그 다음에 **리니어 어텐션** — 어제 얘기한 리니어 어텐션의 **업데이트 룰** 관련한 것도 한 게 있고  
> 그리고 오늘 본 것 중에 **Fairness 매트릭과 관련된 그런 정의들**  
> 네. 그리고 또 하나 뭐였지?  
> … 그런 내용들을 시험 문제로 냈으니까 참고해 주시면 감사하겠습니다.”

# 시험 출제 포인트

교수님 260716 오후 안내와 이틀 강의 강조점을 바탕으로 정리했습니다.  
**설명형**으로 답하는 연습이 핵심입니다.

## 1일차 — SSL (260715 오전)

1. **SSL 정의**: self-generated supervision / pretext task  
2. **평가**: linear probing vs fine-tuning  
3. **Contrastive loss / InfoNCE** ⭐ 출제 언급  
   - positive·negative 정의  
   - loss minimize → pos 유사도↑, neg↓  
4. **SimCLR vs MoCo** (배치 vs queue+momentum) — 출제 가능성 중  
5. BYOL: negative 없음, predictor + stop-grad + EMA  
6. CLIP / AR이 SSL 패밀리인 이유 (개념)

## 1일차 — Linear Attention (260715 오후)

1. Softmax attention 병목: **\(O(L^2)\)**, KV cache  
2. Softmax 제거 + associativity → **\(S_t=\sum vk^\top\)**  
3. **Update rule** ⭐ 출제 언급  
   - Vanilla: \(S_t = S_{t-1} + v_t k_t^\top\), \(o_t = S_t q_t\)  
   - Decay: \(\gamma S_{t-1} + \cdots\)  
   - Delta: 에러 보정 write  
4. Chunk-wise parallel → \(O(LD^2)\)  
5. Recency bias 부재 → decay 필요  
6. Associative memory / capacity limit (\(D\)개 직교)

## 2일차 — Fairness (260716 오전)

1. Spurious correlation / worst group  
2. ERM vs **Group DRO** (worst-group 최적화)  
3. **DP / EOpp / EO 정의** ⭐ 출제 언급  
4. DP의 한계(역차별·base rate)  
5. COMPAS 등 민감 속성 × 라벨 서브그룹

## 2일차 — LLM Bias (260716 오후)

1. Representational vs Allocational harm  
2. BBQ: ambiguous(unknown) vs disambiguated  
3. 편향 유입 경로(pretrain·filter·RLHF 등)  
4. (보너스) FairSteer, T2I 스테레오, 문화 편향

## 체크리스트

- [ ] InfoNCE를 **말로** 설명하고, positive/negative 예를 들 수 있는가?  
- [ ] Linear attention의 \(S\) update를 한 줄로 쓸 수 있는가?  
- [ ] Full attention과 비교해 **무엇을 희생·무엇을 얻는지** 말할 수 있는가?  
- [ ] DP·EOpp·EO를 **\(Y\) 조건 유무**로 구분하는가?  
- [ ] Group DRO가 ERM과 무엇을 다르게 최적화하는가?  
- [ ] BBQ ambiguous에서 왜 정답이 unknown이어야 하는가?  
- [ ] **설명형** — 수식 암기보다 **아이디어·목적·차이**를 문장으로
