## 1. Macro vs Micro Average (오전 마지막 이어서)

### 1.1 왜 둘 다 보여주는가
- **Macro Average** : 클래스별 Precision/Recall을 먼저 구하고 단순 평균.
- **Micro Average** : 모든 클래스를 합친 pooled matrix에서 Precision/Recall 계산 → 인스턴스 단위.
- Macro는 **특정 클래스의 약점**(예: Urgent 한두 건만 있어 성능이 나빠도 전체에선 안 보임)을 드러내고, Micro는 전반적 성능을 본다.
- **두 개를 함께 보고함**으로써 클래스 편차 없이 잘하고 있는지 + 전체적으로 잘하고 있는지를 동시에 평가한다.

### 1.2 Bias와 Model Card
- 데이터 → 모델 → 새 데이터의 **자기 강화 루프**로 bias가 점점 커짐.
- 데이터셋·모델 릴리즈 시 **Model Card**, **Limitations 섹션** 명시 의무화 추세.
- toxicity 분류로 censorship 도구가 되면 **표현권 박탈**이 될 수 있음 → 민감.

## 2. 🔥 Extrinsic vs Intrinsic Evaluation **(시험 출제)**

### 2.1 Extrinsic Evaluation
- **외부 task의 성능**으로 LM을 평가.
- 예: GLUE, MMLU, sentiment analysis, spell correction.
- LM 자체가 아닌 **활용한 downstream task** 의 점수를 본다.
- 단점 : 평가에 며칠씩 걸림.

### 2.2 ⭐ Intrinsic Evaluation : Perplexity
- LM 자체로 평가 가능한 metric.
- **직관** : 좋은 LM은 진짜 등장할 만한 문장에 **높은 확률**, 엉뚱한 문장에 **낮은 확률**을 줘야 한다.
- "I'm perplexed" = "어떤 단어를 골라야 할지 모르겠다" → Perplexity가 낮을수록 LM이 분명한 결정을 내릴 수 있음.

### 2.3 🔥 Perplexity 정의 **(시험 출제)**
```
PP(W) = P(w_1, w_2, ..., w_N)^(-1/N)
      = exp( -1/N · Σ log P(w_i | context) )
```
- **Cross-entropy의 exp** 형태.
- **Log probability의 평균에 부호 뒤집고 exp 취함**.

### 2.4 Random Digit 예제로 해석
- 0~9 균등 확률 (각 1/10)로 N개 뽑은 문장 :
  ```
  P = (1/10)^N → PP = ((1/10)^N)^(-1/N) = 10
  ```
- **Perplexity 10** = "다음 단어 예측 시 평균 10갈래에서 헷갈림."
- Unigram → Bigram → Trigram → LLM 으로 갈수록 PP가 줄어든다.

### 2.5 Perplexity 구현 (실습)
```python
log_prob = sum of log P(w_i | w_{i-1})  for all bigrams
perplexity = exp( -log_prob / m )       # m = 토큰 개수
```

## 3. 🔥 Backoff & Interpolation **(중요)**

### 3.1 동기
- Trigram이 더 정확할 수 있지만 **sparse 해서 관찰이 0인 경우** 가 많음.
- "맛집 리뷰" 비유 : 별 4개 1명짜리(트라이그램) vs 별 4개 4500명짜리(유니그램) — 두 번째가 더 신뢰도 있음.

### 3.2 Backoff
- 상위 모델(예: trigram)에서 카운트 0이면 **하위 모델(bigram, unigram)로 fallback**.

### 3.3 Interpolation
- 하위/상위 모델의 **weighted combination**:
```
P_interp(w | w_{-2}, w_{-1}) = λ_3 · P_3 + λ_2 · P_2 + λ_1 · P_1
```
- 조건 : λ들의 합 = 1.
- λ는 **모의고사(validation set) 로 튜닝**.
- 일반화된 표기 : λ가 직전 단어에 따라 case-by-case로 다를 수도 있다.

### 3.4 🔥 Interpolation의 재미있는 해석
- Bigram + Unigram의 interpolation은 **노이즈가 낀 train data로 학습된 bigram**과 같다.
- 절차 : 원본 train data의 일부 토큰을 **unigram 분포로 랜덤 샘플링한 토큰**으로 바꿔치기 → 그 noisy data로 bigram 학습.
- 노이즈 비율이 unigram의 가중치에 대응.
- 노이즈 100% → unigram만, 노이즈 0% → 원본 bigram.

## 4. OOV (Out-Of-Vocabulary) 처리

### 4.1 신조어 문제
- Train data에 없던 단어 = **OOV**.
- 그대로면 확률 0이 되거나 KeyError.

### 4.2 `<UNK>` 토큰 기법
1. **Train 시점** : 빈도가 낮은 단어 몇 개를 미리 `<UNK>` 토큰으로 치환해 학습.
2. **Inference 시점** : 처음 보는 단어를 `<UNK>` 로 치환 → 학습된 확률을 그대로 활용.

## 6. Naive Bayes Classifier 실습 (IMDB Movie Review)

### 6.1 데이터셋
- IMDB movie review : positive / negative 라벨링된 영화 리뷰.
- Regular expression으로 전처리.

### 6.2 Software 1.0 방식 : Lexicon-based
```python
def classify(review, pos_words, neg_words):
    score = sum(1 for w in review if w in pos_words) \
          - sum(1 for w in review if w in neg_words)
    return "positive" if score > 0 else "negative"
```
- pos_words, neg_words를 **set**으로 두면 멤버십 테스트가 O(1).
- 결과 : Accuracy 약 **66%**.
- 클래스별 accuracy로 보면 negative를 negative로 맞히는 게 positive보다 더 잘됨.

### 6.3 Negation Handling (Heuristic)
- "not"을 만나면 **다음 punctuation까지** 의 단어 앞에 `NOT_` prefix 추가.
- 새로운 토큰을 invent하여 부정 효과를 표현.
- 구현 :
  ```python
  result = []
  negate = False
  for word in tokens:
      if word in NEGATION_WORDS:
          negate = True
          result.append(word)
      elif word in PUNCTUATION:
          negate = False
          result.append(word)
      elif negate:
          result.append("NOT_" + word)
      else:
          result.append(word)
  ```

## 7. 🔥 Bayesian 추론 정리 : MLE / MAP / Full Bayesian **(시험 출제)**

### 7.1 캔디 공장 예시
- 공장이 고정 비율 P로 A맛, (1-P)로 B맛 사탕을 만든다.
- 사 먹으며 P를 추정.

### 7.2 세 가지 추정 방식 ⭐

| 추정법 | 출력 | Prior 사용 | 직관 |
|--------|------|------------|------|
| **MLE** (Maximum Likelihood) | 점추정 | 사용 X | 데이터만 보고 likelihood 최대화 |
| **MAP** (Maximum A Posteriori) | 점추정 | 사용 O | Posterior 최대화 = prior × likelihood |
| **Full Bayesian** | **분포** | 사용 O | Posterior 분포 자체를 유지·업데이트 |

### 7.3 사탕 예시 수치
- 10개 사 먹어 A 6개, B 4개 → MLE는 6/10 = 0.6.
- 만약 prior가 "70%일 것 같다 = (A 7개, B 3개와 동등)" 이고 데이터가 "A 3개, B 7개" 면 MAP = (3+7)/(10+10) = 0.5.
- 데이터가 많아지면 prior 영향력이 점점 줄어 **MLE에 수렴**.

### 7.4 핵심 메시지
- **LLM의 pre-training은 본질적으로 MLE**.
- **MAP는 prior가 있는 경우의 MLE**.
- **Full Bayesian** 은 점추정이 아니라 **분포**를 유지 → 더 많은 정보, 더 무거운 계산.

## 8. Naive Bayes 실습 - 구현 디테일

### 8.1 Prior 계산
```python
def calculate_priors(labels):
    priors = {}
    counter = Counter(labels)
    total = len(labels)
    for c, count in counter.items():
        priors[c] = count / total
    return priors
```

### 8.2 Document Likelihood (Word-level with Smoothing)
- 각 클래스에 대해 분리된 word_counts dictionary.
```python
def word_likelihood(word, c, word_counts, total_words, vocab_size):
    return (word_counts[c].get(word, 0) + 1) / (total_words[c] + vocab_size)
```

### 8.3 최종 Score 계산 (Log Space)
```python
def predict(review):
    scores = {}
    for c in priors:
        log_score = math.log(priors[c])
        for word in review:
            log_score += np.log(word_likelihood(word, c, ...))
        scores[c] = log_score
    return max(scores, key=scores.get)   # argmax
```
- **Underflow 방지** 위해 log 사용. 곱셈 → 덧셈.
- 비교만 하면 되므로 normalize 불필요.

### 8.4 결과 해석
- "good"이라는 단어는 의외로 **변별력 약함** : "not good" 형태로도 자주 등장.
- "bad"는 negative의 강력한 marker.
- 영화 리뷰 코퍼스 특성상 "movie", "film" 같은 단어는 어느 클래스에서나 자주 나와 정보가 거의 없음 → 거의 stop word.

## 10. 🔥 Dense Embedding으로 가는 동기

### 10.1 Count-based LM의 한계
1. **Long-distance dependency** 못 잡음 (윈도우 밖 단어 무시).
2. **Synonym 처리 불가** : "buy"는 많이 봤지만 "purchase"는 적게 봤다면, 좋은 LM이라면 거의 같은 의미니 동등하게 활용해야 하는데 sparse count는 그게 안 됨.
3. **Smoothing의 한계** : 본 적 없는 단어들에 동일한 확률을 줄 뿐, 비슷한 환경의 단어는 더 크게 키워줘야 함.

### 10.2 Sparse vs Dense Representation
| 항목 | Sparse | Dense |
|------|--------|-------|
| 표기 | One-hot, Bag-of-Words, N-gram | Word2Vec, Embedding |
| 차원 | V (vocabulary size) ~ 수만 | 수백 차원 (예: 300) |
| 0 비율 | 99.9% | 거의 없음 |
| 의미 표현 | 같은 단어인지만 표시 | **유사한 단어 = 유사한 벡터** |

## 11. CNN, RNN과 Classification

### 11.1 Bag-of-Words 한계
- 단어를 lexicon-based 5차원 벡터로 표현 (positive, neutral, negative 등) → 단순 합산 → softmax → 예측.
- 단점 : "not love" 같은 인접어 의미를 잡지 못함.

### 11.2 두 가지 발전 방향
1. **네트워크 자체를 복잡하게** : CNN(인접어 표현), RNN(이전 히스토리 요약), LSTM/GRU.
2. **단어 벡터 자체를 학습 대상으로** : 피처 엔지니어링(렉시콘 디자인) 포기, 벡터를 모델 파라미터의 일부로 둠.
   - 학습 후 부산물로 **task-specific representation** 획득.
   - 비슷한 의미 단어는 비슷한 벡터로 모이게 됨.

### 11.3 CBOW (Continuous Bag of Words)
- Sparse BoW의 dense 버전.
- 단어를 dense vector로 변환하고 더해서 예측.
- 차원이 V → 수백 차원으로 줄어 dense.
- **각 차원의 의미는 명시적으로 해석 불가** (블랙박스).

## 12. 🔥 Word2Vec & Distributional Semantics **(중요)**

### 12.1 LM 학습이 만들어내는 부산물 = Embedding
- LM은 빈칸 채우기(self-supervised) task.
- "I gave a ___ at the conference" → lecture, talk, presentation, gift...
- 빈칸을 잘 맞히려면 **비슷한 자리에 등장하는 단어들이 비슷한 벡터를 가져야** 한다.

### 12.2 Distributional Semantics
> **"비슷한 컨텍스트에서 등장하는 두 단어는 동의어다."**

- LM 학습은 자동으로 이 정의에 맞는 representation을 만들어 낸다.
- 의도하지 않았지만 **의미 보존 embedding**이 부산물로 얻어짐.

### 12.3 Word2Vec의 두 변형
- **CBOW** : 주변 단어 → 중앙 단어 예측.
- **Skip-gram** : 중앙 단어 → 주변 단어 예측.
- 어느 방향이든 distributional semantics를 따름.

### 12.4 🔥 Word2Vec의 유명한 Vector 산술
```
v(king) - v(man) + v(woman) ≈ v(queen)
v(uncle) ↔ v(aunt) (성별 차이 벡터)
v(eagle) - v(연대) + v(고대) ≈ v(tiger)
v(영국 신사) ↔ v(한국의 이미지) (국가 매핑)
```

### 12.5 비유 : MBTI = Representation
- 4비트 벡터 (E/I, N/S, T/F, J/P).
- 같은 MBTI = 비슷한 벡터 → 비슷한 사람으로 느낌.
- 사람들이 MBTI 재밌어하는 이유 = representation의 직관성.

### 12.6 Softmax (전체 vocab 분류)
- LM = vocab 크기만큼의 클래스 수를 가진 거대한 classification.
- Softmax로 score → 확률분포 변환.

## 13. Transfer Learning과 Fine-tuning

### 13.1 LM Embedding이 좋은 시작점인 이유
- LM이 **distributional semantics에 따라 의미가 보존된** embedding을 학습.
- like, love가 비슷한 위치 / hate, dislike가 비슷한 위치.
- Sentiment classification 같은 task에서 **이미 좋은 시작점**.

### 13.2 Fine-tuning
- LM에서 학습된 embedding을 시작점으로 잡고, downstream task(sentiment 등)에 맞춰 추가 학습.
- 무작위 초기화보다 적은 데이터로 **훨씬 좋은 성능**.
- 이게 LM이 NLP 전반에서 각광받는 이유.

## 14. Time Delay Neural Network (TDNN) ≈ CNN for Text

### 14.1 NLP의 콘볼루션
- 인접한 두 단어를 합쳐 새 벡터로 만듦.
- 마치 bigram이지만 **soft bigram** : 단순 concatenation이 아닌 학습된 결합.
- 효과: "not"+"hate"가 invented 새 의미 단어처럼 표현됨.

### 14.2 RNN (Recurrent)
- 이전까지의 모든 컨텍스트를 **요약 벡터(hidden state)** 에 누적.
- **Long distance dependency** 표현 가능.
- 단점 :
  - 너무 많은 정보를 저장하려다 "까먹는 것이 더 나은" 정보까지 보유 → LSTM/GRU의 gating 기법으로 보완.
  - 순차 의존성 → **병렬 계산 어려움** → 나중에 Transformer가 해결.

## 15. Dense Embedding 실습

### 15.1 학습 방법 (개념적 옵션들)
1. **유사도 기반 학습** : query-document relevant 페어를 가까이, irrelevant 페어를 멀게 (search/IR).
2. **Autoencoder** : input → bottleneck → reconstruct로 압축된 representation 학습.
3. **LM Pretraining (Word2Vec 등)** : 빈칸 채우기.

### 15.2 One-hot Encoding (Sparse)
```python
def one_hot(word, word2idx, vocab_size):
    vec = np.zeros(vocab_size)
    vec[word2idx[word]] = 1
    return vec
```
- 모든 단어 쌍의 dot product = 0 (자기 자신 외에는 닿지 않음).
- 단어 간 유사도 측정 불가 → 동의어를 알 수 없음.

### 15.3 Bag-of-Words (Document Vector)
```python
def bow(document, word2idx, vocab_size):
    vec = np.zeros(vocab_size)
    for word in document:
        if word in word2idx:
            vec[word2idx[word]] += 1
    return vec
```
- 또는 단어들을 one-hot 표현 후 그냥 sum 해도 같음.
- 문서 간에는 단어 겹침이 있어 cosine similarity가 0이 아닌 값을 줄 수 있음.

### 15.4 🔥 Cosine Similarity
```
cos_sim(u, v) = (u · v) / (||u|| · ||v||)
```
- 벡터 길이의 영향을 제거하고 **각도(방향)** 만 비교.
- Unit norm vector를 학습하는 경우 Euclidean distance와 cosine similarity가 거의 같아짐.

### 15.5 Sparse 표현의 한계
- 문서 간 유사도는 어느 정도 가능하지만, **단어 간 의미 유사도**는 잡지 못함.
- 같은 토픽의 다른 단어들("bicycle" vs "bike")이 등장한 두 문서를 유사하다고 판단할 수 없음.
- 이를 해결하기 위해 **Dense Embedding** 이 필요.

## 16. 한 페이지 요약 (핵심 정리)

### 16.1 Perplexity & 평가
1. **Extrinsic** : downstream task로 LM 평가 (MMLU, GLUE).
2. **Intrinsic = Perplexity** : `exp( -mean(log P) )`. 낮을수록 좋다.
3. PP = K → "K갈래에서 헷갈림" 직관.

### 16.2 N-gram 보완
1. **Backoff** : 상위 → 하위 fallback.
2. **Interpolation** : λ-weighted combination, λ_sum = 1.
3. **Interpolation = noisy data로 학습된 bigram** 의 베이지안 해석.
4. **OOV** 처리 : `<UNK>` 토큰.
5. **압축** : Quantization, Trie, Huffman, Threshold pruning.

### 16.3 🔥 Bayesian 추정 (시험)
- **MLE** : prior 무시, likelihood 최대화 (= LLM pre-training).
- **MAP** : posterior = prior × likelihood, 점추정.
- **Bayesian** : 분포를 유지하며 데이터마다 업데이트.
- 데이터 ↑ → MLE로 수렴.

### 16.4 Naive Bayes 실습 핵심
- **Lexicon-based (1.0)** ~66% accuracy.
- **Negation handling** : `NOT_` prefix.
- **Naive Bayes (2.0)** with Laplace smoothing → 더 나은 성능.
- Log-space 계산으로 underflow 방지.

### 16.5 🔥 Sparse → Dense
- Sparse(one-hot, BoW)는 단어 의미 유사도 표현 불가.
- **Distributional Semantics** : 비슷한 컨텍스트의 단어 = 비슷한 의미.
- **Word2Vec** (CBOW / Skip-gram) : LM pretraining의 부산물로 의미 보존 embedding.
- 산술 가능 : king - man + woman ≈ queen.

### 16.6 신경망 NLP 모델 비교
| 모델 | 강점 | 약점 |
|------|------|------|
| **BoW** | 간단, 빠름 | 순서 무시, 의미 못 잡음 |
| **CNN** | 인접어 결합 (n-gram-like) | Long distance 못 잡음 |
| **RNN** | Long distance 가능 | 순차 의존성, 병렬 X |
| **LSTM/GRU** | 까먹기 게이팅 | 여전히 순차적 |

### 16.7 Transfer Learning
- LM pretraining → fine-tuning 패러다임.
- **이미 의미가 보존된 representation**에서 출발하므로 적은 데이터로 좋은 성능.

### 16.8 강의 중 주요 직관/비유
- **Perplexity = "I'm perplexed"** : 어디로 갈지 모르겠다 → 낮을수록 좋다.
- **맛집 리뷰 별 4개** : 4500명짜리(unigram) vs 1명짜리(trigram) — 신뢰도와 정확도의 trade-off.
- **사탕 공장 P** : MLE / MAP / Bayesian의 차이.
- **MBTI = representation** : 비슷한 4비트 벡터 = 비슷한 사람.
- **연대-독수리, 고대-호랑이** : Word2Vec 벡터 산술의 매직.
- **사은품 vs 본상품** : LM 학습의 진짜 가치는 빈칸 예측이 아니라 부산물 embedding이다.
- **king - man + woman ≈ queen** : Distributional semantics의 시각적 증명.
- **배터리가 방전** : 강사가 수업 후 파티 안 갈 거라고 예측 — representation의 활용 비유.
