## 1. BPE 토크나이저 마무리 (실습 리캡)

### 1.1 BPE의 본질
- **모든 글자 단위로 분절**한 뒤, **bottom-up 방식**으로 누구와 누구를 합칠지 정하는 것이 BPE.
- 같은 bottom-up이라도 어떤 페어를 합칠지 결정하는 방식은 다양하다.
  - **BPE** : 단순히 **frequency**가 높은 페어를 합침.
  - **WordPiece** : 합치기 전 vs 합친 후의 **데이터 likelihood**가 최대화되도록 페어 선정.

### 1.2 BPE 토큰화 수행 (apply 단계)
- 입력 문자열을 **소문자 변환 → 글자 단위 리스트화 → end-of-word 마커(`_`) 추가**.
- 학습된 **merge rule들을 순서대로** 적용해 최종 토큰화 결과를 deterministic하게 얻음.
- 실제 라이브러리는 list concatenation을 매번 새로 만들지 않고 더 효율적으로 처리.

## 2. N-gram 언어모델 복습 및 Bigram 도입

### 2.1 N-gram의 핵심 아이디어
- 이전 강의에서 다룬 초저질 **Unigram**은 토큰별 frequency만 세고 독립적으로 주사위를 굴려 샘플링.
- **Markov Assumption**으로 컨텍스트 길이를 단축 :
  - **Bigram** : 직전 1단어를 컨텍스트로
  - **Trigram** : 직전 2단어를 컨텍스트로
  - 4-gram, 5-gram, 10-gram도 가능

### 2.2 N-gram의 구조적 한계 : Long Distance Dependency
- "crash"가 생성될 확률이 높은 이유는 컨텍스트 어딘가에 **"computer"** 가 있기 때문인데, 컨텍스트가 2g(1단어), 3g(2단어)로 좁으면 "computer"가 컨텍스트 윈도우 밖으로 빠질 수 있음.
- 반대로 컨텍스트를 너무 길게 보면 "5층에서 crash"같은 희귀 컨텍스트가 발생해 sparse 문제가 심해짐.

### 2.3 Bigram 확률 계산
```
P(the | dat) = Count(dat the) / Count(dat)
```
- 이 조건부 확률을 모든 단어 페어에 대해 구하면, 1차원 unigram 표가 **2차원 bigram 테이블**로 확장됨.

## 3. 🔥 Sparsity 와 Underflow / Log Probability **(시험 출제)**

### 3.1 Sparsity (스팔스)
- N-gram 테이블의 **대부분 entry가 0**.
- Shakespeare corpus(약 3만 vocabulary)로 bigram 표를 만들면 **약 99.96%가 0**.
- 트라이그램, 4-gram이면 더 심해짐.
- **0의 두 가지 의미** :
  1. **DVD 저장 공간 낭비**
  2. **Overfitting** (더 큰 문제)

### 3.2 Joint Probability와 Underflow
조인트 확률을 chain rule로 분해 :
```
P(I love u) = P(I) · P(love|I) · P(u|I,love)
```
Markov assumption(bigram) 적용 시 :
```
P(I love u) ≈ P(I) · P(love|I) · P(u|love)
```
모든 확률은 1보다 작으므로 곱하면 점점 0에 수렴 → **Underflow**.

### 3.3 ⭐ 시험 출제 포인트 : Log Probability
- 곱셈을 덧셈으로 바꾸기 위해 **log를 씌움**.
- **랭크는 보존**되므로 절대 확률값이 필요 없는 한 문제없음 (instructor 비유: "중간고사 등수를 log 기준으로 매기겠다 해도 1등은 그대로 1등").
- **두 가지 장점** :
  1. 작아지는 숫자 문제(underflow) 해결
  2. **곱셈 → 덧셈** (많은 기계에서 더 빠른 연산)

## 4. Maximum Likelihood Estimation (MLE)

### 4.1 MLE의 직관
- 토큰 A, B, C가 각각 3, 2, 1번 등장했다면 → 확률 1/2, 1/3, 1/6.
- 즉 **카운트를 토탈수로 나눈 값**이 추정치 = MLE.
- 합이 1이 되는 것은 자명.

### 4.2 왜 이렇게 추정하면 좋은가
- 관측 데이터의 likelihood (P=1/2, P=1/3, P=1/6 일 때 실제로 이 관측 데이터가 나올 확률)를 계산해보면, **다른 어떤 추정값을 써도 항상 작아진다**.
- Lagrange multiplier로 미분하면 직접 증명 가능.
- LLM (ChatGPT 등) **pre-training도 결국 MLE 학습** 이라는 것을 기억하자.

## 5. 🔥 Smoothing **(시험에 나올 수 있음 - 핵심 기법)**

### 5.1 Overfitting과 0 확률 문제
- "denied the claim", "denied the report", "denied the allegations"는 봤지만 "denied the offer"는 한 번도 못 봤다고 가정.
- 테스트셋에 "denied the offer"가 나오면 → **확률 0** 또는 **0으로 나누기 에러**.
- Instructor 비유 : "꼰대" — 자기 경험에만 매몰돼 본 적 없는 일은 일어날 수 없다고 단정.

### 5.2 Laplace Smoothing (Add-1 Smoothing) ⭐ **(시험 출제)**
- 모든 카운트에 **+1** 을 더한다 ("benefit of the doubt").
- 분모 노멀라이즈를 위해 **vocabulary size V** 도 더해야 함.

```
P_Laplace(w_i | w_{i-1}) = (Count(w_{i-1}, w_i) + 1) / (Count(w_{i-1}) + V)
```

### 5.3 Add-K Smoothing
- 1 대신 일반화된 K 사용.
- "왜 항상 1인가?"라는 reviewer 2 공격을 피하려면 **다양한 K를 empirically tuning** 했다고 보고.

```
P_AddK(w_i | w_{i-1}) = (Count(w_{i-1}, w_i) + K) / (Count(w_{i-1}) + K·V)
```

- K < 1 : 0이었던 애들에게 덜 나눠줌, 원래 큰 값들이 보존.
- K > 1 : 더 많이 균등화.

### 5.4 ABC 토큰 예시
- 기존 카운트 4, 2, 0 → 원래 확률 2/3, 1/3, 0
- +1 후 5, 3, 1, 합 9 → 5/9, 3/9, 1/9
- 9가 된 이유 : 원래 6에 V=3 더해진 것.

## 6. 🔥 Noisy Channel Model 기반 Spell Correction **(중요)**

### 6.1 핵심 아이디어 : 언어모델은 만능 사골국
- 슬로건 : **"세상만사 LM이 있으면 NLP 문제는 다 쉬워지고, LM이 좋을수록 downstream task가 더 좋아진다."**
- LM은 corpus에서 끓여진 사골국 같은 수학적 정보. 동일한 사골국으로 spell correction, classification, translation 등 다양한 NLP를 풀 수 있다.

### 6.2 Peter Norvig의 비행기 코드
- 구글 임원이자 AI 교과서 저자 Peter Norvig이 비행기 안(WiFi 없음, dictionary 없음)에서 작성한 ~20줄의 spell corrector 코드.
- 기내에서 가진 것 : `big.txt` corpus 하나.
- 그 corpus에서 unigram 언어모델 + Edit Distance로 후보 추출.

### 6.3 Bayes Rule 적용
관찰 가능 : **noisy word x** (예: graff)
알고 싶음 : **correct word w** (예: graph)

```
ŵ = argmax_w P(w | x)
  = argmax_w P(x | w) · P(w) / P(x)
  = argmax_w P(x | w) · P(w)        ← P(x)는 모든 후보에 동일하므로 제거
                ↑          ↑
             noise model   언어모델 P(w)
```

- **언어모델은 "정방향" 정보** (P(w))는 풍부하지만 역방향(P(w|x))은 어려움 → 베이즈로 뒤집기.
- 비유 : "1등에게 금 100돈만 줄 거라 분포가 1로 합산되지 않아도 등수만 보존되면 됨."

### 6.4 Noisy Word 후보 추출
- `Edit_1(x)` : x로부터 **edit distance 1** 만큼 떨어진 모든 단어 중 corpus에 있는 단어들.
- 4가지 edit operation : insertion, deletion, substitution, **transposition** (Damerau-Levenshtein).
  - **Transposition** : "SA → AS"처럼 두 글자를 바꿔 친 실수.

### 6.5 Likelihood P(x|w) 근사
- 정밀하게 구하려면 어떤 사람이 어떤 키 실수를 하는지 통계가 필요 (Wi-Fi 있으면 가능).
- 노빅의 비행기 버전 : **edit distance가 같으면 likelihood도 같다** 고 가정. 매우 단순.

### 6.6 예제 : "acress"
- 8개의 후보 중 **"across"** 가 1등 (P_LM 높음, edit distance 1).
- 그러나 만약 LM이 **bigram** 이라면 컨텍스트 "versatile ___" 에서 "actress"가 더 어울림.
- 슬로건 재확인 : **LM 업그레이드 → Spell Correction 성능 자동 향상**.

### 6.7 Real-word vs Non-word Errors
- **Non-word error** : 사전에 없는 단어 만듦 (graff). detect는 사전 비교만으로 가능. **쉽다.**
- **Real-word error** : 사전에 있지만 의미가 다른 단어 (minute → minuet, piece → peace).
  - 모든 단어를 의심 후보에 올림.
  - "two of two" → "two of the" (LM 확률 비교).
  - 한 문장에 여러 개 동시 오타 가정은 거의 없으므로 **하나만 틀린다** 고 pruning.

## 7. N-gram 실습 핵심 코드 패턴

### 7.1 Counter / DefaultDict
```python
from collections import Counter, defaultdict
counter = Counter(tokens)        # 한 줄로 카운팅
top_k = counter.most_common(k)   # 상위 k개

dd = defaultdict(int)             # 0으로 초기화 자동
dd['key'] += 1                   # KeyError 없이
```

### 7.2 Unigram 확률
```python
N = sum(unigram_count.values())
unigram_prob = {w: c/N for w, c in unigram_count.items()}
```

### 7.3 Bigram 카운트
- 키를 **튜플 (w_{i-1}, w_i)** 또는 **nested dict**로 저장.
```python
bigrams = [(tokens[i], tokens[i+1]) for i in range(len(tokens)-1)]
bigram_count = Counter(bigrams)
```

### 7.4 Bigram 조건부 확률 (MLE)
```python
P(w_i | w_{i-1}) = bigram_count[(w_{i-1}, w_i)] / unigram_count[w_{i-1}]
```

### 7.5 문장 확률 (Bigram)
```python
prob = 1.0
for w1, w2 in zip(tokens[:-1], tokens[1:]):
    prob *= bigram_prob.get((w1, w2), 0)
```

### 7.6 Log Probability
```python
import math
logprob = 0.0
for w1, w2 in zip(...):
    p = bigram_prob.get((w1,w2), 0)
    if p > 0:
        logprob += math.log(p)
    else:
        logprob = -math.inf      # 0의 log는 -∞
```

### 7.7 Add-1 Smoothed Bigram
```python
P_smooth = (bigram_count[(w1, w2)] + 1) / (unigram_count[w1] + V)
```

### 7.8 BOS/EOS 토큰의 의미
- **확률 정규화** : BOS/EOS 없으면 모든 문장 확률의 합이 1이 안 됨.
- **디코딩 종료 시점** 결정.

## 8. Naive Bayes Classification

### 8.1 분류 문제의 종류
| 문제 | 클래스 |
|------|--------|
| Sentiment Analysis | positive / negative |
| Spam Detection | spam / ham |
| Gender Classification | male / female |
| Star Rating | 1~5 stars |
| **Language Identification** | 세상의 모든 언어 |

### 8.2 Bag of Words (BoW)
- 단어들을 **종이봉투에 담아 흔들어** 순서를 무시한다는 의미.
- 리뷰 분석에선 형용사·부사·감정 동사만 살리고 나머지는 버리기도 함.
- 즉 **유니그램 언어모델**처럼 다룸.

### 8.3 Naive Bayes의 베이지안 분해
```
P(c | d) ∝ P(c) · P(d | c)         (분모 P(d) 제거 - argmax용)
        = P(c) · ∏_i P(w_i | c)    (BoW + conditional independence)
```
- **Class별로 polder의 메가 다큐멘트가 있고, 각 폴더의 unigram 언어모델을 만들었다고 생각**.
- Conditional Independence : **클래스 c 하에서 각 단어의 빈도는 독립**.

### 8.4 🔥 Naive Bayes의 강력한 장점
- 모든 항이 확률이라서 **비-텍스트 피처도 그냥 곱해서 합칠 수 있음**.
  - 예: "이메일에 이미지가 있을 확률", "보낸 사람이 지도교수일 확률" 등을 자유롭게 추가.
- 다른 ML 모델은 이런 임의 결합이 공격받지만, **Naive Bayes는 확률 연산이라 자연스럽게 합쳐짐**.

### 8.5 파라미터 추정 (with Laplace Smoothing)
```
P(c) = N_c / N                                  (prior)
P(w | c) = (Count(w, c) + 1) / (Σ_w' Count(w', c) + V)
```

### 8.6 예제 : Restaurant Type Classification
- 트레이닝 데이터 : Chinese 3개 ("Chinese Beijing Chinese", ...), Japan 1개 ("Tokyo Japan Chinese").
- 테스트 : "Chinese Chinese Chinese Tokyo Japan"
- Prior : P(Chinese) = 3/4, P(Japan) = 1/4
- Likelihood (smoothing 후) :
  - Chinese 폴더에 "Chinese" : (5+1)/(8+6) = 6/14 = 3/7
  - Chinese 폴더에 "Tokyo", "Japan" : (0+1)/14 = 1/14
- P(Chinese | doc) = (3/4) · (3/7)³ · (1/14) · (1/14)
- P(Japan | doc) = (1/4) · (2/9)⁵
- 비교 → **간소한 차이로 Chinese 승**.

## 9. Naive Bayes 개선 기법

### 9.1 Stop Words
- 모든 문서에 흔히 나오는 단어 (the, a, is...) 제거.
- 1등 결정에는 영향 없고 연산만 줄여줌.
- 라이브러리에서 언어별로 제공.

### 9.2 Binarized Multinomial Naive Bayes (Diminishing Returns)
- 한 단어가 한 문서에서 여러 번 나와도 **중요도는 1번 본 효과 정도**라고 본다.
- "햄버거 4개 먹기 비유" : 첫 햄버거는 만족 크지만 두 번째부터 체감↓.
- **Diminishing Return 표현 방법** :
  - log를 씌우거나
  - 1과 0의 binary로 clip (한 문서 내에서 occurrence 수를 1로 cap)
- 주의 : **단일 문서 내 카운트만 클리핑**, 전체 코퍼스 카운트는 여전히 합산.

### 9.3 Negation Handling
- BoW의 약점 : "not like" → "not"과 "like" 따로 다뤄지면 의미 잃음.
- **꼼수** : "not"을 만나면 그 뒤의 영향력 범위 안의 단어에 **NOT_ prefix** 를 붙여 새 토큰으로 만든다.
  - 예: "I don't like this fun movie" → "I don't NOT_like NOT_this NOT_fun NOT_movie"
- 보통 punctuation까지 영향력 범위.
- CNN/RNN은 모델 자체에서 이런 표현이 학습되므로 굳이 필요 없음.

## 10. Software 1.0 / 2.0 / 3.0 Paradigm (Andrej Karpathy)

| 패러다임 | 방식 | NLP 예 |
|----------|------|--------|
| **Software 1.0** | 룰·렉시콘 기반 | positive/negative lexicon으로 점수 합산 |
| **Software 2.0** | 데이터로 모델 학습 | CNN/RNN/Naive Bayes로 sentiment classification |
| **Software 3.0** | LLM에 프롬프트 | "Classify this review as positive or negative: ..." |

### 10.1 어떤 걸 쓸까
- **Lexicon이 풍부하고 잘 알려진 task** → Software 1.0이 즉시 가능, 트레이닝 데이터 불필요.
- **Labeled examples가 많음** → Software 2.0이 자연스러움.
- **GPT 노출 많은 분포 / 쉬운 task** → Software 3.0 (zero-shot or few-shot).

### 10.2 Few-shot Prompting 예시
```
Task: Classify the review as Positive or Negative.
Review: "This movie is amazing." → Positive
Review: "I hated every minute." → Negative
Review: "Boring and predictable." → ?
```

## 11. 🔥 Evaluation Metrics **(시험 출제)**

### 11.1 Confusion Matrix (Binary)
|  | Predicted + | Predicted - |
|--|-------------|-------------|
| **Actual +** | TP | FN |
| **Actual -** | FP | TN |

### 11.2 정의 ⭐
| 지표 | 공식 |
|------|------|
| **Precision** | TP / (TP + FP) |
| **Recall** | TP / (TP + FN) |
| **Accuracy** | (TP + TN) / All |
| **F1** | 2·P·R / (P + R) (harmonic mean) |

### 11.3 🔥 Accuracy의 함정 : Class Imbalance
- 듣보잡 가게 "DeliciousPie"의 트윗 100만 개 중 100개만 관련.
- 직원이 항상 "negative"라고 찍으면 **Accuracy = 99.99%** 가 나오지만 **Recall = 0**.
- **Class imbalance가 심하면 Accuracy 보지 마라.**
- 의료 진단(희귀병)도 같은 문제.

### 11.4 F1 = Harmonic Mean
- 두 값이 차이가 크면 **떨어지는 쪽에 더 가깝게 붙음** → 두 지표 모두 높게 만들도록 강제.

### 11.5 Cross-Validation
- 트레이닝 데이터 일부를 모의고사로 떼어놓음 (validation set).
- 떼어놓는 부분을 바꿔가며 평균 → **k-fold cross-validation** (보통 10-fold).

### 11.6 Multi-class Precision/Recall
**3-class 예시 (Urgent / Normal / Spam)** :
- 클래스마다 그것을 positive로 보는 **2x2 matrix를 만들 수 있음**.
- 예: Urgent의 Precision = 8 / (8+10+1) = 8/19
- 각 클래스별로 Precision, Recall, F1 계산 가능.

### 11.7 Macro vs Micro Average ⭐ **(시험 출제)**
| 평균 방식 | 계산 |
|-----------|------|
| **Macro** | 클래스별 Precision/Recall을 먼저 계산하고 단순 평균 |
| **Micro** | 모든 클래스를 하나로 합쳐 (pooled matrix) Precision/Recall 계산 |

- **Macro** : 소수 클래스도 동등하게 반영 → 클래스 별 약점 발견 가능.
- **Micro** : 인스턴스 단위 평균 → 전체적인 성능.
- 비유 : "AI 센터 1명이 100점, 다른 그룹 0점이면 Macro=50점, Micro는 그룹별 인원수로 weighted."
- **둘 다 보여주는 것이 표준**.

## 12. Bias / Ethics / Model Card

### 12.1 NLP Bias 문제
- corpus의 편향이 모델에 그대로 흡수.
- 예: 자기소개서 분류 모델이 특정 인종의 이름을 부정 리뷰로 분류, 개발자 채용에서 성별 편향 등.
- 모델 → 데이터 → 모델 → 데이터 의 **악순환** 우려.

### 12.2 Model Card / Limitations
- 최근 데이터셋, 벤치마크, 논문에 **Limitations 섹션** 필수.
- 데이터가 어떻게 수집됐고 어떤 용도로 쓸 수 있는지 명시.
- Centerized AI를 통한 **censorship 위험** 언급.

## 13. 한 페이지 요약 (핵심 정리)

### 13.1 N-gram 언어모델 핵심
1. **Markov Assumption** : 직전 N-1 단어만 컨텍스트로.
2. **MLE 추정** : 조건부 카운트를 분모 카운트로 나눔.
3. **Sparsity** : Bigram만 해도 99.96%가 0.
4. **Underflow** → **Log Probability** 로 해결, 곱셈 → 덧셈.
5. **0 카운트 → Laplace (Add-1) Smoothing** : 분자 +1, 분모 +V.
6. **Add-K** : 1 대신 K로 일반화.

### 13.2 Spell Correction (Noisy Channel)
1. 관찰 noisy word x, 후보군 = Edit_1(x) ∩ corpus vocab.
2. **Bayes로 뒤집기** : argmax_w P(x|w) · P(w).
3. **LM(P(w))이 좋아질수록 spell correction 성능↑**.

### 13.3 Naive Bayes Classification
1. **BoW + Conditional Independence** 가정.
2. P(c|d) ∝ P(c) · ∏ P(w_i|c).
3. 추정 : prior = N_c/N, likelihood = (count+1)/(total+V).
4. **장점** : 텍스트 외 임의 확률 피처도 자연스럽게 곱해 합칠 수 있음.
5. **개선** : Stop words, Binarized count (diminishing returns), Negation handling.

### 13.4 🔥 Evaluation 시험 포인트
- **Accuracy ≠ 모든 것** : Class imbalance에서 위험.
- **Precision** = TP/(TP+FP), **Recall** = TP/(TP+FN).
- **F1 = harmonic mean** → 약점 클래스에 더 끌림.
- **Macro vs Micro** : 클래스별 평균 vs 전체 인스턴스 평균.
- **k-fold Cross-Validation** 으로 overfitting 방지.

### 13.5 강의 중 주요 직관/비유
- **사골국** : LM은 corpus를 끓여 만든 다용도 사골국 → 모든 NLP task에 활용.
- **꼰대** : Overfitting된 사람. 본 적 없는 일은 일어날 수 없다고 생각. Smoothing은 "benefit of the doubt".
- **금 100돈** : 1등에게만 줄 금이라면, log 변환·정규화 항 제거 등 **랭크 보존**만 하면 됨.
- **종이봉투(Bag of Words)** : 단어들을 봉투에 담아 흔들면 순서가 사라짐.
- **공장과 사탕(Bayesian)** : 공장이 P 비율로 A맛 사탕을 만든다 — MLE/MAP/Bayesian 차이의 직관.
- **햄버거 4개** : Diminishing returns — 같은 단어의 반복은 가치가 점점 줄어듦.
- **MBTI = 일종의 representation** : 4비트 벡터로 사람을 표현, 비슷한 사람들은 같은 벡터.
- **Software 1.0 / 2.0 / 3.0** (Karpathy) : 룰 → 학습 → LLM 프롬프팅의 패러다임 진화.
- **DeliciousPie 트윗** : Class imbalance에서 accuracy가 의미 없음을 보여주는 사례.
